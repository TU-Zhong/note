我的整体思路是：**分组 -> 计算 -> 合并**。

### 第一步：准备工作：创建计算所需的辅助列

在进行分组计算之前，如果能先把一些中间步骤准备好，会让后续的聚合操作变得非常清晰简单。

#### 1. 计算 `quality` 的基础

`quality` 的定义是 “评分与其位置之间比率的平均值”，也就是对每个 `query_name` 计算 `mean(rating / position)`。 所以，我们可以先直接在原始 DataFrame 中创建一个新列，存放每个查询的 `rating / position` 的比率值。

Python

```
# 新增一列，存放每个查询的 "评分/位置"
queries['quality_metric'] = queries['rating'] / queries['position']
```

#### 2. 计算 `poor_query_percentage` 的基础

`poor_query_percentage` 的定义是 “评分小于 3 的查询占该类别总查询数的百分比”。 我们可以利用 `pandas` 中布尔值（`True`/`False`）可以被当作 `1`/`0` 来计算的特性。

首先，我们可以创建一个布尔列，判断每个查询是否为“劣质查询”（`rating < 3`）。

Python

```
# 新增一列，如果 rating < 3 则为 True (1)，否则为 False (0)
queries['is_poor'] = queries['rating'] < 3
```

这样做有一个极大的好处：直接对这个布尔列求平均值 (`mean`)，得到的就是劣质查询所占的**比例**。例如，对于 `Dog`，这一列会是 `[False, False, True]`，在计算中等同于 `[0, 0, 1]`，求平均就是 `(0+0+1)/3 = 0.333...`，这正是我们想要的分数，最后再乘以 100 即可。

---

### 第二步：核心操作：分组与聚合 (Group and Aggregate)

现在我们有了 `quality_metric` 和 `is_poor` 这两个辅助列，就可以进行分组和聚合了。

1. **分组 (Group by)**：使用 `.groupby('query_name')` 将数据按照查询名称（`Dog`, `Cat`）分成不同的组。
    
2. **聚合 (Aggregate)**：对每个组，我们执行我们需要的计算。`.agg()` 方法是完成此项任务最清晰的方式，它可以让我们同时为不同的新列指定不同的计算方法。
    

Python

```
# 按 query_name 分组，并对每个组进行聚合计算
grouped_stats = queries.groupby('query_name').agg(
    # 新列 quality：对我们创建的 quality_metric 列求平均值
    quality=('quality_metric', 'mean'),

    # 新列 poor_query_percentage：对我们创建的 is_poor 列求平均值
    # 这里得到的是比例，例如 0.3333
    poor_query_percentage=('is_poor', 'mean')
)
```

执行完这一步，我们会得到一个以 `query_name` 为索引（Index）的 DataFrame，里面包含了 `quality` 和劣质查询的**比例**。

---

### 第三步：收尾工作：格式化结果

根据题目要求，我们还需要对结果进行一些处理。

1. **计算百分比**：将 `poor_query_percentage` 列的比例值（如 `0.3333`）乘以 `100`，得到百分比（`33.33`）。
    
2. **四舍五入**：使用 `.round(2)` 方法将 `quality` 和 `poor_query_percentage` 两列都四舍五入到小数点后两位。
    
3. **恢复索引**：`groupby` 操作会把分组的列（`query_name`）变成结果的索引。题目要求的输出格式中 `query_name` 是一个普通的列，所以我们需要使用 `.reset_index()` 将其从索引恢复为数据列。
    

---

### 第四步：整合代码

将以上所有思路整合到你提供的函数中，就得到了最终的解决方案。



```python
import pandas as pd

def queries_stats(queries: pd.DataFrame) -> pd.DataFrame:
    # --- 思路第一步：创建辅助列 ---
    # quality 的计算基础
    queries['quality_metric'] = queries['rating'] / queries['position']
    # poor_query_percentage 的计算基础 (True=1, False=0)
    queries['is_poor'] = queries['rating'] < 3

    # --- 思路第二步：分组与聚合 ---
    result_df = queries.groupby('query_name').agg(
        # 计算 quality: 对 quality_metric 列求平均
        quality=('quality_metric', 'mean'),
        # 计算劣质查询的比例: 对 is_poor 列求平均
        poor_query_percentage=('is_poor', 'mean')
    )

    # --- 思路第三步：格式化结果 ---
    # 1. 将比例转换为百分比
    result_df['poor_query_percentage'] = result_df['poor_query_percentage'] * 100

    # 2. 对两列进行四舍五入到小数点后两位
    result_df = result_df.round(2)

    # 3. 将 query_name 从索引恢复为普通列
    result_df = result_df.reset_index()

    return result_df
```

这个流程清晰地将一个复杂的问题分解成了几个简单、独立的步骤，充分利用了 `pandas` 的向量化操作和 `groupby` 功能，非常高效。
