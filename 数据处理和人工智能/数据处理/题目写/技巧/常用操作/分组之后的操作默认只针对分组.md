对，`.groupby('col')` 之后调用的大多数**聚合类、转换类、过滤类函数**，都是**按分组**来处理的，不是对整个 DataFrame 一起算。

我帮你按类型拆一下，你就能知道哪些是分组运算，哪些不是。

---

## **📌 1. 聚合类（Aggregation）**

**每组 → 1 个结果**

- 常用方法：
    

python

复制编辑

`sum()       # 每组求和 mean()      # 每组均值 median()    # 每组中位数 count()     # 每组非空值数量 nunique()   # 每组唯一值数量 min() / max() agg()       # 多指标聚合`

**特点**：

- 返回的行数 = 分组数量
    
- 用于生成组的汇总结果
    

---

## **📌 2. 转换类（Transformation）**

**每组 → 保持原行数**（结果会“广播”回每行）

- 常用方法：
    

python

复制编辑

`transform('sum')       # 每组求和，并赋给每行 transform(lambda x: x - x.mean())  # 每行减去组均值 rank()                  # 每组内排名 cumcount()              # 每组内累计计数 cumsum() / cumprod()    # 每组内累计和 / 累积乘积`

**特点**：

- 返回行数和原数据一样
    
- 适合加回原 DataFrame 作为新列
    

---

## **📌 3. 过滤类（Filter）**

**按分组条件保留或删除整组**

- 常用方法：
    

python

复制编辑

`filter(lambda x: x['value'].mean() > 100)  # 保留均值大于 100 的组`

**特点**：

- 整个组要么保留，要么去掉
    

---

## **📌 4. 应用类（Apply）**

**最灵活**，可以对每组执行自定义操作

- 常用方法：
    

python

复制编辑

`apply(lambda g: g.sort_values('value').head(2))  # 每组排序取前 2`

**特点**：

- 既可以返回每组一个值，也可以返回多行
    
- 最灵活，但可能比内置方法慢
    

---

## **📌 注意**

- 只要是 `.groupby()` 之后的计算，**默认都是分组内部单独计算**，互不影响。
    
- 除非你 `.reset_index()` 或 `.apply()` 后自己合并，否则分组的边界不会跨组运算。
