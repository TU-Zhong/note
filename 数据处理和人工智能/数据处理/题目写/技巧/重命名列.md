你好，你提供的代码不行，这是一个非常常见的疑惑！问题主要出在两个方面：

1. **变量 `b` 未定义**：在执行 `b[['day','active_users']] = ...` 这行代码时，变量 `b` 还不存在，所以 Python 不知道要把结果赋值给谁。
    
2. **赋值的结构不匹配**：`activity.groupby(...).nunique()` 这部分代码的执行结果是一个 Pandas **Series**，而不是一个有两列的 DataFrame。你不能把一个 Series 直接赋值给一个 DataFrame 的两列。
    

我们来看一下 `activity.groupby(['activity_date'])['user_id'].nunique()` 的输出结果长什么样：

```
activity_date
2019-07-20    2
2019-07-21    2
Name: user_id, dtype: int64
```

它只有一个数据列（值是 `2`, `2`），但是它的**索引 (Index)** 是日期。我们的目标是把这个索引也变成一个普通的列。

---

### 正确的解决方法

有几种标准且高效的方法可以实现你的目标。

#### 方法一：使用 `.reset_index()` (最常用)

这是最直接、最流行的方法。`.reset_index()` 可以把 Series 的索引转换成一个新的列。

Python

```
# 1. 先进行 groupby 聚合，得到一个 Series
result_series = activity.groupby(['activity_date'])['user_id'].nunique()

# 2. 使用 .reset_index() 将其转换为 DataFrame
#    此时，列名会是 'activity_date' 和 'user_id'
b = result_series.reset_index()

# 3. 最后，重命名列为你想要的名字
b.columns = ['day', 'active_users']

print(b)
```

**更简洁的链式写法：**

你也可以把这些步骤连在一起写，这是 Pandas 中非常常见的写法：

Python

```
b = activity.groupby('activity_date')['user_id'].nunique().reset_index()
b.columns = ['day', 'active_users']
```

#### 方法二：使用 `.reset_index()` 并立即重命名

这种方法与方法一类似，但使用 `.rename()` 函数，代码意图更清晰。

Python

```
b = activity.groupby('activity_date')['user_id'].nunique().reset_index()

# 使用 rename 方法来重命名列
b = b.rename(columns={'activity_date': 'day', 'user_id': 'active_users'})

print(b)
```

#### 方法三：使用 `.agg()` 方法 (功能更强大)

当你需要进行更复杂的聚合操作时，`.agg()` 是一个非常好的选择。它可以让你在聚合时直接命名新的列。

Python

```
b = activity.groupby('activity_date').agg(
    # 新列名 = (原始列名, 聚合函数)
    active_users=('user_id', 'nunique')
).reset_index()  # reset_index 仍然需要，用来把 activity_date 从索引变为列

# 只需要重命名日期列即可
b = b.rename(columns={'activity_date': 'day'})

print(b)
```

### 总结

对于你的需求，**方法一的链式写法是目前最简洁、最常用的**：

Python

```
# 推荐使用这种方式
b = activity.groupby('activity_date')['user_id'].nunique().reset_index()
b.columns = ['day', 'active_users']
```

记住这个模式：**`groupby()` -> 聚合函数 -> `reset_index()` -> 重命名列**，这是在 Pandas 中进行数据聚合和整理时非常有用的一个流程。
