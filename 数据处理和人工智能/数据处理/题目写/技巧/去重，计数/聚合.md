不是的，这是一个非常好的问题，因为它触及了 `groupby` 操作的核心概念：**聚合 (Aggregation)**。

`nunique()` **不会**对分组内的每一行数据都进行赋值。它是一个**聚合函数**，作用是：

1. **分组 (Group)：** `activity.groupby(['activity_date'])` 会首先根据 `activity_date` 列中的值将整个 DataFrame 分成若干个组。所有 `activity_date` 相同的行会进入同一个组。
    
    - 例如，所有日期为 `2019-07-20` 的行会形成一个组。
        
    - 所有日期为 `2019-07-21` 的行会形成另一个组。
        
2. **选择列 (Select)：** `['user_id']` 指示在每个组内，我们只关注 `user_id` 这一列的数据。
    
3. **聚合 (Aggregate)：** `.nunique()` 会对**每个组内**的所有 `user_id` 值进行计算，得出其中**不重复的值的数量**，并为**每个组返回一个最终的、单一的结果**。
    

---

### 举例说明

让我们用您之前的示例数据来分解这个过程：

**原始数据 (部分):**

```
| user_id | activity_date |
|---------|---------------|
| 1       | 2019-07-20    |
| 1       | 2019-07-20    |
| 1       | 2019-07-20    |
| 2       | 2019-07-20    |  <-- '2019-07-20' 组结束
| 2       | 2019-07-21    |
| 2       | 2019-07-21    |
| 3       | 2019-07-21    |
| 3       | 2019-07-21    |
| 3       | 2019-07-21    |  <-- '2019-07-21' 组结束
```

**`b = activity.groupby(['activity_date'])['user_id'].nunique()` 的执行过程:**

1. **处理 `'2019-07-20'` 组:**
    
    - 该组内所有的 `user_id` 是: `[1, 1, 1, 2]`。
        
    - 对这个列表 `[1, 1, 1, 2]` 应用 `nunique()`。
        
    - 不重复的值是 `{1, 2}`。
        
    - 不重复值的数量是 **2**。
        
    - 所以，对于 `'2019-07-20'` 这个组，最终结果就是 `2`。
        
2. **处理 `'2019-07-21'` 组:**
    
    - 该组内所有的 `user_id` 是: `[2, 2, 3, 3, 3]`。
        
    - 对这个列表 `[2, 2, 3, 3, 3]` 应用 `nunique()`。
        
    - 不重复的值是 `{2, 3}`。
        
    - 不重复值的数量是 **2**。
        
    - 所以，对于 `'2019-07-21'` 这个组，最终结果就是 `2`。
        

**最终结果 `b`：**

执行完这行代码后，变量 `b` 会是一个新的 Pandas `Series`，其**索引**是分组的依据（`activity_date`），**值**是每个组聚合后的结果（唯一用户数）。它看起来是这样的：

```
activity_date
2019-07-20    2
2019-07-21    2
Name: user_id, dtype: int64
```

### 与您设想的操作对比

如果您想要实现“对分组内每行的数据都进行赋值”（例如，在原始的 DataFrame 中新增一列，显示该行所在日期的总活跃用户数），您需要使用的是 `transform` 函数，而不是 `nunique`。

例如：

Python

```
# 这会返回一个与原始DataFrame行数相同的Series
# 其中每个值是其所在组的nunique计算结果
activity['daily_active_count'] = activity.groupby('activity_date')['user_id'].transform('nunique')
```

执行后，`activity` 会变成：

```
| user_id | activity_date | daily_active_count |
|---------|---------------|--------------------|
| 1       | 2019-07-20    | 2                  |
| 1       | 2019-07-20    | 2                  |
| 2       | 2019-07-20    | 2                  |
| 2       | 2019-07-21    | 2                  |
| 3       | 2019-07-21    | 2                  |
...       ...             ...
```

在这里，`transform('nunique')` 才实现了为组内每一行赋予相同值的效果。

**总结：** `groupby(...).nunique()` 是一个**聚合**操作，它将一个组的**多行数据**缩减为**一个单一的输出值**。
