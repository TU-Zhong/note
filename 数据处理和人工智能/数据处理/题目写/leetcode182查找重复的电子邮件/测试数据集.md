为了方便您实验和验证查找重复电子邮件的函数，我为您准备了一份覆盖多种情况的测试数据集。您可以直接在 Python 中使用这段代码来创建一个 Pandas DataFrame。

这份数据集包含以下几种情况：

- **没有重复**的电子邮件。
    
- **简单重复**：一个 email 出现两次。
    
- **多次重复**：一个 email 出现三次。
    
- **多种重复**：有两个或以上不同的 email 地址是重复的。
    

---

### 用于测试的 Python/Pandas 脚本

您可以直接复制并运行下面的完整代码。它包括了：

1. 您之前编写的 `duplicate_emails` 函数。
    
2. 创建测试数据集 `person_df`。
    
3. 调用函数并打印结果。
    
4. 给出预期的正确结果，方便您比对。
    

Python

```
import pandas as pd

# 1. 您要测试的函数
def duplicate_emails(person: pd.DataFrame) -> pd.DataFrame:
    """
    报告所有重复的电子邮件。
    """
    # 使用 value_counts() 计算每个 email 出现的次数
    email_counts = person['email'].value_counts()
    
    # 筛选出出现次数大于 1 的 email
    duplicate_emails_index = email_counts[email_counts > 1].index
    
    # 将结果转换为要求的 DataFrame 格式
    return pd.DataFrame({'Email': duplicate_emails_index})


# 2. 创建一个包含多种情况的测试数据集
data = {
    'id': [1, 2, 3, 4, 5, 6, 7, 8, 9],
    'email': [
        'user1@example.com',      # 唯一出现
        'user2@test.com',         # 将会重复 (出现2次)
        'admin@corp.com',         # 将会重复 (出现3次)
        'user3@example.com',      # 唯一出现
        'user2@test.com',         # <-- user2 的重复项
        'support@corp.com',       # 唯一出现
        'admin@corp.com',         # <-- admin 的重复项
        'user4@test.com',         # 唯一出现
        'admin@corp.com'          # <-- admin 的第三次出现
    ]
}

person_df = pd.DataFrame(data)

print("--- 输入的测试数据 ---")
print(person_df)
print("-" * 40)


# 3. 调用您的函数进行测试
actual_output = duplicate_emails(person_df)

print("--- 函数的实际输出 ---")
# .reset_index(drop=True) 是为了让输出的索引从0开始，更整洁
print(actual_output.reset_index(drop=True)) 
print("-" * 40)


# 4. 预期得到的正确结果
# 根据上面的数据，admin@corp.com 出现了3次，user2@test.com 出现了2次
# 这两个是重复的 email
print("--- 预期的正确输出 ---")
expected_output_data = {'Email': ['admin@corp.com', 'user2@test.com']}
expected_df = pd.DataFrame(expected_output_data)
print(expected_df)
print("-" * 40)

# 提示: value_counts() 默认按出现次数降序排序，所以 'admin@corp.com' (3次) 会在 'user2@test.com' (2次) 前面。
# 您的函数输出顺序如果与预期一致，说明逻辑完全正确。
```

### 如何使用

1. 将上述代码保存为一个 Python 文件（例如 `test_duplicates.py`）。
    
2. 在您的终端中运行该文件：`python test_duplicates.py`。
    
3. 比较“函数的实际输出”和“预期的正确输出”是否完全一致。如果一致，就说明您的函数在这些测试场景下是正确的。
    

这个数据集能有效地帮助您验证代码逻辑的健壮性。
