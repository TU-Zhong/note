好的，我们来聊聊 Pandas 中另一个非常强大的功能：**数据变形（Data Reshaping）**。

数据变形是指在不改变数据内容的前提下，改变数据的组织形式（即行和列的结构），以适应不同的分析需求。最核心的概念是在 **“长格式 (Long Format)”** 和 **“宽格式 (Wide Format)”** 数据之间进行转换。

- **长格式 (Long/Tidy Format)**：也叫整洁格式。每一行是一个观测（observation），每一列是一个变量（variable）。这是大多数数据分析和可视化的首选格式。
    
- **宽格式 (Wide Format)**：将某些变量的值作为新的列名，使得数据更紧凑，便于人类阅读和比较。
    

我们先创建一个示例 `DataFrame`，它是一个典型的“长格式”数据：

Python

```
import pandas as pd

df_long = pd.DataFrame({
    '日期': ['2025-08-01', '2025-08-01', '2025-08-02', '2025-08-02'],
    '城市': ['北京', '上海', '北京', '上海'],
    '指标': ['温度', '湿度', '温度', '湿度'],
    '值': [32, 60, 30, 65]
})

print("长格式 (Long Format) 数据:")
print(df_long)
```

输出：

```
长格式 (Long Format) 数据:
           日期  城市  指标   值
0  2025-08-01  北京  温度  32
1  2025-08-01  上海  湿度  60
2  2025-08-02  北京  温度  30
3  2025-08-02  上海  湿度  65
```

这种格式很规范，但如果我们想一眼就看出“北京在8月1日的温度和湿度”，就不那么直观了。这时就需要将其变为“宽格式”。

---

### 1. 从长格式到宽格式：`pivot()` 和 `pivot_table()`

这是数据变形最核心的操作，也叫“数据透视”。

#### `pivot()` - 简单透视

`pivot()` 是一个基础的透视方法。你需要指定哪个列作为新的行索引（`index`），哪个列的值作为新的列名（`columns`），以及哪个列作为填充的值（`values`）。

**重要前提**：使用 `pivot()` 时，`index` 和 `columns` 的组合必须是唯一的，否则会报错。

Python

```
# pivot() 在我们的例子中会报错，因为 ('2025-08-01', '北京') 这样的组合并不唯一
# df_long.pivot(index='日期', columns='城市', values='值') # 这行会报错

# 我们需要一个 index 和 columns 组合唯一的例子
df_simple = pd.DataFrame({
    '日期': ['2025-08-01', '2025-08-01', '2025-08-02'],
    '城市': ['北京', '上海', '北京'],
    '值': [32, 28, 30]
})
df_wide_simple = df_simple.pivot(index='日期', columns='城市', values='值')
print("使用 pivot() 进行简单透视:")
print(df_wide_simple)
```

输出：

```
使用 pivot() 进行简单透视:
城市            北京    上海
日期
2025-08-01  32.0  28.0
2025-08-02  30.0   NaN
```

#### `pivot_table()` - 功能更强大的透视表

`pivot_table()` 是 `pivot()` 的增强版，也是实际应用中最常用的工具。它解决了 `pivot()` 的最大痛点：**当 `index` 和 `columns` 组合不唯一时，可以通过聚合函数 (`aggfunc`) 来处理重复值**。

**关键参数**：

- `index`: 新 DataFrame 的行索引。
    
- `columns`: 新 DataFrame 的列。
    
- `values`: 要聚合的数值。
    
- `aggfunc`: 聚合函数，用于处理重复值。默认为 `'mean'` (求平均值)。可以是 `'sum'`, `'count'`, `'max'`, `'min'` 等，也可以是自定义函数。
    

让我们回到最初的 `df_long` 例子：

Python

```
# 创建一个包含重复项的例子
df_sales = pd.DataFrame({
    '日期': ['2025-08-01', '2025-08-01', '2025-08-02', '2025-08-02', '2025-08-01'],
    '城市': ['北京', '上海', '北京', '上海', '北京'],
    '销售额': [100, 150, 120, 180, 50]
})

# 我们想看每个城市每天的总销售额
# index='日期', columns='城市' 的组合 ('2025-08-01', '北京') 出现了两次
df_pivot = df_sales.pivot_table(index='日期', columns='城市', values='销售额', aggfunc='sum')
print("\n使用 pivot_table() 进行透视 (aggfunc='sum'):")
print(df_pivot)
```

输出：

```
使用 pivot_table() 进行透视 (aggfunc='sum'):
城市         北京   上海
日期
2025-08-01  150  150
2025-08-02  120  180
```

这里，`pivot_table` 自动将北京在8月1日的两笔销售额（100和50）相加得到了150。

### 2. 从宽格式到长格式：`melt()`

`melt()` 操作与 `pivot` 相反，它将宽格式的数据“融化”为长格式，非常适合于数据清理和规范化。

**关键参数**：

- `id_vars`: 标识符变量，这些列在转换过程中将保持不变。
    
- `value_vars`: 需要被“融化”的列。如果未指定，则除 `id_vars` 之外的所有列都将被融化。
    
- `var_name`: 用于存放“融化”的列名的新列的名称，默认为 `'variable'`。
    
- `value_name`: 用于存放“融化”的列对应的值的新列的名称，默认为 `'value'`。
    

让我们把刚才创建的宽格式 `df_pivot` 再转换回长格式：

Python

```
# df_pivot.reset_index() 是为了将行索引'日期'变回普通列
df_melted = df_pivot.reset_index().melt(
    id_vars=['日期'],
    value_vars=['北京', '上海'],
    var_name='城市',
    value_name='销售额'
)
print("\n使用 melt() 将宽格式转回长格式:")
print(df_melted)
```

输出：

```
使用 melt() 将宽格式转回长格式:
           日期  城市  销售额
0  2025-08-01  北京  150
1  2025-08-02  北京  120
2  2025-08-01  上海  150
3  2025-08-02  上海  180
```

### 3. 基于多级索引的变形：`stack()` 和 `unstack()`

`stack()` 和 `unstack()` 是处理具有**多级索引 (MultiIndex)** 的 DataFrame 的变形工具。

- **`stack()` (堆叠)**: 将**列**“堆叠”到**行索引**中，使数据变得更长。
    
- **`unstack()` (拆堆)**: `stack()` 的逆操作，将**行索引**的某个级别“拆分”到**列**中，使数据变得更宽。
    

我们先创建一个带有多级索引的 `DataFrame`（通常由 `groupby` 或 `pivot_table` 产生）。

Python

```
# 使用上面的 df_pivot，它本身就是一个多级索引的理想例子
print("\n原始透视表 (列是单级索引):")
print(df_pivot)

# stack() 将列索引 '城市' 堆叠到行索引中
df_stacked = df_pivot.stack()
print("\n使用 stack() 之后 (变成一个 Series):")
print(df_stacked)

# unstack() 可以将 stack() 的结果还原
df_unstacked = df_stacked.unstack()
print("\n使用 unstack() 之后 (还原):")
print(df_unstacked)

# unstack() 也可以操作不同的索引级别
# 假设我们想把 '日期' 拆分到列上
df_unstacked_level0 = df_stacked.unstack(level=0)
print("\n使用 unstack(level=0) 之后:")
print(df_unstacked_level0)
```

输出：

```
原始透视表 (列是单级索引):
城市         北京   上海
日期
2025-08-01  150  150
2025-08-02  120  180

使用 stack() 之后 (变成一个 Series):
日期          城市
2025-08-01  北京    150
            上海    150
2025-08-02  北京    120
            上海    180
dtype: int64

使用 unstack() 之后 (还原):
城市         北京   上海
日期
2025-08-01  150  150
2025-08-02  120  180

使用 unstack(level=0) 之后:
日期  2025-08-01  2025-08-02
城市
北京         150         120
上海         150         180
```

### 总结：如何选择变形方法

|方法|作用|起始格式|目标格式|核心特点|
|---|---|---|---|---|
|**`pivot_table()`**|聚合与透视|长格式|宽格式|功能最强，通过`aggfunc`处理重复值，是数据汇总分析的核心。|
|**`melt()`**|融化/逆透视|宽格式|长格式|将宽表转换为标准的整洁格式，便于后续分析和可视化。|
|**`stack()`**|堆叠|宽/多级索引|更长/Series|将列索引压入行索引，创建多级行索引。|
|**`unstack()`**|拆堆|长/多级索引|更宽/DataFrame|将行索引的一个级别提升为列索引，是`stack`的逆操作。|
