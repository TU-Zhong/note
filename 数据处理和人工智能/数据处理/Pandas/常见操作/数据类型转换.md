好的，我们来详细讲解 Pandas 中一个非常重要且频繁的操作：**数据类型转换**。

在数据处理过程中，数据源中的类型不一定是我们所需要的。例如，数字被存成了字符串，日期被存成了文本。正确的数据类型是进行有效计算、分析和建模的前提。

### 为什么需要进行数据类型转换？

1. **节省内存**：将宽泛的数据类型（如 `object`）转换为更精确的类型（如 `int`, `float`, `category`）可以显著减少内存占用。
    
2. **开启特定功能**：
    
    - **数值类型 (`int`, `float`)**：是进行数学运算（加减乘除、求和、求平均等）的基础。
        
    - **日期时间类型 (`datetime64`)**：可以方便地提取年份、月份、星期，或进行时间差计算。
        
    - **类别类型 (`category`)**：当一个列中只有少数几个固定值时（如“性别”、“星期”），`category` 类型不仅能节省内存，还能提升某些运算（如 `groupby`）的速度。
        
3. **避免错误**：用错误的类型进行操作会引发 `TypeError` 或得到不符合预期的结果。例如，对字符串 `'5' + '3'` 的结果是 `'53'`，而不是 `8`。
    

### Pandas 中的主要数据类型转换方法

我们先创建一个示例 `DataFrame` 用于演示：

Python

```
import pandas as pd

df = pd.DataFrame({
    '订单ID': [1001, 1002, 1003, 1004],
    '销售额': ['59.9', '120.5', '88', '75.3'],
    '评分': [4, 5, 3, '4'], # 混合了数字和字符串
    '订单日期': ['2025-08-01', '2025-08-01', '2025-08-02', '2025-08-03'],
    '产品类别': ['A', 'B', 'A', 'C']
})

print("原始数据类型:")
print(df.dtypes)
```

输出：

```
原始数据类型:
订单ID       int64
销售额       object
评分         object
订单日期      object
产品类别      object
dtype: object
```

从上面可以看到，除了“订单ID”，其他列都被识别为通用的 `object` 类型，这显然不便于分析。

---

### 1. 通用转换方法：`astype()`

`astype()` 是最常用、最直接的类型转换方法。它可以将一个 Series 或 DataFrame 的所有列转换为指定的类型。

**语法**：`df['列名'].astype(新类型)` 或 `df.astype({'列名1': 新类型1, '列名2': 新类型2})`

#### a. 转换为数值类型 (int, float)

Python

```
# 将 '销售额' 从 object 转换为 float
df['销售额'] = df['销售额'].astype(float)

# 将 '评分' 从 object 转换为 int
# 注意：如果列中存在无法转换的值（如 'abc'），这步会直接报错
df['评分'] = df['评分'].astype(int)

print("\n转换为数值类型后:")
print(df.dtypes)
```

输出：

```
转换为数值类型后:
订单ID       int64
销售额      float64
评分         int64
订单日期      object
产品类别      object
dtype: object
```

#### b. 转换为字符串类型 (string)

虽然 `object` 类型通常用来存储字符串，但从 Pandas 1.0 开始，官方推荐使用专门的 `string` 类型，它提供了一些专门针对字符串的优化。

Python

```
df['产品类别'] = df['产品类别'].astype('string')
print("\n转换为string类型后:")
print(df.dtypes)
```

#### c. 转换为类别类型 (category)

当一列的唯一值数量有限时，`category` 类型是最佳选择。

Python

```
# '产品类别' 只有 A, B, C 三种值，非常适合转换为 category
df['产品类别'] = df['产品类别'].astype('category')

print("\n转换为category类型后:")
print(df.dtypes)
print(df['产品类别'])
```

输出：

```
转换为category类型后:
...
产品类别     category
...
0    A
1    B
2    A
3    C
Name: 产品类别, dtype: category
Categories (3, object): ['A', 'B', 'C']
```

可以看到，Pandas 识别出了3个类别。

### 2. 专用转换函数

对于某些特定类型的转换，Pandas 提供了功能更强大的专用函数，它们能更好地处理错误和复杂格式。

#### a. `pd.to_numeric()` - 转换为数值

`to_numeric()` 比 `astype(int)` 或 `astype(float)` 更灵活，尤其是在处理错误值时。

**主要参数 `errors`**:

- `errors='raise'` (默认): 如果遇到无法转换的值，直接引发错误。
    
- `errors='coerce'` (Coerce: 强制): 如果遇到无法转换的值，会将其替换为 `NaN` (Not a Number)。**这个功能非常有用！**
    
- `errors='ignore'`: 如果遇到无法转换的值，保持原样，不进行转换。
    

**示例**： 假设“评分”列有一个无效值 `'bad'`。

Python

```
df_err = pd.DataFrame({'评分': [4, 5, 'bad', 4]})

# 使用 astype 会报错
# df_err['评分'].astype(int) # --> ValueError

# 使用 to_numeric 并设置 errors='coerce'
df_err['评分_numeric'] = pd.to_numeric(df_err['评分'], errors='coerce')
print("\n使用 to_numeric 处理错误:")
print(df_err)
```

输出：

```
使用 to_numeric 处理错误:
    评分  评分_numeric
0    4         4.0
1    5         5.0
2  bad         NaN
3    4         4.0
```

可以看到，`'bad'` 被优雅地处理成了 `NaN`，之后你可以用 `fillna()` 来填充它。

#### b. `pd.to_datetime()` - 转换为日期时间

这是将字符串或数字转换为日期时间类型的标准方法。它能自动解析多种常见的日期格式。

**主要参数 `format`**: 如果你的日期格式很特殊，可以通过 `format` 参数指定格式代码，以提高解析速度和准确性。

Python

```
# Pandas 自动识别标准格式
df['订单日期'] = pd.to_datetime(df['订单日期'])

# 假设有非标准格式
date_series = pd.Series(['2025/08/01', '2025/08/02'])
# 指定格式
pd.to_datetime(date_series, format='%Y/%m/%d')

print("\n转换为datetime类型后:")
print(df.dtypes)
print(df['订单日期'].dt.day_name()) # 转换后就可以使用 .dt 访问器了
```

输出：

```
转换为datetime类型后:
订单ID               int64
销售额              float64
评分                 int64
订单日期     datetime64[ns]
产品类别           category
dtype: object

0      Friday
1      Friday
2    Saturday
3      Sunday
Name: 订单日期, dtype: object
```

转换后，就可以用 `.dt` 访问器轻松获取年份、月份、星期等信息。

#### c. `pd.to_timedelta()` - 转换为时间差

用于将字符串或数字转换为时间差（`Timedelta`）类型，常用于日期时间的加减运算。

Python

```
delta_series = pd.Series(['1 days', '2 days', '3 hours'])
pd.to_datetime(delta_series)
```

### 总结：如何选择转换方法

|方法|主要用途|优点|缺点|
|---|---|---|---|
|**`astype()`**|通用的、快速的类型转换。|语法简洁，可一次性转换多列。|错误处理能力弱，遇到无法转换的值通常直接报错。|
|**`pd.to_numeric()`**|专门将对象转换为数值类型。|强大的错误处理机制（`errors='coerce'`）。|只能用于数值转换。|
|**`pd.to_datetime()`**|专门将对象转换为日期时间类型。|能自动识别多种日期格式，可通过`format`参数处理复杂格式。|只能用于日期时间转换。|

Export to Sheets

**实践建议**：

1. **首选 `astype()`**：对于确定没有问题的、简单的类型转换（如 `int` -> `float`），使用 `astype()` 最方便。
    
2. **当数据可能“脏”时，用 `to_numeric()`**：当你从外部文件（如CSV）读取数据，不确定列中是否混有非数值字符串时，使用 `pd.to_numeric(errors='coerce')` 是最安全、最稳健的做法。
    
3. **处理日期时间，用 `to_datetime()`**：这是处理日期时间数据的标准做法，毫无疑问。
