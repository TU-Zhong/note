### 🔪 面试官的三连灵魂拷问：

**第一问：标签定义（别给我背基础标签）**

常规的“色情”、“暴恐”标签已经失效了。针对这种“隐晦变异”和“上下文诱导”的内容，你作为策略PM，要怎么在数美的系统里**重新定义和拆解这套风险标签体系**？

- **第一道防线（Input端 - 提示词审查）：** 拦截“明确的恶意意图”。像“娇喘”、“诱惑”这种指令词，在 Input 环节直接命中黑名单或意图识别模型，**阻断生成**。不仅保护了安全，还给公司省下了大笔大模型调用的 Token 算力成本。
    
- **第二道防线（Output端 - 文本流抢跑审核）：** 拦截“绕过指令的违规生成”或“AI 幻觉”。有些黑灰产很聪明，用极其干净的词语（比如“吟诗”）通过了第一关，但诱导大模型产出了带有色情隐喻的诗句。这时候，我们在转交 TTS 发声之前，通过底层文本流审核将其掐断。
**第二问：分级管控策略（直面实时性冲突）**

语音是**实时流**，要求极低的延迟。

- 如果直接拦截，容易误杀且体验极差；
    
- 如果进黑匣子人工复核，人工听完一段录音至少要十几秒，这十几秒里违规内容早就被用户听见并录音了。
    
    在这种极端苛刻的“既要安全、又要实时体验”的限制下，你的**分级管控策略（拦截 / 降级 / 延时 / 旁路阻断等）**具体画个决策树出来，每一步的触发条件是什么？
    

**第三问：指标对线与迭代（用数据说话）**

上线你的新策略后第一天，客户的运营总监拿着数据来砸桌子：“你们误杀率太高了！昨天有上千个VIP用户的正常通话被你们掐断了，我要立刻关掉你们的策略！”

你现在手头有数美的后台，你需要提取哪几个维度的**核心数据指标**，怎么排查这是不是真的“误杀”？如果确实误杀了，你怎么安抚他，并给出你的下一步策略迭代方案？
