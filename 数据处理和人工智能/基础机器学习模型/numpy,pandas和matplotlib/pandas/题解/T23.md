好的，向一个空的 DataFrame 逐行添加数据是一个常见的需求，尤其是在处理循环或流式数据时。

这里介绍几种方法，但**请务必阅读到最后**，因为这些方法在性能上有巨大差异，了解最佳实践非常重要。

首先，我们创建一个带有名确列的空 DataFrame。

Python

```
import pandas as pd

# 创建一个定义了列名的空 DataFrame
columns = ['姓名', '年龄', '城市']
df = pd.DataFrame(columns=columns)

print("初始的空 DataFrame:")
print(df)
```

### 方法一：使用 `pd.concat()` (官方推荐的追加方法)

`.append()` 方法在较新版本的 Pandas 中已被弃用，官方推荐使用 `pd.concat()`。你需要为新行创建一个小的 DataFrame，然后将它与原有的 DataFrame 连接起来。

Python

```
# 创建一个空的 DataFrame
df = pd.DataFrame(columns=['姓名', '年龄', '城市'])

# 准备要添加的数据
new_row_data = {'姓名': '张三', '年龄': 28, '城市': '北京'}
new_row_df = pd.DataFrame([new_row_data]) # 注意要将字典放在列表中

# 使用 concat 添加行
df = pd.concat([df, new_row_df], ignore_index=True)

print("\n添加一行后:")
print(df)

# 再添加一行
new_row_data_2 = {'姓名': '李四', '年龄': 35, '城市': '上海'}
df = pd.concat([df, pd.DataFrame([new_row_data_2])], ignore_index=True)

print("\n添加两行后:")
print(df)
```

**解释:**

- `pd.concat([...])`: 将一个 DataFrame 列表连接（concatenate）起来。
    
- `ignore_index=True`: 这是非常重要的参数。它会重新生成从 0 开始的连续索引，否则新行的索引会是 0，导致索引重复。
    

### 方法二：使用 `.loc[]` 赋值

你可以通过 `.loc[]` 直接为 DataFrame 的新索引位置赋值。

Python

```
# 创建一个空的 DataFrame
df = pd.DataFrame(columns=['姓名', '年龄', '城市'])

# 使用 .loc 添加第一行
df.loc[0] = ['王五', 22, '广州']

# 使用 .loc 添加第二行
df.loc[1] = ['赵六', 40, '深圳']

print("\n使用 .loc 添加数据后:")
print(df)
```

---

### **⚠️ 重要警告：性能问题**

上面两种方法在循环中逐行添加数据时，**性能会非常差**。

**原因**：==DataFrame 在内存中是基于不可变的 NumPy 数组构建的==。每次你添加一行，Pandas 并不是简单地在末尾“附加”数据，而是需要：

1. 在内存中创建一个全新的、更大的空间。
    
2. 将**旧 DataFrame 的全部数据**复制到新空间。
    
3. 将新行的数据添加到新空间。
    
4. 销毁旧的 DataFrame。
    

当你要添加成千上万行数据时，这种反复的内存分配和数据复制会导致程序运行得极其缓慢。

---

### ✅ 最佳实践：先收集到列表，再一次性创建 DataFrame

这是处理此类问题的**最高效、最推荐**的方法。思路是：

1. 将每一行的数据组织成字典或列表。
    
2. 将这些字典或列表追加到一个普通的 Python 列表中。
    
3. 在所有数据都收集完毕后，**调用一次 `pd.DataFrame()`** 来创建最终的 DataFrame。
    

这个方法避免了在循环中反复修改 DataFrame 的巨大开销。

**示例代码：**

Python

```
import pandas as pd

# 1. 准备一个 Python 列表来收集数据
data_list = []

# 假设我们在一个循环中生成数据
for i in range(5):
    # 模拟生成一行数据（这里使用字典，更具可读性）
    row_data = {
        '姓名': f'用户_{i}',
        '年龄': 20 + i,
        '城市': ['北京', '上海', '广州', '深圳', '杭州'][i]
    }
    # 2. 将这行数据（字典）追加到列表中
    data_list.append(row_data)
    
print("收集到的数据列表:")
print(data_list)

# 3. 循环结束后，一次性从列表创建 DataFrame
df_best_practice = pd.DataFrame(data_list)

print("\n通过最佳实践创建的 DataFrame:")
print(df_best_practice)
```

### 总结对比

|方法|优点|缺点 / 适用场景|
|---|---|---|
|`pd.concat`|官方推荐，功能强大|**在循环中性能极差**。适合偶尔合并少量行或整个 DataFrame。|
|`.loc`|语法直观|**在循环中性能同样极差**。只适合添加或修改极少数特定的行。|
|**先收集再创建**|**性能最优，内存高效**|**强烈推荐**。几乎是所有需要逐行构建 DataFrame 场景下的标准解决方案。|

结论：

除非你只是想手动添加一两行数据，否则请始终使用“先将数据收集到 Python 列表，最后一次性创建 DataFrame”的方法。这会使你的代码运行速度快上几个数量级。
