### 数据集说明

- **`订单ID`**: 每次交易的唯一标识。
    
- **`客户ID`**: 客户的唯一标识，用于将订单聚合到每个客户。
    
- **`下单日期`**: 客户下订单的日期，这是所有时间分析的基础。
    
- **`订单金额`**: 该笔订单的金额。
    
- **`产品类别`**: 客户购买的产品类别，可用于进一步的客户分群。
    

这份数据中包含了不同行为的客户，例如：

- **C1001**: **高价值忠诚客户**，购买频率高，金额稳定且持续。
    
- **C1002**: **高潜力新客户**，近期才成为客户，但购买频率和金额都在快速增长。
    
- **C1003**: **流失风险客户**，早期购买很活跃，但后期购买频率和金额都在下降，且最近没有购买。
    
- **C1015**: **低价值稳定客户**，购买频率可以，但单次金额一直不高。
    
- **C1050**: **一次性客户**，只有一次购买记录。

### 您可以完成的分析操作

有了这份数据集，您就可以开始进行您提到的所有分析了。我们假设今天的日期是 **`2025-10-01`** 来进行分析。

1. **最近一次购买时间 (Recency)**
    
	- **操作**: 按 `客户ID` 分组，找出每个客户的 `下单日期` 的最大值，然后用 `2025-10-01` 减去该日期。
        
    - **目的**: 衡量客户的活跃度，R值越小，客户越活跃。

==要找到一个基准日期来进行比较==
```python
test1['下单日期']=pd.to_datetime(test1['下单日期'])

d=pd.to_datetime('2025-10-01')

test1['LatestPurchase']=test1.groupby('客户ID')['下单日期'].transform(lambda x:d-x.max())
# 转化成天
test1['LatestPurchase']=test1['LatestPurchase'].dt.days
```
1. **成为客户的时间 (客户服务期/Tenure)**
    
    - **操作**: 按 `客户ID` 分组，找出每个客户的 `下单日期` 的最小值，然后用 `2025-10-01` 减去该日期。
        
    - **目的**: 了解客户关系的持续时间，是新客户还是老客户。
```python
test1['Tenure']=test1.groupby('客户ID')['下单日期'].transform(lambda x:d-x.min())

test1['Tenure']=test1['Tenure'].dt.days
```
1. **购买频率 (Frequency)**
    
    - **操作**: 按 `客户ID` 分组，计算每个客户的 `订单ID` 的数量。
        
    - **目的**: 衡量客户的忠诚度，F值越高，忠诚度越高。
    - ==这两个写法都可以，后者是计算非空值的行数，前者是直接计算有多少行==
```python
#找到客户下单频率
test1['freq']=test1.groupby('客户ID').size()
test1['freq']=test1.groupby('客户ID')['订单ID'].count()
```
1. **平均购买间隔 (Average Purchase Interval)**
    
    - **操作**: 对于每个客户，计算其相邻两次 `下单日期` 的时间差，然后求平均值。
        
    - **目的**: 了解客户的购买周期。
    - 不能说使用当前日期减去注册日期再除以频率-1，因为这样无法处理只购买了一次的用户。==使用先排序再分组求diff最后求均值的方法更好。==
```python
#找到客户下单的平均时间间隔
test1_sorted=test1.sort_values(by=['客户ID','下单日期'])
# test1_sorted
test1_sorted['diff']=test1_sorted.groupby('客户ID')['下单日期'].diff()
test1_sorted[test1_sorted['diff']<pd.Timedelta(0)]
test1['时间间隔']=test1_sorted.groupby('客户ID')['diff'].transform('mean')
# test1=test1.drop(columns='平均时间间隔')
# test1.sort_values(by='客户ID')
test1['时间间隔']=test1['时间间隔'].dt.days
test1.loc[test1['时间间隔'].isnull(),'时间间隔']=test1['LatestPurchase']
# test1['时间间隔'].isnull().sum()
# test1['平均时间间隔']=test1_sorted.groupby('客户ID')['diff'].transform('mean')
# test1
```
1. **增长率 (Growth Rate)**
    
    - **操作**: 对于每个客户，可以比较不同时间段的购买总金额。例如，计算每个客户在2024年和2025年的购买总额，然后计算增长率 `(2025年金额 - 2024年金额) / 2024年金额`。
        
    - **目的**: 识别正在成长的客户和正在衰退的客户。
    - [[除零操作]]            ==这个很重要==
    - 分开再进行合并时有两种方法，==一种不reset_index使用Series的map()==     ,    ==一种直接merge==
	    然后要有所有的客户名单，`all_customers=test1['客户ID'].unique()     m=m.reindex(all_customers)`
		==这行代码 `m = m.reindex(all_customers)` 的核心作用是：**根据一个全新的索引 (`all_customers`) 来重塑或“校准”一个现有的 DataFrame (`m`)。**==
		==简单来说，就是**强制 `m` 的行索引变得和 `all_customers` 列表一模一样**。==

```python
# 增长率
start=pd.to_datetime('2025-04-01')
recent=pd.to_datetime('2025-07-01')
now=pd.to_datetime('2025-10-01')
pre=test1[(test1['下单日期']<recent)&(test1['下单日期']>start)].groupby('客户ID')['订单金额'].sum()
next=test1[test1['下单日期']>recent].groupby('客户ID')['订单金额'].sum()
m=pd.DataFrame({
    'p':pre,
    'n':next
})
all_customers=test1['客户ID'].unique()
m=m.reindex(all_customers)
m=m.fillna(0)
m['growth_rate']=(m['n']-m['p'])/m['p']
m.replace([np.inf,-np.inf],np.nan,inplace=True)
# m.reset_index(inplace=True)
# test1=pd.merge(test1,m,on='客户ID',how='left')
test1['growth_rate']=test1['客户ID'].map(m['growth_rate'])
```
1. **拟合的斜率 (Fitted Slope)**
    
    - **操作**: 这是一个更高级的分析。对于每个客户：
        
        - 将其所有 `下单日期` 转换为距离第一次购买日期的天数（或月数），作为X轴。
            
        - 将其对应的 `订单金额` 作为Y轴。
            
        - 对这些 (X, Y) 数据点进行**线性回归**拟合。
            
        - 得到的**斜率 (slope)** 就是您要的值。
            
    - **目的**: 斜率直观地反映了客户价值随时间的变化趋势。
        
        - **正斜率**: 客户的订单金额随时间呈**增长**趋势 (如 C1002)。
            
        - **接近零的斜率**: 客户的订单金额保持**稳定** (如 C1001)。
            
        - **负斜率**: 客户的订单金额随时间呈**下降**趋势 (如 C1003)。
