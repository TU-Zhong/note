早安！這是一個很好的問題，了解數據結構的內存佔用是編程和數據分析中的一個重要習慣。

要查看 NumPy 數組佔用的內存大小，有兩種非常方便的方法。

---

### 方法一：使用 `.nbytes` 屬性 (最直接、最推薦)

每個 NumPy 數組對象都有一個內置的 `.nbytes` 屬性，它會直接告訴您數組中所有數據佔用的總字節（bytes）數。

**代碼如下：**

Python

```python
import numpy as np

# 重新生成您的 sales_data 數組以確保一致
np.random.seed(42)
sales_data = np.random.randint(50, 200, size=(12, 12))

# 直接訪問 .nbytes 屬性
memory_size = sales_data.nbytes

print(f"sales_data 數組佔用的內存大小為: {memory_size} 字節 (bytes)")
```

**輸出結果 (在常見的64位系統上)：**

```
sales_data 數組佔用的內存大小為: 1152 字節 (bytes)
```

---

### 方法二：手動計算 (`size` * `itemsize`) (有助於理解原理)

如果您想知道這個數字是怎麼來的，可以手動計算。總內存大小等於 **元素總數** 乘以 **單個元素的大小**。

NumPy 也為我們提供了這兩個屬性：

- `.size`: 數組中元素的總個數。
    
- `.itemsize`: 數組中單個元素的字節大小。
    

**代碼如下：**

Python

```python
import numpy as np

# 重新生成數組
np.random.seed(42)
sales_data = np.random.randint(50, 200, size=(12, 12))

# 1. 獲取數組元素的總個數 (12 * 12)
total_elements = sales_data.size
print(f"數組總元素個數 (.size): {total_elements}")

# 2. 獲取單個元素的字節大小
#    因為 sales_data 是由 randint 生成的，預設的 dtype 通常是 int64 (64位整數)
#    一個 int64 佔用 8 個字節 (64 bit / 8 = 8 bytes)
element_size = sales_data.itemsize
print(f"單個元素的字節大小 (.itemsize): {element_size} 字節")
print(f"數組的數據類型 (.dtype): {sales_data.dtype}")

# 3. 手動計算總大小
calculated_size = total_elements * element_size
print(f"手動計算的總內存大小: {calculated_size} 字節")
```

**輸出結果：**

```
數組總元素個數 (.size): 144
單個元素的字節大小 (.itemsize): 8 字節
數組的數據類型 (.dtype): int64
手動計算的總內存大小: 1152 字節
```

您可以看到，手動計算的結果 `144 * 8 = 1152`，與 `.nbytes` 屬性給出的結果完全一致。

---

### 補充說明：和 `sys.getsizeof()` 的區別

您可能還會見到 Python 內置的 `sys.getsizeof()` 函數。但對於 NumPy 數組，它返回的結果會比 `.nbytes` 稍大，因為它還包含了數組對象本身的一些管理開銷（overhead），而不僅僅是數據佔用的內存。

因此，當您關心的是**數據本身佔用多少內存**時，使用 NumPy 的 `.nbytes` 是最準確的方法。

**總結：直接使用 `sales_data.nbytes` 是查看內存佔用最好、最直接的方式。**
