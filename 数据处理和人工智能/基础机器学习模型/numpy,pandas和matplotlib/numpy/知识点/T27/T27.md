交换数组中的两行是一个非常常见的操作，NumPy 提供了非常优雅和高效的方式来完成它。

---

### 重要提示：索引从 0 开始

和之前一样，我们需要记住索引是从 0 开始的：

- **“第1行”** 对应的 **行索引是 `0`**。
    
- **“第2行”** 对应的 **行索引是 `1`**。
    

---
[[花形索引]]
### 方法一：使用花式索引 (Fancy Indexing) (最 NumPy 风格、最安全)

这是在 NumPy 中进行此类操作最地道、最推荐的方法。它允许我们使用一个索引列表来一次性选取和修改多个行。

**代码如下：**

Python

```
import numpy as np

# 重新创建 sales_data 数组
np.random.seed(42)
sales_data = np.random.randint(50, 200, size=(12, 12))

print("交换前的第1行和第2行:")
# 使用花式索引 [[0, 1]] 一次性打印这两行
print(sales_data[[0, 1]])

# 核心交换语句
sales_data[[0, 1]] = sales_data[[1, 0]]

print("\n交换后的第1行和第2行:")
print(sales_data[[0, 1]])
```

**代码解释 `sales_data[[0, 1]] = sales_data[[1, 0]]`**：

- **右侧 `sales_data[[1, 0]]`**: 这部分会先执行。它会创建一个**新的临时数组**，这个新数组的第一行是原始数据的第2行（索引1），第二行是原始数据的第1行（索引0）。
    
- **左侧 `sales_data[[0, 1]] = ...`**: 然后，程序将右侧创建的那个临时数组，赋值给左侧指定的位置。也就是说，把这个临时数组的第一行（即原始的第2行）放到 `sales_data` 的第1行（索引0）上；把临时数组的第二行（即原始的第1行）放到 `sales_data` 的第2行（索引1）上。
    

这个过程清晰、无歧义，并且是原子性的，因此非常安全可靠。

### 方法二：利用 Python 的元组交换语法 (简洁直观)

这种方法利用了 Python 中 `a, b = b, a` 这种简洁的交换语法，可读性非常高。

**代码如下：**

Python

```
import numpy as np

# 为了对比，我们创建另一个数组
np.random.seed(42)
sales_data_2 = np.random.randint(50, 200, size=(12, 12))

print("交换前的第1行和第2行:")
print(sales_data_2[:2])

# 核心交换语句
# 为了确保在所有情况下都万无一失，最好在右侧加上 .copy()
sales_data_2[0], sales_data_2[1] = sales_data_2[1].copy(), sales_data_2[0].copy()

print("\n交换后的第1行和第2行:")
print(sales_data_2[:2])
```

**代码解释：**

- `sales_data_2[0], sales_data_2[1] = ...`: 这是 Python 的多重赋值语法。
    
- `... = sales_data_2[1].copy(), sales_data_2[0].copy()`: 右侧先被求值，创建了第2行和第1行的**副本**，然后分别赋值给左侧的 `sales_data_2[0]` 和 `sales_data_2[1]`。
    
- **为什么加 `.copy()`?** 虽然在很多情况下不加 `.copy()` 也能成功，但在复杂的视图（view）操作中，不加 `copy` 可能会导致意想不到的结果（一个值被错误地覆盖）。加上 `.copy()` 可以确保我们是用交换前那一刻的原始值来进行赋值，从而让这个操作变得100%安全。
    

### 总结

- **方法一（花式索引）** 是最受推荐的 **NumPy 原生方法**，它最能体现 NumPy 的数组编程思想，且无需担心副本或视图的问题。
    
- **方法二（元组交换）** 更 **Pythonic**，可读性极佳，但在 NumPy 中使用时，为了代码的健壮性，建议加上 `.copy()`。
    

对于初学者，**建议您熟练掌握方法一**。
