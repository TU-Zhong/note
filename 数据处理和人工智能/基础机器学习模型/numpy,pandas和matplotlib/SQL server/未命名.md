### 第一部分：SQL Server 更全面的核心知识体系

这次，我们将知识点分为几个层级，从基础业务支撑到DBA级别的底层架构：

**层级一：基础查询与数据清洗 (The Foundation)**

* **核心**: `SELECT`, `FROM`, `WHERE`, `ORDER BY`, 数据类型 (`INT`, `VARCHAR`, `NVARCHAR`, `DATETIME2`)。
* **操作**: 模糊匹配 (`LIKE`)、空值处理 (`IS NULL`, `ISNULL`, `COALESCE`)、类型转换 (`CAST`, `CONVERT`)。这是所有数据操作的基石。

**层级二：数据聚合与多表联接 (Aggregation & Joins)**

* **聚合函数**: `SUM`, `COUNT`, `AVG`, `MAX`, `MIN`，以及结合 `GROUP BY` 和 `HAVING` 的复杂分组。
* **多表关联**:
* 关系代数: `INNER JOIN`, `LEFT JOIN`, `RIGHT JOIN`, `FULL OUTER JOIN`, `CROSS JOIN`。
* 集合运算: `UNION`, `UNION ALL`, `INTERSECT`, `EXCEPT`。


* **子查询**: 标量子查询、表值子查询、相关子查询 (`EXISTS`, `NOT EXISTS`)。

**层级三：高级查询与窗口函数 (Advanced T-SQL)**

* **窗口函数 (Window Functions)**: `ROW_NUMBER()`, `RANK()`, `DENSE_RANK()`, `NTILE()` 以及结合 `OVER(PARTITION BY ... ORDER BY ...)`。
* **通用表表达式 (CTE)**: `WITH` 语法，以及递归 CTE 处理树形/层级结构数据。
* **数据透视**: `PIVOT` 和 `UNPIVOT`。

**层级四：数据库编程与自动化 (Database Programming)**

* **DML与DDL**: `INSERT`, `UPDATE`, `DELETE`, `MERGE` (Upsert操作)，以及表和约束的创建 (`CREATE`, `ALTER`)。
* **控制流与批处理**: `DECLARE` 变量, `IF...ELSE`, `WHILE`, `BEGIN...END`。
* **存储过程与函数**: `CREATE PROCEDURE` (参数化防注入、计划缓存)、标量/表值自定义函数 (UDF)。
* **错误处理与触发器**: `TRY...CATCH`, `RAISERROR`, `THROW`, `AFTER/INSTEAD OF TRIGGER`。

**层级五：底层架构、并发与调优 (Architecture & Tuning)**

* **事务与并发**: `BEGIN TRAN`, `COMMIT`, `ROLLBACK`，死锁排查，以及隔离级别 (`READ COMMITTED`, `READ UNCOMMITTED`/`NOLOCK`, `SNAPSHOT`)。
* **索引原理 (Indexes)**: 聚集索引 (Clustered B-Tree, 页分裂)、非聚集索引、覆盖索引 (`INCLUDE`)、索引碎片。
* **系统元数据 (Metadata)**: 利用系统视图 (`sys.tables`, `sys.indexes`, `sys.dm_exec_requests`) 动态提取底层结构和监控运行状态。
* **执行计划调优**: 参数嗅探 (Parameter Sniffing)，书签查找 (Key Lookup)，全表扫描优化。
