在 Spring Boot 中集成 Knife4j，具体的依赖和写法取决于你的 **Spring Boot 版本**。目前主流的是 Spring Boot 3.x（基于 OpenAPI 3）和 Spring Boot 2.x（基于 Swagger 2）。

考虑到你正在进行较新的项目开发，以下以 **Spring Boot 3.x**（基于 Jakarta 和 OpenAPI 3）的标准配置为例：

### 1. 引入依赖 (`pom.xml`)

对于 Spring Boot 3.x，请使用 `knife4j-openapi3-jakarta-spring-boot-starter`。如果你使用的是 Spring Boot 2.x，请将 `jakarta` 去掉，或者换成 `openapi2` 的 starter。

XML

```
<dependency>
    <groupId>com.github.xiaoymin</groupId>
    <artifactId>knife4j-openapi3-jakarta-spring-boot-starter</artifactId>
    <version>4.4.0</version> <!-- 建议使用最新的 4.x 稳定版本 -->
</dependency>
```

### 2. 编写配置类 (`SwaggerConfig.java`)

在你的项目的 `config` 包下创建一个配置类，用于定义文档的标题、描述、作者等基础信息。
[[这里这个Configuration和Bean注解是干啥的]]
Java

```java
import io.swagger.v3.oas.models.OpenAPI;
import io.swagger.v3.oas.models.info.Contact;
import io.swagger.v3.oas.models.info.Info;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class SwaggerConfig {

    @Bean
    public OpenAPI customOpenAPI() {
        return new OpenAPI()
                .info(new Info()
                        .title("项目 API 接口文档")
                        .description("基于 Spring Boot 3 + Springdoc + Knife4j 的接口文档")
                        .version("1.0.0")
                        .contact(new Contact()
                                .name("开发者姓名")
                                .email("developer@example.com")));
    }
}
```

### 3. 配置 `application.yml`

在配置文件中开启 Knife4j，并指定 Springdoc 去扫描哪个包下的 Controller 以生成文档。

YAML

```
# ===============================
# Springdoc & Knife4j 配置
# ===============================
springdoc:
  api-docs:
    enabled: true
    path: /v3/api-docs            # OpenAPI 规范的默认 JSON 数据接口
  swagger-ui:
    enabled: true
  packages-to-scan: com.你的包名.controller # 🚨重要：替换为你自己 Controller 所在的包路径！

knife4j:
  enable: true                    # 开启 Knife4j 增强功能
  setting:
    language: zh_cn               # 设置界面语言为中文简体
```

### 4. 访问文档

启动 Spring Boot 项目后，在浏览器中输入以下地址即可访问增强版 UI：

👉 **`http://localhost:你的端口/doc.html`**

### ⚠️ 避坑指南：权限拦截（403 错误或重定向）

如果你在项目中集成了 **Spring Security**、**Shiro** 或者自定义了 **HandlerInterceptor（拦截器）** 进行 Token 鉴权，**必须将 Knife4j 的相关静态资源和接口放行**，否则访问 `/doc.html` 时会被拦截导致白屏或报错。

需要在你的安全配置或拦截器放行规则中，添加以下路径白名单：

- `/doc.html` (Knife4j 页面)
    
- `/webjars/` (前端静态资源)
    
- `/v3/api-docs/` (后端提供 OpenAPI 数据的接口)
    
- `/swagger-resources/` (Swagger 资源)
