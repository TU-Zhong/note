在 Spring Boot 3（基于 Springdoc）中，配置接口分组非常简单。你只需要在刚才那个 `SwaggerConfig` 类里，继续利用 `@Bean` 声明几个 `GroupedOpenApi` 对象就可以了。

底层逻辑是：Springdoc 会自动收集容器中所有的 `GroupedOpenApi` 对象，并根据你设置的 `pathsToMatch`（路径匹配规则）去扫描对应的 Controller。

你可以把你的 `SwaggerConfig` 类修改成下面这样：

Java

```
import io.swagger.v3.oas.models.OpenAPI;
import io.swagger.v3.oas.models.info.Contact;
import io.swagger.v3.oas.models.info.Info;
import org.springdoc.core.models.GroupedOpenApi;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class SwaggerConfig {

    // 1. 全局 API 信息（保持不变）
    @Bean
    public OpenAPI customOpenAPI() {
        return new OpenAPI()
                .info(new Info()
                        .title("项目 API 接口文档")
                        .description("基于 Spring Boot 3 + Springdoc + Knife4j 的接口文档")
                        .version("1.0.0")
                        .contact(new Contact()
                                .name("开发者姓名")
                                .email("developer@example.com")));
    }

    // ===============================
    // 下面是分组配置
    // ===============================

    // 2. 管理端接口分组
    @Bean
    public GroupedOpenApi adminApi() {
        return GroupedOpenApi.builder()
                .group("管理端接口")              // 这个名字会显示在 Knife4j 页面左上角的下拉框里
                .pathsToMatch("/admin/**")      // 拦截所有以 /admin/ 开头的接口
                .build();
    }

    // 3. 开放接口分组
    @Bean
    public GroupedOpenApi openApi() {
        return GroupedOpenApi.builder()
                .group("开放接口")
                .pathsToMatch("/api/**")        // 拦截所有以 /api/ 开头的接口
                .build();
    }
}
```

### 配置完成后的效果

当你重启项目并访问 `http://localhost:你的端口/doc.html` 后，Knife4j 界面的左上角（或者顶部导航栏）会出现一个**下拉选择框**。

- 当你选择“管理端接口”时，左侧菜单树只会显示路径中带有 `/admin/` 的 Controller。
    
- 当你选择“开放接口”时，左侧只会显示路径带有 `/api/` 的 Controller。
    

> **补充提示**：如果你在 `application.yml` 里配置了 `springdoc.packages-to-scan`（包扫描），你可以把它删掉。因为在代码里配置了 `pathsToMatch` 之后，按照 URL 路径来分组扫描会比按照包名扫描更加灵活且直观。
