手写一个简易的 IoC（控制反转）容器，其实就是**模拟 Spring 框架最核心的启动和 Bean 加载过程**。

要实现这个功能，整体思路可以分为 **四个核心步骤**。我们可以把它想象成一个“自动化装配车间”，车间需要知道图纸在哪里（包扫描），图纸上有没有特殊标记（解析注解），然后根据标记造出零件（反射实例化），最后把零件妥善存放在仓库里（存入 Map）。

以下是具体的实现思路和简易代码框架：

### 第一步：定义 `@Component` 注解

首先，你需要造一个“标签”。在 Java 中也就是自定义一个注解。这个注解必须在运行期（`RUNTIME`）有效，并且只能作用于类（`TYPE`）上。

Java

```
import java.lang.annotation.*;

@Target(ElementType.TYPE) // 作用在类上
@Retention(RetentionPolicy.RUNTIME) // 运行时保留，这样反射才能读到
public @interface Component {
    String value() default ""; // 允许自定义 Bean 的名字
}
```

### 第二步：准备 IoC 容器（仓库）

IoC 容器的本质就是一个大 Map（通常是 `ConcurrentHashMap`），用来根据 Bean 的名称存储创建好的对象实例。

Java

```
import java.util.Map;
import java.util.concurrent.ConcurrentHashMap;

public class SimpleApplicationContext {
    // 我们的 IoC 容器，Key 是 Bean 的名字，Value 是对象实例
    private Map<String, Object> iocContainer = new ConcurrentHashMap<>();
    
    // 构造函数传入需要扫描的包路径
    public SimpleApplicationContext(String basePackage) {
        scanAndInstantiate(basePackage);
    }
    
    // 提供一个获取 Bean 的方法
    public Object getBean(String beanName) {
        return iocContainer.get(beanName);
    }
}
```

### 第三步：包扫描（找图纸）

这是最繁琐的一步。当我们在构造函数中传入 `com.example` 时，程序需要找到这个包下所有的 `.class` 文件。

1. 将包名 `com.example` 转换为文件路径 `com/example`。
    
2. 使用类加载器（`ClassLoader`）获取这个相对路径在当前项目中的绝对磁盘路径。
    
3. 递归遍历这个磁盘目录，找出所有以 `.class` 结尾的文件。
    
4. 将文件路径反向转换回全限定类名（例如 `com.example.service.UserService`）。
    

### 第四步：反射解析并实例化（造零件并入库）

拿到所有的全限定类名后，就可以利用 Java 的反射机制来大显身手了。

1. 使用 `Class.forName("类的全限定名")` 加载类。
    
2. 判断这个类头上有没有打上我们的 `@Component` 标签：`clazz.isAnnotationPresent(Component.class)`。
    
3. 如果有，利用反射调用无参构造器创建对象：`clazz.getDeclaredConstructor().newInstance()`。
    
4. 决定 Bean 的名字（如果没有在注解中指定，通常把类名首字母小写作为默认名字）。
    
5. 将 `名字 -> 实例` 存入之前准备好的 `iocContainer` 中。
    

### 完整核心逻辑的伪代码示例：

你可以参考下面这个核心逻辑，尝试在你的 IDE 中把它补充完整：

Java

```
private void scanAndInstantiate(String basePackage) {
    // 1. 获取包下的所有类名 (此处省略了递归读取文件的具体代码，假设我们已经拿到了)
    List<String> classNames = getClassNamesFromPackage(basePackage); 

    for (String className : classNames) {
        try {
            // 2. 加载类
            Class<?> clazz = Class.forName(className);

            // 3. 检查是否有 @Component 注解
            if (clazz.isAnnotationPresent(Component.class)) {
                
                // 4. 获取注解实例，看看用户有没有自定义 Bean 名字
                Component componentAnno = clazz.getAnnotation(Component.class);
                String beanName = componentAnno.value();
                
                // 如果没有自定义，就用类名首字母小写作为 BeanName
                if ("".equals(beanName)) {
                    String simpleName = clazz.getSimpleName();
                    beanName = Character.toLowerCase(simpleName.charAt(0)) + simpleName.substring(1);
                }

                // 5. 反射创建实例 (前提是该类有无参构造方法)
                Object instance = clazz.getDeclaredConstructor().newInstance();

                // 6. 放入 IoC 容器
                iocContainer.put(beanName, instance);
                
                System.out.println("成功加载 Bean: " + beanName);
            }
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```

**深入思考与进阶（结合你的技术储备）：**

当你完成了这一步，你会发现目前的容器只能“创建”孤立的对象，不能“组装”。

这就引出了下一个问题：**如何实现 `@Autowired`？**

思路是：在存入 Map 之后，再次遍历 `iocContainer` 中的所有对象，利用反射获取对象的所有的属性（`Field`），检查属性上是否有 `@Autowired` 注解。如果有，就从 `iocContainer` 里面根据类型或名字把对应的依赖捞出来，通过 `field.set(instance, dependency)` 强行赋值（依赖注入）。

你可以先用纯 Java 把上面的包扫描和反射实例化跑通，这是理解 Spring 启动源码 `refresh()` 方法中最重要的一环。
