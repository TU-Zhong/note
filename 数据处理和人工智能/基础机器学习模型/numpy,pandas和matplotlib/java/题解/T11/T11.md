这是一个非常典型的企业级项目需求！你现在的思路是对的，遇到卡壳的地方主要是因为没有完全理解“静态工厂方法”**和**“默认值”的概念。

我先直接回答你的疑问：“这个 code, message, timestamp 怎么拿到？”

1. **`code` 和 `message` 从哪来？**
    
    - 在 `success()` 方法里：既然明确是“成功”了，这两个值通常是直接写死（默认）的，比如状态码给 `200`，提示语给 `"操作成功"`。
        
    - 在 `error()` 方法里：既然是发生错误，到底是什么错误你无法提前预知，所以这两个值要作为**方法的参数**传进来（这也是你题目里要求 `error(int code, String message)` 的原因）。
        
2. **`timestamp` 从哪来？**
    
    - 在创建这个响应对象的一瞬间，直接调用 Java 底层的系统时间 API（如 `System.currentTimeMillis()`）自动生成，不需要别人传给你。
        

### 完整的代码实现

这里有几个细节帮你做了修正：

1. 工厂方法必须是 `static`（静态）的，并且要声明泛型 `<T>`。
    
2. 你题目里说要 `error(int code, ...)`，所以建议把你属性里的 `String code` 改成 `Integer code`，这是业界的通用做法。
    
3. `timestamp` 建议用 `long`（存储毫秒数），前端处理起来最方便，不容易出现时区格式化报错。
    

Java

```
import lombok.Data;

@Data
public class ApiResponse<T> {
    
    // 建议使用 Integer 而不是 String，方便做逻辑判断（比如 code == 200）
    private Integer code;
    private String message;
    private T data;
    // 建议用 long 记录时间戳，或者保留你的 Timestamp
    private long timestamp;

    // 💡 技巧：把构造方法私有化，强制大家只能用 success 和 error 这两个静态方法去创建对象
    private ApiResponse() {
    }

    /**
     * 成功时的静态工厂方法
     * @param data 要返回给前端的具体数据
     */
    public static <T> ApiResponse<T> success(T data) {
        ApiResponse<T> response = new ApiResponse<>();
        response.setCode(200);             // 成功默认状态码
        response.setMessage("操作成功");     // 成功默认提示语
        response.setData(data);            // 装入你传进来的数据
        response.setTimestamp(System.currentTimeMillis()); // 自动获取当前时间
        return response;
    }

    /**
     * 失败/错误时的静态工厂方法
     * @param code    具体的错误码 (如 400, 401, 500)
     * @param message 具体的错误原因说明
     */
    public static <T> ApiResponse<T> error(int code, String message) {
        ApiResponse<T> response = new ApiResponse<>();
        response.setCode(code);            // 使用外部传进来的错误码
        response.setMessage(message);      // 使用外部传进来的错误提示
        response.setData(null);            // 报错了通常就没有业务数据了，给 null
        response.setTimestamp(System.currentTimeMillis()); // 自动获取当前时间
        return response;
    }
}
```

### 为什么非要加 `static`？（怎么使用）

如果你不加 `static`，你在 Controller 里用的时候，就得先 `new` 一个对象出来，像这样：

Java

```
// ❌ 不加 static 的错误/繁琐用法
ApiResponse<String> res = new ApiResponse<>();
return res.success("Hello");
```

加了 `static` 之后，这个方法就属于类本身了，你在 Controller 里写起来会极其丝滑和优雅：

Java

```
@RestController
@RequestMapping("/api")
public class TestController {

    // 模拟成功返回
    @GetMapping("/hello")
    public ApiResponse<String> hello() {
        // ✅ 优雅的用法：直接 类名.方法名()
        return ApiResponse.success("Hello 电商后台"); 
    }

    // 模拟失败返回
    @GetMapping("/fail")
    public ApiResponse<Object> fail() {
        // ✅ 发生异常时，自定义返回状态码和错误信息
        return ApiResponse.error(401, "您还没有登录，请先登录");
    }
}
```

你可以把刚才写的 `TestController` 里的返回类型替换成 `ApiResponse<String>` 试一下，然后再去你的 Knife4j 页面（`/doc.html`）里看一眼“调试”结果，你就会发现返回的 JSON 变得非常规范了！
