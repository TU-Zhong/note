这段配置是 Spring Boot Actuator 的核心权限和显示控制开关。

在完整的 `application.yml` 文件中，它们通常统一嵌套在 `management:` 节点之下。我们可以把这段配置拆成两部分来理解：

### 1. 决定“看多细”

YAML

```
endpoint:
  health:
    show-details: always
```

- **作用**：控制 `/actuator/health` 接口返回信息的详细程度。
    
- **原理解释**：默认情况下，为了防止向外界泄露系统的底层架构（比如你用了什么数据库、什么版本的中间件），Spring Boot 极其保守，只会返回一个简陋的 `{"status": "UP"}`。当你把它设置为 `always` 后，就相当于“解除封印”，允许接口输出所有组件（如 SQL Server、Redis、磁盘空间等）的详细体检报告。
    
- **扩展**：在真正的企业生产环境（线上）中，为了安全，通常会把它设置为 `when-authorized`（只有经过身份验证的管理员登录后才能看到详细信息），而在本地开发环境则多用 `always` 方便排查问题。
    

### 2. 决定“准不准看”

YAML

```
endpoints:
  web:
    exposure:
      include: health
```

- **作用**：控制哪些 Actuator 监控接口可以通过 Web（也就是 HTTP 请求）来访问。
    
- **原理解释**：Actuator 其实提供了十几个极其强大的监控接口（例如访问 `/actuator/env` 能看系统的全部环境变量，访问 `/actuator/beans` 能看 Spring 容器里的所有对象）。为了防止黑客利用这些接口“偷家”，Spring Boot 默认关闭了绝大多数接口的 HTTP 访问权限。
    
- **`include: health` 的意义**：它的意思是“在所有监控接口中，我只允许 `health` 接口暴露在 Web 上”。（如果你想看所有接口，可以配置为 `include: "*"`，但这仅限本地开发时使用，绝不能带到线上去）。
    

总结来说，上半段是让体检报告“毫无保留”**，下半段是给这份体检报告**“开通网络访问权限”。
