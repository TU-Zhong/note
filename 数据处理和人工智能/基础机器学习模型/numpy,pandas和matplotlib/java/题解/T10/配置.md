**Spring Boot Actuator** 是 Spring Boot 官方提供的一个用于“监控和管理”应用程序的工具。

你可以把它理解为给你的电商后台系统装上了一个“健康监视器”或者“心电图”。

在分布式系统或者电商项目中，你的应用通常需要依赖很多外部组件（比如你提到的 SQL Server 数据库、Redis 缓存、RabbitMQ 消息队列）。如果其中任何一个挂掉了，你的程序可能会报错。**`/actuator/health` 这个接口的作用，就是帮你一次性自动探测这些底层组件死没死。**

当它发现你的项目里引入了数据库、Redis 和 RabbitMQ 的依赖时，它会自动帮你去 ping 它们。你只需要访问这个接口，就能看到一份详细的“体检报告”。

要实现这个功能，只需两步：

### 1. 引入依赖

在你的 `pom.xml` 中加入 Actuator 的起步依赖：

XML

```
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>
```

### 2. 修改配置 (`application.yml`)

默认情况下，出于安全考虑，访问 `/actuator/health` 只会返回一个非常简陋的 `{"status": "UP"}`，不会展示具体的数据库、Redis 细节。为了能看到具体状态，我们需要在配置文件里开启详细信息的展示。

把下面的配置加到你的 `application.yml` 中：

YAML

```
management:
  endpoint:
    health:
      show-details: always    # 总是显示详细的健康检查信息
  endpoints:
    web:
      exposure:
        include: health       # 暴露 health 接口（默认已暴露，显式写出来更清晰）
```

### 3. 测试效果

重启项目后，打开浏览器访问 👉 **`http://localhost:你的端口/actuator/health`**

你会看到类似下面这样一长串的 JSON 数据。你可以清楚地看到 `db` (SQL Server)、`redis`、`rabbit` 的连接状态都是 `UP`（正常运行）：

JSON

```
{
  "status": "UP",
  "components": {
    "db": {
      "status": "UP",
      "details": {
        "database": "Microsoft SQL Server",
        "validationQuery": "isValid()"
      }
    },
    "ping": {
      "status": "UP"
    },
    "rabbit": {
      "status": "UP",
      "details": {
        "version": "3.12.0"
      }
    },
    "redis": {
      "status": "UP",
      "details": {
        "version": "7.0.12"
      }
    }
  }
}
```

> **注意**：如果在浏览器里看到的 JSON 是一团乱码没有换行，建议安装一个浏览器插件（如 JSONVue），或者直接在刚才配置好的 Knife4j 的“调试”面板里发送请求，看起来会更整洁。
