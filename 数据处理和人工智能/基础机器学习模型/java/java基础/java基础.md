### 操作系统 (OS) 核心考频榜单

操作系统的考察重点通常集中在**进程/线程管理**、**内存管理**以及**并发安全**。

- **进程与线程的本质区别**：必须能清晰指出它们分别是**资源分配**和**CPU调度**的基本单位。还要能说明它们在内存结构上的差异（例如，线程共享所属进程的堆区和方法区，但拥有独立的虚拟机栈和程序计数器）。
    
- **进程间通信 (IPC) 的方式**：掌握管道、消息队列、共享内存（速度最快）、信号量、Socket。了解它们各自的优缺点及适用场景。
    
- **死锁 (Deadlock) 的四大必要条件**：互斥、占有并等待、非抢占、循环等待。不仅要知道这四个条件，更常考的是**如何预防和避免死锁**（例如经典的银行家算法思想，或者在代码层面如何规定加锁顺序）。
    
- **虚拟内存机制**：为什么要有虚拟内存？它解决了什么问题？需要理解逻辑地址到物理地址的映射过程，以及**缺页中断 (Page Fault)** 的触发和处理流程。
    
- **页面置换算法**：重点掌握 **LRU (最近最少使用)** 算法的思想。面试中非常喜欢让你在手撕代码环节，利用哈希表结合双向链表来实现一个 $O(1)$ 时间复杂度的 LRU 缓存。
    
- **I/O 多路复用 (高阶高频)**：理解 `select`、`poll` 和 `epoll` 的演进与区别。尤其是 `epoll` 基于红黑树和就绪链表的事件驱动机制，这是理解现代高性能网络框架（如 Netty、Nginx、Redis 底层）的基石。

### 计算机网络 (CN) 核心考频榜单

网络部分的考察极度集中在 **TCP/IP 协议簇** 和 **HTTP 协议** 上。

- **TCP 三次握手与四次挥手**：这是绝对的必考题。不仅要在脑海中有状态流转图，还要能应对连环追问：
    
    - 为什么握手是三次，而挥手需要四次？
        
    - `TIME_WAIT` 状态存在的意义是什么（为什么要等待 2MSL）？
        
    - 如果服务器上出现大量 `CLOSE_WAIT` 或 `TIME_WAIT` 状态，说明了什么问题，该如何排查？
        
- **TCP 如何保证可靠传输**：不要只说“面向连接”，要讲清楚底层的机制，包括：校验和、序列号与确认应答、超时重传、**滑动窗口 (流量控制)**，以及复杂的**拥塞控制**（慢启动、拥塞避免、快重传、快恢复）。
    
- **TCP 与 UDP 的区别及应用场景**：除了面向连接/无连接、可靠/不可靠的对比外，要结合实际业务来说明，比如为什么视频流或 DNS 查询常使用 UDP，而文件下载或支付接口必须用 TCP。
    
- **HTTP 与 HTTPS 的区别**：核心考点是 HTTPS 的加密原理。必须讲清楚在 SSL/TLS 握手阶段，是如何巧妙结合**非对称加密**（用于安全交换密钥）和**对称加密**（用于后续高效的数据传输）的，以及 CA 数字证书防伪造的作用。
    
- **经典串联题：“在浏览器输入 URL 到页面显示经历了什么？”**：这是一道考察计算机网络全局观的题目。你需要有条理地串联起：DNS 域名解析、ARP 获取 MAC 地址、TCP 建立连接、发送 HTTP 请求、服务器处理并响应、浏览器渲染页面、最后 TCP 断开连接的全过程。
    
- **常见 HTTP 状态码**：熟知 `200`、`301` (永久重定向)、`302` (临时重定向)、`400`、`403`、`404`、`500`、`502` (Bad Gateway) 和 `504` (Gateway Timeout) 代表的具体业务含义。
