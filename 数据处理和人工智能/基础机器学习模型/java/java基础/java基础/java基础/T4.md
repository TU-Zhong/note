好的，这是一个关于 Java 基础的经典问题。

Java 中共有 **8 种**基本数据类型（Primitive Types）。它们是 Java 语言预定义的，直接存储在**栈内存**中，用于存储简单的值。

这 8 种基本数据类型可以分为四大类：

1. **整型 (Integer Types)**
    
2. **浮点型 (Floating-Point Types)**
    
3. **字符型 (Character Type)**
    
4. **布尔型 (Boolean Type)**
    

下面是它们各自的详细信息，包括所占的字节数、位数和取值范围，我将用一个表格来清晰地展示：

|类别|类型|关键字|**字节数 (Bytes)**|位数 (Bits)|默认值|取值范围|
|---|---|---|---|---|---|---|
|**整型**|字节型|`byte`|**1**|8|`0`|`-128` 到 `127`|
||短整型|`short`|**2**|16|`0`|`-32,768` 到 `32,767`|
||整型|`int`|**4**|32|`0`|`-2,147,483,648` 到 `2,147,483,647` (−231 到 231−1)|
||长整型|`long`|**8**|64|`0L`|`-9,223,372,036,854,775,808` 到 `9,223,372,036,854,775,807` (−263 到 263−1)|
|**浮点型**|单精度|`float`|**4**|32|`0.0f`|大约 ±3.40282347×1038 (有效位数为 6-7 位)|
||双精度|`double`|**8**|64|`0.0d`|大约 ±1.79769313486231570×10308 (有效位数为 15 位)|
|**字符型**|字符|`char`|**2**|16|`'\u0000'`|`0` 到 `65,535` (表示一个 Unicode 字符)|
|**布尔型**|布尔|`boolean`|**未明确定义**|1|`false`|`true` 或 `false`|

---

### 对特殊类型的补充说明

#### 1. 字符型 (`char`)

- `char` 类型在 Java 中占用 **2 个字节**，这是因为它采用了 **Unicode** 编码（具体来说是 UTF-16 编码的码元）。这使得 Java 可以表示世界上绝大多数语言的字符，而不仅仅是像 ASCII 码那样只能表示有限的西欧字符。
    

#### 2. 布尔型 (`boolean`)

- `boolean` 类型的情况比较特殊，它的**字节数没有被 Java 虚拟机（JVM）规范明确定义**。其大小取决于具体的 JVM 实现。
    
- **逻辑上**：它只表示 1 位（bit）的信息（`true` 或 `false`）。
    
- **实际存储上**：
    
    - 当作为一个**单独的变量**使用时，为了内存对齐和寻址方便，JVM 可能会将其当作一个 `int` 或 `byte` 来处理，通常占用 **1 个字节**或 **4 个字节**。
        
    - 当在**数组**（`boolean[]`）中使用时，通常每个元素会占用 **1 个字节**。
        

### 总结要点

- **整数默认是 `int` 类型**：例如 `long num = 10000000000;` 会编译错误，因为右边的数字超出了 `int` 的范围。必须写成 `long num = 10000000000L;`。
    
- **浮点数默认是 `double` 类型**：例如 `float f = 3.14;` 会编译错误，因为 `3.14` 是 `double` 类型，无法直接赋给 `float`。必须写成 `float f = 3.14F;` 或 `float f = (float)3.14;`。
    
- 这些基本数据类型都有对应的**包装类**（Wrapper Classes），例如 `int` 对应 `Integer`，`char` 对应 `Character`。包装类是对象，拥有方法，可以存放在 `ArrayList` 这样的集合中。
