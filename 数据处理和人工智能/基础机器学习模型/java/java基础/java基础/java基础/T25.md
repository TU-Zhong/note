好的，Java 中的访问修饰符（Access Modifiers）是面向对象编程中**封装性（Encapsulation）** 的核心体现。它们用来控制类、接口、构造方法、成员变量和成员方法的可访问性（即可见性范围）。

Java 中共有四种访问修-饰符，按照权限范围从严到宽排序，依次是：

1. **`private`** (私有的)
    
2. **`default`** (默认的，也叫包级私有 package-private，**注意：它没有关键字**)
    
3. **`protected`** (受保护的)
    
4. **`public`** (公共的)
    

---

### 权限范围对比表

这张表格可以非常清晰地展示它们之间的区别：

| 修饰符         | **同一个类中**<br>(Same Class) | **同一个包中**<br>(Same Package) | **不同包的子类中**<br>(Subclass in Different Package) | **不同包的非子类中**<br>(Non-subclass in Different Package) |
| ----------- | ------------------------- | --------------------------- | ---------------------------------------------- | --------------------------------------------------- |
| `private`   | ✔️                        | ❌                           | ❌                                              | ❌                                                   |
| `default`   | ✔️                        | ✔️                          | ❌                                              | ❌                                                   |
| `protected` | ✔️                        | ✔️                          | ✔️                                             | ❌                                                   |
| `public`    | ✔️                        | ✔️                          | ✔️                                             | ✔️                                                  |

### 详细解释与使用场景

#### 1. `private` (私有的)

- **权限范围**：**最严格的访问级别**。被 `private` 修饰的成员，只能在**其所在的类内部**被访问。离开了这个类，==就再也无法直接访问。==
    
- **使用场景**：
    
    - **类的成员变量**：通常都应该声明为 `private`，以隐藏类的内部状态和实现细节。
        
    - **类的辅助方法**：只在类内部被其他方法调用的工具性方法。
        
- **核心思想**：**封装**。通过将数据设为私有，并提供公共的 getter/setter 方法来控制数据的访问，可以保证数据的安全性和一致性。
    

**示例：**

Java

```java
public class Person {
    private String name; // 私有成员变量

    private boolean isValid() { // 私有辅助方法
        return name != null && !name.isEmpty();
    }
    
    public void setName(String name) {
        this.name = name;
    }
}
```

#### 2. `default` (默认 / 包级私有)

- **如何声明**：**不写任何访问修饰符**，就是 `default` 权限。
    
- **权限范围**：可以在**同一个包（package）** 内的任何类中被访问。
    
- **使用场景**：当你希望某些类、接口或成员只在包内部作为“模块内”的组件相互调用，而不希望暴露给包外部时使用。它常用于实现包级别的 API。
    

示例：

com/example/A.java:

Java

```
package com.example;

class A { // 这个类是 default 权限
    String message = "Hello"; // 这个变量也是 default 权限
}
```

`com/example/B.java`:

Java

```
package com.example;

public class B {
    public void test() {
        A a = new A(); // OK, B 和 A 在同一个包
        System.out.println(a.message); // OK, 可以访问 a 的 default 成员
    }
}
```

#### 3. `protected` (受保护的)

- **权限范围**：它包含了 `default` 的所有权限（即**同一个包内**均可访问），此外还**允许不同包中的子类**访问。
    
- **使用场景**：专门为**继承**而设计。当父类中的某个成员，你希望它能被所有的子类（无论在哪个包）继承和使用，但又不希望它被完全公开给其他无关的类时，就用 `protected`。
    

示例：

com/library/Book.java:

Java

```
package com.library;

public class Book {
    protected String title; // 受保护的成员
    
    protected void checkOut() { // 受保护的方法
        System.out.println("Checking out: " + title);
    }
}
```

`com/myapp/Dictionary.java`:

Java

```
package com.myapp;
import com.library.Book;

public class Dictionary extends Book {
    public void describe() {
        this.title = "Oxford Dictionary"; // OK, 子类可以访问父类的 protected 成员
        this.checkOut();                  // OK, 子类可以调用父类的 protected 方法
    }
}
```

#### 4. `public` (公共的)

- **权限范围**：**最宽松的访问级别**。被 `public` 修饰的成员，可以在**任何地方**被访问，没有任何限制。
    
- **使用场景**：用于定义一个类或一个库的**公共 API（应用程序编程接口）**。这些是你希望所有其他类都能使用的功能。
    
- **示例**：
    
    - `main` 方法必须是 `public` 的，以便 JVM 可以从任何地方调用它。
        
    - 类的构造方法通常是 `public` 的，以便其他类可以创建它的实例。
        
    - 工具类（如 `Math`）中的方法通常是 `public static` 的。
        

---

### 选择建议（最佳实践）

在设计类时，遵循**最小权限原则**：

1. 首先，所有成员变量都应默认为 **`private`**。
    
2. 然后，根据需要，为外部提供 **`public`** 的方法（API）。
    
3. 如果一个方法或变量需要被子类继承和使用，但又不想完全公开，使用 **`protected`**。
    
4. 如果一些类或成员需要在同一个包内紧密协作，而不想对包外暴露，使用 **`default`**。
