这个问题绝对是 Java 并发编程面试的**“分水岭”**。能答出底层 `Monitor`（监视器）的运作机制，说明你对 JVM 的底层原理有了实质性的掌握。

要彻底扒开 `synchronized` 的外衣，我们需要从**字节码层面**和**操作系统层面**两个维度来剖析。

### 1. 表面现象：字节码层面的 `monitorenter` 和 `monitorexit`

当你在代码里写下 `synchronized(lock) { ... }` 时，Java 编译器会在编译后的 `.class` 字节码文件中，悄悄插入两个指令：

- **`monitorenter`**：插在同步代码块的开始位置。意思是“尝试获取该对象的 Monitor 锁”。
    
- **`monitorexit`**：插在同步代码块的结束位置（以及异常发生的位置）。意思是“释放该对象的 Monitor 锁”。
    

_注：如果是 `synchronized` 修饰在方法上，字节码中不会有这两个指令，而是会打上一个 `ACC_SYNCHRONIZED` 的访问标志，但底层本质完全一样。_

### 2. 底层核心：什么是 Monitor（监视器）？

在 Java 虚拟机（HotSpot）中，**任何一个 Java 对象天生都自带了一把看不见的锁，这把锁就是 Monitor。**

Monitor 不是 Java 写的，而是用 **C++** 实现的（源码中的 `ObjectMonitor`）。当一个线程执行到 `monitorenter` 时，它其实就是在和这个 C++ 的 `ObjectMonitor` 对象打交道。

这个 `ObjectMonitor` 内部有三个极其核心的属性，你一定要记住，这是整个同步机制的灵魂：

1. **`_owner`（所有者）：** 指向当前**正在持有**这把锁的线程。这个位置最多只能容纳一个线程。
    
2. **`_EntryList`（入口队列）：** 如果此时锁被别人拿走了，后来竞争锁失败的线程，全部会被关进这个队列里，状态变为 **Blocked（阻塞）**。
    
3. **`_WaitSet`（等待集合）：** 如果持有锁的线程主动调用了 `lock.wait()` 方法，它会立刻放弃这把锁，并退到这个集合里，状态变为 **Waiting（等待）**，直到有人调用 `notify()` 把它叫醒。
    

### 3. 动态流转：线程的“争夺战”

为了让你形成深刻的肌肉记忆，我们可以把 Monitor 想象成一个**只有一个主治医生的高级诊室**。

- **争抢：** 线程 A 和 线程 B 同时到达，线程 A 跑得快，挤进了诊室（成为了 **`_owner`**），关上门。
    
- **阻塞：** 线程 B 进不去，只能在门外的走廊长椅上排队等候（进入 **`_EntryList`**）。此时 B 的状态是 Blocked。
    
- **等待 (wait)：** 线程 A 看了病，医生说：“你需要去等化验结果，先出去吧。” 于是线程 A 走出诊室，去大厅的沙发上干等（调用 `wait()`，进入 **`_WaitSet`**）。此时 A 放开了锁，诊室空了。
    
- **上位：** 诊室一空，门外的线程 B 立刻冲进去，成为了新的 **`_owner`**。
    
- **唤醒 (notify)：** 线程 B 看完病，顺手在广播里喊了一句：“刚才那个等化验的，结果出来了！”（调用 `notify()`）。此时，大厅沙发上的线程 A 被唤醒，但它**不能立刻进诊室**，而是必须重新回到走廊的长椅上排队（从 `_WaitSet` 移动到 **`_EntryList`**），准备再次争抢。

### 动手尝试：Monitor 锁机制推演沙盘

纯文字容易绕晕，我为你用图表化做了一个推演沙盘。你可以按照 `A 加锁 -> B 加锁 -> A wait -> B notify` 的顺序，亲眼看看底层这两个队列是如何交替配合的。

**面试终极补充说明：锁升级机制（JDK 1.6 优化）**

早期的 `synchronized` 被称为**“重量级锁”**，因为它每次让线程去 `_EntryList` 阻塞时，都需要操作系统介入，把线程从“用户态”切换到“内核态”，这个切换成本极其高昂。

因此，在 JDK 1.6 之后，JVM 对 `synchronized` 做了史诗级优化，引入了**锁升级机制**。它并不会一开始就直接动用底层粗暴的 Monitor，而是按需逐步升级：

**无锁 -> 偏向锁 -> 轻量级锁 (CAS 自旋) -> 重量级锁 (Monitor)。**

只要在面试时，把 **Monitor 的三个核心属性（owner, EntryList, WaitSet）** 以及 **JDK 1.6 的锁升级** 这两板斧抛出来，这个知识点你就拿到了满分。
