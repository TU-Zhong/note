晚安！这是一个非常实际的工程问题。当 Java 内置的异常类型不足以清晰地描述我们程序中特定的错误状况时，自定义异常就显得尤为重要。

自定义异常是构建健壮、可维护的应用程序的重要一环。

### 为什么要自定义异常？

我们不直接用 `new Exception("...")` 的主要原因有三个：

1. 提高代码可读性和明确性：
    
    异常类的名称本身就应该清晰地说明问题。相比于一个通用的 Exception，一个 UserNotFoundException 或 InsufficientBalanceException 能让其他开发者（以及未来的你）在第一时间就明白错误的具体性质，而不需要去解析异常信息字符串。
    
2. 区分不同的错误类型：
    
    通过创建不同的异常类，你可以在 catch 块中对不同类型的错误进行精确的、不同的处理。
    
    Java
    
    ```
    try {
        // ... 业务逻辑 ...
    } catch (UserNotFoundException e) {
        // 跳转到用户不存在的提示页面
    } catch (InsufficientBalanceException e) {
        // 提示用户余额不足
    } catch (Exception e) {
        // 处理其他未知错误
    }
    ```
    
3. 携带额外信息：
    
    你可以为你的自定义异常类添加额外的字段，用来携带更多关于错误的上下文信息。例如，一个 ValidationException 可以在内部包含一个列表，列出所有未通过校验的字段。
    

---

### 创建自定义异常的步骤

创建自定义异常非常简单，只需要遵循以下几个步骤：

#### ==第一步（最关键）：选择父类==

你需要决定你的自定义异常是**受检异常 (Checked Exception)** 还是**运行时异常 (Unchecked Exception)**。

- **创建受检异常**：让你的类继承自 `java.lang.Exception`。
    
    - **适用场景**：当你认为方法的调用者**有能力也应该**处理这个异常时。例如，`FileNotFoundException` 是受检异常，因为调用者可以尝试使用一个备用文件或提示用户。
        
- **创建运行时异常**：让你的类继承自 `java.lang.RuntimeException`。
    
    - **适用场景**：当异常的发生通常意味着一个**编程错误（Bug）**，调用者无法做任何有意义的恢复操作时。例如，`NullPointerException` 是运行时异常，因为最好的处理方式是修复代码，而不是到处捕获它。
        

#### 第二步：遵循命名约定

通常，自定义异常的类名以 `Exception` 结尾，例如 `InvalidAgeException`。

#### ==第三步：提供标准的构造方法（最佳实践）==

为了让你的自定义异常像 Java 内置异常一样灵活好用，强烈建议提供以下四种标准的构造方法。这些构造方法只是简单地调用父类的对应构造方法即可。

1. 无参构造方法。
    
2. 接收一个字符串消息的构造方法。
    
3. 接收一个 `Throwable` 原因（用于异常链）的构造方法。
    
4. 同时接收字符串消息和 `Throwable` 原因的构造方法。

### 一个完整的实例

**场景**：我们来创建一个自定义的受检异常 `InvalidAgeException`。当注册用户时，如果年龄小于 18 岁，就抛出这个异常。

#### 1. 定义 `InvalidAgeException` 类

Java

```java
// 步骤一：继承自 Exception，所以它是一个受检异常
public class InvalidAgeException extends Exception {
    
    // (可选) 步骤四：添加自定义属性来携带额外信息
    private final int invalidAge;

    // 步骤三：提供标准的构造方法
    public InvalidAgeException() {
        super();
        this.invalidAge = -1; // 默认值
    }

    public InvalidAgeException(String message, int age) {
        super(message);
        this.invalidAge = age;
    }

    public InvalidAgeException(String message, Throwable cause, int age) {
        super(message, cause);
        this.invalidAge = age;
    }

    public InvalidAgeException(Throwable cause, int age) {
        super(cause);
        this.invalidAge = age;
    }
    
    // (可选) 为自定义属性提供 getter 方法
    public int getInvalidAge() {
        return invalidAge;
    }
}
```

#### 2. 在业务逻辑中使用它

Java

```
public class UserService {
    // 因为 InvalidAgeException 是受检异常，所以必须在方法签名上用 throws 声明
    public void register(String name, int age) throws InvalidAgeException {
        if (age < 18) {
            // 抛出我们自定义的异常
            throw new InvalidAgeException("用户年龄不能小于18岁", age);
        }
        System.out.println("用户 " + name + " (年龄 " + age + ") 注册成功！");
    }
}
```

#### 3. 在调用层处理它

Java

```
public class Main {
    public static void main(String[] args) {
        UserService service = new UserService();
        try {
            // 编译器会强制我们处理这个受检异常
            service.register("张三", 25);
            service.register("李四", 16); // 这行会抛出异常
        } catch (InvalidAgeException e) {
            System.err.println("注册失败: " + e.getMessage());
            // 我们可以从异常对象中获取自定义的额外信息
            System.err.println("检测到的不合法年龄是: " + e.getInvalidAge());
        }
    }
}
```

**运行结果：**

```
用户 张三 (年龄 25) 注册成功！
注册失败: 用户年龄不能小于18岁
检测到的不合法年龄是: 16
```

### 总结

自定义异常是 Java 异常处理框架的重要补充。通过创建具有明确业务含义的异常类，可以让你的应用程序架构更清晰、代码更易读、错误处理更精确。在选择继承 `Exception`还是 `RuntimeException` 时，关键要思考：“**这是一个调用者应该预料到并处理的恢复性问题，还是一个应该被修复的程序性 Bug？**”
