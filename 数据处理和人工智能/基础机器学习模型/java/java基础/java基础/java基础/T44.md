晚安！我们刚刚聊完了运行时异常，现在来看看与之对应的**编译时异常（Checked Exception）**，也叫受检异常。

回顾一下，它们的共同特点是，Java 编译器会**强制**开发者必须对它们进行处理（通过 `try-catch` 捕获或 `throws` 声明抛出）。

它们通常代表了程序与**外部环境**（如文件系统、网络、数据库）交互时可能发生的、可预见的意外情况。处理这些异常是编写健壮程序的关键部分。

下面是一些在 Java 开发中最常见的编译时异常。

---

### 1. `IOException` (输入/输出异常)

这是最常见的受检异常“家族”。它是一个基类，包含了所有在进行输入/输出（I/O）操作时可能发生的错误。

- **发生场景**：任何与文件读写、网络数据传输相关的操作，都可能因为各种原因失败（如磁盘已满、文件被锁定、网络连接中断、权限不足等）。
    
- **处理方式**：通常需要关闭资源、记录错误日志、向用户显示“操作失败”的提示，或者进行重试。
    
- **代码示例**：
    
    Java
    
    ```
    import java.io.FileReader;
    import java.io.IOException;
    
    public class CheckedExceptionExample {
        public void readFile(String filePath) {
            // FileReader 的构造方法声明了 throws FileNotFoundException (是 IOException 的子类)
            // 如果不处理，编译器会报错
            try {
                FileReader reader = new FileReader(filePath);
                // ... 读取文件 ...
                reader.close();
            } catch (IOException e) {
                // 必须捕获或在方法签名上用 throws 声明
                System.err.println("文件读写失败: " + e.getMessage());
            }
        }
    }
    ```
    
- **常见的子类**：
    
    - **`FileNotFoundException`**：当试图打开一个不存在的文件时抛出。这是 `IOException` 最具体的、最常见的子类之一。
        

### 2. `SQLException` (SQL 异常)

- **发生场景**：当使用 JDBC 与数据库进行交互时，任何环节出错都可能抛出 `SQLException`。例如：
    
    - 数据库连接失败。
        
    - SQL 语句存在语法错误。
        
    - 试图插入违反数据库约束（如主键冲突）的数据。
        
    - 查询的表或字段不存在。
        
- **处理方式**：记录详细的 SQL 错误信息，进行事务回滚（Transaction Rollback），向用户返回一个通用的“数据库操作失败”提示。
    
- **代码示例 (概念性)**：
    
    Java
    
    ```
    import java.sql.*;
    
    public void queryDatabase() {
        try {
            Connection conn = DriverManager.getConnection("url", "user", "pass");
            Statement stmt = conn.createStatement();
            // executeQuery 方法声明了 throws SQLException
            ResultSet rs = stmt.executeQuery("SELECT * FROM users");
            // ... 处理结果集 ...
            conn.close();
        } catch (SQLException e) {
            // 必须处理 SQLException
            System.err.println("数据库查询出错，错误码: " + e.getErrorCode());
            // ... 进行事务回滚等操作 ...
        }
    }
    ```
    

### 3. `ClassNotFoundException` (类未找到异常)

- **发生场景**：主要发生在**运行时**通过**反射**动态加载类的时候。`Class.forName("...")` 方法在指定的类路径（classpath）下找不到具有该名称的类时，就会抛出此异常。
    
- **处理方式**：通常意味着程序缺少了必要的依赖库（JAR 包），或者类名写错了。应该记录错误并检查部署环境。
    
- **代码示例**：
    
    Java
    
    ```
    public void loadClassDynamically() {
        try {
            // 尝试加载一个不存在的类
            Class.forName("com.example.NonExistentClass");
        } catch (ClassNotFoundException e) {
            // 必须处理 ClassNotFoundException
            System.err.println("动态加载类失败，请检查类路径和类名是否正确。");
            e.printStackTrace();
        }
    }
    ```
    

### 4. `InterruptedException` (中断异常)

- **发生场景**：主要与**多线程**编程相关。当一个线程正在执行一个可中断的操作（如 `Thread.sleep()`, `Object.wait()`, `Future.get()`）而处于阻塞状态时，如果其他线程调用了该线程的 `interrupt()` 方法来“中断”它，那么那个阻塞的方法就会抛出 `InterruptedException`。
    
- **处理方式**：`catch` 到这个异常通常意味着当前线程被请求停止它正在做的事情。标准的处理方式是清理资源，有时会重新设置中断状态（`Thread.currentThread().interrupt()`），然后结束当前任务。
    
- **代码示例**：
    
    Java
    
    ```
    public void sleepForAWhile() {
        try {
            // Thread.sleep() 声明了 throws InterruptedException
            System.out.println("线程准备休眠5秒...");
            Thread.sleep(5000);
        } catch (InterruptedException e) {
            System.err.println("线程休眠被中断了！");
            // 恢复中断状态，是一种好的实践
            Thread.currentThread().interrupt();
        }
    }
    ```
    

### 总结

你可以发现，受检异常（编译时异常）有一个非常鲜明的共同点：它们都与**程序无法直接控制的外部资源或环境**有关。

- 文件系统 (`IOException`)
    
- 数据库 (`SQLException`)
    
- 类加载器和类路径 (`ClassNotFoundException`)
    
- 其他线程 (`InterruptedException`)
    

Java 的设计哲学是，当你的代码需要与这些不可靠的外部因素打交道时，**必须**提前想好如果出现问题该怎么办（`try-catch` 或 `throws`），从而强制你编写出更加健壮和容错的程序。
