当然，这里有一个精心设计的短小精悍的例子，它在一个程序中包含了 `sleep()`, `wait()`, `yield()`, `join()` 四个方法，并配有详细的注释和执行分析，以展示它们各自的作用和区别。

### 示例代码

Java

```java
public class AllInOneThreadExample {

    // 创建一个所有线程共享的对象锁
    private static final Object lock = new Object();

    public static void main(String[] args) throws InterruptedException {
        System.out.println("主线程(main)开始运行...");

        Thread workerThread = new Thread(() -> {
            synchronized (lock) {
                try {
                    System.out.println("子线程(worker): 已获得锁，准备调用 wait() 进入等待状态并释放锁...");
                    // 1. wait(): 释放锁，进入WAITING状态，等待被notify
                    lock.wait();
                    System.out.println("子线程(worker): 已被唤醒并重新获得锁！");
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
            
            System.out.println("子线程(worker): 准备调用 yield() 礼让CPU...");
            // 2. yield(): 暗示调度器可以运行其他线程，但自己仍是RUNNABLE
            Thread.yield();
            System.out.println("子线程(worker): yield() 结束，继续运行。");
            
            System.out.println("子线程(worker): 准备调用 sleep(2000) 休眠2秒，但不释放锁...");
            try {
                // 3. sleep(): 进入TIMED_WAITING状态，不会释放任何锁
                Thread.sleep(2000);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            System.out.println("子线程(worker): 休眠结束，任务完成！");
        }, "Worker");

        // 启动子线程
        workerThread.start();
        
        // 主线程稍微等一下，确保子线程先拿到锁并调用wait()
        Thread.sleep(500);

        synchronized (lock) {
            System.out.println("主线程(main): 已获得锁，准备调用 notify() 唤醒子线程...");
            // 唤醒在lock对象上等待的一个线程
            lock.notify();
            System.out.println("主线程(main): notify() 已调用，但会等同步块结束才真正释放锁。");
        } // 主线程在这里释放锁

        System.out.println("主线程(main): 准备调用 join() 等待子线程执行完毕...");
        // 4. join(): main线程进入WAITING状态，直到workerThread执行完毕
        workerThread.join();

        System.out.println("主线程(main): 子线程已结束，主线程运行完毕。");
    }
}
```

---

### 代码逻辑分解

1. **`wait()` / `notify()`**
    
    - `workerThread` 启动后，首先进入 `synchronized` 代码块获得 `lock` 对象锁。
        
    - 然后它调用 `lock.wait()`，这会使其**立即释放 `lock` 锁**并进入 `WAITING` 状态。
        
    - 主线程 `main` 等待了500毫秒后，也进入 `synchronized` 代码块（此时它可以成功，因为 `workerThread` 已释放锁）。
        
    - 主线程调用 `lock.notify()` 来唤醒 `workerThread`。`workerThread` 被唤醒后，并不会立即执行，而是进入 `BLOCKED` 状态，因为它需要**重新竞争** `lock` 锁。
        
    - 主线程的 `synchronized` 代码块执行完毕，释放了 `lock` 锁。
        
    - `workerThread` 成功获得锁，从 `wait()` 方法处继续执行。
        
2. **`yield()`**
    
    - `workerThread` 在执行完同步代码块后，调用 `Thread.yield()`。
        
    - 这是一个**建议性**的操作，它告诉线程调度器：“我愿意让出CPU”。调度器可能会让其他线程执行，也可能忽略这个建议，让 `workerThread` 继续执行。在这个简单的例子中，你可能看不到明显的效果，但它展示了其“礼让”的意图。
        
3. **`sleep()`**
    
    - 接着 `workerThread` 调用 `Thread.sleep(2000)`。
        
    - 它会进入 `TIMED_WAITING` 状态，**暂停执行2秒**。在此期间，它**不会释放**任何它可能持有的锁（虽然在这个例子中它此时已经不在同步块内了）。
        
4. **`join()`**
    
    - 主线程 `main` 在唤醒 `workerThread` 之后，调用了 `workerThread.join()`。
        
    - 这使得 `main` 线程进入 `WAITING` 状态，它会一直**暂停**在这里，直到 `workerThread` 的 `run()` 方法**完全执行结束**。
        
    - 当 `workerThread` 打印出“任务完成！”并退出 `run()` 方法后，`main` 线程才会从 `join()` 方法处继续执行。

这个例子通过主线程和子线程的互动，紧凑地演示了这四个核心方法的行为和它们之间的关键区别。
