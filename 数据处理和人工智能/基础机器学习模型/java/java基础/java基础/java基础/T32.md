好的，关于 Java `switch` 语句能作用于哪些数据类型，这个问题随着 Java 版本的迭代，答案也在不断地丰富。

总的来说，`switch` 语句支持的类型越来越多，使其变得更加强大和灵活。

---

### 按 Java 版本演进划分

我们可以按照 Java 版本的演进来看 `switch` 支持的类型：

#### 1. Java 5 之前 (The Basics)

在早期版本的 Java 中，`switch` 语句只能作用于和 `int` 类型兼容的**基本数据类型**。

- **`int`**
    
- **`byte`**
    
- **`short`**
    
- **`char`**
    

这四种类型都可以被隐式地提升（promote）为 `int` 而不丢失信息，这是 `switch` 底层实现（基于整数跳转指令）所要求的。

**示例 (使用 `int`)**:

Java

```
int day = 3;
switch (day) {
    case 1: System.out.println("星期一"); break;
    case 2: System.out.println("星期二"); break;
    case 3: System.out.println("星期三"); break;
    default: System.out.println("未知"); break;
}
```

#### 2. Java 5 的增强 (Enum & Wrapper Classes)

Java 5 带来了两个重要的增强：

- 枚举类型 (enum)
    
    switch 对 enum 的支持是该版本的一大亮点，代码可读性和类型安全性都得到了极大的提升。这是 enum 类型最常见的应用场景之一。
    
    **示例 (使用 `enum`)**:
    
    Java
    
    ```
    enum DayOfWeek { MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY }
    
    DayOfWeek today = DayOfWeek.WEDNESDAY;
    switch (today) {
        case MONDAY: System.out.println("处理周一的事务"); break;
        case TUESDAY: System.out.println("处理周二的事务"); break;
        case WEDNESDAY: System.out.println("处理周三的事务"); break;
        default: System.out.println("处理其他事务"); break;
    }
    ```
    
- 基本类型的包装类
    
    与 byte, short, char, int 对应的包装类也可以用于 switch 语句了。
    
    - `Byte`
        
    - `Short`
        
    - `Character`
        
    - `Integer`
        
    
    这是通过 Java 5 引入的**自动拆箱（Auto-unboxing）** 机制实现的。在 `switch` 判断时，包装类对象会自动转换成对应的基本类型值。
    

#### 3. Java 7 的增强 (String)

Java 7 带来了一个开发者期盼已久的功能：

- String 类型
    
    可以直接在 switch 中使用字符串，这使得代码比使用一长串的 if-else if 语句（if (str.equals("..."))）要简洁和清晰得多。
    
    **示例 (使用 `String`)**:
    
    Java
    
    ```
    String command = "LOGIN";
    switch (command) {
        case "LOGIN":
            System.out.println("执行登录操作");
            break;
        case "LOGOUT":
            System.out.println("执行登出操作");
            break;
        case "REGISTER":
            System.out.println("执行注册操作");
            break;
        default:
            System.out.println("无效命令");
            break;
    }
    ```
    
    **注意**: `String` 类型的 `switch` 在底层是通过 `hashCode()` 和 `equals()` 方法组合实现的，编译器会自动生成相应的字节码来保证其正确性。
    

---

### 不支持的类型

了解 `switch` **不支持**哪些类型同样重要：

- **`long`**: 尽管是整数类型，但 `long` 的范围比 `int` 大，无法直接兼容 `switch` 底层的 `int` 跳转指令，因此不被支持。
    
- **`float`, `double`**: 浮点数存在精度问题，直接用 `==` 进行比较本身就是不可靠的，因此 `switch` 不支持它们以避免产生难以预料的 bug。
    
- **`boolean`**: `boolean` 类型只有 `true` 和 `false` 两种值，使用 `if-else` 语句来处理比 `switch` 更简单、更清晰。
    
- **其他任何对象类型** (除了上面提到的 `String` 和枚举包装类)。
    

---

### 总结

|Java 版本|新增支持的数据类型|
|---|---|
|**Java 5 之前**|`byte`, `short`, `char`, `int`|
|**Java 5**|`enum`, `Byte`, `Short`, `Character`, `Integer`|
|**Java 7**|`String`|

### 现代 Java 的演进

值得一提的是，从 **Java 12/14** 开始，`switch` 语句本身也得到了极大的增强，演变成了**Switch 表达式**，可以使用 `->` 语法，并且可以有返回值。

从 **Java 17** 开始，**模式匹配（Pattern Matching for switch）** 作为预览功能引入，并在 **Java 21** 中成为正式功能。这使得 `switch` 甚至可以对**对象的类型**进行匹配，功能变得空前强大。

**示例 (现代 Switch 表达式与模式匹配 - Java 21)**:

Java

```
Object obj = "Hello World";

String result = switch (obj) {
    case Integer i -> "这是一个整数: " + i;
    case String s -> "这是一个字符串，长度为 " + s.length();
    case null -> "这是一个 null";
    default -> "其他类型";
};
System.out.println(result); // 输出: 这是一个字符串，长度为 11
```

这个特性进一步扩展了 `switch` 的应用范围，使其不再局限于少数几种数据类型。但就传统的 `switch` 语句而言，上表总结的类型是其支持的标准范围。
