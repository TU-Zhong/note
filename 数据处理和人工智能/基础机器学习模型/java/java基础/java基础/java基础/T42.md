晚安！这个问题紧接着我们刚刚讨论的异常处理，是理解该机制的关键一步。`throw` 和 `throws` 在拼写上只差一个 's'，但它们在 Java 中的角色和用途**完全不同**。

简单来说，可以这样记忆：

- **`throw`** 是一个**动作**：像动词一样，表示主动**“抛出”一个**异常的实例。
    
- **`throws`** 是一个**声明**：像名词一样，用在方法上**“声明”可能抛出的**异常类型列表。
    

---

下面我们来详细地对比它们。

### 1. `throw` 关键字

- **性质**：**动词（一个动作）**。
    
- **作用**：在代码中**显式地、主动地抛出一个具体的异常对象（实例）**。一旦执行 `throw` 语句，程序的正常流程就会立即中断，并开始在调用栈中寻找能够处理这个异常的 `catch` 块。
    
- **位置**：用在**方法体内部**。
    
- **语法**：`throw throwableInstance;`
    
    - `throwableInstance` 必须是一个 `Throwable` 类或其子类的实例，例如 `throw new IllegalArgumentException("参数不合法！");`。
        
- **数量**：每次只能抛出**一个**异常实例。
    

比喻：拉响火警警报。

当你（代码）发现了一个紧急情况（比如一个非法的参数），你就主动去拉响警报（throw new Exception()），通知整个系统出问题了。

**代码示例：**

Java

```
public void setAge(int age) {
    if (age < 0 || age > 150) {
        // 当年龄不合法时，主动创建一个异常实例并把它“抛”出去
        throw new IllegalArgumentException("年龄必须在 0 到 150 之间。");
    }
    // ... 正常逻辑 ...
}
```

---

### 2. `throws` 关键字

- **性质**：**名词（一个声明）**。
    
- **作用**：用在方法签名上，向这个方法的**调用者**声明：“我这个方法在执行过程中，**可能会抛出**这些列出的**受检异常（Checked Exception）**，你自己看着办吧！” 它本质上是一种**契约**和**警告**。
    
- **位置**：用在**方法签名中**，位于参数列表 `()` 之后，方法体 `{}` 之前。
    
- **语法**：`returnType methodName() throws ExceptionType1, ExceptionType2, ... { }`
    
    - `ExceptionType` 是异常的**类型（Class）**，而不是实例。
        
- **数量**：可以声明抛出**一个或多个**异常类型，用逗号隔开。
    

比喻：在门口挂上“内有危险，可能起火”的警示牌。

这个警示牌（throws 声明）本身并不会引发火灾，它只是提醒所有想进入这个房间（调用这个方法）的人，你需要有应对火灾的准备（try-catch）或者准备好把火灾警报传递给你的上级（在自己的方法上也声明 throws）。

---

### 一个结合使用的例子

这个例子能最好地展示两者的关系：`throws` 是“声明”，`throw` 是“行动”。

Java

```
import java.io.IOException;

public class FileManager {

    // 1. 使用 throws 在方法签名上“声明”，这个方法可能会遇到 IOException
    public void readFile(String filePath) throws IOException {
        if (filePath == null || filePath.isEmpty()) {
            // 2. 使用 throw 在方法体内“抛出”一个具体的异常实例
            // 注意：IllegalArgumentException 是 RuntimeException，无需强制声明
            throw new IllegalArgumentException("文件路径不能为空！");
        }

        if (!filePath.endsWith(".txt")) {
            // 2. IOException 是受检异常，必须被处理
            // 这里我们选择把它“抛”出去，让调用者处理
            throw new IOException("只支持读取 .txt 文件！");
        }

        System.out.println("正在读取文件: " + filePath);
        // ... 真正的文件读取逻辑 ...
    }

    public static void main(String[] args) {
        FileManager fm = new FileManager();
        try {
            // 3. 因为 readFile() 声明了 throws IOException，所以调用者必须处理它
            fm.readFile("my-document.md");
        } catch (IOException | IllegalArgumentException e) {
            // 4. 在这里捕获到 readFile() 方法中 throw 出来的异常
            System.err.println("出错了: " + e.getMessage());
        }
    }
}
```

### 总结对比表

|特性|`throw`|`throws`|
|---|---|---|
|**性质**|**动词 (动作)**|**名词 (声明)**|
|**位置**|**方法体内部**|**方法签名中**|
|**作用对象**|一个具体的**异常实例** (`new Exception()`)|一个或多个**异常类** (`Exception.class`)|
|**数量**|每次只能抛出**一个**|可以声明**多个**，用逗号隔开|
|**功能**|**真正执行**抛出异常的操作，中断程序流程|**声明**一种异常可能性，将处理责任转嫁给调用者|
