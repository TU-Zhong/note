好的，这是一个非常核心且重要的Java基础问题。我们来深入探讨一下 `hashCode()` 和 `equals()` 方法之间的关系，以及为什么必须同时重写它们。

### 打个比方：图书馆找书

为了方便理解，我们先用一个生活中的例子来类比：

- **`hashCode()` 就像是书的分类号（例如 "TP312JA"）。** 它能让你快速定位到书架的某个区域。你不需要看遍整个图书馆，只需要去对应的区域找书，大大提高了效率。
    
- **`equals()` 就像是逐一比对书名、作者、出版社等信息。** 当你根据分类号找到对应的区域后，那个区域可能有多本书。这时你就需要一本一本地比对，确认是不是你想要的那本。
    

### 1. `equals()` 方法是什么？

`equals()` 方法定义在 `java.lang.Object` 类中，它的作用是**判断两个对象在逻辑上是否相等**。

- **默认实现**：在 `Object` 类中，`equals()` 的默认行为是比较两个对象的内存地址，等同于 `==` 操作符。只有当两个引用指向同一个对象时，才会返回 `true`。
    
- **重写目的**：通常，我们需要根据对象的内部状态（即成员变量）来判断它们是否相等。例如，两个 `Person` 对象，即使它们是内存中两个独立的对象，但如果它们的身份证号相同，我们就认为它们在逻辑上是“相等”的。这时就需要重写 `equals()` 方法。
    

Java

```java
public class Person {
    private String idCard;
    private String name;

    // ... 构造函数、getter/setter省略 ...

    @Override
    public boolean equals(Object o) {
        // 1. 检查是否为同一对象引用
        if (this == o) return true;
        // 2. 检查对象是否为null或类型不匹配
        if (o == null || getClass() != o.getClass()) return false;
        // 3. 类型转换并比较核心字段
        Person person = (Person) o;
        return idCard != null ? idCard.equals(person.idCard) : person.idCard == null;
    }
}
```

### 2. `hashCode()` 方法是什么？

`hashCode()` 方法也定义在 `java.lang.Object` 类中，它的作用是返回该对象的一个**哈希码（hash code）**，它是一个 `int` 类型的整数。

- **主要用途**：这个哈希码主要用于提升哈希表（Hash-based collections）的性能，例如 `HashMap`, `HashSet`, `Hashtable`。
    
- **工作原理**：当你向 `HashSet` 中添加一个对象时，它会首先调用该对象的 `hashCode()` 方法得到一个哈希码，这个哈希码决定了对象在集合内部存储的位置（即“桶”的位置）。这样，当需要查找该对象时，可以直接根据哈希码定位到那个“桶”，而不需要遍历整个集合，从而实现近乎 O(1) 的查找效率。
    

### 3. 两者之间至关重要的“约定”

Java官方对 `hashCode()` 和 `equals()` 的实现制定了一个非常严格的**约定（Contract）**：

1. **如果两个对象通过 `equals()` 方法比较是相等的，那么它们的 `hashCode()` 方法必须返回相同的值。** ( `a.equals(b)` 为 `true` implies `a.hashCode() == b.hashCode()` )
    
2. **如果两个对象通过 `equals()` 方法比较是不相等的，那么它们的 `hashCode()` 方法不必返回不同的值。** 但为了性能，最好让不相等的对象产生不同的哈希码，以减少“哈希冲突”（hash collisions）。
    
3. 在一个Java应用的执行期间，只要一个对象用于 `equals()` 比较的信息没有被修改，那么无论调用多少次 `hashCode()`，都必须始终返回同一个整数。
    

**核心就是第一条：相等的对象必须有相等的哈希码。**

### 4. 为什么重写 `equals()` 时必须重写 `hashCode()`？

**答案是：为了遵守约定，保证哈希表等集合能够正常工作。**

如果我们违反了这个约定，只重写 `equals()` 而不重写 `hashCode()`，就会导致灾难性的后果。

让我们用前面那个 `Person` 类的例子来说明：

假设我们有两个 `Person` 对象，它们的 `idCard` 相同，但它们是两个不同的实例。

Java

```
Person p1 = new Person("12345", "张三");
Person p2 = new Person("12345", "张三");

// 根据我们重写的 equals 方法，它们是相等的
System.out.println(p1.equals(p2)); // 输出: true
```

**场景：只重写了 `equals()`，没有重写 `hashCode()`**

此时，`p1` 和 `p2` 会使用 `Object` 类默认的 `hashCode()` 方法。这个方法通常是根据对象的内存地址来计算的，所以 `p1` 和 `p2` 的哈希码几乎肯定是不一样的。

Java

```
System.out.println(p1.hashCode()); // 例如输出: 1829164700
System.out.println(p2.hashCode()); // 例如输出: 2018699554
```

现在，我们把 `p1` 放进一个 `HashSet` 里，然后尝试判断 `p2` 是否存在于这个集合中。

Java

```
Set<Person> set = new HashSet<>();
set.add(p1);

System.out.println(set.contains(p2)); // 输出: false (错误！)
```

**为什么会得到错误的结果 `false`？**

1. **`set.add(p1)`**：`HashSet` 计算 `p1.hashCode()` (例如 1829164700)，然后把 `p1` 存放在这个哈希码对应的“桶”里。
    
2. **`set.contains(p2)`**：`HashSet` 要判断 `p2` 是否存在。它首先计算 `p2.hashCode()` (例如 2018699554)。
    
3. `HashSet` 发现 `p2` 的哈希码对应的“桶”是空的，或者里面没有与 `p2` `equals()` 的对象。它**根本不会**去 `p1` 所在的那个桶里进行 `equals()` 比较。
    
4. 因此，`HashSet` 错误地认为 `p2` 不存在于集合中，尽管从逻辑上讲 (`p1.equals(p2)`) 它应该存在。
    

**正确的做法：同时重写 `hashCode()`**

我们需要保证当 `p1.equals(p2)` 为 `true` 时，`p1.hashCode() == p2.hashCode()` 也为 `true`。

Java

```
public class Person {
    private String idCard;
    private String name;

    // ... equals 方法同上 ...

    @Override
    public int hashCode() {
        // 保证当 idCard 相同时，返回的 hashCode 也相同
        return idCard != null ? idCard.hashCode() : 0;
        // 更健壮的写法是使用 Objects.hash()
        // return java.util.Objects.hash(idCard, name);
    }
}
```

现在再运行上面的代码：

Java

```
Person p1 = new Person("12345", "张三");
Person p2 = new Person("12345", "张三");

System.out.println(p1.equals(p2));     // 输出: true
System.out.println(p1.hashCode());    // 输出: 48691       (例如)
System.out.println(p2.hashCode());    // 输出: 48691       (与p1相同)

Set<Person> set = new HashSet<>();
set.add(p1);

System.out.println(set.contains(p2)); // 输出: true (正确！)
```

**这次为什么正确了？**

1. **`set.add(p1)`**：计算 `p1.hashCode()` (48691)，将 `p1` 存入对应桶。
    
2. **`set.contains(p2)`**：计算 `p2.hashCode()` (48691)，**找到了与 `p1` 相同的桶**。
    
3. `HashSet` 发现这个桶里已经有对象 (`p1`)，于是它会进一步调用 `equals()` 方法进行精确比较 (`p2.equals(p1)`)。
    
4. `equals()` 方法返回 `true`，因此 `HashSet` 正确地判断出 `p2` 已经存在于集合中。
    

### 总结

|方法|作用|何时使用|
|---|---|---|
|**`equals()`**|判断两个对象在**逻辑上是否相等**。|在需要比较对象内容而不是内存地址时。|
|**`hashCode()`**|返回一个整数哈希码，用于**快速定位**对象在哈希表中的存储位置。|主要由 `HashMap`, `HashSet` 等哈希集合内部使用，以提高性能。|

关系核心：

hashCode() 是为了提高 equals() 在哈希集合中的执行效率而存在的。两者必须协同工作，遵守“相等的对象必须有相等的哈希码”这一核心约定。当你只重写 equals() 时，就破坏了这个约定，导致所有依赖哈希算法的类都无法正常工作。
