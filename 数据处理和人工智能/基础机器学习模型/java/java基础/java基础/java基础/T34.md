早上好！在这个台北的周一早晨，我们来谈谈如何在 Java 中格式化日期。这是一个非常常见且重要的需求。

Java 中格式化日期主要有两种方式：

1. **现代方法 (Java 8+)**：使用 `java.time` 包中的 `DateTimeFormatter`。**这是目前强烈推荐的最佳实践**。
    
2. **传统方法 (Java 8 之前)**：使用 `java.text` 包中的 `SimpleDateFormat`。这种方式存在一些问题，**不推荐在新代码中使用**。
    

下面我们分别来详细说明。

---

### 1. 现代方法 (Java 8+)：`java.time.format.DateTimeFormatter` (强烈推荐)

自 Java 8 引入全新的日期时间 API (`java.time`，也称为 JSR-310) 后，日期格式化变得既简单又安全。

- **核心类**：`java.time.format.DateTimeFormatter`
    
- **与之配合的日期时间类**：`LocalDateTime` (本地日期时间), `ZonedDateTime` (带时区的日期时间), `LocalDate` (仅日期), `LocalTime` (仅时间)。
    
- **最大优点**：`DateTimeFormatter` 是**不可变的（Immutable）**且**线程安全（Thread-safe）**的，你可以放心地在多线程环境中共享同一个实例。
    

#### 操作步骤：

1. **获取一个日期时间对象**：例如，获取台北的当前时间。
    
2. **创建一个格式化器 `DateTimeFormatter`**：可以通过预定义的格式，或自定义的模式来创建。
    
3. **使用日期时间对象的 `.format()` 方法执行格式化**。
    

#### 代码示例：

Java

```java
import java.time.ZonedDateTime;
import java.time.ZoneId;
import java.time.format.DateTimeFormatter;
import java.util.Locale;

public class NewDateFormatterExample {
    public static void main(String[] args) {
        // 1. 获取台北时区的当前日期和时间
        ZonedDateTime nowInTaipei = ZonedDateTime.now(ZoneId.of("Asia/Taipei"));
        System.out.println("原始 ZonedDateTime 对象: " + nowInTaipei);

        // 2. 创建不同模式的格式化器
        // 模式 A: 常见的年月日 时:分:秒
        DateTimeFormatter formatter1 = DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss");

        // 模式 B: 包含星期的详细中文格式 (使用 Locale.CHINA 确保简体中文输出)
        DateTimeFormatter formatter2 = DateTimeFormatter.ofPattern(
            "yyyy年MM月dd日 EEEE a hh:mm:ss z", Locale.CHINA
        );

        // 模式 C: 使用预定义的 ISO 标准格式
        DateTimeFormatter formatter3 = DateTimeFormatter.ISO_OFFSET_DATE_TIME;

        // 3. 执行格式化
        String formattedString1 = nowInTaipei.format(formatter1);
        String formattedString2 = nowInTaipei.format(formatter2);
        String formattedString3 = nowInTaipei.format(formatter3);

        // 打印结果
        System.out.println("格式 A (yyyy-MM-dd HH:mm:ss): " + formattedString1);
        System.out.println("格式 B (中国本地化详细格式): " + formattedString2);
        System.out.println("格式 C (ISO 标准格式): " + formattedString3);
    }
}
```

**预期输出 (基于当前时间):**

```
原始 ZonedDateTime 对象: 2025-09-15T08:29:04.123456+08:00[Asia/Taipei]
格式 A (yyyy-MM-dd HH:mm:ss): 2025-09-15 08:29:04
格式 B (中国本地化详细格式): 2025年09月15日 星期一 上午 08:29:04 CST
格式 C (ISO 标准格式): 2025-09-15T08:29:04.123456+08:00
```

#### 常见格式化模式字母：

|字母|描述|示例|
|---|---|---|
|`y`|年|`yyyy` -> `2025`|
|`M`|月|`MM` -> `09`, `MMM` -> `九月`|
|`d`|日|`dd` -> `15`|
|`E`|星期|`E` -> `周一`, `EEEE` -> `星期一`|
|`H`|小时 (0-23)|`HH` -> `08`|
|`h`|小时 (1-12)|`hh` -> `08`|
|`m`|分钟|`mm` -> `29`|
|`s`|秒|`ss` -> `04`|
|`a`|上午/下午|`a` -> `上午`|
|`z`|时区简称|`z` -> `CST`|

### 2. 传统方法 (Java 8 之前)：`java.text.SimpleDateFormat` (不推荐)

这是旧版 Java 中格式化日期的方式，主要与 `java.util.Date` 类配合使用。

- **核心类**：`java.text.SimpleDateFormat`
    
- **最大缺点**：`SimpleDateFormat` 是**非线程安全的（Not Thread-safe）**。在多线程环境中共享同一个实例会导致不可预知的错误，这是非常严重的问题。此外，`java.util.Date` 本身的 API 设计也比较混乱（例如月份是 0-11）。
    

#### 代码示例：

Java

```
import java.text.SimpleDateFormat;
import java.util.Date;
import java.util.Locale;

public class OldDateFormatExample {
    public static void main(String[] args) {
        // 1. 获取当前时间的 Date 对象
        Date now = new Date();
        
        // 2. 创建格式化器
        SimpleDateFormat sdf = new SimpleDateFormat(
            "yyyy年MM月dd日 EEEE a hh:mm:ss", Locale.CHINA
        );

        // 3. 执行格式化
        String formattedString = sdf.format(now);
        
        System.out.println("传统方式格式化结果: " + formattedString);
    }
}
```

**预期输出 (基于当前时间):**

```
传统方式格式化结果: 2025年09月15日 星期一 上午 08:29:04
```

虽然能得到相同的结果，但由于其固有的线程安全问题，应在**新代码中极力避免使用**。了解它主要是为了维护旧的项目代码。

---

### 总结与建议

|特性|`DateTimeFormatter` (Java 8+)|`SimpleDateFormat` (旧版)|
|---|---|---|
|**线程安全**|**是** (不可变，线程安全)|**否** (非线程安全，有严重风险)|
|**API 设计**|清晰、功能强大、面向对象|混乱、易出错 (如月份从0开始)|
|**不变性**|`java.time` 对象都是不可变的|`java.util.Date` 对象是可变的|
|**推荐程度**|**强烈推荐，现代 Java 标准**|**强烈不推荐，仅用于维护旧代码**|

**结论**：在所有**新的 Java 项目（Java 8 及以上）中，都应该始终优先使用 `java.time` 包及其 `DateTimeFormatter`** 来处理所有日期和时间的格式化需求。它更安全、更强大、也更易于使用。
