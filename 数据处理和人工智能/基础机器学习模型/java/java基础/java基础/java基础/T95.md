下午好！在台北的周四下午，我们来深入探讨 `ArrayList` 的一个核心内部机制——**扩容（Resizing）**。

这是一个在面试中经常被问到的问题，因为它能很好地考察面试者对 `ArrayList` 底层工作原理的理解。

简单来说，`ArrayList` 的扩容机制就像是：

> 当你往一个**“只能装10个苹果的篮子”**里放第11个苹果时，你发现放不下了。于是，你找来一个**“能装15个苹果的大篮子”**，把原来篮子里的10个苹果全部搬到大篮子里，然后再把第11个苹果放进去。

---

### 1. 为什么需要扩容？—— `ArrayList` 的本质

要理解扩容，首先要明白 `ArrayList` 的本质。`ArrayList` 的底层数据结构就是一个**普通的 Java 数组**。在 `ArrayList` 源码中，这个数组被定义为：

Java

```
transient Object[] elementData;
```

我们知道，Java 中的**数组长度是固定的**，一旦被创建，其大小就不能再改变。

但是 `ArrayList` 又是一个“动态数组”，可以让我们无限地 `add()` 元素。为了实现这种“动态”的假象，当内部的 `elementData` 数组被填满时，`ArrayList` 就必须进行**扩容**，即创建一个更大的新数组来替代旧数组。

---

### 2. 扩容的关键步骤与时机

#### 触发时机

扩容主要发生在调用 `add()` 或 `addAll()` 等添加元素的方法时。在添加元素之前，`ArrayList` 会进行一次容量检查，判断当前数组是否还有空间。如果 `size`（当前元素数量）即将等于 `elementData.length`（数组容量），就会触发扩容。

#### 核心扩容逻辑

扩容的核心逻辑在 `ArrayList` 的 `grow()` 方法中，可以概括为以下四步：

1. 计算新容量
    
    ArrayList 并不是每次只增加一个空间。为了摊销成本，它会一次性地增加较多的容量。
    
    新容量的计算规则是：
    
    > **新容量 = 旧容量 + (旧容量 / 2)**
    
    也就是将容量增加到原来的 **1.5 倍**。在源码中，这是通过位运算 `(oldCapacity >> 1)` 来高效实现的，`>> 1` 相当于除以2。
    
2. 创建新数组
    
    根据计算出的新容量，使用 new Object[newCapacity] 创建一个新的、更大的数组。
    
3. 复制旧数据
    
    使用 Arrays.copyOf() 或 System.arraycopy() 方法，将旧数组（elementData）中的所有元素原封不动地复制到新创建的大数组中。
    
4. 引用指向新数组
    
    让 ArrayList 内部的 elementData 引用指向这个新的、更大的数组。旧的小数组由于不再被任何引用指向，会在未来的某个时刻被 Java 的垃圾回收器（GC）回收。
    

---

### 3. 两个特殊情况：初始容量与第一次添加

1. 无参构造方法 (new ArrayList())
    
    当你使用 new ArrayList() 创建一个实例时，为了节省内存，Java 采用了懒加载（Lazy Initialization）的策略。此时，elementData 数组并不会被立即创建，它只是一个空的占位数组，容量为 0。
    
2. 第一次 add()
    
    当你第一次向这个空的 ArrayList 中添加元素时，程序发现容量（0）不足。此时，它并不会按 0 * 1.5 来计算新容量，而是会直接将容量扩充到一个默认值，这个值通常是 10。
    

**所以，一个无参构造的 `ArrayList` 的容量变化路径是：0 -> 10 -> 15 -> 22 -> 33 -> ...**

---

### 4. 性能影响与最佳实践

性能影响

扩容操作，特别是第三步的数组复制（Arrays.copyOf()），是一个相对耗时的操作。它的时间复杂度是 O(n)，其中 n 是当前元素的数量。如果一个 ArrayList 中已经有百万个元素，那么每一次扩容都需要复制这百万个元素，会带来明显的性能开销。

最佳实践

为了避免这种不必要的性能损耗，一个非常重要的最佳实践是：

> **在预知将要存储大量元素的情况下，在创建 `ArrayList` 时就指定一个合理的初始容量。**

Java

```
// 不推荐：如果需要添加 10000 个元素，会触发多次耗时的扩容
List<String> list1 = new ArrayList<>(); 

// 推荐：直接创建一个容量为 10000 的数组，避免了中间的扩容过程
List<String> list2 = new ArrayList<>(10000); 
```

通过在构造方法中传入一个初始容量，`ArrayList` 会直接创建一个足够大的内部数组，从而避免了在后续 `add()` 过程中因多次扩容而带来的性能开销。这是一个简单而有效的优化手段。

### 总结

- `ArrayList` 的底层是**固定长度的数组**。
    
- 当数组满时，调用 `add()` 会触发**扩容**。
    
- 扩容的核心是创建一个**1.5 倍**于旧容量的新数组，并**复制**所有旧元素过去。
    
- 使用无参构造方法创建的 `ArrayList`，在第一次添加元素时，容量会从 **0 扩展到 10**。
    
- **最佳实践**是在已知元素数量时，**预设初始容量**，以避免不必要的性能开销。
