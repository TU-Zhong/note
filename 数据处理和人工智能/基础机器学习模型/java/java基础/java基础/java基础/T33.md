这是一个非常经典的 Java 面试题，很多开发者会直觉地回答 `-2`，但正确答案其实是 **-1**。

---

### 正确答案

在 Java 中，`Math.round(-1.5)` 的计算结果是 **-1**。

### 为什么是 -1？

要理解这个结果，我们需要知道 `Math.round()` 方法的**运算规则**。

根据 Java 官方文档，对于 `double` 类型的参数 `a`，`Math.round(a)` 的计算方法等同于：

Java

```
(long) Math.floor(a + 0.5)
```

现在，我们把 `-1.5` 带入这个公式来一步步计算：

1. 参数 a 加上 0.5:
    
    a + 0.5 => -1.5 + 0.5 = -1.0
    
2. 对结果进行 Math.floor() 操作:
    
    Math.floor() 的作用是“向下取整”，即返回不大于参数的最大整数。
    
    Math.floor(-1.0) 的结果是 -1.0。
    
3. 将结果强制转换为 long 类型:
    
    (long) -1.0 的结果是 -1。
    

所以，`Math.round(-1.5)` 的最终结果是 **-1**。

### 举例对比

为了加深理解，我们可以看几个相关的例子，它们都遵循 `(long)Math.floor(a + 0.5)` 这个规则：

|表达式|计算过程 (a + 0.5)|`Math.floor()` 结果|最终结果|备注|
|---|---|---|---|---|
|`Math.round(1.5)`|`1.5 + 0.5` = `2.0`|`2.0`|**`2`**|正数，四舍五入中的“五入”|
|`Math.round(1.4)`|`1.4 + 0.5` = `1.9`|`1.0`|**`1`**|正数，四舍五入中的“四舍”|
|`Math.round(-1.4)`|`-1.4 + 0.5` = `-0.9`|`-1.0`|**`-1`**|负数，四舍五入中的“四舍”|
|`Math.round(-1.5)`|`-1.5 + 0.5` = `-1.0`|`-1.0`|**`-1`**|**负数，`.5` 的情况，向正无穷方向取整**|
|`Math.round(-1.6)`|`-1.6 + 0.5` = `-1.1`|`-2.0`|**`-2`**|负数，四舍五入中的“五入”|

### 总结记忆

`Math.round()` 的取整规则可以简单总结为：**在数轴上，将参数加上 0.5，然后向左（负无穷方向）取最近的整数。**

- 对于**正数**，这个规则等同于我们日常理解的“四舍五入”。
    
- 对于**负数**，这个规则导致 `.5` 的情况会向**正无穷**方向取整（即向 0 的方向取整）。所以 `-1.5` 会变成 `-1`，`-2.5` 会变成 `-2`。
