晚安！我们来聊聊那些在开发中最常“偶遇”的运行时异常（RuntimeException）。

理解这些常见的运行时异常对于编写健壮、可靠的 Java 代码至关重要。它们的共同点是都继承自 `RuntimeException` 类，编译器不强制我们处理它们，但它们的出现通常都意味着代码中存在需要修正的**逻辑错误（Bug）**。

下面是一些最常见的运行时异常，以及它们的触发原因和示例。

---

### 1. `NullPointerException` (空指针异常)

这是 Java 开发者最常遇到的“老朋友”，通常被戏称为“NPE”。

- **发生原因**：当一个变量的引用为 `null` 时，你尝试使用这个引用去调用方法或访问其字段。
    
- **代码示例**：
    
    Java
    
    ```
    public static void main(String[] args) {
        String name = null;
        // 尝试在一个 null 引用上调用 .length() 方法
        System.out.println(name.length()); // 抛出 NullPointerException
    }
    ```
    
- **如何避免**：在调用对象的方法前，进行 `null` 检查。
    

### 2. `ArrayIndexOutOfBoundsException` (数组索引越界异常)

- **发生原因**：访问数组时，使用的索引（下标）超出了数组的有效范围（有效范围是 `0` 到 `数组长度 - 1`）。
    
- **代码示例**：
    
    Java
    
    ```
    public static void main(String[] args) {
        int[] numbers = new int[3]; // 数组的有效索引是 0, 1, 2
        // 尝试访问索引为 3 的元素
        numbers[3] = 42; // 抛出 ArrayIndexOutOfBoundsException
    }
    ```
    
- **如何避免**：在访问数组前，确保索引在有效范围内。
    

### 3. `IllegalArgumentException` (非法参数异常)

- **发生原因**：向方法传递了一个不合法或不符合预期的参数。这个异常通常是由程序员**主动抛出**的，用来进行参数校验。
    
- **代码示例**：
    
    Java
    
    ```
    public static void setAge(int age) {
        if (age < 0 || age > 150) {
            // 主动抛出异常，表示传入的 age 参数不合法
            throw new IllegalArgumentException("年龄必须在 0 到 150 之间");
        }
        System.out.println("年龄设置为: " + age);
    }
    
    public static void main(String[] args) {
        setAge(-5); // 调用此方法会抛出 IllegalArgumentException
    }
    ```
    

### 4. `NumberFormatException` (数字格式化异常)

这是 `IllegalArgumentException` 的一个常见子类。

- **发生原因**：当试图将一个**非数字格式的字符串**转换为数字类型时。
    
- **代码示例**：
    
    Java
    
    ```
    public static void main(String[] args) {
        String notANumber = "abc";
        // 尝试将 "abc" 转换为整数
        int number = Integer.parseInt(notANumber); // 抛出 NumberFormatException
    }
    ```
    

### 5. `ClassCastException` (类转换异常)

- **发生原因**：试图将一个对象强制转换为它并非其实例的类型。
    
- **代码示例**：
    
    Java
    
    ```
    public static void main(String[] args) {
        Object obj = 123; // 这是一个 Integer 对象
        // 尝试将 Integer 对象强制转换为 String 类型
        String str = (String) obj; // 抛出 ClassCastException
    }
    ```
    
- **如何避免**：在强制类型转换前，使用 `instanceof` 运算符进行检查。
    

### 6. `ArithmeticException` (算术异常)

- **发生原因**：出现异常的算术条件。最常见的情况是**整数除以零**。
    
- **代码示例**：
    
    Java
    
    ```
    public static void main(String[] args) {
        int result = 10 / 0; // 抛出 ArithmeticException: / by zero
    }
    ```
    

### 7. `ConcurrentModificationException` (并发修改异常)

在操作集合时非常常见。

- **发生原因**：当使用迭代器（Iterator）或增强 for 循环遍历一个集合时，如果**通过集合自身的方法（而不是迭代器的方法）** 对集合的结构进行了修改（如添加或删除元素），就会抛出此异常。这是一种“快速失败”（fail-fast）机制。
    
- **代码示例**：
    
    Java
    
    ```
    import java.util.ArrayList;
    import java.util.List;
    
    public static void main(String[] args) {
        List<String> list = new ArrayList<>();
        list.add("A");
        list.add("B");
        list.add("C");
    
        // 在使用增强 for 循环（其底层是迭代器）遍历时，尝试删除元素
        for (String item : list) {
            if ("B".equals(item)) {
                list.remove(item); // 抛出 ConcurrentModificationException
            }
        }
    }
    ```
    
- **如何避免**：使用迭代器自身的 `remove()` 方法，或者使用 Java 8 的 `removeIf()` 方法。
    

### 总结与建议

遇到运行时异常，我们的第一反应不应该是简单地用 `try-catch` 把它“包”起来让程序继续运行。

**正确的处理方式是**：将这些异常的出现视为**代码中存在 Bug 的信号**。我们应该通过**防御性编程**来避免它们，例如：

- **添加 `null` 检查**来预防 `NullPointerException`。
    
- **检查数组边界**来预防 `ArrayIndexOutOfBoundsException`。
    
- **校验方法参数**来预防 `IllegalArgumentException`。
    
- **使用 `instanceof`** 来预防 `ClassCastException`。
    

修复代码逻辑，从根本上杜绝这些异常的发生，是编写高质量 Java 代码的关键。
