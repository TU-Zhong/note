`@annotation(LogExecutionTime)` 是 Spring AOP 中的一种**切入点表达式（Pointcut Expression）**。

简单来说，它的意思是：**“我要拦截所有加了 `@LogExecutionTime` 这个注解的方法。”**

这里的 `LogExecutionTime` 通常是你（或你的同事）自定义的一个注解。只要在任何业务方法上加上 `@LogExecutionTime`，AOP 就会自动拦截它并执行你写的日志逻辑。这种方式非常灵活且具有侵入性低的特点（想切哪个方法，加个注解就行）。

---

### Spring AOP 还有哪些“切入点表达式”的用法？

除了 `@annotation`，Spring AOP 还有很多种定义“该拦截谁”的规则。可以将它们理解为强大的“过滤器”。以下是最常用的几种：

#### 1. `execution(...)`：最强大、最常用的表达式

它可以精确匹配到方法的**返回值、包名、类名、方法名和参数类型**。

- **语法格式：** `execution(返回值类型 包名.类名.方法名(参数))`
    
- **常用示例：**
    
    - `execution(* com.example.service.*.*(..))`
        
        _含义：_ 拦截 `com.example.service` 包下**所有类**的**所有方法**。
        
        _解释：_ 第一个 `*` 代表任何返回值类型；第二个 `*` 代表该包下的所有类；第三个 `*` 代表所有方法名；`(..)` 代表任意数量和类型的参数。
        
    - `execution(public String com.example.UserService.find*(..))`
        
        _含义：_ 拦截 `UserService` 类中，所有以 `find` 开头、且返回值是 `String` 的 `public` 方法。
        

#### 2. `within(...)`：按“类”或“包”的范围匹配

如果你不想精确到方法，只想圈定一个范围，用 `within` 会更简洁。

- **常用示例：**
    
    - `within(com.example.service.*)`
        
        _含义：_ 拦截 `com.example.service` 包下**所有类中的所有方法**（不包括子包）。
        
    - `within(com.example.service..*)`
        
        _含义：_ 拦截 `com.example.service` 包及**其所有子包**下的所有类的所有方法（注意这里的 `..` 代表包含子包）。
        

#### 3. `@within(...)`：按“类上的注解”匹配

不要和 `@annotation` 搞混了：

- `@annotation` 是看**方法**上有没有注解。
    
- `@within` 是看**类**上有没有注解。
    
- **常用示例：**
    
    - `@within(org.springframework.web.bind.annotation.RestController)`
        
        _含义：_ 拦截所有标注了 `@RestController` 的**类里面的所有方法**（常用于全局拦截 Controller 层）。
        

#### 4. `args(...)`：按“方法参数”匹配

专门用来匹配方法的入参类型，不关心它在哪个包、叫什么名字。

- **常用示例：**
    
    - `args(java.lang.String, ..)`
        
        _含义：_ 拦截所有**第一个参数是 String 类型**的方法（后面的参数随意）。
        

#### 5. 逻辑运算符组合（&&、||、!）

你可以把上面的这些规则组合起来，实现极其精细的控制。

- **常用示例：**
    
    - **与（&&）：** `execution(* com.example..*.*(..)) && @annotation(LogExecutionTime)`
        
        _含义：_ 必须是在 `com.example` 包（或其子包）下，**并且**方法上加了 `@LogExecutionTime` 注解的才会被拦截。
        
    - **非（!）：** `execution(* com.example.service.*.*(..)) && !execution(* com.example.service.*.get*(..))`
        
        _含义：_ 拦截 `service` 包下的所有方法，**但是排除掉**所有以 `get` 开头的方法。
        

### 总结建议

在实际项目开发中，最普遍的做法是两种：

1. **全局拦截：** 使用 `execution` 表达式，直接切入整个 Service 层或 Controller 层（例如用于全局异常处理或统一日志）。
    
2. **按需拦截（你现在的写法）：** 自定义一个注解，然后使用 `@annotation`。这种方式最清晰，别人看代码时，看到方法上有注解，就立刻知道这个方法被 AOP 增强了。
