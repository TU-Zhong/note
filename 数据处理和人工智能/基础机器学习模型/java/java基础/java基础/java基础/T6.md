好的，`String`、`StringBuffer` 和 `StringBuilder` 是 Java 中处理字符串时非常重要的三个类。它们之间的区别主要体现在**可变性**、**线程安全性**和**性能**这三个核心方面。

### 核心区别概览

- **`String`**：**不可变的**（Immutable）字符序列。
    
- **`StringBuffer`**：**可变的**（Mutable）字符序列，**线程安全**。
    
- **`StringBuilder`**：**可变的**（Mutable）字符序列，**线程不安全**。
    

下面我们来详细地逐一对比。

---

### 1. 可变性 (Mutability)

这是 `String` 与另外两个类的最根本区别。

- String (不可变)
    
    一旦一个 String 对象被创建，它内部的字符序列就不能再被改变。所有对 String 对象看似修改的操作（如 + 拼接、substring()、replace() 等）实际上都会创建一个全新的 String 对象，而原始对象保持不变。
    
    Java
    
    ```
    String s = "Hello";
    // s.concat(" World") 会创建一个新的对象 "Hello World"
    // 原来的 "Hello" 对象并没有改变，只是 s 引用指向了新对象
    s = s.concat(" World"); 
    ```
    
    **缺点**：在循环中或大量进行字符串拼接时，会频繁创建新对象，不仅占用大量内存空间，还会引发频繁的垃圾回收（GC），导致性能低下。
    
- StringBuffer 和 StringBuilder (可变)
    
    这两个类的对象允许直接在原始对象上进行修改（追加、插入、删除等），而不会创建新的对象。它们内部维护一个可变的字符数组，当空间不足时会自动扩容。
    
    Java
    
    ```
    StringBuilder sb = new StringBuilder("Hello");
    // .append() 方法直接在 sb 对象本身上进行修改
    sb.append(" World"); // sb 的内容变为 "Hello World"，没有创建新对象
    ```
    

### 2. 线程安全性 (Thread Safety)

这是 `StringBuffer` 和 `StringBuilder` 之间的主要区别。

- String
    
    因为 String 对象是不可变的，所以它的状态不会被任何线程改变。因此，String 对象是天生线程安全的，可以在多线程环境中安全地共享。
    
- StringBuffer
    
    StringBuffer 是线程安全的。它的几乎所有公开方法（如 append()、insert()）都使用了 synchronized 关键字进行同步。这意味着在同一时刻，只允许一个线程操作 StringBuffer 对象。
    
    优点：在多线程环境下，可以保证数据的同步和一致性。
    
    缺点：同步操作会带来额外的性能开销（加锁和解锁）。
    
- StringBuilder
    
    StringBuilder 是线程不安全的。它的方法没有进行同步处理。
    
    优点：由于没有同步开销，其性能要远高于 StringBuffer。
    
    缺点：在多线程环境下，如果多个线程同时操作同一个 StringBuilder 对象，可能会导致数据错乱。
    

### 3. 性能 (Performance)

性能差异主要源于它们的可变性和线程安全性。

- 性能排序：
    
    在单线程环境下进行大量字符串修改操作时，性能从高到低依次是：
    
    StringBuilder > StringBuffer > String
    
- **原因分析**：
    
    - **`String`**：最慢，因为它每次修改都会创建新对象。
        
    - **`StringBuffer`**：比 `String` 快很多，因为它在原对象上操作。但由于方法同步，相比 `StringBuilder` 有性能损耗。
        
    - **`StringBuilder`**：最快，因为它既在原对象上操作，又没有同步开销。它是 `StringBuffer` 的一个非同步版本，是 Java 5 中引入的。
        

---

### 总结与如何选择

为了方便记忆和选择，这里有一个清晰的总结表格：

|特性|`String`|`StringBuffer`|`StringBuilder`|
|---|---|---|---|
|**可变性**|否 (Immutable)|是 (Mutable)|是 (Mutable)|
|**线程安全性**|安全|**安全** (方法同步)|**不安全** (无同步)|
|**性能**|低 (适用于少量修改)|中|**高**|
|**继承关系**|直接继承 `Object`|继承 `AbstractStringBuilder`|继承 `AbstractStringBuilder`|

### 选择指南：

1. **优先使用 `String`**：
    
    - 如果字符串内容基本不发生改变，或者操作非常少。
        
    - 例如：常量、配置信息、类的属性等。
        
    - `String` 是最常用的，因为它的不可变性保证了安全和简洁。
        
2. **单线程下大量拼接或修改，使用 `StringBuilder`**：
    
    - 在单线程环境中，需要对字符串进行频繁的追加、修改操作。
        
    - 例如：在循环中拼接SQL语句、构造JSON或XML字符串。
        
    - **这是 `StringBuilder` 最典型的应用场景，也是官方推荐的。**
        
3. **多线程下共享并修改字符串，使用 `StringBuffer`**：
    
    - 在多线程环境中，有一个共享的字符串变量需要被多个线程同时修改。
        
    - 例如：一个被多个线程共享的日志记录器缓冲区。
        
    - **注意**：这种情况在实际开发中相对较少，大多数时候可以通过其他方式避免共享可变状态。
        

**简单来说，记住这个法则：**

- **少量操作用 `String`。**
    
- **单线程循环拼接用 `StringBuilder`。**
    
- **多线程共享修改用 `StringBuffer`。**
