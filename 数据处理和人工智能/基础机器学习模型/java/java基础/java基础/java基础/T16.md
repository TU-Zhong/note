好的，深拷贝（Deep Copy）和浅拷贝（Shallow Copy）是编程中处理对象复制时非常重要的两个概念，尤其是在像 Java 这样拥有引用类型的语言中。它们的核心区别在于**如何处理对象内部的引用类型成员**。

### 打个比方：复印文件夹

为了方便理解，我们先用一个比喻：

- 你有一个文件夹 `A`（原始对象）。
    
- 文件夹里有一份普通文件 `doc.txt`（**基本数据类型**）。
    
- 文件夹里还有一个指向另一个文件夹 `B` 的**快捷方式**（**引用数据类型**）。
    

浅拷贝 (Shallow Copy)：

你创建了一个新的文件夹 A'，然后：

1. 你复印了一份 `doc.txt` 放进 `A'`。
    
2. 你只复制了那个快捷方式，也放进了 A'。
    
    结果：现在 A 和 A' 里的快捷方式都指向同一个文件夹 B。如果你通过 A' 的快捷方式去修改了文件夹 B 里的内容，那么通过 A 的快捷方式也会看到这个修改。这就是“牵一发而动全身”。
    

深拷贝 (Deep Copy)：

你创建了一个新的文件夹 A'，然后：

1. 你复印了一份 `doc.txt` 放进 `A'`。
    
2. 你顺着快捷方式找到了文件夹 B，把整个文件夹 B 也完整地复制了一份，得到 B'，然后在新文件夹 A' 里创建了一个指向 B' 的新快捷方式。
    
    结果：现在 A 指向 B，A' 指向 B'。B 和 B' 是两个完全独立的文件夹。你修改 B 不会影响 B'，反之亦然。两者完全隔离。
    

---

### 1. 浅拷贝 (Shallow Copy)

#### 定义

创建一个新对象，然后将原始对象中的字段值**原封不动地**复制到新对象中。

- **对于基本数据类型**（如 `int`, `double`, `boolean`）：直接复制其**值**。
    
- **对于引用数据类型**（如 `String`, `Array`, 自定义对象）：只复制其**引用地址**，而不是复制引用所指向的实际对象。
    

#### 结果

原始对象和拷贝后的对象内部的引用类型成员指向的是**同一个**堆内存中的对象。

#### 如何实现？

方法一：重写 Object.clone() 方法（默认实现）

Java 中 Object 类的 clone() 方法默认执行的就是浅拷贝。

1. 类需要实现 `Cloneable` 接口（这是一个标记接口，表示该类可以被克隆）。
    
2. 重写 `clone()` 方法，并将其访问权限提升为 `public`。
    

**代码示例：**

Java

```
// 地址类
class Address {
    public String city;
    public Address(String city) { this.city = city; }
}

// 用户类，实现了 Cloneable 接口
class User implements Cloneable {
    public int age; // 基本类型
    public Address address; // 引用类型

    public User(int age, Address address) {
        this.age = age;
        this.address = address;
    }

    @Override
    public User clone() throws CloneNotSupportedException {
        // 调用 Object.clone() 执行浅拷贝
        return (User) super.clone();
    }
}

public class ShallowCopyTest {
    public static void main(String[] args) throws CloneNotSupportedException {
        Address address = new Address("北京");
        User originalUser = new User(30, address);

        // 进行浅拷贝
        User clonedUser = originalUser.clone();

        // 修改拷贝后对象的内部引用类型
        clonedUser.address.city = "上海";

        System.out.println("原始用户的城市: " + originalUser.address.city); // 输出：上海
        System.out.println("拷贝用户的城市: " + clonedUser.address.city); // 输出：上海
        // 结论：修改拷贝对象影响了原始对象，因为它们共享同一个 Address 对象
    }
}
```

### 2. 深拷贝 (Deep Copy)

#### 定义

创建一个新对象，并将原始对象中的字段值复制到新对象中。

- **对于基本数据类型**：同样是直接复制其**值**。
    
- **对于引用数据类型**：**递归地**为每一个引用类型成员创建一个新的对象，并复制其内容。
    

#### 结果

原始对象和拷贝后的对象是**完全独立**的，它们内部的任何引用类型成员都指向不同的对象。修改一方不会对另一方产生任何影响。

#### 如何实现？

方法一：重写 clone() 并进行递归克隆

在浅拷贝的基础上，对内部的每个引用类型都手动调用它们的 clone() 方法。

**代码示例（改进版）：**

Java

```
// 地址类也需要实现 Cloneable
class Address implements Cloneable {
    public String city;
    public Address(String city) { this.city = city; }
    
    @Override
    public Address clone() throws CloneNotSupportedException {
        return (Address) super.clone();
    }
}

class User implements Cloneable {
    public int age;
    public Address address;

    public User(int age, Address address) {
        this.age = age;
        this.address = address;
    }

    @Override
    public User clone() throws CloneNotSupportedException {
        // 1. 先进行浅拷贝，得到一个 User 对象
        User clonedUser = (User) super.clone();
        // 2. 对内部的引用类型进行手动的深拷贝
        clonedUser.address = this.address.clone();
        return clonedUser;
    }
}

public class DeepCopyTest {
    public static void main(String[] args) throws CloneNotSupportedException {
        Address address = new Address("北京");
        User originalUser = new User(30, address);
        
        // 进行深拷贝
        User clonedUser = originalUser.clone();

        // 修改拷贝后对象的内部引用类型
        clonedUser.address.city = "上海";

        System.out.println("原始用户的城市: " + originalUser.address.city); // 输出：北京
        System.out.println("拷贝用户的城市: " + clonedUser.address.city); // 输出：上海
        // 结论：修改拷贝对象不再影响原始对象，实现了完全隔离
    }
}
```

方法二：通过序列化实现（推荐，更通用）

利用 Java 的序列化机制，将对象写入字节流再从字节流中读出，这个过程自然而然地创建了一个全新的对象图，从而实现深拷贝。

- **优点**：代码简单，无论对象结构多复杂，都能正确处理。
    
- **缺点**：被拷贝的类及其所有引用的类都必须实现 `java.io.Serializable` 接口。性能开销比 `clone()` 略大。
    

**代码示例：**

Java

```
import java.io.*;

// 所有相关类都需要实现 Serializable 接口
class Address implements Serializable { ... }
class User implements Serializable { ... }

public class DeepCopyBySerialization {
    public static <T> T deepCopy(T object) throws IOException, ClassNotFoundException {
        // 写入字节流
        ByteArrayOutputStream bos = new ByteArrayOutputStream();
        ObjectOutputStream oos = new ObjectOutputStream(bos);
        oos.writeObject(object);

        // 从字节流中读出
        ByteArrayInputStream bis = new ByteArrayInputStream(bos.toByteArray());
        ObjectInputStream ois = new ObjectInputStream(bis);
        return (T) ois.readObject();
    }
    
    // main 方法中调用 deepCopy() 即可
}
```

方法三：使用第三方库

像 Gson、Jackson 等 JSON 库也可以实现深拷贝：先将对象转换为 JSON 字符串，再将字符串转换回新的对象。

### 总结对比

|特性|浅拷贝 (Shallow Copy)|深拷贝 (Deep Copy)|
|---|---|---|
|**处理基本类型**|复制值|复制值|
|**处理引用类型**|**复制引用地址**（共享对象）|**递归创建并复制新对象**（完全独立）|
|**独立性**|不完全独立，内部引用对象是共享的|**完全独立**，互不影响|
|**实现复杂度**|简单，`Object.clone()` 默认就是|复杂，需要手动递归克隆或使用序列化等技术|
|**适用场景**|对象内部没有可变引用类型，或者希望共享内部对象时|需要一个对象的**完全独立副本**，避免相互影响|
