实现 `clone()` 方法远比看起来要复杂，有很多**“约定”和“陷阱”**需要注意。正因如此，许多 Java 专家（包括《Effective Java》的作者 Joshua Bloch）都建议**谨慎使用它**，并优先考虑其他复制对象的方式（如我们昨天讨论的拷贝构造方法和序列化）。

==序列化深拷贝==
==或者使用MapStruct进行拷贝==
```java
import java.io.*;

public class SerializationDeepCopy {

    // 一个通用的深拷贝工具方法
    @SuppressWarnings("unchecked")
    public static <T extends Serializable> T deepCopy(T object) {
        try {
            // 写入内存中的字节流
            ByteArrayOutputStream bos = new ByteArrayOutputStream();
            ObjectOutputStream oos = new ObjectOutputStream(bos);
            oos.writeObject(object);

            // 从内存中的字节流读出
            ByteArrayInputStream bis = new ByteArrayInputStream(bos.toByteArray());
            ObjectInputStream ois = new ObjectInputStream(bis);
            return (T) ois.readObject();
        } catch (IOException | ClassNotFoundException e) {
            // 在实际项目中，这里应该有更完善的异常处理
            throw new RuntimeException("深拷贝失败", e);
        }
    }

    public static void main(String[] args) {
        Person originalPerson = new Person("张三", new Address("北京"));
        System.out.println("原始对象: " + originalPerson);

        // 使用序列化工具方法进行深拷贝
        Person deepCopiedPerson = deepCopy(originalPerson);
        System.out.println("拷贝对象: " + deepCopiedPerson);

        // 修改拷贝对象的内部引用
        deepCopiedPerson.address.city = "上海";

        System.out.println("--- 修改后 ---");
        // 验证：原始对象的 address.city 同样没有被改变
        System.out.println("原始对象: " + originalPerson);
        System.out.println("拷贝对象: " + deepCopiedPerson);
    }
}
```
### 1. 正确实现 `clone()` 的五个步骤

要正确地实现 `clone()`，你需要严格遵守一个约定俗成的“配方”：

#### **第一步：==实现 `Cloneable` 接口**==

这是 `clone()` 机制的“开关”。`Cloneable` 是一个**标记接口**（内部没有方法）。

- 如果一个类**实现了 `Cloneable`**，那么调用 `super.clone()` 时，JVM 会为其创建一个字段对字段的副本。
    
- 如果**没有实现 `Cloneable`**，调用 `super.clone()` 会直接抛出 `CloneNotSupportedException`。
    

Java

```
public class Person implements Cloneable {
    // ...
}
```

#### **第二步：==重写 `clone()` 方法并提升访问权限**==

`Object` 类中的 `clone()` 方法是 `protected` 的，这意味着它只能被同一个包的类或子类访问。为了让其他任何地方的代码都能调用它，你需要将其重写为 `public`。

Java

```
@Override
public Person clone() { // 返回类型建议使用协变返回类型，即类本身
    // ...
}
```

#### **第三步：==首先调用 `super.clone()`**

这是最关键、也是最基础的一步。调用 `super.clone()` 会利用 JVM 底层的本地（native）能力，高效地创建一个新对象，并将原始对象的所有字段值**原封不动地复制**到新对象中。

**这个操作默认产生的是一个“浅拷贝” (Shallow Copy)**。

#### **第四步：==（深拷贝的关键）处理引用类型字段**==

`super.clone()` 完成后，你得到了一个浅拷贝对象。如果你需要的是**深拷贝**，那么你必须手动处理所有**可变的引用类型**字段。

你需要为每一个这样的字段，也调用它的 `clone()` 方法，并将返回的新引用赋值给拷贝后对象的对应字段。

#### **第五步：==处理 `CloneNotSupportedException`**

`Object.clone()` 声明了会抛出 `CloneNotSupportedException`（一个受检异常）。即使你的类已经实现了 `Cloneable`，编译器仍然会强制你处理这个异常。

最佳实践是捕获它，并将其包装成一个非受检异常（如 `AssertionError`）抛出，因为理论上，如果你的类正确实现了 `Cloneable`，这个异常是永远不会发生的。

### 2. 代码示例：从浅拷贝到深拷贝

我们继续使用 `Person` 和 `Address` 的例子。

**首先，让 `Address` 也变得“可克隆”：**

Java

```java
public class Address implements Cloneable {
    public String city;
    public Address(String city) { this.city = city; }

    @Override
    public Address clone() {
        try {
            return (Address) super.clone();
        } catch (CloneNotSupportedException e) {
            // 这不应该发生，因为我们实现了 Cloneable
            throw new AssertionError();
        }
    }
}
```

**然后，在 `Person` 类中实现 `clone()`：**

Java

```java
public class Person implements Cloneable {
    public String name;
    public Address address; // 可变的引用类型字段

    public Person(String name, Address address) {
        this.name = name;
        this.address = address;
    }

    @Override
    public Person clone() {
        try {
            // 步骤三：首先调用 super.clone() 得到一个浅拷贝对象
            Person clonedPerson = (Person) super.clone();
            
            // 步骤四：手动处理引用类型，实现深拷贝
            // 让拷贝出来的 Person 对象指向一个全新的 Address 对象
            clonedPerson.address = this.address.clone();
            
            return clonedPerson;
        } catch (CloneNotSupportedException e) {
            // 步骤五：处理不可能发生的异常
            throw new AssertionError();
        }
    }

    public static void main(String[] args) {
        Person original = new Person("张三", new Address("北京"));
        Person cloned = original.clone();

        // 修改拷贝对象的内部引用
        cloned.address.city = "上海";

        // 验证：原始对象的 address 没有被改变，证明是深拷贝
        System.out.println("Original: " + original.address.city); // 输出: Original: 北京
        System.out.println("Cloned: " + cloned.address.city);   // 输出: Cloned: 上海
    }
}
```

如果你在**步骤四**中注释掉 `clonedPerson.address = this.address.clone();` 这一行，那么 `clone()` 方法执行的就只是浅拷贝，最后的输出会是两个“上海”。

### 3. 实现 `clone()` 的陷阱与缺点

了解了如何实现，也必须知道为什么它不受欢迎。

1. 与 final 字段的冲突：
    
    clone() 机制与 final 字段的配合很差。super.clone() 可以复制 final 字段，但如果你想对一个 final 的引用类型字段进行深拷贝，你将无法重新为 clonedObject.finalField 赋值，因为它是 final 的。
    
2. 复杂的契约与易错性：
    
    Cloneable 作为一个没有方法的标记接口，却改变了 Object.clone() 的行为，这种设计本身就不够直观。开发者很容易忘记上面五个步骤中的任何一步，导致 clone() 实现错误。
    
3. 信任链问题：
    
    实现深拷贝，你必须确保你引用的所有可变对象都正确地实现了 clone()。这个信任链中只要有一个环节出了错，整个深拷贝就会失败，变成部分深、部分浅的奇怪状态。
    

### 总结与替代方案

`clone()` 是一个强大但充满陷阱的特性。在实现它时，必须严格遵守上述的五个步骤，并特别注意深拷贝和浅拷贝的区别。

由于其固有的复杂性和风险，**在现代 Java 开发中，更推荐使用以下替代方案**：

- 拷贝构造方法 (Copy Constructors)：
    
    public Person(Person other) { ... }
    
    这种方式更明确、类型安全，且不会与 final 字段冲突。
    
- 静态工厂方法 (Static Factory Methods)：
    
    public static Person newInstance(Person other) { ... }
    
    功能上类似拷贝构造方法，但命名更灵活。
    
- 使用序列化 (Serialization)：
    
    对于复杂的对象图，序列化是一种实现深拷贝的便捷技巧。
    

总而言之，当你需要复制对象时，请优先考虑拷贝构造方法等更现代、更安全的方式。只有在你必须与某些强制要求 `clone()` 的旧框架交互，或者有特殊性能需求时，才应该去实现 `clone()`。
