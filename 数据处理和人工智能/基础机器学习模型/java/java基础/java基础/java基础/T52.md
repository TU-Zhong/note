早上好！在台北的周二早晨，我们来聊聊 Java 世界的“始祖”——`Object` 类。

`java.lang.Object` 类是 Java 类继承体系的根节点。由于 Java 中所有的类都直接或间接地继承自 `Object` 类，因此 `Object` 类中的方法是所有 Java 对象都具备的通用能力。

下面是 `Object` 类中最常用、也最常被问到的一些核心方法。

---

### 1. `toString()` 方法

- 默认行为：
    
    Object 类中 toString() 的默认实现是返回一个由“类名 + @ + 哈希码的十六进制表示”组成的字符串。例如：com.example.Person@15db9742。这个输出通常没有什么实际意义，可读性很差。
    
- 重写目的：
    
    在子类中重写此方法，以提供一个清晰、有意义的、人类可读的字符串表示，来描述该对象的状态。这对于日志记录（Logging）、调试（Debugging）和程序输出都至关重要。
    
- **代码示例**：
    
    Java
    
    ```
    public class Person {
        private String name = "张三";
        private int age = 25;
    
        // 重写 toString() 方法
        @Override
        public String toString() {
            return "Person{name='" + name + "', age=" + age + '}';
        }
    }
    // 调用时：
    // Person p = new Person();
    // System.out.println(p); // 如果不重写，输出类似 Person@...
    //                       // 重写后，输出: Person{name='张三', age=25}
    ```
    

### 2. `equals(Object obj)` 和 `hashCode()` 方法

这两个方法是“形影不离的兄弟”，因为它们之间有一个非常重要的约定。它们通常需要**同时被重写**。

#### `equals(Object obj)`

- 默认行为：
    
    等同于 == 运算符。比较的是两个对象的内存地址，即判断两个引用是否指向同一个对象（引用相等）。
    
- 重写目的：
    
    定义对象之间“逻辑相等”的概念。例如，我们认为两个 Person 对象，只要它们的身份证号相同，它们就代表同一个人，即逻辑上是相等的，即使它们是内存中两个独立的对象。
    

#### `hashCode()`

- 默认行为：
    
    通常是根据对象的内存地址计算得出的一个整数。
    
- 重写目的：
    
    主要为了配合哈希集合（如 HashMap, HashSet, Hashtable）正常工作。哈希集合使用 hashCode() 的返回值来确定对象的存储位置，以实现快速查找。
    
- **核心约定（Contract）**：
    
    > **如果两个对象通过 `equals()` 方法比较是相等的，那么它们的 `hashCode()` 方法必须返回相同的值。**
    
    如果只重写 `equals()` 而不重写 `hashCode()`，会导致对象在哈希集合中存储和查找时出现严重错误。
    
- **代码示例**：
    
    Java
    
    ```
    import java.util.Objects;
    
    public class Person {
        private String idCard;
        // ...
    
        @Override
        public boolean equals(Object o) {
            if (this == o) return true;
            if (o == null || getClass() != o.getClass()) return false;
            Person person = (Person) o;
            return Objects.equals(idCard, person.idCard); // 只要身份证号相同就认为相等
        }
    
        @Override
        public int hashCode() {
            return Objects.hash(idCard); // 保证身份证号相同的对象有相同的哈希码
        }
    }
    ```
    

### 3. `getClass()` 方法

- **作用**：返回一个对象在**运行时**的 `Class` 对象。这个 `Class` 对象包含了类的所有元数据信息（如类名、方法、字段等）。
    
- **特点**：这是一个 `final` 方法，**不能被重写**。
    
- **用途**：是 **Java 反射（Reflection）机制**的入口。
    
- **代码示例**：
    
    Java
    
    ```
    Person p = new Person();
    Class<?> personClass = p.getClass();
    System.out.println("该对象的运行时类是: " + personClass.getName());
    // 输出: 该对象的运行时类是: Person
    ```
    

### 4. 多线程相关方法：`wait()`, `notify()`, `notifyAll()`

这三个方法是 Java 传统多线程编程中用于**线程间通信**和**同步**的核心工具。

- **核心前提**：这三个方法都**必须在 `synchronized` 同步代码块或同步方法中调用**，并且是针对“锁对象”来调用的。如果不在同步上下文中调用，会抛出 `IllegalMonitorStateException`。
    
- wait():
    
    导致当前线程释放锁并进入等待状态，直到其他线程调用同一个锁对象的 notify() 或 notifyAll() 方法。
    
- notify():
    
    随机唤醒一个正在等待该锁对象的线程。
    
- notifyAll():
    
    唤醒所有正在等待该锁对象的线程。
    
- **用途**：主要用于实现经典的并发模式，如**生产者-消费者模型**。在现代 Java 开发中，更推荐使用 `java.util.concurrent` 包下更高级、更安全的并发工具（如 `BlockingQueue`, `CountDownLatch`, `ReentrantLock` 等）。
    

### 5. `clone()` 方法

- **作用**：创建一个并返回当前对象的一个**副本**。
    
- **特点**：
    
    - 它是 `protected` 的，通常需要子类重写并改为 `public` 才能在外部调用。
        
    - 一个类必须实现 `Cloneable` **标记接口**，否则调用 `clone()` 方法会抛出 `CloneNotSupportedException`。
        
    - `Object` 类的 `clone()` 方法执行的是**浅拷贝（Shallow Copy）**。
        
- **用途**：用于复制对象。但由于其设计的复杂性，现在很多开发者更倾向于使用**拷贝构造方法**或**序列化**等方式来实现对象的深拷贝。
    

### 6. `finalize()` 方法 (已废弃)

- **作用**：在对象被垃圾回收器回收之前，提供一个执行清理操作的机会。
    
- **状态**：从 **Java 9 开始已被正式废弃**，强烈不推荐使用。它的执行时机不确定、性能差，并且容易导致问题。
    
- **替代方案**：使用 `try-with-resources` 语句和 `AutoCloseable` 接口来进行可靠的资源清理。
