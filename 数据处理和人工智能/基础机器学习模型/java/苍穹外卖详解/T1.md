我来介绍一下我的“苍穹外卖”项目。这是一个完整的、仿照主流外卖平台设计的B2C在线订餐项目。它的核心是实现用户在线浏览、下单，以及商家接单、处理的完整业务闭环。整个项目采用前后端分离的架构进行开发。

这个项目主要可以分为两大功能模块：**用户端**和**管理后台**。

**首先，在用户端（C端）**，我们为普通消费者提供了流畅完整的点餐体验，主要功能包括：

1. **用户模块**：支持微信授权一键登录、个人信息管理和收货地址管理。
    
2. **店铺与商品浏览**：用户可以查看平台上的店铺列表，进入店铺后可以浏览详细的菜品分类、单个菜品信息以及套餐信息。
    
3. **购物车功能**：用户可以将心仪的菜品或套餐加入购物车，并能实时修改商品数量或清空。
    
4. **核心下单流程**：用户确认购物车商品后，可以选择收货地址、备注信息，然后下单并模拟支付。
    
5. **订单中心**：用户可以随时查看自己的历史订单，以及当前订单的实时状态，比如“待付款”、“待接单”、“派送中”等等。
    

**项目的另一大块，是提供给商家和平台管理员使用的管理后台（B端）**。这部分功能也非常完善，确保了平台的正常运营，主要包括：

1. **菜品管理**：这是核心功能之一，不仅支持单个菜品的增删改查、启售停售，还实现了较为复杂的**套餐管理**，可以将不同菜品组合成套餐进行售卖。
    
2. **订单管理**：后台可以实时查看所有用户订单，进行条件搜索，并执行**接单、派送、完成订单**等核心操作。
    
3. **店铺管理**：可以设置店铺的基本信息，最重要的是能够实时**修改店铺的营业状态**，决定用户端能否下单。
    
4. **员工管理**：支持后台管理员账号的增删改查和权限控制。
    
5. **数据统计**：提供了简单的数据看板，可以统计指定时间范围内的营业额、订单量、以及销量排名前10的菜品，为商家的经营决策提供数据支持。
    

---

**在这个项目中，我主要担任的是后端开发的角色，基本上独立负责了整个后端系统的设计和实现。我的具体职责包括：**

1. **数据库设计**：从0到1负责项目的数据库设计，包括用户、地址、店铺、菜品、订单等十几张核心表的结构设计和关联关系建立。
    
2. **核心业务API的开发**：上面提到的所有功能模块，比如用户登录、购物车、订单状态流转、后台管理等，其全部的后端RESTful API接口都是由我来编写和实现的。
    
3. **关键技术方案的实现**：比如，为了提升性能和并发能力，我使用 **Redis** 来实现购物车功能，减少了数据库的访问压力；使用 **JWT** 技术方案结合拦截器，实现了用户端和管理后台的登录认证与权限校验。
    
4. **项目通用模块的搭建**：为了提升代码质量和开发效率，我还搭建了项目的通用模块，比如**全局异常处理、公共字段自动填充（AOP）、统一结果返回**等。
    

总的来说，通过这个项目，我完整地经历了一个商业项目从需求分析、技术选型、数据库设计到编码实现的全过程，对企业级应用的开发流程有了非常深入的理解和实践。
