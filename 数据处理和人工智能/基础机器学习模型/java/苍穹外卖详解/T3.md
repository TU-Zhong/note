## 🗄️ 淘淘外卖数据库设计详解

### **8张核心表及其关系**

#### **1. Users (用户表)** - 系统用户基础表
- **主键**: `id` (BIGINT, 自增)
- **核心字段**: 
  - `username` (用户名，唯一)
  - `password` (加密密码)
  - `user_type` (1:消费者, 2:商家/管理员)
- **作用**: 存储所有系统用户信息，支持双端用户

#### **2. Shops (店铺表)** - 商家店铺信息
- **主键**: `id` (BIGINT, 自增)
- **核心字段**:
  - `name` (店铺名称)
  - `status` (1:营业中, 0:休息中)
- **作用**: 管理商家店铺基本信息和营业状态

#### **3. Categories (菜品分类表)** - 菜品分类管理
- **主键**: `id` (BIGINT, 自增)
- **外键**: `shop_id` → `Shops(id)`
- **关系**: **一个店铺有多个分类** (1:N)
- **作用**: 为每个店铺的菜品进行分类管理

#### **4. Dishes (菜品表)** - 菜品信息核心表
- **主键**: `id` (BIGINT, 自增)
- **外键**: `category_id` → `Categories(id)`
- **关系**: **一个分类有多个菜品** (1:N)
- **核心字段**:
  - `name`, `price`, `description`
  - `status` (1:在售, 0:停售)
- **作用**: 存储具体菜品信息

#### **5. Combos (套餐表)** - 套餐信息管理
- **主键**: `id` (BIGINT, 自增)
- **外键**: `shop_id` → `Shops(id)`
- **关系**: **一个店铺有多个套餐** (1:N)
- **作用**: 管理店铺的套餐产品

#### **6. Combo_Dish_Mappings (套餐菜品关联表)** - 多对多关系表
- **外键**: 
  - `combo_id` → `Combos(id)`
  - `dish_id` → `Dishes(id)`
- **关系**: **套餐与菜品的多对多关系** (M:N)
- **作用**: 定义套餐包含哪些菜品

#### **7. Orders (订单表)** - 订单主表
- **主键**: `id` (BIGINT, 自增)
- **外键**: 
  - `user_id` → `Users(id)`
  - `shop_id` → `Shops(id)`
- **关系**: 
  - **一个用户有多个订单** (1:N)
  - **一个店铺有多个订单** (1:N)
- **核心字段**:
  - `order_number` (订单号，唯一)
  - `status` (1-6:订单状态流转)
  - `total_amount` (订单总金额)

#### **8. Order_Details (订单详情表)** - 订单商品明细
- **主键**: `id` (BIGINT, 自增)
- **外键**:
  - `order_id` → `Orders(id)`
  - `dish_id` → `Dishes(id)` (可选)
  - `combo_id` → `Combos(id)` (可选)
- **关系**: **一个订单有多个详情** (1:N)
- **特点**: 支持菜品和套餐混合下单

### **🔗 核心关系链路**

#### **商品管理链路**:
```
Shops → Categories → Dishes
  ↓
Combos ← Combo_Dish_Mappings → Dishes
```

#### **订单业务链路**:
```
Users → Orders ← Shops
  ↓
Order_Details → Dishes/Combos
```

### **🎯 设计亮点**

1. **灵活的用户体系**: 通过`user_type`区分消费者和商家
2. **完整的商品层次**: 店铺→分类→菜品的三级结构
3. **套餐支持**: 通过关联表实现套餐与菜品的多对多关系
4. **订单详情灵活性**: 支持单品和套餐混合下单
5. **公共字段设计**: 所有表都有`create_time`、`update_time`等审计字段
6. **状态管理**: 店铺、菜品、套餐、订单都有状态字段
7. **性能优化**: 创建了关键字段的索引
8. **数据完整性**: 完善的外键约束和检查约束

### **📊 业务场景支持**

- **用户注册登录**: Users表支持
- **店铺管理**: Shops表 + 状态缓存
- **菜品浏览**: Categories + Dishes的层次查询
- **购物车**: Redis Hash结构存储
- **下单流程**: Orders + Order_Details记录完整订单信息
- **套餐销售**: Combos + Combo_Dish_Mappings支持

这个数据库设计完全支持了外卖平台的核心业务流程，具备良好的扩展性和维护性！
