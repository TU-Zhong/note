实现的功能

- 从 Redis Hash cart:{userId} 读取购物车项（ dish:{id} 或 combo:{id} ，值为数量）。
- 查询 Dish / Combo 单价，计算订单总价，生成订单号并写入 orders 表。
- 批量写入订单明细到 order_details 表（包含菜品或套餐的快照单价与数量）。
- 提交成功后清空用户购物车键。
接口路径

- POST /user/order/submit
  - 请求体（JSON）：
    - deliveryAddress ：字符串，收货地址
    - contactPhone ：字符串，联系电话
    - notes ：字符串，备注（可选）
  - 响应： Result<Order> ，包含生成的 orderNumber 、 totalAmount 、 status 等。
提交逻辑说明

- 用户ID来自登录上下文（ BaseContext.getCurrentId() ）。
- 订单号格式： ORD{userId}{timestamp} 。
- 默认 shopId=1 ，订单状态置为 2 （待接单）。
- 明细价格以下单时的 Dish.price 或 Combo.price 快照为准。
- 空购物车会抛出错误提示。
如何达到目标

- 复用现有购物车的 Redis 结构与键设计，严谨解析 dish: 与 combo: 项，确保总价计算与明细入库正确。
- 使用批量插入提升订单明细写入效率，并在成功后清理购物车，保证状态一致性。
