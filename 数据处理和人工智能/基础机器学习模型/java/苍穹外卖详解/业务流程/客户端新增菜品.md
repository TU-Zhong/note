  已为移动端新增菜品浏览接口，并梳理完整购物车调用流程与示例，便于前端对接。

更新与实现
- 新增文件：`src/main/java/com/taotao/controller/user/DishController.java`
- 新增特性：
  - `GET /api/user/dish/list?categoryId=<可选>` 返回在售菜品列表（按分类可选过滤）。
  - `GET /api/user/dish/{id}` 返回菜品详情。
- 购物车已具备：`/api/user/shoppingCart/add`、`/api/user/shoppingCart/list`、`/api/user/shoppingCart/sub`、`/api/user/shoppingCart/clean`。

整体流程
- 用户登录获取JWT后，在移动端请求中携带 `Authorization: Bearer <token>`。
- 浏览菜品：调用 `GET /api/user/dish/list` 获取在售菜品；需要详情时调用 `GET /api/user/dish/{id}`。
- 加入购物车：调用 `POST /api/user/shoppingCart/add`，传 `dishId` 或 `comboId`（二选一）。==数量在后端通过 Redis Hash 自增。==
- 查看购物车：`GET /api/user/shoppingCart/list` 返回购物车明细（菜品会带回 `name`、`image`）。
- 减少数量与清空：分别调用 `POST /api/user/shoppingCart/sub` 和 `DELETE /api/user/shoppingCart/clean`。

接口概览
- `GET /api/user/dish/list?categoryId=<可选>`：返回在售菜品列表。
- `GET /api/user/dish/{id}`：返回菜品详情。
- `POST /api/user/shoppingCart/add`：加入购物车（`dishId` 或 `comboId`）。
- `GET /api/user/shoppingCart/list`：查看购物车。
- `POST /api/user/shoppingCart/sub`：减少购物车中一个商品的数量。
- `DELETE /api/user/shoppingCart/clean`：清空购物车。

数据模型（请求体 `ShoppingCartDTO`）
- 字段：`dishId`、`comboId`、`name`、`image`、`userId`、`amount`
- 约定：加入购物车时只需传 `dishId` 或 `comboId`；后端通过当前登录用户上下文(`BaseContext.getCurrentId()`)识别用户，并在 Redis 中以 `cart:<userId>` 存储。

请求示例
- 获取菜品列表（在售）：
  - `GET /api/user/dish/list?categoryId=1`
- 加入购物车（菜品）：
  - `POST /api/user/shoppingCart/add`，JSON：`{ "dishId": 1 }`
- 查看购物车：
  - `GET /api/user/shoppingCart/list`
- 减少数量：
  - `POST /api/user/shoppingCart/sub`，JSON：`{ "dishId": 1 }`
- 清空购物车：
  - `DELETE /api/user/shoppingCart/clean`

实现细节（后端逻辑）
- 购物车：`ShoppingCartServiceImpl` 基于 Redis Hash 实现，键：`cart:<userId>`；项键：`dish:<dishId>` 或 `combo:<comboId>`；已存在则数量 `amount` 自增，不存在则置为 `1`。
- 查看购物车：会查询菜品详情填充 `name`、`image`；套餐（`comboId`）的详情填充目前预留，需后续增加 Combo 查询。
- 菜品列表：`DishService.list(categoryId)` 返回在售菜品，`status=1`。
