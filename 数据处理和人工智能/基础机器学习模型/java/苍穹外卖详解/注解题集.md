### 第一部分：基础 Bean 与依赖注入 (1-4题)

**目标：** 掌握如何定义 Bean 以及如何将它们装配在一起。

1. **`@Component` 与 `@ComponentScan`**
    
    - **任务：** 创建一个 `com.example.components` 包，在其中定义一个 `MyComponent` 类，并使用 `@Component` 注解它。在你的 Spring Boot 主类中，确保 `@SpringBootApplication` (它包含了 `@ComponentScan`) 能够扫描到这个包。编写一个测试或在主类的 `CommandLineRunner` 中，从 Spring 容器中获取 `MyComponent` 的实例并打印出来，验证它是否被成功加载。
        
2. **`@Service`, `@Repository` 与 `@Autowired`**
    
    - **任务：** 创建一个经典的三层结构：
        
        - `UserController` (暂时只是一个普通类)
            
        - `UserService` (使用 `@Service`)
            
        - UserRepository (使用 @Repository)
            
            在 UserService 中注入 UserRepository，在 UserController 中注入 UserService。使用字段注入、Setter 注入和构造器注入分别实现一次，体会它们的区别（推荐使用构造器注入）。
            
3. **`@Qualifier`**
    
    - **任务：** 创建一个 `NotificationService` 接口，它有一个 `send(String message)` 方法。然后创建两个实现类：`EmailNotificationService` 和 `SmsNotificationService`，都用 `@Service` 注解。在一个 `OrderService` 中，尝试使用 `@Autowired` 注入 `NotificationService`。你会发现启动失败。现在，使用 `@Qualifier("emailNotificationService")` 来精确指定注入哪一个实现。
        
4. **`@Configuration` 与 `@Bean`**
    
    - **任务：** 假设你需要使用一个第三方库的类，比如 `PasswordEncoder` (密码编码器)。创建一个 `AppConfig` 配置类，并用 `@Configuration` 注解。在其中定义一个方法，返回 `BCryptPasswordEncoder` 的实例，并用 `@Bean` 注解该方法，将其注册为 Spring Bean。然后在你的 `UserService` 中注入这个 `PasswordEncoder`。

### 第二部分：Web API 开发 (5-9题)

**目标：** 掌握如何使用 Spring MVC 注解构建 RESTful API。

5. **`@RestController` 与 `@GetMapping`**
    
    - **任务：** 创建一个 `HelloController`，使用 `@RestController` 注解。在其中创建一个方法，映射到 `/hello` 路径，返回字符串 "Hello, Spring Boot!"。使用浏览器或 Postman 访问 `http://localhost:8080/hello` 进行测试。
        
6. **`@PathVariable`**
    
    - **任务：** 创建一个 `UserController`。在其中创建一个方法，可以根据 URL 中的 ID 获取用户信息。路径应为 `/users/{id}`。使用 `@PathVariable` 将 `id` 绑定到方法参数，并返回一个类似 "Fetching user with ID: {id}" 的字符串。
        
7. **`@RequestParam`**
    
    - **任务：** 在 `UserController` 中添加一个方法，用于根据条件查询用户列表，路径为 `/users`。使用 `@RequestParam` 来接收查询参数，例如 `?status=active`。让这个参数是可选的，并给它一个默认值。
        
8. **`@PostMapping` 与 `@RequestBody`**
    
    - **任务：** 创建一个简单的 `User` DTO 类（只有 id, name, email 字段）。在 `UserController` 中创建一个方法，映射到 `POST /users`，使用 `@RequestBody` 将请求体中的 JSON 自动转换为 `User` 对象，并返回 "User created: " + user.toString()。
        
9. **综合 CRUD API**
    
    - **任务：** 为一个 `Product` 实体（有 id, name, price 字段）创建一个完整的 CRUD API。
        
        - `GET /products`：获取所有产品
            
        - `GET /products/{id}`：获取单个产品
            
        - `POST /products`：创建新产品
            
        - `PUT /products/{id}`：更新产品
            
        - DELETE /products/{id}：删除产品
            
            你需要综合运用 @GetMapping, @PostMapping, @PutMapping, @DeleteMapping, @PathVariable, @RequestBody。

### 第三部分：配置与属性注入 (10-12题)

**目标：** 掌握如何管理和注入应用的配置。

10. **`@Value`**
    
    - **任务：** 在 `application.properties` 文件中添加一个自定义属性 `app.name=My Awesome App`。在一个 `@Service` 中，使用 `@Value("${app.name}")` 将这个属性值注入到一个字段中，并将其打印出来。
        
11. **`@ConfigurationProperties`**
    
    - **任务：** 创建一个 `JwtProperties` 类，包含 `secretKey` 和 `expirationTime` 两个字段。在 `application.properties` 中定义 `jwt.secret-key=mysecret` 和 `jwt.expiration-time=3600`。使用 `@ConfigurationProperties(prefix = "jwt")` 将这些属性自动绑定到 `JwtProperties` 类的实例上。
        
12. **`@Profile`**
    
    - **任务：** 创建两个配置类 `DevConfig` 和 `ProdConfig`。在 `DevConfig` 中创建一个用于开发的 Bean (例如，一个返回 "dev datasource" 的字符串Bean)，并用 `@Profile("dev")` 注解。在 `ProdConfig` 中创建另一个用于生产的 Bean (返回 "prod datasource")，并用 `@Profile("prod")` 注解。在 `application.properties` 中通过 `spring.profiles.active=dev` 来切换环境，并验证容器中加载的是哪一个 Bean。

### 第四部分：数据访问与事务管理 (13-15题)

目标： 掌握如何与数据库交互并保证数据一致性。

(你需要添加 Spring Data JPA 和 H2 Database 的依赖)

13. **JPA `@Entity` 与 `@Repository`**
    
    - **任务：** 创建一个 `Product` 实体类，使用 `@Entity`, `@Id`, `@GeneratedValue` 等注解。然后创建一个 `ProductRepository` 接口，继承 `JpaRepository<Product, Long>`，并用 `@Repository` 注解它。
        
14. **`@Transactional`**
    
    - **任务：** 在 `ProductService` 中创建一个 `updateProductPrice` 方法。这个方法需要先读取产品，然后更新价格。在方法上添加 `@Transactional` 注解。为了模拟失败，可以在保存后手动抛出一个 `RuntimeException`。验证当异常发生时，数据库中的价格是否回滚到了修改前的状态。
        
15. **事务传播行为**
    
    - **任务：** 创建两个 Service，`ServiceA` 和 `ServiceB`。`ServiceA` 的方法 `methodA` 调用 `ServiceB` 的方法 `methodB`。在 `methodA` 上添加 `@Transactional`。尝试给 `methodB` 添加不同的事务传播属性（如 `Propagation.REQUIRED` 和 `Propagation.REQUIRES_NEW`），并在 `methodB` 中抛出异常，观察 `methodA` 中的数据操作是否会回滚，理解不同传播行为的区别。

### 第五部分：综合与进阶应用 (16-20题)

**目标：** 掌握异常处理、AOP、定时任务等高级特性。

16. **`@RestControllerAdvice` 与 `@ExceptionHandler`**
    
    - **任务：** 自定义一个 `ResourceNotFoundException` 异常。在你的 Service 中，当查询不到某个资源时，抛出这个异常。然后创建一个全局异常处理器类，使用 `@RestControllerAdvice` 注解，并在其中创建一个方法用 `@ExceptionHandler(ResourceNotFoundException.class)` 来捕获这个异常，返回一个统一的 JSON 错误格式和 404 状态码。
        
17. **AOP (`@Aspect`, `@Before`)**
    
    - **任务：** 创建一个日志切面 `LoggingAspect`。使用 `@Aspect` 注解它。定义一个切点（Pointcut）来匹配所有 `Controller` 包下的所有公共方法。然后使用 `@Before` 通知，在每个 Controller 方法执行前，打印出"即将执行的方法: [方法签名]"。
        
18. **`@Scheduled`**
    
    - **任务：** 创建一个 `ScheduledTasks` 组件。在其中创建一个方法，使用 `@Scheduled(fixedRate = 5000)` 注解，让它每隔 5 秒钟在控制台打印一次当前时间。
        
19. **`@Scope`**
    
    - **任务：** 创建一个 `PrototypeBean`，并使用 `@Scope("prototype")` 注解。同时创建一个默认的 `SingletonBean`。在一个 Service 中同时注入这两个 Bean。连续两次从 Spring 容器中获取 `PrototypeBean` 和 `SingletonBean` 的实例，打印它们的哈希码，验证 Singleton 只创建一个实例，而 Prototype 每次都创建新实例。
        
20. **【终极挑战】综合项目**
    
    - **任务：** 构建一个迷你的博客文章发布 API。
        
        - 使用 `@Entity` 创建 `Post` 实体。
            
        - 使用 Spring Data JPA 创建 `@Repository`。
            
        - 使用 `@Service` 和 `@Transactional` 编写业务逻辑（如发布、删除文章）。
            
        - 使用 `@RestController` 暴露完整的 CRUD API。
            
        - 使用 `@ConfigurationProperties` 管理自定义配置。
            
        - 使用 `@RestControllerAdvice` 处理文章未找到的异常。
            
        - **要求：** 将之前学到的所有相关注解都融会贯通地应用到这个小项目中。
