当然，这是一个非常深刻的问题，它触及了算法背后的数学原理。我们能这么写，是基于两个非常基础且强大的数学事实。

我们再次回到目标：计算 313。

---
```java
private long power(long base, int exponent, int mod) {
    long res = 1;      // res 用于存储最终结果，初始化为乘法的单位元 1
    base %= mod;       // 预处理，防止 base 过大。 (3 % 1000 = 3)

    while (exponent > 0) { // 循环直到 exponent 变为 0，相当于处理完二进制的每一位
        
        // --- 核心判断 ---
        // if (exponent % 2 == 1) 检查 exponent 二进制的最右一位是否为 1
        // 如果是 1，说明最终结果需要乘以当前这一项的 base
        if (exponent % 2 == 1) {
            res = (res * base) % mod; 
        }

        // --- 准备下一轮 ---
        // base 自乘，实现“连续平方法”。
        // 这一步让 base 从 3^1 -> 3^2 -> 3^4 -> 3^8 ...
        base = (base * base) % mod; 
        
        // exponent 除以 2，相当于二进制右移一位，处理下一位
        exponent /= 2; 
    }
    return res;
}
```
### 事实一：任何指数都可以“拆分”成加法

这是我们小学就学过的指数法则：am+n=am×an

- 例如：35=32+3=32×33。
    
- 这个法则告诉我们：**一个复杂的指数运算，可以被拆分成几个更简单的指数运算的乘积。**
    

### 事实二：任何一个十进制数字，都可以用“二进制”来表示

这是计算机科学的基石。二进制的本质就是**把一个数字表示成“2的幂次方”的和**。

我们再来看 `13`：

- `13` 的二进制是 `1101`。
    
- 这个 `1101` 是什么意思呢？它是按“权位”来计算的，从右到左，权位分别是 20,21,22,23，也就是 `1, 2, 4, 8`。
    
- 所以，1101 就表示：
    
    13=(1×8)+(1×4)+(0×2)+(1×1)
    
    13=8+4+1
    

### 关键的结合：将“指数”用二进制拆分

现在，我们把上面两个事实结合起来，奇迹就发生了：

我们想计算 313。

根据 事实二，我们可以把指数 13 拆分成 8+4+1。

所以，313=3(8+4+1)。

现在，根据 事实一，我们可以把这个式子变成乘法：

313=38×34×31

**这就是整个算法的数学灵魂！**

我们成功地把一个看起来需要乘以13次的复杂问题，转化成了一个**全新的、更简单的问题**：

1. 我只需要计算出 31,34,38 这几个“特殊项”。
    
2. 然后把这几个特殊项乘起来，就得到了最终答案。
    

---

### 代码是如何实现这个数学思想的？

现在我们回头看代码，就会发现它的每一行都在完美地执行这个数学思想。

`while` 循环就是在模拟“拆分指数”和“合成结果”的过程。

|轮次|指数 `exponent` 和它的二进制|`base` 的值 (代表的特殊项)|`res` 购物车的变化|
|---|---|---|---|
|**开始**|13 (`1101`)|`base` = 3 (代表 31)|`res` = 1|
|**第1轮**|13 (`110**1**`)|`base` = 3 (代表 31)|最右位是`1`，**需要** 31 这一项。`res` 乘以 `base` 变成 **3**。|
|_`base`升级_||`base` 变成 9 (代表 32)||
|**第2轮**|6 (`11**0**`)|`base` = 9 (代表 32)|最右位是`0`，**不需要** 32 这一项。`res` 不变。|
|_`base`升级_||`base` 变成 81 (代表 34)||
|**第3轮**|3 (`1**1**`)|`base` = 81 (代表 34)|最右位是`1`，**需要** 34 这一项。`res` 乘以 `base` 变成 **243** (3×81)|
|_`base`升级_||`base` 变成 6561 (代表 38)||
|**第4轮**|1 (`**1**`)|`base` = 6561 (代表 38)|最右位是`1`，**需要** 38 这一项。`res` 乘以 `base` 变成 **1594323** (243×6561)|
|_`base`升级_||`base` 变成 ... (代表 316)||
|**结束**|0||最终 `res` 就是 31×34×38 的结果。|

**所以，我们可以这么总结代码的作用：**

- `exponent /= 2` 和 `exponent % 2`：这一对操作就是在**从右到左地读取指数的二进制位**，判断出我们需要 31,34,38 这几项。
    
- `base = base * base`：这一行代码就是通过**连续平方，快速地制造出** 31,32,34,38,... 这些“特殊项”的原材料。
    
- `res = res * base`：这一行就是**在需要的时候，把对应的特殊项加入到最终的乘积中**。
    

这个算法之所以能这么写，是因为它巧妙地利用了任何数字都能用二进制（2的幂之和）来表示的原理，再结合指数的乘法法则，将一个复杂的、连续的乘法问题，分解成了一系列简单、独立的“检查”和“累乘”操作。
