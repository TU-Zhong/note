您好！这是一个 Java 面试中非常经典的问题。从经典到现代，Java 创建和管理线程的方式主要有以下几种，我将逐一为您解释并提供示例。

---

### **方式一：继承 `Thread` 类**

这是最直观的一种方式，直接创建一个 `Thread` 的子类。

**步骤：**

1. 定义一个类，继承自 `java.lang.Thread` 类。
    
2. 重写父类的 `run()` 方法，将线程需要执行的任务逻辑写在 `run()` 方法中。
    
3. 创建该子类的实例。
    
4. 调用实例的 `start()` 方法来启动线程（注意：不是直接调用 `run()` 方法！）。
    

**示例代码：**

Java

```
// 1. 定义一个 Thread 的子类
class MyThread extends Thread {
    // 2. 重写 run() 方法
    @Override
    public void run() {
        System.out.println("方式一：通过继承 Thread 类创建的线程正在运行...");
        System.out.println("当前线程名称: " + Thread.currentThread().getName());
    }
}

public class Main {
    public static void main(String[] args) {
        // 3. 创建子类实例
        MyThread myThread = new MyThread();
        // 4. 调用 start() 启动线程
        myThread.start();
    }
}
```

**缺点：**

- Java 是单继承的，如果你的类已经继承了另一个类，就无法再继承 `Thread` 类了，扩展性差。
    
- 任务（`run` 方法的逻辑）和线程（`Thread` 类）高度耦合，不符合面向对象的设计原则。
    

---

### **方式二：实现 `Runnable` 接口 (推荐)**

这是更常用、更推荐的一种经典方式，它将“任务”和“线程”解耦。

**步骤：**

1. 定义一个类，实现 `java.lang.Runnable` 接口。
    
2. 实现接口中的 `run()` 方法，编写任务逻辑。
    
3. 创建该实现类的实例（这就是“任务”）。
    
4. 创建一个 `Thread` 对象，并将上一步创建的 `Runnable` 实例作为参数传入。
    
5. 调用 `Thread` 对象的 `start()` 方法。
    

**示例代码：**

Java

```java
// 1. 定义一个实现 Runnable 接口的类
class MyRunnable implements Runnable {
    // 2. 实现 run() 方法
    @Override
    public void run() {
        System.out.println("方式二：通过实现 Runnable 接口创建的线程正在运行...");
        System.out.println("当前线程名称: " + Thread.currentThread().getName());
    }
}

public class Main {
    public static void main(String[] args) {
        // 3. 创建 Runnable 实现类的实例
        MyRunnable myRunnable = new MyRunnable();
        // 4. 创建 Thread 对象，并将任务传递进去
        Thread thread = new Thread(myRunnable);
        // 5. 启动线程
        thread.start();

        // 使用 Lambda 表达式可以更简洁 (Java 8+)
        Thread lambdaThread = new Thread(() -> {
            System.out.println("方式二 (Lambda)：线程正在运行...");
        });
        lambdaThread.start();
    }
}
```

**优点：**

- 任务代码（`MyRunnable`）和线程机制（`Thread`）分离，耦合度低。
    
- 实现接口的方式没有单继承的限制，类可以继承其他类。
    
- 多个线程可以共享同一个 `Runnable` 实例，适合处理共享资源的场景。

### **方式三：实现 `Callable` 接口 (可获取返回值)**

当你的线程任务执行完毕后需要一个返回结果时，前两种方式就无能为力了。`Callable` 就是为此而生的。

**步骤：**

1. 定义一个类，实现 `java.util.concurrent.Callable<V>` 接口，`V` 是返回值的类型。
    
2. 实现接口中的 `call()` 方法，该方法可以有返回值，也可以抛出异常。
    
3. 通常配合**线程池 (`ExecutorService`)** 来使用。将 `Callable` 任务提交给线程池。
    
4. 线程池的 `submit()` 方法会返回一个 `Future<V>` 对象。
    
5. 通过调用 `Future` 对象的 `get()` 方法，可以**阻塞式地**获取线程执行完毕后的返回结果。
    

**示例代码：**

Java

```
import java.util.concurrent.*;

// 1. 定义一个实现 Callable 接口的类
class MyCallable implements Callable<String> {
    // 2. 实现 call() 方法，有返回值
    @Override
    public String call() throws Exception {
        System.out.println("方式三：通过实现 Callable 接口创建的线程正在运行...");
        Thread.sleep(2000); // 模拟耗时操作
        return "任务执行完毕，这是返回结果！";
    }
}

public class Main {
    public static void main(String[] args) throws ExecutionException, InterruptedException {
        // 3. 创建一个线程池
        ExecutorService executor = Executors.newSingleThreadExecutor();
        
        // 4. 创建 Callable 实例并提交给线程池，获取 Future 对象
        MyCallable myCallable = new MyCallable();
        Future<String> future = executor.submit(myCallable);

        System.out.println("主线程继续做其他事情...");

        // 5. 调用 get() 方法获取结果（这里会阻塞，直到任务完成）
        String result = future.get();
        System.out.println("主线程获取到结果: " + result);

        // 关闭线程池
        executor.shutdown();
    }
}
```

**优点：**

- 可以获得任务的执行结果。
    
- 可以在 `call()` 方法中抛出受检异常。

### **现代最佳实践：使用线程池**

虽然上面介绍了三种创建线程任务的方法，但在现代企业级开发中，我们**几乎不会手动 `new Thread()`**。因为频繁创建和销毁线程开销很大。

**最佳实践是使用 `ExecutorService` (线程池)**，将任务（`Runnable` 或 `Callable`）提交给线程池，由线程池来管理线程的生命周期、复用和调度。

**示例代码：**

Java

```java
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

public class Main {
    public static void main(String[] args) {
        // 创建一个固定大小的线程池
        ExecutorService threadPool = Executors.newFixedThreadPool(5);

        try {
            // 提交 10 个任务给线程池
            for (int i = 0; i < 10; i++) {
                final int taskIndex = i;
                // 提交 Runnable 任务
                threadPool.submit(() -> {
                    System.out.println("任务 " + taskIndex + " 由线程 " + Thread.currentThread().getName() + " 执行");
                });
            }
        } finally {
            // 任务提交完毕后，关闭线程池
            threadPool.shutdown();
        }
    }
}
```

### **总结**

| 特性         | **继承 `Thread`** | **实现 `Runnable`** | **实现 `Callable`** |
| ---------- | --------------- | ----------------- | ----------------- |
| **继承性**    | 有单继承限制          | 无限制               | 无限制               |
| **返回值**    | 无               | 无                 | **有 (`Future`)**  |
| **异常处理**   | 只能处理非受检异常       | 只能处理非受检异常         | **可抛出受检异常**       |
| **与线程耦合度** | 高               | **低**             | **低**             |
| **推荐度**    | ★☆☆☆☆ (不推荐)     | ★★★★☆ (推荐)        | ★★★★★ (需要返回值时首选)  |

**面试核心要点：**

1. 答出 `Thread` 和 `Runnable` 两种方式，并说明 `Runnable` 因解耦和无继承限制而更优。
    
2. 进一步答出 `Callable` 和 `Future`，说明其可以获取返回值的优点。
    
3. 最后升华到**线程池**，表明在实际开发中，我们通过线程池来管理和执行任务，这是现代并发编程的最佳实践。
