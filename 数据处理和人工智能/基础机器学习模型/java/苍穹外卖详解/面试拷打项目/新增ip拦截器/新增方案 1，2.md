好的，为您的项目添加一个IP地址拦截器（俗称“IP墙”），是一个非常常见的安全和风控需求。这通常用于阻止已知的恶意IP、爬虫或者进行区域限制。

在Java Web项目（特别是基于Spring Boot的项目）中，实现这个功能通常有两种主流且优雅的方式：

1. **使用 Servlet Filter (过滤器)**：这是最底层的、最标准的Java EE解决方案。它在请求进入Spring的`DispatcherServlet`之前就进行拦截，适用性最广，效率最高。
    
2. **使用 Spring Interceptor (拦截器)**：这是Spring MVC框架提供的功能。它在请求进入Controller方法之前执行，可以更方便地利用Spring容器中的服务（Bean）。
    

对于IP拦截这种普适性的安全功能，**我更推荐使用 Servlet Filter**，因为它更早介入请求处理流程。下面，我将为您详细介绍如何实现，并同时提供两种方案供您选择。

---

### 整体思路

无论使用哪种技术，核心步骤都是一样的：

1. **创建IP黑名单/白名单**：定义一个列表，存放需要拦截（或放行）的IP地址。
    
2. **获取请求的真实IP**：正确地从`HttpServletRequest`中解析出客户端的真实IP地址，需要考虑代理服务器的情况。
    
3. **编写拦截逻辑**：在Filter或Interceptor中，获取IP并与名单进行比对。
    
4. **执行拦截/放行**：如果IP在黑名单中，则中断请求，返回一个错误响应（如 `403 Forbidden`）；否则，放行请求，让其继续处理。

### 方案一：使用 Servlet Filter (推荐)

这是最通用和高效的实现方式。

#### 第一步：在 `application.yml` 中配置IP黑名单

将IP列表配置在外部文件中，便于修改，无需重新编译代码。

YAML

```java
# application.yml
security:
  ip-blocklist:
    - "192.168.1.101"  # 这是一个示例恶意IP
    - "10.0.0.5"
    - "127.0.0.1"      # 如果你想禁止本地访问测试，可以加上
```

#### 第二步：编写获取真实IP的工具类

在实际环境中，用户请求可能会经过Nginx、负载均衡器等代理。`request.getRemoteAddr()`可能只拿到代理服务器的IP。因此，我们需要一个工具类来获取真实IP。

```java
// src/main/java/com/yourproject/util/IpUtils.java
package com.yourproject.util;

import javax.servlet.http.HttpServletRequest;

public class IpUtils {

    private static final String[] IP_HEADER_CANDIDATES = {
            "X-Forwarded-For",
            "Proxy-Client-IP",
            "WL-Proxy-Client-IP",
            "HTTP_X_FORWARDED_FOR",
            "HTTP_X_FORWARDED",
            "HTTP_X_CLUSTER_CLIENT_IP",
            "HTTP_CLIENT_IP",
            "HTTP_FORWARDED_FOR",
            "HTTP_FORWARDED",
            "HTTP_VIA",
            "REMOTE_ADDR"
    };

    public static String getClientIpAddr(HttpServletRequest request) {
        for (String header : IP_HEADER_CANDIDATES) {
            String ip = request.getHeader(header);
            if (ip != null && ip.length() != 0 && !"unknown".equalsIgnoreCase(ip)) {
                // 对于X-Forwarded-For头，它可能包含多个IP，第一个是真实IP
                if ("X-Forwarded-For".equalsIgnoreCase(header)) {
                    int index = ip.indexOf(',');
                    if (index != -1) {
                        return ip.substring(0, index);
                    }
                }
                return ip;
            }
        }
        return request.getRemoteAddr();
    }
}
```

#### 第三步：创建 IP 拦截过滤器 `IpBlockFilter`

这是核心的拦截逻辑。

Java

```java
// src/main/java/com/yourproject/filter/IpBlockFilter.java
package com.yourproject.filter;

import com.yourproject.util.IpUtils;
import lombok.extern.slf4j.Slf4j;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Component;

import javax.servlet.*;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import java.io.IOException;
import java.util.HashSet;
import java.util.List;
import java.util.Set;

@Component
@Slf4j
public class IpBlockFilter implements Filter {

    // 使用 @Value 注解从 application.yml 注入黑名单
    @Value("${security.ip-blocklist}")
    private List<String> ipBlocklist;

    private Set<String> ipBlockSet;

    // Filter初始化时，将List转换为Set，提高查询效率
    @Override
    public void init(FilterConfig filterConfig) throws ServletException {
        ipBlockSet = new HashSet<>(ipBlocklist);
        log.info("IP黑名单加载完成: {}", ipBlockSet);
    }

    @Override
    public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain)
            throws IOException, ServletException {
        
        HttpServletRequest httpServletRequest = (HttpServletRequest) request;
        String clientIp = IpUtils.getClientIpAddr(httpServletRequest);

        // 检查IP是否在黑名单中
        if (ipBlockSet.contains(clientIp)) {
            log.warn("检测到黑名单IP访问，已拦截。IP: {}, URL: {}", clientIp, httpServletRequest.getRequestURI());
            
            HttpServletResponse httpServletResponse = (HttpServletResponse) response;
            httpServletResponse.setStatus(HttpServletResponse.SC_FORBIDDEN); // 设置 403 Forbidden 状态码
            httpServletResponse.setContentType("application/json;charset=UTF-8");
            httpServletResponse.getWriter().write("{\"error\": \"Access Denied\"}");
            return; // 请求被拦截，直接返回，不调用 chain.doFilter
        }

        // IP不在黑名单中，放行请求
        chain.doFilter(request, response);
    }
}
```

**说明**：

- `@Component`: 将这个Filter注册为Spring Bean。
    
- `@Value`: 从配置文件中读取黑名单列表。
    
- `init()`: 在服务启动时将List转为HashSet，后续每次检查IP时，Set的查询性能是O(1)，远高于List的O(n)。
    
- `doFilter()`: 核心方法。获取IP，检查是否存在于`ipBlockSet`中。如果存在，则直接返回403响应，**不调用`chain.doFilter()`**，请求处理到此为止。如果不存在，则调用`chain.doFilter()`将请求传递给下一个过滤器或Servlet。

### 方案二：使用 Spring Interceptor

如果你希望拦截功能与Spring MVC的生命周期结合得更紧密（例如，你只想拦截特定Controller的请求），可以使用拦截器。

```java
// src/main/java/com/yourproject/interceptor/IpBlockInterceptor.java
package com.yourproject.interceptor;

import com.yourproject.util.IpUtils;
import lombok.extern.slf4j.Slf4j;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Component;
import org.springframework.web.servlet.HandlerInterceptor;

import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import java.util.HashSet;
import java.util.List;
import java.util.Set;
import javax.annotation.PostConstruct;

@Component
@Slf4j
public class IpBlockInterceptor implements HandlerInterceptor {

    @Value("${security.ip-blocklist}")
    private List<String> ipBlocklist;

    private Set<String> ipBlockSet;

    // Bean初始化后执行
    @PostConstruct
    public void init() {
	        ipBlockSet = new HashSet<>(ipBlocklist);
        log.info("IP黑名单拦截器加载完成: {}", ipBlockSet);
    }

    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler)
            throws Exception {
        
        String clientIp = IpUtils.getClientIpAddr(request);

        if (ipBlockSet.contains(clientIp)) {
            log.warn("检测到黑名单IP访问，已被Interceptor拦截。IP: {}, URL: {}", clientIp, request.getRequestURI());
            
            response.setStatus(HttpServletResponse.SC_FORBIDDEN);
            response.setContentType("application/json;charset=UTF-8");
            response.getWriter().write("{\"error\": \"Access Denied\"}");
            return false; // 返回false，表示请求被中断
        }

        return true; // 返回true，表示放行，请求继续由Controller处理
    }
}
```

#### 第四步：注册拦截器

拦截器需要一个配置类来注册到Spring MVC中。

Java

```java
// src/main/java/com/yourproject/config/WebMvcConfig.java
package com.yourproject.config;

import com.yourproject.interceptor.IpBlockInterceptor;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.context.annotation.Configuration;
import org.springframework.web.servlet.config.annotation.InterceptorRegistry;
import org.springframework.web.servlet.config.annotation.WebMvcConfigurer;

@Configuration
public class WebMvcConfig implements WebMvcConfigurer {

    @Autowired
    private IpBlockInterceptor ipBlockInterceptor;

    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(ipBlockInterceptor)
                .addPathPatterns("/**") // 拦截所有请求
                .excludePathPatterns("/public/**", "/login"); // 可以配置排除的路径
    }
}
```

### 对比与选择

|特性|Servlet Filter|Spring Interceptor|
|---|---|---|
|**执行时机**|在Spring `DispatcherServlet`之前，更早|在`DispatcherServlet`之后，Controller方法之前|
|**依赖**|依赖Servlet API，与框架无关|强依赖Spring MVC框架|
|**控制粒度**|只能基于URL pattern进行粗粒度拦截|可基于URL，且能获取到要执行的`HandlerMethod`（Controller方法），控制更精细|
|**适用场景**|全局性的、与业务逻辑无关的功能，如日志、编码、**安全拦截(IP墙)**|与业务处理流程相关的、需要访问Controller上下文的功能，如登录校验、权限检查|

**结论**：对于实现“IP墙”这种纯粹的、全局性的安全功能，**Servlet Filter是更佳的选择**。如果您未来的需求是针对某些特定的API做拦截，那么Spring Interceptor会更灵活。
