### 第一部分：`AutoFill.java` (自定义注解)

首先我们来解释 `@interface AutoFill`，因为它是整个功能的基础。它定义了一个“标记”，我们用这个标记来告诉 AOP 切面应该在哪些方法上执行自动填充逻辑。

Java

```java
// @Target 注解用于指定我们自定义的这个注解可以应用在哪些程序元素上。
// ElementType.METHOD 表示 @AutoFill 注解只能用于方法上。
@Target(ElementType.METHOD)

// @Retention 注解指定了该注解的生命周期。
// RetentionPolicy.RUNTIME 表示这个注解会被保留到运行时，这样我们才能在程序运行时通过反射机制读取到它。
// 这对于AOP至关重要，因为AOP正是在运行时动态地织入逻辑。
@Retention(RetentionPolicy.RUNTIME)

// public @interface AutoFill: 这行代码定义了一个名为 AutoFill 的注解。
public @interface AutoFill {
    
    // 这行在注解中定义了一个名为 "value" 的成员（或称为属性）。
    // 它的类型是 OperationType，这是一个枚举类型，可能包含了 INSERT 和 UPDATE 两个值。
    // 这个 value 属性的作用是让使用者在应用此注解时，明确指出当前方法执行的是哪种数据库操作。
    // 因为 INSERT 和 UPDATE 操作需要填充的公共字段是不同的。
    OperationType value();
}
```

**小结 `AutoFill.java`**:

这段代码定义了一个自定义注解 `@AutoFill`。

- 它只能用在方法上 (`@Target(ElementType.METHOD)`)。
    
- 它在程序运行时依然有效，可以被读取 (`@Retention(RetentionPolicy.RUNTIME)`)。
    
- 它有一个必须填写的 `value` 属性，用来指明数据库操作的类型（是插入还是更新）。
    

使用示例:

你会在 Mapper 层的方法上看到这样的用法：

Java

```java
@AutoFill(OperationType.INSERT)
void insert(Employee employee);

@AutoFill(OperationType.UPDATE)
void update(Employee employee);
```

### 第二部分：`AutoFillAspect.java` (AOP 切面类)

这个类是实现自动填充功能的核心逻辑所在。它是一个“切面”，会“监听”所有被 `@AutoFill` 注解标记的方法，并在这些方法执行前，执行一段我们预设的逻辑（即填充公共字段）。

Java

```java
package com.taotao.aspect;
// ... 导入相关的类

/**
 * 自定义切面，实现公共字段自动填充处理逻辑
 */
// @Aspect 表明这个类是一个切面类，是AOP的核心组件。
@Aspect
// @Component 将这个切面类声明为Spring容器中的一个Bean，以便Spring能够管理它并让它生效。
@Component
// @Slf4j 是 Lombok 提供的注解，它会自动为这个类生成一个名为 'log' 的 SLF4J 日志对象，方便我们打印日志。
@Slf4j
public class AutoFillAspect {

    /**
     * 定义切入点（Pointcut）
     * 切入点告诉AOP，我们的逻辑应该在“哪里”执行。
     */
    // @Pointcut 定义了一个可重用的切入点表达式。
    // "execution(* com.taotao.mapper.*.*(..))" : 这是一个执行表达式。
    //   - `*`: 匹配任意返回值类型。
    //   - `com.taotao.mapper.*`: 匹配 `com.taotao.mapper` 包下的任意类。
    //   - `.*`: 匹配类中的任意方法名。
    //   - `(..)`: 匹配任意数量和类型的参数。
    // "&& @annotation(com.taotao.common.annotation.AutoFill)" : 这是一个条件。
    //   - `&&`: 逻辑与，表示两个条件必须同时满足。
    //   - `@annotation(...)`: 目标方法必须被指定的注解（这里是`@AutoFill`）所标记。
    // 总结：这个切入点会匹配 com.taotao.mapper 包下所有被 @AutoFill 注解标记的方法。
    @Pointcut("execution(* com.taotao.mapper.*.*(..)) && @annotation(com.taotao.common.annotation.AutoFill)")
    public void autoFillPointCut(){} // 这个方法体是空的，它只作为切入点表达式的载体。

    /**
     * 定义前置通知（Before Advice）
     * 通知是切面具体要执行的动作。前置通知表示这个动作会在目标方法执行“之前”运行。
     */
    // @Before("autoFillPointCut()") 将下面的 autoFill 方法定义为一个前置通知，
    // 并指定它应用在上面定义的 autoFillPointCut 切入点上。
    @Before("autoFillPointCut()")
    public void autoFill(JoinPoint joinPoint){ // JoinPoint 对象包含了目标方法的所有信息（如方法名、参数等）。
        log.info("开始进行公共字段自动填充...");

        // 1. 获取当前被拦截方法上的数据库操作类型（INSERT 或 UPDATE）
        // joinPoint.getSignature() 获取方法签名，并强转为 MethodSignature 以获取更详细的方法信息。
        MethodSignature signature = (MethodSignature) joinPoint.getSignature();
        // 从方法签名中获取方法对象，再从方法对象上获取 @AutoFill 注解的实例。
        AutoFill autoFill = signature.getMethod().getAnnotation(AutoFill.class);
        // 调用注解的 value() 方法，获得开发者在注解中设置的操作类型。
        OperationType operationType = autoFill.value();

        // 2. 获取当前被拦截方法的参数——即实体对象（如 Employee, Dish 等）
        // joinPoint.getArgs() 返回一个包含所有方法参数的数组。
        Object[] args = joinPoint.getArgs();
        // 进行健壮性判断，如果方法没有参数，则直接返回。
        if(args == null || args.length == 0){
            return;
        }
        
        // 按照约定，实体对象是方法的第一个参数。
        Object entity = args[0];

        // 3. 准备要填充的数据
        // 获取当前时间。
        LocalDateTime now = LocalDateTime.now();
        // 从一个线程局部变量工具类 BaseContext 中获取当前登录用户的ID。
        // 这是一种常见的在不同层之间传递用户ID的方式。
        Long currentId = BaseContext.getCurrentId();

        // 4. 根据不同的操作类型，通过反射为实体对象的属性赋值
        if(operationType == OperationType.INSERT){
            // 如果是 INSERT 操作，需要为4个公共字段赋值：createTime, createUser, updateTime, updateUser
            try {
                // 使用反射获取实体类中名为 setCreateTime 且参数为 LocalDateTime 的方法对象。
                Method setCreateTime = entity.getClass().getDeclaredMethod(AutoFillConstant.SET_CREATE_TIME, LocalDateTime.class);
                Method setCreateUser = entity.getClass().getDeclaredMethod(AutoFillConstant.SET_CREATE_USER, Long.class);
                Method setUpdateTime = entity.getClass().getDeclaredMethod(AutoFillConstant.SET_UPDATE_TIME, LocalDateTime.class);
                Method setUpdateUser = entity.getClass().getDeclaredMethod(AutoFillConstant.SET_UPDATE_USER, Long.class);

                // 通过反射调用这些 setter 方法，为实体对象的属性赋值。
                setCreateTime.invoke(entity, now);
                setCreateUser.invoke(entity, currentId);
                setUpdateTime.invoke(entity, now);
                setUpdateUser.invoke(entity, currentId);
            } catch (Exception e) {
                // 打印异常堆栈信息。
                e.printStackTrace();
            }
        }else if(operationType == OperationType.UPDATE){
            // 如果是 UPDATE 操作，只需要为2个公共字段赋值：updateTime, updateUser
            try {
                // 类似地，使用反射获取并调用相应的 setter 方法。
                Method setUpdateTime = entity.getClass().getDeclaredMethod(AutoFillConstant.SET_UPDATE_TIME, LocalDateTime.class);
                Method setUpdateUser = entity.getClass().getDeclaredMethod(AutoFillConstant.SET_UPDATE_USER, Long.class);

                setUpdateTime.invoke(entity, now);
                setUpdateUser.invoke(entity, currentId);
            } catch (Exception e) {
                e.printStackTrace();
            }
        }
    }
}
```

### 整体工作流程总结

1. **定义标记**：开发者创建了 `@AutoFill` 注解，用来标记哪些数据库操作方法需要被特殊处理。
    
2. **标记方法**：在 `Mapper` 接口中，开发者给 `insert` 和 `update` 方法加上 `@AutoFill` 注解，并指明操作类型，例如 `@AutoFill(OperationType.INSERT)`。
    
3. **定义切面**：`AutoFillAspect` 这个切面类被 Spring 扫描并加载。
    
4. **拦截方法调用**：当业务代码调用一个被 `@AutoFill` 标记的 `Mapper` 方法时（例如 `employeeMapper.insert(employee)`)，Spring AOP 框架会拦截这次调用。
    
5. **执行前置通知**：在真正执行 `Mapper` 的 `insert` 方法之前，AOP 会先执行 `AutoFillAspect` 中的 `autoFill` 方法。
    
6. **执行填充逻辑**：`autoFill` 方法会：
    
    - 检查注解，发现是 `INSERT` 操作。
        
    - 获取到 `insert` 方法的参数，即 `employee` 对象。
        
    - 获取当前时间和当前用户ID。
        
    - 通过 Java 的**反射**机制，动态地找到 `employee` 对象的 `setCreateTime`, `setCreateUser`, `setUpdateTime`, `setUpdateUser` 方法，并将获取到的时间和用户ID设置进去。
        
7. **执行原始方法**：当 `autoFill` 方法执行完毕后，原始的 `employeeMapper.insert(employee)` 方法才会继续执行。此时，传入的 `employee` 对象已经被自动填充好了公共字段。
    

**核心优势**：这种方式将公共字段填充的逻辑从业务代码中完全分离出来，实现了代码的解耦和重用，让开发者可以更专注于核心业务逻辑的编写。
