### ## `sismember` 是什么命令？

`sismember` 是 Redis 中用于**集合 (Set)** 数据类型的命令。

它的作用是**判断一个指定的成员 (member) 是否存在于一个集合 (Set) 中**。

- 如果成员存在于集合中，命令返回 `1`。
    
- 如果成员不存在于集合中，或者该集合键 (key) 本身就不存在，命令返回 `0`。
    

这个命令非常高效，其时间复杂度为 O(1)，无论集合有多大，检查速度都极快。

在我们的秒杀功能中，我们就用它来高效地判断一个用户ID是否已经存在于“成功抢购”的集合中，从而实现防止重复下单的功能。

### ## 常用 Redis 命令合集 (按数据类型分类)

这里只列出命令及其核心作用，不包含详细的语法示例。

#### 一、键 (Key) 通用操作

- `DEL` (删除一个或多个键)
    
- `EXISTS` (判断键是否存在)
    
- `EXPIRE` (为一个键设置过期时间，单位秒)
    
- `TTL` (查看键的剩余过期时间，单位秒)
    
- `KEYS` (查找所有符合给定模式的键，**注意：在生产环境慎用，可能阻塞服务器**)
    
- `SCAN` (迭代数据库中的键，`KEYS` 的安全替代方案)
    
- `TYPE` (获取键所储存的值的类型)
    
- `RENAME` (重命名键)
    

#### 二、字符串 (String / 字串)

- `SET` (设置指定键的值)
    
- `GET` (获取指定键的值)
    
- `SETEX` (设置键值并指定过期时间)
    
- `SETNX` (只有在键不存在时才设置键的值)
    
- `INCR` (将键中储存的数字值增一)
    
- `DECR` (将键中储存的数字值减一)
    
- `INCRBY` (将键所储存的值加上指定的增量值)
    
- `DECRBY` (将键所储存的值减去指定的减量值)
    
- `MSET` (同时设置一个或多个键值对)
    
- `MGET` (同时获取一个或多个键的值)
    
- `APPEND` (向指定键的值追加内容)
    

#### 三、哈希 (Hash / 雜湊)

- `HSET` (将哈希表 key 中的字段 field 的值设为 value)
    
- `HGET` (获取存储在哈希表中指定字段的值)
    
- `HMSET` (同时将多个 field-value 对设置到哈希表中，**新版Redis中已不推荐，建议多次HSET**)
    
- `HMGET` (同时获取在哈希表中多个给定字段的值)
    
- `HGETALL` (获取在哈希表中指定 key 的所有字段和值)
    
- `HDEL` (删除一个或多个哈希表字段)
    
- `HKEYS` (获取哈希表中的所有字段名)
    
- `HVALS` (获取哈希表中的所有值)
    
- `HEXISTS` (查看哈希表的指定字段是否存在)
    
- `HINCRBY` (为哈希表中的字段值加上指定的增量)
    

#### 四、列表 (List / 串列)

- `LPUSH` (将一个或多个值插入到列表头部)
    
- `RPUSH` (将一个或多个值插入到列表尾部)
    
- `LPOP` (移除并获取列表的第一个元素)
    
- `RPOP` (移除并获取列表的最后一个元素)
    
- `LRANGE` (获取列表指定范围内的元素)
    
- `LLEN` (获取列表的长度)
    
- `LINDEX` (通过索引获取列表中的元素)
    
- `BLPOP` (阻塞式地移除并获取列表的第一个元素，常用于实现消息队列)
    
- `BRPOP` (阻塞式地移除并获取列表的最后一个元素)
    

#### 五、集合 (Set / 集合)

- `SADD` (向集合添加一个或多个成员)
    
- `SREM` (移除集合中一个或多个成员)
    
- `SMEMBERS` (返回集合中的所有成员)
    
- `SISMEMBER` (判断 member 元素是否是集合 key 的成员)
    
- `SCARD` (获取集合的成员数)
    
- `SPOP` (随机移除并返回集合中的一个或多个元素)
    
- `SUNION` (返回所有给定集合的并集)
    
- `SINTER` (返回所有给定集合的交集)
    
- `SDIFF` (返回所有给定集合的差集)
    

#### 六、有序集合 (Sorted Set / ZSet / 有序集合)

- `ZADD` (向有序集合添加一个或多个成员，或者更新已存在成员的分数)
    
- `ZREM` (移除有序集合中的一个或多个成员)
    
- `ZRANGE` (通过索引区间返回有序集合成中指定区间内的成员，按分数升序)
    
- `ZREVRANGE` (通过索引区间返回有序集合成中指定区间内的成员，按分数降序)
    
- `ZRANGEBYSCORE` (通过分数区间返回有序集合中指定区间内的成员)
    
- `ZCARD` (获取有序集合的成员数)
    
- `ZSCORE` (返回有序集中，成员的分数值)
    
- `ZINCRBY` (为有序集合中指定成员的分数加上增量)
    
- `ZCOUNT` (计算在有序集合中指定分数区间的成员数)
    

#### 七、服务器/连接 (Server/Connection)

- `PING` (检查服务是否正在运行，通常返回 "PONG")
    
- `FLUSHDB` (清空当前数据库中的所有 key)
    
- `FLUSHALL` (清空所有数据库中的所有 key)
    
- `INFO` (获取 Redis 服务器的各种信息和统计数值)
