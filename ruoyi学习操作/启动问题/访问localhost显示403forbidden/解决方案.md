好的，你提供的日志是 Nginx 的**启动日志**，它显示 `Configuration complete; ready for start up`。这说明 Nginx 容器**已经成功启动了**，日志本身没有问题。

你访问 `localhost:80` 报 403 Forbidden，问题几乎可以肯定是 Nginx **找不到要显示的前端页面文件**。

---

## 诊断：为什么是 403？

1. 你看一下你的 `docker-compose.yml` 文件中 `ruoyi-nginx` 服务的 `volumes` 配置：
    
    YAML
    
    ```
    volumes:
      - ./nginx/html/dist:/home/ruoyi/projects/ruoyi-ui
      # ... 其他挂载
    ```
    
2. 这行配置的意思是：把**你电脑本地的 `./nginx/html/dist` 目录** 挂载到 Nginx 容器内部的 `/home/ruoyi/projects/ruoyi-ui` 目录。
    
3. Nginx 的配置文件 (`nginx.conf`) 已经被设置好了，会去 `/home/ruoyi/projects/ruoyi-ui` 目录里寻找 `index.html` 文件来展示给用户。
    
4. **问题的关键：** 如果你**本地的 `./nginx/html/dist` 目录是空的**，或者根本不存在，那么 Nginx 容器里的 `/home/ruoyi/projects/ruoyi-ui` 目录也就是空的。
    
5. 当 Nginx 找不到 `index.html` 文件，并且默认配置下又禁止显示目录列表时，就会返回 `403 Forbidden` 错误。
    

---

## 解决方案

你需要**先编译前端项目 (RuoYi-UI)，然后把编译好的文件放到 Nginx 的挂载目录中**。

1. **编译前端项目**：
    
    - 进入你的 RuoYi-UI 前端项目源代码目录（一般叫 `ruoyi-ui`）。
        
    - 运行 `npm install` 安装依赖。
        
    - 运行 `npm run build:prod` 来编译项目。
        
2. **复制编译产物**：
    
    - 编译成功后，你的 `ruoyi-ui` 目录里会出现一个 `dist` 文件夹。
        
    - **把这个 `dist` 文件夹里的所有内容** (包括 `index.html` 和 `static` 文件夹等) **复制**到你 `docker-compose.yml` 文件所在的 `./nginx/html/dist` 目录中。
        
3. **重启 Nginx 容器**：
    
    - 在 `docker-compose.yml` 所在的目录运行命令，单独重启 Nginx 服务：
        
        Bash
        
        ```
        docker-compose restart ruoyi-nginx
        ```
        
    - 或者，如果你是 `docker-compose up` 运行的，可以按 `Ctrl+C` 停止，然后重新 `docker-compose up -d`。
        
4. **验证**：
    
    - 清除浏览器缓存，然后重新访问 `http://localhost:80`。
        

现在 Nginx 应该就能找到 `index.html` 文件并正确显示登录页面了。
