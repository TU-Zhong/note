在计算机专业的毕业论文中，**系统架构设计如果不画图，基本等同于“没写”**。

在你的湘潭大学计算机学院论文模板中，虽然文字描述是基础，但**“图”才是架构设计的灵魂** 。对于你这种涉及多数据源、动态加载和复杂前端状态的项目，图表具有以下几个不可替代的作用：

### 1. 满足“复杂工程问题”的评审要求

你的论文模板明确要求体现“复杂工程问题”的特征，其中第（3）条提到：**“需要通过建立合适的抽象模型才能解决”** 。

- **文字是具象的，图才是抽象模型**。一张清晰的架构图能直接向评委展示你如何通过抽象层（如 JDBC 规范、SPI 接口）来解决驱动冲突这一复杂问题。
    

### 2. 降低导师和评委的理解成本

评委在答辩时通常只有几分钟扫视你的论文。如果你用三页纸描述 `DriverLoaderService` 是如何工作的，没人看得完；但如果你画一张**时序图（Sequence Diagram）**，展示从前端点击连接到后端 SPI 动态加载 JAR 包的全过程，逻辑一目了然。

### 3. 填补篇幅与美化排版

模板中专门留出了“图1-1 图名”的位置，并要求有图片来源说明 。一份全是文字的论文会显得极其枯燥且缺乏工程实践感，合理的图表布局能显著提升论文的专业度。

### 💡 针对你的项目，建议画这三张图：

#### ① 系统总体逻辑架构图（必选）

展示系统的层次结构：

- **展示层**：Vue 3 + Element Plus (Diff Pool 逻辑)。
    
- **业务层**：Spring Boot 控制器与服务。
    
- **核心驱动层**（你的亮点）：自定义 URLClassLoader + SPI 接口。
    
- **数据层**：各版本的 MySQL、SQL Server 等。
    

#### ② 驱动动态加载流程图/时序图（亮点图）

专门描述 `DriverLoaderService` 的工作原理：

1. 前端传回 `jarPath` 和 `url`。
    
2. `URLClassLoader` 加载目标 JAR 包。
    
3. `ServiceLoader` 扫描 `java.sql.Driver` 实现类。
    
4. `acceptsURL` 匹配并返回连接。
    

#### ③ 前端数据状态机图（针对 Diff-Pool）

展示一条数据在前端的生命周期：

- **Original 态** -> **Modified 态**（变黄） -> **Deleted 态**（删除线） -> **Synced 态**（刷新）。
    
- 这能完美对应你代码中 `diffPool` 的 `Set` 操作逻辑。
