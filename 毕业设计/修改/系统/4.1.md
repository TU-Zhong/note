## 4.1 多源异构数据库动态挂载实现

本模块是系统的核心入口，旨在解决 Web 端对异构数据库（如 MySQL、SQL Server）驱动的动态识别、隔离加载及物理连接的高效调度问题。系统实现了从驱动 JAR 包上传到逻辑会话（Token）建立的全链路闭环。

### 4.1.1 交互接口与元数据凭证封装

在系统前端，用户通过统一的配置界面输入数据库连接的元数据。为了保证前后端数据交互的结构化与类型安全，系统设计了 `ConnectionDTO` 实体类作为数据载体。

### 4.1.2 基于 SPI 的驱动隔离加载机制

由于不同数据库甚至同一数据库的不同版本驱动可能存在类冲突，本研究通过 `DriverLoaderService` 实现了基于自定义类加载器的隔离方案。

- **打破双亲委派**：利用 `URLClassLoader` 载入用户指定的物理驱动文件，确保驱动类在独立的命名空间中实例化，避免了 JVM 层面的版本碰撞。
    
- **SPI 自动发现**：通过 Java 原生的 `ServiceLoader` 机制动态扫描 JAR 包内的 `java.sql.Driver` 实现类，无需用户手动输入驱动全限定名，提升了系统的智能化程度。
    

代码 4-2：动态驱动识别核心逻辑：

Java

```
// 1. 创建自定义类加载器
URLClassLoader ucl = new URLClassLoader(urls, Thread.currentThread().getContextClassLoader()); 

// 2. 利用 SPI 机制自动扫描并实例化驱动
ServiceLoader<Driver> driverLoader = ServiceLoader.load(Driver.class, ucl);
for (Driver driver : driverLoader) {
    if (driver.acceptsURL(url)) { // 自动匹配对应的数据库方言驱动[cite: 2]
        targetDriver = driver;
        break;
    }
}
```

### 4.1.3 高性能连接池托管与会话路由

在成功识别驱动后，系统并不直接建立单次物理连接，而是实例化 `HikariDataSource` 连接池进行资源托管。

- **物理资源配置**：配置核心参数如 `MaximumPoolSize: 20`（支持最高 20 个并发连接）及 `ConnectionTimeout: 30s`，确保高并发场景下资源分配的稳健性[cite: 2]。
    
- **会话 Token 映射**：连接池建立后，系统调用 `ConnectionCathe` 生成唯一的 UUID 字符串作为会话 Token。该 Token 将作为后续所有数据操作的路由键，存入 Caffeine 本地缓存中，实现了前端逻辑会话与后端物理连接的精准绑定。

### 4.1.4 资源安全回收与防御闭环

为了防止多租户环境下连接泄露导致的服务器宕机，系统构建了双重防御体系：

1. **自动资源卸载**：`ConnectionCathe` 配置了基于存活权标的淘汰机制。通过 `expireAfterAccess(30, Minutes)`，一旦某连接池在 30 分钟内无活跃调用，监听器会自动触发 `ds.close()` 释放物理 TCP 连接，从源头杜绝了“僵尸连接”对资源的占用。
    
2. **全局异常拦截**：通过 `GlobalExceptionHandler` 统一接管挂载过程中可能出现的 `ClassNotFoundException` 或连接超时异常，通过 `Result.error` 格式返回友好的交互提示，隐藏了系统底层的堆栈细节，增强了系统的安全性。
