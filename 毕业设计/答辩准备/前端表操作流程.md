在这套基于 Vue 3 Composition API 的系统中，表格的数据流转、UI 交互和状态管理被非常优雅地拆分到了视图层（`TableDataViewer.vue`）和逻辑层（`useTableUI.js`, `useTableDiff.js`）中。

这里为你详细梳理**单击**和**双击**操作背后的全流程状态流转和 UI 变化。一切的核心锚点是每一行数据独有的虚拟主键：`__row_id`。

---

### 一、 单击流程：选中与单元格高亮 (Single Click)

当你单击表格中的某一个单元格时，主要触发的是“选中”态的 UI 变化。

#### 1. 触发事件

在 `TableDataViewer.vue` 中，`el-table` 绑定了两个点击事件：

- `@row-click="handleCurrentRowChange"`
    
- `@cell-click="handleCellClick"`
    

#### 2. 状态变更 (位于 `useTableUI.js`)

- **`handleCurrentRowChange(row)`**：
    
    将当前点击的整行数据赋值给响应式变量 **`currentRow`**。这个变量主要用于后续判断“删除”操作时，知道该删哪一行。
    
- **`handleCellClick(row, column)`**：
    
    将当前点击的单元格坐标存入响应式变量 **`selectedCell`**。
    
    JavaScript
    
    ```
    selectedCell.value = {
        rowId: row.__row_id,      // 行的唯一标识
        colKey: column.property   // 列的字段名
    }
    ```
    

#### 3. UI 样式响应

状态改变后，Vue 的响应式机制会触发 `el-table` 重新计算类名：

- **计算单元格类名 (`getCellClassName`)**：
    
    遍历渲染每个单元格时，判断当前单元格的坐标是否和 `selectedCell.value` 一致。如果一致，返回 CSS 类名 `'cell-selected-highlight'`。
    
- **样式呈现**：
    
    视图层的 `<style>` 捕获到该类名，应用高亮边框和背景色：
    
    CSS
    
    ```
    box-shadow: inset 0 0 0 2px #f483ad !important; /* 粉红色内阴影边框 */
    background-color: #10324a !important;           /* 深色背景 */
    ```
    

---

### 二、 双击流程：进入沉浸式编辑态 (Double Click)

双击的核心是将普通的文本展示（`<span>`）瞬间替换为输入框（`<el-input>`）。

#### 1. 触发事件

`el-table` 触发双击事件：`@cell-dblclick="handleCellDbClick"`

#### 2. 拦截与校验 (位于 `useTableUI.js`)

在 `handleCellDbClick` 内部，系统会先做两道拦截：

- **拦截1**：检查该行是否在 **`diffPool.deleted`** (删除池) 中，如果是，直接 `return`（被删除的数据不允许编辑）。
    
- **拦截2**：检查该列是否是自增列/主键列（`type.includes('identity')`），如果是，弹出警告并 `return`（数据库自动分配的列不可手动修改）。
    

#### 3. 状态变更：开启编辑

如果通过校验，系统会更新响应式变量 **`editingCell`**：

JavaScript

```
editingCell.value = {
    rowId: row.__row_id,
    colKey: column.property
}
```

#### 4. UI 响应：组件切换

在 `TableDataViewer.vue` 的列模板中，有一个 `v-if` 判断：

HTML

```
<template v-if="editingCell.rowId === scope.row.__row_id && editingCell.colKey === col.columnName">
    </template>
<template v-else>
    </template>
```

由于 `editingCell` 更新，`v-if` 条件成立，文本变成输入框。同时，利用 Vue 的钩子 `@vue:mounted="vnode => vnode.el.querySelector('input').focus()"`，**输入框一出现就会自动获取光标**，让用户直接可以打字。

---

### 三、 失去焦点 / 回车流程：差异对比与行变色 (Blur / Enter)

用户修改完数据，点击其他地方或按下回车时，触发数据比对和行状态变更。

#### 1. 触发事件

`<el-input>` 绑定了失去焦点和回车事件：`@blur="handleInputBlur(scope.row)"` 和 `@keyup.enter="handleInputBlur(scope.row)"`

#### 2. 状态变更：退出编辑态 (位于 `useTableUI.js`)

`handleInputBlur` 函数首先会清空编辑状态：

JavaScript

```
editingCell.value = { rowId: null, colKey: null }
```

UI 立即响应，输入框消失，变回普通的 `<span>` 显示最新修改的文本。

#### 3. 核心差异比对 (调用 `diffCore.checkRowDiff(row)`)

接着，调用 `useTableDiff.js` 中的底层对比逻辑：

- **豁免判定**：如果这行数据本身就是刚新增的（存在于 `diffPool.inserted`），不需要进入修改池，直接结束。
    
- **找原数据**：通过 `__row_id` 去 **`originalData`** (刚从后端拉取的原始数据快照) 中找到对应的原始行。
    
- **全字段比对**：遍历该行的所有列，将当前值和原始值都转为字符串进行比对（防止 JavaScript 隐式转换报错）。
    
- **进出更新池**：
    
    - 如果发现不一致：把该行的 `__row_id` 加入 **`diffPool.updated`** 集合。
        
    - 如果用户又改回了原来的值（一致）：把该行的 `__row_id` 从 `diffPool.updated` 集合中剔除。
        

#### 4. UI 响应：行背景变色

因为 `diffPool.updated` 发生了变化，触发 `el-table` 的行类名计算方法：

- **计算行类名 (`getRowClassName`)**：
    
    判断 `diffPool.updated.has(row.__row_id)` 是否为真，若为真，返回类名 `'row-updated'`。
    
- **样式呈现**：
    
    视图层的 `<style>` 捕获到该类名，整行背景变色：
    
    CSS
    
    ```
    background-color: #ffefd8 !important; /* 浅黄色/橙色，提示用户这行被修改了还没保存 */
    ```

### 💡 核心变量总结清单

为了方便你全局理解，这里总结了贯穿整个流程的 **6 个核心变量**：

1. **`__row_id`**: 每一行数据的唯一前端虚拟 ID。所有的选中、编辑、对比，全靠它来精确定位。
    
2. **`originalData`** (`useTableDiff`): 数据的原始快照，用来对比到底有没有发生真正的修改。
    
3. **`diffPool`** (`useTableDiff`): 差异状态机，包含 `inserted`, `updated`, `deleted` 三个 Set，记录哪些行的 `__row_id` 发生了什么变化。
    
4. **`currentRow`** (`useTableUI`): 记录当前点击的是哪一行（主要为了提供给删除按钮使用）。
    
5. **`selectedCell`** (`useTableUI`): `{rowId, colKey}`，控制哪个单元格显示被“选中”的高亮边框。
    
6. **`editingCell`** (`useTableUI`): `{rowId, colKey}`，控制哪个单元格显示 `el-input` 输入框。
