<script setup>
import { ref } from 'vue'

// 1. 定义待办事项列表
// Java 类比: List<Task> todoList = new ArrayList<>();
const todoList = ref([
  { id: 1, content: '学习 Vue 3' },
  { id: 2, content: '跑步 5 公里' }
])

// 2. 定义输入框绑定的变量
const newTodo = ref('')

// 3. 添加功能
function addTodo() {
  // 简单的非空校验 (StringUtils.hasText)
  if (!newTodo.value.trim()) {
    alert("内容不能为空！")
    return
  }

  // 往数组里“追加”一个对象
  // 注意：在 Script 里操作 ref 数组，要用 todoList.value.push
  todoList.value.push({
    id: Date.now(), // 用时间戳模拟唯一 ID
    content: newTodo.value
  })

  // 清空输入框
  newTodo.value = ''
}

// 4. 删除功能 (额外赠送的功能，列表通常都需要删除)
function removeTodo(id) {
  // Java 类比: list.removeIf(item -> item.getId() == id);
  // JS 中常用 filter 来过滤掉要删除的项，然后重新赋值
  todoList.value = todoList.value.filter(item => item.id !== id)
}
</script>

<template>
  <div class="container">
    <h2>待办清单 (v-for 循环)</h2>

    <div class="input-box">
      <input 
        v-model="newTodo" 
        @keyup.enter="addTodo" 
        placeholder="输入待办事项，按回车添加"
      />
      <button @click="addTodo">添加</button>
    </div>

    <ul>
      <li v-for="item in todoList" :key="item.id">
        <span>{{ item.content }}</span>
        
        <button class="del-btn" @click="removeTodo(item.id)">删除</button>
      </li>
    </ul>

    <div v-if="todoList.length === 0" class="empty-tip">
      恭喜！所有任务都完成了 🎉
    </div>
  </div>
</template>

<style scoped>

### 🧐 核心知识点解析

#### 1. `v-for` (循环渲染)

- **写法**：`v-for="item in list"`。这和 Java 的 `for (Item item : list)` 逻辑一模一样。
    
- **`:key` (唯一标识)**：
    
    - 这是面试必问，也是实战必坑点。
        
    - Vue 的渲染引擎是“偷懒”的，如果你不给每个 `<li>` 一个身份证号（key），当你删除了第 1 项，Vue 可能会复用第 1 项的 DOM 节点来显示第 2 项的数据，导致错乱。
        
    - **原则：** 永远使用数据库 ID 或唯一值作为 key，**尽量别用索引 index**。
        

#### 2. 数组操作

- 在 Java 里你习惯 `list.add()` 和 `list.remove()`。
    
- 在 Vue 3 (JS) 里：
    
    - 添加：`list.value.push(obj)`
        
    - 删除：通常使用“过滤法”，即 `list.value = list.value.filter(...)`，这相当于把“不要那个元素的新数组”重新赋值回去。
        

#### 3. 关于 `defineEmits` (补充说明)

你的题目里提到了 `defineEmits`。在上面的代码中，因为我们把所有逻辑都写在了一个文件 (`App.vue`) 里，所以用不到它。

什么时候用到 defineEmits？

当你把每一行 <li> 拆分成一个独立的组件 TodoItem.vue 时：

1. **父组件 (App.vue)**：循环调用 `<TodoItem />`。
    
2. **子组件 (TodoItem.vue)**：里面有一个“删除按钮”。
    
3. **场景**：子组件里的按钮被点了，但数据在父组件里。子组件必须**打电话**通知父组件：“喂，ID 为 5 的这条数据要删掉”。
    
4. 这时候子组件就需要用 `const emit = defineEmits(['delete'])`，然后执行 `emit('delete', 5)`。
