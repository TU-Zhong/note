这份计划表是专门为你“零基础”量身定制的。我将这14天分为两个阶段：**第一周搭建静态骨架**，**第二周实现数据交互逻辑**。

目标不是让你成为前端工程师，而是**造出毕设需要的那个“壳”**。

### 📅 第一周：环境搭建与静态页面 (把界面画出来)

**目标：** 电脑上能运行 Vue 项目，且浏览器里能看到左侧菜单树、右侧内容区的布局。

- **Day 1: 环境准备与 Hello World**
    命令：
    ```bash
    # 此时确保你的路径是以 ...\my-vue-project 结尾
	docker run --rm -it -v ${PWD}:/app -w /app node:22-alpine npm create vue@latest .
	#安装依赖vite和vue等库
	docker run --rm -it -v ${PWD}:/app -w /app node:22-alpine npm install 
	#启动
	docker run --rm -it -p 5173:5173 -v ${PWD}:/app -w /app node:22-alpine npm run dev -- --host
    ```
    - **任务：**
        
        1. 安装 Node.js (推荐 LTS 版本)。
            
        2. 安装编辑器 VS Code，并安装插件 `Vue - Official` (以前叫 Volar)。
            
        3. 运行命令 `npm init vue@latest` 创建项目（项目名建议 `metadata-web`）。
            
        4. 在项目目录下运行 `npm install` 和 `npm run dev`。
            
    - **验收：** 浏览器打开 `http://localhost:5173` 能看到 Vue 的绿色欢迎界面。
        
- **Day 2: 引入 Element Plus**
    
    - **任务：**
        ```bash
        #对应docker安装这个组件库
        docker run --rm -it -v ${PWD}:/app -w /app node:22-alpine npm install element-plus
        
        #重新启动
        docker run --rm -it -p 5173:5173 -v ${PWD}:/app -w /app node:22-alpine npm run dev -- --host
        ```
    
        1. 停止运行项目 (Ctrl+C)，运行 `npm install element-plus`。
            
        2. 修改 `main.js`，全局引入 Element Plus 样式（参考官方文档“快速开始”章节）。
            
        3. 清理 `App.vue`，删掉默认代码，只留一个 `<template><el-button>测试按钮</el-button></template>`。
            
    - **验收：** 页面变白，只显示一个蓝色的 Element 风格按钮。
        
- **Day 3: Vue 3 核心语法 (只学这三板斧)**
    [[毕业设计/大致环节/计划/第一周/day2/计划|计划]]
    - **任务：**
    小demo:
    ==计数器，简易表单，待办清单==
    [[计数器]]
    [[员工入职信息录入]]
    [[待办清单]]
        1. 学习 `<script setup>` 写法。
            
        2. 学习 `ref` 定义变量 (例如：`const name = ref('何翊')`)。
            
        3. 学习 `{{ }}` 插值表达式 (例如：`<div>{{ name }}</div>`)。
            
    - **验收：** 点击按钮，让页面上的文字发生变化 (如从 "未连接" 变成 "已连接")。
        
- **Day 4: 页面整体布局 (Container)**
    
    - **任务：**
        
        1. 查阅 Element Plus 文档中的 `Container` 布局容器。
            
        2. 选择“倒T型”布局：Header (顶栏) + Aside (侧边栏) + Main (主体)。
            
        3. 在 `App.vue` 中把这个结构写死。
            
    - **验收：** 页面出现明显的三个区域，可以用不同的背景色区分开。
        
- **Day 5: 侧边栏——静态树形菜单 (Tree)**
    
    - **任务：**
        
        1. 这是你毕设的核心组件。查阅 Element Plus `Tree` 树形控件文档 1。
            
        2. 复制文档中的“基础用法”代码到你的 `Aside` 区域。
            
        3. 手动把数据改成数据库的样子：`School_DB` -> `Table` -> `Student` 2。
            
    - **验收：** 左侧能看到一个可以展开/折叠的数据库层级结构。
        
- **Day 6: 主体区域——表格展示 (Table)**
    
    - **任务：**
        
        1. 查阅 Element Plus `Table` 表格组件文档。
            
        2. 在 `Main` 区域放入一个表格，列头定义为：字段名、类型、主键、非空 3。
            
        3. 造几条假数据（JSON格式）填进去。
            
    - **验收：** 右侧显示出一个漂亮的元数据详情表格。
        
- **Day 7: 第一周复盘与美化**
    
    - **任务：**
        
        1. 调整一下宽度、高度，让布局占满整个屏幕。
            
        2. 把代码中没用的注释删掉。
            
    - **验收：** 截图保存，你已经完成了毕设的前端UI框架。

### 📅 第二周：数据绑定与 Axios 交互 (让界面动起来)

**目标：** 理解 Vue 是如何通过 Axios 拿到数据并更新界面的（虽然暂时用假数据模拟）。

- **Day 8: 深入 Vue 响应式 (v-for)**
    
    - **任务：**
        
        1. 学习 `v-for` 指令，这是用来循环显示列表的。
            
        2. 将 Day 5 写死的树形数据，提取到 `<script>` 中的一个 `ref` 数组里。
            
        3. 在 `<template>` 中使用变量绑定，而不是写死文字。
            
    - **验收：** 修改 JS 里的数组数据，保存后，页面上的树自动更新，不需要刷新浏览器。
        
- **Day 9: 事件监听 (Click)**
    
    - **任务：**
        
        1. 学习 `@node-click` 事件（Tree 组件自带的）。
            
        2. 写一个函数 `handleNodeClick`，当点击左侧树的“学生表”节点时，在控制台 (`console.log`) 打印出这个节点的名字。
            
    - **验收：** 打开浏览器 F12 控制台，点击树节点，能看到日志输出。
        
- **Day 10: 引入 Axios**
    
    - **任务：**
        
        1. 运行 `npm install axios` 4。
            
        2. 在 `App.vue` 中 `import axios from 'axios'`。
            
        3. 理解“异步”概念：前端发请求 -> 等待 -> 拿到数据 -> 更新页面。
            
    - **验收：** 项目编译不报错。
        
- **Day 11: 模拟后端数据 (Mock)**
    
    - **任务：**
        
        1. 因为后端还没写，我们在 `public` 文件夹下新建一个 `mock.json` 文件。
            
        2. 把 PPT 里提到的数据库元数据结构（表、字段信息）写进这个 JSON 文件 5。
            
    - **验收：** 浏览器访问 `http://localhost:5173/mock.json` 能看到数据。
        
- **Day 12: 发送第一个 HTTP 请求**
    
    - **任务：**
        
        1. 使用 `axios.get('/mock.json')` 读取昨天的文件。
            
        2. 在 `onMounted` (Vue 的生命周期钩子，指页面加载完毕时) 中发送请求。
            
        3. 把拿到的结果赋值给定义好的 `ref` 变量。
            
    - **验收：** 刷新页面，表格里的数据不再是写死的，而是从 `mock.json` 里读取出来的。
        
- **Day 13: 前端交互逻辑打通**
    
    - **任务：**
        
        1. 实现逻辑：点击左侧“树节点” -> 触发函数 -> (模拟)去后台拿该表的详情 -> 更新右侧“表格”。
            
        2. 虽然现在是读静态文件，但逻辑和读真实后端是一模一样的。
            
    - **验收：** 整个流程跑通，虽然数据是假的，但交互体验已经和最终成品一样了。
        
- **Day 14: 总结与整理**
    
    - **任务：**
        
        1. 将代码整理好，提交到 Git (如果会用的话)。
            
        2. **信心建设：** 此时你已经掌握了毕设前端 80% 的工作量。剩下的 20% 只是把 `mock.json` 的 URL 换成将来 Spring Boot 写的真实接口地址 `http://localhost:8080/api/...`。

### 💡 给你的学习建议

1. **遇到报错怎么办？**
    
    - 不要慌。把红色的报错信息直接复制给我（Gemini），或者复制到 Google。前端的报错通常很直观。
        
2. **不要买书，不要看长视频。**
    
    - 直接看 **Element Plus 官方文档** 的“代码示例”。你需要什么组件（按钮、输入框、树、表格），就去复制它的代码，然后改改字。这是最快的“速成”法。
        
3. **VS Code 插件：**
    
    - 一定要装 **"Vue - Official"** 插件，它能让你的代码高亮，还能提示你哪写错了。
