学习 `<script setup>` 是目前 Vue 3 开发的**绝对主流**，也是最高效的方式。

对于你（熟悉 Java/Spring Boot）来说，你可以把它理解为：**从“填写配置表格”变成了“写直观的逻辑代码”**。

以下是一份为你定制的学习路线和核心知识点总结。

---

### 第一阶段：核心概念（必学）

这 4 个概念覆盖了你 80% 的日常开发工作。

#### 1. 变量定义 (Ref & Reactive)

在 `<script setup>` 中，变量不是直接声明就行的，必须让 Vue “监控”到它的变化，这样页面才会自动更新。

- **`ref` (推荐)**：万能钥匙。不管是数字、字符串还是对象都能用。
    
    - **注意：** 在 JS 代码里修改值要加 `.value`，在 HTML 模板里不用加。
        
- **`reactive`**：专门用于对象/数组。不用写 `.value`，但解构会丢失响应性（新手易踩坑，建议先只用 `ref`）。
    

Code snippet

```
<script setup>
import { ref } from 'vue'

// 定义响应式变量
const count = ref(0) // 类似于 private Integer count = 0;
const name = ref("TuZhong")

// 定义方法 (直接写函数即可)
function increment() {
  count.value++ // ⚠️ JS里必须加 .value
  console.log(count.value)
}
</script>

<template>
  <h1>{{ name }} 的计数: {{ count }}</h1>
  <button @click="increment">+1</button>
</template>
```

#### 2. 父子传参 (defineProps)

类似于 Java 方法的入参。子组件接收父组件传来的数据。

- **不需要 import**，直接用。
    
- **类型安全**：可以使用 TypeScript 定义类型。
    

Code snippet

```
<script setup>
// 定义接收参数
const props = defineProps({
  title: String,
  userInfo: Object
})

console.log(props.title)
</script>
```

#### 3. 抛出事件 (defineEmits)

类似于 Java 的回调或者观察者模式。子组件通知父组件“我这边发生事情了”。

Code snippet

```
<script setup>
// 定义可以抛出的事件名
const emit = defineEmits(['submit', 'delete'])

function confirm() {
  // 触发 'submit' 事件，并携带参数
  emit('submit', { id: 1, status: 'ok' })
}
</script>
```

#### 4. 生命周期 (onMounted)

最常用的是 onMounted。它等同于 Spring 的 @PostConstruct 或者 JS 的 window.onload。

场景： 页面加载完立刻去后端请求数据。

Code snippet

```
<script setup>
import { onMounted } from 'vue'

onMounted(() => {
  console.log('组件挂载完毕，发送 API 请求...')
  // fetchUserData()
})
</script>
```

### 第二阶段：进阶逻辑（常用）

#### 1. 计算属性 (computed)

类似于 Java Bean 中的 getFoo() 方法，或者数据库的视图。

特点： 它有缓存。如果它依赖的变量没变，它就不会重新计算。

Code snippet

```
<script setup>
import { ref, computed } from 'vue'

const price = ref(10)
const count = ref(3)

// 自动计算总价，只要 price 或 count 变了，total 自动变
const total = computed(() => price.value * count.value)
</script>
```

#### 2. 监听器 (watch)

类似于“触发器”。当某个变量变了，我要做点**副作用**（比如存 LocalStorage，或者打日志）。

Code snippet

```
<script setup>
import { ref, watch } from 'vue'

const keyword = ref('')

// 只要 keyword 变了，就执行 console.log
watch(keyword, (newVal, oldVal) => {
  console.log(`正在搜索: ${newVal}`)
})
</script>
```

### 第三阶段：你需要转变的“Java 思维”

作为后端开发者，这两个点最容易让你困惑：

1. **`.value` 陷阱：**
    
    - 在 `<script>` 标签里，操作 `ref` 定义的变量**必须**加 `.value`（`count.value = 1`）。
        
    - 很多人写顺手了会漏掉，导致 bug。
        
2. **顶层绑定：**
    
    - 以前 Vue 2 需要把变量写在 `return {}` 里才能给 HTML 用。
        
    - 现在 `<script setup>` 里定义的**任何**变量、函数、import 进来的包，**自动**可以在 HTML 模板里直接使用，不需要导出。
