你写的这个文件就是一个完整的 **Vue 组件**。要“使用”它，你需要做三件事：

1. **修正一个小错误**：在你第二个 `<span>` 中，你仍然使用了 `{{ }}`，这会导致 `maliciousHtml` 仍然被显示为纯文本。
    
2. **创建文件**：把它保存到一个 `.vue` 文件中（例如 `TestView.vue`）。
    
3. **配置路由**：告诉 `router` 在哪个 URL 路径下显示这个组件。
    

让我们一步一步来：

---

### 步骤一：修正你代码中的一个小错误

你的目标是想对比 `{{ }}` 和 `v-html` 的区别，但是你在两个地方都用了 `{{ }}`。

请把你的代码**修改**成下面这样，注意看 `v-html` 的使用：

Code snippet

```
<script setup lang="ts">
const data = {
  basicHtml: '<i>这是一段斜体文本</i>',
  maliciousHtml: '这是一个<script>alert("XSS 攻击！");<\/script>尝试'
}
</script>

<template>
  <div class="content">
    <h3>--- 示例开始 ---</h3>

    <p>
      <strong>文本插值：</strong>
      <span>{{ data.basicHtml }}</span>
    </p>

    <p>
      <strong>v-html 插值：</strong>
      <span v-html="data.maliciousHtml"></span>
    </p>

    <p>
      <strong>maliciousHtml (用文本插值)：</strong>
      <span>{{ data.maliciousHtml }}</span>
    </p>

    <h3>--- 示例结束 ---</h3>
  </div>
</template>

<style scoped>
/* 加一点样式，让它更清晰 */
.content {
  font-family: sans-serif;
  border: 2px solid #eee;
  padding: 20px;
}
p {
  line-height: 1.6;
}
strong {
  color: #333;
}
span {
  background-color: #f5f5f5;
  padding: 2px 5px;
  border-radius: 4px;
}
</style>
```

### 步骤二：创建文件并配置路由

现在，我们把这个修正后的组件作为一个“页面”添加到你的应用中。

#### 1. 保存 `.vue` 文件

1. 根据我们之前的讨论，在 `src/` 目录下创建一个新文件夹，命名为 `views`。
    
2. 在 `src/views/` 文件夹内，创建一个新文件，命名为 `TestView.vue`。
    
3. 把上面**修正后**的完整代码（`<script>`, `<template>`, `<style>`）**粘贴**到 `TestView.vue` 中并保存。
    

#### 2. 配置 `router`

1. 打开你项目中的 `src/router/index.ts` 文件。
    
2. 你需要做两处修改：
    
    - **导入**你刚创建的 `TestView.vue`。
        
    - 在 `routes` 数组中为它**添加一条新规则**。
        

你的 `src/router/index.ts` 文件看起来会像这样：

TypeScript

```
import { createRouter, createWebHistory } from 'vue-router'
// 假设你已经有一个 HomeView
import HomeView from '../views/HomeView.vue'

// 1. 在这里导入你新创建的组件
import TestView from '../views/TestView.vue' 

const router = createRouter({
  history: createWebHistory(import.meta.env.BASE_URL),
  routes: [
    {
      path: '/',
      name: 'home',
      component: HomeView
    },
    // { ... 其他路由 ... }

    // 2. 在这里添加一条新规则，让 /test 路径指向 TestView 组件
    {
      path: '/test', // 你希望在浏览器地址栏输入的路径
      name: 'test',  // 一个唯一的路由名字
      component: TestView // 告诉路由使用这个组件
    }
  ]
})

export default router
```

### 步骤三：检查 `App.vue`（重要！）

你的 `router` 需要知道在哪里显示它所管理的页面（比如 `HomeView` 或 `TestView`）。它会寻找一个叫 `<router-view>` 的特殊标签。

1. 打开你的 `src/App.vue` 文件。
    
2. 确保它**至少包含** `<router-view />` 标签。一个最基础的 `App.vue` 可能是这样的：
    

Code snippet

```
<template>
  <router-view />
</template>
```

_（你也可以在 `<router-view>` 旁边添加导航栏、页眉、页脚等。)_

### 步骤四：运行和查看结果

1. 在你的项目终端中，运行开发服务器（通常是 `npm run dev`）。
    
2. 打开浏览器，访问你配置的路径：
    
    - 访问 `http://localhost:5173/` (或你本地服务器的地址)，你会看到 `HomeView`。
        
    - 访问 `http://localhost:5173/test`，你就会看到你刚写的 `TestView.vue` 组件了！
        

当你访问 `/test` 页面时，你将会看到：

- `basicHtml` 被正确渲染为_斜体_。
    
- `maliciousHtml` 在使用 `v-html` 时，会**弹出一个警告框**（"XSS 攻击！"），并且页面上只显示 "这是一个尝试"。
    
- `maliciousHtml` 在使用 `{{ }}` 时，会**安全地显示**完整的字符串，包括 `<script>` 标签文本，**不会**触发弹窗。
    

这下你就亲眼验证了 `v-html` 的效果和风险！
