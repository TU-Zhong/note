```java
// 1. 创建一个固定大小为2的线程池

ExecutorService executorService = Executors.newFixedThreadPool(2);

// 2. 提交任务到线程池执行

executorService.execute(() -> System.out.println("6. 线程池创建线程：" + Thread.currentThread().getName()));
```
主要涉及以下几个关键概念：

1. `ExecutorService`：
    - 线程池的顶层接口
    - 提供了管理线程池的方法，如提交任务、关闭线程池等
2. `Executors`：
    - 线程池的工厂类
    - 提供了多种创建线程池的静态方法：
        - `newFixedThreadPool(n)`: 固定大小的线程池
        - `newCachedThreadPool()`: 可缓存的线程池
        - `newSingleThreadExecutor()`: 单线程的线程池
        - `newScheduledThreadPool(n)`: 支持定时任务的线程池
3. 提交任务的方法：
    - `execute(Runnable)`: 提交一个 Runnable 任务
    - `submit(Runnable)`: 提交任务并返回 `Future<?>`
    - `submit(Callable)`: 提交任务并返回 Future

建议的完整使用方式应该是：

```java
ExecutorService executorService = Executors.newFixedThreadPool(2);

try {

    // 提交多个任务

    executorService.execute(() ->

        System.out.println("6. 线程池创建线程：" + Thread.currentThread().getName())

    );

    // 可以提交返回结果的任务

    Future<String> future = executorService.submit(() -> "任务执行结果");

} finally {

    // 关闭线程池

    executorService.shutdown();

    // 等待所有任务完成，最多等待5秒

    executorService.awaitTermination(5, TimeUnit.SECONDS);

}
```
使用线程池的优势：

1. 重用线程，避免频繁创建和销毁线程的开销
2. 控制并发线程数量，避免资源耗尽
3. 提供任务队列，管理任务的执行
4. 提供了完整的生命周期管理
