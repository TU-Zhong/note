

以下是关于Java多线程核心知识体系的专业化技术解析：

---

### **一、线程基础机制**
1. **线程创建方式**  
[[创建线程]]
[[线程池]]
   - 继承`Thread`类（单继承限制）  
     ```java 
     class MyThread extends Thread { public void run() {...} }
     ```  
   - 实现`Runnable`接口（推荐解耦方案）  
     ```java
     new Thread(() -> {...}).start();
     ```  
   - 实现`Callable`+`FutureTask`（支持返回值与异常）  
     ```java
     FutureTask<Integer> task = new FutureTask<>(() -> {return 42;});
     ```

2. **线程生命周期**  
   - `NEW`：创建未启动状态  
   - `RUNNABLE`：可运行状态（包含操作系统层面的Ready/Running）  
   - `BLOCKED`：等待监视器锁的阻塞状态  
   - `WAITING`：无限期等待（`Object.wait()`/`LockSupport.park()`）  
   - `TIMED_WAITING`：带超时的等待（`Thread.sleep()`/`LockSupport.parkNanos()`）  
   - `TERMINATED`：线程终止

---

### **二、线程同步与锁机制**
1. **synchronized关键字**  
   - **锁升级机制**：无锁 → 偏向锁 → 轻量级锁（自旋） → 重量级锁  
   - 使用形式：  
     ```java
     // 实例锁
     synchronized(this) {...}  
     // 类锁
     synchronized(MyClass.class) {...}  
     ```

2. **Lock接口体系**  
   - `ReentrantLock`：  
     - 可中断锁（`lockInterruptibly()`）  
     - 公平锁（构造参数`fair=true`）  
     - 条件变量（`Condition`实现精准通知）  
   - `ReentrantReadWriteLock`：  
     - 读写分离（读锁共享，写锁互斥）  
     - 锁降级特性

3. **内存可见性保证**  
   - `volatile`关键字：  
     - 禁止指令重排序（内存屏障）  
     - 强制线程工作内存与主内存同步  
   - `happens-before`原则：  
     - 程序顺序规则  
     - 锁规则（解锁先于加锁）  
     - volatile变量规则

---

### **三、并发问题与解决方案**
1. **原子性问题**  
   - `i++`的非原子操作（读取→修改→写入）  
   - 解决方案：  
     - `AtomicInteger`（CAS实现）  
     ```java
     AtomicInteger ai = new AtomicInteger();  
     ai.getAndIncrement();
     ```  
     - `LongAdder`（分段CAS优化高并发）

2. **死锁条件与检测**  
   - 必要条件：  
     1. 互斥访问  
     2. 持有且等待  
     3. 不可剥夺  
     4. 循环等待  
   - 检测工具：  
     - `jstack`生成线程转储  
     - `VisualVM`死锁检测

3. **线程安全容器**  
   - `ConcurrentHashMap`（分段锁/CAS+红黑树）  
   - `CopyOnWriteArrayList`（写时复制）  
   - `BlockingQueue`实现（Array/ LinkedBlockingQueue）

---

### **四、并发工具与线程池**
1. **JUC工具类**  
   - `CountDownLatch`：一次性栅栏（计数器归零触发）  
   - `CyclicBarrier`：可重复使用的屏障  
   - `Semaphore`：信号量控制资源访问  
   - `Exchanger`：线程间数据交换

2. **线程池体系**  
   - 核心参数：  
     ```java
     new ThreadPoolExecutor(  
         corePoolSize,  
         maximumPoolSize,  
         keepAliveTime,  
         unit,  
         workQueue,  
         threadFactory,  
         handler);
     ```  
   - 任务队列类型：  
     - `SynchronousQueue`（直接提交策略）  
     - `LinkedBlockingQueue`（无界队列）  
     - `ArrayBlockingQueue`（有界队列）  
   - 拒绝策略：  
     - `AbortPolicy`（默认抛出RejectedExecutionException）  
     - `CallerRunsPolicy`（调用方线程执行）

3. **Fork/Join框架**  
   - 工作窃取算法（Work-Stealing）  
   - 分治任务实现：  
     ```java
     class MyTask extends RecursiveTask<Integer> {  
         protected Integer compute() {...}  
     }
     ```

---

### **五、高阶并发模式**
1. **ThreadLocal原理**  
   - 每个线程维护`ThreadLocalMap`  
   - Key为弱引用防止内存泄漏  
   - 典型应用场景：  
     - Spring事务管理  
     - 日期格式化器

2. **CAS底层实现**  
   - `sun.misc.Unsafe`类提供硬件级别原子操作  
   - CPU原语：`LOCK CMPXCHG`指令  
   - ABA问题解决方案：`AtomicStampedReference`

3. **无锁并发框架**  
   - Disruptor环形缓冲区  
   - StampedLock乐观读锁

---

### **六、最佳实践与陷阱规避**
1. **锁优化策略**  
   - 缩小同步代码块范围  
   - 避免嵌套锁（降低死锁概率）  
   - 使用`tryLock()`设置超时

2. **线程池使用规范**  
   - 禁止使用`Executors`创建（易导致OOM）  
   - 合理设置队列容量和拒绝策略  
   - 监控线程池状态（通过`ThreadPoolExecutor`扩展）

3. **资源泄漏防范**  
   - 确保`Lock`在`finally`块中释放  
   - `ThreadLocal`使用后必须调用`remove()`  
   - 正确关闭线程池（`shutdown()`/`shutdownNow()`）
