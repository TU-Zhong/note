```java
public void copyUsingStream() throws IOException{  
    try(FileInputStream in=new FileInputStream("src/resource/a.txt")){  
        FileOutputStream out=new FileOutputStream("src/resource/b.txt");  
        byte[] buffer=new byte[1024];  
        int bytesRead;  
        while((bytesRead=in.read(buffer))!=-1){  
            out.write(buffer,0,bytesRead);  
        }  
        }  
}
```

[[代码解释2]]
### **代码流程比喻**
想象你有一个煎饼进货车（`a.txt`）和出货车（`b.txt`），你要把煎饼一筐筐搬运过去：

1. **准备工具**：
   - 🧺 进货筐 `buffer`（容量1024个煎饼）
   - 🚚 进货员 `in`（从a.txt拿煎饼）
   - 🚛 出货员 `out`（往b.txt放煎饼）

2. **搬运过程**：
```java
while((bytesRead = in.read(buffer)) != -1) {  // 进货员每次最多装1024个煎饼
    out.write(buffer, 0, bytesRead);          // 出货员只拿实际装到的数量
}
```

---

### **逐句解读**
#### **`bytesRead = in.read(buffer)`**
- 🤌 **动作**：进货员伸手从a.txt抓一把煎饼
- ⚖️ **规则**：
  - 最多抓1024个（因为筐子buffer大小是1024）
  - 如果只剩50个煎饼了，就只抓50个
  - 如果抓空了（文件末尾），返回-1

#### **`!= -1`**
- 🚦 **判断**：
  - 抓到煎饼 → 继续干活（返回正数）
  - 抓空了 → 下班收工（返回-1时停止循环）

#### **`out.write(buffer, 0, bytesRead)`**
- 👐 **精细操作**：
  - 把筐里的煎饼**按实际抓取数量**放到b.txt
  - 比如筐里有50个煎饼，就只放50个（避免把上次残留的1024个都放进去）

---

### **关键点提醒**
1. **为什么要用bytesRead**？（⭐核心）
   - 最后一次抓取时，筐子可能没装满
   - 示例：文件共1500字节 → 第1次读1024，第2次读476

2. **错误写法示范**：
```java
out.write(buffer); // ❌ 危险！会把整个1024字节都写入，即使实际只读取了部分
```

3. **资源泄漏隐患**：
```java
// 当前写法：out没有被自动关闭（像出货员下班没锁仓库）
// 正确改进：把out也放进try-with-resources
try (FileInputStream in = ...;
     FileOutputStream out = ...) { // ✅ 安全写法
}
```
