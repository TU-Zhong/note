```java
public class ZeroTest {
    public static void main(String[] args) {
     try{
       int i = 100 / 0;
       System.out.print(i);
      }catch(Exception e){
       System.out.print(1);
       throw new RuntimeException();
      }finally{
       System.out.print(2);
      }
      System.out.print(3);
     }
 }
```
官方解析：这道题目考察了Java异常处理机制中的执行流程和finally块的特性。  
  
代码执行过程分析：  
1. 首先执行try块中的代码 100/0,这会抛出算术异常(ArithmeticException)  
2. 进入catch块,输出1  
3. catch块中抛出新的RuntimeException异常  
4. 无论是否发生异常,finally块都会执行,输出2  
5. 由于catch块中抛出了未被捕获的RuntimeException,程序在执行完finally块后会终止  
6. System.out.print(3)这行代码不会被执行到


需要特别注意：  
1. <mark style="background: #FFF3A3A6;">finally块中的代码总是会执行,即使try或catch块中抛出了异常  </mark>
2. <mark style="background: #FFF3A3A6;"> 如果catch块中抛出新异常且未被捕获,则在finally块执行完后程序会终止  </mark>
3. <mark style="background: #FFF3A3A6;">finally块后的代码只有在整个try-catch-finally结构正常执行完毕后才会执行</mark>
