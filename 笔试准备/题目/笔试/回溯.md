这是一道非常经典且高频的**回溯算法（Backtracking）**题目。在实际笔试或面试中（比如剑指 Offer 第 38 题），这道题的隐藏难点在于**如何处理字符串中可能包含重复字符的情况**，以确保输出的排列中没有重复项。

解决这个问题的最优标准做法是：**排序 + 回溯 + 剪枝**。

### 核心解题思路

1. **排序：** 将字符串转换为字符数组并按字典序排序。把相同的字符排在一起，是后续进行“剪枝去重”的基础。
    
2. **回溯（DFS）：** 逐个将未被使用的字符加入到当前正在拼接的字符串（`path`）中。当 `path` 的长度等于原字符串的长度时，说明找到了一个完整的排列。
    
3. **剪枝（去重）：** 在遍历字符时，如果发现当前字符与前一个字符相同，并且前一个字符在当前这一轮选择中**尚未被使用**（即 `!visited[i-1]`），说明继续往下走一定会产生和之前完全相同的分支，此时直接 `continue` 跳过。
    

---

### 标准 Java 实现代码

以下是结构清晰、可以直接通过大部分在线评测系统（OJ）的标准模板代码：

Java

```
import java.util.ArrayList;
import java.util.Arrays;

public class Solution {
    public ArrayList<String> Permutation(String str) {
        ArrayList<String> res = new ArrayList<>();
        // 边界条件处理
        if (str == null || str.length() == 0) {
            return res;
        }
        
        // 1. 将字符串转为字符数组并排序，这是去重剪枝的关键前提
        char[] chars = str.toCharArray();
        Arrays.sort(chars);
        
        // 记录哪些字符在当前的排列路径中已经被使用了
        boolean[] visited = new boolean[chars.length];
        // 用于动态拼接当前的排列
        StringBuilder path = new StringBuilder();
        
        // 开启回溯
        backtrack(chars, visited, path, res);
        
        return res;
    }
    
    private void backtrack(char[] chars, boolean[] visited, StringBuilder path, ArrayList<String> res) {
        // 终止条件：当收集到的字符数量等于原字符串长度时，说明一个完整的排列生成完毕
        if (path.length() == chars.length) {
            res.add(path.toString());
            return;
        }
        
        for (int i = 0; i < chars.length; i++) {
            // 如果当前字符已经被用过，跳过
            if (visited[i]) {
                continue;
            }
            
            // 【核心去重逻辑】：
            // 如果当前字符和前一个字符相同，且前一个字符在这个位置没有被使用过
            // 说明这个字符作为当前位置的情况已经被穷举过了，直接剪枝跳过
            if (i > 0 && chars[i] == chars[i - 1] && !visited[i - 1]) {
                continue;
            }
            
            // 做选择：标记为已使用，并加入路径
            visited[i] = true;
            path.append(chars[i]);
            
            // 递归进入下一层，继续选择剩下的字符
            backtrack(chars, visited, path, res);
            
            // 撤销选择（回溯）：状态重置，以便尝试其他分支
            path.deleteCharAt(path.length() - 1);
            visited[i] = false;
        }
    }
}
```

### 复杂度分析

- **时间复杂度：** $O(n \times n!)$。长度为 $n$ 的字符串最多有 $n!$ 种排列，每次拼接字符串并加入结果集需要 $O(n)$ 的时间。
    
- **空间复杂度：** $O(n)$。递归调用栈的最大深度为 $n$，同时使用了额外的 `visited` 数组和 `StringBuilder`，它们占用的空间也均为 $O(n)$。
    

这套模板非常通用，只要掌握了里面 `!visited[i - 1]` 的剪枝原理，遇到任何需要去重的组合、排列问题都能轻松套用。
