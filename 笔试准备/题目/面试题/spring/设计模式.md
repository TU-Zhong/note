这个问题在国企和银行的面试中简直是**“送分又送大招”**的绝佳机会！

面试官问这个问题，其实不是想听你像报菜名一样背出 23 种设计模式，而是想考察你**“是否真的看过源码”**，以及**“是否具备优秀的架构设计审美”**。

在回答时，一定要采用**“模式名称 + 在 Spring 中的具体类 + 解决什么痛点”**的结构。你只需要挑出以下 5 个最核心、最常打交道的模式，就能给面试官留下非常专业的印象：

### 1. 工厂模式 (Factory Pattern) —— Spring 的绝对基石

- **在哪里用到了：** `BeanFactory` 和 `ApplicationContext`。
    
- **解决什么痛点：** 这是 Spring IoC（控制反转）的核心。在没有 Spring 之前，我们总是要在代码里手动 `new` 对象，导致代码高度耦合。Spring 把创建对象的权力收归到一个“大工厂”里，业务代码只需要找工厂要对象就行了，实现了极致的解耦。
    

### 2. 单例模式 (Singleton Pattern) —— 性能与资源的守护者

- **在哪里用到了：** Spring 默认的 Bean 作用域（Scope）就是单例。
    
- **底层实现小亮点（面试加分）：** Spring 并不是靠简单的私有构造方法来实现单例的，而是内部维护了一个 `ConcurrentHashMap`（也就是咱们常说的“单例池”或者叫“一级缓存”），以此来保证全局唯一，并兼顾了多线程环境下的并发安全。
    

### 3. 代理模式 (Proxy Pattern) —— AOP 的灵魂

- **在哪里用到了：** Spring AOP 的底层实现。
    
- **解决什么痛点：** 当我们需要给现有的系统统一加上日志记录、或者是**数据库事务控制（银行极其看重这个）** 时，如果去改原来的代码就太蠢了。Spring 会在运行时帮你动态生成一个“代理对象”，把原来的对象包装起来，在执行核心逻辑前后偷偷塞进事务控制的代码。
    
- **关键术语必提：** 底层用到了 **JDK 动态代理**（针对实现了接口的类）和 **CGLIB 动态代理**（针对没有实现接口的普通类）。
    

### 4. 模板方法模式 (Template Method Pattern) —— 优雅地消除重复代码

- **在哪里用到了：** 以 `Template` 结尾的类，最经典的就是 `JdbcTemplate`，或者是 `RedisTemplate`。
    
- **解决什么痛点：** 回想一下原生的 JDBC 编程，每次查数据库都要经历“获取连接 -> 创建 Statement -> 执行 SQL -> 处理结果集 -> `finally` 块里挨个关闭连接”的痛苦过程。
    
- **自然结合实战：** 模板方法模式就是把这些“永远不变的死板步骤”封装在父类里定好骨架，而把“具体的 SQL 怎么写、结果怎么映射”这种灵活的逻辑留给子类或回调函数去实现。这在你做类似数据库管理平台或者复杂的外卖报表查询时，是大幅提升代码复用率的神器。
    

### 5. 观察者模式 (Observer Pattern) —— 模块间的解耦利器

- **在哪里用到了：** Spring 的事件驱动模型（Event & Listener），核心类如 `ApplicationEvent` 和 `ApplicationListener`。
    
- **解决什么痛点：** 比如用户下单成功后，你需要发短信、扣积分、发优惠券。如果把这些逻辑全写在订单方法里，代码就臃肿不堪了。使用观察者模式，订单系统只需要大喊一声“订单建好了（发布事件）”，其他短信、积分系统作为“观察者”监听到事件后自动执行，互不干扰。

### 💡 面试实战战术

当面试官问出：“Spring 源码里用到了哪些设计模式？”

**你可以这样顺滑地回答：**

> “Spring 源码中大量运用了设计模式来保证框架的高扩展性和优雅性。我印象最深的主要有四个：
> 
> 第一是**工厂模式**，通过 `BeanFactory` 实现了对象创建和业务逻辑的解耦；
> 
> 第二是**单例模式**，Spring 默认的 Bean 都是单例的，底层通过 `ConcurrentHashMap` 构成的单例注册表来管理；
> 
> 第三是**代理模式**，这是 AOP 和事务管理 `@Transactional` 的底层灵魂，依靠 JDK 动态代理和 CGLIB 实现；
> 
> 第四是**模板方法模式**，比如我平时开发中经常接触底层数据交互，用到的 `JdbcTemplate` 就完美地把获取连接、关闭连接等样板代码封装了起来，只让我们关注核心的 SQL 执行。
> 
> 这几种模式不仅在看源码时给了我很多启发，我在自己写业务代码时也有意识地在模仿它们。”

你看，这样一套组合拳打下来，既展示了广度，又突出了深度，最后还落脚到了你的实际开发体验上。

这 5 个模式中，**代理模式（动态代理机制）** 基本上是面试官听完后一定会紧接着往下挖的考点。需要我帮你把 JDK 动态代理和 CGLIB 的核心区别，用最直白的人话梳理一下吗？
