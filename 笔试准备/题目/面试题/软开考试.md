银行软件开发中心（简称“银行软开”）的笔试通常由各个银行自行组织或委托第三方机构（如北森）进行。虽然不同银行（如六大国有行与各类股份制银行）的侧重点略有不同，但整体考试题目的分布和考察方向存在高度的共性。

对于后端开发岗位的笔试，题目分布通常分为以下几个主要模块：

### 一、 计算机专业知识（约占 40% - 60%）

这是软开笔试的核心部分，主要考察计算机基础底蕴和后端开发技术栈。题型多为单选题和多选题。

- **编程语言：** **Java** 是银行后端系统的绝对主力，因此 Java 基础（集合、并发、JVM、面向对象特性）是重中之重。偶尔会涉及部分 C++ 或 Python 基础。
    
- **数据库：** 重点考察关系型数据库。包括 SQL 语法、事务隔离级别、索引原理、锁机制以及数据库调优。
    
- **计算机基础：**
    
    - **操作系统：** 进程与线程、内存管理，以及 **Linux 系统**的常用命令和基本操作。
        
    - **计算机网络：** TCP/IP 模型、HTTP/HTTPS 协议、三次握手/四次挥手等网络通信基础。
        
    - **数据结构与算法：** 数组、链表、树、图、排序算法、查找算法的时间/空间复杂度分析。
        
- **软件工程与前沿技术：** 软件生命周期、设计模式、分布式系统概念，以及偶尔出现的微服务（如 Spring Boot 相关概念）和前端基础（如 Vue/React 常识）。
    

### 二、 编程题 / 算法题（约占 10% - 20%）

一般包含 2 到 3 道手撕代码题，支持多种语言（强烈建议使用 Java 答题）。

- **难度：** 通常介于 LeetCode 的简单（Easy）到中等（Medium）难度之间。极少出现竞赛级别的难题。
    
- **高频考点：** 字符串处理、数组操作、动态规划基础、哈希表应用、树的遍历等。
    
- **SQL 编写：** 部分银行会单独设置 1 道手写 SQL 的题目，通常涉及多表连接（JOIN）、分组聚合（GROUP BY）以及窗口函数。
    

### 三、 行测 / 综合能力（EPI）（约占 20% - 30%）

这部分类似于公务员考试的行测，主要考察逻辑思维和抗压能力。题量大，时间紧。

- **言语理解：** 选词填空、段落主旨阅读。
    
- **逻辑推理：** 图形推理、数字规律、类比推理、演绎推理。
    
- **资料分析：** 给定图表或数据，进行快速计算和信息提取。
    
- **数量关系：** 行程问题、工程问题、概率问题等。
    

### 四、 银行综合知识与英语（约占 5% - 10%）

国有大行（如工、农、中、建）的笔试通常会保留这部分，而股份制银行可能会弱化或省略。

- **综合知识：** 宏观经济学基础、金融常识、银行企业文化、近期的时事政治。
    
- **英语：** 类似于四六级难度的单项选择（语法/词汇）和 1-2 篇阅读理解。重点通常偏向商务或计算机英语。
    

### 五、 性格测试（不计入总分，但具有一票否决权）

通常在笔试的最后进行，约 100 题左右。

- **测试目的：** 评估求职者的性格是否适合团队合作，是否有极端的心理倾向。
    
- **答题策略：** 保持前后一致，展现出情绪稳定、踏实肯干、有责任心和团队协作精神即可。
    

---

**备考建议：** 银行软开的笔试广度大于深度。你的重点应该放在巩固 Java 核心基础、熟悉 MySQL 数据库底层原理与 SQL 编写，以及保持每天刷 1-2 道算法题保持手感上。
