二、 基础项目：淘淘外卖

这个项目偏向传统的电商/外卖业务，面试官主要会通过它来考察你对主流中间件（Redis、MyBatis）和高并发场景的理解 。

**1. 关于 Redis 与高并发**

- **基础追问：** 你引入 Redis 缓存了店铺状态和套餐等热点数据 。你是如何保证数据库（MySQL）与 Redis 缓存之间的数据一致性的？（经典题：旁路缓存模式、延迟双删、Canal 监听 Binlog 等）
    
- **深度拓展：** 你提到用 Redis 高效实现了购物车功能 。Redis 中你具体使用了哪种数据结构（Hash、List 还是 String）？为什么这么选？如果 Redis 宕机了，用户的购物车数据会丢失吗？你是怎么处理持久化的？
    
- **场景题：** 外卖平台在中午高峰期会有极高的并发读写。你的系统目前能抗住多大的 QPS？如果让你对系统进行压测并进一步优化，你会从哪些方面入手？
    

**2. 关于框架与设计模式**

- **基础追问：** 你利用 AOP 实现了公共字段自动填充 。请讲讲 AOP 的底层原理（JDK 动态代理与 CGLIB 的区别）。在什么情况下，Spring 的 AOP 事务或者切面会失效？
    
- **深度拓展：** 你的 C/B 两端登录认证都使用了 JWT 技术结合 Interceptor 。JWT 相较于传统的 Session 有什么优缺点？如果用户的权限在中途被管理员修改了，或者你需要强制某个用户下线，使用无状态的 JWT 该怎么实现？
