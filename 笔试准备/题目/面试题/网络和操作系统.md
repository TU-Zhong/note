网络和操作系统是**后端开发**真正的地基。在国企和银行的面试中，这两个板块的权重甚至不亚于 Java 语言本身。因为无论上层的框架怎么变，底层的服务端通信逻辑、高并发下的资源调度机制是永远不会变的。特别是涉及数据库底层原理、服务器性能调优时，这两块知识是必不可少的。

为你整理了这两个板块的“必考题单”，按照重要程度和面试频率进行了排序：

### 一、 计算机网络 (重点考察：稳定传输与通信原理)

银行业务对数据的可靠传输有着极其严苛的要求，所以 **TCP 协议** 是绝对的霸主级考点。

**1. 传输层双雄（绝对核心）**

- **TCP 的三次握手和四次挥手过程是怎样的？**（必考！不仅要知道过程，还要知道为什么是三次而不是两次？为什么挥手要四次？`TIME_WAIT` 状态有什么作用？）
    
- **TCP 和 UDP 的区别是什么？**（分别适用于什么场景？如何基于 UDP 实现可靠传输？）
    
- **TCP 是如何保证可靠性的？**（滑动窗口、流量控制、拥塞控制机制，一定要能用白话把“慢启动、拥塞避免”的逻辑说清楚。）
    

**2. 应用层协议（日常开发最常用）**

- **HTTP 和 HTTPS 的区别？**（HTTPS 的加密过程是怎么样的？对称加密和非对称加密是如何结合使用的？这个在金融级安全通信中必问。）
    
- **常见的 HTTP 状态码有哪些？**（比如 301 和 302 的区别，401 和 403 的区别，500 和 502 的区别。）
    
- **GET 和 POST 的区别？**（除了表面上的参数位置，最好能答出幂等性和底层 TCP 数据包发送次数的差异。）
    
- **在浏览器输入 URL 到页面显示，中间发生了什么？**（经典串联题，考验对 DNS 解析、TCP 连接、HTTP 请求、服务器处理以及页面渲染的全局掌握程度。）
    

---

### 二、 操作系统 (重点考察：并发控制与资源调度)

当你开发的后台系统需要处理海量并发请求，或者涉及复杂的数据库读写时，操作系统的底层知识就会直接决定你代码的性能上限。

**1. 进程与线程（并发编程的灵魂）**

- **进程和线程的区别是什么？**（资源分配的基本单位 vs CPU 调度的基本单位，内存共享与隔离的差异。）
    
- **进程间通信（IPC）的方式有哪些？**（管道、消息队列、共享内存、信号量等，知道哪种最快即可。）
    
- **什么是死锁？**（结合之前做过的银行家算法，说出死锁的四个必要条件，以及如何预防和避免。）
    

**2. 内存管理（缓存与性能的底层）**

- **什么是虚拟内存？**（为什么需要它？解决了什么问题？）
    
- **页面置换算法有哪些？**（**LRU** 是重中之重！不仅要知道原理，面试官甚至可能让你手写一个 LRU 缓存，因为它在 Redis 和数据库缓冲池中应用太广泛了。）
    

**3. I/O 模型（高阶加分项）**

- **说说你对阻塞/非阻塞、同步/异步的理解？**
    
- **什么是 I/O 多路复用？**（能说清楚 `select`、`poll` 和 `epoll` 的区别，基本就能拿下极高的分数了，这是 Nginx、Redis 等高性能服务器的底层基石。）

### 💡 备考小建议

对于这些偏理论的内容，千万不要死记硬背枯燥的概念，**一定要结合后端开发的实际场景来记**。比如，背 `epoll` 的时候，就想想它是怎么让服务器同时处理成千上万个客户端连接的；背 LRU 算法的时候，就想想如果你在设计一个外卖系统的菜单缓存，满了该怎么淘汰数据。

这其中，**“TCP 的三次握手”** 和 **“手写 LRU 缓存算法”** 是两个极易被深挖的硬骨头。你想先挑战哪一个？我可以帮你把它的核心逻辑彻底拆解明白。
