这不仅是计算机网络的基础，更是春招季后端笔试和面试中最核心的必考考点。理解透彻这些底层网络协议，对于拿下心仪的后端研发岗位非常有帮助。

TCP 是一种面向连接的、可靠的传输层协议。简单来说，“三次握手”是为了**安全地建立连接**，而“四次挥手”是为了**优雅地断开连接**。

### 一、 三次握手 (Three-way Handshake) —— 建立连接

建立连接的过程就像是两个人要在嘈杂的环境中确认彼此都能听见对方说话。

1. **第一次握手（客户端 -> 服务端）：** 客户端主动发起连接，向服务端发送一个带有 **SYN**（同步序列编号）标志的数据包，并随机生成一个初始序列号 `seq = x`。
    
    - _状态变化：_ 客户端进入 `SYN_SENT`（同步已发送）状态。
        
    - _潜台词：_ “喂，老哥，能听到吗？我想跟你建立连接，我的初始暗号是 x。”
        
2. **第二次握手（服务端 -> 客户端）：**
    
    服务端收到 SYN 包后，必须确认客户端的请求。于是它发送一个带有 **SYN** 和 **ACK**（确认）标志的数据包。它会确认收到客户端的暗号（`ack = x + 1`），并给出自己的初始序列号 `seq = y`。
    
    - _状态变化：_ 服务端进入 `SYN_RCVD`（同步收到）状态。
        
    - _潜台词：_ “听到了！我也想跟你建立连接，你的暗号 x 我收到了，我的初始暗号是 y，你能听到我吗？”
        
3. **第三次握手（客户端 -> 服务端）：**
    
    客户端收到服务端的 SYN+ACK 包后，向服务端发送一个带有 **ACK** 标志的确认包（`ack = y + 1`）。
    
    - _状态变化：_ 客户端进入 `ESTABLISHED`（已建立连接）状态。服务端收到此包后，也进入 `ESTABLISHED` 状态。
        
    - _潜台词：_ “我也能听到你！你的暗号 y 我收到了，咱们开始通信吧！”
        

_(此时，双方的收发通道都已经过验证，开始正式传输数据。)_

### 二、 四次挥手 (Four-way Wave) —— 断开连接

断开连接之所以比建立连接多一次，是因为 TCP 是**全双工通信**（双方可以同时收发），所以必须**两边分别独立关闭**。这被称为“半关闭（Half-Close）”机制。

1. **第一次挥手（主动方 -> 被动方）：**
    
    通常是客户端数据发送完毕，主动向服务端发送一个 **FIN**（结束）标志的数据包，请求断开连接（`seq = u`）。
    
    - _状态变化：_ 客户端进入 `FIN_WAIT_1` 状态。停止发送应用数据，但仍可以接收数据。
        
    - _潜台词：_ “我的话都说完了，我准备挂电话了。”
        
2. **第二次挥手（被动方 -> 主动方）：**
    
    服务端收到 FIN 包后，必须立刻回复一个 **ACK** 确认包（`ack = u + 1`），告诉客户端收到了关闭请求。
    
    - _状态变化：_ 服务端进入 `CLOSE_WAIT` 状态。客户端收到后进入 `FIN_WAIT_2` 状态。
        
    - _潜台词：_ “好的，我知道你没话说了。但我这里可能还有点事情没交代完，你先等我会。”
        
        _(注意：此时客户端到服务端的连接已经关闭，但服务端到客户端的连接还在，服务端如果还有数据没发完，可以继续发。)_
        
3. **第三次挥手（被动方 -> 主动方）：**
    
    服务端把剩下的数据都发送完毕后，向客户端发送一个 **FIN** 标志的数据包（`seq = w`）。
    
    - _状态变化：_ 服务端进入 `LAST_ACK`（最后确认）状态。
        
    - _潜台词：_ “我这边的数据也都交代完了，我也准备挂了。”
        
4. **第四次挥手（主动方 -> 被动方）：**
    
    客户端收到服务端的 FIN 后，必须最后回复一个 **ACK** 确认包（`ack = w + 1`）。
    
    - _状态变化：_ 客户端进入 **`TIME_WAIT`** 状态。服务端收到这个 ACK 后，直接进入 `CLOSED` 状态，彻底关闭连接。
        
    - _潜台词：_ “好的，那咱们就彻底拜拜吧！”

### 面试高频追问提示

在经历完四次挥手后，客户端并没有立刻进入 `CLOSED` 状态，而是必须在 **`TIME_WAIT`** 状态下强制等待 **2MSL**（最大报文段生存时间，通常是 1-4 分钟）。
