国企和银行的后端开发面试，风格与互联网大厂略有不同。他们通常不追求最前沿、花哨的技术，而是极度看重**基础的扎实程度、系统的稳定性以及对核心机制的底层理解**。

对于 Spring 框架，**事务管理 (Transaction)** 和 **AOP** 是绝对的重中之重，因为这直接关系到金融级数据的安全与一致性。

为你整理了一份针对国企/银行高频且精准的 Spring 核心面试题单，我将其分为了四个核心考区：

### 一、 核心容器与运行机制 (IoC & Bean)

这是考察基本功的重灾区，一定要能流畅地把流程说清楚。

- **谈谈你对 Spring IoC 和 DI 的理解？** （重点答出控制反转带来的“解耦”思想）
    
- **Spring Bean 的生命周期是怎样的？** （必考题，建议分为：实例化、属性赋值、初始化、销毁 四个大阶段来记，并提一下 `BeanPostProcessor` 的作用）
    [[bean生命周期]]
- **Spring Bean 默认是单例的吗？如果是，存在线程安全问题吗？如何解决？** （银行非常看重并发安全，重点回答无状态 Bean 以及通过 `ThreadLocal` 解决有状态 Bean 安全问题的方案）
    [[spring boot bean是单例的吗]]
- **Spring 是如何解决循环依赖的？** （经典区分度考题，说明白三级缓存机制，以及为什么需要第三级缓存来处理 AOP 代理对象）
    [[三级缓存处理循环依赖]]
- **Spring 源码中用到了哪些设计模式？** （单例、工厂、代理、模板方法、观察者等，最好能结合具体的类，比如 `JdbcTemplate`）
    [[设计模式]]

[[ThreadLocal]]
### 二、 面向切面编程 (AOP)

通常会结合实际项目经验来问。

- **Spring AOP 的底层实现原理是什么？** （必须熟练掌握 JDK 动态代理和 CGLIB 代理的区别，以及 Spring 在什么条件下会切换这两种代理）
    [[Aop实现原理]]
- **在实际的后端开发中，你通常用 AOP 来做什么？** （结合你的实际开发经验，可以说说在做复杂的后台管理系统或业务系统时，如何用 AOP 实现全局的操作日志记录、接口鉴权拦截或是性能监控）
    [[用AOP来干什么]]

### 三、 事务管理 (Transaction - 银行最爱考)

金融业务对数据一致性要求极高，这部分如果答不好，面试极大概率会挂。

- **Spring 事务的传播行为 (Propagation) 有哪些？** （重点区分默认的 `REQUIRED` 和 `REQUIRES_NEW`，最好能在脑海里准备一个“A 方法调用 B 方法，谁报错谁回滚”的具体例子来讲述）
    
- **`@Transactional` 注解在哪些情况下会失效？** （非常高频的实战避坑题，例如：修饰的方法不是 public、发生了同类中的方法自调用、异常被 try-catch 吞掉了、底层的数据库引擎本身不支持事务等）
    
- **Spring 事务的隔离级别有哪些？它跟底层关系型数据库的隔离级别是强绑定的吗？** （考察你对 Spring 事务抽象层和底层数据库交互的综合理解）
    

### 四、 Spring Boot 延伸

既然现在大家基本都在用 Spring Boot 构建项目，它的核心原理也必须掌握。

- **说说 Spring Boot 的自动装配 (Auto-Configuration) 原理？** （核心是答出 `@SpringBootApplication` 注解里的 `@EnableAutoConfiguration`，以及它如何读取 `spring.factories` 或 `AutoConfiguration.imports` 文件中的配置类）
    
- **Spring Boot 启动流程是怎样的？** （简述推断 Web 环境、加载配置、创建 ApplicationContext 等核心步骤）
    [[Spring boot启动流程]]

**💡 面试实战小建议**：

在回答这些问题时，尽量不要只干巴巴地背诵概念。比如被问到 AOP 时，你可以顺势带出你在处理类似 ERP 系统或多表关联的后台管理系统时，是如何统一设计数据拦截和日志收集的，这样能立刻展示出你的实战能力。
