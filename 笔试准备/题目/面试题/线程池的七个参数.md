`ThreadPoolExecutor` 是 Java 并发编程中实现线程池的核心类。要彻底理解线程池的工作原理，掌握其构造函数中的 7 个核心参数至关重要。

以下是这 7 个参数的详细解释：

### 1. `corePoolSize`（核心线程数）

- **含义**：线程池中长期维护的最小线程数量。
    
- **作用**：即使这些线程处于空闲状态，它们也不会被回收（除非显式调用了 `allowCoreThreadTimeOut(true)`）。当有新任务提交时，如果当前运行的线程数少于 `corePoolSize`，线程池会创建一个新的核心线程来执行任务，即使池中存在其他空闲线程。
    

### 2. `maximumPoolSize`（最大线程数）

- **含义**：线程池允许创建的最大总线程数量。
    
- **作用**：当核心线程都在忙碌，且**任务队列（workQueue）也已经装满**时，线程池会继续创建新的非核心线程来处理任务，直到池中的总线程数达到 `maximumPoolSize`。
    

### 3. `keepAliveTime`（空闲线程存活时间）

- **含义**：非核心线程空闲时的最大存活时间。
    
- **作用**：当线程池中的线程数量超过 `corePoolSize` 时，如果某些线程空闲了 `keepAliveTime` 这么久，它们就会被销毁，直到线程数量降回 `corePoolSize`。这有助于在系统负载减轻时释放系统资源。
    

### 4. `unit`（时间单位）

- **含义**：`keepAliveTime` 的时间单位。
    
- **作用**：通常使用 `TimeUnit` 枚举类，例如 `TimeUnit.SECONDS`（秒）、`TimeUnit.MILLISECONDS`（毫秒）等。
    

### 5. `workQueue`（任务阻塞队列）

- **含义**：用于存放等待执行任务的阻塞队列。
    
- **作用**：当核心线程都在忙碌时，新提交的任务会被放入这个队列中排队等待。常见的队列有：
    
    - `ArrayBlockingQueue`：基于数组的有界队列。
        
    - `LinkedBlockingQueue`：基于链表的队列（通常作为无界队列使用，默认大小为 `Integer.MAX_VALUE`）。
        
    - `SynchronousQueue`：不存储元素的阻塞队列，每个插入操作必须等待另一个线程的移除操作。
        

### 6. `threadFactory`（线程工厂）

- **含义**：用于创建新线程的工厂类。
    
- **作用**：可以通过自定义线程工厂来为线程池中创建的线程命名、设置守护线程状态、或者设置线程优先级等，这在排查线上问题时非常有帮助（例如通过特定的线程前缀快速定位日志）。
    

### 7. `handler`（拒绝策略）

- **含义**：当线程池和任务队列都达到饱和状态时，处理新提交任务的策略。
    
- **作用**：如果总线程数已达到 `maximumPoolSize`，且 `workQueue` 也满了，说明系统已无法承受更多的任务，此时会触发拒绝策略。Java 提供了 4 种内置策略：
    
    - `AbortPolicy`（默认）：直接抛出 `RejectedExecutionException` 异常。
        
    - `CallerRunsPolicy`：由调用 `execute()` 方法的线程（通常是主线程）自己来执行该任务。
        
    - `DiscardPolicy`：直接默默丢弃该任务，不抛出异常。
        
    - `DiscardOldestPolicy`：丢弃队列中最老（最早进入队列）的一个任务，然后尝试重新提交当前任务。
        

---

### 参数之间的协作流程（执行逻辑）

理解这些参数最好的方式是看它们如何协同处理一个新提交的任务：

1. **第一步**：当前工作线程数 < `corePoolSize`？如果是，**创建核心线程**执行任务。如果否，进入下一步。
    
2. **第二步**：尝试将任务加入 `workQueue`。如果队列未满，**任务入队**等待。如果队列已满，进入下一步。
    
3. **第三步**：当前工作线程数 < `maximumPoolSize`？如果是，**创建非核心线程**执行任务。如果否，进入下一步。
    
4. **第四步**：触发 **`handler` 拒绝策略**。
