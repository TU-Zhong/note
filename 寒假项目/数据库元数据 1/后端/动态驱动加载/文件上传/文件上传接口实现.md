```java
package com.example.dbdesign.controller; // 记得修改为你的实际包名

import org.springframework.web.bind.annotation.*;
import org.springframework.web.multipart.MultipartFile;
import java.io.File;
import java.io.IOException;

@RestController
@RequestMapping("/api/driver")
@CrossOrigin // 关键！允许前端(Vue 5173)跨域访问后端(8080)
public class DriverController {

    // 定义上传文件的存储目录：项目根目录下的 drivers 文件夹
    // System.getProperty("user.dir") 获取的是项目启动时的根路径
    private static final String UPLOAD_DIR = System.getProperty("user.dir") + File.separator + "drivers";

    /**
     * 上传数据库驱动 Jar 包
     * @param file 前端传递的文件对象
     * @return 上传成功后的绝对路径
     */
    @PostMapping("/upload")
    public String uploadDriver(@RequestParam("file") MultipartFile file) {
        if (file.isEmpty()) {
            return "上传失败，请选择文件";
        }

        try {
            // 1. 确保目录存在
            File directory = new File(UPLOAD_DIR);
            if (!directory.exists()) {
                boolean created = directory.mkdirs(); // 如果目录不存在，自动创建
                if (created) {
                    System.out.println("创建驱动目录: " + UPLOAD_DIR);
                }
            }

            // 2. 构建目标文件对象
            // 获取原始文件名 (例如 mssql-jdbc-12.2.0.jar)
            String originalFilename = file.getOriginalFilename();
            // 拼接完整路径
            File destFile = new File(UPLOAD_DIR + File.separator + originalFilename);

            // 3. 执行保存 (Spring 提供的便捷方法)
            file.transferTo(destFile);

            // 4. 返回绝对路径 (给下一步动态加载使用)
            System.out.println("驱动上传成功: " + destFile.getAbsolutePath());
            return destFile.getAbsolutePath();

        } catch (IOException e) {
            e.printStackTrace();
            return "上传出错: " + e.getMessage();
        }
    }
}
```

### 2. 代码核心点解析 (写论文/答辩可用)

1. **`@CrossOrigin`**:
    
    - **作用**：这是一个**极其重要**的注解。因为你的 Vue 运行在 `localhost:5173`，而后端运行在 `localhost:8080`，浏览器默认会拦截跨域请求。加上这个注解，前端才能成功上传文件。
        
2. **`System.getProperty("user.dir")`**:
    
    - **作用**：获取当前项目的根目录。
        
    - **优势**：这样写代码具有**可移植性**。无论你在 Windows 还是 Linux (Docker) 上运行，它都能找到正确的相对路径，而不是写死 `D:\\drivers` 这种换个电脑就报错的路径。
        
3. **`file.transferTo(destFile)`**:
    
    - **作用**：Spring MVC 提供的核心方法，直接把临时文件写入磁盘，性能高且代码简洁。
        

### 3. 如何测试它是否工作？

你不需要写前端页面就可以测试。

使用 Postman (或者 Apifox) 进行测试：

1. **请求方式**: `POST`
    
2. **URL**: `http://localhost:8080/api/driver/upload`
    
3. **Body**:
    
    - 选择 `form-data`。
        
    - Key 填写 `file` (必须和代码里的 `@RequestParam("file")` 一致)。
        
    - 类型从 Text 改为 **File**。
        
    - Value 选择你电脑上的任意一个 `.jar` 文件（或者随便一个 txt 文件也行）。
        
4. **点击 Send**。
    

**预期结果**：

1. Postman 返回一串长路径，例如：`D:\yingyong2\CodingLearning2\JavaProject\Study\DBDesign\drivers\mssql-jdbc.jar`。
    
2. 你去你的项目文件夹下看，会自动多出一个 `drivers` 文件夹，里面有你刚才传的文件。
