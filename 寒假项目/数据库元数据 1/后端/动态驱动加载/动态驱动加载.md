### 🎯 这一步要做到什么程度？（验收标准）

你不需要做复杂的数据库树形展示，**只要达到以下 3 点就算成功**：

1. **文件落地**：能把 `mssql-jdbc.jar` 传给后端，后端把它存到了硬盘上的某个文件夹里（例如 `D:\drivers\`）。
    
2. **驱动加载**：后端能通过代码读取这个 Jar 包，并且能在内存中加载出 `com.microsoft.sqlserver.jdbc.SQLServerDriver` 这个类 1111。
    
3. **握手成功**：后端能用加载出来的驱动，连接上你 Docker 里的 SQL Server，并执行一个简单的 `SELECT 1`，不报错即为通过 2222。

B. 动态加载服务 (Service) —— 核心中的核心

这是实现任务书里“运行时热插拔”的关键 3333。这里不能用平时常用的 DriverManager.getConnection()，因为传统的 JDBC 管理器无法识别外部加载的类。你必须直接操作 Driver 对象。

我为你准备了这一步的核心代码骨架，你可以直接参考：
[[动态加载驱动]]
Java

```
// 伪代码参考
public Connection getDynamicConnection(String jarPath, String url, String user, String password) throws Exception {
    // 1. 创建 URLClassLoader，指向 jar 包路径 [cite: 45]
    File jarFile = new File(jarPath);
    URL urlPath = jarFile.toURI().toURL();
    URLClassLoader ucl = new URLClassLoader(new URL[]{urlPath}, Thread.currentThread().getContextClassLoader());

    // 2. 加载驱动类 (SQL Server 的驱动类名是固定的)
    // 任务书要求：反射实例化 [cite: 46]
    Class<?> driverClass = ucl.loadClass("com.microsoft.sqlserver.jdbc.SQLServerDriver");
    Driver driver = (Driver) driverClass.newInstance();

    // 3. 建立连接 [cite: 49]
    // 注意：这里使用 driver.connect 而不是 DriverManager.getConnection
    Properties props = new Properties();
    props.setProperty("user", user);
    props.setProperty("password", password);
    
    Connection conn = driver.connect(url, props);
    return conn;
}
```

#### 2. 前端 (Vue): 10% 的工作量

这一步前端只需要做一个最简陋的界面，用来触发后端的逻辑。

- **界面**：放一个文件选择框（`<input type="file">` 或 Element Plus 的 `<el-upload>`）和一个“测试连接”按钮。
    
- **逻辑**：
    
    1. 点击上传，把 Jar 包传给后端，拿到后端返回的文件路径。
        
    2. 填入数据库账号密码，点击“测试”，把刚才的文件路径 + 账号密码发给后端。
        
    3. 如果后端返回 "Success"，你就成功了。

### ⚠️ 需要避坑的 2 个点

1. **JDK 版本问题**：你现在用的是 Docker 里的 `node`，但后端是本地跑的。确保你本地的 JDK 版本（建议 JDK 17 或 8）和下载的 `mssql-jdbc.jar` 版本匹配。通常下载 `jre8` 或 `jre11` 后缀的 jar 包最稳。
    
2. **Docker 网络**：后端（本机）连接 Docker（容器）里的数据库时，URL 要写 `jdbc:sqlserver://localhost:1433...`。确保你的 Docker 容器端口 `1433` 是对外开放的（我们在 Day 1 的命令里已经配置了 `-p 1433:1433`，所以应该是通的）。
    

**总结**：这一步是**后端重、前端轻**。只要后端能通过加载外部 Jar 包连上 Docker 里的数据库，你的毕设核心功能就完成了一半！
