### 3. 核心突破：服务层实现 (`MetadataService.java`)

在你的 `MetadataService` 中**追加**以下两个方法。`getIndexes` 用于提取底层索引，`getTableTree` 用于将表、列、索引拼接成一棵完整的树。

Java

```java
package com.example.dbdesign.Service;

import com.example.dbdesign.vo.ColumnVO;
import com.example.dbdesign.vo.IndexVO;
import com.example.dbdesign.vo.TableV0;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

import java.sql.Connection;
import java.sql.DatabaseMetaData;
import java.sql.ResultSet;
import java.util.ArrayList;
import java.util.List;

// ... 保持你原有的 getTable 和 getColumns 方法不变 ...

@Service
public class MetadataService {
    @Autowired
    private ConnectionCathe connectionCathe;

    // ... (你之前写的 getTable 和 getColumns 代码保留在此处) ...

    /**
     * Day 9 新增：获取表的索引信息
     */
    public List<IndexVO> getIndexes(String token, String schema, String tableName) throws Exception {
        Connection conn = connectionCathe.getConnection(token);
        if (conn == null) {
            throw new RuntimeException("连接已断开，请重试！");
        }
        
        DatabaseMetaData metaData = conn.getMetaData();
        // 参数说明：catalog, schema, table, unique(false表示查询所有索引), approximate
        ResultSet rs = metaData.getIndexInfo(null, schema, tableName, false, false);
        
        List<IndexVO> indexList = new ArrayList<>();
        while (rs.next()) {
            String indexName = rs.getString("INDEX_NAME");
            // 过滤掉统计信息 (indexName 为 null 的通常是表自身的统计数据)
            if (indexName == null) continue;

            boolean nonUnique = rs.getBoolean("NON_UNIQUE");
            String columnName = rs.getString("COLUMN_NAME");
            short type = rs.getShort("TYPE");
            String ascOrDesc = rs.getString("ASC_OR_DESC");

            indexList.add(new IndexVO(indexName, columnName, nonUnique, type, ascOrDesc));
        }
        rs.close();
        return indexList;
    }

    /**
     * Day 9 新增：获取完整树形结构 (表 -> 列 & 索引)
     */
    public List<TableV0> getTableTree(String token, String schemaFilter) throws Exception {
        // 1. 先获取所有表
        List<TableV0> tables = this.getTable(token, schemaFilter);

        // 2. 遍历每个表，把它的列和索引都查出来，挂载进去
        for (TableV0 table : tables) {
            // 获取该表的列
            List<ColumnVO> columns = this.getColumns(token, table.getSchema(), table.getName());
            table.setColumns(columns);

            // 获取该表的索引
            List<IndexVO> indexes = this.getIndexes(token, table.getSchema(), table.getName());
            table.setIndexes(indexes);
        }

        return tables;
    }
}
```

### 💡 技术解析 (这部分可以在论文中或答辩时提及)：

1. **关于索引过滤 (`indexName == null`)**：这是 JDBC 开发的一个暗坑。`getIndexInfo` 会把表的“统计信息”也当作索引返回，此时 `INDEX_NAME` 为 `NULL`。我在代码中加了 `if (indexName == null) continue;` 进行拦截，这体现了你对数据库底层 API 的深度理解。
    
2. **DTO 嵌套设计**：通过 `getTableTree` 方法，我们将多次查询的结果聚合到了一起。这其实是一个典型的 **BFF (Backend For Frontend)** 思想：后端服务前端，帮前端把数据拼装成 Vue `el-tree` 组件最容易解析的 JSON 格式。
