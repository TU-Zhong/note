### 第一步：后端接口“单兵测试” (Backend Sanity Check)

在让前端去请求之前，必须先确保后端 API 是活着的，而且能吐出正确的数据。

1. **启动后端**：确保你的 Spring Boot 项目已经 Run 起来了，控制台没有报错。
    
2. **浏览器直连**：
    
    打开浏览器（或者 Postman），直接访问你的后端接口。
    
    - 假设你的端口是 `8080`，获取表数据的接口是 `/api/db/table/data`（具体看你 Controller 怎么写的）。
        
    - 输入类似：`http://localhost:8080/api/db/table/data?tableName=t_student`
        
3. **预期结果**：
    
    你应该看到类似这样的 JSON 返回：
    
    JSON
    
    ```
    {
      "code": 200,
      "data": [
        {"id": 1, "name": "张三", "age": 20},
        {"id": 2, "name": "李四", "age": 22}
      ],
      "msg": "success"
    }
    ```
    
    _❌ 如果浏览器转圈或报错 404/500，先别动前端，去修后端 Controller！_

### 第二步：给表格组件装上“真引擎”

你之前的 `TableDataViewer.vue` 里用的是 `mockData`（假数据）。现在我们要把它换成真的 API 请求。

请修改 `src/components/data/TableDataViewer.vue`：

Code snippet

```vue
<template>
  <div class="data-viewer" v-loading="loading">
    <div class="toolbar">
      <el-button type="primary" link @click="fetchData">
        <el-icon><Refresh /></el-icon> 刷新
      </el-button>
      <span class="info">正在查看表: {{ params.tableName }}</span>
    </div>

    <el-table :data="tableData" style="width: 100%" height="100%" border stripe>
      <el-table-column
        v-for="key in tableKeys"
        :key="key"
        :prop="key"
        :label="key"
        min-width="120"
      />
    </el-table>
  </div>
</template>

<script setup>
import { ref, onMounted, computed } from 'vue'
import axios from 'axios' // 确保安装了 axios: npm install axios
import { ElMessage } from 'element-plus'

// 1. 接收参数
const props = defineProps(['tabId', 'params'])

// 2. 定义状态
const tableData = ref([])
const loading = ref(false)

// 3. 计算表头 (简单粗暴版：取第一行数据的 key)
const tableKeys = computed(() => {
  if (tableData.value.length > 0) {
    return Object.keys(tableData.value[0])
  }
  return []
})

// 4. 获取数据的函数 (真请求)
const fetchData = async () => {
  if (!props.params.tableName) return
  
  loading.value = true
  try {
    // ⚠️ 注意：这里的 URL 必须替换成你真实的后端接口地址
    // 假设后端接口是 POST /api/query/table
    const res = await axios.post('http://localhost:8080/api/query/table', {
      tableName: props.params.tableName
    })
    
    // 根据你后端的 Result 结构取值，通常是 res.data.data
    if (res.data.code === 200) {
      tableData.value = res.data.data
    } else {
      ElMessage.error(res.data.msg || '数据加载失败')
    }
  } catch (error) {
    console.error(error)
    ElMessage.error('网络请求异常')
  } finally {
    loading.value = false
  }
}

// 5. 组件挂载时自动调用
onMounted(() => {
  fetchData()
})
</script>

<style scoped>
.data-viewer {
  height: 100%;
  display: flex;
  flex-direction: column;
  padding: 10px;
}
.toolbar {
  margin-bottom: 10px;
  display: flex;
  align-items: center;
  gap: 10px;
}
.info {
  font-size: 12px;
  color: #909399;
}
</style>
```

现在，激动人心的时刻到了。请打开你的前端页面 (`npm run dev`) 和 Chrome 浏览器的 **F12 开发者工具**。

#### 3. 测试动作 B：双击打开 Tab

### 🚑 常见报错急救包

如果在测试中遇到问题，大概率是下面三个原因之一：

1. **跨域问题 (CORS Error)**
    
    - **现象**：Network 里报红，Console 里显示 `Access to XMLHttpRequest... has been blocked by CORS policy`。
        
    - **原因**：前端 5173 访问后端 8080，浏览器处于安全考虑拦截了。
        
    - **解决**：在 Spring Boot 后端加一个配置类：
        
        Java
        
        ```
        @Configuration
        public class CorsConfig implements WebMvcConfigurer {
            @Override
            public void addCorsMappings(CorsRegistry registry) {
                registry.addMapping("/**")
                        .allowedOriginPatterns("*")
                        .allowedMethods("GET", "POST", "PUT", "DELETE")
                        .allowCredentials(true);
            }
        }
        ```
        
2. **字段名对不上**
    
    - **现象**：数据回来了，Network 里有数据，但表格是空的。
        
    - **原因**：后端返回的 JSON 结构可能是 `{ code: 200, result: [...] }`，但前端代码写的是 `res.data.data`。
        
    - **解决**：看 `console.log(res.data)` 打印出来的结构，调整前端取值路径。
        
3. **连接被拒绝 (Connection Refused)**
    
    - **现象**：`ERR_CONNECTION_REFUSED`。
        
    - **原因**：后端根本没启动，或者端口写错了（比如后端是 8081，前端写了 8080）。
