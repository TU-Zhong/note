这三个概念是 Vue 组件通信的**铁三角**。如果你把一个 Vue 应用比作一家公司，那么 `Props`、`Emits` 和 `Watch` 分别代表了**下达指令**、**汇报工作**和**盯着进度**。

---

### 1. Props（属性）：下达指令 (Parent -> Child)

**Props** 是父组件向子组件传递数据的方式。它是**单向**的，子组件只能读取，不能直接修改。

- **形象比喻**：父亲给孩子一张**只能看不能改**的说明书。
    
- **代码写法**：
    
    JavaScript
    
    ```
    // 子组件：接收指令
    const props = defineProps(['title', 'count'])
    ```
    
- **关键点**：
    
    - **只读性**：子组件如果试图修改 `props.title`，Vue 会在控制台给你报警告。
        
    - **动态性**：父组件的数据变了，子组件收到的 `props` 也会跟着变。
        

---

### 2. Emits（发射）：汇报工作 (Child -> Parent)

由于子组件不能改 `props`，如果它想告诉父组件“我这边完事了”或者“我想改个数据”，它就得发一个**信号**（事件）。

- **形象比喻**：员工给老板发邮件汇报进度：“老板，活干完了，请指示。”
    
- **代码写法**：
    
    JavaScript
    
    ```
    // 子组件：发射信号
    const emit = defineEmits(['success', 'close'])
    
    const submit = () => {
      emit('success', { id: 123 }) // 发送信号，并附带数据
    }
    ```
    
- **关键点**：
    
    - **事件名**：可以自定义，通常用 `@success` 这种形式在父组件监听。
        
    - **解耦**：子组件不需要知道父组件是谁，它只负责把信号“喊”出去。
        

---

### 3. Watch（侦听）：盯着进度 (Side Effects)

**Watch** 用来观察某个数据的变化，并在变化时执行特定的**操作**。

- **形象比喻**：保安盯着监控。一旦发现有人进入（数据变了），立刻按响警报（执行函数）。
    
- **代码写法**：
    
    JavaScript
    
    ```
    const visible = ref(false)
    
    // 盯着 visible 变量
    watch(visible, (newValue, oldValue) => {
      console.log(`状态从 ${oldValue} 变成了 ${newValue}`)
      if (newValue) {
        // 如果变为了 true，执行一些逻辑，比如自动聚焦输入框
      }
    })
    ```
    
- **关键点**：
    
    - **异步操作**：非常适合在数据变动时去发请求（Axios）或者修改其他变量。
        
    - **深度侦听**：如果要盯着一个对象内部属性的变化，需要加 `{ deep: true }`。
        

---

### 🚀 总结：它们是如何协同工作的？

让我们把你刚才写的数据库连接弹窗逻辑串起来：

1. **Props**：父组件通过 `:modelValue` 告诉子组件：“现在你应该显示还是隐藏？”
    
2. **Watch**：子组件盯着这个 `props.modelValue`。一旦发现它变成 `true`，子组件内部也把窗口打开。
    
3. **Emit**：
    
    - 用户点击关闭：子组件发射 `@update:modelValue` 信号，让父组件把开关关掉。
        
    - 连接成功：子组件发射 `@success` 信号，并把后端给的 `token` 给父组件。
        
4. **Watch**（在父组件中）：父组件盯着连接成功的信号，发现有了新数据，立刻把新节点更新到左侧的树（Tree）上。
