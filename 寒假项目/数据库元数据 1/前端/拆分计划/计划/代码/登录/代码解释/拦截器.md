这两句代码是在**拦截器（Interceptor）**或**过滤器（Filter）**中非常经典的操作。

因为拦截器是运行在 Controller 之前的，它没法像 Controller 那样直接 `return Result.error()` 让 Spring Boot 帮你自动转换成 JSON。

所以在拦截器里，如果我们要拒绝用户访问（比如没登录），我们需要**手动**构造一个 HTTP 响应发回给浏览器。

这两行代码就是分工明确的**“发快递”**过程：

---

### 第一句：贴快递单（声明格式）

Java

```
response.setContentType("application/json;charset=utf-8");
```

这行代码是在设置 **HTTP 响应头 (Header)**。它告诉前端（浏览器或 Vue）：

1. **`application/json`**：**“我给你发的是 JSON 数据，不是 HTML 网页”**。
    
    - 如果不写这个，浏览器可能会把它当成普通字符串处理，前端 `axios` 就不会自动把它解析成对象。
        
2. **`charset=utf-8`**：**“我用了 UTF-8 编码”**。
    
    - **这句至关重要！** 因为你的返回信息里包含了中文 `“未登录或登录过期”`。如果不加这句，前端收到的中文大概率会变成 `????` 或者 `åää½` 这种乱码。
        

---

### 第二句：装箱发货（写入内容）

Java

```
response.getWriter().write("{\"code\": 401, \"msg\": \"未登录或登录过期\"}");
```

这行代码是在写 **HTTP 响应体 (Body)**。

1. **`response.getWriter()`**：
    
    - 向 Response 对象申请一只“笔”（`PrintWriter`），准备开始往回写数据了。
        
2. **`write(...)`**：
    
    - 这是真正把数据写进网络流里。
        
3. **`"{\"code\": 401, ...}"`**：
    
    - 这是一个**手写的 JSON 字符串**。
        
    - 因为在拦截器里不能用 Jackson/Gson 等工具自动序列化对象（虽然能用但比较麻烦），所以对于简单的错误提示，程序员通常直接手写这个 JSON 字符串。
        
    - 它对应前端接收到的数据结构：
        
        JSON
        
        ```
        {
            "code": 401,
            "msg": "未登录或登录过期"
        }
        ```
        

---

### 总结

这就好比餐厅服务员（拦截器）拦住了没穿正装的顾客（没 Token）：

- **第一句**：服务员先拿出一张**中文告示牌**（UTF-8），上面写着“通知”（JSON格式）。
    
- **第二句**：服务员在牌子上写下：“对不起，禁止入内”（写入具体错误信息）。
    

**如果没有这两句，前端请求接口时不仅会显示“请求成功”（因为没有报错），而且拿不到任何错误提示，用户就会一脸懵逼地看着页面没反应。**
