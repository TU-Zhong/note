`request.getMethod()` 是 Java Web 开发中最基础、但也最常用的方法之一。

用一句话解释：**它是用来问浏览器：“你想对服务器做什么？”**

它的返回值是一个**全大写的字符串**，代表 HTTP 请求的**动作类型**（也叫 HTTP 动词）。

---

### 1. 它可以拿到什么值？

当你调用 `request.getMethod()` 时，根据前端（Vue/Axios）发起的请求不同，它会返回以下常见的值：

|**返回值 (String)**|**含义**|**前端 Axios 对应写法**|
|---|---|---|
|**"GET"**|**查**。我想获取数据。|`axios.get(...)`|
|**"POST"**|**增**。我想提交表单或新建数据。|`axios.post(...)`|
|**"PUT"**|**改**。我想更新数据。|`axios.put(...)`|
|**"DELETE"**|**删**。我想删除数据。|`axios.delete(...)`|
|**"OPTIONS"**|**问**。探路，询问服务器允许什么操作。|(浏览器自动发送)|

---

### 2. 结合你的代码场景

回到你刚才写的拦截器代码：

Java

```
// request 就是那个“HTTP 请求包裹”
// .getMethod() 就是看包裹上贴的标签是啥
if ("OPTIONS".equalsIgnoreCase(request.getMethod())) {
    return true; // 放行
}
```

这段代码的翻译就是：

1. **`request.getMethod()`**：拿到动作标签（比如 "GET" 或 "OPTIONS"）。
    
2. **`"OPTIONS".equalsIgnoreCase(...)`**：判断这个标签是不是 "OPTIONS"（忽略大小写，防止有些奇葩浏览器发 "options"）。
    
3. **逻辑**：
    
    - 如果标签是 **"OPTIONS"**（说明是浏览器派来的探路兵）：**放行！**
        
    - 如果标签是 **"GET" / "POST"**（说明是真正的业务请求）：**继续往下走**，去检查 Authorization Token。
        

---

### 3. 形象的比喻

把 **服务器** 想象成一个 **公司前台**（拦截器）。

把 **请求（Request）** 想象成 **访客**。

`request.getMethod()` 就是前台问访客的那句话：**“您今天是来办什么业务的？”**

- 访客 A 说：**“GET”**（我是来拿文件的）。
    
    - 前台：好，请出示证件（检查 Token）。
        
- 访客 B 说：**“POST”**（我是来送文件的）。
    
    - 前台：好，请出示证件（检查 Token）。
        
- 访客 C 说：**“OPTIONS”**（我就是路过问问你们几点下班，不进门）。
    
    - 前台：哦，问话啊？那你不用登记证件了，直接告诉你吧（直接放行，`return true`）。
        

### 总结

`request.getMethod()` 就是用来**识别动作类型**的。在拦截器里，我们利用它来把“仅仅是来问路”的 `OPTIONS` 请求挑出来，给予特殊待遇（免检）。
