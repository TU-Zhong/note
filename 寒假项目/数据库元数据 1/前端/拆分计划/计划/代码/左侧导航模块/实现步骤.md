既然你已经搭好了页面的大框架（MainLayout），并且理解了“大脑”（Pinia）的运作逻辑，现在我们要开始编写**最核心的交互触发点**——`DbSidebar.vue`。

在手敲这个组件时，建议按照**“由表及里”**的顺序，分三步走：

---

### 1. 第一步：创建文件并建立“骨架” (Template)

在 `src/components/database/` 目录下新建 `DbSidebar.vue`。先写 HTML 结构，理清这里面有几个区域。

- **为什么要先写这个？** 这样你能在浏览器里立刻看到侧边栏被划分为：顶部的按钮区、中间的搜索框、底面的树形列表。
    

Code snippet

```
<template>
  <div class="db-sidebar">
    <div class="sidebar-toolbar">...</div>

    <div class="sidebar-search">...</div>

    <div class="tree-container">...</div>
  </div>
</template>
```

---

### 2. 第二步：引入“大脑”和图标 (Script Setup)

在 `<script setup>` 中，先完成三件事：**导包、引 Store、定数据**。

- **重点关注**：`useWorkspaceStore` 的引入。这是 Sidebar 能指挥右侧工作区的唯一通道。
    
- **模拟数据**：因为后端接口还没联调，先在 `treeData` 里手写几行模拟数据（如 `t_student`），方便你测试双击效果。
    

JavaScript

```
<script setup>
import { ref, onMounted } from 'vue'
import { Plus, Refresh, Search, Coin, Grid } from '@element-plus/icons-vue' // 导入图标
import { useWorkspaceStore } from '../../stores/useWorkspaceStore' // 引入大脑

const store = useWorkspaceStore()
const treeData = ref([]) // 这里先给点 mock 数据测试
// ... 接下来写 handleNewConsole 和 handleNodeDblClick
</script>
```

---

### 3. 第三步：编写关键逻辑 (Actions)

这是最体现你软件工程思维的地方，你需要实现两个**事件监听**：

1. **点击“+”图标**：调用 `store.addTab({ type: 'console' })`。
    
2. **双击树节点**：判断如果是“表（table）”类型，调用 `store.addTab({ type: 'table', title: node.label })`。
    

---

### 4. 第四步：穿上“IDEA 的西装” (Style)

最后写 CSS。这一步最费时间，建议边写边在浏览器 F12 里调颜色。

- **专业建议**：一定要给 `el-tree` 设置深色背景。Element Plus 默认是白色的，放在你的深色大框架里会非常刺眼。你需要用到 `:deep(.el-tree)` 这个伪类来强制修改组件内部样式。
