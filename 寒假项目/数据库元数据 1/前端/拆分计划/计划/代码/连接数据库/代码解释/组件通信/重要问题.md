这两个写法有什么区别？难道就是上面的是赋定值，下面的是绑定读取js里面的变量值？



你的直觉非常准！这正是 Vue 中 **“静态属性”** 与 **“动态绑定”** 的核心区别。

简单来说，有没有那个冒号（`:`），决定了引号里的内容是被当作 **“普通字符串”** 还是 **“JavaScript 代码”**。

---

### 1. 核心对比表

|**写法**|**名称**|**引号里的内容是什么？**|**最终传给子组件的值**|
|---|---|---|---|
|`title="101"`|**静态属性**|纯文本字符串|字符串 `"101"`|
|`:id="101"`|**动态绑定 (v-bind)**|**JavaScript 表达式**|数字 `101`|

---

### 2. 深度拆解

#### ① 静态写法：`title="101"`

这种写法和原生 HTML 一模一样。Vue 不会对引号里的内容做任何处理。

- **传值类型**：永远是 **String**。
    
- **适用场景**：传一些固定不变的文本，比如标题、占位符。
    
- **例子**：`placeholder="请输入用户名"`。
    

#### ② 动态写法：`:id="101"`

这里的 `:` 是 `v-bind:` 的缩写。它告诉 Vue：“嘿，把引号里的东西当成 JS 代码运行一下，再把结果传过去。”

- **传值类型**：可以是 **数字、布尔值、对象、数组** 等任何 JS 类型。
    
- **适用场景**：
    
    - **传变量**：`:id="userId"`（读取 JS 里的 `userId` 变量）。
        
    - **传非字符串类型**：`:is-loading="true"`（传的是布尔值，不加冒号传的就是字符串 `"true"`）。
        
    - **传表达式**：`:title="'Prefix-' + name"`。
        

---

### 3. 一个极易踩坑的例子：布尔值

作为一名软件工程专业的学生，你在处理组件逻辑（比如 Loading 状态）时，必须注意这一点：

HTML

```
<MyComponent is-loading="false" />

<MyComponent :is-loading="false" />
```

### 总结你的疑问

- **`title="新建MySQL连接"`**：Vue 看到没有冒号，直接把 `"新建MySQL连接"` 这几个字原封不动扔给子组件。
    
- **`:id="101"`**：Vue 看到有冒号，它会想：“哦，这是 JS。101 在 JS 里是个数字。” 于是它把**数字类型**的 `101` 扔给子组件。
    

如果你在 JS 里定义了一个变量 `const myId = ref(101)`，那么写成 `:id="myId"`，子组件拿到的就是这个变量的值。
