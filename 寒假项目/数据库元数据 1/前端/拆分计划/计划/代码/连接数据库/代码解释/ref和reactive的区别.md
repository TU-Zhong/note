这是 Vue 3 中最经典的面试题，也是新手最容易晕的地方。

简单来说：**`ref` 是通用的（万能的），`reactive` 是专用的（局限的）。**

我用一个**“包装盒”**的比喻来帮你彻底搞懂：

---

### 1. `ref()` —— 万能的“包装盒”

想象 `ref` 是一个**小盒子**。

- **你可以往里面放任何东西**：数字、字符串、布尔值，甚至是一个对象。
    
- **读写规则**：因为东西在盒子里，所以你在 JS 里操作时，必须**打开盒子**才能拿到东西，这就是为什么要加 **`.value`**。
    
- **模板里**：Vue 帮你自动打开了盒子，所以在 `<template>` 里**不用写** `.value`。
    

JavaScript

```
// ✅ 放简单类型
const count = ref(0)
count.value = 1 // 必须加 .value

// ✅ 放对象也可以
const user = ref({ name: 'admin' })
user.value.name = 'root' // 必须加 .value
```

### 2. `reactive()` —— 透明的“代理人”

想象 `reactive` 是一个**智能外壳**（Proxy）。

- **只能包对象**：它**不能**包数字、字符串等基本类型。它只能包对象 (`{}`)、数组 (`[]`) 等。
    
- **读写规则**：它就像一个透明的代理，你**不需要**加 `.value`，直接用属性名就行。
    

JavaScript

```
// ❌ 报错！不能放基本类型
// const count = reactive(0) 

// ✅ 只能放对象
const user = reactive({ name: 'admin' })
user.name = 'root' // 不需要 .value，直接改
```

---

### 3. 核心区别对比表（背下这个就够了）

|**特性**|**ref (推荐)**|**reactive**|
|---|---|---|
|**支持类型**|**所有类型** (数字, 字符串, 对象, 数组)|**仅限引用类型** (对象, 数组, Map, Set)|
|**JS中访问**|需要 **`.value`**|**不需要** `.value`|
|**Template中访问**|不需要 `.value` (自动解包)|不需要 `.value`|
|**重新赋值(大坑)**|**可以整体替换** (`form.value = {}`)|**严禁整体替换** (`form = {}` 会导致失去响应式)|

---

### 4. 为什么大家现在更喜欢用 `ref`？

也就是你之前遇到的那个场景。

**`reactive` 有一个致命的死穴：不能直接赋值替换！**

假设你有一个表单 `form`，你想在清空表单时直接把它重置：

#### ❌ 用 reactive 的惨案：

JavaScript

```
let form = reactive({ name: 'admin', pwd: '123' })

// 你想重置表单，于是你写了：
form = { name: '', pwd: '' } 

// 💀 完了！form 变成了一个普通的死对象，界面再也不会更新了！
// 因为你切断了 reactive 原来的连接。
```

#### ✅ 用 ref 的优雅：

JavaScript

```
const form = ref({ name: 'admin', pwd: '123' })

// 你想重置表单：
form.value = { name: '', pwd: '' }

// 🎉 完美！因为你换的是盒子里的东西，盒子本身（ref）还在，响应式依然有效。
```

### 💡 总结建议

作为初学者，为了避免心智负担和踩坑，**我强烈建议你 95% 的情况都使用 `ref`**。

- **什么时候用 `ref`**：
    
    - 数字、字符串、布尔值（必须用）。
        
    - 后端返回的数据（即使是对象/数组，也建议用 ref，方便整体替换）。
        
    - DOM 元素引用（比如 `treeRef`，必须用 ref）。
        
- **什么时候用 `reactive`**：
    
    - 只有当你非常确定这个对象里的字段很固定，且**永远不会发生整体替换**（比如一些固定的配置项 config）时，才考虑用 `reactive` 少写几个 `.value`。
        

**一句话口诀：遇事不决用 `ref`，JS 里记得加 `.value`。**
