### 📜 核心协议：`TabObject` (标签页对象)

这是我们整个前端应用的**“通用货币”**。无论是在侧边栏、Pinia 仓库，还是右侧组件里，我们流转的都**必须**是这个结构。

TypeScript

```
// 伪代码定义 (Interface)
interface TabObject {
  id: string;       // 【唯一身份证】用于 Vue 的 :key 和删除操作 (例如: 't_1', 'console_1')
  title: string;    // 【标题】显示在标签页头上的文字 (例如: 't_student', 'SQL控制台')
  type: string;     // 【类型】决定加载哪个组件 (枚举值: 'table' | 'console')
  params: object;   // 【私货】子组件干活需要的具体数据 (例如: { tableName: 'xxx' })
}
```

### 🔍 场景举例：两种不同的“包裹”

根据这个协议，我们来看看你的系统里目前仅有的两种包裹长什么样：

#### 场景 A：用户双击了 `t_student` 表

这是你要传给 `TableDataViewer.vue` 的数据：

JavaScript

```
{
  id: 't_1',                // 对应树节点的 id
  title: 't_student',       // 对应树节点的 label
  type: 'table',            // 🔥 告诉 WorkArea：加载 TableDataViewer 组件
  params: {
    tableName: 't_student', // 🔥 告诉 TableDataViewer：你去查这张表的数据
    dbName: 'mysql_prod'    // (可选) 将来可能还需要数据库名
  }
}
```

#### 场景 B：用户点击了“新建查询”

这是你要传给 `SqlConsole.vue` 的数据：

JavaScript

```
{
  id: 'console_17382055',   // 随机生成的唯一ID
  title: 'Console',         // 固定标题
  type: 'console',          // 🔥 告诉 WorkArea：加载 SqlConsole 组件
  params: {
    initSql: 'SELECT * FROM...' // (可选) 比如你想给它预填一些 SQL
  }
}
```

### 🛠️ 全链路修正 (按照规范来)

现在我们要根据这个规范，检查并统一你三个地方的代码。

#### 1. 发货方 (`DbSidebar.vue`)

检查 `store.addTab` 里的参数是否符合规范。

JavaScript

```
// DbSidebar.vue
const handleNodeDblClick = (data) => {
  if (data.type === 'table') {
    // ✅ 发货单填写规范
    store.addTab({
      id: data.id,             // 必填：唯一ID
      title: data.label,       // 必填：标题
      type: 'table',           // 必填：类型
      params: {                // 必填：参数包
        tableName: data.label,
        // 可以在这里扩展，比如加 database: 'xxx'
      }
    });
  }
};
```

#### 2. 中转站 (`WorkArea.vue`)

检查它是如何拆包并分发的。

HTML

```
<component 
  :is="componentMap[item.type]" 
  :tab-id="item.id"      :params="item.params"  />
```

#### 3. 收货方 (`TableDataViewer.vue`)

检查子组件是否声明了它要接收的东西。

JavaScript

```
// TableDataViewer.vue
<script setup>
// ✅ 声明接收：我只关心 tabId 和 params
const props = defineProps(['tabId', 'params'])

// 使用数据
console.log("我要查询的表是：", props.params.tableName) 
</script>
```

### 💡 现在的清晰度

现在你的脑海里应该有这样一张图：

1. **Sidebar** 生产一个标准包裹 `{ id, type, params }`。
    
2. **Pinia** 把包裹存进 `tabs` 数组。
    
3. **WorkArea** 遍历数组，看到 `type='table'` 就叫来 **表格组件**。
    
4. **WorkArea** 把 `params` 递给 **表格组件**。
    
5. **表格组件** 打开 `params`，拿出 `tableName`，开始干活。
    

**这样梳理一下，你对 `item.params` 和 `item.type` 的关系清楚了吗？如果清楚了，我们就可以放心地继续下一步了！**
