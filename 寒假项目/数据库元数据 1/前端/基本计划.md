## 📅 复刻 IDEA 数据库控制台：五阶段实施计划

### 第一阶段：基础布局与“工作空间”搭建

这个阶段的目标是搭建出 IDEA 的“外壳”，确立多标签页的核心交互逻辑。

1. **全局容器布局**：
    
    - 使用 `el-container` 构建三段式布局：顶部全局状态栏、左侧导航侧边栏、右侧主展示区。
        
2. **多标签页 (Tabs) 系统设计**：
    
    - 引入 Pinia 或 Vuex 管理全局的 `tabs` 数组。
        
    - 定义标签页类型：`SQL_CONSOLE`（SQL 编辑器）、`TABLE_DATA`（表数据查看）、`DDL_VIEW`（结构预览）。
        
    - 实现点击树节点或工具栏按钮时，动态向数组 `push` 对象并激活对应标签。
        
3. **主题与样式统一**：
    
    - 配置一套深色/浅色切换的 CSS 变量，模仿 IDEA 的 Darcula 主题风格。
        

---

### 第二阶段：左侧“Database”工具窗口实现

对应你计划中的 Day 11，重点在于元数据的可视化展示。

1. **顶部工具栏 (Icon Toolbar)**：
    
    - **New Connection (`+`)**：点击呼出连接配置对话框。
        
    - **Refresh (`↻`)**：重载当前连接的树形结构。
        
    - **New Console (`SQL`)**：在当前选中的数据库上下文下开启一个新的 SQL 标签页。
        
2. **多层级元数据树 (Metadata Tree)**：
    
    - 实现四级加载逻辑：`Connection` -> `Database/Schema` -> `Table/View` -> `Columns/Indexes`。
        
    - **自定义渲染**：为不同类型的节点配置 IDEA 风格的图标（如🔑代表主键，🍃代表普通索引）。
        
3. **节点交互逻辑**：
    
    - **单击**：在右侧展示该节点的属性摘要。
        
    - **双击（表节点）**：自动开启一个 `TABLE_DATA` 标签页，查询该表前 100 条数据。
        

---

### 第三阶段：SQL 编辑器与控制台集成

这是最核心的“硬核”功能，对应计划中的 Day 12。

1. **Monaco Editor 集成**：
    
    - 在 `SQL_CONSOLE` 标签页中嵌入编辑器。
        
    - 配置 SQL 语言包，开启代码自动补全和语法高亮。
        
2. **上下文关联**：
    
    - 确保每个控制台标签页都绑定了对应的 `connectionToken` 和当前 `schema`。
        
3. **结果集渲染区域**：
    
    - 在编辑器下方设计一个可拖拽高度的面板。
        
    - 使用 `el-table` 动态渲染 SQL 执行结果（支持动态列生成）。
        
    - 添加“执行耗时”和“受影响行数”的状态提示信息。
        

---

### 第四阶段：连接管理与弹窗交互

完善系统的入口，确保用户可以动态增加新的数据源。

1. **动态连接表单**：
    
    - 设计支持多种驱动（SQL Server, MySQL, Postgre）的通用表单。
        
2. **连接测试 (Test Connection)**：
    
    - 在保存前调用后端接口测试连通性，并反馈详细的错误日志。
        
3. **本地存储 (Persistence)**：
    
    - 将连接配置（不含敏感密码）保存在浏览器的 `localStorage` 中，实现“记住连接”功能。
        

---

### 第五阶段：后端接口适配与安全风控

确保后端能支撑前端的“IDEA 式”操作，对应计划中的 Day 13-14。

1. **SQL 路由接口**：
    
    - 编写 `/api/sql/execute`，支持接收任意 SQL 字符串。
        
2. **安全黑名单拦截**：
    
    - 在后端增加解析层，拦截非管理员执行 `DROP`、`TRUNCATE` 等高危 DDL 语句。
        
3. **DDS (Dynamic Driver Loading) 优化**：
    
    - 确保在前端频繁切换标签页时，后端能快速根据 Token 找回数据库连接。
