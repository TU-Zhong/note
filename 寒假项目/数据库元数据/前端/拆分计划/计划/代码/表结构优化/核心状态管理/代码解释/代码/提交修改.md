太激动人心了！我们终于来到了前端数据交互的最终章——**将差异池（Diff Pool）里的数据打包，发给后端！**

作为一个专业的数据库 IDE，我们在提交数据时，绝对不能把整个 100 行的表格全发给后端，那样性能太差了。我们**只需要把“发生变化”的那几行挑出来**发过去。

这里有一个**非常硬核的架构设计点**：

对于“新增”和“删除”，我们只需要发一行数据。但对于**“修改（Update）”**，我们最好同时把**修改前（oldRow）**和**修改后（newRow）**的数据都发给后端。因为有些表可能没有主键，后端必须靠 `oldRow` 的各个字段去精确拼装 `WHERE` 条件，才能保证不改错数据。

请按照以下两步，把提交逻辑补充完整：

### 1. 给提交按钮加上 Loading 状态

为了防止用户网络卡顿时疯狂连点，我们需要加一个防抖的 `isSubmitting` 状态。

找到你的 `<template>` 里的提交按钮，加上 `:loading` 属性：

HTML

```html
<el-button 
  size="small" 
  type="primary" 
  :icon="Check" 
  @click="handleSubmit" 
  :disabled="!hasChanges" 
  :loading="isSubmitting"
  title="提交更改到数据库"
>
  提交 ({{ totalChanges }})
</el-button>
```

### 2. 编写硬核的 `handleSubmit` 逻辑

在 `<script setup>` 中，找到之前那个只有 `console.log` 的空壳 `handleSubmit`，把它替换成下面这套**完整的数据清洗与组装逻辑**：

JavaScript

```js
// 新增一个提交中的状态
const isSubmitting = ref(false)

// 🔪 数据清洗刀：专门用来在提交前，把前端专用的 __row_id 给剔除掉，防止后端报错
const cleanRow = (row) => {
  const cleanData = { ...row }
  delete cleanData.__row_id
  return cleanData
}

// 🚀 真实的提交逻辑
const handleSubmit = async () => {
  if (!hasChanges.value) return
  isSubmitting.value = true

  try {
    // 1. 组装新增的数据 (从 tableData 里找)
    const insertedList = tableData.value
      .filter(row => diffPool.inserted.has(row.__row_id))
      .map(row => cleanRow(row))

    // 2. 组装修改的数据 (需要同时拿出修改前和修改后的数据)
    const updatedList = []
    diffPool.updated.forEach(rowId => {
      // 当前表格里的新数据
      const newRow = tableData.value.find(r => r.__row_id === rowId)
      // 快照库里的老数据
      const oldRow = originalData.value.find(r => r.__row_id === rowId)
      
      if (newRow && oldRow) {
        updatedList.push({
          oldRow: cleanRow(oldRow),
          newRow: cleanRow(newRow)
        })
      }
    })

    // 3. 组装删除的数据 (⚠️ 必须从 originalData 里找，因为它们可能已经从 tableData 里被 splice 抹除了！)
    const deletedList = originalData.value
      .filter(row => diffPool.deleted.has(row.__row_id))
      .map(row => cleanRow(row))

    // 4. 打包成最终要发给后端的 Payload
    const payload = {
      tableName: props.params.tableName,
      schema: props.params.schema,
      inserted: insertedList,
      updated: updatedList,
      deleted: deletedList
    }

    console.log("准备发送给后端的终极数据包：", payload)

    // 5. 发起真实请求 (假设你的后端同步接口叫 /api/metadata/table/data/sync)
    const res = await axios.post('http://localhost:8080/api/metadata/table/data/sync', payload, {
      headers: {
        'Authorization': localStorage.getItem('ConnectionToken') || ''
      }
    })

    if (res.data.code === 200) {
      ElMessage.success(`提交成功！共影响 ${totalChanges.value} 行数据`)
      // ✨ 提交成功后，必须重新拉取一遍最新数据！
      // fetchData 内部包含清空 diffPool 的逻辑，表格会自动恢复成干净的状态
      await fetchData() 
    } else {
      ElMessage.error('提交失败：' + (res.data.msg || '未知错误'))
    }

  } catch (error) {
    console.error(error)
    ElMessage.error('请求异常，请检查网络或后端状态')
  } finally {
    isSubmitting.value = false
  }
}
```
