为了实现真正的 DataGrip 体验，我们必须把现在表格里的数据，跟咱们一开始存在 `originalData`（源数据快照）里的数据进行**逐个格子的对比**。

### 🛠️ 终极修复方案（同时避开另一个隐藏天坑）

不仅要对比，这里还有一个前端极其容易踩中的**“数据类型天坑”**：

当你双击年龄（原本是数字 `20`），`<el-input>` 经常会把数字变成字符串 `"20"`。如果用严格不等于 `!==`，系统会认为 `20 !== "20"`，从而误判你修改了数据！

所以我们需要把它们统一转成字符串再对比。

请用下面这段代码，**完全覆盖**你现在的 `handleInputBlur` 函数：

JavaScript

```
// 输入框失去焦点，或者按下回车键时触发
const handleInputBlur = (row) => {
  // 1. 关闭输入框模式，变回普通文本
  editingCell.value = { rowId: null, colKey: null }
  
  // 2. 如果是刚刚新增的空行，它永远是 inserted 状态，不需要做对比
  if (diffPool.inserted.has(row.__row_id)) return

  // 3. 核心鉴定机制：去“快照库”里找出这行数据最原始的样子
  const originalRow = originalData.value.find(r => r.__row_id === row.__row_id)
  
  if (originalRow) {
    let isChanged = false
    // 遍历这行的每一列，逐个格子进行比对
    for (const col of tableColumns.value) {
      
      // 【防坑指南】统一转成字符串对比，防止数字 20 和字符串 "20" 造成误判，同时处理 null 的情况
      const val1 = row[col] === null || row[col] === undefined ? '' : String(row[col])
      const val2 = originalRow[col] === null || originalRow[col] === undefined ? '' : String(originalRow[col])

      if (val1 !== val2) {
        isChanged = true
        break // 只要发现有一个格子不一样，直接判定为“已修改”，跳出循环
      }
    }

    // 4. 根据比对结果决定要不要变色
    if (isChanged) {
      // 确实改了，放入修改池（变黄）
      diffPool.updated.add(row.__row_id)
    } else {
      // ✨ 极其惊艳的体验：如果没有改，或者用户改了一圈又改回了原来的值
      // 我们不仅不加它，还要把它从修改池里踢出去（撤销黄底色）！
      diffPool.updated.delete(row.__row_id)
    }
  }
}
```
