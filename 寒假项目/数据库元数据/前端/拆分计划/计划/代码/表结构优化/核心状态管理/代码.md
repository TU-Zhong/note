要追踪数据的变化，我们首先面临一个技术挑战：**如何唯一识别表格中的一行？** 后端返回的数据虽然可能有主键（比如 `student_id`），但并不是每张表都有主键，而且如果是我们刚刚点击“新增”产生的空行，它在提交给后端之前是**没有主键**的。

因此，业界标准的做法是：在前端拿到数据的那一瞬间，给每一行打上一个**前端专属的虚拟唯一 ID（UUID/雪花ID）**。以后所有的“增删改”追踪，全部认准这个虚拟 ID。

咱们直接在 **`TableDataViewer.vue`** 中实现这三大核心状态！为了让你能立刻看到“脏检查”的视觉效果，我顺便加了三个**“临时测试按钮”**。

请用以下代码完全覆盖你的 `TableDataViewer.vue`：

Code snippet

[[TableDataViewer中的defineProps]]
```html
<template>
  <div style="height: 100%; display: flex; flex-direction: column;">
    
    <div class="temp-toolbar">
      <span style="font-size: 14px; font-weight: bold; margin-right: 15px;">
        数据预览：{{params.tableName}}
      </span>
      <el-button size="small" type="success" plain @click="testAddRow">测试：新增行</el-button>
      <el-button size="small" type="warning" plain @click="testUpdateRow">测试：修改首行</el-button>
      <el-button size="small" type="danger" plain @click="testDeleteRow">测试：删除末行</el-button>
    </div>

    <div style="flex: 1; overflow: hidden; min-height: 0;">
      <el-table
        :data="tableData"
        height="100%" 
        border
        style="width: 100%"
        :row-class-name="getRowClassName"
      >
        <el-table-column
          v-for="col in tableColumns"
          :key="col"
          :prop="col"
          :label="col"
          min-width="120"
          show-overflow-tooltip
          sortable
        />
        <template #empty>
          <el-empty :image-size="100" description="暂无数据"></el-empty>
        </template>
      </el-table>
    </div>
  </div>
</template>

<script setup>
import { ref, reactive, onMounted } from 'vue'
import { ElMessage } from 'element-plus'
import axios from 'axios'

const props = defineProps({
  tabId: String,
  params: {
    type: Object,
    default: () => ({ tableName: '', schema: '' })
  }
})

// === 第一阶段：核心状态管理 (脏检查机制) ===

const tableColumns = ref([]) // 表头列名
const tableData = ref([])    // 当前表格显示的数据（双向绑定）
const originalData = ref([]) // 1. 定义源数据快照（用于对比和撤销）
const loading = ref(false)   // 加载状态

// 2. 构建差异追踪池 (Diff Pool)
// 我们使用 Set 来存储发生变化的行的 __row_id，保证唯一性且查找效率高
const diffPool = reactive({
  inserted: new Set(),
  updated: new Set(),
  deleted: new Set()
})

// 生成前端虚拟主键的方法
const generateRowId = () => `row_${Date.now()}_${Math.floor(Math.random() * 10000)}`

onMounted(() => {
  fetchData()
})

const fetchData = async () => {
  if (!props.params.tableName) {
    ElMessage.warning('未指定表名')
    return
  }
  loading.value = true

  try {
    const res = await axios.get('http://localhost:8080/api/metadata/table/data', {
      headers: {
        'Authorization': localStorage.getItem('ConnectionToken') || ''
      },
      params: {
        tableName: props.params.tableName,
        schema: props.params.schema
      }
    })
    
    if (res.data.code === 200) {
      const resultData = res.data.data
      
      if (resultData.columns && Array.isArray(resultData.columns)) {
        tableColumns.value = resultData.columns
      }
      
      if (resultData.rows && Array.isArray(resultData.rows)) {
        // 【核心逻辑】：拿到数据后，立刻给每一行注入前端虚拟主键 __row_id
        const rowsWithVirtualId = resultData.rows.map(row => ({
          ...row,
          __row_id: generateRowId() 
        }))

        tableData.value = rowsWithVirtualId
        
        // 1. 深拷贝一份作为快照 (这里用 JSON 序列化做深拷贝足够应付基础表格数据)
        originalData.value = JSON.parse(JSON.stringify(rowsWithVirtualId))
        
        // 清空之前的追踪池（刷新数据相当于重置状态）
        diffPool.inserted.clear()
        diffPool.updated.clear()
        diffPool.deleted.clear()
      }
      ElMessage.success('查询成功！')
    }
  } catch (error) {
    console.error(error)
  } finally {
    loading.value = false
  }
}

// 3. UI 状态标记：动态赋予 CSS 类名
const getRowClassName = ({ row }) => {
  // 如果这行的虚拟 ID 存在于删除池中，赋予删除样式
  if (diffPool.deleted.has(row.__row_id)) {
    return 'row-deleted'
  }
  // 如果在新增池中，赋予新增样式
  if (diffPool.inserted.has(row.__row_id)) {
    return 'row-inserted'
  }
  // 如果在修改池中，赋予修改样式
  if (diffPool.updated.has(row.__row_id)) {
    return 'row-updated'
  }
  return '' // 没有变化的行，不加特殊样式
}

// === 测试用的模拟操作函数 ===

const testAddRow = () => {
  // 构造一个空行对象，所有的列先填 null
  const newRow = { __row_id: generateRowId() }
  tableColumns.value.forEach(col => {
    newRow[col] = null
  })
  
  // 插到表格最前面
  tableData.value.unshift(newRow)
  // 记录到 Diff Pool 的 inserted 集合中
  diffPool.inserted.add(newRow.__row_id)
}

const testUpdateRow = () => {
  // 找一个没有被删除的行来修改
  const targetRow = tableData.value.find(row => !diffPool.deleted.has(row.__row_id))
  if (targetRow && tableColumns.value.length > 0) {
    const firstCol = tableColumns.value[0]
    targetRow[firstCol] = '被修改的值!'
    
    // 如果它不是新插入的行，才需要标记为 updated（新插入的行修改了也还是 inserted）
    if (!diffPool.inserted.has(targetRow.__row_id)) {
      diffPool.updated.add(targetRow.__row_id)
    }
  }
}

const testDeleteRow = () => {
  // 从后往前找一个还没被删除的行
  for (let i = tableData.value.length - 1; i >= 0; i--) {
    const row = tableData.value[i]
    if (!diffPool.deleted.has(row.__row_id)) {
      
      // 如果这个行是刚新增的，直接从数组里删掉就行，不用标记为 deleted
      if (diffPool.inserted.has(row.__row_id)) {
        tableData.value.splice(i, 1)
        diffPool.inserted.delete(row.__row_id)
      } else {
        // 如果是原本就存在的行，不能从数组删，只能标记为已删除（因为最后要发给后端执行 DELETE 语句）
        diffPool.deleted.add(row.__row_id)
      }
      break
    }
  }
}
</script>

<style scoped>
/* 临时工具栏样式 */
.temp-toolbar {
  flex-shrink: 0;
  padding: 8px 12px;
  background-color: #f5f7fa;
  border-bottom: 1px solid #e4e7ed;
  display: flex;
  align-items: center;
}

/* === 脏检查状态对应的颜色样式 === 
  注意：使用 :deep() 穿透 el-table，并且加上 !important 覆盖 Element Plus 默认的斑马纹或 Hover 颜色
*/

/* 新增行：浅绿色背景 */
:deep(.el-table .row-inserted) {
  background-color: #f0f9eb !important; 
}

/* 修改行：浅黄色背景 */
:deep(.el-table .row-updated) {
  background-color: #fdf6ec !important; 
}

/* 删除行：浅红色背景 + 灰色文字 + 删除线 */
:deep(.el-table .row-deleted) {
  background-color: #fef0f0 !important; 
  color: #a8abb2 !important;
  text-decoration: line-through;
}

/* 让被删除的行鼠标悬停时不要恢复颜色，保持灰色 */
:deep(.el-table .row-deleted:hover > td) {
  background-color: #fef0f0 !important;
}
</style>
```

### 💡 这段代码实现了什么魔法？

1. **虚拟主键注入**：每次后端数据拉回来（`resultData.rows`），代码都会悄悄给它们塞入一个形如 `row_1700123456_4567` 的隐藏属性 `__row_id`。
    
2. **状态分离机制**：你在页面上点击“删除”时，注意看 `testDeleteRow` 的代码逻辑：如果是原本就存在的数据，我们并没有把它从数组里 `splice` 删掉，而是把它扔进了 `diffPool.deleted` 集合里。**这就是商业级 IDE 的做法**：前端假删，只是加上删除线，等用户最后点“提交”时，再把这些 ID 发给后端执行真正的 SQL 删除。
    
3. **响应式 UI 映射**：Element Plus 的 `row-class-name` 属性会自动监听 `diffPool` 的变化。只要一行数据的 `__row_id` 进入了对应的池子，这一行就会瞬间变色。
