你需要修改 `handleNewConsole` 函数的逻辑。不再依赖一个局部的 `indexOfConsole` 变量，而是每次点击新建按钮时，**实时计算**当前 `store.tabs` 中最大的控制台序号，然后加 1。

以下是具体的修改方案：

### 修改思路

1. **获取现有控制台**：从 `store.tabs` 中筛选出 `type` 为 `'console'` 的所有标签。
    
2. **提取序号**：遍历这些标签的 `title`（例如 "Console-1"），提取横杠后面的数字。
    
3. **计算最大值**：找出这些数字中的最大值。如果没有控制台，最大值默认为 0。
    
4. **生成新序号**：新序号 = 最大值 + 1。
    

### 代码修改

请在你的 `<script setup>` 中替换原来的 `handleNewConsole` 逻辑，并删除 `indexOfConsole` 变量。

JavaScript

```js
// ... existing imports ...

// 【删除】这行不再需要了，因为我们不依赖本地状态，而是依赖 store 状态
// let indexOfConsole=ref(1) 

// 【修改】新的处理函数
const handleNewConsole = () => {
  // 1. 筛选出所有类型为 console 的标签
  const consoleTabs = store.tabs.filter(tab => tab.type === 'console');
  
  // 2. 计算下一个序号
  let nextIndex = 1;
  
  if (consoleTabs.length > 0) {
    // 提取所有 title 中的数字 (假设格式固定为 "Console-数字")
    const indices = consoleTabs.map(tab => {
      // 分割字符串获取后面的数字部分
      const parts = tab.title.split('-'); 
      // 转换成整数，如果解析失败返回 0
      return parts.length > 1 ? parseInt(parts[1]) : 0;
    });
    
    // 找出最大值并 +1
    const maxIndex = Math.max(...indices);
    nextIndex = maxIndex + 1;
  }

  console.log("添加一个控制台，序号：", nextIndex);
  
  // 3. 添加 Tab
  store.addTab({
    // 建议 ID 加上前缀以保证唯一性，且为 string 类型
    id: `console_${nextIndex}`, 
    title: `Console-${nextIndex}`,
    type: 'console'
  })
}

// ... existing code ...
```

### 为什么这样做更好？

1. **防刷新丢失**：原来的写法中，如果你刷新页面，`indexOfConsole` 会重置为 1，导致再次点击时会生成重复的 "Console-1"。现在的写法只要 Store 里有数据（比如你做了持久化），它就能正确计算下一个数字。
    
2. **智能填补**：虽然目前的逻辑是 "最大值 + 1"，这符合大多数 IDE（如 Navicat/DataGrip）的逻辑。例如你有 "Console-1" 和 "Console-3"，新建时会生成 "Console-4"，避免了 ID 冲突。
    

### 对应的 Store 类型建议

注意，你的 `useWorkspaceStore` 中定义的 `addTab` 接收的 `id` 类型可能是 string。在上面的代码中，我将 `id` 设置为了 `` `console_${nextIndex}` `` 这种字符串形式，这比纯数字更安全，能防止与数据库表的 ID（如果也是数字）发生冲突。

确保你的 Store 里 `tabs` 的定义能正确响应即可（Pinia 默认就是响应式的）。
