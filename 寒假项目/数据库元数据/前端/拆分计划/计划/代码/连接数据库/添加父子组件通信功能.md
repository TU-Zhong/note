为了让你这个组件从“单机版”变成真正的“联机版”，我们需要把刚才讲的 **Props**、**Emits** 和 **Watch** 填进去。

你的代码需要改动三个关键地方：

1. **接听信号**：定义 `props` 接收父组件的 `v-model` 指令。
    
2. **建立同步**：用 `watch` 让内部的 `visible` 和外部的信号保持一致。
    
3. **汇报成果**：在连接成功后，通过 `emit` 把后端给的 `token` 传给父组件，以便更新左侧的树。

### ✅ 修改后的 `<script setup>`

直接替换你原来的脚本部分：

JavaScript

```js
<script setup>
import { ref, reactive, watch } from 'vue' // 1. 别忘了引入 watch
import axios from 'axios'
import { ElMessage } from 'element-plus'

// --- 🛠️ 第一部分：双向绑定同步逻辑 ---

// 接收父组件传来的 v-model 值（Vue 3 默认名字是 modelValue）
const props = defineProps(['modelValue'])
// 定义发射器：一个用于同步开关，一个用于汇报成功
const emit = defineEmits(['update:modelValue', 'success'])

// 内部控制弹窗显示的变量，初始设为 false
const visible = ref(false)

// 【盯着外部】：父组件开关变了，我内部跟着变
watch(() => props.modelValue, (newVal) => {
  visible.value = newVal
})

// 【盯着内部】：我自己关了（点叉号或取消），告诉父组件把开关也关了
watch(visible, (newVal) => {
  emit('update:modelValue', newVal)
})

// --- 🛠️ 第二部分：表单与逻辑 ---

const formRef = ref()
const loading = ref(false)

const form = reactive({
  name: 'Local-Database',
  jarPath: 'D:\\maven_repo\\mysql\\mysql-connector-j\\8.0.33\\mysql-connector-j-8.0.33.jar',
  host: 'localhost',
  port: '3306',
  dbName: 'mysql', 
  url: 'jdbc:mysql://localhost:3306/mysql?serverTimezone=UTC', // 建议根据你当前的库调整
  userName: 'root',
  password: ''
})

const rules = {
  name: [{ required: true, message: '请输入连接名', trigger: 'blur' }],
  jarPath: [{ required: true, message: '请输入驱动包地址', trigger: 'blur' }],
  url: [{ required: true, message: '请输入url', trigger: 'blur' }],
  userName: [{ required: true, message: '请输入用户名', trigger: 'blur' }]
}

const handleConnect = async () => {
  // 注意：validate 内部的回调函数如果涉及 await，建议写成 async
  await formRef.value.validate(async (valid) => {
    if (!valid) return
    loading.value = true
    try {
      const res = await axios.post('http://localhost:8080/api/driver/connect', form)
      if (res.data.code === 200) {
        ElMessage.success(res.data.msg)
        
        // --- 🚀 第三部分：汇报成果 ---
        // 成功后，把后端生成的 token 变成一个树节点发回给父组件
        emit('success', {
          id: res.data.data.token, // 后端返回的 connectionToken
          label: form.name,        // 树上显示的名字
          type: 'database',        // 节点类型
          children: []             // 预留给后面的“懒加载”表
        })
        
        // 关闭弹窗
        visible.value = false 
      } else {
        ElMessage.error(res.data.msg || '连接失败')
      }
    } catch (e) {
      console.error('连接异常:', e)
      ElMessage.error('连接服务器失败，请检查后端设置')
    } finally {
      loading.value = false
    }
  })
}
</script>
```

### 🧐 为什么要这么改？

- **关于 `watch`**：
    
    如果你不写 `watch`，父组件点击图标把变量设为 `true` 时，子组件的 `visible` 还是 `false`，弹窗永远跳不出来。这就是你之前说的“同步代码”。
    
- **关于 `emit('success', ...)`**：
    
    这是为了让你的 `Sidebar.vue` 知道连接成功了。父组件会执行你之前写的 `onConnectionSuccess` 函数，把你传回去的对象 `push` 到 `treeData` 里，这样左侧的树上才会立刻冒出一个新数据库节点。
    
- **关于 `form` 的处理**：
    
    你在 `handleConnect` 里直接传了 `form` 给 `axios`。因为 `form` 是用 `reactive` 定义的，它不需要加 `.value`，Axios 会直接把它转成 JSON 发给你的 Spring Boot 后端。

### 💡 现在的流程：

1. **Sidebar**：点击图标 -> `showConnectionDialog = true`。
    
2. **Dialog**：`watch` 发现变 `true` -> 弹窗显示。
    
3. **Dialog**：用户点“测试连接” -> 后端返回 `token` -> 触发 `emit('success', ...)` 并 `visible = false`。
    
4. **Sidebar**：监听到 `@success` -> 把新节点加进树里 -> 你的数据库列表里出现了新连接。
