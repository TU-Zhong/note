这就对了！在真正的商业级数据库管理软件（如 DataGrip 或 Navicat）中，纯 CSS 的 `resize` 只是一个玩具。标准的工程化做法是：**在侧边栏最右侧放置一根绝对定位的“透明触发条”，通过监听鼠标的拖拽事件，实时动态修改侧边栏的宽度。**

这种做法不仅体验如丝般顺滑，而且不受高度限制。咱们分三步把它拿下：

### 第一步：修改 HTML 模板 (绑定宽度 + 增加拖拽条)

在你的 `<div class="db-sidebar">` 标签上绑定动态宽度，并在它的内部**最下面**加上一个专门用来拖拽的 `div`。

HTML

```html
<template>
  <div class="db-sidebar" :style="{ width: sidebarWidth + 'px' }">
    
    <div 
      class="sidebar-resizer" 
      @mousedown="startResize"
    ></div>
  </div>
</template>
```

---

### 第二步：编写 Vue 拖拽逻辑 (JavaScript)

在你的 `<script setup>` 中，引入这些变量和鼠标事件方法。

核心逻辑：按下鼠标时（`mousedown`）给整个网页挂上 `mousemove` 监听器，鼠标移动的 X 坐标就是侧边栏的新宽度。

JavaScript

```js
import { ref, onBeforeUnmount } from 'vue';

// --- 侧边栏拖拽逻辑 ---
const sidebarWidth = ref(280); // 默认宽度 280px
const isResizing = ref(false); // 标记当前是否正在拖拽

// 1. 鼠标按下：开始拖拽
const startResize = (e) => {
  isResizing.value = true;
  // 拖拽时为了体验流畅，禁止页面选中文字，并把全局鼠标强制变成左右拖拽图标
  document.body.style.userSelect = 'none';
  document.body.style.cursor = 'ew-resize';

  // 将移动和抬起事件绑定到 document 上，防止鼠标移动过快脱离触发条
  document.addEventListener('mousemove', handleMouseMove);
  document.addEventListener('mouseup', stopResize);
};

// 2. 鼠标移动：实时计算宽度
const handleMouseMove = (e) => {
  if (!isResizing.value) return;
  
  // e.clientX 就是鼠标距离浏览器左边缘的横坐标
  let newWidth = e.clientX; 
  
  // 加上极值保护，防止缩得太小或拉得太大
  if (newWidth < 200) newWidth = 200; 
  if (newWidth > 600) newWidth = 600; 
  
  sidebarWidth.value = newWidth;
};

// 3. 鼠标抬起：结束拖拽
const stopResize = () => {
  isResizing.value = false;
  // 恢复页面默认样式
  document.body.style.userSelect = '';
  document.body.style.cursor = '';
  // 卸载监听器，释放内存
  document.removeEventListener('mousemove', handleMouseMove);
  document.removeEventListener('mouseup', stopResize);
};

// 作为一个规范的组件，销毁前记得清理可能遗留的全局监听器
onBeforeUnmount(() => {
  document.removeEventListener('mousemove', handleMouseMove);
  document.removeEventListener('mouseup', stopResize);
});
```

### 第三步：更新 CSS 样式

把你之前加的 `resize: horizontal` 等原生属性删掉，换成这套基于绝对定位的样式。

CSS

```
/* 修改原有的 db-sidebar */
.db-sidebar {
  position: relative; /* 关键：为了让内部的触发条能够绝对定位 */
  display: flex;
  flex-direction: column;
  height: 100%;
  border-right: 1px solid #e4e7ed;
  /* 删掉了之前的 width, min-width, max-width 和 resize，因为现在由 JS 接管了 */
}

/* 保持你原本的 tree-container 样式，让内部能上下滚动 */

  flex: 1;
  overflow-y: auto;
  overflow-x: auto;
  padding-bottom: 20px;
}

/* --- 新增：拖拽触发条的样式 --- */
.sidebar-resizer {
  position: absolute;
  top: 0;
  right: -2px;     /* 向右偏移一点，正好压在边框上 */
  width: 5px;      /* 触发区域宽度，5px 比较容易被鼠标捕捉到 */
  height: 100%;    /* 撑满整条边框的高度 */
  cursor: ew-resize; /* 左右拖拽的鼠标指针 */
  z-index: 10;
  background-color: transparent; /* 默认透明 */
  transition: background-color 0.2s; /* 颜色渐变动画 */
}

/* 鼠标悬停或正在拖拽时，触发条变成 Element Plus 的经典蓝色 */
.sidebar-resizer:hover,
.sidebar-resizer:active {
  background-color: #409eff; 
}
```

### 💡 体验升级点在哪？

1. **全边框触发**：你现在可以在边框的**任意高度**（哪怕是右上角）按住鼠标进行拉伸。
    
2. **防脱轨机制**：由于 `mousemove` 是绑定在 `document` 上的，即使你鼠标拉得飞快，瞬间甩出了边框的范围，侧边栏依然会精准地跟着你的鼠标走。
    
3. **视觉反馈**：鼠标放上去边框会微微泛蓝，非常有现代 IDE 的质感。
