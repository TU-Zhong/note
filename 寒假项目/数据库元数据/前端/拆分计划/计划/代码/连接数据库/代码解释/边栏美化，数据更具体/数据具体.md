要实现 `image_45007d.png` 中那种类似 DataGrip 或 Navicat 的专业树形结构，你需要在前端做两件事：**重构数据映射逻辑**（增加中间的“文件夹”层级）和**增强树节点模板**（显示图标、数量和辅助文本）。

作为**软件工程专业**的学生，你应该追求这种“结构化”的数据展示方式，这不仅美观，也更符合数据库管理工具的逻辑。

### 1. 修改数据转换逻辑（在 `ConnectonDialog.vue` 中）

你需要把后端返回的扁平 `columns` 和 `indexes` 数组，手动包装进三个“虚拟文件夹”节点：`columns`、`keys` 和 `indexes`。

TypeScript

```ts
// 在 handleConnect 成功拿到 treeRes.data.data 后进行转换
const tableList = treeRes.data.data;

const tableNodes = tableList.map(table => {
  // 1. 转换列 (增加主键图标判断)
  const columnNodes = table.columns.map(col => ({
    id: `col_${table.name}_${col.name}`,
    label: col.name,
    type: 'column',
    dataType: col.type,       // 存下数据类型
    isPK: col.primarykey,    // 存下是否为主键
  }));

  // 2. 转换索引
  const indexNodes = table.indexes.map(idx => ({
    id: `idx_${table.name}_${idx.indexName}`,
    label: idx.indexName,
    type: 'index',
    info: `(${idx.columnNames.join(',')})` // 显示索引包含的列
  }));

  // 3. 组装成具有文件夹结构的表节点
  return {
    id: `table_${table.name}`,
    label: table.name,
    type: 'table',
    children: [
      {
        id: `folder_col_${table.name}`,
        label: 'columns',
        type: 'folder',
        count: columnNodes.length, // 文件夹右侧显示的数量
        children: columnNodes
      },
      {
        id: `folder_key_${table.name}`,
        label: 'keys',
        type: 'folder',
        count: table.columns.filter(c => c.primarykey).length,
        children: [] // 你可以把主键列单独放这，或者简单展示
      },
      {
        id: `folder_idx_${table.name}`,
        label: 'indexes',
        type: 'folder',
        count: indexNodes.length,
        children: indexNodes
      }
    ]
  };
});
```

### 2. 增强树节点模板（在 `DbSidebar.vue` 中）

你需要使用 `el-tree` 的插槽，根据不同的 `type` 显示不同的图标和样式。

HTML

```vue
<template #default="{ node, data }">
  <span class="custom-tree-node">
    <el-icon v-if="data.type === 'database'"><Coin /></el-icon>
    <el-icon v-else-if="data.type === 'table'"><Grid /></el-icon>
    <el-icon v-else-if="data.type === 'folder'"><FolderOpened v-if="node.expanded"/><Folder v-else/></el-icon>
    
    <el-icon v-else-if="data.type === 'column'">
      <Key v-if="data.isPK" style="color: #e6a23c;" />
      <Operation v-else />
    </el-icon>

    <el-icon v-else-if="data.type === 'index'"><PriceTag /></el-icon>

    <span class="node-label">{{ data.label }}</span>

    <span v-if="data.type === 'folder'" class="node-count">{{ data.count }}</span>
    <span v-if="data.type === 'column'" class="node-type">{{ data.dataType }}</span>
    <span v-if="data.type === 'index'" class="node-type">{{ data.info }}</span>
  </span>
</template>
```

### 3. 添加样式美化（在 `DbSidebar.vue` 的 `<style>` 中）

为了达到 `image_45007d.png` 那种“深浅有致”的效果，你需要给辅助文本加点颜色。

CSS

```
.custom-tree-node {

  align-items: center;

  width: 100%;

.custom-tree-node .el-icon {

/* 数量显示样式 (小数字) */

  color: #909399;
  margin-left: 4px;

/* 类型提示样式 (灰色字体) */
.node-type {

  color: #a8abb2;
  margin-left: 8px;
}

/* 鼠标悬停变色，增强交互感 */
```

### 💡 为什么你的旧结构（image_450017.png）看起来很乱？

1. **没有分类**：所有的东西都堆在第一层，用户一眼看过去很难区分哪个是字段，哪个是约束。
    
2. **没有对比**：字段名和类型连在一起（`course_id : int identity`），视觉重心不平衡。
    
3. **缺少图标暗示**：文件夹图标（Folder）能给人一种“可收起/可展开”的心理暗示，这在管理大量表字段时能极大地减轻用户的视觉压力。
