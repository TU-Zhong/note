# 我看有的时候传[],有的时候传{}这是怎么区分的？




这其实是 Vue 提供的两种**严格程度不同**的声明方式。作为一名**软件工程专业**的学生，你可以直接把这理解为“动态类型”与“静态类型（强类型）”的区别。

---

### 1. 数组写法 `['modelValue']` —— “快速原型”

这是一种 **简写**，相当于告诉 Vue：“这几个名字的参数我会传进来，你帮我收一下就行，别管它们是什么。”

- **特点**：简单、省事。
    
- **缺点**：**没有任何校验**。你传个数字、字符串还是对象，Vue 都不会报错。
    
- **适用场景**：自己写着玩、非常简单的内部小组件，或者你百分之百确定传值不会出错时。
    

---

### 2. 对象写法 `{ modelValue: Boolean }` —— “生产级标准”

这是一种 **规范化声明**。它不仅接收数据，还充当了**“安检员”**的角色。

- **特点**：**强类型校验**。
    
- **功能**：
    
    - **类型检查**：如果你声明了 `Number` 却传了 `String`，控制台会直接报错。
        
    - **必填限制**：可以设置 `required: true`。
        
    - **默认值**：可以设置 `default: 'xxx'`。
        
- **适用场景**：**正式项目、多人协作、或者作为毕业设计项目时**。
    

---

### 3. 核心对比表

|**维度**|**数组写法 []**|**对象写法 {}**|
|---|---|---|
|**严谨性**|随意（Low）|严谨（High）|
|**报错提醒**|传错了也不吭声|传错了直接在控制台红字警告|
|**默认值支持**|不支持|支持|
|**代码可读性**|差（不知道传进来干嘛用的）|好（一眼看出数据类型和要求）|

---

### 4. 为什么我建议你用“对象写法”？

在开发你的**动态数据库元数据浏览器**时，逻辑会越来越复杂。使用对象写法对你至少有三个好处：

1. **像 Java 一样安全**：既然你熟悉 Java 后端，你应该习惯了类型定义。对象写法能让你在前端也享受到“类型安全”的心理安慰。
    
2. **自动文档化**：别人（或者一个月后的你自己）看到组件代码，立刻就知道 `id` 应该是数字，`modelValue` 应该是布尔值。
    
3. **调试方便**：如果你在父组件不小心写错了 `:id="'101'"`（传了字符串），Vue 会立刻跳出来告诉你：“兄弟，这里要的是 Number，你给我个 String 干啥？”
    

#### 代码示范（推荐写法）：

JavaScript

```
const props = defineProps({
  // 基础类型检测
  modelValue: Boolean,
  
  // 必填且带类型检测
  id: {
    type: Number,
    required: true
  },
  
  // 带默认值的字符串
  title: {
    type: String,
    default: '新建连接'
  }
})
```

---

### 总结

- **用数组 `[]`**：是在“裸奔”，追求速度。
    
- **用对象 `{}`**：是在“穿盔甲”，追求稳健。
    

既然你正在进行**毕业设计开发**，我强烈建议你养成使用**对象写法**的习惯，这会让你的前端代码看起来更有“专业软件工程”的味道。
