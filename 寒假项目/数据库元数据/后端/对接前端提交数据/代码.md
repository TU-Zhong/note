### 第一步：定义 DTO（数据传输对象）

我们需要在 Spring Boot 里建一个类，用来完美接住前端传过来的复杂 JSON。

新建一个 `DataSyncRequest.java`：

```java
import lombok.Data;
import java.util.List;
import java.util.Map;

@Data
public class DataSyncRequest {
    private String tableName;
    private String schema;
    
    // 对应前端的 inserted 数组
    private List<Map<String, Object>> inserted;
    
    // 对应前端的 updated 数组
    private List<UpdatePair> updated;
    
    // 对应前端的 deleted 数组
    private List<Map<String, Object>> deleted;

    // 内部类：专门用来接修改操作的 oldRow 和 newRow
    @Data
    public static class UpdatePair {
        private Map<String, Object> oldRow;
        private Map<String, Object> newRow;
    }
}
```

### 第三步：编写 Service（全场最硬核的底层逻辑）

这是整个项目最体现含金量的地方。我们需要根据 `Map` 里的键值对，**动态拼装**出 `INSERT`、`UPDATE` 和 `DELETE` 的原生 SQL 语句，并且必须开启**事务（Transaction）**，保证这一批修改要么全成功，要么全失败。

```java
import org.springframework.stereotype.Service;
import java.sql.Connection;
import java.sql.PreparedStatement;
import java.util.Map;
import java.util.StringJoiner;

@Service
public class TableDataService {

    public int syncData(String token, DataSyncRequest request) throws Exception {
        // 1. 获取当前用户的数据库连接 (这里调用你之前写好的连接池或动态获取连接的方法)
        // Connection conn = connectionManager.getConnection(token);
        Connection conn = null; // ⚠️ 请替换为你真实的获取连接代码
        
        int totalAffected = 0;
        String tableName = request.getTableName();
        
        // 如果有 schema，拼装成 schema.tableName
        if (request.getSchema() != null && !request.getSchema().isEmpty()) {
            tableName = request.getSchema() + "." + tableName;
        }

        try {
            // ✨ 开启手动事务控制！极其重要，防止改了一半报错导致数据不一致
            conn.setAutoCommit(false);

            // 2. 处理 DELETE (真删除)
            if (request.getDeleted() != null && !request.getDeleted().isEmpty()) {
                for (Map<String, Object> row : request.getDeleted()) {
                    totalAffected += executeDelete(conn, tableName, row);
                }
            }

            // 3. 处理 INSERT (新增)
            if (request.getInserted() != null && !request.getInserted().isEmpty()) {
                for (Map<String, Object> row : request.getInserted()) {
                    totalAffected += executeInsert(conn, tableName, row);
                }
            }

            // 4. 处理 UPDATE (修改)
            if (request.getUpdated() != null && !request.getUpdated().isEmpty()) {
                for (DataSyncRequest.UpdatePair pair : request.getUpdated()) {
                    totalAffected += executeUpdate(conn, tableName, pair.getOldRow(), pair.getNewRow());
                }
            }

            // ✨ 所有操作没报错，提交事务！
            conn.commit();
            return totalAffected;

        } catch (Exception e) {
            // 🚨 一旦中间任何一条 SQL 报错，全部回滚（撤销），保护数据库！
            if (conn != null) {
                conn.rollback();
            }
            throw new Exception("数据同步执行异常，已全部回滚：" + e.getMessage());
        } finally {
            if (conn != null) {
                conn.setAutoCommit(true); // 恢复默认行为
                // 不要在这里关闭连接，如果你的连接是复用的
            }
        }
    }

    // ================= 以下是底层动态 SQL 生成器 =================

    // 动态生成并执行 DELETE
    private int executeDelete(Connection conn, String tableName, Map<String, Object> row) throws Exception {
        StringBuilder sql = new StringBuilder("DELETE FROM ").append(tableName).append(" WHERE 1=1 ");
        
        // 拼装 WHERE 条件 (用原数据作为定位条件)
        for (String colName : row.keySet()) {
            Object value = row.get(colName);
            if (value == null) {
                sql.append(" AND ").append(colName).append(" IS NULL ");
            } else {
                sql.append(" AND ").append(colName).append(" = ? ");
            }
        }

        try (PreparedStatement ps = conn.prepareStatement(sql.toString())) {
            int paramIndex = 1;
            for (Object value : row.values()) {
                if (value != null) {
                    ps.setObject(paramIndex++, value);
                }
            }
            return ps.executeUpdate();
        }
    }

    // 动态生成并执行 INSERT
    private int executeInsert(Connection conn, String tableName, Map<String, Object> row) throws Exception {
        StringJoiner cols = new StringJoiner(", ");
        StringJoiner placeholders = new StringJoiner(", ");

        for (String colName : row.keySet()) {
            cols.add(colName);
            placeholders.add("?");
        }

        String sql = "INSERT INTO " + tableName + " (" + cols.toString() + ") VALUES (" + placeholders.toString() + ")";

        try (PreparedStatement ps = conn.prepareStatement(sql)) {
            int paramIndex = 1;
            for (Object value : row.values()) {
                ps.setObject(paramIndex++, value);
            }
            return ps.executeUpdate();
        }
    }

    // 动态生成并执行 UPDATE
    private int executeUpdate(Connection conn, String tableName, Map<String, Object> oldRow, Map<String, Object> newRow) throws Exception {
        StringBuilder sql = new StringBuilder("UPDATE ").append(tableName).append(" SET ");
        StringJoiner sets = new StringJoiner(", ");
        
        // 拼装 SET
        for (String colName : newRow.keySet()) {
            sets.add(colName + " = ?");
        }
        sql.append(sets.toString()).append(" WHERE 1=1 ");

        // 拼装 WHERE (使用 oldRow 确保并发安全，就像乐观锁一样)
        for (String colName : oldRow.keySet()) {
            Object value = oldRow.get(colName);
            if (value == null) {
                sql.append(" AND ").append(colName).append(" IS NULL ");
            } else {
                sql.append(" AND ").append(colName).append(" = ? ");
            }
        }

        try (PreparedStatement ps = conn.prepareStatement(sql.toString())) {
            int paramIndex = 1;
            // 填入 SET 的值
            for (Object value : newRow.values()) {
                ps.setObject(paramIndex++, value);
            }
            // 填入 WHERE 的值
            for (Object value : oldRow.values()) {
                if (value != null) {
                    ps.setObject(paramIndex++, value);
                }
            }
            int updatedCount = ps.executeUpdate();
            if (updatedCount == 0) {
                throw new Exception("更新失败，数据可能已被其他用户修改 (行已被覆盖)");
            }
            return updatedCount;
        }
    }
}
```

### 💡 为什么这段代码“含金量极高”？

在面试时，你可以非常骄傲地向面试官介绍这段代码的 3 个亮点：

1. **彻底摆脱 ORM 束缚**：没有 MyBatis，没有实体类，纯靠底层的 `PreparedStatement` 和 `Map` 遍历，拼装出了能兼容**世界上所有表结构**的动态 SQL。（也就是造了个极其简易的 MyBatis 核心引擎！）
    
2. **严谨的并发控制（乐观锁思想）**：在 `UPDATE` 和 `DELETE` 时，我们把 `oldRow` 里的所有字段都拼进了 `WHERE` 子句里。如果在你修改期间，有另一个 DBA 把这条数据偷偷改了，你的 `WHERE` 条件就匹配不上，导致更新失败。这就是大厂必备的防并发覆盖机制！
    
3. **原生 JDBC 事务管理**：利用 `conn.setAutoCommit(false)` 手动接管事务。如果批量更新 100 条数据时第 99 条报错了，前 98 条会自动回滚，绝不会在数据库留下脏数据。
