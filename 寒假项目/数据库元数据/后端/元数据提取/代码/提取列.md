这部分代码的核心逻辑是：**“先查户口（主键），再查明细（列信息），最后合并”。**

为了不让你在 `List` 循环里套循环（性能杀手），我会教你用 `Set` 来优化主键匹配。

以下是完整的 Day 8 代码实现。

### 第一步：创建 `ColumnVO.java` (快递盒)

我们需要在盒子（VO）里多加一个格子 `isPrimaryKey`，用来告诉前端：“这一列旁边要不要画一个小钥匙 🔑”。

在 `com.example.dbdesign.vo` 包下新建：

Java


```java
package com.example.dbdesign.vo;

import lombok.AllArgsConstructor;
import lombok.Data;
import lombok.NoArgsConstructor;

@Data
@AllArgsConstructor
@NoArgsConstructor
public class ColumnVO {
    private String name;          // 列名 (COLUMN_NAME)
    private String type;          // 类型名 (TYPE_NAME, 如 "varchar", "int")
    private Integer size;         // 长度 (COLUMN_SIZE)
    private String isNullable;    // 是否允许为空 (IS_NULLABLE, 返回 "YES" 或 "NO")
    private String remarks;       // 注释 (REMARKS)
    private Boolean isPrimaryKey; // ✅ 是否为主键 (重点新增)
}
```

### 第二步：编写 `MetadataService.java` (核心逻辑)

这里有一个编程小技巧：

因为 JDBC 是分两次查询（一次查列，一次查主键）。如果不优化，你可能需要拿着每一列去遍历主键列表，效率很低。

优化方案：先把主键查出来，扔进一个 HashSet 里。查列的时候，只要问一句 set.contains(name)，马上就知道是不是主键。

在 `MetadataService` 类中添加 `getColumns` 方法：

Java

```java
    /**
     * 获取指定表的列信息（包含主键标记）
     */
    public List<ColumnVO> getColumns(String token, String schema, String tableName) throws Exception {
        // 1. 获取连接
        Connection conn = connectionCache.getConnection(token); // 注意：这里用 Cache 类名，如果你还没改名记得用 ConnectionCathe
        if (conn == null) {
            throw new RuntimeException("连接已断开，请重试！");
        }
        DatabaseMetaData metaData = conn.getMetaData();

        // ==========================================
        // 第一步：先查主键，存入 HashSet (查找速度快)
        // ==========================================
        // getPrimaryKeys 参数：Catalog, Schema, Table
        ResultSet pkRs = metaData.getPrimaryKeys(null, schema, tableName);
        
        java.util.Set<String> pkSet = new java.util.HashSet<>();
        while (pkRs.next()) {
            String pkColumnName = pkRs.getString("COLUMN_NAME");
            pkSet.add(pkColumnName); // 把 "id" 这种主键列名扔进去
        }
        pkRs.close();

        // ==========================================
        // 第二步：查所有列，并匹配主键
        // ==========================================
        // getColumns 参数：Catalog, Schema, Table, ColumnNamePattern(null/%)
        ResultSet colRs = metaData.getColumns(null, schema, tableName, null);
        
        List<ColumnVO> columnList = new ArrayList<>();
        while (colRs.next()) {
            String name = colRs.getString("COLUMN_NAME");
            
            // ✅ 重点：直接取原生类型名 (如 "datetime2", "nvarchar")，展示给用户看最直观
            String type = colRs.getString("TYPE_NAME"); 
            
            int size = colRs.getInt("COLUMN_SIZE");
            String isNullable = colRs.getString("IS_NULLABLE"); // "YES" 或 "NO"
            String remarks = colRs.getString("REMARKS");

            // ✅ 判断是不是主键：去刚才的 Set 里查一下都在不在
            boolean isPk = pkSet.contains(name);

            ColumnVO vo = new ColumnVO(name, type, size, isNullable, remarks, isPk);
            columnList.add(vo);
        }
        colRs.close();

        return columnList;
    }
```

### 第三步：编写 `MetadataController.java` (接口)

在 Controller 里增加一个 /columns 接口。

注意：tableName 是必须传的参数。

Java

```java
    @GetMapping("/columns")
    public Result<List<ColumnVO>> getColumns(
            @RequestHeader(value = "Authorization", required = false) String token, // 记得用 @RequestHeader
            @RequestParam String schema,    // 必填，比如 "dbo"
            @RequestParam String tableName  // 必填，比如 "Student"
    ) {
        if (token == null || token.isEmpty()) {
            return Result.error("未授权，请提供连接 Token");
        }
        
        try {
            // 调用 Service
            List<ColumnVO> columns = metadataService.getColumns(token, schema, tableName);
            return Result.success("获取列结构成功", columns);
        } catch (Exception e) {
            e.printStackTrace();
            return Result.error("获取列失败：" + e.getMessage());
        }
    }
```
