在关系型数据库中，**联合索引（Composite Index / 复合索引）**非常常见。比如为了加速按姓名和年龄的组合查询，我们会建一个 `idx_name_age (name, age)` 的索引。

### 🔍 JDBC 的底层表现（坑点）

在使用 Java 的 `DatabaseMetaData.getIndexInfo()` 时，如果你查询的表有联合索引，JDBC **不会**只返回一行数据。

**它会为联合索引中的每一列，单独返回一行数据。**

> **举例**：假设有一个联合索引 `idx_combine` 包含了 `(A列, B列, C列)`。
> 
> JDBC 的 ResultSet 会循环 **3次**，分别返回：
> 
> 第1次：`index_name = idx_combine`, `column_name = A`
> 
> 第2次：`index_name = idx_combine`, `column_name = B`
> 
> 第3次：`index_name = idx_combine`, `column_name = C`

如果我们直接用原来的代码，前端会看到 3 个**同名但字段不同**的索引，这显然是错的。

---

### 🛠️ 解决方案：用 Map 进行数据聚合

我们需要把原本的一对一（一个索引一个字段）改为**一对多**。具体分两步改造：

#### 第一步：改造 `IndexVO.java`

把 `String columnName` 变成 `List<String> columnNames`。

```java

package com.example.dbdesign.vo;

import lombok.AllArgsConstructor;
import lombok.Data;
import lombok.NoArgsConstructor;
import java.util.List;

@Data
@AllArgsConstructor
@NoArgsConstructor
public class IndexVO {
    private String indexName;       // 索引名称
    
    // --- 改造点：支持联合索引，改为 List ---
    private List<String> columnNames; 
    
    private boolean nonUnique;      // 是否非唯一
    private short type;             // 索引类型
    private String ascOrDesc;       // 排序
}
```

#### 第二步：改造 `MetadataService.java` 的 `getIndexes` 方法

我们在循环遍历 `ResultSet` 时，使用一个 `Map<String, IndexVO>`（Key 为索引名）来**合并**相同名字的索引。

Java
[[rs返回index]]
```java
    // ... 在 MetadataService.java 中修改 getIndexes 方法 ...

    public List<IndexVO> getIndexes(String token, String schema, String tableName) throws Exception {
        Connection conn = connectionCathe.getConnection(token);
        if (conn == null) throw new RuntimeException("连接已断开，请重试！");
        
        DatabaseMetaData metaData = conn.getMetaData();
        ResultSet rs = metaData.getIndexInfo(null, schema, tableName, false, false);
        
        // --- 改造点：使用 LinkedHashMap 来聚合联合索引，Key 是索引名 ---
        // 使用 LinkedHashMap 保持索引查询出的原始顺序
        Map<String, IndexVO> indexMap = new java.util.LinkedHashMap<>();

        while (rs.next()) {
            String indexName = rs.getString("INDEX_NAME");
            if (indexName == null) continue; // 过滤统计信息

            String columnName = rs.getString("COLUMN_NAME");

            // 判断这个索引是否已经被记录过
            if (indexMap.containsKey(indexName)) {
                // 如果已存在（说明是联合索引的第二、第三个字段），直接把字段名塞进 List 里
                indexMap.get(indexName).getColumnNames().add(columnName);
            } else {
                // 如果不存在（新索引的第一个字段），初始化对象并放入 Map
                boolean nonUnique = rs.getBoolean("NON_UNIQUE");
                short type = rs.getShort("TYPE");
                String ascOrDesc = rs.getString("ASC_OR_DESC");

                List<String> columnList = new ArrayList<>();
                columnList.add(columnName);

                IndexVO newIndex = new IndexVO(indexName, columnList, nonUnique, type, ascOrDesc);
                indexMap.put(indexName, newIndex);
            }
        }
        rs.close();
        
        // 将 Map 的 Value 转换回 List 返回
        return new ArrayList<>(indexMap.values());
    }
```
