这正是我们需要修改的地方！
[[为什么不使用之前的代码进行复用]]
你现有的 `MetadataService` 写得非常标准（使用了 `DatabaseMetaData` 来获取结构），这很好。但是，它目前**只负责“查结构”**，缺少一个**“查数据”**的方法。

为了实现我们刚才讨论的“完美方案”（即：即使是空表也能返回列名），我们需要在你的 Service 中增加一个 `getTableData` 方法。

这个新方法**不使用** `DatabaseMetaData`（那个是查表定义的），而是使用 JDBC 执行真正的 SQL 查询 (`SELECT *`)，然后利用 `ResultSetMetaData` 来提取查询结果的列信息。

### 1. 修改后的 MetadataService.java

请将以下代码复制到你的类中。我保留了你原有的拼写（`ConnectionCathe`），并添加了新的 `getTableData` 方法。

[[table的写法]]
Java

```java
package com.example.dbdesign.Service;

import com.example.dbdesign.vo.ColumnVO;
import com.example.dbdesign.vo.IndexVO;
import com.example.dbdesign.vo.TableV0;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

import java.sql.*;
import java.util.*;

@Service
public class MetadataService {
    @Autowired
    private ConnectionCathe connectionCathe; // 保持你原有的拼写

    // ... (你原有的 getTable, getColumns, getIndexs, getTableTree 方法不用动) ...
    // ... 为了篇幅省略原有代码，请不要删除它们 ...

    /**
     * 🔥 新增方法：获取表数据 + 表结构
     * 专门用于数据预览，解决了“空表无法显示表头”的问题
     */
    public Map<String, Object> getTableData(String token, String schema, String tableName) throws Exception {
        Connection conn = connectionCathe.getConnection(token);
        if (conn == null) {
            throw new RuntimeException("连接已断开，请重试！");
        }

        // 1. 拼接 SQL (注意：为了安全，生产环境通常需要校验 tableName，但毕设可暂时忽略)
        // 加上 LIMIT 防止表数据太大把后端撑爆
        String sql;
        if (schema != null && !schema.isEmpty()) {
            sql = "SELECT * FROM " + schema + "." + tableName + " LIMIT 100";
        } else {
            sql = "SELECT * FROM " + tableName + " LIMIT 100";
        }

        Statement stmt = conn.createStatement();
        ResultSet rs = stmt.executeQuery(sql);

        // 2. 准备返回结果的容器
        Map<String, Object> result = new HashMap<>();
        List<String> columnNames = new ArrayList<>();
        List<Map<String, Object>> rows = new ArrayList<>();

        // 3. 🔥 核心步骤：先获取元数据 (Columns)
        // 即使 rs.next() 为 false (空表)，这里也能拿到列名！
        ResultSetMetaData metaData = rs.getMetaData();
        int columnCount = metaData.getColumnCount();

        for (int i = 1; i <= columnCount; i++) {
            // getColumnLabel 优于 getColumnName，因为它能处理 'select a as b' 的别名情况
            columnNames.add(metaData.getColumnLabel(i));
        }

        // 4. 获取行数据 (Rows)
        while (rs.next()) {
            // 使用 LinkedHashMap 保证 JSON 中的字段顺序和数据库列顺序一致
            Map<String, Object> row = new LinkedHashMap<>();
            for (String colName : columnNames) {
                row.put(colName, rs.getObject(colName));
            }
            rows.add(row);
        }

        // 5. 组装最终结果
        result.put("columns", columnNames); // 前端用来画表头
        result.put("rows", rows);           // 前端用来填数据

        // 关闭资源
        rs.close();
        stmt.close();
        // 注意：不要关闭 conn，因为它是缓存的长连接

        return result;
    }
}
```

### 2. 配套的 Controller 写法

既然 Service 写好了，我们需要在 Controller 里暴露这个接口。假设你有一个 `SqlController` 或者就在 `MetadataController` 里。

Java

```java
@RestController
@RequestMapping("/api/db")
public class MetadataController {

    @Autowired
    private MetadataService metadataService;

    // ... 其他接口 ...

    /**
     * 数据预览接口
     * 接收参数：tableName, schema (可选)
     */
    @PostMapping("/table/data")
    public Result getTableData(@RequestHeader("Authorization") String token, 
                               @RequestBody Map<String, String> params) {
        try {
            String schema = params.get("schema"); // 如果是 MySQL 可能是 null
            String tableName = params.get("tableName");

            if (tableName == null) {
                return Result.error("表名不能为空");
            }

            // 调用刚才写的 Service 方法
            Map<String, Object> data = metadataService.getTableData(token, schema, tableName);
            
            return Result.success(data);
        } catch (Exception e) {
            e.printStackTrace();
            return Result.error("查询失败: " + e.getMessage());
        }
    }
}
```

### 3. 这里涉及的一个关键知识点

你可能会问：**“为什么不用你原来写的 `getColumns` 方法去查列名？”**

虽然 `getColumns` (基于 `DatabaseMetaData`) 也能查到列名，但在**数据预览**这个场景下，使用 `ResultSetMetaData` (基于当前查询结果) 更好，原因有二：

1. **一致性**：`ResultSetMetaData` 是直接从 `SELECT *` 结果里拿的，它能 100% 确保拿到的列名和 `rows` 里的数据 Key 是对得上的。
    
2. **效率**：我们只发了一次 SQL 查询 (`SELECT *`) 就同时拿到了结构和数据，不需要查两次数据库。
