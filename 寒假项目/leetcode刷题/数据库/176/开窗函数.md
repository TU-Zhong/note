没错，开窗函数（Window Functions）确实有很多种用法。在面试和实际开发（比如你的图书馆管理系统报表）中，最容易混淆、也最常考的是**排序类**函数的区别。

一般来说，开窗函数主要分为三大类：**排序类**、**位移类**（前后行查找）和**聚合类**。

基本语法结构都是：

SQL

```
函数名() OVER (
    PARTITION BY 分组字段 
    ORDER BY 排序字段
)
```

下面我通过一个具体的例子（学生成绩表）来帮你彻底理清它们的区别。

假设有 4 个学生的成绩：`100, 90, 90, 80` （注意有两个 90 分）。

---

### 1. 排序类（面试必考：三大金刚）

这是最容易搞混的，它们的区别主要在于**如何处理“并列”的情况**。

|**函数名**|**逻辑描述**|**结果 (针对 100, 90, 90, 80)**|**特点总结**|
|---|---|---|---|
|**ROW_NUMBER()**|**不管并列，硬排**|1, 2, 3, 4|**唯一标识**。哪怕分数一样，排名也不重复（通常按行号）。|
|**RANK()**|**并列占位，跳跃排名**|1, 2, **2**, **4**|**奥运会规则**。有两个第二名，就没有第三名了，直接跳到第四。|
|**DENSE_RANK()**|**并列不占位，紧凑排名**|1, 2, **2**, **3**|**紧密排序**。有两个第二名，下一个人依然是第三名。|

**代码对比：**

SQL

```
SELECT score,
    ROW_NUMBER() OVER (ORDER BY score DESC) as 'rn',
    RANK()       OVER (ORDER BY score DESC) as 'rank',
    DENSE_RANK() OVER (ORDER BY score DESC) as 'dense'
FROM Scores;
```

**结果：**

Plaintext

```
+-------+----+------+-------+
| score | rn | rank | dense |
+-------+----+------+-------+
| 100   | 1  | 1    | 1     |
| 90    | 2  | 2    | 2     | -- 并列
| 90    | 3  | 2    | 2     | -- 并列
| 80    | 4  | 4    | 3     | -- RANK跳到了4，DENSE紧接为3
+-------+----+------+-------+
```

- **应用场景**：
    
    - **ROW_NUMBER**: 用于“分页”或者“删除重复数据只保留一条”。
        
    - **DENSE_RANK**: 用于“求第二高薪水”这种不允许排名中断的场景。
        

---

### 2. 位移类（前后行查找）

这类函数用于获取**当前行**的**前一行**或**后一行**的数据。做**同比增长**、**环比增长**分析时神器。

- **`LAG(col, n, default)`**: 往前看（Lag = 滞后）。取当前行之前的第 n 行数据。
    
- **`LEAD(col, n, default)`**: 往后看（Lead = 领先）。取当前行之后的第 n 行数据。
    

**场景**：你要分析某本书这个月的借阅量比上个月增长了多少。

SQL

```
SELECT 
    month,
    borrow_count,
    -- 取出上一行的借阅量，如果没有上一行（第一月），显示为0
    LAG(borrow_count, 1, 0) OVER (ORDER BY month) as last_month_count
FROM BookStats;
```

计算结果：

| month | borrow_count | last_month_count |

| :--- | :--- | :--- |

| 1月 | 100 | 0 |

| 2月 | 120 | 100 |

| 3月 | 150 | 120 |

---

### 3. 聚合类（累计计算）

普通的 `SUM`、`AVG`、`MAX` 等聚合函数，如果配合 `OVER()` 使用，就不会把多行合并成一行，而是算出**“累计值”**（Running Total）。

**场景**：计算直到当前月份的**年度累计**借阅总数。

SQL

```
SELECT 
    month, 
    borrow_count,
    -- 按照月份排序，计算从第一行到当前行的累计和
    SUM(borrow_count) OVER (ORDER BY month) as running_total
FROM BookStats;
```

计算结果：

| month | borrow_count | running_total |

| :--- | :--- | :--- |

| 1月 | 100 | 100 |

| 2月 | 120 | 220 (100+120) |

| 3月 | 150 | 370 (220+150) |

---

### 总结记忆法

1. **想排名？**
    
    - 要唯一的：`ROW_NUMBER`
        
    - 允许并列且不跳号（如求第N高）：`DENSE_RANK`
        
    - 允许并列但要跳号（奥运会）：`RANK`
        
2. **想看前面后面？**
    
    - 前面：`LAG`
        
    - 后面：`LEAD`
        
3. **想算累计？**
    
    - `SUM() OVER(...)`
