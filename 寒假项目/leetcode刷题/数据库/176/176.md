这是一个非常经典的 SQL 面试题（通常对应 LeetCode 176 题）。核心难点在于两点：
[[开窗函数]]
1. **去重**：如果有多个相同的最高薪水，第二高不应该是第一高的那个数值，而是排在后面的不同的值。
    
2. **NULL处理**：如果表中只有一条记录，或者所有人工资都一样，不存在“第二高”时，必须返回 `NULL`。
    

以下是三种不同的 SQL 解法，分别适用于不同的场景和数据库版本：

### 方法一：使用子查询 + LIMIT OFFSET (最常用/MySQL)

这是最直观的解法。我们先去重，然后倒序排列，跳过第1个，取第1个。

为了解决“如果不存在则返回 NULL”的问题，我们将整个查询包裹在另一个 SELECT 中，作为一个临时字段。

SQL

```sql
SELECT (
    SELECT DISTINCT salary
    FROM Employee
    ORDER BY salary DESC
    LIMIT 1 OFFSET 1
) AS SecondHighestSalary;
```

- **原理解析：**
    
    - `DISTINCT`：确保薪水唯一。
        
    - `ORDER BY salary DESC`：从高到低排序。
        
    - `LIMIT 1 OFFSET 1`：跳过第一行（最高薪），取下一行（第二高薪）。
        
    - **关于 NULL 的处理**：如果内部查询没有结果（例如只有一行数据），直接运行内部查询会返回“空结果集”。但把它放在外部的 `SELECT (...)` 中作为标量子查询时，如果内部没结果，它会自动返回 `NULL`。
        

---

### 方法二：使用 MAX() 函数 (通用性最强)

这个逻辑是通过数学大小关系来找：**第二高的薪水，就是所有“小于最大值”的薪水中的最大值。**

SQL

```
SELECT MAX(salary) AS SecondHighestSalary
FROM Employee
WHERE salary < (
    SELECT MAX(salary)
    FROM Employee
);
```

- **原理解析：**
    
    - 内部子查询 `SELECT MAX(salary) FROM Employee` 找到了全表的最高薪水。
        
    - 外部查询筛选出所有比最高薪水小的薪水。
        
    - 再对剩下的薪水取 `MAX()`，自然就是第二高的。
        
    - **关于 NULL 的处理**：如果表中只有一条数据，`WHERE` 条件筛选后为空集。SQL 的聚合函数 `MAX()` 在面对空集时，默认返回 `NULL`，完美符合题目要求。
        

---

### 方法三：使用窗口函数 DENSE_RANK() (适用于现代数据库/复杂排名)

如果你的数据库支持窗口函数（MySQL 8.0+, PostgreSQL, SQL Server, Oracle），这是处理“第 N 高”问题最标准的方法。使用 `DENSE_RANK()` 可以处理并列排名的情况（例如：100, 100, 90，90就是第二名）。

SQL

```
SELECT MAX(salary) AS SecondHighestSalary
FROM (
    SELECT salary, DENSE_RANK() OVER (ORDER BY salary DESC) AS rnk
    FROM Employee
) AS temp
WHERE rnk = 2;
```

- **原理解析：**
    
    - `DENSE_RANK()`：生成排名。如果最高薪水有两个 200，它们都会标记为 rank 1，下一个不同的薪水会被标记为 rank 2。
        
    - `WHERE rnk = 2`：筛选出排名第二的行。
        
    - **关于 NULL 的处理**：同样，如果直接 `SELECT salary ... WHERE rnk=2`，如果没有第二名，会返回空集而不是 NULL。所以我们在最外层加了一个 `SELECT MAX(...)`（或者 `SELECT DISTINCT` 也可以，但 MAX 更常用），作用是当子查询结果为空时，强制聚合函数返回 `NULL`。
        

### 总结建议

- **面试/刷题时**：推荐使用 **方法二** (`MAX < MAX`)，因为它不需要考虑数据库方言（LIMIT 是 MySQL/PG 的，TOP 是 SQL Server 的），且逻辑非常清晰。
    
- **实际开发中**：如果需要求第 N 高（比如第 50 高），推荐使用 **方法三** (`窗口函数`)，因为方法二和方法一在 N 很大时写起来会很麻烦或性能较差。
