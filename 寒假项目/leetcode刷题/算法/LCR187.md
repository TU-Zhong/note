这是一个典型的 **“约瑟夫环” (Josephus Problem)** 问题。

针对 `num` 高达 $10^5$ 的数据规模，我为你提供三种解法，从**最直观的模拟**到**最高效的数学推导**，循序渐进。

### 方法一：链表/数组模拟法 (直观但性能差)

最符合人类思维的方法是：真的创建一个列表代表这些人，然后不断循环计数、删除，直到只剩下一个。

- **思路**：使用一个 `ArrayList` 存储 `0` 到 `num-1`。每次计算出要删除的索引 `index`，将其移除。
    
- **缺点**：`ArrayList` 的 `remove` 操作是 $O(N)$ 的（因为要移动后面的元素），总时间复杂度是 $O(N^2)$。对于 $10^5$ 的数据，这通常会**超时 (Time Limit Exceeded)**。
    

Java

```
import java.util.ArrayList;

class Solution {
    public int iceBreakingGame(int num, int target) {
        ArrayList<Integer> list = new ArrayList<>();
        // 1. 初始化列表
        for (int i = 0; i < num; i++) {
            list.add(i);
        }

        int index = 0;
        // 2. 循环直到只剩一人
        while (list.size() > 1) {
            // 计算下一个要删除的位置
            // 这里的公式是：(当前位置 + 步长 - 1) % 剩余人数
            // -1 是因为计数是 1-based，而索引是 0-based
            index = (index + target - 1) % list.size();
            
            // 3. 物理删除
            list.remove(index);
        }

        return list.get(0);
    }
}
```

### 方法二：递归公式法 (数学解法)

为了解决模拟法太慢的问题，我们需要寻找规律。

假设 $f(n, m)$ 表示有 $n$ 个人，每次报数 $m$ (target) 时，最后幸存者的索引。

- **核心逻辑**：
    
    1. 如果只有 1 个人 ($n=1$)，幸存者索引肯定是 0。
        
    2. 如果有 $n$ 个人，删除第 $m$ 个位置的人后，剩下了 $n-1$ 个人。
        
    3. 此时问题变成了一个规模为 $n-1$ 的子问题。
        
    4. **关键推导**：幸存者在“旧数组”中的位置 = `(幸存者在“新数组”中的位置 + m) % n`。
        
- 公式：
    
    $$f(n, m) = (f(n-1, m) + m) \% n$$
    

Java

```
class Solution {
    public int iceBreakingGame(int num, int target) {
        // 递归终止条件：只剩一人时，索引为 0
        if (num == 1) {
            return 0;
        }
        // 递归公式：(上一轮的结果 + target) % 本轮人数
        int x = iceBreakingGame(num - 1, target);
        return (x + target) % num;
    }
}
```

- **复杂度**：时间 $O(N)$，空间 $O(N)$（递归栈深度）。虽然快了很多，但如果 `num` 极大，可能会导致**栈溢出 (Stack Overflow)**。

### 方法三：迭代公式法 (最优解)

这是实际面试和工程中最推荐的写法。它将方法二的递归改写为循环 (迭代)。

这样既保留了 $O(N)$ 的时间效率，又将空间复杂度降到了 $O(1)$。

- 思路：
    
    我们不需要从 $n$ 倒推到 1，而是可以从 1 反推到 $n$。
    
    - $i=1$ 时，幸存者位置为 0。
        
    - $i=2$ 时，位置为 $(0 + target) \% 2$。
        
    - $i=3$ 时，位置为 $(上一步位置 + target) \% 3$。
        
    - ...
        
    - 直到 $i=num$。
        

Java

```
class Solution {
    public int iceBreakingGame(int num, int target) {
        int ans = 0; // i = 1 时的解（最后剩下的人在只有1人时的索引是0）
        
        // 从 2 个人开始推导，直到 num 个人
        for (int i = 2; i <= num; i++) {
            // 公式：(当前索引 + target) % 当前人数
            ans = (ans + target) % i;
        }
        
        return ans;
    }
}
```

### 总结与建议

|**方法**|**时间复杂度**|**空间复杂度**|**评价**|
|---|---|---|---|
|**模拟法**|$O(N^2)$|$O(N)$|思路简单，但数据量 > 5000 就会很慢，**不推荐**。|
|**递归法**|$O(N)$|$O(N)$|代码简洁，但有栈溢出风险。|
|**迭代法**|$O(N)$|$O(1)$|**最优解**，性能最好，面试首选。|

**针对你的提示 ($num \le 10^5$)，请务必使用方法三（迭代法）。**
