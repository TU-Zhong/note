没错，这正是我们 Phase 2 的核心蓝图！

你现在手里拿着的是**设计图纸**，我们接下来的任务就是把这张图纸变成**代码**。这一步非常关键，因为只要**偏移量 (Offset)** 算错一个字节，或者 Pipeline 顺序搞反，服务器就会直接断开或者报一堆乱码。

我们将分三步走：**先写编码器（发），再写解码器（收），最后组装 Pipeline（服务器）。**

### 第一步：编写编码器 (Encoder)

作用：把你的 Java 对象 AegisPacket 变成二进制字节流 ByteBuf，发给网络。

位置：com.example.demo.protocol 包下。

**新建文件 `AegisProtocolEncoder.java`**：

Java

```java
package com.example.demo.protocol;

import io.netty.buffer.ByteBuf;
import io.netty.channel.ChannelHandlerContext;
import io.netty.handler.codec.MessageToByteEncoder;
import java.nio.charset.StandardCharsets;

public class AegisProtocolEncoder extends MessageToByteEncoder<AegisPacket> {

    @Override
    protected void encode(ChannelHandlerContext ctx, AegisPacket msg, ByteBuf out) throws Exception {
        // 1. 写入魔数 (2字节)
        out.writeShort(msg.getMagic());

        // 2. 写入版本 (1字节)
        out.writeByte(msg.getVersion());

        // 3. 写入类型 (1字节)
        out.writeByte(msg.getType());

        // 4. 写入指令 (2字节)
        out.writeShort(msg.getCmd());

        // 5. 计算 Body 长度并写入 (4字节)
        byte[] bodyBytes = msg.getBody().getBytes(StandardCharsets.UTF_8);
        out.writeInt(bodyBytes.length); // 这里写入的是长度

        // 6. 写入实际数据 (N字节)
        out.writeBytes(bodyBytes);
    }
}
```

### 第二步：编写解码器 (Decoder) —— 最难的一步

作用：把网络传过来的一堆字节，按照协议还原成 AegisPacket 对象。

注意：我们这里只负责“翻译”，“断句”（解决粘包）的工作交给 Netty 自带的 LengthFieldBasedFrameDecoder（在第三步配置）。所以进入这个类的 ByteBuf 肯定是一个完整的包。

**新建文件 `AegisProtocolDecoder.java`**：

Java

```java
package com.example.demo.protocol;

import io.netty.buffer.ByteBuf;
import io.netty.channel.ChannelHandlerContext;
import io.netty.handler.codec.ByteToMessageDecoder;
import java.nio.charset.StandardCharsets;
import java.util.List;

public class AegisProtocolDecoder extends ByteToMessageDecoder {

    @Override
    protected void decode(ChannelHandlerContext ctx, ByteBuf in, List<Object> out) throws Exception {
        // 安全防御：虽然前面有 LengthFieldBasedFrameDecoder，但防止意外，还是要判断长度
        if (in.readableBytes() < 10) { // 头部总长度 = 2+1+1+2+4 = 10字节
            return;
        }

        // 1. 检查魔数 (Magic)
        short magic = in.readShort();
        if (magic != 0xCAFE) {
            // 🚨 非法连接！直接关闭，不解释
            System.out.println("检测到非法魔数: " + Integer.toHexString(magic));
            ctx.close();
            return;
        }

        // 2. 读取头部其他信息
        byte version = in.readByte();
        byte type = in.readByte();
        short cmd = in.readShort();
        int length = in.readInt();

        // 3. 读取 Body
        // 安全检查：如果剩下的字节不够 length 长度，说明包还没收全（通常 FrameDecoder 已经保证了，这里是双保险）
        if (in.readableBytes() < length) {
            in.resetReaderIndex(); // 回滚读取进度，等待下一次
            return;
        }

        byte[] bodyBytes = new byte[length];
        in.readBytes(bodyBytes);
        String body = new String(bodyBytes, StandardCharsets.UTF_8);

        // 4. 封装成对象，传递给下一个 Handler
        AegisPacket packet = AegisPacket.builder()
                .magic(magic)
                .version(version)
                .type(type)
                .cmd(cmd)
                .length(length)
                .body(body)
                .build();

        out.add(packet);
    }
}
```

### 第三步：组装服务器 (NettyServer) —— 核心配置

现在回到你的启动类 DemoApplication.java（或者你单独写的 NettyServer 类）。

我们要修改 initChannel 方法，把刚才写的编解码器加进去，并配置核心参数。

**请仔细核对每一个数字，尤其是 Offset！**

Java

```
// 在 initChannel 方法里修改：

protected void initChannel(SocketChannel ch) {
    // Pipeline 就是处理流水线，顺序非常重要！
    
    // ---------------- 入站 (Inbound) 处理 ----------------
    
    // 1. [关键] 解决粘包/拆包
    // maxFrameLength = 65536 (最大包长)
    // lengthFieldOffset = 6 (长度字段在第几个字节？ Magic(2)+Ver(1)+Type(1)+Cmd(2)=6)
    // lengthFieldLength = 4 (长度字段本身占几个字节？ int=4)
    // lengthAdjustment = 0 (长度字段的值是 Body 长度，不需要调整)
    // initialBytesToStrip = 0 (是否把头部切掉？不切，我们需要在 Decoder 里校验 Magic)
    ch.pipeline().addLast(new LengthFieldBasedFrameDecoder(65536, 6, 4, 0, 0));

    // 2. 自定义解码器 (把二进制变成 AegisPacket 对象)
    ch.pipeline().addLast(new AegisProtocolDecoder());

    
    // ---------------- 出站 (Outbound) 处理 ----------------
    
    // 3. 自定义编码器 (把 AegisPacket 对象变成二进制，顺序无所谓，只要在 write 时经过它就行)
    ch.pipeline().addLast(new AegisProtocolEncoder());


    // ---------------- 业务逻辑 ----------------
    
    // 4. 最终的业务处理器 (处理 AegisPacket)
    ch.pipeline().addLast(new SimpleChannelInboundHandler<AegisPacket>() {
        @Override
        protected void channelRead0(ChannelHandlerContext ctx, AegisPacket packet) {
            // 打印看看！
            System.out.println("收到客户端消息: " + packet.toString());
            
            // 简单回复一个 PONG
            if (packet.getType() == 1) { // 假设 1 是请求
                 AegisPacket response = AegisPacket.builder()
                        .magic((short) 0xCAFE)
                        .version((byte) 1)
                        .type((byte) 2) // 2 是响应
                        .cmd(packet.getCmd())
                        .length(0) // 会被 Encoder 自动覆盖
                        .body("Server Received: " + packet.getBody())
                        .build();
                 ctx.writeAndFlush(response);
            }
        }
    });
}
```
