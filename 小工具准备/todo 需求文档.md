### 最终需求与技术方案草案 (V3)

#### 1. 核心思路与模型设计 (Model)

- **独立的时间概念**：任务实体（Task）将包含两个独立的时间字段：
    
    - `scheduledTime`（安排时间）：用于视图分类（比如安排在今天，或者安排在本周）。
        
    - `deadline`（截止时间）：一个独立的可选属性，表示该任务最迟需要完成的绝对日期。
        
- **状态与元数据**：包含 `isCompleted` (布尔值) 和 `notes` (字符串，用于备忘记录)。
    

#### 2. 数据存储方案 (Storage)

- **核心方案**：采用纯文本 **JSON 格式** 持久化到本地文件（例如 `todo-data.json`）。
    
- **技术选型**：使用 `Gson` 或 `Jackson` 库进行 Java 对象与 JSON 字符串的相互转换（序列化/反序列化）。
    
- **为什么选这个方案**：完全符合你“轻量级纯文本”和“单机离线”的约束。JSON 格式结构化强、易于扩展，且不需要用户安装配置任何数据库服务，应用随时可以打包成一个 Jar 包或可执行文件随处运行。
    

#### 3. 界面表现与交互 (View)

- **核心方案**：采用 **JavaFX** 作为 GUI 框架，结合 FXML 来分离界面布局与业务逻辑。
    
- **布局构思**：
    
    - **左侧导航栏**：用于切换“每日视图”和“每周视图”，并包含一个简单的日期/周次选择器。
        
    - **中部主区域**：任务列表，每个任务前带 Checkbox。
        
    - **右侧/弹窗区域**：当点击某个具体任务时，展示其详细信息（修改日期、设定 Deadline、编辑备注）。
        
- **为什么选这个方案**：JavaFX 是 Java 生态内最现代化、功能最完整的原生桌面框架，能够不依赖任何 Web 技术（HTML/JS）构建出流畅且美观的单机程序。
    

#### 4. 工程化构建

- **构建工具**：建议使用 Maven 进行依赖管理（主要用于引入 JavaFX SDK 和 JSON 解析库）。
