

以下是一个复杂的正则表达式示例：**解析并验证带有毫秒和时区的 ISO 8601 格式日期时间字符串**，同时提取年、月、日、时间等信息。

---

### **正则表达式**
```java
String iso8601Regex = 
    "^" +
    "(\\d{4})" +                 // 年 (组1)
    "[-/]" +                     // 分隔符（- 或 /）
    "(0[1-9]|1[0-2])" +          // 月 (组2)
    "[-/]" +
    "(0[1-9]|[12]\\d|3[01])" +   // 日 (组3)
    "[T ]" +                     // 分隔符（T 或空格）
    "([01]\\d|2[0-3])" +         // 小时 (组4)
    "[:.-]" +
    "([0-5]\\d)" +               // 分钟 (组5)
    "[:.-]" +
    "([0-5]\\d)" +               // 秒 (组6)
    "(?:\\.(\\d{1,3}))?" +       // 毫秒（可选，组7）
    "(Z|[+-](?:0\\d|1[0-3]):[0-5]\\d)?" + // 时区（可选，组8）
    "$";
```

---

### **代码示例**
```java
import java.util.regex.*;

public class ComplexRegexExample {
    public static void main(String[] args) {
        String iso8601Regex = "^(\\d{4})[-/](0[1-9]|1[0-2])[-/](0[1-9]|[12]\\d|3[01])[T ]([01]\\d|2[0-3])[:.-]([0-5]\\d)[:.-]([0-5]\\d)(?:\\.(\\d{1,3}))?(Z|[+-](?:0\\d|1[0-3]):[0-5]\\d)?$";
        String[] testDates = {
            "2023-10-05T14:30:45.123+08:00",
            "2023/12/31 23:59.59.999Z",
            "2023-02-30 25:00:00", // 非法日期
            "2023-13-01 12:00:00"  // 非法月份
        };

        Pattern pattern = Pattern.compile(iso8601Regex);
        for (String date : testDates) {
            Matcher matcher = pattern.matcher(date);
            if (matcher.matches()) {
                System.out.println("合法日期: " + date);
                System.out.println("年: " + matcher.group(1));
                System.out.println("月: " + matcher.group(2));
                System.out.println("日: " + matcher.group(3));
                System.out.println("时: " + matcher.group(4));
                System.out.println("分: " + matcher.group(5));
                System.out.println("秒: " + matcher.group(6));
                System.out.println("毫秒: " + (matcher.group(7) != null ? matcher.group(7) : "无"));
                System.out.println("时区: " + (matcher.group(8) != null ? matcher.group(8) : "无"));
                System.out.println("-------------------");
            } else {
                System.out.println("非法日期: " + date);
                System.out.println("-------------------");
            }
        }
    }
}
```

### **正则表达式解析**
1. **日期部分**：
   - `(\\d{4})`：年（4位数字）。
   - `(0[1-9]|1[0-2])`：月（01-12）。
   - `(0[1-9]|[12]\\d|3[01])`：日（01-31，未严格验证每月天数）。

2. **时间部分**：
   - `([01]\\d|2[0-3])`：小时（00-23）。
   - `([0-5]\\d)`：分钟和秒（00-59）。
   - `(?:\\.(\\d{1,3}))?`：可选的毫秒（1~3位数字）。

3. **时区部分**：
   - `Z`：UTC 时间。
   - `[+-](0\\d|1[0-3]):[0-5]\\d`：时区偏移（如 +08:00 或 -05:30）。

---

### **局限性**
1. **未严格验证日期有效性**（如 2月30日、4月31日等）。
2. **毫秒位数限制**：仅允许1~3位（实际ISO标准允许1~6位）。
3. **时区偏移限制**：未覆盖全部可能值（如 `+14:00`）。

---

### **改进方向**
1. 结合 `java.time` 库进行日期合法性验证。
2. 允许更灵活的毫秒位数（例如 `\\.\\d{1,6}`）。
3. 支持更多时区格式（如 `±HHmm`）。

---

这个例子展示了正则表达式如何用于：
- **复杂格式验证**
- **结构化数据提取**
- **可选字段处理**
- **多分隔符兼容**

但需注意：正则表达式适合格式验证和初步解析，实际业务中应结合其他方法（如日期库）确保数据完全正确。
