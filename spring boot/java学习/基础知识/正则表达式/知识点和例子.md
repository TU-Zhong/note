### **1. 核心类**
Java 的正则表达式处理主要依赖两个类：

- **`Pattern`**：<mark style="background: #FFF3A3A6;">表示一个编译好的正则表达式</mark>（通过 `Pattern.compile(regex)` 创建）。
- **`Matcher`**：<mark style="background: #FFF3A3A6;">基于 `Pattern` 对某个字符串执行匹配操作的引擎。</mark>，对输入的字符串进行匹配等操作

`.matcher()` 方法的作用**

- 调用方式：`Pattern` 实例调用 `.matcher(input)`。
- **输入参数**：一个待匹配的字符串（如 `"apple@example.com"`）。
- **返回值**：一个 `Matcher` 对象，用于后续的匹配操作。
[[Matcher的常用方法]]
```java
Pattern pattern = Pattern.compile("a*b"); // 预编译正则表达式
Matcher matcher = pattern.matcher("aaaaab"); 
boolean isMatch = matcher.matches(); // true
```

### **2. 常用语法**
#### **2.1 元字符**
| 元字符 | 描述                   | 示例                      |
|--------|------------------------|---------------------------|
| `.`    | 匹配任意字符（除换行） | `"a.c"` → "abc", "a1c"    |
| `\d`   | 数字（等价于 `[0-9]`） | `"\\d+"` → "123"          |
| `\D`   | 非数字                 | `"\\D+"` → "abc"          |
| `\s`   | 空白字符（空格、制表等）| `"a\\sb"` → "a b"         |
| `\w`   | 单词字符（字母数字下划线）| `"\\w+"` → "user_123"   |
| `^`    | 字符串开头             | `"^Java"` → "Java is..."  |
| `$`    | 字符串结尾             | `"end$"` → "the end"      |

---

#### **2.2 量词**
| 量词   | 描述               | 示例                       |
|--------|--------------------|----------------------------|
| `*`    | 0次或多次          | `"a*"` → "", "a", "aa"     |
| `+`    | 1次或多次          | `"a+"` → "a", "aa"         |
| `?`    | 0次或1次           | `"a?"` → "", "a"           |
| `{n}`  | 恰好n次            | `"a{3}"` → "aaa"           |
| `{n,m}`| 至少n次，最多m次   | `"a{2,4}"` → "aa", "aaaa"  |

---

#### **2.3 字符类**
| 语法         | 描述                       | 示例                     |
|--------------|----------------------------|--------------------------|
| `[abc]`      | 匹配a、b或c                | `"[aeiou]"` → "a"        |
| `[^abc]`     | 排除a、b、c                | `"[^0-9]"` → "a", "x"    |
| `[a-zA-Z]`   | 范围匹配                   | `"[A-Za-z]+"` → "Hello"  |

---

#### **2.4 分组与捕获**
- **`( )`**：定义分组并捕获。
- **`\1`**：反向引用第一个分组（Java 中使用 `\\1`）。

```java
String regex = "(\\d{3})-(\\d{4})"; // 分组捕获区号和后4位
Matcher m = Pattern.compile(regex).matcher("123-4567");
if (m.find()) {
    System.out.println(m.group(1)); // 123
    System.out.println(m.group(2)); // 4567
}
```

### **3. 常见操作**
#### **3.1 验证字符串格式**
```java
// 验证邮箱
String emailRegex = "^[a-zA-Z0-9_+&*-]+(?:\\.[a-zA-Z0-9_+&*-]+)*@(?:[a-zA-Z0-9-]+\\.)+[a-zA-Z]{2,7}$";
boolean isValidEmail = "user@example.com".matches(emailRegex); // true

// 验证手机号（简单示例）
String phoneRegex = "^1[3-9]\\d{9}$";
boolean isValidPhone = "13812345678".matches(phoneRegex); // true
```

---

#### **3.2 提取匹配内容**
```java
String text = "Prices are $100 and $200.";
Pattern pattern = Pattern.compile("\\$\\d+");
Matcher matcher = pattern.matcher(text);
while (matcher.find()) {
    System.out.println(matcher.group()); // $100, $200
}
```

---

#### **3.3 替换字符串**
```java
// 替换所有数字为"#"
String replaced = "123abc456".replaceAll("\\d", "#"); // "###abc###"

// 使用反向引用
String formatted = "2023-12-31".replaceAll("(\\d{4})-(\\d{2})-(\\d{2})", "$3/$2/$1"); // "31/12/2023"
```

---

#### **3.4 分割字符串**
```java
String[] parts = "one,two,three".split("\\s*,\\s*"); // ["one", "two", "three"]
```

### **4. 高级技巧**
#### **4.1 贪婪与非贪婪**
- **贪婪匹配**：默认量词（如 `.*`）尽可能匹配长内容。
- **非贪婪匹配**：在量词后加 `?`（如 `.*?`）尽可能匹配短内容。

```java
// 贪婪匹配
String greedy = "a<b>test</b>".replaceAll("<.*>", ""); // 结果: "a"

// 非贪婪匹配
String nonGreedy = "a<b>test</b>".replaceAll("<.*?>", ""); // 结果: "atest"
```

---

#### **4.2 预编译正则表达式**
提升性能（特别在多次使用时）：
```java
private static final Pattern DATE_PATTERN = Pattern.compile("\\d{4}-\\d{2}-\\d{2}");

public boolean isValidDate(String input) {
    return DATE_PATTERN.matcher(input).matches();
}
```

---

### **5. 注意事项**
1. **转义字符**：Java 字符串中需用双反斜杠（如 `\\d` 表示 `\d`）。
2. **性能**：避免在循环中重复编译正则表达式。
3. **边界匹配**：用 `^` 和 `$` 确保完整字符串匹配。

---

### **完整示例：提取URL中的域名**
```java
String url = "https://www.example.com/path?query=123";
Pattern pattern = Pattern.compile("^(https?://)?([^/]+)");
Matcher matcher = pattern.matcher(url);
if (matcher.find()) {
    System.out.println(matcher.group(2)); // www.example.com
}
```
