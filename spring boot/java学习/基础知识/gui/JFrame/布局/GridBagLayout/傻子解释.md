

**GridBagLayout 就像拼拼图** 🧩

想象你有一个空白的画板（比如一张纸），你要在上面贴各种形状的贴纸（比如按钮、文本框）。GridBagLayout 就是帮你决定每个贴纸怎么摆的规则，但比普通网格更灵活！

---

### **核心规则（用贴纸举例）**
1. **画格子线** 📏  
   虽然看不见，但整个画板被分成很多小格子（像田字格本）。

2. **贴纸可以占多个格子** 🔲  
   - 比如一个大贴纸可以占 2 行 × 3 列的小格子。
   - 小贴纸只占 1 个格子。

3. **贴哪里？** 🎯  
   - 告诉系统：左上角从第 2 行、第 1 列的格子开始贴。
   - 如果窗口变大，贴纸可以按比例拉伸（比如优先横向拉宽）。

---

### **实际代码里的关键参数（贴纸说明书）**
```java
GridBagConstraints gbc = new GridBagConstraints();

// 1. 贴哪儿？
gbc.gridx = 1;  // 横坐标（第 2 列）
gbc.gridy = 0;  // 纵坐标（第 1 行）

// 2. 占多大？
gbc.gridwidth = 2;  // 占 2 列宽
gbc.gridheight = 1; // 占 1 行高

// 3. 如果窗口变大，怎么拉伸？
gbc.weightx = 1.0; // 横向拉伸优先级（数字越大，分到的空间越多）
gbc.weighty = 0.5; // 纵向拉伸优先级

// 4. 贴纸怎么对齐？
gbc.anchor = GridBagConstraints.WEST; // 靠左对齐

// 5. 填充方式
gbc.fill = GridBagConstraints.HORIZONTAL; // 横向填满格子
```

---

### **举个实际例子**
假设你要做这个界面：
```
[用户名：____________][确认按钮]
[密码：  ____________]
```
用 GridBagLayout 可以这样设置：
1. **用户名输入框**：横跨两列，放在第1行。
2. **确认按钮**：放在第1行第3列。
3. **密码输入框**：横跨两列，放在第2行。

---

### **总结**
- **优点**：超级灵活！想贴哪就贴哪，想占多大就占多大。
- **缺点**：规则多，写起来麻烦（像拼复杂拼图）。
- **适用场景**：需要精细控制布局时（比如登录表单、复杂设置界面）。
