如果接口有 **多个方法**，就不能直接使用 Lambda 表达式了！因为 Lambda 表达式的核心是 **只针对只有一个抽象方法的接口**（即 **函数式接口**）。

---

### 1. **接口有多个方法时的限制**
如果接口有多个方法，比如：
```java
interface StringProcessor {
    String process(String input); // 抽象方法1
    void log(String message);     // 抽象方法2
}
```
这个接口有两个抽象方法，因此 **不是函数式接口**，无法用 Lambda 表达式来实现。

---

### 2. **解决方案**
如果需要使用 Lambda 表达式，可以有以下几种方式：

#### 方式 1：**确保接口是函数式接口**
只保留一个抽象方法，其他方法可以是默认方法或静态方法：
```java
@FunctionalInterface
interface StringProcessor {
    String process(String input); // 只有这一个抽象方法

    default void log(String message) { // 默认方法
        System.out.println("日志: " + message);
    }
}
```
这样依然可以用 Lambda 表达式：
```java
StringProcessor toUpper = s -> s.toUpperCase();
System.out.println(toUpper.process("hello")); // 输出 "HELLO"
toUpper.log("任务完成"); // 调用默认方法
```

#### 方式 2：**使用匿名内部类**
如果接口有多个方法，只能用传统的匿名内部类来实现：
```java
StringProcessor processor = new StringProcessor() {
    @Override
    public String process(String input) {
        return input.toUpperCase();
    }

    @Override
    public void log(String message) {
        System.out.println("日志: " + message);
    }
};

System.out.println(processor.process("hello")); // 输出 "HELLO"
processor.log("任务完成"); // 输出 "日志: 任务完成"
```

#### 方式 3：**拆分成多个接口**
将不同功能拆分成多个接口，每个接口只有一个抽象方法：
```java
@FunctionalInterface
interface Processor {
    String process(String input);
}

interface Logger {
    void log(String message);
}
```
然后分别用 Lambda 表达式实现：
```java
Processor toUpper = s -> s.toUpperCase();
Logger logger = message -> System.out.println("日志: " + message);

System.out.println(toUpper.process("hello")); // 输出 "HELLO"
logger.log("任务完成"); // 输出 "日志: 任务完成"
```

---

### 3. **函数式接口的定义**
Java 8 引入了 **`@FunctionalInterface`** 注解，用于明确标记一个接口是函数式接口。如果接口不符合函数式接口的定义（即不止一个抽象方法），编译器会报错：
```java
@FunctionalInterface
interface InvalidProcessor {
    String process(String input); // 抽象方法1
    void log(String message);     // 抽象方法2
}
```
**编译错误**：  
`InvalidProcessor` 不是函数式接口，因为它有多个抽象方法。

---

### 4. **默认方法和静态方法**
函数式接口可以有默认方法或静态方法，因为它们不是抽象方法：
```java
@FunctionalInterface
interface StringProcessor {
    String process(String input); // 抽象方法

    default void log(String message) { // 默认方法
        System.out.println("日志: " + message);
    }

    static void printVersion() { // 静态方法
        System.out.println("版本: 1.0");
    }
}
```

---

### 5. **总结**
- **Lambda 表达式只能用于函数式接口**（只有一个抽象方法的接口）。
- 如果接口有多个方法，可以用以下方式解决：  
  1. 将接口改为函数式接口（只保留一个抽象方法）。  
  2. 使用匿名内部类。  
  3. 拆分成多个接口。
- 默认方法和静态方法不会影响函数式接口的定义。

---

### 类比生活中的例子：
假设你有一个任务：“把名字转为大写”和“记录日志”。如果两个任务都写在一张纸上（接口），Lambda 表达式只能处理其中一个任务（函数式接口）。如果想同时处理两个任务，要么分开写（拆分成多个接口），要么写一张详细的清单（匿名内部类）。

通过这种方式，可以更好地理解 Lambda 表达式的使用限制和解决方案！
