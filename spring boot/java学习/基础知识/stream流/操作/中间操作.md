好的，我们来详细深入地了解一下 Java Stream 中这些核心的中间操作。中间操作的核心特点是“惰性求值”，它们不会立即执行，而是构建一个操作流水线，直到终端操作被调用时才一次性处理。

---

### 1. `filter(Predicate<T> predicate)` - 过滤

`filter` 操作可以说是最常用的中间操作之一。它像一个筛子，只允许满足特定条件的元素通过。

- **功能**：遍历流中的每个元素，并应用你提供的判断条件。如果条件返回 `true`，则该元素被保留；如果返回 `false`，则该元素被丢弃。
- **参数 `Predicate<T>`**：
    - 这是一个**函数式接口**，代表一个“断言”或“谓词”。
    - 它只有一个抽象方法：`boolean test(T t)`。
    - `T` 是流中元素的类型。
    - 你需要提供一个 Lambda 表达式来实现这个 `test` 方法，定义你的过滤逻辑。该 Lambda 表达式接收一个元素 `t`，返回一个布尔值。

#### 示例：筛选出长度大于 3 的字符串

Java

```
import java.util.List;
import java.util.stream.Collectors;

List<String> words = List.of("Apple", "Banana", "Sky", "Cat", "Dog", "Elephant");

List<String> longWords = words.stream()
    .filter(s -> s.length() > 3) // Lambda表达式 s -> s.length() > 3 实现了 Predicate<String>
    .collect(Collectors.toList());

System.out.println(longWords); // 输出: [Apple, Banana, Elephant]
```

**工作流程**：

1. "Apple" (长度5) -> `5 > 3` -> `true` -> 保留
2. "Banana" (长度6) -> `6 > 3` -> `true` -> 保留
3. "Sky" (长度3) -> `3 > 3` -> `false` -> 丢弃
4. "Cat" (长度3) -> `3 > 3` -> `false` -> 丢弃
5. "Dog" (长度3) -> `3 > 3` -> `false` -> 丢弃
6. "Elephant" (长度8) -> `8 > 3` -> `true` -> 保留

### 2. `map(Function<T, R> mapper)` - 映射/转换

`map` 操作用于将流中的每个元素进行一对一的转换，将原始元素映射成一个新的元素。转换前后的元素类型可以不同。

- **功能**：将流中的每个元素应用你提供的转换函数，并将该函数的返回值作为新的流元素。
- **参数 `Function<T, R>`**：
    - 这是一个**函数式接口**，代表一个“函数”。
    - 它只有一个抽象方法：`R apply(T t)`。
    - `T` 是输入元素的类型（原始流中元素的类型）。
    - `R` 是输出元素的类型（新流中元素的类型）。
    - 你需要提供一个 Lambda 表达式来实现 `apply` 方法，定义你的转换逻辑。

#### 示例：获取每个字符串的长度

Java

```
import java.util.List;
import java.util.stream.Collectors;

List<String> words = List.of("Apple", "Banana", "Sky");

List<Integer> wordLengths = words.stream()
    .map(s -> s.length()) // Lambda表达式 s -> s.length() 实现了 Function<String, Integer>
    .collect(Collectors.toList());

System.out.println(wordLengths); // 输出: [5, 6, 3]
```

**工作流程**：

1. "Apple" -> `s.length()` -> `5` (流中的元素从 `String` 变为 `Integer`)
2. "Banana" -> `s.length()` -> `6`
3. "Sky" -> `s.length()` -> `3`

### 3. `sorted()` - 排序

`sorted` 操作用于对流中的元素进行排序。它有两个重载版本。

- **`sorted()` (无参版本)**：
    
    - **功能**：对流中的元素进行**自然排序**。
    - **要求**：流中的元素必须实现了 `Comparable` 接口（例如 `String`, `Integer` 等基本包装类型都已实现）。否则，在执行时会抛出 `ClassCastException`。
- **`sorted(Comparator<? super T> comparator)` (带参版本)**：
    
    - **功能**：使用你提供的**自定义比较器**对元素进行排序。
    - **参数 `Comparator<? super T>`**：这是一个函数式接口，用于定义两个对象之间的比较逻辑。

#### 示例 1：自然排序

Java

```
import java.util.List;
import java.util.stream.Collectors;

List<String> words = List.of("Elephant", "Apple", "Banana");

List<String> sortedWords = words.stream()
    .sorted()
    .collect(Collectors.toList());

System.out.println(sortedWords); // 输出: [Apple, Banana, Elephant] (按字母顺序)
```

#### 示例 2：按字符串长度排序（自定义比较器）

Java

```
import java.util.List;
import java.util.Comparator;
import java.util.stream.Collectors;

List<String> words = List.of("Elephant", "Apple", "Banana");

// 使用 Comparator.comparingInt() 简化写法
List<String> sortedByLength = words.stream()
    .sorted(Comparator.comparingInt(String::length)) // 按长度升序
    .collect(Collectors.toList());

System.out.println(sortedByLength); // 输出: [Apple, Banana, Elephant]

// 降序排序
List<String> sortedByLengthDesc = words.stream()
    .sorted(Comparator.comparingInt(String::length).reversed()) // 按长度降序
    .collect(Collectors.toList());

System.out.println(sortedByLengthDesc); // 输出: [Elephant, Banana, Apple]
```

### 4. `distinct()` - 去重

`distinct` 操作用于去除流中重复的元素。

- **功能**：基于元素的 `equals()` 方法来判断两个元素是否相等。它会保留第一次出现的元素，并丢弃后续出现的相同元素。
- **要求**：为了正确工作，流中元素的类应该正确地覆写 `hashCode()` 和 `equals()` 方法。对于 `String`, `Integer` 等标准库类，这已经做好了。

#### 示例：去除列表中的重复数字

Java

```
import java.util.List;
import java.util.stream.Collectors;

List<Integer> numbers = List.of(1, 2, 2, 3, 4, 4, 4, 5);

List<Integer> distinctNumbers = numbers.stream()
    .distinct()
    .collect(Collectors.toList());

System.out.println(distinctNumbers); // 输出: [1, 2, 3, 4, 5]
```

### 5. `limit(long maxSize)` - 截断

`limit` 操作是一个**短路（short-circuiting）**操作。它用于限制从流中获取的元素数量。

- **功能**：返回一个不超过给定长度的新流。一旦流中的元素数量达到了 `maxSize`，处理就会停止，后续的元素将不再被处理。这在处理无限流或大型流时非常高效。
- **参数 `maxSize`**：一个非负整数，表示所需的最大元素数量。

#### 示例：获取前 3 个数字

Java

```
import java.util.List;
import java.util.stream.Collectors;

List<Integer> numbers = List.of(10, 20, 30, 40, 50, 60);

List<Integer> limitedNumbers = numbers.stream()
    .limit(3)
    .collect(Collectors.toList());

System.out.println(limitedNumbers); // 输出: [10, 20, 30]
```

---

### 6. `skip(long n)` - 跳过

`skip` 操作与 `limit` 相对，它用于跳过流中的前 n 个元素。

- **功能**：丢弃流中的前 `n` 个元素，返回一个包含剩余元素的新流。如果流中的元素数量不足 `n`，则会返回一个空流。
- **参数 `n`**：一个非负整数，表示要跳过的元素数量。

#### 示例：跳过前 2 个元素后，获取之后的所有元素

Java

```
import java.util.List;
import java.util.stream.Collectors;

List<Integer> numbers = List.of(10, 20, 30, 40, 50);

List<Integer> skippedNumbers = numbers.stream()
    .skip(2)
    .collect(Collectors.toList());

System.out.println(skippedNumbers); // 输出: [30, 40, 50]
```

### 组合使用：实现分页逻辑

`skip` 和 `limit` 经常被组合使用来实现分页功能。

#### 示例：每页 3 个元素，获取第 2 页的数据

Java

```
import java.util.List;
import java.util.stream.Collectors;
import java.util.stream.Stream;

List<String> data = List.of("A", "B", "C", "D", "E", "F", "G", "H");
int pageNumber = 2;
int pageSize = 3;

List<String> pageData = data.stream()
    .skip((pageNumber - 1) * pageSize) // 跳过 (2-1)*3 = 3 个元素
    .limit(pageSize)                   // 获取 3 个元素
    .collect(Collectors.toList());

System.out.println(pageData); // 输出: [D, E, F]
```

这些中间操作可以像积木一样自由组合，以声明式的方式构建出复杂而强大的数据处理流水线。
