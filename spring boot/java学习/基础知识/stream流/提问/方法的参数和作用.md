```java
List<String> result = names.stream() // 创建流
                           .filter(name -> name.length() > 3) 
                           .map(String::toUpperCase)          
                           .collect(Collectors.toList());     
```
## **Q:** 
这个代码块.stream(),.filter()都是方法吗，这些方法的参数都是什么呢？

## **A**：
是的，这段代码中的 `.stream()`、`.filter()` 和 `.map()` 都是 **方法**，它们属于 Java 的 **Stream API**。这些方法的参数是 **函数式接口**（Functional Interface），通常用 **Lambda 表达式** 或 **方法引用** 来传递。以下是每个方法的详细说明：

---
[[常见函数式接口]]
### 1. **`.stream()`**
- **作用**：将集合（如 `List`、`Set`）转换为一个流（`Stream`）。
- **参数**：无参数。
- **返回值**：返回一个 `Stream` 对象。

**示例**：
```java
List<String> names = Arrays.asList("Alice", "Bob", "Charlie");
Stream<String> stream = names.stream(); // 将 List 转换为流
```

---

### 2. **`.filter()`**
- **作用**：过滤流中的元素，保留符合条件的元素。
- **参数**：接受一个 `Predicate<T>` 函数式接口。
  - `Predicate<T>` 是一个函数式接口，它的抽象方法是 `boolean test(T t)`。
  - 可以用 Lambda 表达式或方法引用来实现。
- **返回值**：返回一个新的 `Stream`，包含过滤后的元素。

**示例**：
```java
Stream<String> filteredStream = stream.filter(name -> name.length() > 3);
```
**解释**：
- `name -> name.length() > 3` 是一个 Lambda 表达式，实现了 `Predicate<String>` 的 `test` 方法。
- 它会过滤出长度大于 3 的字符串。

---

### 3. **`.map()`**
- **作用**：将流中的每个元素映射为另一个值。
- **参数**：接受一个 `Function<T, R>` 函数式接口。
  - `Function<T, R>` 是一个函数式接口，它的抽象方法是 `R apply(T t)`。
  - 可以用 Lambda 表达式或方法引用来实现。
- **返回值**：返回一个新的 `Stream`，包含映射后的元素。

**示例**：
```java
Stream<String> mappedStream = filteredStream.map(String::toUpperCase);
```
**解释**：
- `String::toUpperCase` 是一个方法引用，相当于 `name -> name.toUpperCase()`。
- 它会将每个字符串转换为大写。

---

### 4. **`.collect()`**
- **作用**：将流中的元素收集到一个结果容器中（如 `List`、`Set`、`Map` 等）。
- **参数**：接受一个 `Collector<T, A, R>` 对象。
  - `Collectors.toList()` 是一个常用的 `Collector`，用于将元素收集到 `List` 中。
- **返回值**：返回一个结果容器（如 `List`）。

**示例**：
```java
List<String> result = mappedStream.collect(Collectors.toList());
```
**解释**：
- `Collectors.toList()` 将流中的元素收集到一个 `List` 中。

---

### 5. **完整代码解析**
```java
List<String> result = names.stream() // 1. 将 List 转换为流
                           .filter(name -> name.length() > 3) // 2. 过滤长度大于 3 的字符串
                           .map(String::toUpperCase) // 3. 将字符串转换为大写
                           .collect(Collectors.toList()); // 4. 收集为 List
```
**步骤**：
1. **`.stream()`**：将 `names` 转换为流。
2. **`.filter()`**：过滤出长度大于 3 的字符串。
3. **`.map()`**：将每个字符串转换为大写。
4. **`.collect()`**：将结果收集到 `List` 中。

---

### 6. **方法参数总结**
| 方法名      | 参数类型               | 参数说明                                                                 |
|-------------|------------------------|------------------------------------------------------------------------|
| `.stream()` | 无                     | 将集合转换为流。                                                        |
| `.filter()` | `Predicate<T>`         | 接受一个 `Predicate`，用于过滤流中的元素。                                |
| `.map()`    | `Function<T, R>`       | 接受一个 `Function`，用于将流中的每个元素映射为另一个值。                  |
| `.collect()`| `Collector<T, A, R>`   | 接受一个 `Collector`，用于将流中的元素收集到结果容器中（如 `List`、`Set`）。 |

---

### 7. **函数式接口的实现方式**
这些方法的参数都是函数式接口，可以用以下方式实现：
- **Lambda 表达式**：如 `name -> name.length() > 3`。
- **方法引用**：如 `String::toUpperCase`。
- **匿名内部类**：如 `new Predicate<String>() { ... }`（不推荐）。

---

### 8. **总结**
- `.stream()`、`.filter()`、`.map()` 和 `.collect()` 都是 Stream API 的方法。
- 它们的参数是函数式接口，通常用 Lambda 表达式或方法引用来传递。
- 通过这些方法，可以对集合进行链式操作，实现过滤、映射、收集等功能。

通过理解这些方法的参数和用法，可以更好地掌握 Stream API 的使用！
