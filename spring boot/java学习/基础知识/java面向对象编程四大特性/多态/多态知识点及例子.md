Java 中的 **多态（Polymorphism）** 是面向对象编程的四大特性之一（其他三个是封装、继承和抽象）。多态允许一个对象在不同的情况下表现出不同的行为。它主要通过 **方法重写（Override）** 和 **向上转型（Upcasting）** 来实现。以下是 Java 多态的详细知识点及其示例：

---

### 1. **多态的定义**
多态是指同一个方法或操作在不同对象上有不同的表现形式。它分为两种类型：
- <mark style="background: #FFF3A3A6;">**编译时多态（静态多态）**：通过方法重载（Overload）实现。</mark>
- <mark style="background: #FFF3A3A6;">**运行时多态（动态多态）**：通过方法重写（Override）和向上转型实现。</mark>

---

### 2. **多态的实现方式**
#### (1) **方法重写（Override）**
- 子类可以重写父类的方法，提供自己的实现。
- 调用时，实际执行的是子类重写后的方法。

**示例：**
```java
class Animal {
    void sound() {
        System.out.println("Animal makes a sound");
    }
}

class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("Dog barks");
    }
}

class Cat extends Animal {
    @Override
    void sound() {
        System.out.println("Cat meows");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal myDog = new Dog(); // 向上转型
        Animal myCat = new Cat(); // 向上转型

        myDog.sound(); // 输出: Dog barks
        myCat.sound(); // 输出: Cat meows
    }
}
```

#### (2)
## <mark style="background: #FFF3A3A6;">**向上转型（Upcasting）**</mark>
[[向上转型，向下转型，动态绑定]]
[[向上转型注意事项]]
- 将子类对象赋值给父类引用。
- 通过父类引用调用方法时，实际执行的是子类重写的方法。

**示例：**
```java
Animal myAnimal = new Dog(); // 向上转型
myAnimal.sound(); // 输出: Dog barks
```
## <mark style="background: #FFF3A3A6;">限制</mark>
- **无法调用子类特有的方法**：
  向上转型后，父类引用只能调用父类中定义的方法，无法调用子类特有的方法。如果需要调用子类特有的方法，必须进行 **向下转型（Downcasting）**。

**示例：**
```java
Animal myAnimal = new Dog(); // 向上转型
// myAnimal.fetch(); // 错误：Animal 类中没有 fetch 方法

if (myAnimal instanceof Dog) {
    Dog myDog = (Dog) myAnimal; // 向下转型
    myDog.fetch(); // 正确：调用 Dog 类特有的方法
}
```

#### (3) **动态绑定（Dynamic Binding）**
- 在运行时确定调用哪个类的方法。
- 这是 Java 默认的行为。

**示例：**
```java
Animal myAnimal;
myAnimal = new Dog(); // 动态绑定到 Dog 类
myAnimal.sound(); // 输出: Dog barks

myAnimal = new Cat(); // 动态绑定到 Cat 类
myAnimal.sound(); // 输出: Cat meows
```

---

### 3. **多态的优点**
- **代码复用**：通过继承和方法重写，减少重复代码。
- **可扩展性**：可以轻松添加新的子类，而不需要修改现有代码。
- **灵活性**：同一个方法可以在不同子类中有不同的实现。

---

### 4. **多态的应用场景**
#### (1) **方法参数多态**
可以将父类类型作为方法参数，传递子类对象。

**示例：**
```java
class Main {
    static void makeSound(Animal animal) {
        animal.sound();
    }

    public static void main(String[] args) {
        makeSound(new Dog()); // 输出: Dog barks
        makeSound(new Cat()); // 输出: Cat meows
    }
}
```
## Q: 为什么可以传递子类对象？
## A:
<mark style="background: #FFF3A3A6;">Java 允许将子类对象传递给父类类型的参数，原因如下：</mark>

- **继承关系**：<mark style="background: #FFF3A3A6;">子类是父类的一种，所以子类对象可以被当作父类对象使用。</mark>
- **多态**：<mark style="background: #FFF3A3A6;">父类引用可以指向子类对象，并且在运行时调用子类重写的方法。</mark>

**类比：**

- <mark style="background: #FFF3A3A6;">假设 `Animal` 是“车”，`Dog` 是“轿车”，`Cat` 是“卡车”。</mark>
- <mark style="background: #FFF3A3A6;">当你定义一个方法 `drive(Vehicle vehicle)` 时，你可以传递“轿车”或“卡车”，因为它们都是“车”的一种。</mark>
#### (2) **集合多态**
可以将不同子类对象存储在一个父类类型的集合中。

**示例：**
```java
List<Animal> animals = new ArrayList<>();
animals.add(new Dog());
animals.add(new Cat());

for (Animal animal : animals) {
    animal.sound();
}
// 输出:
// Dog barks
// Cat meows
```

#### (3) **工厂模式**
通过多态实现对象的动态创建。

**示例：**
```java
class AnimalFactory {
    static Animal getAnimal(String type) {
        if (type.equals("Dog")) {
            return new Dog();
        } else if (type.equals("Cat")) {
            return new Cat();
        }
        return null;
    }
}

public class Main {
    public static void main(String[] args) {
        Animal myAnimal = AnimalFactory.getAnimal("Dog");
        myAnimal.sound(); // 输出: Dog barks
    }
}
```

---

### 5. **多态与重载的区别**
| 特性           | 多态（重写）                          | 重载（Overload）                     |
|----------------|---------------------------------------|--------------------------------------|
| **定义**       | 子类重写父类的方法                    | 同一个类中方法名相同，参数不同       |
| **绑定时间**   | 运行时（动态绑定）                    | 编译时（静态绑定）                   |
| **作用范围**   | 继承关系中的方法                      | 同一个类中的方法                     |
| **示例**       | `void sound()` 在子类中重写           | `void sound()` 和 `void sound(int)`  |

---

### 6. **总结**
Java 中的多态通过方法重写和向上转型实现，它使代码更加灵活、可扩展和易于维护。理解多态的关键是掌握以下几点：
- 方法重写和动态绑定。
- 向上转型和向下转型。
- 多态的应用场景，如方法参数多态、集合多态和设计模式。
