在Java中，**抽象**是一种重要的面向对象编程（OOP）概念，它允许我们隐藏复杂的实现细节，只暴露必要的接口或功能。抽象主要通过**抽象类**和**接口**来实现。下面我们将详细介绍抽象的相关知识点，并通过示例来帮助理解。

<mark style="background: #FFF3A3A6;">也就是抽象类和接口，没其他的</mark>
---

### 1. **抽象类（Abstract Class）**
抽象类是一种不能实例化的类，通常作为其他类的基类（父类）。抽象类可以包含抽象方法（没有实现的方法）和具体方法（有实现的方法）。

#### 1.1 抽象类的特点
- 使用 `abstract` 关键字定义。
- 不能被实例化（不能创建对象）。
- 可以包含抽象方法和具体方法。
- 子类必须实现抽象类中的所有抽象方法（除非子类也是抽象类）。

#### 1.2 示例
```java
// 定义一个抽象类
abstract class Animal {
    // 抽象方法，没有方法体
    public abstract void makeSound();

    // 具体方法
    public void sleep() {
        System.out.println("This animal is sleeping.");
    }
}

// 子类继承抽象类
class Dog extends Animal {
    // 实现抽象方法
    @Override
    public void makeSound() {
        System.out.println("Woof! Woof!");
    }
}

// 测试类
public class Main {
    public static void main(String[] args) {
        Dog dog = new Dog();
        dog.makeSound(); // 输出: Woof! Woof!
        dog.sleep();     // 输出: This animal is sleeping.
    }
}
```

### 2. **接口（Interface）**
接口是一种完全抽象的类，它只能包含抽象方法（在Java 8之前）和常量（`public static final`）。从Java 8开始，接口可以包含**默认方法**（`default` 方法）和**静态方法**。

#### 2.1 接口的特点
- 使用 `interface` 关键字定义。
- 不能被实例化。
- 所有方法默认是 `public abstract` 的（在Java 8之前）。
- 所有字段默认是 `public static final` 的。
- 类通过 `implements` 关键字实现接口。
- 一个类可以实现多个接口（Java支持多继承接口）。

#### 2.2 示例
```java
// 定义一个接口
interface Vehicle {
    // 抽象方法
    void start();
    void stop();
}

// 类实现接口
class Car implements Vehicle {
    @Override
    public void start() {
        System.out.println("Car is starting.");
    }

    @Override
    public void stop() {
        System.out.println("Car is stopping.");
    }
}

// 测试类
public class Main {
    public static void main(String[] args) {
        Car car = new Car();
        car.start(); // 输出: Car is starting.
        car.stop();  // 输出: Car is stopping.
    }
}
```

### 3. **抽象类 vs 接口**
| 特性                | 抽象类                          | 接口                          |
|---------------------|--------------------------------|-------------------------------|
| 关键字              | `abstract`                    | `interface`                  |
| 实例化              | 不能实例化                     | 不能实例化                    |
| 方法实现            | 可以包含具体方法和抽象方法     | 只能包含抽象方法（Java 8之前）|
| 字段                | 可以是任意类型                 | 只能是 `public static final`  |
| 继承                | 单继承（一个类只能继承一个抽象类）| 多继承（一个类可以实现多个接口）|
| 默认方法            | 不支持                         | 支持（Java 8开始）            |
| 静态方法            | 不支持                         | 支持（Java 8开始）            |

### 4. **抽象类的应用场景**
- 当多个类有共同的属性和行为，但具体实现不同时，可以使用抽象类。
- 当需要为子类提供一些默认实现时，可以使用抽象类。

#### 示例
```java
abstract class Shape {
    // 抽象方法
    public abstract double area();

    // 具体方法
    public void display() {
        System.out.println("This is a shape.");
    }
}

class Circle extends Shape {
    private double radius;

    public Circle(double radius) {
        this.radius = radius;
    }

    @Override
    public double area() {
        return Math.PI * radius * radius;
    }
}

class Rectangle extends Shape {
    private double width, height;

    public Rectangle(double width, double height) {
        this.width = width;
        this.height = height;
    }

    @Override
    public double area() {
        return width * height;
    }
}
```

---

### 5. **接口的应用场景**
- 当需要定义一组行为（方法）而不关心具体实现时，可以使用接口。
- 当需要实现多继承时，可以使用接口。

#### 示例
```java
interface Flyable {
    void fly();
}

interface Swimmable {
    void swim();
}

class Bird implements Flyable {
    @Override
    public void fly() {
        System.out.println("Bird is flying.");
    }
}

class Duck implements Flyable, Swimmable {
    @Override
    public void fly() {
        System.out.println("Duck is flying.");
    }

    @Override
    public void swim() {
        System.out.println("Duck is swimming.");
    }
}
```

---

### 6. **总结**
- **抽象类**用于定义具有共同属性和行为的类的模板，可以包含具体方法和抽象方法。
- **接口**用于定义一组行为，强调“能做什么”，不关心具体实现。
- 在实际开发中，抽象类和接口通常结合使用，以实现更灵活的设计。
