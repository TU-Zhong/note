Java封装（Encapsulation）是面向对象编程（OOP）的四大基本原则之一，它通过将数据和操作数据的方法绑定在一起，并隐藏内部实现细节，来保护对象的状态。封装的主要目的是提高代码的安全性、可维护性和可复用性。

### 封装的核心知识点

1. **访问修饰符（Access Modifiers）**
	[[private,public,default,protected修饰符解析]]
   - `private`：私有访问权限，只能在类内部访问。
   - `protected`：受保护的访问权限，可以在类内部、子类和同一个包内访问。
   - `default`（默认，不写修饰符）：包内访问权限，只能在同一个包内访问。
   - `public`：公共访问权限，可以在任何地方访问。

2. **Getter 和 Setter 方法**
   - Getter方法用于获取私有属性的值。
   - Setter方法用于设置私有属性的值。
   - 通过Getter和Setter方法，可以控制对属性的访问和修改。

3. **隐藏实现细节**
   - 通过封装，可以将类的内部实现细节隐藏起来，只暴露必要的接口给外部使用。

4. **提高安全性**
   - 封装可以防止外部代码直接修改对象的内部状态，从而提高数据的安全性。

5. **提高可维护性**
   - 通过封装，可以在不改变外部接口的情况下，修改类的内部实现，从而提高代码的可维护性。

### 封装的例子

#### 1. 使用`private`修饰符封装属性

```java
public class Person {
    // 私有属性
    private String name;
    private int age;

    // 公共的Getter方法
    public String getName() {
        return name;
    }

    // 公共的Setter方法
    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        if (age > 0) { // 通过Setter方法进行数据验证
            this.age = age;
        } else {
            System.out.println("年龄必须大于0");
        }
    }
}
```

#### 2. 使用`protected`修饰符封装属性

```java
public class Animal {
    // 受保护的属性
    protected String species;

    // 受保护的方法
    protected void makeSound() {
        System.out.println("动物发出声音");
    }
}

class Dog extends Animal {
    public void displaySpecies() {
        species = "犬科"; // 子类可以访问受保护的属性
        System.out.println("物种: " + species);
    }

    @Override
    protected void makeSound() {
        System.out.println("狗在叫");
    }
}
```

#### 3. 使用`default`修饰符封装属性

```java
class Car {
    // 默认访问权限的属性
    String brand;

    // 默认访问权限的方法
    void start() {
        System.out.println("汽车启动");
    }
}

// 同一个包内的另一个类
class TestCar {
    public static void main(String[] args) {
        Car car = new Car();
        car.brand = "Toyota"; // 在同一个包内可以访问默认权限的属性
        car.start();
    }
}
```

#### 4. 使用`public`修饰符封装方法

```java
public class BankAccount {
    // 私有属性
    private double balance;

    // 公共的Getter方法
    public double getBalance() {
        return balance;
    }

    // 公共的存款方法
    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
        } else {
            System.out.println("存款金额必须大于0");
        }
    }

    // 公共的取款方法
    public void withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
        } else {
            System.out.println("取款金额无效");
        }
    }
}
```

### 封装的好处

1. **安全性**：通过隐藏内部实现，防止外部代码直接修改对象的状态。
2. **灵活性**：可以在不改变外部接口的情况下，修改类的内部实现。
3. **可维护性**：封装使得代码更容易理解和维护，因为内部实现细节被隐藏，外部代码只关心接口。
4. **代码复用**：封装后的类可以作为独立的模块，方便在其他项目中复用。

### 总结

封装是Java中非常重要的概念，它通过访问修饰符、Getter和Setter方法等手段，将数据和操作数据的方法绑定在一起，并隐藏内部实现细节。通过封装，可以提高代码的安全性、可维护性和可复用性。
