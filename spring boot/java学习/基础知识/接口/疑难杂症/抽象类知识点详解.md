### 抽象类（Abstract Class）知识点详解

抽象类（Abstract Class）是Java中一种特殊的类，它不能被实例化，只能被继承。抽象类通常用于定义一些通用的行为或属性，并强制子类实现特定的方法。以下是抽象类的详细知识点及其示例。

---

#### 1. **抽象类的定义**
抽象类使用 `abstract` 关键字声明。它可以包含抽象方法（没有方法体的方法）和具体方法（有方法体的方法）。

```java
abstract class Animal {
    // 抽象方法
    abstract void makeSound();

    // 具体方法
    void breathe() {
        System.out.println("Breathing...");
    }
}
```

---

#### 2. **抽象类的特点**
- **不能实例化**：抽象类不能直接创建对象，只能通过子类继承。
- **可以包含抽象方法和具体方法**：抽象方法没有方法体，必须在子类中实现；具体方法可以有方法体，子类可以直接继承或重写。
- **可以包含成员变量**：抽象类可以定义实例变量和静态变量。
- **可以包含构造方法**：抽象类可以有构造方法，用于初始化成员变量，但不能直接调用。
- **可以继承其他类**：抽象类可以继承其他类（包括抽象类和非抽象类）。
- **可以实现接口**：抽象类可以实现一个或多个接口。

---

#### 3. **抽象方法的定义**
抽象方法使用 `abstract` 关键字声明，没有方法体。必须在子类中实现。

```java
abstract class Animal {
    abstract void makeSound();
}
```

---

#### 4. **抽象类的继承**
子类通过 `extends` 关键字继承抽象类，并且必须实现抽象类中的所有抽象方法（除非子类也是抽象类）。

```java
class Dog extends Animal {
    @Override
    void makeSound() {
        System.out.println("Woof!");
    }
}
```

---

#### 5. **抽象类的构造方法**
抽象类可以有构造方法，用于初始化成员变量。构造方法在子类实例化时调用。

```java
abstract class Animal {
    String name;

    Animal(String name) {
        this.name = name;
    }

    abstract void makeSound();
}

class Dog extends Animal {
    Dog(String name) {
        super(name);
    }

    @Override
    void makeSound() {
        System.out.println(name + " says Woof!");
    }
}
```

---

#### 6. **抽象类的成员变量**
抽象类可以定义实例变量和静态变量。

```java
abstract class Animal {
    String name; // 实例变量
    static int count = 0; // 静态变量

    Animal(String name) {
        this.name = name;
        count++;
    }

    abstract void makeSound();
}
```

---

#### 7. **抽象类与接口的区别**
| 特性               | 抽象类                          | 接口                          |
|--------------------|---------------------------------|------------------------------|
| **实例化**         | 不能实例化                      | 不能实例化                    |
| **方法**           | 可以包含抽象方法和具体方法       | 只能包含抽象方法、默认方法、静态方法和私有方法 |
| **成员变量**       | 可以包含实例变量和静态变量       | 只能包含常量（`public static final` 变量） |
| **构造方法**       | 可以有构造方法                  | 不能有构造方法                |
| **继承**           | 只能单继承（一个类只能继承一个抽象类） | 可以多继承（一个类可以实现多个接口） |
| **实现**           | 不能实现接口                    | 可以实现多个接口              |

---

#### 8. **抽象类的应用场景**
- **定义通用行为**：当多个类共享一些通用行为时，可以将这些行为定义在抽象类中。
- **强制子类实现特定方法**：当某些方法必须由子类实现时，可以将这些方法定义为抽象方法。
- **代码复用**：通过抽象类可以复用代码，减少重复代码。

---

#### 9. **示例代码**
```java
// 抽象类
abstract class Shape {
    String color;

    Shape(String color) {
        this.color = color;
    }

    // 抽象方法
    abstract double area();

    // 具体方法
    void displayColor() {
        System.out.println("Color: " + color);
    }
}

// 子类
class Circle extends Shape {
    double radius;

    Circle(String color, double radius) {
        super(color);
        this.radius = radius;
    }

    @Override
    double area() {
        return Math.PI * radius * radius;
    }
}

class Rectangle extends Shape {
    double length;
    double width;

    Rectangle(String color, double length, double width) {
        super(color);
        this.length = length;
        this.width = width;
    }

    @Override
    double area() {
        return length * width;
    }
}

public class Main {
    public static void main(String[] args) {
        Shape circle = new Circle("Red", 5.0);
        Shape rectangle = new Rectangle("Blue", 4.0, 6.0);

        circle.displayColor();
        System.out.println("Area of circle: " + circle.area());

        rectangle.displayColor();
        System.out.println("Area of rectangle: " + rectangle.area());
    }
}
```

---

#### 10. **总结**
抽象类是Java中实现代码复用和定义通用行为的重要工具。通过抽象类，可以定义一些通用的属性和方法，并强制子类实现特定的行为。抽象类与接口的区别在于，抽象类更适合定义通用的行为，而接口更适合定义行为契约。合理使用抽象类可以提高代码的可维护性和复用性。
