**场景：** 我们将模拟一个智能家居生态系统，其中包含不同类型的可控制设备。

Java

```java
// ============== 1. 基础接口：可控制设备 ==============
interface Controllable {
    // 1.1 常量 (隐式 public static final)
    String DEVICE_CATEGORY = "Generic Controllable";

    // 1.2 抽象方法 (隐式 public abstract)
    void powerOn();
    void powerOff();
    boolean isPoweredOn();

    // 1.3 默认方法 (Java 8+)
    default void togglePower() {
        if (isPoweredOn()) {
            powerOff();
        } else {
            powerOn();
        }
        // 1.5.1 调用接口内的私有方法
        logStatusChange("Toggled");
    }

    // 1.5.2 私有方法 (Java 9+) - 仅限接口内部使用，辅助默认方法
    private void logStatusChange(String action) {
        // 假设这是当前时间
        String timestamp = new java.text.SimpleDateFormat("HH:mm:ss").format(new java.util.Date());
        System.out.println("[" + timestamp + " CONTROLLABLE LOG] " + this.getClass().getSimpleName() + " (via interface): Power " + action);
    }
}

// ============== 2. 基础接口：可连接网络设备 ==============
interface Connectable {
    String DEFAULT_WIFI_SSID = "MyHomeNetwork";

    boolean connect(String networkId);
    void disconnect();
    String getNetworkStatus();

    // 2.1 默认方法
    default void displayConnectionInfo() {
        System.out.println("Network Status: " + getNetworkStatus() + " (Protocol: " + getNetworkProtocol() + ")");
    }

    // 2.2 抽象方法，被默认方法调用
    String getNetworkProtocol(); // 实现类需要提供具体的协议

    // 2.3 静态方法 (Java 8+)
    static void showNetworkTips() {
        System.out.println(getStaticTipHeader() + "Ensure your Wi-Fi router is powered on and the password is correct.");
    }

    // 2.4 私有静态方法 (Java 9+) - 辅助静态方法
    private static String getStaticTipHeader() {
        return "[NETWORK TIP] ";
    }
}

// ============== 3. 扩展接口：智能设备 (继承 Controllable 和 Connectable) ==============
interface SmartDevice extends Controllable, Connectable {
    // 3.1 新增常量
    String SMART_DEVICE_TAG = "IOT_ENABLED";

    // 3.2 新增抽象方法
    String getDeviceName();
    void performSelfCheck(); // 执行自检

    // 3.3 默认方法，覆盖或利用父接口的默认方法
    @Override // 演示可以覆盖父接口的默认方法，尽管这里 Controllable 没有同名默认方法可覆盖
              // 更常见的场景是提供一个更具体的默认实现
    default void togglePower() {
        Controllable.super.togglePower(); // 可以调用父接口的默认实现
        System.out.println(getDeviceName() + " (SmartDevice) power state toggled intelligently.");
        // 调用此接口的私有方法
        privateLogSmartAction("Smart Power Toggle");
    }

    // 3.4 此接口的私有方法
    private void privateLogSmartAction(String action) {
         System.out.println("[SMART LOG] " + getDeviceName() + ": " + action);
    }

    // 3.5 默认方法，展示复杂逻辑
    default String getFullDeviceInfo() {
        return String.format("Device: %s [%s]\n  Power: %s\n  Network: %s\n  Protocol: %s",
                getDeviceName(),
                SMART_DEVICE_TAG,
                isPoweredOn() ? "ON" : "OFF",
                getNetworkStatus(),
                getNetworkProtocol()
        );
    }
}

// ============== 4. 具体实现类：智能灯泡 ==============
class SmartLight implements SmartDevice { // 实现扩展接口
    private String name;
    private boolean power = false;
    private String networkStatus = "Disconnected";
    private final String protocol = "Zigbee";

    public SmartLight(String name) {
        this.name = name;
    }

    @Override
    public String getDeviceName() {
        return this.name;
    }

    @Override
    public void powerOn() {
        this.power = true;
        System.out.println(name + " light is now ON.");
    }

    @Override
    public void powerOff() {
        this.power = false;
        System.out.println(name + " light is now OFF.");
    }

    @Override
    public boolean isPoweredOn() {
        return this.power;
    }

    @Override
    public boolean connect(String networkId) {
        if (DEFAULT_WIFI_SSID.equals(networkId) || "ZigbeeHub".equals(networkId)) {
            this.networkStatus = "Connected to " + networkId;
            System.out.println(name + " connected to " + networkId + " via " + protocol);
            return true;
        }
        this.networkStatus = "Failed to connect to " + networkId;
        System.out.println(name + " failed to connect.");
        return false;
    }

    @Override
    public void disconnect() {
        this.networkStatus = "Disconnected";
        System.out.println(name + " disconnected.");
    }

    @Override
    public String getNetworkStatus() {
        return this.networkStatus;
    }

    @Override
    public String getNetworkProtocol() {
        return this.protocol;
    }

    @Override
    public void performSelfCheck() {
        System.out.println(name + " performing self-check... Bulb OK. Connection: " + networkStatus);
    }

    // SmartLight 特有的方法
    public void setBrightness(int level) {
        if (isPoweredOn()) {
            System.out.println(name + " brightness set to " + level + "%.");
        } else {
            System.out.println(name + " is off. Cannot set brightness.");
        }
    }
}

// ============== 5. 具体实现类：智能音箱 (实现多个接口) ==============
// 假设我们还有一个简单的 Notifiable 接口
interface Notifiable {
    void sendAlert(String message);
}

class SmartSpeaker implements SmartDevice, Notifiable { // 实现 SmartDevice (间接 Controllable, Connectable) 和 Notifiable
    private String speakerName;
    private boolean isPowered = false;
    private String connection = "Offline";
    private final String netProtocol = "Wi-Fi/Bluetooth";

    public SmartSpeaker(String speakerName) {
        this.speakerName = speakerName;
    }

    @Override
    public String getDeviceName() {
        return this.speakerName;
    }

    @Override
    public void powerOn() {
        this.isPowered = true;
        System.out.println(speakerName + " is powered ON.");
    }

    @Override
    public void powerOff() {
        this.isPowered = false;
        System.out.println(speakerName + " is powered OFF.");
    }

    @Override
    public boolean isPoweredOn() {
        return this.isPowered;
    }

    @Override
    public boolean connect(String networkId) {
        this.connection = "Connected to " + networkId;
        System.out.println(speakerName + " is now connected to " + networkId + " using " + netProtocol);
        return true;
    }

    @Override
    public void disconnect() {
        this.connection = "Offline";
        System.out.println(speakerName + " disconnected.");
    }

    @Override
    public String getNetworkStatus() {
        return this.connection;
    }

    @Override
    public String getNetworkProtocol() {
        return this.netProtocol;
    }

    @Override
    public void performSelfCheck() {
        System.out.println(speakerName + " audio self-check: OK. Network: " + connection);
    }

    // 实现 Notifiable 接口的方法
    @Override
    public void sendAlert(String message) {
        if(isPoweredOn()){
            System.out.println(speakerName + " ALERT: \"" + message + "\"");
        } else {
            System.out.println(speakerName + " is OFF. Cannot send alert.");
        }
    }

    // SmartSpeaker 特有的方法
    public void playMusic(String track) {
        if (isPoweredOn() && !connection.equals("Offline")) {
            System.out.println(speakerName + " is playing: " + track);
        } else {
            System.out.println(speakerName + " cannot play music (check power/connection).");
        }
    }
}


// ============== 6. 主程序：演示所有功能 ==============
public class ComprehensiveInterfaceDemo {
    public static void main(String[] args) {
        System.out.println("===== 1. 接口常量和静态方法 =====");
        System.out.println("Controllable Device Category: " + Controllable.DEVICE_CATEGORY);
        System.out.println("Default Wi-Fi SSID: " + Connectable.DEFAULT_WIFI_SSID);
        Connectable.showNetworkTips(); // 调用 Connectable 接口的静态方法

        System.out.println("\n===== 2. 创建和操作 SmartLight (实现 SmartDevice) =====");
        SmartLight livingRoomLight = new SmartLight("Living Room Light");
        SmartDevice lightDevice = livingRoomLight; // 多态性

        System.out.println("Device Name: " + lightDevice.getDeviceName());
        lightDevice.powerOn();
        System.out.println("Is Light On? " + lightDevice.isPoweredOn());
        lightDevice.connect("ZigbeeHub");
        lightDevice.displayConnectionInfo(); // 调用 Connectable 的默认方法
        lightDevice.performSelfCheck();
        livingRoomLight.setBrightness(80); // 调用 SmartLight 的特有方法

        System.out.println("\n--- 测试 SmartDevice 的 togglePower (调用了父接口默认方法和私有方法) ---");
        lightDevice.togglePower(); // 会触发 SmartDevice 的 togglePower, 内部调用 Controllable.super.togglePower()
        lightDevice.togglePower();

        System.out.println("\n--- 测试 SmartDevice 的 getFullDeviceInfo (默认方法) ---");
        System.out.println(lightDevice.getFullDeviceInfo());

        System.out.println("\n===== 3. 创建和操作 SmartSpeaker (实现 SmartDevice 和 Notifiable) =====");
        SmartSpeaker kitchenSpeaker = new SmartSpeaker("Kitchen Speaker");
        SmartDevice speakerDevice = kitchenSpeaker; // 多态
        Notifiable speakerNotifier = kitchenSpeaker; // 多态

        speakerDevice.powerOn();
        speakerDevice.connect(Connectable.DEFAULT_WIFI_SSID); // 使用接口常量
        speakerDevice.performSelfCheck();
        speakerNotifier.sendAlert("Dinner is almost ready!"); // 调用 Notifiable 的方法
        kitchenSpeaker.playMusic("Favorite Jazz Playlist"); // 调用 SmartSpeaker 的特有方法

        System.out.println("\n--- 测试 SmartSpeaker 的 togglePower (SmartDevice 默认方法) ---");
        speakerDevice.togglePower(); // 灯光和音箱都使用 SmartDevice 的 togglePower 实现

        System.out.println("\n--- 测试 SmartSpeaker 的 getFullDeviceInfo ---");
        System.out.println(speakerDevice.getFullDeviceInfo());

        System.out.println("\n===== 4. 使用接口类型进行统一处理 (多态性) =====");
        Controllable[] devices = {livingRoomLight, kitchenSpeaker};
        for (Controllable device : devices) {
            System.out.println("\nOperating device (as Controllable): " + ((SmartDevice)device).getDeviceName()); // 需要转型获取名字
            if (!device.isPoweredOn()) {
                device.powerOn();
            }
            // device.togglePower(); // 调用的是 SmartDevice 覆盖/提供的 togglePower
            if (device instanceof SmartDevice) { // 安全检查和转型
                SmartDevice sd = (SmartDevice) device;
                System.out.println("  Current Network Status: " + sd.getNetworkStatus());
                sd.performSelfCheck();
            }
            if (device instanceof Notifiable) {
                ((Notifiable) device).sendAlert("Routine check alert!");
            }
            device.powerOff();
        }

        System.out.println("\n===== 演示结束 =====");
    }
}

```

**这个例子覆盖的知识点：**

1. **接口定义 (`interface Controllable`, `Connectable`, `SmartDevice`, `Notifiable`)**:
    - **常量**: `Controllable.DEVICE_CATEGORY`, `Connectable.DEFAULT_WIFI_SSID`, `SmartDevice.SMART_DEVICE_TAG` (都是隐式的 `public static final`).
    - **抽象方法**: `powerOn()`, `connect()`, `getDeviceName()`, `sendAlert()` 等 (都是隐式的 `public abstract`).
2. **接口继承 (`SmartDevice extends Controllable, Connectable`)**: `SmartDevice` 继承了 `Controllable` 和 `Connectable` 的所有抽象方法、常量和默认方法（如果它选择不覆盖）。
3. **默认方法 (Java 8+)**:
    - `Controllable.togglePower()`: 提供了基本的开关切换逻辑。
    - `Connectable.displayConnectionInfo()`: 显示连接信息。
    - `SmartDevice.togglePower()`: **覆盖/增强**了 `Controllable` 的 `togglePower`（通过 `Controllable.super.togglePower()` 调用父接口版本）。
    - `SmartDevice.getFullDeviceInfo()`: 一个新的默认方法。
4. **静态方法 (Java 8+)**:
    - `Connectable.showNetworkTips()`: 提供通用的网络提示。
5. **私有方法和私有静态方法 (Java 9+)**:
    - **私有方法**: `Controllable.logStatusChange()`, `SmartDevice.privateLogSmartAction()`，被各自接口内的默认方法调用。
    - **私有静态方法**: `Connectable.getStaticTipHeader()`，被 `Connectable.showNetworkTips()` (静态方法) 调用。
6. **类实现接口**:
    - `SmartLight implements SmartDevice`: `SmartLight` 必须实现 `SmartDevice` 以及其父接口 `Controllable` 和 `Connectable` 中的所有抽象方法。
    - `SmartSpeaker implements SmartDevice, Notifiable`: `SmartSpeaker` 实现了多个接口 (`SmartDevice`本身是一个扩展了两个接口的接口，再加上`Notifiable`)。
7. **多态性**:
    - `SmartDevice lightDevice = livingRoomLight;`
    - `SmartDevice speakerDevice = kitchenSpeaker;`
    - `Notifiable speakerNotifier = kitchenSpeaker;`
    - 在 `Controllable[] devices` 数组中统一处理不同类型的智能设备。
8. **调用父接口的默认方法**: 在 `SmartDevice` 的 `togglePower` 中使用 `Controllable.super.togglePower();`。

这个例子通过构建一个小型“智能家居”模型，将各个知识点融入到实际的类和接口设计中，希望能帮助您更好地理解和巩固Java接口的用法。
