### Java 抽象类 (Abstract Class) 详细知识点

1. **定义 (Definition):**
    
    - 使用 `abstract` 关键字修饰的类被称为抽象类。
    - 抽象类是用来被继承的，它代表了对一类事物的共同特征的抽象，但这种抽象可能还不够具体到可以实例化一个对象。
    - **目的：** 主要是为了代码复用和定义模板，强制子类实现某些特定的行为。
2. **语法 (Syntax):**
    
    Java
    
    ```java
    public abstract class ClassName {
        // 成员变量
        // 构造方法
        // 抽象方法
        // 具体方法
    }
    ```
    
3. **特点 (Characteristics):**
    
    - **不能被实例化 (Cannot be Instantiated):**
        - 你不能使用 `new` 关键字直接创建抽象类的对象。例如：`AbstractClass obj = new AbstractClass();` 这是错误的。
        - 如果尝试实例化抽象类，编译器会报错。
        - **原因：** 抽象类可能包含抽象方法（没有具体实现的方法），如果允许实例化，那么调用这些抽象方法时将无法确定执行哪个具体实现。
    - **可以包含抽象方法 (May Contain Abstract Methods):**
        - **抽象方法：** 使用 `abstract` 关键字修饰，并且没有方法体（即没有 `{}` 代码块），以分号 `;` 结尾。
        - **作用：** 抽象方法定义了子类必须实现的行为规范。子类继承抽象类后，必须重写（实现）父类中所有的抽象方法，除非子类自己也是一个抽象类。
        - 语法：`public abstract returnType methodName(parameters);`
    - **也可以包含具体方法 (May Contain Concrete Methods):**
        - 抽象类可以拥有普通的方法（有方法体，提供具体实现）。
        - 这些具体方法可以被子类继承和直接使用，也可以被子类重写。
        - **作用：** 提供通用的功能，减少子类代码的重复。
    - **可以包含成员变量 (May Contain Member Variables):**
        - 抽象类可以有各种类型的成员变量（实例变量、静态变量）。
        - 这些成员变量的访问修饰符规则与普通类相同。
    - **可以包含构造方法 (May Contain Constructors):**
        - 抽象类可以有构造方法。
        - 这些构造方法不能用于创建抽象类的实例（因为抽象类不能实例化）。
        - **主要用途：** 在子类创建对象时，子类的构造方法会隐式或显式地调用父类（抽象类）的构造方法，用于初始化从父类继承来的成员变量。
        - 如果抽象类没有提供构造方法，编译器会自动添加一个默认的无参构造方法。
    - **子类的约束 (Subclass Constraints):**
        - 如果一个非抽象子类继承了一个抽象类，那么该子类必须实现（重写）父抽象类中所有的抽象方法。
        - 如果子类没有实现父抽象类中所有的抽象方法，那么该子类也必须声明为抽象类。
    - **`abstract` 关键字不能与某些关键字同时使用：**
        - `private`: 抽象方法需要被子类重写，而 `private` 方法在子类中不可见，无法重写。
        - `static`: 抽象方法依赖于对象实例去实现，而 `static` 方法属于类本身，不依赖对象。
        - `final`: `final` 方法不能被子类重写，而抽象方法恰恰需要被子类重写。 `final` 类不能被继承，而抽象类存在的意义就是被继承。
4. **作用与使用场景 (Purpose and Use Cases):**
    
    - **代码复用：** 将子类共有的属性和方法提取到抽象父类中，子类可以直接继承使用，避免代码冗余。
    - **定义模板方法模式 (Template Method Pattern):** 抽象类可以定义一个算法的骨架（模板方法，通常是具体方法），而将算法中的某些可变步骤延迟到子类中实现（通过抽象方法）。
    - **强制规范：** 通过抽象方法，强制子类必须实现某些特定的功能，确保子类具有某些必要的行为。
    - **表示 "is-a" 的层级关系：** 当多个相关的类具有共同的特征和行为，但某些行为的具体实现方式不同时，可以将这些共同点抽象成一个抽象类。
    - **作为一种中间层：** 在复杂的继承体系中，抽象类可以作为连接具体类和顶层接口或类的桥梁。

### Java 抽象类示例

**场景：** 我们需要定义不同形状（Shape），所有形状都有计算面积 (area) 和周长 (perimeter) 的功能，但具体计算方式不同。同时，所有形状都可以有一个名字 (name)。

Java

```java
// 1. 定义抽象类 Shape
abstract class Shape {
    protected String name; // 形状的名称，子类可以访问

    // 构造方法，用于子类初始化名称
    public Shape(String name) {
        this.name = name;
    }

    // 抽象方法：计算面积，没有方法体，由子类实现
    public abstract double calculateArea();

    // 抽象方法：计算周长，没有方法体，由子类实现
    public abstract double calculatePerimeter();

    // 具体方法：获取形状名称，所有子类共享这个实现
    public String getName() {
        return name;
    }

    // 具体方法：显示形状信息 (模板方法的一种简单体现)
    public void displayInfo() {
        System.out.println("形状名称: " + getName());
        System.out.println("面积: " + calculateArea());
        System.out.println("周长: " + calculatePerimeter());
    }
}

// 2. 定义具体子类 Circle (圆形)
class Circle extends Shape {
    private double radius;

    public Circle(String name, double radius) {
        super(name); // 调用父类 Shape 的构造方法
        this.radius = radius;
    }

    // 实现父类的抽象方法 calculateArea
    @Override
    public double calculateArea() {
        return Math.PI * radius * radius;
    }

    // 实现父类的抽象方法 calculatePerimeter
    @Override
    public double calculatePerimeter() {
        return 2 * Math.PI * radius;
    }

    // Circle 特有的方法
    public double getRadius() {
        return radius;
    }
}

// 3. 定义具体子类 Rectangle (矩形)
class Rectangle extends Shape {
    private double width;
    private double height;

    public Rectangle(String name, double width, double height) {
        super(name);
        this.width = width;
        this.height = height;
    }

    @Override
    public double calculateArea() {
        return width * height;
    }

    @Override
    public double calculatePerimeter() {
        return 2 * (width + height);
    }
}

// 4. 定义抽象子类 (如果某个子类还不能完全实现所有抽象方法)
abstract class Polygon extends Shape { // 多边形也是一种形状，但计算方式可能更复杂，这里假设它仍是抽象的
    public Polygon(String name) {
        super(name);
    }

    // Polygon 可能有自己的一些通用方法或属性
    public abstract int getNumberOfSides(); // 例如，多边形有边数
}

// 5. 测试类
public class AbstractClassDemo {
    public static void main(String[] args) {
        // Shape myShape = new Shape("Generic Shape"); // 错误！不能实例化抽象类

        Shape circle = new Circle("我的圆形", 5.0);
        Shape rectangle = new Rectangle("我的矩形", 4.0, 6.0);

        System.out.println("--- 圆形信息 ---");
        circle.displayInfo(); // 调用 Shape 类中的 displayInfo，内部会调用 Circle 重写的 calculateArea 和 calculatePerimeter

        System.out.println("\n--- 矩形信息 ---");
        rectangle.displayInfo();

        System.out.println("\n--- 单独调用 ---");
        System.out.println(circle.getName() + " 的半径是: " + ((Circle) circle).getRadius()); // 需要向下转型才能调用子类特有方法
        System.out.println(rectangle.getName() + " 的面积是: " + rectangle.calculateArea());

        // 如果有一个 Polygon 的具体子类，例如 Triangle
        // Triangle triangle = new Triangle("我的三角形", 3, 4, 5);
        // triangle.displayInfo();
        // System.out.println("边数: " + triangle.getNumberOfSides());
    }
}
```

**示例解释：**

1. `Shape` 类是抽象类，它定义了所有形状共有的属性 `name` 和行为 `calculateArea()`、`calculatePerimeter()`、`getName()`、`displayInfo()`。
2. `calculateArea()` 和 `calculatePerimeter()` 是抽象方法，因为不同形状的计算方式不同，`Shape` 类无法提供统一实现，需要子类去具体实现。
3. `getName()` 和 `displayInfo()` 是具体方法，所有形状获取名称和展示信息的逻辑是相似的，可以直接在父类中实现，子类共享。
4. `Circle` 和 `Rectangle` 是 `Shape` 的具体子类，它们都继承了 `Shape`，并实现了 `Shape` 中的所有抽象方法。它们也有自己的构造方法，并通过 `super()` 调用父类的构造方法。
5. `Polygon` 是一个抽象子类的例子，它继承了 `Shape` 但自身也是抽象的，可能因为它还有一些未具体化的行为（如 `getNumberOfSides()`）。如果一个类继承了 `Polygon`，则需要实现 `calculateArea()`, `calculatePerimeter()`以及 `getNumberOfSides()`。
6. 在 `main` 方法中，我们不能直接创建 `Shape` 的实例，但可以创建 `Circle` 和 `Rectangle` 的实例，并将它们赋值给 `Shape` 类型的引用（向上转型）。通过父类引用调用被子类重写的方法时，会发生动态绑定，执行子类的具体实现。

这个例子清晰地展示了抽象类的核心作用：提供一个共同的基类，定义共享的结构和必须被子类实现的行为规范，同时允许包含一些通用的具体实现。
