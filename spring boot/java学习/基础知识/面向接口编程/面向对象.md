Java 是一门纯粹的面向对象编程（Object-Oriented Programming, OOP）语言，其核心思想是将现实世界中的事物抽象为程序中的对象，通过对象之间的交互来完成程序的功能。

**Java 面向对象的核心概念：**

1. **类 (Class) 和对象 (Object)：**
    
    - **类：** 类是对象的蓝图或模板，它定义了一类事物所共有的属性（成员变量）和行为（成员方法）。它是一种抽象的数据类型。
    - **对象：** 对象是类的实例（Instance）。它是实际存在的、具体的个体，拥有类所定义的属性和行为。程序中操作的实体就是对象。
    - **重点：** <mark style="background: #FFF3A3A6;">理解类和对象的关系是学习面向对象的基础。类是抽象的定义，对象是具体的实体。</mark>
2. **封装 (Encapsulation)：**

	下面是 Java 中 `public`, `private`, `default` (默认，即不写任何访问修饰符) 和 `protected` 四种访问修饰符的访问域（也称为可见性或作用范围）：

| 访问修饰符       | 同一个类 (Same Class) | 同一个包 (Same Package) - 子类 | 同一个包 (Same Package) - 非子类 | 不同包 (Different Package) - 子类 | 不同包 (Different Package) - 非子类 |
| :---------- | :---------------- | :----------------------- | :------------------------ | :--------------------------- | :---------------------------- |
| `public`    | ✅                 | ✅                        | ✅                         | ✅                            | ✅                             |
| `protected` | ✅                 | ✅                        | ✅                         | ✅ (通过继承访问)                   | ❌                             |
| `default`   | ✅                 | ✅                        | ✅                         | ❌                            | ❌                             |
| `private`   | ✅                 | ❌                        | ❌                         | ❌                            | ❌                             |

- **概念：** 将对象的状态信息（属性）和行为（方法）封装在一起，形成一个不可分割的独立单位。同时，尽可能隐藏对象的内部细节，只对外提供公共的访问方式（通过方法）。
    - **实现方式：** 使用访问修饰符（`private`, `protected`, `public`, 默认）来控制属性和方法的可见性。通常将属性设为 `private`，通过 `public` 的 `getter` 和 `setter` 方法来访问和修改属性。
    - **优点：** 提高代码的安全性、可维护性和模块化。
    - **重点：** 封装是面向对象的重要特性之一，它强调的是“隐藏实现，暴露接口”。
3. **继承 (Inheritance)：**
    [[spring boot/java学习/基础知识/面向接口编程/多态/向上转型，向下转型，动态绑定|向上转型，向下转型，动态绑定]]
    [[is-a]]
    [[子类构造方法调用流程]] <mark style="background: #FFF3A3A6;">期间父类并不会进行实例化</mark>
    - **概念：** 允许一个类（子类/派生类）继承另一个类（父类/基类）的属性和方法。子类可以拥有父类的所有非私有成员，并且可以添加自己特有的属性和方法，也可以重写（Override）父类的方法。
    - **实现方式：** 使用 `extends` 关键字。
    - **优点：** 实现代码复用，提高代码的可扩展性和可维护性。形成类的层次结构，体现“is-a”的关系。
    - **缺点：** 增加了类之间的耦合度。Java 是单继承的，即一个类只能直接继承一个父类，但可以通过接口实现多重继承的效果。
    - **重点：** 继承是代码复用的重要手段，理解继承的机制、`super` 关键字的使用以及方法的重写至关重要。
4. **多态 (Polymorphism)：**
    <mark style="background: #FFF3A3A6;">super.方法(),调用父类中的方法，即使子类进行了@Override重写，也不会覆盖父类中的方法，而是会进行隐藏。</mark>
    - **概念：** <mark style="background: #FFF3A3A6;">同一种类型的引用变量，在指向不同类的对象时，调用同一个方法会表现出不同的行为。</mark>或者说，允许不同类的对象对同一消息做出响应。
    - **实现方式：**
        - **方法的重写 (Override)：** <mark style="background: #FFF3A3A6;">子类重新定义父类中已有的方法</mark>。
        - **方法的重载 (Overload)：** <mark style="background: #FFF3A3A6;">在同一个类中，方法名相同，但参数列表不同</mark>（参数个数、类型或顺序不同）。
        - **对象类型的转换 (Casting)：** 向上转型（子类对象赋值给父类引用）和向下转型（父类引用强制转换为子类引用，需要使用 `instanceof` 进行判断以避免 `ClassCastException`）。
    - **优点：** 增强程序的灵活性和可扩展性，降低代码的耦合度。
    - **重点：** 多态是面向对象最具魅力的特性之一。理解多态的三个必要条件（<mark style="background: #FFF3A3A6;">继承、重写、父类引用指向子类对象</mark>）以及其带来的好处是核心。`instanceof` 运算符在多态中也扮演重要角色。

**其他重要的 Java 面向对象知识点：**

5. **抽象类 (Abstract Class)：**
    [[抽象类]]
    抽象类可能包含抽象方法（没有具体实现的方法），如果允许实例化，那么调<mark style="background: #FFF3A3A6;">用这些抽象方法时将无法确定执行哪个具体实现</mark>。
    - **概念：** 使用 `abstract` 关键字修饰的类。抽象类不能被实例化，主要用于被子类继承。抽象类中可以包含抽象方法和非抽象方法。（抽象方法）
    - **抽象方法：** 使用 `abstract` 关键字修饰且没有方法体的方法，必须在子类中被重写。
    - **作用：** 定义一种规范或模板，强制子类实现某些特定的行为。
    - **重点：** 抽象类体现了“is-a”关系，并且可以提供部分实现。
6. **接口 (Interface)：**
    [[接口例子]]
    - **概念：** 使用 `interface` 关键字定义的引用类型。接口是抽象方法和常量值的集合。在 Java 8 之后，接口可以包含默认方法 (default methods) 和静态方法 (static methods)。
    - **作用：** 定义一种规范或契约，规定了实现该接口的类必须实现哪些方法。实现接口使用 `implements` 关键字。一个类可以实现多个接口。
    - **优点：** 实现多重继承的效果，降低耦合度，定义标准。
    - **重点：** 接口体现了“has-a”或“can-do”的关系，是实现多态和定义规范的重要手段。理解接口与抽象类的区别和联系。
7. **构造方法 (Constructor)：**
    
    - **概念：** 类中一个特殊的方法，用于<mark style="background: #FFF3A3A6;">创建和初始化对象</mark>。方法名与类名相同，没有返回类型（也不能用 `void`）。
    - **作用：** 在创建对象时，对对象的属性进行初始化。
    - **分类：** 无参构造方法和有参构造方法。如果类中没有显式定义构造方法，Java 会提供一个默认的无参构造方法。如果定义了有参构造方法，则不会再提供默认的无参构造方法（除非显式定义）。
    - **重点：** 构造方法是对象创建的入口，理解其作用和调用时机。
8. **`this` 关键字：**
    <mark style="background: #FFF3A3A6;">super指的是当前类的直接父类</mark>
    - **概念：** 指向当前对象的引用。
    - **用途：**
        - 在方法中访问当前对象的成员变量（当成员变量与局部变量同名时）。
        - 在构造方法中调用同一个类中的其他构造方法（必须是构造方法的第一条语句）。
        - <mark style="background: #FFF3A3A6;">作为方法的返回值，返回当前对象。</mark>
    - **重点：** `this` 是理解对象内部方法如何引用对象自身属性和方法的关键。
9. **`static` 关键字：**
    
    - **概念：** 可以用来修饰成员变量、成员方法、代码块和内部类。
    - **静态成员变量 (类变量)：** <mark style="background: #FFF3A3A6;">被类的所有对象共享，在内存中只有一份拷贝。可以通过类名直接访问，也可以通过对象名访问（不推荐）。</mark>
    - **静态成员方法 (类方法)：** 属于类本身，不依赖于任何对象。可以通过类名直接调用。静态方法中不能直接访问非静态成员（成员变量和成员方法），也不能使用 `this` 和 `super` 关键字。
    - **静态代码块：** 在类加载时执行，且只执行一次，通常用于初始化静态资源。
    - **重点：** `static` 成员不属于某个特定对象，而是属于整个类。
10. **`final` 关键字：**
    
    - **概念：** 可以用来修饰类、方法和变量。
    - **`final` 类：** 不能被继承。
    - **`final` 方法：** 不能被子类重写。
    - **`final` 变量 (常量)：**
        - **基本数据类型变量：** 其值一旦初始化后不能被改变。
        - **引用数据类型变量：** 其引用地址一旦初始化后不能被改变，但该引用指向的对象的内容是可以改变的。
    - **重点：** `final` 用于表示“最终的”、“不可改变的”。
11. **对象的创建和内存管理：**
    
    - **对象创建过程：**
        1. 类加载检查。
        2. 为新生对象分配内存。
        3. 将分配到的内存空间都初始化为零值。
        4. 设置对象头。
        5. 执行 `<init>` 方法（构造方法）。
    - **内存区域：** 主要关注堆 (Heap) 和栈 (Stack)。对象实例存储在堆中，对象的引用存储在栈中。
    - **垃圾回收 (Garbage Collection, GC)：** Java 自动管理内存，通过垃圾回收器回收不再被引用的对象所占用的内存。程序员不需要手动释放内存，但理解 GC 的基本原理有助于编写更高效的代码。
    - **重点：** 理解对象是如何被创建、存储以及如何被回收的，是编写健壮 Java 程序的基础。

**总结非常重要的重点：**

- **封装、继承、多态：** 这是面向对象编程的三大基石，是理解和运用 OOP 的核心。务必深刻理解它们的概念、实现方式以及带来的好处。
- **类和对象：** 一切皆对象，理解类是对象的模板，对象是类的实例，这是 OOP 的出发点。
- **抽象类和接口：** 理解它们在定义规范、实现多态和代码复用方面的不同作用和应用场景。
- **构造方法：** 对象初始化的关键。
- **`this` 和 `static` 关键字：** 理解它们在不同上下文中的含义和用法。

掌握这些核心概念和知识点，将为你学习和使用 Java 进行面向对象编程打下坚实的基础。建议通过编写实际的代码来加深理解和熟练运用这些知识。
