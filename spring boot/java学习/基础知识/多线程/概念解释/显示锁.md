用“手动开关锁”的比喻，彻底搞懂 `ReentrantLock` 的用法！🔐

---

### **1. ReentrantLock 基础用法**
#### **核心步骤**：
1. **创建锁**：实例级或静态级
2. **手动加锁**：`lock()`
3. **释放锁**：`unlock()`（必须放在 `finally` 块中）
4. **可选功能**：可中断、超时、公平锁等

---

### **场景一：在实例方法中使用**
```java
public class BankAccount {
    private final ReentrantLock lock = new ReentrantLock(); // 实例锁
    private int balance = 0;

    public void deposit(int amount) {
        lock.lock(); // 加锁
        try {
            balance += amount;
        } finally {
            lock.unlock(); // 必须释放！
        }
    }
}
```
**关键点**：
- 每个实例有自己的锁（`lock`），保护实例变量
- 和 `synchronized` 实例方法效果相同，但需要手动管理锁

---

### **场景二：在静态方法中使用**
```java
public class Bank {
    private static final ReentrantLock staticLock = new ReentrantLock(); // 静态锁
    private static int totalMoney = 0;

    public static void addToTotal(int amount) {
        staticLock.lock();
        try {
            totalMoney += amount;
        } finally {
            staticLock.unlock();
        }
    }
}
```
**关键点**：
- 静态锁保护静态变量，所有实例调用静态方法时共享同一把锁
- 类似 `synchronized` 静态方法的效果

---

### **对比 synchronized 和 ReentrantLock**
| **特性**    | **synchronized**       | **ReentrantLock**                        |
| --------- | ---------------------- | ---------------------------------------- |
| **锁类型**   | 内置锁（自动获取/释放）           | 显式锁（手动加锁/释放）                             |
| **锁的公平性** | 非公平锁（默认）               | 可设为公平锁（构造函数传 `true`）                     |
| **可中断**   | ❌ 不可中断等待               | ✅ `lockInterruptibly()`                  |
| **超时获取**  | ❌ 不支持                  | ✅ `tryLock(long timeout, TimeUnit unit)` |
| **条件变量**  | 一个对象只能关联一个条件（`wait()`） | 可创建多个 `Condition` 对象（更灵活）                |
| **锁的细粒度** | 锁对象固定（this 或类对象）       | 可自由选择锁对象                                 |
| **代码复杂度** | 简单（自动管理）               | 复杂（需手动管理，容易忘记释放）                         |

---

### **ReentrantLock 的高级功能**
#### **1. 尝试获取锁（避免死锁）**
```java
if (lock.tryLock(3, TimeUnit.SECONDS)) { // 最多等3秒
    try {
        // 获取到锁后的操作
    } finally {
        lock.unlock();
    }
} else {
    System.out.println("等不及了，去做其他事");
}
```

#### **2. 公平锁（先到先得）**
```java
ReentrantLock fairLock = new ReentrantLock(true); // 公平锁
```

#### **3. 条件变量（精准唤醒）**
```java
Condition condition = lock.newCondition();

// 等待条件
condition.await();

// 唤醒等待线程
condition.signalAll();
```

---

### **最佳实践与注意事项**
1. **必须释放锁**：`unlock()` 要放在 `finally` 块中，防止异常导致锁未释放
2. **避免锁泄露**：确保所有代码路径都有解锁逻辑
3. **不要混合使用**：同一个锁范围的逻辑不要混用 `synchronized` 和 `ReentrantLock`
4. **性能考量**：高竞争场景下，`ReentrantLock` 的性能通常优于 `synchronized`
