### **一、线程的4种创建方式**
## Q:线程就是Runnable接口，Callable接口和Thread接口的实现吗

A :[[线程就是Runnable接口，Callable接口和Thread接口的实现吗]]
<mark style="background: #FFF3A3A6;">简答：Thread创建线程，而Runnable和Callable是任务单 。而且Callable是要和线程池结合使用，Runnable可用于简单线程实例。</mark>
## Q:我看到Runnable可直接作为Thread构造方法的参数，Callable不可以吗？
[[我看到Runnable可直接作为Thread构造方法的参数，Callable不可以吗？]]
简答：不可以
#### 1. 继承 `Thread` 类（最基础）
Thread继承Runnable函数式接口，必须实现 run ( ) 方法
```java
public class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println("我是线程：" + Thread.currentThread().getName());
    }

    public static void main(String[] args) {
        MyThread t1 = new MyThread();
        t1.start(); // 调用start()启动线程（不是run()）
    }
}
```

#### 2. 实现 `Runnable` 接口（更灵活）
```java
public class MyRunnable implements Runnable {
    @Override
    public void run() {
        System.out.println("我是线程：" + Thread.currentThread().getName());
    }

    public static void main(String[] args) {
        Thread t2 = new Thread(new MyRunnable());
        t2.start();
    }
}
```

#### 3. 实现 `Callable`（带返回值）
这个Callable接口和线程池配套使用，不能传给Thread作为构造函数参数，用于处理较复杂的情况（因为可以返回结果和抛出异常）
[[Callable示例代码解释]]
名词解释：ExecutorService，Executors, Future ,submit shutdown
[[spring boot/java学习/基础知识/多线程/概念解释/名词|名词]]
guture.get()的值是哪里来的
[[名词2]]

```java
import java.util.concurrent.*;

public class MyCallable implements Callable<String> {
    @Override
    public String call() throws Exception {
        return "线程执行完毕";
    }

    public static void main(String[] args) throws Exception {
        ExecutorService executor = Executors.newSingleThreadExecutor();
        Future<String> future = executor.submit(new MyCallable());
        System.out.println(future.get()); // 输出：线程执行完毕
        executor.shutdown();
    }
}
```

#### 4. 线程池创建（推荐！）
```java
ExecutorService pool = Executors.newFixedThreadPool(3); // 固定3个线程的池子
pool.execute(() -> System.out.println("用线程池执行任务"));
pool.shutdown();
```

### **二、线程安全问题（卖票经典案例）**
> **问题**：多个线程卖票时出现重复票、超卖  
> **原因**：线程竞争导致数据不一致

```java
public class TicketProblem {
    private int tickets = 100;

    public void sell() {
        while (tickets > 0) {
            try {
                Thread.sleep(10); // 模拟延时
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            System.out.println("卖出第" + tickets-- + "张票");
        }
    }

    public static void main(String[] args) {
        TicketProblem problem = new TicketProblem();
        new Thread(problem::sell, "窗口1").start();
        new Thread(problem::sell, "窗口2").start();
    }
}
```
**运行结果**：可能出现 `tickets` 负数或重复票！

### **三、解决线程安全**
#### 1. 同步代码块（`synchronized`）
```java
public synchronized void sell() { // 锁住整个方法
    while (tickets > 0) {
        // ... 代码同上 ...
    }
}
```

#### 2. 显式锁（`ReentrantLock`）
```java
private final ReentrantLock lock = new ReentrantLock();

public void sell() {
    lock.lock();
    try {
        while (tickets > 0) {
            // ... 代码同上 ...
        }
    } finally {
        lock.unlock();
    }
}
```

#### 3. 原子类（`AtomicInteger`）
```java
private AtomicInteger tickets = new AtomicInteger(100);

public void sell() {
    while (tickets.get() > 0) {
        int current = tickets.get();
        if (current > 0) {
            System.out.println("卖出第" + tickets.decrementAndGet() + "张票");
        }
    }
}
```

### **四、线程间通信**
> 使用 `wait()` 和 `notify()`（生产者消费者模型）
```java
public class ProducerConsumer {
    private Queue<Integer> queue = new LinkedList<>();
    private int maxSize = 5;

    public synchronized void produce(int value) throws InterruptedException {
        while (queue.size() == maxSize) {
            wait(); // 队列满时等待
        }
        queue.add(value);
        System.out.println("生产：" + value);
        notify(); // 唤醒消费者
    }

    public synchronized int consume() throws InterruptedException {
        while (queue.isEmpty()) {
            wait(); // 队列空时等待
        }
        int value = queue.poll();
        System.out.println("消费：" + value);
        notify(); // 唤醒生产者
        return value;
    }
}
```

### **五、线程池的核心参数**
```java
ThreadPoolExecutor pool = new ThreadPoolExecutor(
    5, // 核心线程数
    10, // 最大线程数
    60L, // 空闲线程存活时间
    TimeUnit.SECONDS, // 时间单位
    new LinkedBlockingQueue<>(100), // 任务队列
    Executors.defaultThreadFactory(), // 线程工厂
    new ThreadPoolExecutor.CallerRunsPolicy() // 拒绝策略
);
```

### **六、常用并发工具类**
#### 1. **CountDownLatch**（倒计时门栓）
```java
CountDownLatch latch = new CountDownLatch(3);
new Thread(() -> { 
    // 任务1
    latch.countDown(); 
}).start();
// ...其他两个线程
latch.await(); // 等待所有线程完成
System.out.println("所有任务完成");
```

#### 2. **CyclicBarrier**（循环栅栏）
```java
CyclicBarrier barrier = new CyclicBarrier(3, () -> {
    System.out.println("所有线程到达栅栏");
});
for (int i = 0; i < 3; i++) {
    new Thread(() -> {
        try {
            barrier.await(); // 等待其他线程
        } catch (Exception e) {
            e.printStackTrace();
        }
    }).start();
}
```

### **注意事项**
1. **避免死锁**：不要嵌套锁，按顺序获取锁
2. **不要用`stop()`强制停止线程**：可能导致数据不一致
3. **推荐用线程池**：减少线程创建销毁的开销
