好的，我们来创建一个例子，这个例子会尝试将线程 (Thread)、内部对象锁 (Intrinsic Lock via `synchronized`)、以及通过内部锁实现的条件等待 (`wait()`, `notifyAll()`) 和隐式的等待集 (Wait Set) 串联起来。

这个例子将模拟一个简单的场景：一个**信箱 (Mailbox)**，一个**写信者 (Writer)** 线程往信箱里放信，一个**读信者 (Reader)** 线程从信箱里取信。

- **共享资源：** `Mailbox` 对象。
- **内部对象锁：** `Mailbox` 对象本身的锁。
- **条件：**
    - 写信者：信箱为空时才能放信。如果信箱里已经有信了，写信者需要等待。
    - 读信者：信箱有信时才能取信。如果信箱是空的，读信者需要等待。
- **等待集：** 与 `Mailbox` 对象关联的隐式等待集，用于存放等待的写信者或读信者线程。

Java

```java
// Mailbox.java (共享资源)
class Mailbox {
    private String message;
    private boolean hasMessage; // true 表示信箱里有信，false 表示信箱为空

    // 使用内部对象锁 (this Mailbox 实例的锁)
    public synchronized void putMessage(String message, String writerName) throws InterruptedException {
        // 条件检查：如果信箱里已经有信了，写信者必须等待
        while (hasMessage) {
            System.out.println(writerName + ": 信箱已满 (内容: \"" + this.message + "\")，等待...");
            wait(); // 释放 Mailbox 对象的锁，并进入该对象的等待集
            // 被唤醒后，会重新获取锁，并再次检查 while 条件
            System.out.println(writerName + ": 被唤醒，重新检查信箱...");
        }

        // 信箱为空，可以放信
        this.message = message;
        this.hasMessage = true;
        System.out.println(writerName + ": 成功放入信件 -> \"" + message + "\"");

        // 通知所有可能在等待的读信者线程 (也可能是其他写信者，但此处主要针对读信者)
        System.out.println(writerName + ": 通知所有等待者...");
        notifyAll(); // 唤醒在此对象监视器上等待的所有线程
    }

    // 使用内部对象锁 (this Mailbox 实例的锁)
    public synchronized String getMessage(String readerName) throws InterruptedException {
        // 条件检查：如果信箱是空的，读信者必须等待
        while (!hasMessage) {
            System.out.println(readerName + ": 信箱为空，等待...");
            wait(); // 释放 Mailbox 对象的锁，并进入该对象的等待集
            // 被唤醒后，会重新获取锁，并再次检查 while 条件
            System.out.println(readerName + ": 被唤醒，重新检查信箱...");
        }

        // 信箱有信，可以取信
        String retrievedMessage = this.message;
        this.hasMessage = false;
        System.out.println(readerName + ": 成功取出信件 <- \"" + retrievedMessage + "\"");

        // 通知所有可能在等待的写信者线程 (也可能是其他读信者，但此处主要针对写信者)
        System.out.println(readerName + ": 通知所有等待者...");
        notifyAll(); // 唤醒在此对象监视器上等待的所有线程
        return retrievedMessage;
    }
}

// Writer.java (生产者线程)
class Writer implements Runnable {
    private final Mailbox mailbox;
    private final String name;
    private int messageCount = 0;

    public Writer(Mailbox mailbox, String name) {
        this.mailbox = mailbox;
        this.name = name;
    }

    @Override
    public void run() {
        try {
            for (int i = 0; i < 5; i++) { // 每个写信者尝试写5封信
                String messageToSend = "来自 " + name + " 的第 " + (++messageCount) + " 封信";
                mailbox.putMessage(messageToSend, name);
                Thread.sleep((long) (Math.random() * 1000)); // 模拟写信的耗时
            }
            System.out.println("--- " + name + " 写完了所有信件 ---");
        } catch (InterruptedException e) {
            System.out.println(name + " 被中断了.");
            Thread.currentThread().interrupt();
        }
    }
}

// Reader.java (消费者线程)
class Reader implements Runnable {
    private final Mailbox mailbox;
    private final String name;

    public Reader(Mailbox mailbox, String name) {
        this.mailbox = mailbox;
        this.name = name;
    }

    @Override
    public void run() {
        try {
            for (int i = 0; i < 5; i++) { // 每个读信者尝试读5封信
                String receivedMessage = mailbox.getMessage(name);
                Thread.sleep((long) (Math.random() * 1500)); // 模拟读信和处理的耗时
            }
            System.out.println("--- " + name + " 读完了分配的信件 ---");
        } catch (InterruptedException e) {
            System.out.println(name + " 被中断了.");
            Thread.currentThread().interrupt();
        }
    }
}

// Main.java (演示程序入口)
public class MailboxDemo {
    public static void main(String[] args) {
        System.out.println("程序开始，当前时间：" + java.time.LocalDateTime.now());
        Mailbox sharedMailbox = new Mailbox();

        // 创建线程
        Thread writer1 = new Thread(new Writer(sharedMailbox, "写信者-小明"));
        Thread writer2 = new Thread(new Writer(sharedMailbox, "写信者-小红"));
        Thread reader1 = new Thread(new Reader(sharedMailbox, "读信者-大雄"));
        Thread reader2 = new Thread(new Reader(sharedMailbox, "读信者-静香"));

        System.out.println("启动线程...");

        // 启动线程
        writer1.start();
        reader1.start();
        writer2.start();
        reader2.start();

        // 等待所有线程执行完毕 (可选，用于观察完整流程)
        try {
            writer1.join();
            writer2.join();
            reader1.join();
            reader2.join();
        } catch (InterruptedException e) {
            System.err.println("主线程等待时被中断: " + e.getMessage());
            Thread.currentThread().interrupt();
        }

        System.out.println("所有线程执行完毕，程序结束。");
    }
}
```

**代码中体现的知识点：**

1. **线程 (Thread):**
    
    - `Writer` 和 `Reader` 类都实现了 `Runnable` 接口，它们的 `run()` 方法定义了线程的执行逻辑。
    - 在 `MailboxDemo` 的 `main` 方法中，我们创建了 `Thread` 对象（例如 `new Thread(new Writer(...))`）并调用 `start()` 来启动它们。这些线程会并发执行。
2. **内部对象锁 (`synchronized`):**
    
    - `Mailbox` 类中的 `putMessage` 和 `getMessage` 方法都使用了 `synchronized` 关键字。
    - 这意味着任何线程在执行这两个方法之前，都必须先获得 `sharedMailbox` 对象（调用该方法的那个 `Mailbox` 实例）的**内部锁**。
    - 同一时刻，只有一个线程能够持有 `sharedMailbox` 的内部锁，因此只有一个线程能执行 `putMessage` 或 `getMessage` 中的代码，从而保证了对 `message` 和 `hasMessage` 字段的互斥访问，防止了数据竞争。
3. **条件等待 (`wait()`):**
    
    - 在 `putMessage` 中，如果 `hasMessage` 为 `true`（信箱已满），写信者线程会调用 `wait()`。
        - 调用 `wait()` 时，该线程会**释放它持有的 `sharedMailbox` 对象的内部锁**。
        - 然后，线程进入与 `sharedMailbox` 对象关联的**隐式等待集**中，并暂停执行，等待被其他线程唤醒。
    - 类似地，在 `getMessage` 中，如果 `!hasMessage` 为 `true`（信箱为空），读信者线程会调用 `wait()`，释放锁并进入等待集。
    - **注意 `while` 循环的使用：** `wait()` 必须在循环中被调用（`while (condition) { wait(); }`）。这是因为线程可能因为“虚假唤醒”(spurious wakeup) 而被唤醒，或者当多个线程被 `notifyAll()` 唤醒时，只有第一个重新获得锁的线程能改变条件，其他线程醒来后需要重新检查条件。
4. **通知 (`notifyAll()`):**
    
    - 当写信者成功放入信件后（即改变了条件，信箱从空变为有信），它调用 `notifyAll()`。
        - `notifyAll()` 会唤醒所有因调用 `sharedMailbox.wait()` 而在该对象等待集中的线程（可能是等待取信的读信者，也可能是其他等待放信的写信者）。
    - 当读信者成功取出信件后（即改变了条件，信箱从有信变为空），它也调用 `notifyAll()`，以唤醒可能在等待的写信者。
    - 被唤醒的线程会从等待集中移出，并尝试重新获取 `sharedMailbox` 对象的内部锁。只有成功获取锁的线程才能从 `wait()` 调用中返回，并继续执行（通常是再次检查 `while` 循环的条件）。
5. **等待集 (Implicit Wait Set):**
    
    - 这个例子中没有显式地看到一个名为“等待集”的集合对象。它是与每个 Java 对象（这里是 `sharedMailbox`）的监视器 (monitor) 相关联的一个内部结构。
    - 当线程调用 `sharedMailbox.wait()`，它就被放入了这个对象的等待集中。
    - `sharedMailbox.notifyAll()` 会作用于这个等待集中的线程。

**如何观察和理解输出：**

当你运行 `MailboxDemo` 时，注意观察控制台的输出。你会看到：

- 线程启动的顺序。
- 写信者和读信者尝试获取锁并执行操作。
- 当条件不满足时（例如，写信者发现信箱满了，或读信者发现信箱空了），线程会打印 "等待..." 的信息，然后调用 `wait()`。
- 当一个线程改变了条件并调用 `notifyAll()` 后，你会看到之前等待的线程打印 "被唤醒，重新检查信箱..." 的信息。
- 由于 `Thread.sleep()` 的随机性以及线程调度的不确定性，每次运行的输出顺序细节可能会有所不同，但整体的同步和协作逻辑会保持一致。你会发现，不会出现写信者在信箱满的时候继续写，或者读信者在信箱空的时候尝试读的情况。

这个例子清晰地展示了这些核心概念是如何协同工作以实现线程安全和有效的并发控制的。
