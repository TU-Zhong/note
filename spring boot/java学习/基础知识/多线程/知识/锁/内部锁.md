「内部对象锁」 (Internal Object Lock)，在 Java 中通常被称为 **固有锁 (Intrinsic Lock)** 或 **监视器锁 (Monitor Lock)**。

简单来说，**Java 中的每一个对象都自动关联着一个唯一的、内部的锁。** 你不需要显式地去创建这个锁，它随着对象的创建而自然存在。

以下是关于内部对象锁的关键点：

1. **与每个对象关联：**
    
    - 无论是你 `new` 出来的任何类的实例，还是类本身（通过 `ClassName.class` 对象），都有一个内部锁。
    - 这个锁是对象本身的一部分，是 JVM 实现的。
2. **`synchronized` 关键字：**
    [[锁静态方法和实例方法对其他线程访问该类有什么区别]]
    - 内部对象锁最常通过 `synchronized` 关键字来使用。
    - **同步方法 (Synchronized Methods):** 当你用 `synchronized` <mark style="background: #FFF3A3A6;">修饰一个实例方法时</mark>，线程在执行该方法前必须获取该方法所属**对象实例的内部锁**。
        
        
        
        ```java
        public class MyClass {
            public synchronized void synchronizedInstanceMethod() {
                // 访问共享资源
                // 此时当前线程持有 MyClass 实例的内部锁
            }
        }
        ```
        
    - **同步静态方法 (Synchronized Static Methods):** 当你用 `synchronized` <mark style="background: #FFF3A3A6;">修饰一个静态方法时</mark>，线程在执行该方法前必须获取该方法所属**类的 Class 对象的内部锁** (`MyClass.class.getLock()` 这样的概念，虽然没有这个直接的API，但锁是与 `Class` 对象关联的)。
        
        
        
        ```java
        public class MyClass {
            public static synchronized void synchronizedStaticMethod() {
                // 访问静态共享资源
                // 此时当前线程持有 MyClass.class 对象的内部锁
            }
        }
        ```
        
    - **同步代码块 (Synchronized Blocks):** <mark style="background: #FFF3A3A6;">你可以明确指定哪个对象的内部锁需要被获取。</mark>
        
        理解为要执行代码块中的代码，必须提前获得括号里面的锁
        <mark style="background: #FFF3A3A6;">synchronized关键字，隐式地获取锁，使用了这个就相当于lock.lock()估计。</mark>
        ```java
        public class MyClass {
            private final Object lockObject = new Object(); // 可以是一个专用的锁对象
        
            public void someMethod() {
                // ... 非同步代码 ...
        
                synchronized (this) { // 使用当前实例的内部锁
                    // 需要同步的代码块
                }
        
                synchronized (lockObject) { // 使用特定对象的内部锁
                    // 另一块需要同步的代码
                }
        
                synchronized (MyClass.class) { // 使用 MyClass.class 对象的内部锁 (常用于保护静态成员)
                    // 保护静态资源的同步代码块
                }
                // ... 非同步代码 ...
            }
        }
        ```
        
3. **互斥访问 (Mutual Exclusion):**
    
    - 一个内部锁在任何时候只能被一个线程持有。
    - 如果一个线程已经持有了某个对象的内部锁，其他线程如果尝试获取同一个对象的内部锁，它们将会被阻塞（进入该锁的等待队列），直到持有锁的线程释放它。
4. **可重入性 (Reentrancy):**
    
    - Java 中的内部锁是可重入的。这意味着如果一个线程已经持有了某个对象的内部锁，它可以在不释放该锁的情况下再次获取它（例如，一个同步方法调用了同一个对象上的另一个同步方法）。每次获取都会增加一个持有计数，每次释放会减少计数，只有当计数降为零时，锁才真正被释放。
5. **与 `wait()`, `notify()`, `notifyAll()` 的关系：**
    
    - `Object` 类中的 `wait()`, `notify()`, 和 `notifyAll()`<mark style="background: #FFF3A3A6;"> 方法必须在当前线程持有该对象的内部锁</mark>（<mark style="background: #FFF3A3A6;">即在 `synchronized` 方法或块中</mark>）时调用。
    - 当线程调用 `object.wait()` 时，它会释放该 `object` 的内部锁，并进入该对象的等待集 (wait set)，直到被其他线程通过 `object.notify()` 或 `object.notifyAll()` 唤醒。唤醒后，它需要重新竞争获取该对象的内部锁才能继续执行。

**总结来说，内部对象锁是 Java 提供的一种基础的、内置的同步机制，它与每个对象紧密绑定，通过 `synchronized` 关键字方便地实现线程间的互斥访问，并配合 `wait/notify` 机制实现线程间的协作。**

它的优点是使用简单直观，但相比于 `java.util.concurrent.locks.Lock` 接口（如 `ReentrantLock`），它的灵活性稍差（例如，无法中断等待锁的线程，不能尝试非阻塞地获取锁，一个锁只能关联一个条件等待集等）。尽管如此，内部对象锁仍然是 Java 并发编程中非常常用和重要的一个概念。
