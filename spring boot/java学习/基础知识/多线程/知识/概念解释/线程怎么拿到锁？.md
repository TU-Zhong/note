### **1. 锁放在哪里？**
**答案**：<mark style="background: #FFF3A3A6;">锁对象通常作为**共享资源的成员变量**，和需要保护的数据放在一起！</mark>

#### **场景示例**：
假设有一个银行账户类 `BankAccount`，需要保护余额（`balance`）不被多线程同时修改：

```java
public class BankAccount {
    private int balance = 0;
    private final Object lock = new Object(); // ✅ 锁对象是账户的成员变量
    
    public void deposit(int amount) {
        synchronized(lock) { // 用账户自己的锁保护余额
            balance += amount;
        }
    }
    
    public void withdraw(int amount) {
        synchronized(lock) {
            if (balance >= amount) {
                balance -= amount;
            }
        }
    }
}
```
**关键点**：
- 每个 `BankAccount` 实例有自己的锁（`lock`），不同账户的操作互不影响。
- 锁对象推荐用 `private final`，避免被外部篡改。

### **2. 线程如何拿到锁？**
**分步解释**：
1. <mark style="background: #FFF3A3A6;">**线程遇到 `synchronized`**：</mark>
   ```java
   synchronized(lock) { // 准备获取锁
       // 代码
   }
   ```
2. <mark style="background: #FFF3A3A6;">**尝试获取锁**：</mark>
   - <mark style="background: #FFF3A3A6;">**锁空闲** → 线程立即获取锁，进入同步块执行代码。</mark>
   - <mark style="background: #FFF3A3A6;">**锁被占用** → 线程进入阻塞状态，直到锁被释放。
</mark>
1. **释放锁**：
   - <mark style="background: #FFF3A3A6;">线程执行完同步块 → **自动释放锁**。</mark>
   - <mark style="background: #FFF3A3A6;">线程在同步块内抛出异常 → **自动释放锁**。</mark>

---

### **比喻理解**
把锁想象成**保险箱的密码锁**：
- **锁的位置**：每个保险箱（`BankAccount` 实例）自带一把锁（`lock`）。
- **线程拿锁**：柜员（线程）想操作保险箱，必须输入正确密码（获取锁）才能打开。

### **代码验证锁获取流程**
```java
public class LockDemo {
    private static final Object lock = new Object();

    public static void main(String[] args) {
        // 线程1：先获取锁
        new Thread(() -> {
            synchronized(lock) {
                System.out.println("线程1拿到锁，开始工作...");
                try { Thread.sleep(3000); } catch (Exception e) {}
                System.out.println("线程1释放锁");
            }
        }).start();

        // 线程2：后尝试获取锁
        new Thread(() -> {
            System.out.println("线程2等待锁...");
            synchronized(lock) {
                System.out.println("线程2拿到锁！");
            }
        }).start();
    }
}
```
**输出结果**：
```
线程1拿到锁，开始工作...
线程2等待锁...
...3秒后...
线程1释放锁
线程2拿到锁！
```

### **锁的底层机制（不深入JVM）**
- **监视器（Monitor）**：每个对象关联一个监视器，包含锁状态、等待队列等信息。
- **锁记录（Lock Record）**：线程获取锁时，JVM在栈帧中记录锁信息。
- **可重入性**：同一线程可多次获取同一把锁（如递归调用同步方法）。

---

### **最佳实践**
| **场景**                   | **推荐做法**                                 |
|---------------------------|--------------------------------------------|
| **保护实例变量**           | 用实例内的私有锁对象（`private final Object lock`）|
| **保护静态变量**           | 用类的Class对象或静态锁对象                   |
| **避免死锁**               | 按固定顺序获取多把锁（如先锁A，再锁B）          |
| **高并发场景**             | 优先用`ReentrantLock`（可设置公平锁、超时等）  |
