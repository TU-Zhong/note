

用“打电话等厕所”的比喻，秒懂可中断锁！ 📞🚽

---

### **1. 什么是锁的可中断性？**
**答案**：线程在等待锁的过程中，**可以被打断**（通过调用 `interrupt()`），无需死等到底！

#### **对比普通锁（不可中断）**：
- **普通锁**：线程会一直阻塞，直到拿到锁（像在厕所门口死等）
- **可中断锁**：如果等太久，别人可以打电话叫你离开（中断等待）

---

### **2. `lockInterruptibly()` 的作用**
- **方法签名**：`void lockInterruptibly() throws InterruptedException`
- **使用场景**：当线程长时间等待锁时，允许外部强制中断等待
- **核心优势**：避免线程无限期阻塞，提升程序健壮性

---

### **3. 代码示例**
#### **场景**：线程等待锁时被中断
```java
ReentrantLock lock = new ReentrantLock();

Thread thread = new Thread(() -> {
    try {
        System.out.println("线程尝试获取锁...");
        lock.lockInterruptibly(); // ✅ 可中断式获取锁
        try {
            System.out.println("线程成功获取锁！");
        } finally {
            lock.unlock();
        }
    } catch (InterruptedException e) {
        System.out.println("线程被中断，放弃等待锁！");
    }
});

lock.lock(); // 主线程先获取锁，不让子线程拿到
thread.start();
Thread.sleep(2000); // 让子线程等待2秒
thread.interrupt(); // 中断子线程的等待
```

**输出**：
```
线程尝试获取锁...
线程被中断，放弃等待锁！
```

---

### **4. 对比不可中断锁（`lock()`）**
```java
lock.lock(); // 普通获取锁（不可中断）
try {
    // ...
} finally {
    lock.unlock();
}
```
**行为**：
- 调用 `thread.interrupt()` **无法中断**正在等待锁的线程
- 线程必须等到锁释放才能继续

---

### **关键点总结**
| **特性**          | **可中断锁（`lockInterruptibly()`）** | **不可中断锁（`lock()`）** |
|-------------------|---------------------------------------|---------------------------|
| **响应中断**       | ✅ 抛出 `InterruptedException`        | ❌ 忽略中断               |
| **代码复杂度**     | 需要处理中断异常                      | 简单                      |
| **适用场景**       | 需要取消长时间等待的任务              | 确定锁很快释放的简单场景   |
| **风险**           | 可能因中断导致任务未完成              | 可能死锁或长时间阻塞      |

---

### **实际应用场景**
#### **场景1：任务取消**
用户提交任务后，等待获取锁处理数据。当用户点击取消时，调用 `interrupt()` 中断等待。

#### **场景2：死锁恢复**
监控系统检测到死锁时，中断其中一个线程，打破僵局。

---

### **注意事项**
1. **正确处理中断**：捕获 `InterruptedException` 后，通常需要清理资源或记录日志
2. **避免重复中断**：中断状态会被清除，需根据业务决定是否恢复中断标志
   ```java
   catch (InterruptedException e) {
       Thread.currentThread().interrupt(); // 重新设置中断标志
       // 其他处理...
   }
   ```
3. **锁释放**：即使被中断，如果已经获取到锁，仍需在 `finally` 中释放
