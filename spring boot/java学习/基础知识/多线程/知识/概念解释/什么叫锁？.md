

在Java多线程编程中，`synchronized`关键字用于实现线程同步，以避免并发访问共享资源时出现数据不一致的问题。当`同步块`使用`synchronized(obj)`形式时，括号中的`obj`参数决定了锁定的对象。以下是逐步解释：

1. **锁的基本概念**：
   - **锁（Lock）** 是一种同步机制，用于控制多个线程对共享资源的访问。当一个线程获得某个对象的锁后，其他线程必须等待该锁释放后才能继续执行相关代码。[[线程怎么拿到锁？]]
   - 每个Java对象都有一个与之关联的**内部锁（Intrinsic Lock）**，也称为监视器锁（Monitor Lock）。

2. **synchronized关键字的作用**：
   - 修饰方法：锁定当前实例（`this`）或类（静态方法），确保同一时间只有一个线程执行该方法。
   - 修饰代码块：显式指定锁对象，允许更细粒度的同步控制。

3. **括号中的参数（锁对象）**：
   - **作用**：决定同步块的锁范围。同一锁对象的所有同步块将互斥执行，不同锁对象的同步块互不干扰。
   - **示例**：
     ```java
     public class Example {
         private final Object lock1 = new Object();
         private final Object lock2 = new Object();
     
         public void methodA() {
             synchronized(lock1) { // 锁定lock1
                 // 访问共享资源
             }
         }
     
         public void methodB() {
             synchronized(lock2) { // 锁定lock2
                 // 访问另一共享资源
             }
         }
     }
     ```
     - `methodA`和`methodB`使用不同的锁对象，可以并行执行，提高并发效率。

4. **锁对象的选择原则**：
   - **专用锁对象**：推荐使用私有且不可变（`final`）的锁对象，避免外部干扰或意外修改。
   - **避免字符串或基础类型**：如`String`或`Integer`可能因JVM优化导致隐藏的锁冲突。
   - **粒度控制**：根据共享资源的独立性选择合适的锁对象，避免不必要的大范围锁定。

5. **锁的释放**：
   - **自动释放**：无论是正常退出同步块还是抛出异常，锁都会自动释放。
   - **作用域限制**：锁的有效范围仅限于同步块内部，执行完毕后立即释放，不影响同步块外的代码执行。

6. **常见错误与注意事项**：
   - **错误锁定可变对象**：如锁对象在同步块内被修改，可能导致锁失效或意外行为。
     ```java
     Object lock = new Object();
     synchronized(lock) {
         lock = new Object(); // ❌ 错误！锁对象被修改，后续线程可能使用不同的锁
     }
     ```
   - **死锁风险**：多个线程互相等待对方持有的锁时可能导致死锁。需通过设计一致的锁获取顺序来避免。

**总结**：
- `synchronized(obj)`中的`obj`参数决定了锁定的对象，控制同步块的并发访问。
- 正确选择锁对象可实现细粒度同步，平衡线程安全与程序性能。
- 理解锁的机制及合理应用，是编写高效、可靠多线程程序的关键。
