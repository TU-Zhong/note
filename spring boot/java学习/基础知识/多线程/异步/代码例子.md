好的，我们通过一些例子来帮助你掌握 `CompletableFuture` 的核心知识点。

`CompletableFuture` 是 Java 8 引入的，它极大地简化了异步编程，提供了强大的链式操作和异常处理机制。

**核心概念回顾：**

- **创建异步任务：** `supplyAsync()` (有返回值), `runAsync()` (无返回值)
- **结果处理与转换：** `thenApply()`, `thenAccept()`, `thenRun()`
- **组合多个 CompletableFuture：** `thenCompose()`, `thenCombine()`, `allOf()`, `anyOf()`
- **异常处理：** `exceptionally()`, `handle()`

下面我们来看一些具体的例子：

**例子 1：基本用法 - 异步执行任务并获取结果**

Java

```java
import java.util.concurrent.CompletableFuture;
import java.util.concurrent.ExecutionException;
import java.util.concurrent.Executors;
import java.util.concurrent.TimeUnit;

public class CompletableFutureExample1 {

    public static void main(String[] args) throws ExecutionException, InterruptedException {
        // 1. 使用 supplyAsync 执行一个有返回值的异步任务
        // supplyAsync 会使用 ForkJoinPool.commonPool() 作为默认线程池
        CompletableFuture<String> future = CompletableFuture.supplyAsync(() -> {
            try {
                // 模拟一个耗时操作
                System.out.println(Thread.currentThread().getName() + "：开始执行异步任务...");
                TimeUnit.SECONDS.sleep(2);
                System.out.println(Thread.currentThread().getName() + "：异步任务执行完毕！");
            } catch (InterruptedException e) {
                throw new IllegalStateException(e);
            }
            return "异步任务结果：Hello from CompletableFuture!";
        });

        System.out.println(Thread.currentThread().getName() + "：主线程继续执行其他任务...");

        // 2. 获取异步任务的结果 (会阻塞直到任务完成)
        // String result = future.get(); // 阻塞式获取
        // System.out.println(Thread.currentThread().getName() + "：获取到异步结果 - " + result);

        // 3. 非阻塞式处理结果 - 使用 thenAccept (消费结果)
        future.thenAccept(result -> {
            System.out.println(Thread.currentThread().getName() + "：(thenAccept) 获取到异步结果 - " + result);
        });

        // 4. 非阻塞式处理结果 - 使用 thenApply (转换结果)
        CompletableFuture<Integer> lengthFuture = future.thenApply(result -> {
            System.out.println(Thread.currentThread().getName() + "：(thenApply) 正在计算结果的长度...");
            return result.length();
        });

        lengthFuture.thenAccept(length -> {
            System.out.println(Thread.currentThread().getName() + "：(thenAccept after thenApply) 结果的长度是 - " + length);
        });


        // 5. 如果只是想在任务完成后执行某个操作，不关心结果，使用 thenRun
        future.thenRun(() -> {
            System.out.println(Thread.currentThread().getName() + "：(thenRun) 异步任务已完成，执行后续清理操作。");
        });


        System.out.println(Thread.currentThread().getName() + "：主线程任务执行完毕，等待异步任务回调...");

        // 为了演示效果，让主线程稍微等待一下，确保异步回调能执行完毕
        // 在实际应用中，通常不需要这样手动 sleep，因为 CompletableFuture 的回调会自动执行
        TimeUnit.SECONDS.sleep(3);
    }
}
```

**例子 1 解析：**

- `CompletableFuture.supplyAsync(() -> { ... })`：创建并立即执行一个异步任务。Lambda 表达式中的代码会在另一个线程（通常是 ForkJoinPool 中的线程）中执行。
- `future.get()`：这是一个阻塞方法。调用它会使当前线程（这里是主线程）等待，直到异步任务完成并返回结果。为了避免阻塞，我们通常使用回调风格的方法。
- `future.thenAccept(result -> { ... })`：当 `future` 完成时，会自动调用这个 Lambda 表达式，并将结果 `result` 传递进来。这个回调通常会在执行 `future` 的同一个线程或者 ForkJoinPool 中的其他线程执行（取决于具体情况，也可以指定线程池）。
- `future.thenApply(result -> { ... })`：与 `thenAccept` 类似，但它允许你对结果进行转换，并返回一个新的 `CompletableFuture`。在这个例子中，我们将字符串结果转换成了它的长度。
- `future.thenRun(() -> { ... })`：当 `future` 完成时执行一个 `Runnable`，它不关心前一个阶段的结果，也没有返回值。

**例子 2：链式操作与 `thenCompose`**

假设我们有两个异步操作，第二个操作依赖于第一个操作的结果。

Java

```
import java.util.concurrent.CompletableFuture;
import java.util.concurrent.ExecutionException;
import java.util.concurrent.TimeUnit;

public class CompletableFutureExample2 {

    // 模拟获取用户ID的异步方法
    public static CompletableFuture<Integer> getUserIdAsync() {
        return CompletableFuture.supplyAsync(() -> {
            System.out.println(Thread.currentThread().getName() + ": 获取用户ID...");
            try {
                TimeUnit.SECONDS.sleep(1);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            return 123; // 模拟用户ID
        });
    }

    // 模拟根据用户ID获取用户详情的异步方法
    public static CompletableFuture<String> getUserDetailsAsync(int userId) {
        return CompletableFuture.supplyAsync(() -> {
            System.out.println(Thread.currentThread().getName() + ": 根据用户ID " + userId + " 获取用户详情...");
            try {
                TimeUnit.SECONDS.sleep(1);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            return "用户详情：ID=" + userId + ", Name=张三";
        });
    }

    public static void main(String[] args) throws ExecutionException, InterruptedException {
        System.out.println(Thread.currentThread().getName() + ": 开始执行主任务");

        CompletableFuture<String> userDetailsFuture = getUserIdAsync()
                .thenCompose(userId -> getUserDetailsAsync(userId)); // 关键点：thenCompose

        // thenApply 会导致 CompletableFuture<CompletableFuture<String>>
        // CompletableFuture<CompletableFuture<String>> nestedFuture = getUserIdAsync()
        //         .thenApply(userId -> getUserDetailsAsync(userId));

        userDetailsFuture.thenAccept(details -> {
            System.out.println(Thread.currentThread().getName() + ": (thenAccept) 最终获取到 - " + details);
        });

        System.out.println(Thread.currentThread().getName() + ": 主任务提交完毕，等待异步结果...");
        TimeUnit.SECONDS.sleep(3); // 等待异步操作完成
    }
}
```

**例子 2 解析：**

- `thenCompose(Function)`：当你的第一个异步操作完成后，它的结果会作为输入传递给一个函数，这个函数**返回另一个 `CompletableFuture`**。`thenCompose` 会等待这个新的 `CompletableFuture` 完成，并将其结果作为最终结果。这对于避免 `CompletableFuture<CompletableFuture<T>>` 这种嵌套结构非常有用。
- 对比 `thenApply`：如果这里使用 `thenApply(userId -> getUserDetailsAsync(userId))`，那么 `thenApply` 的结果将会是 `CompletableFuture<CompletableFuture<String>>`。而 `thenCompose` 会将其“扁平化”为 `CompletableFuture<String>`。

**例子 3：异常处理 - `exceptionally` 和 `handle`**

Java

```
import java.util.concurrent.CompletableFuture;
import java.util.concurrent.ExecutionException;
import java.util.concurrent.TimeUnit;

public class CompletableFutureExample3 {

    public static void main(String[] args) throws ExecutionException, InterruptedException {
        // 1. 使用 exceptionally 处理异常
        CompletableFuture<String> futureWithException = CompletableFuture.supplyAsync(() -> {
            System.out.println(Thread.currentThread().getName() + ": 异步任务开始，准备抛出异常...");
            try {
                TimeUnit.MILLISECONDS.sleep(500);
            } catch (InterruptedException e) { /* ignore */ }
            if (true) { // 模拟一个错误条件
                throw new RuntimeException("异步任务执行出错啦！");
            }
            return "正常结果";
        }).exceptionally(ex -> { // 当上一个阶段抛出异常时，这个函数会被调用
            System.err.println(Thread.currentThread().getName() + ": (exceptionally) 捕获到异常 - " + ex.getMessage());
            return "默认值或错误提示"; // 返回一个替代结果
        });

        futureWithException.thenAccept(result -> {
            System.out.println(Thread.currentThread().getName() + ": (thenAccept after exceptionally) 得到结果 - " + result);
        });


        System.out.println("------------------------------------");
        TimeUnit.SECONDS.sleep(1); // 等待上一个任务完成


        // 2. 使用 handle 处理正常结果和异常
        CompletableFuture<String> futureWithHandle = CompletableFuture.supplyAsync(() -> {
            System.out.println(Thread.currentThread().getName() + ": 异步任务开始 (handle)...");
            try {
                TimeUnit.MILLISECONDS.sleep(500);
            } catch (InterruptedException e) { /* ignore */ }
            // 模拟成功或失败
            boolean success = Math.random() > 0.5;
            if (!success) {
                System.out.println(Thread.currentThread().getName() + ": 任务失败 (handle)，抛出异常...");
                throw new RuntimeException("模拟随机任务失败 (handle)！");
            }
            return "随机任务成功结果 (handle)";
        }).handle((result, ex) -> { // 无论上一个阶段是正常完成还是异常完成，这个函数都会被调用
            if (ex != null) { // 如果有异常
                System.err.println(Thread.currentThread().getName() + ": (handle) 捕获到异常 - " + ex.getMessage());
                return "处理异常后的默认值 (handle)";
            }
            // 如果没有异常，可以对正常结果进行处理或转换
            System.out.println(Thread.currentThread().getName() + ": (handle) 任务正常完成，结果是 - " + result);
            return result.toUpperCase();
        });

        futureWithHandle.thenAccept(finalResult -> {
            System.out.println(Thread.currentThread().getName() + ": (thenAccept after handle) 最终得到结果 - " + finalResult);
        });


        TimeUnit.SECONDS.sleep(2); // 等待所有任务完成
    }
}
```

**例子 3 解析：**

- `exceptionally(Function<Throwable, ? extends T> fn)`：当 `CompletableFuture` 的计算过程中抛出异常时，会调用 `fn` 函数。这个函数接收 `Throwable` 作为参数，并返回一个替代的结果，从而使得 `CompletableFuture` 能够正常完成（尽管是带着一个错误处理后的结果）。
- `handle(BiFunction<? super T, Throwable, ? extends U> fn)`：无论前面的 `CompletableFuture` 是正常完成还是异常完成，`handle` 方法都会被执行。它接收两个参数：正常结果 `T` (如果异常则为 `null`) 和异常 `Throwable` (如果正常完成则为 `null`)。你可以在 `handle` 中根据这两个参数来决定返回什么结果。`handle` 总是会返回一个新的 `CompletableFuture`。

**例子 4：组合多个 CompletableFuture - `thenCombine`, `allOf`, `anyOf`**

Java

```
import java.util.concurrent.CompletableFuture;
import java.util.concurrent.ExecutionException;
import java.util.concurrent.TimeUnit;
import java.util.stream.Collectors;
import java.util.stream.Stream;

public class CompletableFutureExample4 {

    public static CompletableFuture<String> fetchUserData(int userId) {
        return CompletableFuture.supplyAsync(() -> {
            System.out.println(Thread.currentThread().getName() + ": 获取用户 " + userId + " 数据...");
            try { TimeUnit.SECONDS.sleep(userId == 1 ? 1 : 2); } catch (InterruptedException e) {}
            return "用户数据" + userId;
        });
    }

    public static CompletableFuture<String> fetchOrderData(int userId) {
        return CompletableFuture.supplyAsync(() -> {
            System.out.println(Thread.currentThread().getName() + ": 获取用户 " + userId + " 订单数据...");
            try { TimeUnit.SECONDS.sleep(userId == 1 ? 2 : 1); } catch (InterruptedException e) {}
            return "订单数据" + userId;
        });
    }

    public static void main(String[] args) throws ExecutionException, InterruptedException {
        int userId = 1;

        // 1. thenCombine: 合并两个独立的 CompletableFuture 的结果
        System.out.println("--- thenCombine ---");
        CompletableFuture<String> userDataFuture = fetchUserData(userId);
        CompletableFuture<String> orderDataFuture = fetchOrderData(userId);

        CompletableFuture<String> combinedFuture = userDataFuture.thenCombine(
                orderDataFuture,
                (userData, orderData) -> { // BiFunction，接收两个 future 的结果
                    System.out.println(Thread.currentThread().getName() + ": (thenCombine) 合并数据...");
                    return userData + " & " + orderData;
                }
        );

        System.out.println(Thread.currentThread().getName() + ": (thenCombine) 主线程等待合并结果...");
        System.out.println(Thread.currentThread().getName() + ": (thenCombine) 合并结果: " + combinedFuture.get()); // 阻塞获取


        TimeUnit.SECONDS.sleep(1); // 分隔一下输出
        System.out.println("\n--- allOf ---");

        // 2. allOf: 等待所有 CompletableFuture 完成 (无直接返回值，需要自己处理)
        CompletableFuture<String> future1 = fetchUserData(1);
        CompletableFuture<String> future2 = fetchOrderData(2); // 模拟另一个用户的订单
        CompletableFuture<String> future3 = CompletableFuture.supplyAsync(() -> {
            try { TimeUnit.MILLISECONDS.sleep(500); } catch (InterruptedException e) {}
            return "其他数据";
        });

        CompletableFuture<Void> allFutures = CompletableFuture.allOf(future1, future2, future3);

        // allOf 本身返回 CompletableFuture<Void>
        // 如果需要获取所有结果，需要进一步处理
        CompletableFuture<String> allResultsFuture = allFutures.thenApply(v -> {
            // 当 allOf 完成后，所有 future 都已完成，可以安全地调用 get() (但注意 get() 本身是阻塞的，
            // 在 thenApply 的回调中调用通常是安全的，因为此时它们已经完成了)
            // 或者更好的方式是使用 join()，它在 CompletableFuture 完成时非阻塞地获取结果，如果未完成则抛出未检查异常
            String result1 = future1.join();
            String result2 = future2.join();
            String result3 = future3.join();
            System.out.println(Thread.currentThread().getName() + ": (allOf thenApply) 所有任务完成，收集结果...");
            return Stream.of(result1, result2, result3).collect(Collectors.joining(", "));
        });

        System.out.println(Thread.currentThread().getName() + ": (allOf) 主线程等待所有任务完成...");
        System.out.println(Thread.currentThread().getName() + ": (allOf) 所有任务结果: " + allResultsFuture.get());


        TimeUnit.SECONDS.sleep(1);
        System.out.println("\n--- anyOf ---");

        // 3. anyOf: 等待任意一个 CompletableFuture 完成
        CompletableFuture<String> fastFuture = CompletableFuture.supplyAsync(() -> {
            try { TimeUnit.MILLISECONDS.sleep(300); } catch (InterruptedException e) {}
            System.out.println(Thread.currentThread().getName() + ": (anyOf) 快速任务完成");
            return "快速任务的结果";
        });
        CompletableFuture<String> slowFuture = CompletableFuture.supplyAsync(() -> {
            try { TimeUnit.SECONDS.sleep(1); } catch (InterruptedException e) {}
            System.out.println(Thread.currentThread().getName() + ": (anyOf) 慢速任务完成");
            return "慢速任务的结果";
        });

        CompletableFuture<Object> anyFuture = CompletableFuture.anyOf(fastFuture, slowFuture);

        System.out.println(Thread.currentThread().getName() + ": (anyOf) 主线程等待任一任务完成...");
        // anyOf 返回的是 CompletableFuture<Object>，因为不知道哪个会先完成，结果类型不确定
        System.out.println(Thread.currentThread().getName() + ": (anyOf) 第一个完成的任务结果: " + anyFuture.get());


        TimeUnit.SECONDS.sleep(2); // 确保所有异步操作有机会完成
    }
}
```

**例子 4 解析：**

- `thenCombine(otherFuture, BiFunction)`：当当前 `CompletableFuture` 和 `otherFuture` 都完成时，会将它们的结果传递给 `BiFunction`，`BiFunction` 的返回值将作为新的 `CompletableFuture` 的结果。
- `allOf(CompletableFuture<?>... cfs)`：创建一个新的 `CompletableFuture`，它会在所有给定的 `CompletableFuture` 都完成时才完成。注意，`allOf` 本身返回 `CompletableFuture<Void>`。如果你需要所有原始 `CompletableFuture` 的结果，你需要在 `allOf` 完成后再手动收集它们（通常在 `thenApply` 或类似的回调中通过 `join()` 或已完成的 `get()` 获取）。
- `anyOf(CompletableFuture<?>... cfs)`：创建一个新的 `CompletableFuture`，它会在给定的 `CompletableFuture` 中任意一个完成时就完成。返回的 `CompletableFuture` 的结果是第一个完成的那个 `CompletableFuture` 的结果，类型是 `Object`。

**一些额外的技巧和注意事项：**

- **指定线程池：** `supplyAsync` 和 `runAsync` 以及很多 `thenXXX` 方法都有一个重载版本，允许你传入一个 `Executor` 来指定执行异步任务或回调的线程池。这对于避免耗尽公共的 `ForkJoinPool` 或者需要特定线程策略的场景很有用。
    
    Java
    
    ```
    ExecutorService customExecutor = Executors.newFixedThreadPool(5);
    CompletableFuture<String> future = CompletableFuture.supplyAsync(() -> {
        // ...
        return "result";
    }, customExecutor);
    
    future.thenAcceptAsync(result -> {
        // 这个回调也会在 customExecutor 中执行
    }, customExecutor);
    ```
    
- **`join()` vs `get()`：**
    - `get()`：阻塞等待结果，如果任务抛出受检异常，`get()` 会将其包装在 `ExecutionException` 中抛出。
    - `join()`：与 `get()` 类似，但如果任务抛出异常，它会抛出未经检查的 `CompletionException` (包装了原始异常)。在 Lambda 表达式或流操作中更方便使用，因为不需要显式处理 `ExecutionException` 或 `InterruptedException`。
- **避免阻塞：** `CompletableFuture` 的核心优势在于其非阻塞的链式回调。尽量使用 `thenApply`, `thenAccept`, `thenRun`, `thenCompose`, `handle`, `exceptionally` 等方法来处理结果和异常，而不是在主流程中直接调用 `get()` 或 `join()` (除非你确实需要在那个点等待结果)。

通过学习和实践这些例子，你应该能对 `CompletableFuture` 的核心功能有一个很好的理解。建议你亲自运行这些代码，并尝试修改它们来观察不同的行为。
