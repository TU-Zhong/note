要让多个服务员线程安全地生成唯一递增的订单ID，使用 `AtomicInteger` 的正确方式如下：

---

### **一、问题分析**
原代码中 `orderId` 存在的问题：
```java
class Waiter implements Runnable {
    private int orderId = 1; // 实例级变量
    // 多线程下会导致ID重复
}
```

---

### **二、AtomicInteger解决方案**
#### **1. 全局唯一ID生成器**
```java
class OrderIdGenerator {
    // 静态原子计数器，保证全局唯一
    private static final AtomicInteger idCounter = new AtomicInteger(1);
    
    public static int getNextId() {
        return idCounter.getAndIncrement();
    }
}
```

#### **2. 修改Waiter类**
```java
class Waiter implements Runnable {
    // 删除实例级orderId
    private final OrderQueue orderQueue;

    public Waiter(OrderQueue orderQueue) {
        this.orderQueue = orderQueue;
    }

    @Override
    public void run() {
        try {
            while (isRunning) {
                // 使用全局唯一ID
                int currentId = OrderIdGenerator.getNextId();
                String customer = "顾客-" + (currentId % 5 + 1);
                Order order = new Order(currentId, customer);
                orderQueue.addOrder(order);
                Thread.sleep(500);
            }
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
}
```

### **三、技术细节说明**
1. **原子性保证**：
   - `getAndIncrement()` 是原子操作，确保多线程下ID不重复
   - 底层实现：
     ```java
     public final int getAndIncrement() {
         return unsafe.getAndAddInt(this, valueOffset, 1);
     }
     ```

2. **ID序列优化**：
   - 若需要控制ID范围（如订单系统需用32位整数），可添加循环检测：
     ```java
     public static int getNextId() {
         int id;
         do {
             id = idCounter.getAndIncrement();
             if (id > Integer.MAX_VALUE - 100) { // 接近溢出时重置
                 synchronized (OrderIdGenerator.class) {
                     if (idCounter.get() > Integer.MAX_VALUE - 100) {
                         idCounter.set(1);
                     }
                 }
             }
         } while (id < 0); // 防御负数ID
         return id;
     }
     ```

### **五、进阶优化（分布式系统）**
如果需要跨JVM的唯一ID，改用雪花算法：
```java
public class SnowflakeIdGenerator {
    private final long datacenterId;
    private final long machineId;
    private long sequence = 0L;
    private long lastTimestamp = -1L;

    public SnowflakeIdGenerator(long datacenterId, long machineId) {
        this.datacenterId = datacenterId;
        this.machineId = machineId;
    }

    public synchronized long nextId() {
        long timestamp = System.currentTimeMillis();
        if (timestamp < lastTimestamp) {
            throw new IllegalStateException("时钟回拨");
        }

        if (timestamp == lastTimestamp) {
            sequence = (sequence + 1) & 0xFFF;
            if (sequence == 0) {
                timestamp = tilNextMillis(lastTimestamp);
            }
        } else {
            sequence = 0L;
        }

        lastTimestamp = timestamp;

        return ((timestamp - 1609459200000L) << 22) // 2021年开始的毫秒数
                | (datacenterId << 17)
                | (machineId << 12)
                | sequence;
    }

    private long tilNextMillis(long lastTimestamp) {
        long timestamp = System.currentTimeMillis();
        while (timestamp <= lastTimestamp) {
            timestamp = System.currentTimeMillis();
        }
        return timestamp;
    }
}
```

```java
public class IdTest {
    public static void main(String[] args) {
        final int THREADS = 10;
        ExecutorService executor = Executors.newFixedThreadPool(THREADS);
        Set<Integer> idSet = Collections.synchronizedSet(new HashSet<>());

        for (int i = 0; i < 100_000; i++) {
            executor.execute(() -> {
                int id = OrderIdGenerator.getNextId();
                if (!idSet.add(id)) {
                    System.err.println("发现重复ID: " + id);
                }
            });
        }

        executor.shutdown();
        try {
            executor.awaitTermination(1, TimeUnit.MINUTES);
            System.out.println("生成唯一ID数量: " + idSet.size());
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
}
```
