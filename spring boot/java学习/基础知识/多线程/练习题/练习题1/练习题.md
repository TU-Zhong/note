### **题目：简单餐厅订单处理系统**
**需求描述**：  
模拟一个小餐厅的订单处理流程，包含以下角色：  
1. **服务员（多个线程）**：生成订单并放入订单队列  
2. **厨师（多个线程）**：从队列取出订单并烹饪  
3. **订单队列（共享资源）**：线程安全的数据结构，需加锁处理  

**具体要求**：  
- 实现订单队列的线程安全（使用 `synchronized` 或 `ReentrantLock`）  
- 当队列为空时，厨师线程应等待新订单  
- 当队列满时（设置最大容量），服务员线程应等待  
- 支持优雅关闭（所有订单处理完后关闭线程）
