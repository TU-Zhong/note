

在Java中，**方法重写（Override）**是继承的核心特性之一，它允许子类对父类的方法提供新的实现。但为了确保重写的正确性和避免潜在问题，需要遵循以下关键注意事项和细节：

---

### **1. 方法签名必须完全相同**
- **规则**：子类重写的方法必须与父类方法**方法名、参数列表和返回类型**完全一致（Java 5+ 支持协变返回类型，后文会说明）。
- **示例**：
  ```java
  class Parent {
      public void show() { /* ... */ }
  }
  class Child extends Parent {
      @Override
      public void show() { /* 正确重写 */ } // ✔️
  }
  ```

---

### **2. 访问权限不能更严格**
- **规则**：子类重写的方法的访问修饰符必须**≥父类方法的访问权限**。
  - 例如父类方法是 `public`，子类不能改为 `protected` 或 `private`。
- **示例**：
  ```java
  class Parent {
      protected void print() { /* ... */ }
  }
  class Child extends Parent {
      @Override
      public void print() { /* 正确：权限扩大 */ } // ✔️

      // @Override
      // private void print() { /* 错误：权限缩小 */ } // ❌
  }
  ```

---

### **3. 协变返回类型（Java 5+ 支持）**
- **规则**：子类方法的返回类型可以是父类方法返回类型的**子类**。
- **示例**：
  ```java
  class Animal {}
  class Dog extends Animal {}

  class Parent {
      Animal getAnimal() { return new Animal(); }
  }
  class Child extends Parent {
      @Override
      Dog getAnimal() { return new Dog(); } // ✔️ 协变返回类型
  }
  ```

---

### **4. 异常处理必须兼容**
- **规则**：
  - 子类方法不能抛出比父类方法更宽泛的**检查型异常（Checked Exception）**。
  - 可以抛出更具体的异常，或者不抛出异常。
- **示例**：
  ```java
  class Parent {
      void process() throws IOException { /* ... */ }
  }
  class Child extends Parent {
      @Override
      void process() throws FileNotFoundException { // ✔️ FileNotFoundException是IOException的子类
          // 或者不抛出任何异常 ✔️
      }

      // @Override
      // void process() throws Exception { ❌ Exception比IOException更宽泛
      // }
  }
  ```

---

### **5. 静态方法不能被重写**
- **规则**：静态方法属于类，不能被实例方法重写。如果子类定义了同名静态方法，这是**方法隐藏（Method Hiding）**，而非重写。
- **示例**：
  ```java
  class Parent {
      static void staticMethod() { System.out.println("Parent"); }
  }
  class Child extends Parent {
      static void staticMethod() { System.out.println("Child"); } // ✔️ 方法隐藏
  }

  public class Main {
      public static void main(String[] args) {
          Parent.staticMethod(); // 输出 Parent
          Child.staticMethod();  // 输出 Child
      }
  }
  ```

---

### **6. `final` 方法不能被重写**
- **规则**：父类中声明为 `final` 的方法不能被子类重写。
- **示例**：
  ```java
  class Parent {
      final void finalMethod() { /* ... */ }
  }
  class Child extends Parent {
      // @Override
      // void finalMethod() { } // ❌ 编译错误
  }
  ```

---

### **7. `private` 方法不能重写**
- **规则**：父类的 `private` 方法对子类不可见，因此子类中同名方法仅是“新方法”，而非重写。
- **示例**：
  ```java
  class Parent {
      private void privateMethod() { System.out.println("Parent"); }
  }
  class Child extends Parent {
      // 这是一个新方法，与父类无关
      public void privateMethod() { System.out.println("Child"); }
  }

  public class Main {
      public static void main(String[] args) {
          Child child = new Child();
          child.privateMethod(); // 输出 Child
      }
  }
  ```

---

### **8. 使用 `@Override` 注解**
- **建议**：始终在重写方法上添加 `@Override` 注解。
  - 作用：帮助编译器检查是否满足重写条件（避免拼写错误或参数不匹配）。
- **示例**：
  ```java
  class Parent {
      void doSomething() {}
  }
  class Child extends Parent {
      @Override
      void doSomething() {} // ✔️ 编译器会检查重写是否正确
  }
  ```

---

### **9. 构造方法不能被重写**
- **规则**：构造方法名必须与类名相同，所以不存在重写。子类构造方法通过 `super()` 调用父类构造方法。

---

### **10. `synchronized` 关键字不影响重写**
- **规则**：子类方法可以选择是否添加 `synchronized` 关键字，与父类无关。
- **示例**：
  ```java
  class Parent {
      synchronized void syncMethod() {}
  }
  class Child extends Parent {
      @Override
      void syncMethod() {} // ✔️ 可以不加synchronized
  }
  ```

---

### **总结：方法重写的核心原则**
| 特性                | 规则                               |
|---------------------|-----------------------------------|
| 方法签名            | 名称、参数列表、返回类型必须一致   |
| 访问权限            | 不能更严格                        |
| 异常                | 不能抛出更宽泛的检查型异常         |
| 静态方法            | 静态方法隐藏，无法重写            |
| `final`/`private`   | 不能重写                          |
| `@Override`         | 强烈推荐使用                      |

---

### **实战建议**
1. 尽量通过 `@Override` 确保重写正确性。
2. 优先使用重写实现多态，而非通过静态方法或隐藏。
3. 避免在父类方法中使用 `final`，除非明确禁止扩展。
