

在Java中，成员变量的继承与方法的继承有所不同，成员变量**没有“重写”**的概念，而是通过**隐藏（Shadowing）**或**直接访问**的方式处理。以下是关于成员变量继承的关键细节和注意事项：

---

### **1. 成员变量的继承规则**
- **直接继承**：子类会继承父类中所有**非私有（`public`、`protected`、默认包权限）**的成员变量。
- **私有变量不可直接访问**：父类的私有成员变量（`private`）对子类不可见，但可以通过父类提供的公共方法（如 `getter`/`setter`）间接访问。

#### **示例：直接继承非私有变量**
```java
class Parent {
    public String publicVar = "Parent Public";
    protected String protectedVar = "Parent Protected";
    String defaultVar = "Parent Default";
    private String privateVar = "Parent Private"; // 子类不可见
}

class Child extends Parent {
    public void printVariables() {
        System.out.println(publicVar);     // ✔️ 可访问
        System.out.println(protectedVar);  // ✔️ 可访问
        System.out.println(defaultVar);    // ✔️ 同包下可访问
        // System.out.println(privateVar); // ❌ 不可访问
    }
}
```

---

### **2. 变量隐藏（Variable Shadowing）**
- **同名变量**：如果子类定义了与父类**同名**的成员变量，父类的变量会被“隐藏”（不是覆盖）。
- **访问方式**：
  - 子类通过`this`访问自己的变量。
  - 子类通过`super`访问父类的变量。

#### **示例：变量隐藏与访问**
```java
class Parent {
    String name = "Parent";
}

class Child extends Parent {
    String name = "Child"; // 隐藏父类的name变量

    public void printNames() {
        System.out.println(name);      // 输出 Child（子类变量）
        System.out.println(this.name); // 同上
        System.out.println(super.name);// 输出 Parent（父类变量）
    }
}

public class Main {
    public static void main(String[] args) {
        Child child = new Child();
        child.printNames();
    }
}
```

#### **输出结果**
```
Child
Child
Parent
```

---

### **3. 成员变量的初始化顺序**
1. **父类静态变量** → **子类静态变量**（类加载时初始化）。
2. **父类实例变量** → **父类构造方法**。
3. **子类实例变量** → **子类构造方法**。

#### **示例：初始化顺序**
```java
class Parent {
    static { System.out.println("父类静态代码块"); }
    { System.out.println("父类实例代码块"); }
    Parent() { System.out.println("父类构造方法"); }
}

class Child extends Parent {
    static { System.out.println("子类静态代码块"); }
    { System.out.println("子类实例代码块"); }
    Child() { System.out.println("子类构造方法"); }
}

public class Main {
    public static void main(String[] args) {
        new Child();
    }
}
```

#### **输出结果**
```
父类静态代码块
子类静态代码块
父类实例代码块
父类构造方法
子类实例代码块
子类构造方法
```

---

### **4. 多态与成员变量**
- **成员变量没有多态性**：变量的访问由**引用类型**决定，而不是实际对象类型。
- **方法有多态性**：方法的调用由实际对象类型决定。

#### **示例：变量与多态的对比**
```java
class Parent {
    String name = "Parent";
    void print() { System.out.println("Parent Method"); }
}

class Child extends Parent {
    String name = "Child";  // 隐藏父类的name
    @Override
    void print() { System.out.println("Child Method"); }
}

public class Main {
    public static void main(String[] args) {
        Parent obj = new Child();
        System.out.println(obj.name);  // 输出 Parent（变量看引用类型）
        obj.print();                   // 输出 Child Method（方法看对象类型）
    }
}
```

---

### **5. 实例变量与静态变量**
- **静态变量（类变量）**：所有子类共享父类的静态变量（除非子类重新定义）。
- **实例变量**：每个子类实例会拥有独立的父类实例变量。

#### **示例：静态变量的继承**
```java
class Parent {
    static int count = 10;
}

class Child extends Parent {
    // 继承Parent的count
}

public class Main {
    public static void main(String[] args) {
        Parent.count = 20;
        System.out.println(Child.count); // 输出 20（共享同一变量）
    }
}
```

---

### **6. 成员变量的访问权限**
- **`private`**：仅本类可见。
- **`protected`**：本类、子类、同包类可见。
- **默认（包权限）**：同包类可见。
- **`public`**：全局可见。

#### **示例：通过父类方法访问私有变量**
```java
class Parent {
    private String secret = "Parent Secret";
    public String getSecret() { return secret; }
}

class Child extends Parent {
    public void printSecret() {
        // System.out.println(secret); // ❌ 直接访问失败
        System.out.println(getSecret()); // ✔️ 通过父类方法访问
    }
}
```

---

### **7. 成员变量的隐藏与构造方法**
- **父类变量在子类构造方法中的初始化**：
  - 父类的成员变量在父类构造方法中初始化。
  - 子类构造方法通过`super()`调用父类构造方法完成初始化。

---

### **总结：成员变量继承的核心注意事项**
| 特性                  | 规则                                                                 |
|-----------------------|----------------------------------------------------------------------|
| **继承性**            | 非私有变量直接继承，私有变量通过方法访问                             |
| **变量隐藏**          | 子类定义同名变量会隐藏父类变量，需用`super`访问                      |
| **多态性**            | 成员变量无多态性，访问由引用类型决定                                 |
| **静态变量**          | 子类共享父类静态变量（除非重新定义）                                 |
| **初始化顺序**        | 父类成员变量→父类构造方法→子类成员变量→子类构造方法                   |
| **访问权限**          | 父类变量是否可见取决于修饰符（`private`、`protected`、`public`等）   |

---

### **实际开发建议**
1. **避免变量隐藏**：尽量不在子类中定义与父类同名的变量，容易引发混淆。
2. **优先使用方法封装**：通过`getter`/`setter`方法访问变量，增强灵活性和维护性。
3. **明确初始化逻辑**：在构造方法中通过`super()`确保父类变量正确初始化。
4. **慎用静态变量继承**：静态变量共享可能导致副作用，建议通过类名直接访问（如`Parent.count`）。
