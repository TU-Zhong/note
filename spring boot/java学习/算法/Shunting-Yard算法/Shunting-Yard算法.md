

Shunting-Yard算法的核心原理源于**数学表达式求值的本质需求**和**栈结构的特性**。其设计逻辑可拆解为四大基石：

---

### 一、数学表达式的隐式结构 (Implied Hierarchy)
中缀表达式 `3 + 4 × 2 / (1-5)` 隐含以下计算层级：
```math
           +
         /   \
        3     ÷
             / \
            ×   -
           / \ / \
          4  2 1   5
```
算法需要将这一树形结构**线性化**为后缀表达式，同时保持运算顺序不变。栈结构的**后进先出(LIFO)**特性完美适配运算符的**嵌套关系处理**。

---

### 二、双重优先级控制系统

#### 1. 静态优先级 (Static Precedence)
定义运算符的基础优先级：
```python
优先级表 = { '(':0, '+':1, '-':1, '×':2, '÷':2, '^':3 }
```
> **关键作用**：当新运算符的优先级 **>** 栈顶运算符时，允许延迟处理（新运算符入栈）；若 **≤** 则触发栈顶出列。

#### 2. 动态上下文优先级 (Dynamic Contextual Precedence)
- **括号构造的隔离域**：括号内的运算符优先级临时提升至系统最大值
- **右结合运算符的逆序处理**（如幂运算`^`）：
  ```python
  while (栈非空) and (栈顶优先级 > 当前优先级)  # 左结合常规处理
  或
  while (栈非空) and (栈顶优先级 ≥ 当前优先级)  # 右结合特殊处理
  ```

---

### 三、状态机驱动的操作决策
算法在扫描时根据**前一个符号类型**和**当前符号类型**决定操作，形成有限状态机：

| 前一符号类型 | 当前符号类型 | 允许操作          |
| ------ | ------ | ------------- |
| 操作数    | 运算符    | 启动运算符优先级比较    |
| 运算符    | 左括号    | 函数调用检测或普通括号处理 |
| 右括号    | 运算符    | 结束嵌套域后的优先级延续  |

> **示例**：表达式 `sum(2)×3` 中`)`后的`×`需触发栈内操作符弹出。

---

### 四、后缀表达式的计算优势
后缀表达式 `3 4 2 × 1 5 - ÷ +` 的求值过程仅需**单趟扫描+栈计算**：
```python
计算栈 = []
for token in 后缀表达式:
    if token是数字:
        计算栈.append(float(token))
    else:
        b = 计算栈.pop()
        a = 计算栈.pop()
        计算栈.append(apply_operator(a, b, token))
return 计算栈[0]
```
相较于中缀表达式，避免了：
1. 递归优先级判断
2. 括号匹配验证
3. 运算符位置歧义

---

### 五、算法扩展性原理
通过修改优先级表和结合性规则，可支持更多运算类型：

| 运算符类型   | 优先级 | 结合性   | 处理逻辑                              |
|--------------|--------|----------|-------------------------------------|
| 逻辑运算符   | 0      | 左结合   | 在最低优先级处理                    |
| 三元运算符   | 特殊   | 右结合   | 需要插入虚拟标记                    |
| 函数调用     | 最高   | 无结合   | 通过逗号分割参数                    |

---

### 六、时空复杂度证明
令输入表达式长度为n：
- **时间复杂度O(n)**：每个token经历1次入栈+1次出栈（摊还分析）
- **空间复杂度O(n)**：输出队列和操作符栈大小之和不超过2n

---

### 七、与人类计算思维的映射对比
当人类计算 `3 + 4×2/(1-5)` 时，潜意识执行以下步骤：
1. 识别括号内的`1-5=-4`
2. 计算`4×2=8`
3. 执行除法`8/-4=-2`
4. 最后求和`3+(-2)=1`

Shunting-Yard算法将此过程**机械化**：
- **栈** → 大脑的短期记忆区
- **输出队列** → 长期记忆中的步骤记录
- **优先级比较** → 计算时的注意力分配机制

---

### 结论
该算法的本质是**用栈结构模拟人脑处理运算优先级的过程**，通过明确的优先级规则和栈操作，将隐含的运算顺序显式编码为线性结构。这种转换使得计算机可以像人类一样"理解"运算顺序，但以完全确定性的机械方式实现。
