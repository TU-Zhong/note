

以下是Shunting-Yard算法的专业化流程说明：

**Shunting-Yard算法**（调度场算法，Dijkstra, 1961）是用于将中缀表达式转换为后缀表达式（逆波兰式）的线性时间复杂度算法，其核心操作基于两个数据结构：**操作符栈**（operator stack）和**输出队列**（output queue）。具体流程如下：

1. **符号分类处理**
   - **操作数**：直接推入输出队列
   - **函数/左括号**：直接压入操作符栈
   - **右括号**：持续弹出栈顶元素至输出队列，直到遇到左括号（括号丢弃不输出）
   - **运算符**：按优先级规则处理栈操作

2. **运算符优先级规则**
   设当前运算符为`op_current`，栈顶运算符为`op_top`，需满足以下条件时弹出栈顶：
   ```
   ( precedence(op_top) > precedence(op_current) ) ∨ 
   ( precedence(op_top) = precedence(op_current) ∧ left_associative(op_current) )
   ```
   其中：
   - 算数运算符典型优先级：`^` > `*/` > `+-`
   - 左结合性适用于多数运算符（除幂运算等右结合运算符）

3. **算法流程**
   ```python
   while tokens exist:
       token = next_token()
       if is_operand(token):
           output.append(token)
       elif token == '(':
           stack.push(token)
       elif token == ')':
           while stack.top() != '(':
               output.append(stack.pop())
           stack.pop()  # 丢弃左括号
       else:  # operator
           while (not stack.empty()) and 
                 (stack.top() != '(') and 
                 ( (prec(stack.top()) > prec(token)) or 
                   (prec(stack.top()) == prec(token) and assoc(token) == LEFT) ):
               output.append(stack.pop())
           stack.push(token)
   # 清空操作符栈
   while not stack.empty():
       output.append(stack.pop())
   ```

4. **复杂度分析**
   - **时间复杂度**：O(n) （每个操作符最多经历1次入栈+1次出栈）
   - **空间复杂度**：O(n) （输出队列与栈的联合空间）

5. **典型应用场景**
   - 编译器语法分析阶段
   - 表达式求值器（如HP计算器实现）
   - 正则表达式引擎
   - SQL查询优化器

**示例演算**（表达式: 3 + 4 × 2 ÷ (1 - 5)）

| 处理字符 | 操作符栈       | 输出队列              |
|----------|----------------|-----------------------|
| 3        | []             | [3]                   |
| +        | [+]            | [3]                   |
| 4        | [+]            | [3,4]                 |
| ×        | [+,×]          | [3,4]                 |
| 2        | [+,×]          | [3,4,2]               |
| ÷        | [+,÷]          | [3,4,2,×]             |
| (        | [+,÷,(]        | [3,4,2,×]             |
| 1        | [+,÷,(]        | [3,4,2,×,1]           |
| -        | [+,÷,(,-]      | [3,4,2,×,1]           |
| 5        | [+,÷,(,-]      | [3,4,2,×,1,5]         |
| )        | [+,÷]          | [3,4,2,×,1,5,-]       |
| EOF      | []             | [3,4,2,×,1,5,-,÷,+]   |

最终后缀表达式：`3 4 2 × 1 5 - ÷ +`，准确保持了原式的运算顺序：
```math
3 + \frac{4 \times 2}{1 - 5} = 3 + \frac{8}{-4} = 3 - 2 = 1
```


## **具体例子**
我们以中缀表达式 **"3 + 4 × 2 / (1 - 5)"** 为例，通过分步表格展示Shunting-Yard算法的完整处理过程。以下每行代表算法处理一个新符号时的状态变化：

---

### **步骤解析表**
| 当前符号 | 操作符栈 (Stack) | 输出队列 (Output) | 操作说明                                                                 |
|----------|-------------------|--------------------|--------------------------------------------------------------------------|
| **3**    | []                | [3]                | 数字直接加入输出队列                                                     |
| **+**    | [+]               | [3]                | 栈空，运算符直接入栈                                                     |
| **4**    | [+]               | [3,4]              | 数字直接加入输出队列                                                     |
| **×**    | [+, ×]            | [3,4]              | ×优先级 > +，直接入栈                                                   |
| **2**    | [+, ×]            | [3,4,2]            | 数字直接加入输出队列                                                     |
| **/**    | [+, /]            | [3,4,2, ×]         | /与×同优先级且左结合，弹出×后再入栈/                                    |
| **(**    | [+, /, (]         | [3,4,2, ×]         | 左括号直接入栈                                                           |
| **1**    | [+, /, (]         | [3,4,2, ×, 1]      | 数字直接加入输出队列                                                     |
| **-**    | [+, /, (, -]      | [3,4,2, ×, 1]      | 栈顶是(，运算符直接入栈                                                 |
| **5**    | [+, /, (, -]      | [3,4,2, ×, 1,5]    | 数字直接加入输出队列                                                     |
| **)**    | [+, /]            | [3,4,2, ×,1,5,-]   | 弹出栈顶元素直到遇到(，丢弃括号对                                       |
| **处理完毕** | []           | [3,4,2,×,1,5,-,/,+] | 清空栈中所有剩余运算符                                                   |

---

### **关键步骤详解**

#### **第6步（处理 / 符号）**
- 当前栈顶是 ×，新符号是 /
- 计算优先级：× (优先级2) = / (优先级2)
- 结合性规则：二者均为左结合，所以需要先弹出栈顶 ×
- 操作：先弹出 × 加入输出队列，再将 / 压入栈

#### **第10步（处理 ) 符号）**
1. 开始持续弹出栈顶元素：
   - 弹出 - → 输出队列变为 [..., -]
   - 弹出 ( → 遇到左括号，停止弹出
2. 丢弃括号对，此时栈恢复为处理括号前的状态：[+, /]

#### **最终清空栈**
- 按栈顶到栈底的顺序依次弹出：
  - 弹出 / → 输出队列 [..., /]
  - 弹出 + → 输出队列 [..., +]

---

### **后缀表达式验证**
生成的后缀表达式：`3 4 2 × 1 5 - / +`

**计算过程**（使用栈式计算）：
1. 3 → 栈：[3]
2. 4 → 栈：[4,3]
3. 2 → 栈：[2,4,3]
4. × → 计算 4×2=8 → 栈：[8,3]
5. 1 → 栈：[1,8,3]
6. 5 → 栈：[5,1,8,3]
7. - → 计算 1-5=-4 → 栈：[-4,8,3]
8. / → 计算 8/-4=-2 → 栈：[-2,3]
9. + → 计算 3+(-2)=1 → 最终结果：1

---

### **算法核心原则**
1. **优先级统治**：高优先级运算符优先出栈（如×比+先处理）
2. **括号即边界**：括号内的运算符形成独立处理域
3. **左结合性法则**：当运算符优先级相同时，先出现的运算符先处理
4. **栈即调度场**：通过栈暂存未处理的运算符

通过这个具体案例的逐步推演，可以清晰理解Shunting-Yard算法如何在不改变运算逻辑的前提下，将人类易读的中缀表达式转换为计算机易处理的后缀表达式。
