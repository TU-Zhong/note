

正则表达式的 **断言（Assertions）** 是一类特殊的语法，用于**限制匹配发生的条件**，但不实际消耗字符（即不匹配具体内容）。断言帮助我们在匹配时检查某个位置的前后是否符合特定规则。

---

### **四大类断言**
断言分为四种类型，按方向和逻辑组合：

| 类型                | 语法          | 作用                                   | 示例说明                     |
|---------------------|---------------|----------------------------------------|----------------------------|
| **正向先行断言**    | `(?=pattern)` | 匹配位置**后面**必须满足 `pattern`     | `\d+(?=px)` 匹配后面是`px`的数字 |
| **负向先行断言**    | `(?!pattern)` | 匹配位置**后面**必须**不满足** `pattern` | `\d{3}(?!px)` 匹配非`px`结尾的3位数字 |
| **正向后行断言**    | `(?<=pattern)`| 匹配位置**前面**必须满足 `pattern`     | `(?<=\$)\d+` 匹配`$`后面的数字 |
| **负向后行断言**    | `(?<!pattern)`| 匹配位置**前面**必须**不满足** `pattern` | `(?<!\$)\d+` 匹配非`$`开头的数字 |

---

### **具体示例**
#### 1. **正向先行断言（Lookahead）**
用途：匹配后面紧跟特定内容的字符串。
```regex
\d+(?=px)    # 匹配后面是 "px" 的数字，如 "12px" 中的 "12"（不匹配"12em"）
```

#### 2. **负向先行断言（Negative Lookahead）**
用途：排除后面紧跟特定内容的字符串。
```regex
Windows(?!11) # 匹配 "Windows" 但后面不是 "11"，如 "Windows10"（不匹配 "Windows11"）
```

#### 3. **正向后行断言（Lookbehind）**
用途：匹配前面是特定内容的字符串。
```regex
(?<=\$)\d+    # 匹配前面是 "$" 的数字，如 "$100" 中的 "100"（不匹配 "€100"）
```

#### 4. **负向后行断言（Negative Lookbehind）**
用途：排除前面是特定内容的字符串。
```regex
(?<!\$)\d+    # 匹配前面不是 "$" 的数字，如 "100" 或 "€100" 中的 "100"（不匹配 "$100"）
```

---

### **断言的关键特性**
1. **不消耗字符**  
   断言只检查条件，不匹配实际内容。例如：  
   ```regex
   (?=a)b     # 匹配后面是 `a` 的 `b`，如 "ba" 中的 `b`（但实际无法成立，因为 `b` 后不是 `a`）
   ```

2. **支持复杂条件**  
   断言内可以写任意正则表达式，包括嵌套断言：
   ```regex
   (?=.*\d)(?=.*[A-Z])  # 密码必须包含数字和大写字母（多个条件叠加）
   ```

3. **变长后行断言的限制**  
   某些正则引擎（如 JavaScript）要求后行断言中的模式为固定长度（如 `(?<=ab|c)` 允许，但 `(?<=a+)` 不允许）。

---

### **常见应用场景**
#### 1. **密码强度验证**
```regex
^(?=.*\d)(?=.*[A-Z])(?=.*[a-z]).{8,}$  # 必须包含数字、大写、小写，长度≥8
```

#### 2. **提取特定内容**
```regex
(?<=<title>).*?(?=</title>)  # 提取 HTML 中的标题文本（不包含标签）
```

#### 3. **排除特定模式**
```regex
\d{3}(?!-\\d{2})  # 匹配3位数字，且后面不能是 "-两位数字"
```

#### 4. **边界检查**
```regex
(?<=\s)\w+         # 匹配以空格开头的单词（如 " hello" 中的 "hello"）
```

---

### **总结**
- **断言的作用**：增强正则的精确性，避免误匹配。
- **使用场景**：适合需要依赖上下文的条件匹配。
- **注意点**：不同编程语言/工具对断言的支持可能不同（尤其是变长后行断言）。
