

以下是一个结合实际应用场景的Java正则表达式练习题，融合了面向接口编程、异常处理、集合框架、泛型、设计模式等知识点：

题目：实现一个可扩展的智能数据脱敏系统

需求描述：
1. 开发一个数据脱敏工具，能够自动识别并处理文本中的敏感信息
2. 支持多种脱敏策略：手机号、身份证号、邮箱地址、银行卡号
3. 使用策略模式实现不同脱敏规则
4. 支持从文件读取数据，处理后将结果输出到新文件
5. 提供统计功能，记录每种类型数据的脱敏次数
6. 实现自定义异常体系
7. 使用工厂模式创建处理策略

代码结构大纲：
1. DataMaskingStrategy 接口 - 策略模式核心接口
2. 不同策略实现类（手机号、身份证等）
3. 策略工厂类
4. 脱敏上下文类
5. 自定义异常类
6. 文件处理工具类
7. 统计信息类
8. 主程序类
9. 单元测试类

实现亮点：
1. 策略模式：通过DataMaskingStrategy接口实现不同脱敏算法
2. 工厂模式：MaskingStrategyFactory统一管理策略对象
3. 正则表达式应用：每个策略类使用不同正则匹配敏感信息
4. 异常体系：自定义异常处理业务错误
5. 文件IO操作：使用try-with-resources处理资源
6. 统计功能：记录每种类型的脱敏次数
7. 单元测试：使用JUnit验证正则表达式有效性
8. 设计原则：开闭原则（易于扩展新策略）、单一职责原则

实际应用场景：
- 处理客户服务系统中的日志文件
- 金融行业用户数据导出时的隐私保护
- 医疗信息系统中的患者信息脱敏
- 生产环境日志敏感信息过滤

扩展建议：
1. 添加新的脱敏策略（如地址、姓名等）
2. 实现策略配置化（通过配置文件加载策略）
3. 添加多线程处理提高大文件处理效率
4. 集成Spring框架实现依赖注入
5. 添加日志记录功能
