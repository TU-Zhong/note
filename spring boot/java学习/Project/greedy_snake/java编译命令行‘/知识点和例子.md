

以下是 Java 编译命令行的核心知识点和实用示例总结：

---

### **一、基础命令**
1. **javac** - Java 编译器命令
   ```bash
   # 编译单个文件
   javac HelloWorld.java

   # 编译多个文件
   javac Main.java Util.java Model.java

   # 指定输出目录
   javac -d ./bin src/HelloWorld.java
   ```
<mark style="background: #FFF3A3A6;">javac -d ./bin src/HelloWorld.java这个命令生成的.class文件输出到哪里？</mark>
### **关键结论**

1. **如果 `test.java` 没有包声明（无 `package` 语句）**  
    → `.class` 文件直接生成在 **当前命令行所在目录**
    
    BASH
    
    `# 假设当前目录是 C:\project # 编译后会生成：C:\project\test.class`
    
2. **如果 `test.java` 有包声明（例如 `package com.demo;`）**  
    → `.class` 文件生成在 **当前目录下的包路径中**
    
    BASH
    
    `# 假设当前目录是 C:\project # 编译后会生成：C:\project\com\demo\test.class`
3. **java** - 运行字节码命令
   ```bash
   # 运行主类（无包结构）
   java HelloWorld

   # 运行带包结构的类
   java com.example.Main

   # 指定类路径
   java -classpath ./bin com.example.Main
   ```

---

### **二、核心参数**
1. **类路径控制**
   ```bash
   # 指定编译时的类路径
   javac -cp lib/*:config/ MyApp.java

   # 运行时的复杂类路径
   java -cp "lib/gson.jar:./bin" com.example.App
   ```

2. **编码设置**
   ```bash
   javac -encoding UTF-8 Main.java  # 处理中文等字符
   ```

3. **调试信息**
   ```bash
   javac -g:lines,vars,source DebugDemo.java  # 生成完整调试信息
   ```

4. **版本控制**
   ```bash
   javac -source 11 -target 11 OldCode.java  # 强制兼容旧版本
   ```

---

### **三、进阶场景**
1. **模块化编译（Java 9+）**
   ```bash
   javac --module-path libs -d out $(find src -name "*.java")
   ```

2. **注解处理**
   ```bash
   javac -processor com.example.MyProcessor AnnotatedClass.java
   ```

3. **多文件批量编译**
   ```bash
   javac @sources.txt  # sources.txt 内含文件列表
   ```

4. **生成文档链接**
   ```bash
   javac -XDuseUnsharedTable=true BigProject.java  # 提升大项目编译速度
   ```

---

### **四、典型错误处理**
1. **包目录不匹配**
   ```bash
   # 文件内容：package com.test;
   # 正确路径：mkdir -p com/test && mv MyClass.java com/test/
   javac com/test/MyClass.java
   ```

2. **缺少依赖项**
   ```bash
   javac -cp mysql-connector.jar DatabaseDemo.java
   ```

---

### **五、实用技巧**
- `javac -Xlint:unchecked` 开启详细警告检查
- 使用通配符编译：`javac src/*.java src/utils/*.java`
- 查看编译器版本：`javac -version`

---

通过灵活组合这些命令和参数，您可以应对从简单脚本到复杂项目的各种编译需求。建议通过 `javac -help` 和 `java -help` 查看完整参数列表。
