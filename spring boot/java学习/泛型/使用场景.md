泛型在编程中非常有用，尤其是在你需要**处理多种类型的数据**，同时希望代码**简洁、复用性强且类型安全**时。以下是一些常见的使用场景：

---

### 1. **集合类（如 List、Set、Map）**
Java 的集合框架（如 `ArrayList`、`HashMap` 等）广泛使用泛型，用来存储不同类型的数据。

#### 例子：
```java
List<String> names = new ArrayList<>();
names.add("Alice");
names.add("Bob");

List<Integer> ages = new ArrayList<>();
ages.add(25);
ages.add(30);
```

**为什么用泛型？**
- 如果没有泛型，集合中只能存储 `Object` 类型，取出数据时需要手动强制转换，容易出错。
- 使用泛型后，集合中只能存储指定类型的数据，编译器会检查类型，避免错误。

---

### 2. **自定义容器类**
如果你需要定义一个通用的容器类（如盒子、队列、栈等），泛型可以让你的容器支持多种数据类型。

#### 例子：
```java
class Box<T> {
    private T item;

    public void setItem(T item) {
        this.item = item;
    }

    public T getItem() {
        return item;
    }
}

Box<String> stringBox = new Box<>();
stringBox.setItem("Hello");

Box<Integer> integerBox = new Box<>();
integerBox.setItem(123);
```

**为什么用泛型？**
- 如果没有泛型，你需要为每种类型写一个单独的容器类（如 `BoxForString`、`BoxForInteger`），代码重复。
- 使用泛型后，一个容器类可以适用于任何类型。

---

### 3. **工具方法**
如果你需要编写一个工具方法，用于处理不同类型的数据，泛型可以让方法更通用。

#### 例子：
```java
public <T> void printArray(T[] array) {
    for (T item : array) {
        System.out.println(item);
    }
}

// 使用
String[] strings = {"A", "B", "C"};
Integer[] numbers = {1, 2, 3};

printArray(strings); // 输出: A B C
printArray(numbers); // 输出: 1 2 3
```

**为什么用泛型？**
- 如果没有泛型，你需要为每种类型写一个单独的方法（如 `printStringArray`、`printIntArray`）。
- 使用泛型后，一个方法可以处理所有类型的数组。

---

### 4. **接口和实现类**
如果你需要定义一个通用的接口或抽象类，泛型可以让接口或抽象类支持多种类型。

#### 例子：
```java
interface Pair<K, V> {
    K getKey();
    V getValue();
}

class OrderedPair<K, V> implements Pair<K, V> {
    private K key;
    private V value;

    public OrderedPair(K key, V value) {
        this.key = key;
        this.value = value;
    }

    public K getKey() { return key; }
    public V getValue() { return value; }
}

// 使用
Pair<String, Integer> pair = new OrderedPair<>("One", 1);
System.out.println(pair.getKey() + " -> " + pair.getValue()); // 输出: One -> 1
```

**为什么用泛型？**
- 如果没有泛型，接口只能处理特定类型，灵活性差。
- 使用泛型后，接口可以处理任意类型。

---

### 5. **避免类型强制转换**
泛型可以避免在代码中频繁使用强制类型转换，减少运行时错误。

#### 例子：
```java
// 没有泛型
List list = new ArrayList();
list.add("Hello");
String name = (String) list.get(0); // 需要强制转换

// 使用泛型
List<String> list = new ArrayList<>();
list.add("Hello");
String name = list.get(0); // 不需要强制转换
```

**为什么用泛型？**
- 没有泛型时，强制转换容易导致 `ClassCastException`。
- 使用泛型后，编译器会检查类型，避免运行时错误。

---

### 6. **框架和库设计**
如果你在开发框架或库，泛型可以让你的代码更通用，适用于多种场景。

#### 例子：
Java 的集合框架（如 `ArrayList`、`HashMap`）和工具类（如 `Optional`、`Stream`）都使用了泛型。

**为什么用泛型？**
- 框架和库需要处理各种类型的数据，泛型可以让它们更灵活、更通用。

---

### 7. **递归类型**
如果你需要定义递归结构（如树、链表等），泛型可以让你的结构支持多种类型。

#### 例子：
```java
class TreeNode<T> {
    private T value;
    private List<TreeNode<T>> children = new ArrayList<>();

    public TreeNode(T value) {
        this.value = value;
    }

    public void addChild(TreeNode<T> child) {
        children.add(child);
    }

    public List<TreeNode<T>> getChildren() {
        return children;
    }
}

// 使用
TreeNode<String> root = new TreeNode<>("Root");
root.addChild(new TreeNode<>("Child1"));
```

**为什么用泛型？**
- 如果没有泛型，你需要为每种类型写一个单独的结构类（如 `StringTreeNode`、`IntegerTreeNode`）。
- 使用泛型后，一个结构类可以适用于任何类型。

---

### 8. **通配符和类型限制**
如果你需要处理未知类型，或者限制类型的范围，泛型通配符（`?`）和上下界（`extends`、`super`）可以帮助你实现。

#### 例子：
```java
public static void printList(List<?> list) {
    for (Object item : list) {
        System.out.println(item);
    }
}

public static double sumOfList(List<? extends Number> list) {
    double sum = 0.0;
    for (Number number : list) {
        sum += number.doubleValue();
    }
    return sum;
}
```

**为什么用泛型？**
- 通配符和上下界可以让你的代码更灵活，同时保持类型安全。

---

### 总结
你会在以下场景中使用泛型：
1. 处理集合类（如 `List`、`Map`）。
2. 定义通用的容器类（如盒子、队列）。
3. 编写工具方法，处理多种类型的数据。
4. 定义通用的接口或抽象类。
5. 避免类型强制转换，减少错误。
6. 开发框架或库，让代码更通用。
7. 定义递归结构（如树、链表）。
8. 处理未知类型或限制类型的范围。

泛型让你的代码更灵活、更安全、更易复用，是 Java 中非常重要的特性！
