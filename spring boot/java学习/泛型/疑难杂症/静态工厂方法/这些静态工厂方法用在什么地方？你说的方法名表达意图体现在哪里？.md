

以下是 Java 中常见的静态工厂方法命名规范及其使用场景，这些方法名通过 **语义化的命名** 明确表达了对象的创建意图，让代码更易读、更直观：

---

### **1. `of()`：组合元素创建对象**
- **意图**：表示通过 **一组参数** 组合成一个新对象。
- **典型应用场景**：创建包含多个元素的集合或复合对象。

**示例：**
```java
// 创建一个包含多个元素的不可变集合
List<String> names = List.of("Alice", "Bob", "Charlie");

// 创建一个键值对 Pair 对象
Map.Entry<String, Integer> entry = Map.of("Score", 100);
```

---

### **2. `from()`：转换或适配创建对象**
- **意图**：表示从一个 **已有的对象或数据源** 转换/适配成当前对象。
- **典型应用场景**：类型转换、数据解析。

**示例：**
```java
// 从 Temporal 对象转换为 LocalDate
LocalDate date = LocalDate.from(temporal);

// 从 JSON 字符串转换为 User 对象
User user = User.fromJson("{\"name\":\"Alice\"}");
```

---

### **3. `valueOf()`：解析或转换参数为对象**
- **意图**：表示将参数 **解析或转换** 为当前类的实例。
- **典型应用场景**：基本类型包装类的解析、字符串到对象的转换。

**示例：**
```java
// 将字符串转换为 Integer
Integer num = Integer.valueOf("100");

// 将枚举字符串转换为枚举实例
DayOfWeek day = DayOfWeek.valueOf("MONDAY");
```

---

### **4. `getInstance()`：获取单例或特定实例**
- **意图**：表示获取一个 **特定环境下的实例**（可能是单例，也可能是根据上下文生成的实例）。
- **典型应用场景**：单例模式、工具类实例获取。

**示例：**
```java
// 获取默认时区的 Calendar 实例
Calendar calendar = Calendar.getInstance();

// 获取当前线程的数据库连接
Connection conn = Database.getInstance();
```

---

### **5. `newInstance()`：明确创建新对象**
- **意图**：表示 **每次调用都创建一个新实例**，强调对象的“新鲜性”。
- **典型应用场景**：反射创建对象、原型模式。

**示例：**
```java
// 反射创建 String 对象
String str = String.class.newInstance();

// 创建一个长度为 3 的字符串数组
String[] arr = Array.newInstance(String.class, 3);
```

---

### **6. `parse()`：从字符串解析为对象**
- **意图**：表示将 **字符串解析** 为当前类的实例。
- **典型应用场景**：日期、数字、自定义格式的解析。

**示例：**
```java
// 解析日期字符串
LocalDate date = LocalDate.parse("2023-10-01");

// 解析 JSON 字符串为对象
User user = User.parse("{\"name\":\"Alice\"}");
```

---

### **总结：方法名如何体现意图？**
| 方法名           | 意图关键词              | 示例代码                                  |
|------------------|-------------------------|------------------------------------------|
| `of()`           | **组合**               | `List.of("A", "B")`                     |
| `from()`         | **转换**               | `LocalDate.from(temporal)`              |
| `valueOf()`      | **解析/转换**          | `Integer.valueOf("100")`                |
| `getInstance()`  | **获取实例**（可能复用）| `Calendar.getInstance()`                |
| `newInstance()`  | **创建新实例**          | `String.class.newInstance()`            |
| `parse()`        | **字符串解析**          | `LocalDate.parse("2023-10-01")`         |

通过这些约定俗成的方法名，开发者可以一眼看出对象的创建方式，无需深入阅读文档或代码。这是 Java 设计中 **“代码即文档”** 理念的典型体现！
