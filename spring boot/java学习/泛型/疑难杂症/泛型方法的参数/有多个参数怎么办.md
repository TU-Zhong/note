

在 Java 泛型中，如果一个方法需要处理 **多个未知类型参数**，可以通过以下两种方式实现：

---

### **1. 使用多个泛型类型参数（如 `<T, U>`）**
如果方法需要处理 **多个独立且不同类型的参数**，可以在方法声明时定义多个泛型类型（如 `T`、`U`、`V` 等），用逗号分隔。

#### **语法：**
```java
修饰符 <T, U, ...> 返回类型 方法名(参数列表) { ... }
```

#### **示例：两个独立类型的参数**
```java
public static <T, U> void printTwoTypes(T param1, U param2) {
    System.out.println("参数1的类型: " + param1.getClass().getSimpleName());
    System.out.println("参数2的类型: " + param2.getClass().getSimpleName());
}

// 使用示例
printTwoTypes("Hello", 100); // T=String, U=Integer
// 输出:
// 参数1的类型: String
// 参数2的类型: Integer
```

---

### **2. 使用通配符 `?` 处理多个参数**
如果方法的参数是泛型类型，且需要接受 **不同类型的泛型实例**（但不需要具体操作类型），可以使用通配符 `?`。

#### **语法：**
```java
修饰符 返回类型 方法名(泛型类<?> param1, 泛型类<?> param2) { ... }
```

#### **示例：接受任意类型的集合**
```java
public static void printAll(List<?> list1, List<?> list2) {
    System.out.println("集合1: " + list1);
    System.out.println("集合2: " + list2);
}

// 使用示例
List<String> strList = List.of("A", "B");
List<Integer> intList = List.of(1, 2);
printAll(strList, intList); // 输出两个不同类型的集合
```

---

### **3. 多个参数共享同一类型约束**
如果多个参数需要 **类型一致**，只需定义一个泛型类型 `T`，并在所有参数中复用。

#### **示例：两个参数类型必须相同**
```java
public static <T> void checkEquality(T obj1, T obj2) {
    System.out.println("类型是否一致？" + obj1.getClass().equals(obj2.getClass()));
}

// 使用示例
checkEquality("Hello", "World"); // 类型一致 ✔️
checkEquality(100, "Java");      // 类型不一致，但编译通过 ❓（因为 T 被推断为 Object）
```

---

### **4. 多个参数的泛型类型存在关系**
如果多个类型参数之间有 **继承或实现关系**，可以用 `extends` 或 `super` 约束。

#### **示例：类型参数间的继承关系**
```java
public static <T, U extends T> void copyData(List<T> dest, List<U> src) {
    dest.addAll(src); // U 是 T 的子类，安全添加
}

// 使用示例
List<Number> numbers = new ArrayList<>();
List<Integer> integers = List.of(1, 2, 3);
copyData(numbers, integers); // 将 Integer 列表复制到 Number 列表中
```

---

### **5. 通配符的上下界限制**
如果多个参数需要 **类型范围约束**，可以用通配符的上下界（`? extends T` 或 `? super T`）。

#### **示例：方法接受两个 `Number` 的子类集合**
```java
public static void sumNumbers(List<? extends Number> list1, List<? extends Number> list2) {
    double sum = list1.stream().mapToDouble(Number::doubleValue).sum() +
                 list2.stream().mapToDouble(Number::doubleValue).sum();
    System.out.println("总和: " + sum);
}

// 使用示例
List<Integer> intList = List.of(1, 2);
List<Double> doubleList = List.of(1.1, 2.2);
sumNumbers(intList, doubleList); // 输出: 总和: 6.3
```

---

### **总结：如何选择符号？**
| 场景                                | 符号选择                     | 示例                                         |
|-----------------------------------|----------------------------|--------------------------------------------|
| **多个独立类型参数**               | 泛型类型参数 `<T, U>`         | `printTwoTypes(T param1, U param2)`        |
| **接受任意类型的泛型参数**         | 通配符 `<?>`                 | `printAll(List<?> list1, List<?> list2)`   |
| **参数类型必须一致**               | 单一泛型类型 `<T>`            | `checkEquality(T obj1, T obj2)`            |
| **参数类型有继承关系**             | 泛型类型约束 `<T, U extends T>` | `copyData(List<T> dest, List<U> src)`      |
| **参数类型有固定范围**             | 通配符上下界 `<? extends T>`  | `sumNumbers(List<? extends Number> list)` |

---

### **注意事项**
1. **通配符 `?` 无法直接操作类型**：
   - 通配符表示未知类型，不能在方法内部直接调用其具体方法（除非通过 `Object` 的方法）。
   - 如果需要操作具体类型，优先使用泛型类型参数（如 `<T>`）。
   
2. **避免过度使用通配符**：
   - 通配符适用于“只读”场景（如遍历集合元素），如果涉及写入操作，建议用泛型类型参数或上下界约束。

3. **类型擦除的影响**：
   - 泛型类型参数在编译后会擦除为上限类型（如 `Object`），运行时无法获取具体类型信息。

---

### **最终建议**
- **优先使用泛型类型参数**（如 `<T, U>`）：当需要明确操作类型时。
- **灵活使用通配符**（`?`）：当方法只需“接受多种泛型类型”且不操作具体类型时。

通过合理选择符号，可以编写出既灵活又类型安全的泛型代码！ 🚀
