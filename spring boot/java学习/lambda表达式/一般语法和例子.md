### 一、Lambda 表达式的语法结构
Lambda 表达式的基本语法如下：
```java
(parameters) -> { expression or statements }
```
由三部分组成：
1. **参数列表**（Parameters）：方法参数，括号 `()` 包裹，参数类型可省略
2. **箭头符号** `->`：分隔参数和方法体
3. **方法体**（Body）：单行语句可省略 `{}`，多行或返回语句需 `{}`

---

### 二、具体语法规则及示例

#### 1. **无参数情况**
```java
// 语法
() -> { ... }

// 示例
Runnable runnable = () -> System.out.println("无参数Lambda");
new Thread(runnable).start();
```

#### 2. **单参数（可省略括号）**
```java
// 语法
(param) -> { ... }
param -> { ... }  // 省略括号

// 示例
Consumer<String> printer = s -> System.out.println(s);
printer.accept("Hello");
```

#### 3. **多参数（必须有括号）**
```java
// 语法
(p1, p2) -> { ... }

// 示例
Comparator<Integer> comparator = (a, b) -> a - b;
List<Integer> list = Arrays.asList(3, 1, 4);
list.sort(comparator); // 排序结果 [1,3,4]
```

#### 4. **指定参数类型**
```java
// 语法
(类型1 p1, 类型2 p2) -> { ... }

// 示例（虽然不推荐，但演示明确类型）
BinaryOperator<Integer> add = (Integer x, Integer y) -> x + y;
System.out.println(add.apply(3, 5)); // 输出 8
```

#### 5. **方法体包含多行语句**
必须用 `{}` 包裹，且需要 `return`（如果有返回值）：
```java
// 语法
(param) -> {
    statement1;
    statement2;
    return value;
}

// 示例
Function<Integer, String> converter = num -> {
    String result = "Value: " + num;
    return result;
};
System.out.println(converter.apply(10)); // 输出 "Value: 10"
```

#### 6. **返回值简化写法**
单行表达式自动返回结果，无需 `return`：
```java
// 语法
(param) -> expression

// 示例
Function<Integer, Integer> square = x -> x * x;
System.out.println(square.apply(4)); // 输出 16
```

### 三、类型推断机制
Java 编译器能通过上下文推断 Lambda 参数类型，因此通常可以省略类型声明：
```java
// 传统写法（明确类型）
Comparator<String> c1 = (String s1, String s2) -> s1.compareTo(s2);

// 简化写法（类型推断）
Comparator<String> c2 = (s1, s2) -> s1.compareTo(s2);
```

---

### 四、函数式接口的概念
Lambda 表达式只能用于 **函数式接口**（Functional Interface），即仅有一个抽象方法的接口，例如：
- `Runnable` → `void run()`
- `Consumer<T>` → `void accept(T t)`
- `Predicate<T>` → `boolean test(T t)`
- `Function<T, R>` → `R apply(T t)`

#### 自定义函数式接口示例：
```java
@FunctionalInterface
interface StringProcessor {
    String process(String input);
}

public class Demo {
    public static void main(String[] args) {
        StringProcessor toUpper = s -> s.toUpperCase();
        System.out.println(toUpper.process("hello")); // 输出 "HELLO"
    }
}
```

### 五、变量捕获规则
Lambda 表达式可以访问外层作用域的变量，但要求变量是 `final` 或事实上的 `final`（未被修改）：
```java
int base = 10; // 事实上的 final
Function<Integer, Integer> adder = x -> x + base;
System.out.println(adder.apply(5)); // 输出 15

// 如果后续修改 base，编译会报错！
// base = 20; // 错误: 从Lambda表达式引用的局部变量必须是final或事实final
```

---

### 六、对比传统匿名内部类
Lambda 表达式是匿名内部类的语法糖，但更简洁：
```java
// 匿名内部类写法
button.addActionListener(new ActionListener() {
    @Override
    public void actionPerformed(ActionEvent e) {
        System.out.println("按钮点击");
    }
});

// Lambda 写法
button.addActionListener(e -> System.out.println("按钮点击"));
```

---

### 七、注意事项
4. **不能独立存在**：必须依附于函数式接口的实例。
5. **`this` 关键字**：Lambda 中的 `this` 指向外层类，而非 Lambda 自身。
6. **异常处理**：Lambda 可能抛出检查型异常（Checked Exception）时，接口的抽象方法需声明该异常。
