在 Java 中，Lambda 表达式的返回值取决于它的 **方法体** 是 **单条语句** 还是 **多条语句**。以下是详细说明：

---

### 1. **单条语句**
如果 Lambda 表达式的方法体只有一条语句，那么：
- **自动返回**：这条语句的结果就是 Lambda 的返回值。
- **无需 `return` 关键字**。

**示例**：
```java
// Lambda 表达式返回一个数的平方
Function<Integer, Integer> square = x -> x * x;
System.out.println(square.apply(5)); // 输出 25
```
**解释**：  
`x * x` 是单条语句，它的结果（`25`）自动作为 Lambda 的返回值。

---

### 2. **多条语句**
如果 Lambda 表达式的方法体有多条语句，那么：
- **必须用 `{}` 包裹**。
- **需要显式使用 `return` 关键字** 来返回值。

**示例**：
```java
// Lambda 表达式返回一个数的平方，并打印日志
Function<Integer, Integer> squareAndLog = x -> {
    int result = x * x;
    System.out.println("计算平方：" + result);
    return result; // 必须显式返回
};
System.out.println(squareAndLog.apply(5)); // 输出 "计算平方：25" 和 25
```
**解释**：  
因为方法体有多条语句，所以需要用 `{}` 包裹，并且显式使用 `return` 返回结果。

---

### 3. **无返回值**
如果 Lambda 表达式没有返回值（比如实现了 `Runnable` 接口），则不需要 `return` 语句。

**示例**：
```java
// Lambda 表达式执行一个无返回值的任务
Runnable task = () -> System.out.println("任务完成");
task.run(); // 输出 "任务完成"
```
**解释**：  
`Runnable` 接口的 `run()` 方法没有返回值，因此 Lambda 表达式也不需要返回值。

---

### 总结
| 方法体类型   | 是否需要 `{}` | 是否需要 `return` | 示例                                      |
|--------------|---------------|-------------------|-------------------------------------------|
| **单条语句** | 不需要        | 不需要            | `x -> x * x`                              |
| **多条语句** | 需要          | 需要              | `x -> { int result = x * x; return result; }` |
| **无返回值** | 视情况        | 不需要            | `() -> System.out.println("任务完成")`    |

通过以上规则，你可以根据需要编写合适的 Lambda 表达式！
