**函数式接口**（Functional Interface）是 Java 8 引入的核心概念之一，它是 Lambda 表达式的基础。以下是函数式接口的四大特征：

---

### 1. **只有一个抽象方法**
函数式接口的核心特征是它必须 **只有一个抽象方法**（Single Abstract Method, SAM）。这个方法是 Lambda 表达式的目标方法。

**示例**：
```java
@FunctionalInterface
interface MyFunction {
    void run(); // 唯一的抽象方法
}
```

**注意**：
- 默认方法（`default`）和静态方法（`static`）不算抽象方法。
- 如果接口有多个抽象方法，就不是函数式接口。

---

### 2. **可以有默认方法**
函数式接口可以包含 **默认方法**（`default` 方法），因为默认方法有实现，不是抽象的。

**示例**：
```java
@FunctionalInterface
interface MyFunction {
    void run(); // 唯一的抽象方法

    default void log(String message) { // 默认方法
        System.out.println("日志: " + message);
    }
}
```

---

### 3. **可以有静态方法**
函数式接口可以包含 **静态方法**（`static` 方法），因为静态方法有实现，不是抽象的。

**示例**：
```java
@FunctionalInterface
interface MyFunction {
    void run(); // 唯一的抽象方法

    static void printVersion() { // 静态方法
        System.out.println("版本: 1.0");
    }
}
```

---

### 4. **可以用 `@FunctionalInterface` 注解标记**
虽然不是必须的，但建议用 `@FunctionalInterface` 注解标记函数式接口，以便编译器检查其是否符合函数式接口的定义。

**示例**：
```java
@FunctionalInterface
interface MyFunction {
    void run(); // 唯一的抽象方法
}
```

**作用**：
- 明确表示这是一个函数式接口。
- 如果接口不符合函数式接口的定义（比如有多个抽象方法），编译器会报错。

---

### 总结
| 特征                  | 说明                                                                 |
|-----------------------|--------------------------------------------------------------------|
| **只有一个抽象方法**   | 函数式接口的核心特征，Lambda 表达式的目标方法。                        |
| **可以有默认方法**     | 默认方法不算抽象方法，不影响函数式接口的定义。                           |
| **可以有静态方法**     | 静态方法不算抽象方法，不影响函数式接口的定义。                           |
| **可用 `@FunctionalInterface` 注解** | 明确标记函数式接口，编译器会检查是否符合定义。 |

---

### 示例代码
```java
@FunctionalInterface
interface MyFunction {
    void run(); // 唯一的抽象方法

    default void log(String message) { // 默认方法
        System.out.println("日志: " + message);
    }

    static void printVersion() { // 静态方法
        System.out.println("版本: 1.0");
    }
}

public class Demo {
    public static void main(String[] args) {
        MyFunction task = () -> System.out.println("任务执行中...");
        task.run();       // 输出 "任务执行中..."
        task.log("完成"); // 输出 "日志: 完成"
        MyFunction.printVersion(); // 输出 "版本: 1.0"
    }
}
```

---

### 类比生活中的例子：
函数式接口就像一个 **任务清单**：
- **唯一抽象方法**：清单上只有一个核心任务（比如“做饭”）。
- **默认方法**：清单上还可以有一些可选步骤（比如“买菜”）。
- **静态方法**：清单上还可以有一些固定的规则（比如“每周日做一次饭”）。
- **`@FunctionalInterface` 注解**：清单上明确写着“这是一个任务清单”，如果写错了（比如写了多个核心任务），就会有人提醒你。

通过理解这四大特征，可以更好地掌握函数式接口的设计和使用！
