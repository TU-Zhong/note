在众多的设计模式中，有一些因为其解决的问题非常普遍，或者在核心库和流行框架中被广泛应用，所以被认为是“最常用”的。以下这些模式在 Java 开发中尤为常见和重要：

1. **单例模式 (Singleton Pattern)**
    
    - **为什么常用**：确保系统中某个类只有一个实例，并提供全局访问点。非常适用于管理共享资源，如配置信息、线程池、日志对象、数据库连接池等。几乎每个项目都会在某些地方用到。
2. **工厂方法模式 (Factory Method Pattern)**
    [[工厂模式核心思想]]
    
    - **为什么常用**：允许子类决定实例化哪个类，将对象的创建过程封装起来。在 JDK 和各种框架中广泛用于解耦对象的创建和使用，例如 `java.util.Calendar.getInstance()`、`java.text.NumberFormat.getInstance()`。
3. **建造者模式 (Builder Pattern)**
    
    - **为什么常用**：当一个对象有多个可选参数或构建过程复杂时，使用建造者模式可以使代码更清晰、易读，并能优雅地处理默认值。`StringBuilder` 是一个经典例子，许多 API 客户端（如 OkHttp 的 Request 构建）也广泛使用此模式。
4. **适配器模式 (Adapter Pattern)**
    
    - **为什么常用**：解决接口不兼容的问题，让原本无法一起工作的类能够协同工作。在集成第三方库或处理遗留系统时非常有用。Java I/O 中的 `InputStreamReader` (字节流转字符流) 就是一个典型应用。
5. **装饰器模式 (Decorator Pattern)**
    
    - **为什么常用**：动态地给对象添加额外的功能，比继承更灵活。Java I/O 类库是装饰器模式的典范，如 `BufferedInputStream` 为 `InputStream` 添加缓冲功能。
6. **代理模式 (Proxy Pattern)**
    
    - **为什么常用**：控制对对象的访问。在很多场景下都有应用，如：
        - **远程代理** (RMI)
        - **虚拟代理** (延迟加载大对象)
        - **保护代理** (权限控制)
        - **动态代理** (`java.lang.reflect.Proxy`) 是 Spring AOP (事务、安全、日志)、RPC 框架等技术的核心。
7. **外观模式 (Facade Pattern)**
    
    - **为什么常用**：为一个复杂的子系统提供一个简化的、统一的接口。使得子系统更容易使用。例如，SLF4J (Simple Logging Facade for Java) 为多种日志系统提供了一个统一的门面。
8. **观察者模式 (Observer Pattern)**
    
    - **为什么常用**：定义对象间的一对多依赖，当一个对象状态改变时，所有依赖者都会收到通知。广泛用于事件驱动编程，如 GUI 的事件监听器 (Swing/JavaFX 中的 `ActionListener`)、`PropertyChangeListener`、以及各种消息发布/订阅系统。
9. **策略模式 (Strategy Pattern)**
    <mark style="background: #FFF3A3A6;">苹果老农要求按照他的需求进行排序</mark>
    - **为什么常用**：定义一系列算法，将每个算法封装起来，并使它们可以互相替换。使得算法的选择可以独立于使用它的客户端。例如，`Collections.sort()` 可以接受不同的 `Comparator` (策略)；线程池的拒绝策略。
10. **模板方法模式 (Template Method Pattern)**
    [[模板方法模式核心思想]]
    - **为什么常用**：在一个方法中定义一个算法的骨架，并将一些步骤延迟到子类中实现。常用于框架设计，提供扩展点。例如，Java Servlet 的 `HttpServlet` 中的 `service` 方法和 `doGet/doPost` 方法的关系；一些抽象基类中定义了处理流程，子类实现具体细节。
11. **迭代器模式 (Iterator Pattern)**
    [[迭代器模式]]
    - **为什么常用**：提供一种统一的方式遍历集合中的元素，而无需暴露集合的内部结构。Java 集合框架的核心组成部分，几乎所有使用集合的 Java 开发者都在不知不觉中使用它。

**总结一下，哪些“最常用”：**

如果让我选出日常开发中最常直接接触或间接使用（通过框架）的模式，我会重点突出：

- **创建型**：**单例 (Singleton)**、**工厂方法 (Factory Method)**、**建造者 (Builder)**
- **结构型**：**适配器 (Adapter)**、**装饰器 (Decorator)**、**代理 (Proxy)**、**外观 (Facade)**
- **行为型**：**观察者 (Observer)**、**策略 (Strategy)**、**模板方法 (Template Method)**、**迭代器 (Iterator)**

当然，“最常用”也会根据具体的应用领域（例如，Web 开发、嵌入式、游戏开发等）和个人经验略有不同，但上面列出的这些模式具有非常广泛的普适性。理解并掌握它们对于写出高质量的 Java 代码非常有帮助。
