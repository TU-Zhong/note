

以下是 **Java 内部类**和**匿名类**的详细知识点及示例，帮助你更好地理解它们的用途和区别。

---

### 一、**内部类（Inner Class）**
内部类是定义在另一个类内部的类，主要分为以下 4 种类型：

#### 1. **成员内部类（Member Inner Class）**
- **定义**：直接定义在外部类的内部，与成员变量平级。
- **特点**：
  - 可以直接访问外部类的所有成员（包括 `private`）。
  - 必须依赖外部类的实例才能创建对象。

**示例**：
```java
public class Outer {
    private String outerField = "Outer Field";

    // 成员内部类
    class Inner {
        void display() {
            System.out.println("访问外部类的字段: " + outerField);
        }
    }

    public static void main(String[] args) {
        Outer outer = new Outer();
        Outer.Inner inner = outer.new Inner(); // 通过外部类实例创建内部类对象
        inner.display(); // 输出: 访问外部类的字段: Outer Field
    }
}
```

---

#### 2. **静态内部类（Static Nested Class）**
- **定义**：用 `static` 修饰的内部类。
- **特点**：
  - **不依赖外部类实例**，可以直接创建。
  - 只能访问外部类的静态成员。

**示例**：
```java
public class Outer {
    private static String staticOuterField = "Static Outer Field";

    // 静态内部类
    static class StaticInner {
        void display() {
            System.out.println("访问外部类的静态字段: " + staticOuterField);
        }
    }

    public static void main(String[] args) {
        // 直接创建静态内部类对象
        Outer.StaticInner staticInner = new Outer.StaticInner();
        staticInner.display(); // 输出: 访问外部类的静态字段: Static Outer Field
    }
}
```

---

#### 3. **局部内部类（Local Inner Class）**
- **定义**：定义在方法或代码块内部的类。
- **特点**：
  - 作用域仅限于所在的方法或代码块。
  - 可以访问外部类的成员，但**只能访问方法内的 `final` 或等效 final 的局部变量**。

**示例**：
```java
public class Outer {
    private String outerField = "Outer Field";

    public void method() {
        final String localVar = "Local Variable";

        // 局部内部类（定义在方法内部）
        class LocalInner {
            void display() {
                System.out.println("外部类字段: " + outerField);
                System.out.println("局部变量: " + localVar);
            }
        }

        LocalInner inner = new LocalInner();
        inner.display();
    }

    public static void main(String[] args) {
        Outer outer = new Outer();
        outer.method(); 
        // 输出:
        // 外部类字段: Outer Field
        // 局部变量: Local Variable
    }
}
```

---

#### 4. **匿名内部类（Anonymous Inner Class）**
- **定义**：没有名字的局部内部类，通常用于快速实现接口或抽象类。
- **特点**：
  - **直接通过 `new` 关键字定义并实例化**。
  - 适用于一次性使用的场景（如事件监听器）。

### 二、**匿名类（Anonymous Class）**
匿名类是一种没有名字的局部内部类，通常用于简化代码。

#### 1. **语法规则**
```java
接口/父类 对象名 = new 接口/父类() {
    // 实现接口方法或重写父类方法
};
```

#### 2. **示例场景**
##### (1) 实现接口（如 `Runnable`）
```java
public class Demo {
    public static void main(String[] args) {
        // 匿名类实现 Runnable 接口
        Runnable task = new Runnable() {
            @Override
            public void run() {
                System.out.println("匿名类实现的 run() 方法");
            }
        };

        new Thread(task).start(); // 输出: 匿名类实现的 run() 方法
    }
}
```
## Q:
这个task还是Runnable类吗？为什么可以直接重写？

##### (2) 继承抽象类
```java
abstract class Animal {
    abstract void speak();
}

public class Demo {
    public static void main(String[] args) {
        // 匿名类继承 Animal
        Animal dog = new Animal() {
            @Override
            void speak() {
                System.out.println("汪汪汪！");
            }
        };

        dog.speak(); // 输出: 汪汪汪！
    }
}
```

##### (3) 事件监听器（如 Swing 按钮点击）
```java
import javax.swing.*;
import java.awt.event.ActionEvent;
import java.awt.event.ActionListener;

public class ButtonDemo {
    public static void main(String[] args) {
        JFrame frame = new JFrame();
        JButton button = new JButton("点击我");

        // 匿名类实现按钮点击事件
        button.addActionListener(new ActionListener() {
            @Override
            public void actionPerformed(ActionEvent e) {
                System.out.println("按钮被点击了！");
            }
        });

        frame.add(button);
        frame.pack();
        frame.setVisible(true);
    }
}
```

### 三、**内部类 vs 匿名类**

| **特性**         | **内部类**                          | **匿名类**                          |
|------------------|-------------------------------------|-------------------------------------|
| **名称**         | 有明确类名                          | 无类名                              |
| **重用性**       | 可多次使用                          | 一次性使用                          |
| **代码位置**     | 可定义在类、方法或代码块中          | 通常在方法内直接定义                |
| **适用场景**     | 需要多次实例化的逻辑                | 快速实现接口/抽象类（如事件监听）   |
| **访问权限**     | 可访问外部类成员和 `final` 局部变量 | 同局部内部类                        |

---

### 四、**关键注意事项**
1. **成员内部类持有外部类的引用**：  
   非静态内部类会隐式持有外部类的实例，可能导致内存泄漏（可通过静态内部类避免）。

2. **匿名类的变量限制**：  
   在匿名类中访问局部变量时，变量必须是 `final` 或等效 final（Java 8+）。

3. **静态上下文限制**：  
   非静态内部类不能在静态方法中直接实例化（需先创建外部类对象）。

---

### 五、**总结**
- **内部类**适合需要封装紧密逻辑的场景（如事件处理器、迭代器）。
- **匿名类**适合简化代码，快速实现接口或抽象类。
- 根据需求选择合适类型：需要复用选内部类，一次性逻辑用匿名类。
