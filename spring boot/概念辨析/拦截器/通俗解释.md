

我试着用买奶茶的例子来解释拦截器，应该很好懂：

🍦 想象你去奶茶店点单：
1. **preHandle**（点单前检查）
   → 店员先检查你："带钱了吗？会员卡出示下？"
   → 没带钱就不给你点单（拦截请求）

2. **做奶茶**（Controller处理请求）
   → 检查通过后，店员开始制作你的奶茶

3. **postHandle**（奶茶交给你之前）
   → 店员问："要加吸管吗？需要袋子吗？"
   → 最后调整奶茶的包装（修改返回结果）

4. **afterCompletion**（你离开后）
   → 店员擦桌子、洗杯子（收尾工作）

🔧 **拦截器就是帮你自动做这些"检查"和"收尾"的智能小助手**
- 适合做：登录检查、记录访问日志、计算程序耗时...
- 特点：可以同时设置多个检查步骤（比如先查是否登录，再查权限）

⚡ 和过滤器（Filter）的区别：
- 过滤器像大楼保安（只管进出，不知道你要去哪层）
- 拦截器像公司前台（知道你要找哪个部门，能查工牌）
