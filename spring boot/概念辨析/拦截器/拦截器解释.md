[[spring boot/概念辨析/拦截器/通俗解释|通俗解释]]

在Spring Boot中，拦截器（Interceptor）是一种强大的请求处理机制，允许你在Controller执行前后插入自定义逻辑。以下是对拦截器的详细说明：

### 1. 拦截器核心机制
- **执行阶段**：
  - **preHandle**：请求到达Controller前执行（适合权限校验）
  - **postHandle**：Controller执行后，视图渲染前执行（可修改ModelAndView）
  - **afterCompletion**：整个请求完成后执行（适合资源清理）

- **对比Filter**：
  - 过滤器属于Servlet层级，拦截器基于Spring MVC
  - 拦截器可以获取HandlerMethod信息
  - 拦截器能使用Spring依赖注入

### 2. 创建拦截器（代码示例）
```java
@Component
public class AuthInterceptor implements HandlerInterceptor {

    @Autowired
    private AuthService authService;

    @Override
    public boolean preHandle(HttpServletRequest request, 
                            HttpServletResponse response, 
                            Object handler) throws Exception {
        String token = request.getHeader("Authorization");
        if (!authService.validateToken(token)) {
            response.sendError(401, "Unauthorized");
            return false; // 中断请求
        }
        return true;
    }

    @Override
    public void postHandle(HttpServletRequest request,
                          HttpServletResponse response,
                          Object handler,
                          ModelAndView modelAndView) {
        log.info("Request processing time: {}ms", 
                System.currentTimeMillis() - startTime.get());
    }

    @Override
    public void afterCompletion(HttpServletRequest request,
                               HttpServletResponse response,
                               Object handler,
                               Exception ex) {
        // 清理线程本地变量等资源
    }
}
```

### 3. 高级配置方式
```java
@Configuration
public class WebConfig implements WebMvcConfigurer {

    @Autowired
    private AuthInterceptor authInterceptor;

    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(authInterceptor)
                .addPathPatterns("/api/**")  // 拦截API路径
                .excludePathPatterns("/api/public/**") // 排除公开接口
                .order(1); // 设置执行顺序
    }
}
```

### 4. 关键注意事项
- **执行顺序**：
  - preHandle按注册顺序正序执行
  - postHandle和afterCompletion按注册逆序执行

- **路径匹配**：
  - 支持Ant风格：`/admin/**`、`/user/*/profile`
  - 支持排除模式：`excludePathPatterns("/static/**")`

- **性能影响**：
  - 避免在preHandle中执行耗时操作
  - 谨慎使用ThreadLocal，需确保及时清理

### 5. 典型应用场景
1. **认证授权**：验证JWT令牌
2. **日志审计**：记录请求参数和响应时间
3. **数据预处理**：自动解析多语言头信息
4. **防重复提交**：基于请求指纹的幂等校验
5. **流量控制**：API的限流处理

### 6. 高级技巧
- 使用`@Order`注解控制多个拦截器的顺序
- 通过`HandlerMethod`参数获取Controller方法元数据
- 配合自定义注解实现细粒度控制：
```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface AuditLog {
    String value() default "";
}

// 在拦截器中检查方法注解
if (handlerMethod.hasMethodAnnotation(AuditLog.class)) {
    // 记录审计日志
}
```

### 7. 常见问题排查
- **拦截器不生效**：
  - 检查是否配置了`@EnableWebMvc`
  - 确认路径匹配规则是否正确
  - 确保没有通过`excludePathPatterns`排除

- **Bean注入失败**：
  - 确认拦截器类被Spring管理（使用`@Component`）
  - 在配置类中使用@Autowired注入拦截器实例

拦截器相比AOP更适合处理与HTTP请求生命周期密切相关的横切关注点，而业务层的横切逻辑建议使用AOP实现。合理使用拦截器可以显著提升系统的可维护性和安全性。
