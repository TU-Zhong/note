在 Spring Boot 中，bean 的**懒加载** (lazy loading) 是一种延迟初始化 bean 的机制。

通常情况下，Spring IoC 容器会在启动时创建并初始化所有**单例** (singleton)作用域的 bean。但是，如果一个 bean 被配置为懒加载，那么只有当它**第一次被其他 bean 引用或通过 `ApplicationContext` 显式请求**时，Spring 容器才会创建和初始化这个 bean 的实例。

---

## 懒加载的好处

- **加快应用启动速度** 🚀：对于那些在应用启动初期并非必需的 bean，懒加载可以避免在启动时就创建它们，从而缩短应用的启动时间。这在大型应用中尤其明显。
- **节省资源** 💾：如果某些 bean 在应用的整个生命周期中都可能不会被使用，懒加载可以避免为它们分配不必要的内存和计算资源。

---

## 如何配置懒加载

在 Spring Boot 中，有几种方式可以配置 bean 的懒加载：

1. **使用 `@Lazy` 注解**:
    
    - 可以直接在 `@Component`、`@Service`、`@Repository` 等组件注解的类上添加 `@Lazy` 注解。
        
        Java
        
        ```
        import org.springframework.context.annotation.Lazy;
        import org.springframework.stereotype.Component;
        
        @Lazy
        @Component
        public class LazyService {
            public LazyService() {
                System.out.println("LazyService initialized!");
            }
            // ...
        }
        ```
        
    - 也可以在 `@Bean` 方法上使用 `@Lazy` 注解，当通过 Java 配置类定义 bean 时。
        
        Java
        
        ```
        import org.springframework.context.annotation.Bean;
        import org.springframework.context.annotation.Configuration;
        import org.springframework.context.annotation.Lazy;
        
        @Configuration
        public class AppConfig {
        
            @Lazy
            @Bean
            public AnotherService anotherService() {
                System.out.println("AnotherService initialized!");
                return new AnotherService();
            }
        }
        ```
        
    - 还可以在注入点（例如 `@Autowired` 字段或构造函数参数）上使用 `@Lazy` 注解。这表示被注入的 bean 应该被懒加载（如果它还没有被初始化的话）。
        
        Java
        
        ```
        import org.springframework.beans.factory.annotation.Autowired;
        import org.springframework.context.annotation.Lazy;
        import org.springframework.stereotype.Service;
        
        @Service
        public class MyService {
        
            private final LazyService lazyService;
        
            @Autowired
            public MyService(@Lazy LazyService lazyService) {
                this.lazyService = lazyService;
                System.out.println("MyService initialized!");
            }
        
            public void doSomething() {
                // 第一次调用 lazyService 的方法时，才会真正创建 LazyService 实例
                // lazyService.someMethod();
                System.out.println("Doing something in MyService.");
            }
        }
        ```
        
2. **全局配置 (Spring Boot 2.2 及更高版本)**: 可以在 `application.properties` 或 `application.yml` 文件中设置全局懒加载：
    
    - `application.properties`:
        
        Properties
        
        ```
        spring.main.lazy-initialization=true
        ```
        
    - `application.yml`:
        
        YAML
        
        ```
        spring:
          main:
            lazy-initialization: true
        ```
        
    
    当这个属性设置为 `true` 时，所有 bean 默认都会被懒加载，除非显式地使用 `@Lazy(false)` 注解来覆盖。
    

---

## 注意事项

- **循环依赖**：懒加载可能会影响 Spring 处理循环依赖的方式。在某些情况下，懒加载可以帮助解决循环依赖问题，但在其他情况下可能会掩盖问题或导致不同的行为。
- **启动时错误发现**：非懒加载的 bean 会在应用启动时被创建，这有助于及早发现配置错误或初始化问题。如果 bean 被懒加载，这些问题可能只有在该 bean 第一次被使用时才会暴露出来。
- **并非所有 bean 都适合懒加载**：有些 bean 可能需要在应用启动时就完成初始化，例如执行一些预加载任务或注册监听器等。

简单来说，bean 的懒加载就是**按需创建**，而不是**启动时立即创建**。这是一种优化应用性能和资源利用率的有效手段。
