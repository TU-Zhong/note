在当前的`LoginInterceptor`类中，`StringRedisTemplate`不会自动注入，因为这个拦截器不是由Spring容器直接管理的。当前的代码会导致`NullPointerException`。

问题在于`LoginInterceptor`是在`MvcConfig`中通过`new LoginInterceptor()`手动创建的，而不是通过Spring的依赖注入创建的，所以`@Autowired`注解在这里不会生效。

有两种解决方案：

1. **构造函数注入**（推荐）：
```java

LoginInterceptor.java

src/main/java/com/hmdp/utils

Apply

public class LoginInterceptor implements 

HandlerInterceptor {

    private StringRedisTemplate stringRedisTemplate;

    public LoginInterceptor(StringRedisTemplate 

    stringRedisTemplate) {

        this.stringRedisTemplate = 

        stringRedisTemplate;

    }

    @Override

    public boolean preHandle(HttpServletRequest 

    request, HttpServletResponse response, Object 

    handler) throws Exception {

        // 现有代码...

    }

    // 其他方法...

}
```
然后在`MvcConfig`中修改：
```java

MvcConfig.java

src/main/java/com/hmdp/config

Apply

@Configuration

public class MvcConfig implements WebMvcConfigurer {

    @Resource

    private StringRedisTemplate stringRedisTemplate;

    @Override

    public void addInterceptors(InterceptorRegistry 

    registry) {

        registry.addInterceptor(new LoginInterceptor

        (stringRedisTemplate))

                .excludePathPatterns(

                        // 现有排除路径...

                );

    }

}
```
2. **将拦截器注册为Bean**：
```java

LoginInterceptor.java

src/main/java/com/hmdp/utils

Apply

@Component

public class LoginInterceptor implements 

HandlerInterceptor {

    @Autowired

    private StringRedisTemplate stringRedisTemplate;

    // 现有方法...

}
```
然后在
```java
`MvcConfig`

中：

MvcConfig.java

src/main/java/com/hmdp/config

Apply

@Configuration

public class MvcConfig implements WebMvcConfigurer {

    @Autowired

    private LoginInterceptor loginInterceptor;

    @Override

    public void addInterceptors(InterceptorRegistry 

    registry) {

        registry.addInterceptor(loginInterceptor)

                .excludePathPatterns(

                        // 现有排除路径...

                );

    }

}
```
第一种方法（构造函数注入）更推荐，因为它更明确地表达了依赖关系，并且不需要将拦截器注册为Spring Bean。
