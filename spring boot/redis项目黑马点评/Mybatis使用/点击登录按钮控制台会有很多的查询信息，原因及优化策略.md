这是正常的业务流程，因为在

`BlogController`中的`queryHotBlog`

方法会查询用户信息：
```java

BlogController.java

Copy

@GetMapping("/hot")

public Result queryHotBlog(@RequestParam(value = 

"current", defaultValue = "1") Integer current) {

    // 查询热门博客

    Page<Blog> page = blogService.query()

            .orderByDesc("liked")

            .page(new Page<>(current, SystemConstants.

            MAX_PAGE_SIZE));

    List<Blog> records = page.getRecords();

    // 查询用户

    records.forEach(blog ->{

        Long userId = blog.getUserId();

        User user = userService.getById(userId);  // 这里

        会产生多次用户查询

        blog.setName(user.getNickName());

        blog.setIcon(user.getIcon());

    });

    return Result.ok(records);

}
```
为了优化这些查询，你可以：

1. 使用批量查询替代循环查询用户信息
2. 添加缓存减少重复查询
3. 使用联表查询一次性获取博客和用户信息

例如，优化后的代码可以是：
