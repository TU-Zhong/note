在Spring MVC的`HandlerInterceptor`（处理器拦截器）的`preHandle`方法中，这三个参数 `HttpServletRequest request`, `HttpServletResponse response`, 和 `Object handler` 扮演着非常重要的角色。它们提供了访问和控制HTTP请求处理流程所需的所有信息和工具。

下面分别解释这三个参数：

### 1. `HttpServletRequest request`

- **是什么？** `HttpServletRequest` 是Java Servlet API中的一个核心接口，它代表了客户端（通常是浏览器）向服务器发送的HTTP请求。当一个用户通过浏览器访问你的Web应用时，Web服务器（如Tomcat）会为这个请求创建一个`HttpServletRequest`对象。这个对象封装了关于该请求的所有信息。
    
- **怎么用？（常见用途）** 通过`request`对象，你可以获取到请求的方方面面：
    
    - **获取请求参数 (Query Parameters or Form Data):**
        - `request.getParameter(String name)`: 获取指定名称的单个参数值（通常用于GET请求的查询参数或POST请求的表单数据）。
        - `request.getParameterValues(String name)`: 获取指定名称的所有参数值（当一个参数有多个值时，如复选框）。
        - `request.getParameterMap()`: 获取所有请求参数的Map。
    - **获取请求头 (Headers):**
        - `request.getHeader(String name)`: 获取指定请求头的值（如 `User-Agent`, `Content-Type`, `Authorization`）。
        - `request.getHeaders(String name)`: 获取指定请求头的所有值。
        - `request.getHeaderNames()`: 获取所有请求头的名称。
    - **获取请求体 (Request Body):**
        - `request.getInputStream()`: 获取一个输入流，用于读取请求体的内容（常用于处理如JSON、XML等非表单数据）。
        - `request.getReader()`: 获取一个字符读取器，用于读取文本类型的请求体。
    - **获取请求路径和方法:**
        - `request.getRequestURI()`: 获取请求的URI（不包括域名部分，例如 `/myapp/user/profile`）。
        - `request.getRequestURL()`: 获取完整的请求URL（例如 `http://example.com/myapp/user/profile`）。
        - `request.getContextPath()`: 获取应用的上下文路径（例如 `/myapp`）。
        - `request.getServletPath()`: 获取Servlet的路径。
        - `request.getMethod()`: 获取HTTP请求方法（如 `GET`, `POST`, `PUT`, `DELETE`）。
    - **获取会话 (Session):**
        - `request.getSession()`: 获取或创建一个`HttpSession`对象，用于在多次请求之间跟踪用户状态。
    - **获取Cookies:**
        - `request.getCookies()`: 获取一个包含所有Cookie的数组。
    - **获取客户端信息:**
        - `request.getRemoteAddr()`: 获取客户端的IP地址。
        - `request.getRemoteHost()`: 获取客户端的主机名。
    - **请求属性 (Attributes):**
        - `request.setAttribute(String name, Object o)`: 在请求范围内存储一个对象，通常用于在请求处理的不同组件（如拦截器到Controller，或Controller到View）之间传递数据。
        - `request.getAttribute(String name)`: 获取请求范围内的属性。
        - `request.removeAttribute(String name)`: 移除请求范围内的属性。
    
    在你提供的代码中，`request.getHeader(jwtProperties.getAdminTokenName())` 就是用来从请求头中获取JWT令牌的。
    

### 2. `HttpServletResponse response`

- **是什么？** `HttpServletResponse` 也是Java Servlet API的核心接口，它代表了服务器对客户端HTTP请求的响应。通过这个对象，你可以控制服务器将要发送回客户端的内容和状态。
    
- **怎么用？（常见用途）** 通过`response`对象，你可以构建和发送HTTP响应：
    
    - **设置响应状态码:**
        - `response.setStatus(int sc)`: 设置HTTP状态码（如 `200 OK`, `401 Unauthorized`, `404 Not Found`, `500 Internal Server Error`）。
        - `response.sendError(int sc, String msg)`: 发送一个错误状态码和可选的错误信息。
    - **设置响应头 (Headers):**
        - `response.setHeader(String name, String value)`: 设置一个响应头的值（会覆盖同名的旧值）。
        - `response.addHeader(String name, String value)`: 添加一个响应头的值（允许同名头存在多个值）。
        - `response.setContentType(String type)`: 设置响应内容的MIME类型（如 `text/html`, `application/json`, `image/jpeg`）。这实际上是设置 `Content-Type` 响应头的一个便捷方法。
        - `response.setContentLength(int len)`: 设置响应体的长度。
    - **写入响应体 (Response Body):**
        - `response.getWriter()`: 获取一个`PrintWriter`对象，用于向客户端发送字符文本数据（如HTML, JSON字符串）。
        - `response.getOutputStream()`: 获取一个`ServletOutputStream`对象，用于向客户端发送二进制数据（如图片、文件下载）。
    - **重定向 (Redirect):**
        - `response.sendRedirect(String location)`: 指示客户端重定向到另一个URL。
    - **设置Cookies:**
        - `response.addCookie(Cookie cookie)`: 向客户端添加一个Cookie。
    
    在你提供的代码中，`response.setStatus(401)` 就是在JWT校验失败时，设置响应状态码为401。
    

### 3. `Object handler`

- **是什么？** 在Spring MVC的`HandlerInterceptor`上下文中，`handler`参数代表了**即将要处理当前请求的处理器（Handler）**。这个处理器通常是你定义的Controller类中的一个具体方法（被`@RequestMapping`, `@GetMapping`等注解标记的方法）。
    
- **怎么用？（常见用途）**
    
    - **类型检查和信息获取:**
        
        - 你可以检查`handler`的实际类型，最常见的是判断它是否为`HandlerMethod`的实例：
            
            Java
            
            ```
            if (handler instanceof HandlerMethod) {
                HandlerMethod handlerMethod = (HandlerMethod) handler;
            ```
            
        
        // 获取方法名 String methodName = handlerMethod.getMethod().getName(); // 获取方法所在的Bean (Controller实例) Object bean = handlerMethod.getBean(); // 获取方法上的注解 MyAnnotation annotation = handlerMethod.getMethodAnnotation(MyAnnotation.class); // ... 其他操作 } ```
        - `HandlerMethod` 对象提供了关于目标Controller方法的大量元数据信息，例如方法本身、方法所在的bean（Controller实例）、方法上的注解等。这对于实现一些高级功能（如基于注解的权限控制、日志记录）非常有用。
    - **决策依据:**
        - 拦截器可以根据`handler`的信息来决定是否要拦截请求。例如，你可能只想对某些Controller的特定方法或带有特定注解的方法执行拦截逻辑。
    
    在你提供的代码中：
    
    Java
    
    ```
    if (!(handler instanceof HandlerMethod)) {
        // 当前拦截到的不是动态方法，直接放行
        return true;
    }
    ```
    
    这段代码就是检查`handler`是否是一个`HandlerMethod`实例。如果不是（例如，请求的是一个静态资源，它可能由Spring的`ResourceHttpRequestHandler`处理），那么JWT校验逻辑就不需要执行，直接放行。这确保了JWT拦截器只作用于Controller的动态请求。
    

**总结：**

- `request`：封装了客户端发来的所有请求信息，让你能**读取**请求数据。
- `response`：代表了服务器要发回给客户端的响应，让你能**构建和发送**响应数据。
- `handler`：代表了即将处理这个请求的具体Controller方法（或其他处理器），让你能**了解**请求将由谁处理，并据此做出决策。

这三个参数共同构成了拦截器工作的基础，使得拦截器能够在请求处理的生命周期中有效地介入和控制流程。
