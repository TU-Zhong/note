### 什么是 Spring Boot 拦截器 (Interceptor)？

您可以将拦截器想象成一个 **“API 的安检站”**。

在一个 Web 应用中，当一个请求 (Request) 从客户端（例如浏览器）发送到您的控制器 (Controller) 之前，以及在控制器处理完请求、要返回响应 (Response) 给客户端之后，拦截器提供了一个机会，让我们可以在这两个时间点**介入**并执行一些通用的前置或后置处理逻辑。

#### 1. 在请求处理流程中的位置

为了更清楚地理解，我们来看一个简化的请求流程：

`客户端请求` -> `Filter (过滤器)` -> `DispatcherServlet (Spring核心)` -> **`Interceptor.preHandle (前置处理)`** -> `Controller (您的业务逻辑)` -> **`Interceptor.postHandle (后置处理)`** -> `DispatcherServlet 渲染视图` -> **`Interceptor.afterCompletion (完成处理)`** -> `Filter` -> `客户端响应`

从流程中可以看到，拦截器处于 Spring MVC 的核心位置，它比过滤器 (Filter) 更接近您的业务逻辑 (Controller)，因此可以获取到更多与请求处理相关的信息，例如即将执行哪个 Controller 方法。

#### 2. 拦截器的三大核心方法

一个拦截器通常会实现 `HandlerInterceptor` 接口，这个接口提供了三个核心方法：

1. **`preHandle(request, response, handler)` - 控制器方法执行前**
    
    - **时机**: 在 Controller 方法执行**之前**被调用。
    - **用途**: 这是最重要的方法，主要用于**权限验证、登录检查、请求参数日志记录**等。
    - **返回值**:
        - 返回 `true`：请求会继续向下传递，执行下一个拦截器或 Controller。
        - 返回 `false`：请求被**中断**，不会再往下执行，后续的拦截器和 Controller 都不会被调用。您可以直接在这里通过 `response` 对象返回错误信息给客户端。
2. **`postHandle(request, response, handler, modelAndView)` - 控制器方法执行后，视图渲染前**
    
    - **时机**: 在 Controller 方法执行**之后**，但在视图 (View) 被渲染**之前**被调用。
    - **用途**: 有机会在响应送回客户端前，修改 `ModelAndView` 对象，例如在页面中加入一些通用的数据。
    - **注意**: 如果 Controller 方法中发生异常，此方法**不会**被执行。
3. **`afterCompletion(request, response, handler, ex)` - 请求处理完成后**
    
    - **时机**: 在整个请求处理完成后被调用，也就是在视图渲染完毕之后。
    - **用途**: 主要用于**资源清理、性能监控、日志记录**等。无论请求处理过程中是否发生异常 (`ex` 参数会捕捉到异常)，此方法**总是会被执行**。

#### 3. 主要应用场景

- **权限验证**：检查用户是否登录、是否有权限访问该 API。
- **日志记录**：记录每个请求的来源 IP、请求参数、执行时间等。
- **性能监控**：记录每个请求的处理耗时。
- **通用数据处理**：为所有请求的模型添加通用属性（例如网站名称、当前用户信息等）。

### 如何在 Spring Boot 中使用拦截器？

实现一个拦截器只需要两个步骤：**“创建拦截器”** 和 **“注册拦截器”**。

#### 第一步：创建拦截器类

首先，创建一个类并实现 `HandlerInterceptor` 接口。通常我们会将它标注为 `@Component`，方便让 Spring 容器管理。

**示例：一个简单的登录检查拦截器 `LoginInterceptor.java`**

这个拦截器会检查请求的 Header 中是否包含一个名为 `token` 的字段。

Java

```
package com.example.demo.interceptor;

import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;
import lombok.extern.slf4j.Slf4j;
import org.springframework.stereotype.Component;
import org.springframework.web.servlet.HandlerInterceptor;
import org.springframework.web.servlet.ModelAndView;

@Component
@Slf4j // 使用 Lombok 提供的日志注解，方便输出日志
public class LoginInterceptor implements HandlerInterceptor {

    /**
     * 在 Controller 方法执行前进行处理
     * @return true 表示放行，false 表示拦截
     */
    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) throws Exception {
        log.info("拦截器 preHandle：请求 {} 即将进入 Controller...", request.getRequestURI());

        // 模拟检查 token
        String token = request.getHeader("token");
        if (token == null || token.isEmpty()) {
            log.warn("请求被拦截：Header 中缺少 token");
            response.setStatus(401); // 设置 HTTP 状态码为未授权
            response.getWriter().write("Unauthorized: Token is missing.");
            return false; // 拦截请求
        }

        log.info("Token 验证通过，请求放行。");
        return true; // 放行请求
    }

    /**
     * 在 Controller 方法执行后进行处理
     */
    @Override
    public void postHandle(HttpServletRequest request, HttpServletResponse response, Object handler, ModelAndView modelAndView) throws Exception {
        log.info("拦截器 postHandle：Controller 已执行完毕。");
    }

    /**
     * 在整个请求处理完成后进行处理
     */
    @Override
    public void afterCompletion(HttpServletRequest request, HttpServletResponse response, Object handler, Exception ex) throws Exception {
        log.info("拦截器 afterCompletion：请求 {} 已处理完毕。", request.getRequestURI());
        if (ex != null) {
            log.error("请求处理过程中发生异常：", ex);
        }
    }
}
```

#### 第二步：注册拦截器并设置拦截规则

光创建拦截器还不够，您需要告诉 Spring Boot：“我有这样一个拦截器，请将它应用到指定的 URL 路径上”。

这一步通常是通过一个配置类 (Configuration Class) 来完成，这个类需要实现 `WebMvcConfigurer` 接口。

**示例：`WebConfig.java`**

Java

```
package com.example.demo.config;

import com.example.demo.interceptor.LoginInterceptor;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.context.annotation.Configuration;
import org.springframework.web.servlet.config.annotation.InterceptorRegistry;
import org.springframework.web.servlet.config.annotation.WebMvcConfigurer;

@Configuration
public class WebConfig implements WebMvcConfigurer {

    // 将我们创建的拦截器注入进来
    @Autowired
    private LoginInterceptor loginInterceptor;

    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(loginInterceptor)
                // addPathPatterns 用于设置要拦截的请求路径，"/**" 表示拦截所有请求
                .addPathPatterns("/**") 
                // excludePathPatterns 用于设置要排除的请求路径（不拦截）
                .excludePathPatterns("/user/login", "/user/register", "/css/**", "/js/**");
    }
}
```

**代码解释：**

1. `@Configuration`：告诉 Spring Boot 这是一个配置类。
2. `implements WebMvcConfigurer`：实现这个接口才能够自定义 Spring MVC 的配置。
3. `addInterceptors(InterceptorRegistry registry)`：重写这个方法来注册拦截器。
4. `registry.addInterceptor(loginInterceptor)`：将我们写好的 `LoginInterceptor` 实例加进去。
5. `.addPathPatterns("/**")`：设置拦截规则。`/**` 是一个通配符，代表拦截所有路径的请求。
6. `.excludePathPatterns(...)`：设置排除规则。通常我们会排除登录、注册页面以及静态资源（如 CSS、JS 文件），否则连登录页面都进不去。

### 拦截器 (Interceptor) vs. 过滤器 (Filter)

这两者功能相似，但有细微的差别，了解它们有助于您在合适的场景使用合适的工具。

|特性|**拦截器 (Interceptor)**|**过滤器 (Filter)**|
|:--|:--|:--|
|**所属范围**|Spring MVC 框架的一部分|Java Servlet 规范的一部分，不依赖 Spring|
|**执行时机**|在 `DispatcherServlet` 之后，Controller 方法执行前后|在 `DispatcherServlet` 之前，在 Servlet 容器层面工作|
|**可获取信息**|可以获取到要执行的 **Handler (Controller 方法)**|无法直接获取 Spring 的 Handler 信息|
|**控制粒度**|更细，基于 Spring Context，可以方便地使用 DI 注入其他 Bean|更粗，主要处理原始的 `HttpServletRequest` 和 `HttpServletResponse`|
|**主要用途**|权限检查、日志记录、性能监控等**与业务逻辑相关**的通用处理|字符编码、数据压缩、XSS 防护等**与业务逻辑无关**的通用处理|

总结来说，如果您需要做的通用处理与 Spring 的业务逻辑（如哪个 Controller 被调用）相关，**拦截器是更好的选择**。如果您需要处理更底层、更通用的 HTTP 请求本身，过滤器更合适。
