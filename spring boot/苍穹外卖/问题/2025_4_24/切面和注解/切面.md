```java
package com.sky.Aspect;  
  
import lombok.extern.slf4j.Slf4j;  
import org.aspectj.lang.JoinPoint;  
import org.aspectj.lang.annotation.Aspect;  
import org.aspectj.lang.annotation.Before;  
import org.aspectj.lang.annotation.Pointcut;  
import org.springframework.stereotype.Component;  
  
/**  
 * 自定义的切面类  
 */  
@Aspect  
@Component  
@Slf4j  
public class AutoFillAspect {  
    /**  
     * 切入点  
     */  
    @Pointcut("execution(* com.sky.mapper.*.*(..)) && @annotation(com.sky.annotation.AutoFill)")  
    public void autoFillPointcut(){}  
  
    /**  
     * 前置通知，在通知中进行公共字段的赋值  
     */  
    @Before("autoFillPointcut()")  
    public void autoFill(JoinPoint joinPoint){  
        log.info("开始进行公共字段自动填充...");  
    }  
}
```

### **这段代码的作用：**  
想象你有一个机器人小助手叫「自动填表员」，它的任务是：  
**每当有人要填表格（执行数据库操作）时，它就会偷偷在表格里提前填好一些固定内容（比如填上今天的日期、谁填的等等）。**

---

### **具体怎么工作的？**  
1. **小助手在哪待命？**  
   - `@Aspect` → 告诉机器人：“你是专门负责帮人填表的！”  
   - `@Component` → 机器人被注册到工厂（Spring容器），随时可以干活。

2. **小助手的任务范围：**  
   - `@Pointcut` → 机器人盯着**所有来自 `com.sky.mapper` 包的方法**，并且这些方法**贴了 `@AutoFill` 小纸条**（注解）。  
   （就像说：“只要有人在这些地方填表格，我就去帮忙！”）

3. **小助手的工作时间：**  
   - `@Before` → 机器人会在**正式填表前**抢先动手。  
   （比如在保存数据到数据库前，自动给表格加时间、填用户名）

4. **小助手现在做了什么？**  
   - 现在它只是大喊一声：“我开始填表啦！”（日志记录）。  
   （实际填表的代码还没写，但以后可以在这里加）

---

### 🌰 举个栗子：  
假设小明要保存一篇新文章到数据库：  
1. 小明调用 `ArticleMapper.insert()` 方法（这个方法贴了 `@AutoFill` 纸条）。  
2. 机器人小助手发现后，立刻跳出来：  
   - 偷偷在文章数据里加上 `创建时间=现在`、`作者=小明`。  
3. 最后数据库里存的文章自动带上了时间和作者！

---

### 一、整体结构
[[@annotation注解是干什么的]]
`execution(...) && @annotation(...)`  
表示**同时满足两个条件**的方法才会被拦截：
1. 方法在指定包路径下
2. 方法上贴有 `@AutoFill` 注解

---

### 二、`execution` 表达式详解

#### 语法模板：
```java
execution(
   [访问修饰符] 返回值类型 [包名.类名.]方法名(参数列表) [throws 异常类型]
)
```

#### 当前表达式：
`execution(* com.sky.mapper.*.*(..))`

##### 1. `*`（返回值通配符）
- **作用**：匹配任意返回值类型（如 `void`、`String`、`Object` 等）
- **示例**：`String` → ✅，`void` → ✅，`int` → ✅

##### 2. `com.sky.mapper.*`（包路径）
- `com.sky.mapper` → 包名
- 最后的 `*` → 表示匹配**该包下所有类**（但不包括子包）
- **示例**：
  - `com.sky.mapper.UserMapper` → ✅  
  - `com.sky.mapper.sub.OrderMapper` → ❌（属于子包）

##### 3. `.*`（类名和方法名）
- **第一个 `.`** → 类名与包名的分隔符
- **`*`** → 类名通配符（匹配所有类）
- **第二个 `*`** → 方法名通配符（匹配所有方法）
- **组合效果**：`com.sky.mapper` 包下的所有类的所有方法

##### 4. `(..)`（参数列表）
- `..` → 匹配任意数量、任意类型的参数
- **示例**：  
  - `save()` → ✅  
  - `findById(int)` → ✅  
  - `update(String, Object)` → ✅

---

### 三、`@annotation` 表达式
`@annotation(com.sky.annotation.AutoFill)`  
- **作用**：匹配所有**被 `@AutoFill` 注解标记的方法**
- **示例**：
  ```java
  @AutoFill(OperationType.INSERT)
  public void save(User user) { ... }  // ✅ 会被匹配
  ```

---

### 四、组合逻辑 (`&&`)
- `&&` → 逻辑与，要求**同时满足**两个条件
- **等价于**：  
  `方法在 com.sky.mapper 包下` **并且** `方法有 @AutoFill 注解`

---

### 五、Java 知识点总结
| 知识点              | 说明                                                                 |
|---------------------|----------------------------------------------------------------------|
| **AOP 切点表达式**   | 用于定义在哪些方法上应用横切逻辑（如日志、事务）                     |
| **通配符**           | `*` 匹配任意单个元素，`..` 匹配多个元素（如参数、子包）              |
| **注解驱动拦截**     | 通过 `@annotation` 精准控制需要增强的方法                            |
| **逻辑运算符**       | `&&`（与）、`||`（或）、`!`（非）组合复杂条件                        |
| **Spring AOP 底层**  | 基于动态代理实现，对匹配的方法插入通知逻辑                           |

### 六、最终效果
这段代码实现了一个精准的拦截规则：  
**只处理 `com.sky.mapper` 包下所有类中，被 `@AutoFill` 注解标记的方法**，并在这些方法执行前自动填充公共字段（如创建时间、修改人等）。
