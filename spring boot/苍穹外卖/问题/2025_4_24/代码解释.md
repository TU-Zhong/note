```java
public static Claims parseJWT(String secretKey, String token) {  
    // 得到DefaultJwtParser  
    Claims claims = Jwts.parser()
```

这段代码是使用 **JJWT 库**（Java JWT）来 **解析并验证 JWT 令牌** 的核心方法，目的是从 JWT 中提取数据声明（Claims）。以下是逐行解释：

---

### **代码功能**
```java
public static Claims parseJWT(String secretKey, String token) {
    Claims claims = Jwts.parser()                      // 1. 创建 JWT 解析器
            .setSigningKey(secretKey.getBytes(StandardCharsets.UTF_8)) // 2. 设置签名密钥
            .parseClaimsJws(token)                     // 3. 解析并验证令牌
            .getBody();                                 // 4. 提取 Claims（数据载荷）
    return claims;
}
```

---

### **逐行解析**
#### **1. `Jwts.parser()`**
- **作用**：创建一个 JWT 解析器（`JwtParser`）实例。
- **底层机制**：JJWT 库的入口点，用于配置解析规则（如签名密钥、容忍时间等）。

---

#### **2. `.setSigningKey(secretKey.getBytes(StandardCharsets.UTF_8))`**
- **作用**：设置用于验证 JWT 签名的密钥。
- **关键点**：
  - `secretKey` 是对称加密的密钥（如 HMAC-SHA256 算法），需与生成 JWT 时使用的密钥一致。
  - **编码转换**：将字符串密钥转换为字节数组，并明确指定 UTF-8 编码，避免不同系统默认编码不一致导致密钥错误。
- **安全提示**：如果使用非对称加密（如 RSA），这里应设置为公钥（Public Key）。

---

#### **3. `.parseClaimsJws(token)`**
- **作用**：解析并验证 JWT 的完整性和签名。
- **流程**：
  1. **拆分 JWT**：将 `token` 分割为 Header、Payload、Signature 三部分。
  2. **验证签名**：使用 `setSigningKey` 提供的密钥重新计算签名，并与 JWT 中的签名比对，确保未被篡改。
  3. **验证有效期**：检查 JWT 中的 `exp`（过期时间）和 `nbf`（生效时间），若当前时间不在有效范围内，抛出异常（如 `ExpiredJwtException`）。

---

#### **4. `.getBody()`**
- **作用**：从验证通过的 JWT 中提取 **Payload（载荷）** 部分，转换为 `Claims` 对象。
- **`Claims` 内容**：包含 JWT 中存储的所有数据，例如：
  ```java
  String username = claims.getSubject();         // 获取 sub（主题）
  String role = claims.get("role", String.class); // 获取自定义字段
  Date expiration = claims.getExpiration();      // 获取过期时间
  ```

---

### **异常处理**
此代码未显式处理异常，但实际使用中可能抛出以下异常：
- **`SignatureException`**：签名不匹配（密钥错误或数据篡改）。
- **`MalformedJwtException`**：JWT 格式错误（如缺少部分）。
- **`ExpiredJwtException`**：JWT 已过期。
- **`UnsupportedJwtException`**：不支持的 JWT 类型。

**建议补充 try-catch**：
```java
try {
    Claims claims = parseJWT(secretKey, token);
    // 处理 claims
} catch (ExpiredJwtException e) {
    System.out.println("令牌已过期");
} catch (SignatureException e) {
    System.out.println("签名无效");
} catch (Exception e) {
    System.out.println("解析失败: " + e.getMessage());
}
```

### **关键总结**
- **核心步骤**：配置解析器 → 设置密钥 → 解析验证 → 提取数据。
- **安全性**：依赖密钥验证签名，确保数据未被篡改。
- **适用场景**：用户身份验证、微服务间安全数据传输等。

```java

    .parseClaimsJws(token);
```
