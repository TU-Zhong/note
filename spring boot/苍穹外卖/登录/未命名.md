您的问题涉及到HTTP请求处理和JWT认证的流程。让我详细解释这个过程：

### 登录流程与JWT令牌传递

1. **登录请求发送**:
    - 客户端发送POST请求到`/admin/employee/login`端点
    - 请求体中包含用户名和密码的JSON对象(
        
         `EmployeeLoginDTO`
        
        )
2. **服务器处理登录**:
```java
EmployeeController.java

Copy

@PostMapping("/login")

@ApiOperation(value = "员工登录")

public Result<EmployeeLoginVO> login(@RequestBody EmployeeLoginDTO 

employeeLoginDTO) {

    log.info("员工登录：{}", employeeLoginDTO);

    Employee employee = employeeService.login(employeeLoginDTO);

    //登录成功后，生成jwt令牌

    Map<String, Object> claims = new HashMap<>();

    claims.put(JwtClaimsConstant.EMP_ID, employee.getId());

    String token = JwtUtil.createJWT(

            jwtProperties.getAdminSecretKey(),

            jwtProperties.getAdminTtl(),

            claims);

    EmployeeLoginVO employeeLoginVO = EmployeeLoginVO.builder()

            .id(employee.getId())

            .userName(employee.getUsername())

            .name(employee.getName())

            .token(token)

            .build();

    return Result.success(employeeLoginVO);

}
```


3. **客户端接收令牌**:
    - 服务器返回包含 `token`的`EmployeeLoginVO`对象
    - 客户端保存这个`token`(通常存在localStorage或Cookie中)
4. **后续请求携带令牌**:
    - 客户端在后续请求中，将`token`放在HTTP请求头中
    - 请求头的名称是`jwtProperties.getAdminTokenName()`配置的值(默认为"token")
5. **拦截器获取令牌**:

### 对象转换与请求处理

您问的"对象是怎么转换成request中的信息"涉及到几个关键点：

1. **@RequestBody注解的作用**:
    - 当客户端发送POST请求时，请求体中的JSON会被自动转换为
        
         `EmployeeLoginDTO`
        
        对象
    - 这个转换由Spring MVC的`HttpMessageConverter`完成
    - 您的项目中配置了
        
         `JacksonObjectMapper`
        
        来处理这个转换

WebMvcConfiguration.java

sky-server/src/main/java/com/sky/config

Copy
```java
protected void extendMessageConverters(List<HttpMessageConverter<?>> converters) {

    MappingJackson2HttpMessageConverter converter=new MappingJackson2HttpMessageConverter();

    converter.setObjectMapper(new JacksonObjectMapper());

    converters.add(0,converter);

}
```


2. **登录成功后的令牌生成**:
    - 服务器生成JWT令牌，并将其包含在响应中
    - 客户端接收到响应后，保存令牌
3. **令牌在后续请求中的传递**:
    - 客户端代码(前端JavaScript/移动应用)负责在每次请求时添加令牌到请求头
    - 典型的前端代码可能如下:

```javascript
// 登录成功后保存token

function login(username, password) {

  fetch('/admin/employee/login', {

    method: 'POST',

    headers: { 'Content-Type': 'application/json' },

    body: JSON.stringify({ username, password })

  })

  .then(response => response.json())

  .then(data => {

    // 保存token

    localStorage.setItem('token', data.data.token);

  });

}



// 后续请求添加token到请求头

function fetchWithAuth(url) {

  const token = localStorage.getItem('token');

  return fetch(url, {

    headers: {

      'token': token  // 这里的'token'就是jwtProperties.

      getAdminTokenName()配置的值

    }

  });

}
```

4. **请求头到拦截器的传递**:
    - 当请求到达服务器时，Spring MVC将HTTP请求封装为`HttpServletRequest`对象
    - 拦截器通过`request.getHeader()`方法获取请求头中的令牌

### 总结

1. 登录时，客户端发送包含用户名密码的JSON对象
2. 服务器通过`@RequestBody`将JSON转为
    
     `EmployeeLoginDTO`
    
    对象
3. 验证成功后，服务器生成JWT令牌并返回
4. 客户端保存令牌
5. 后续请求中，客户端在请求头中添加令牌
6. 服务器拦截器通过`request.getHeader()`获取令牌并验证

这个过程中没有直接的"对象转换成request"，而是:

- 客户端负责将令牌添加到请求头
- 服务器从请求头中提取令牌

这是一种无状态认证机制，服务器不需要存储会话信息，提高了系统的可扩展性。
