好的，这个练习题“查询订单、订单项及产品信息 (复杂嵌套)”是一个考察 MyBatis 中处理复杂对象图（one-to-many-to-one）的典型场景。你需要使用多表 `JOIN` 和一个嵌套的 `<resultMap>` 来完成。

以下是解题步骤和代码示例：

**目标：** 根据 `orderId` 查询一个 `Order` 对象，该对象需要包含：

1. `Order` 自身的基本信息。
2. 一个 `List<OrderItem>` 订单项列表 (`order.getOrderItems()`)。
3. 列表中的每一个 `OrderItem` 对象都需要包含其关联的 `Product` 对象 (`orderItem.getProduct()`)。

**1. Java 实体类 (回顾)**

确保你的Java实体类之间的关系已经正确定义：

- `Order.java` 包含 `private List<OrderItem> orderItems;`
- `OrderItem.java` 包含 `private Product product;` (以及 `private Integer productId;` 用来存储外键)

**2. SQL 查询语句**

你需要编写一个SQL语句，通过`JOIN`一次性从`Orders`, `OrderItems`, 和 `Products` 表中获取所有需要的数据。使用`LEFT JOIN`可以确保即使订单没有订单项，或订单项关联的产品信息缺失（理论上不应发生，除非数据不一致），主订单信息依然能被查出。

SQL

```
SELECT
    -- Order 表的列 (使用别名 'o')
    o.orderId AS o_orderId,
    o.customerId AS o_customerId,
    o.orderDate AS o_orderDate,
    o.totalAmount AS o_totalAmount,
    o.status AS o_status,

    -- OrderItem 表的列 (使用别名 'oi')
    oi.orderItemId AS oi_orderItemId,
    oi.orderId AS oi_fk_orderId,     -- OrderItem 表中的 orderId 外键
    oi.productId AS oi_fk_productId, -- OrderItem 表中的 productId 外键
    oi.quantity AS oi_quantity,
    oi.priceAtPurchase AS oi_priceAtPurchase,

    -- Product 表的列 (使用别名 'p')
    p.productId AS p_productId,
    p.productName AS p_productName,
    p.unitPrice AS p_unitPrice,
    p.category AS p_category
FROM
    Orders o
LEFT JOIN
    OrderItems oi ON o.orderId = oi.orderId  -- Orders 和 OrderItems 通过 orderId 关联
LEFT JOIN
    Products p ON oi.productId = p.productId -- OrderItems 和 Products 通过 productId 关联
WHERE
    o.orderId = #{orderIdParam}; -- 传入的参数，建议使用@Param注解指定名称
```

- **列别名**: 我为每个表的列都加上了前缀（如 `o_`, `oi_`, `p_`）并使用了别名。这在复杂查询中是一个好习惯，可以避免不同表中相同列名的冲突，并在 `<resultMap>` 中清晰地引用它们。
- `#{orderIdParam}`: 这是传入的订单ID参数。

**3. Mapper 接口方法 (`OrderMapper.java`)**

Java

```
package com.example.mapper; // 你的包名

import com.example.model.Order;
import org.apache.ibatis.annotations.Param;

public interface OrderMapper {
    /**
     * 根据订单ID查询订单详细信息，包括订单项和每个订单项关联的产品信息。
     * @param orderId 订单ID
     * @return 包含详细信息的 Order 对象，如果未找到则返回 null。
     */
    Order findOrderWithDetails(@Param("orderIdParam") Integer orderId);
}
```

- `@Param("orderIdParam")`: 强烈建议为参数使用 `@Param` 注解，这样在XML中就可以通过 `#{orderIdParam}` 来引用，更清晰。

**4. Mapper XML 配置 (`OrderMapper.xml`)**

这是最核心的部分，你需要定义一个复杂的 `<resultMap>` 来处理这种嵌套结构。

XML

```
<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN" "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="com.example.mapper.OrderMapper">

    <resultMap id="OrderWithDetailsResultMap" type="com.example.model.Order">
        <id property="orderId" column="o_orderId"/>
        <result property="customerId" column="o_customerId"/>
        <result property="orderDate" column="o_orderDate"/>
        <result property="totalAmount" column="o_totalAmount"/>
        <result property="status" column="o_status"/>

        <collection property="orderItems" ofType="com.example.model.OrderItem">
            <id property="orderItemId" column="oi_orderItemId"/>
            <result property="orderId" column="oi_fk_orderId"/> <result property="productId" column="oi_fk_productId"/> <result property="quantity" column="oi_quantity"/>
            <result property="priceAtPurchase" column="oi_priceAtPurchase"/>

            <association property="product" javaType="com.example.model.Product">
                <id property="productId" column="p_productId"/>
                <result property="productName" column="p_productName"/>
                <result property="unitPrice" column="p_unitPrice"/>
                <result property="category" column="p_category"/>
            </association>
        </collection>
    </resultMap>

    <select id="findOrderWithDetails" parameterType="int" resultMap="OrderWithDetailsResultMap">
        SELECT
            o.orderId AS o_orderId,
            o.customerId AS o_customerId,
            o.orderDate AS o_orderDate,
            o.totalAmount AS o_totalAmount,
            o.status AS o_status,

            oi.orderItemId AS oi_orderItemId,
            oi.orderId AS oi_fk_orderId,
            oi.productId AS oi_fk_productId,
            oi.quantity AS oi_quantity,
            oi.priceAtPurchase AS oi_priceAtPurchase,

            p.productId AS p_productId,
            p.productName AS p_productName,
            p.unitPrice AS p_unitPrice,
            p.category AS p_category
        FROM
            Orders o
        LEFT JOIN
            OrderItems oi ON o.orderId = oi.orderId
        LEFT JOIN
            Products p ON oi.productId = p.productId
        WHERE
            o.orderId = #{orderIdParam}
    </select>

</mapper>
```

**解释 `<resultMap>`：**

1. **`resultMap id="OrderWithDetailsResultMap" type="com.example.model.Order"`**:
    
    - 定义了一个名为 `OrderWithDetailsResultMap` 的结果映射，其目标Java类型是 `com.example.model.Order`。
2. **Order 自身的属性映射**:
    
    - `<id property="orderId" column="o_orderId"/>`: 将 `o_orderId` 列的值映射到 `Order` 对象的 `orderId` 属性（并标记为主键）。
    - `<result property="..." column="..."/>`: 将其他 `Orders` 表的列映射到 `Order` 对象的相应属性。
3. **`<collection property="orderItems" ofType="com.example.model.OrderItem">`**:
    
    - `property="orderItems"`: 映射到 `Order` 对象中的 `orderItems` 这个 `List` 属性。
    - `ofType="com.example.model.OrderItem"`: 指定这个 `List` 中元素的类型是 `com.example.model.OrderItem`。
    - MyBatis在处理结果集时，如果发现 `o_orderId` 相同但其他 `oi_` 或 `p_` 列不同的行，它会自动将这些行的数据构造成 `OrderItem` 对象，并添加到当前 `Order` 对象的 `orderItems` 列表中。
4. **OrderItem 内部的属性映射 (在 `<collection>` 标签内)**:
    
    - `<id property="orderItemId" column="oi_orderItemId"/>`: 将 `oi_orderItemId` 列映射到 `OrderItem` 对象的 `orderItemId` 属性。
    - `<result property="productId" column="oi_fk_productId"/>`: 这一点很重要，它将 `OrderItems` 表中的 `productId` 外键列的值赋给了 `OrderItem` Java对象中的 `productId` 属性。
    - 其他 `OrderItem` 属性也类似映射。
5. **`<association property="product" javaType="com.example.model.Product">` (在 `<collection>` 标签内，表示 `OrderItem` 的一部分)**:
    
    - `property="product"`: 映射到 `OrderItem` 对象中的 `product` 这个 `Product` 类型的属性。
    - `javaType="com.example.model.Product"`: 指定关联对象的类型。
    - 内部的 `<id>` 和 `<result>` 标签将 `Products` 表的列（以 `p_` 开头的别名）映射到 `Product` 对象的属性。

**工作原理：**

当执行 `findOrderWithDetails` 查询时：

1. SQL 语句通过 JOIN 获取一个扁平化的结果集，其中包含了 `Order`、`OrderItem` 和 `Product` 的所有相关信息。如果一个订单有多个订单项，那么该订单的信息会在结果集中重复多次，每次对应一个订单项。
2. MyBatis 使用 `OrderWithDetailsResultMap` 来处理这个结果集。
3. 它首先根据 `o_orderId` (Order的主键) 来识别和创建/填充 `Order` 对象。
4. 对于每个唯一的 `Order` 对象，它会查看关联的 `OrderItem` 数据 (通过 `oi_` 前缀的列)。
5. 如果 `oi_orderItemId` 不为 NULL（即 `LEFT JOIN` 找到了匹配的订单项），它会创建一个 `OrderItem` 对象，并填充其属性。
6. 在填充每个 `OrderItem` 对象时，它会进一步查看关联的 `Product` 数据 (通过 `p_` 前缀的列)。
7. 如果 `p_productId` 不为 NULL，它会创建一个 `Product` 对象，填充其属性，并将其设置到当前 `OrderItem` 对象的 `product` 属性上。
8. 所有属于同一个 `o_orderId` 的 `OrderItem` 对象会被收集到一个列表中，并设置到对应 `Order` 对象的 `orderItems` 属性上。

**测试：** 编写一个测试用例，调用 `orderMapper.findOrderWithDetails(someOrderId)`，然后断言返回的 `Order` 对象不为null，其 `orderItems` 列表不为null且包含预期的订单项数量，并且每个订单项的 `product` 对象也被正确加载和填充。

这种使用 JOIN 和复杂 `<resultMap>` 的方式是 MyBatis 处理对象图关联（尤其是 eagerly fetching 渴望获取）的标准和推荐做法，因为它通常比多次单独查询（可能导致N+1问题）更高效。
