好的，我们来解决这个练习题：“动态更新订单状态”。

**目标：** 编写一个方法 `updateOrderStatusSelective(Order order)`，它只更新 `Orders` 表中 `order` 对象里非 `null` 的属性（主要关注 `status` 和 `totalAmount`），并且更新操作必须基于 `order` 对象中提供的 `orderId`。

---

### 1. Mapper 接口方法 (`OrderMapper.java`)

Java

```java
package com.example.mapper; // 你的包名

import com.example.model.Order;

public interface OrderMapper {
    /**
     * 选择性地更新订单信息。
     * 只更新传入的 order 对象中非 null 的属性 (例如 status, totalAmount)。
     * **必须**提供 orderId 作为更新条件。
     *
     * @param order 包含要更新的订单ID (orderId 不能为空) 以及需要更新的字段（如果字段为null则不更新）。
     * @return 影响的数据库行数。如果 orderId 不存在或没有字段需要更新，可能返回 0。
     */
    int updateOrderStatusSelective(Order order);
}
```

- **重要提示**：接口方法的 Javadoc 或相关文档应明确指出 `order.orderId` 是必需的。

---

### 2. Mapper XML 配置 (`OrderMapper.xml`)

这里我们将使用 `<set>` 和 `<if>` 标签来实现动态 `UPDATE`。

XML

```xml
<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN" "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="com.example.mapper.OrderMapper">

    <update id="updateOrderStatusSelective" parameterType="com.example.model.Order">
        UPDATE Orders
        <set>
            <if test="status != null">
                status = #{status},
            </if>
            <if test="totalAmount != null">
                totalAmount = #{totalAmount},
            </if>
            </set>
        WHERE orderId = #{orderId}
    </update>

</mapper>
```

**解释:**

- **`parameterType="com.example.model.Order"`**: 指定传入参数是 `Order` 对象。
- **`<update id="updateOrderStatusSelective"...>`**: 定义一个更新操作。
- **`<set>`**: 这个标签非常关键。
    - 它会自动在生成的 SQL 前面加上 `SET` 关键字。
    - 它会自动移除其内部 SQL 片段**末尾**可能多余的逗号 `,`。例如，如果只有 `status` 非 null，生成的 SET 子句会是 `SET status = ?` 而不是 `SET status = ?,`。
- **`<if test="status != null">`**: 检查传入 `order` 对象的 `status` 属性是否为 `null`。
    - 如果不为 `null`，则 `status = #{status},` 这部分 SQL 会被包含在 `<set>` 标签内。
- **`<if test="totalAmount != null">`**: 同理，检查 `totalAmount` 属性。
- **`WHERE orderId = #{orderId}`**:
    - 这是更新操作的条件子句。
    - `#{orderId}` 会从传入的 `order` 对象中获取 `orderId` 的值。
    - **这里隐含了一个假设：调用者必须确保传入的 `order` 对象的 `orderId` 属性是有值的。** MyBatis 的 XML 映射文件本身通常不适合做复杂的业务逻辑校验（比如检查 `orderId` 是否为 null 并据此抛出异常）。

---

### 3. `orderId` 必须提供，如果未提供怎么处理？

这是一个非常重要的问题，涉及到**职责划分**。

- **处理责任方**: **调用者代码**（通常是 Service 层或 Controller 层）负责在调用 Mapper 方法之前，验证输入参数的有效性。
- **处理地点**: **Java 代码中，而不是 MyBatis XML 中**。

**为什么不在 MyBatis XML 中处理?**

1. **职责不清**: XML 主要负责 SQL 映射和生成，而不是业务规则校验。
2. **能力有限**: MyBatis XML 中的表达式能力（OGNL）虽然强大，但用于复杂的流程控制、异常抛出等并不方便，也不直观。
3. **反馈不明确**: 如果在 XML 中简单地不执行更新（比如加一个 `<if test="orderId != null">` 包裹整个 `<update>`），调用者可能无法区分是“ID为空导致未更新”还是“ID不存在导致未更新”。

**正确的处理方式 (在 Java Service 层):**

在调用 `orderMapper.updateOrderStatusSelective(order)` 之前，进行检查。

Java

```java
import com.example.mapper.OrderMapper;
import com.example.model.Order;
import com.example.util.MyBatisSqlSessionFactoryUtil;
import org.apache.ibatis.session.SqlSession;
import java.util.Objects; // 引入 Objects 工具类

public class OrderService { // 假设的服务类

    public boolean updateOrderDetails(Order order) {
        // --- 参数校验 ---
        if (order == null) {
            // 或者记录日志, 返回错误信息等
            throw new IllegalArgumentException("Order object cannot be null for update.");
        }
        // **核心：检查 orderId 是否提供**
        if (order.getOrderId() == null) {
            // 抛出异常是常见的做法，明确告知调用者违反了接口约定
            throw new IllegalArgumentException("Order ID must be provided for updating an order.");
            // 或者根据业务场景返回 false 或特定的错误码
            // System.err.println("Update failed: Order ID is missing.");
            // return false;
        }

        // (可选) 校验是否有实际需要更新的字段，避免无效的数据库调用
        if (order.getStatus() == null && order.getTotalAmount() == null /* && 其他可更新字段也为null */) {
            System.out.println("No fields to update for Order ID: " + order.getOrderId());
            return true; // 认为操作成功，因为无需更新
        }

        // --- 执行数据库操作 ---
        SqlSession session = null;
        boolean success = false;
        try {
            session = MyBatisSqlSessionFactoryUtil.openSession();
            OrderMapper orderMapper = session.getMapper(OrderMapper.class);

            int affectedRows = orderMapper.updateOrderStatusSelective(order);

            if (affectedRows > 0) {
                session.commit(); // 提交事务
                success = true;
                System.out.println("Order " + order.getOrderId() + " updated successfully.");
            } else {
                // 更新了0行，可能是因为orderId在数据库中不存在
                System.out.println("Order " + order.getOrderId() + " not found or no changes needed.");
                // 此时也可能认为是“成功”的，取决于业务定义，但数据未变
                // 如果需要区分，可以在更新前先查询一次
                success = true; // 或者 false，根据业务需求定义
                // 不需要commit因为没有修改，也不需要rollback
            }
        } catch (Exception e) {
            if (session != null) {
                session.rollback(); // 发生异常，回滚事务
            }
            System.err.println("Error updating order: " + e.getMessage());
            // 重新抛出异常或记录日志
            throw new RuntimeException("Database error during order update", e);
        } finally {
            if (session != null) {
                session.close(); // 确保关闭 Session
            }
        }
        return success;
    }
}
```

**总结：**

1. **MyBatis XML (`<set>`, `<if>`)**: 负责根据传入对象**已有的**数据动态生成 `UPDATE` 语句的 `SET` 部分。它假设 `WHERE` 子句所需的 `#{orderId}` 是有效的。
2. **Java Service 层**: 负责**业务逻辑和参数校验**。在调用 Mapper 方法前，必须检查 `order.getOrderId()` 是否为 `null`，并在其为 `null` 时采取适当措施（如抛出 `IllegalArgumentException`）。这是保证接口健壮性和明确职责的最佳实践。
