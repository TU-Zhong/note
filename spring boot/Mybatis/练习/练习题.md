### 1. 数据库表结构 (SQL - 以MySQL为例)

SQL

```sql
-- Customer 表
CREATE TABLE Customers (
    customerId INT AUTO_INCREMENT PRIMARY KEY,
    customerName VARCHAR(255) NOT NULL,
    contactEmail VARCHAR(255) UNIQUE,
    city VARCHAR(100)
);

-- Product 表
CREATE TABLE Products (
    productId INT AUTO_INCREMENT PRIMARY KEY,
    productName VARCHAR(255) NOT NULL,
    unitPrice DECIMAL(10, 2) NOT NULL,
    category VARCHAR(100)
);

-- Order 表
CREATE TABLE Orders (
    orderId INT AUTO_INCREMENT PRIMARY KEY,
    customerId INT,          -- 外键关联到 Customers 表
    orderDate DATE NOT NULL,
    totalAmount DECIMAL(12, 2), -- 订单总金额 (可以由OrderItem计算得出)
    status VARCHAR(50),      -- 例如: PENDING, PROCESSING, SHIPPED, DELIVERED, CANCELLED
    FOREIGN KEY (customerId) REFERENCES Customers(customerId)
);

-- OrderItem 表 (订单项，连接 Order 和 Product)
CREATE TABLE OrderItems (
    orderItemId INT AUTO_INCREMENT PRIMARY KEY,
    orderId INT,             -- 外键关联到 Orders 表
    productId INT,           -- 外键关联到 Products 表
    quantity INT NOT NULL,
    priceAtPurchase DECIMAL(10, 2) NOT NULL, -- 下单时的单价
    FOREIGN KEY (orderId) REFERENCES Orders(orderId),
    FOREIGN KEY (productId) REFERENCES Products(productId)
);
```

### 2. Java 实体类

- **`Customer.java`**
    
    Java
    
    ```java
    import java.util.List;
    
    public class Customer {
        private Integer customerId;
        private String customerName;
        private String contactEmail;
        private String city;
        private List<Order> orders; // 一个客户可以有多个订单
    ```

### 4. MyBatis 练习题

你需要为以下每个需求创建对应的Mapper接口方法和Mapper XML配置。

**基础操作与 ResultMap**

1. **查询客户信息：**
    
    - a. 编写一个方法 `findCustomerById(Integer customerId)`，根据客户ID查询客户基本信息 (不含订单列表)。
        - 知识点: 单参数传递, `resultType`。
    - b. 编写一个方法 `findAllCustomers()`，查询所有客户的基本信息。
        - 知识点: 无参数, `resultType`。
2. **新增客户并返回主键：**
    
    - 编写一个方法 `insertCustomer(Customer customer)`，插入新客户，并确保插入后 `customer` 对象的 `customerId` 属性被正确填充为数据库生成的主键。
        - 知识点: `parameterType`, `useGeneratedKeys`, `keyProperty`。
3. **更新客户邮箱：**
    
    - 编写一个方法 `updateCustomerEmail(@Param("customerId") Integer customerId, @Param("newEmail") String newEmail)`，根据客户ID更新其邮箱地址。
        - 知识点: 多参数传递 (`@Param`), `UPDATE` 语句。
4. **删除客户：**
    
    - 编写一个方法 `deleteCustomer(Integer customerId)`，根据客户ID删除客户。思考：如果客户有关联订单，直接删除是否合适？（此为数据库设计问题，练习MyBatis时可假设能直接删除）。
        - 知识点: `DELETE` 语句。

**关联查询与嵌套 ResultMap**

5. **查询客户及其所有订单 (一对多):**
    
    - 编写一个方法 `findCustomerWithOrders(Integer customerId)`，根据客户ID查询客户信息，同时加载该客户的所有订单信息 (`List<Order> orders`)。使用JOIN查询和嵌套结果映射（`<collection>`）。
        - 知识点: `<resultMap>`, `<collection property="..." ofType="...">`, JOIN SQL。
6. **查询订单及其客户信息 (多对一):**
    
    - 编写一个方法 `findOrderWithCustomer(Integer orderId)`，根据订单ID查询订单信息，同时加载该订单所属的客户信息 (`Customer customer`)。使用JOIN查询和嵌套结果映射（`<association>`）。
        - 知识点: `<resultMap>`, `<association property="..." javaType="...">`, JOIN SQL。
7. **查询订单、订单项及产品信息 (复杂嵌套):** <mark style="background: #FFF3A3A6;">这题很操蛋</mark>
    [[题解]]
    - 编写一个方法 `findOrderWithDetails(Integer orderId)`，根据订单ID查询订单，要求包含所有订单项 (`List<OrderItem> orderItems`)，并且每个订单项中包含其对应的产品信息 (`Product product`)。
        - 知识点: 嵌套 `<collection>` 和 `<association>`, 多表JOIN。

**动态 SQL**

8. **动态条件查询客户：**
    [[动态sql]]
    - 编写一个方法 `findCustomersByCriteria(Customer criteria)`，允许根据 `Customer` 对象中的非空属性（如 `customerName`, `city`, `contactEmail` 可能包含部分匹配）进行动态查询。如果某个属性为null或空字符串，则不作为查询条件。
        - 知识点: `<if>`, `<where>` (或 `<trim>`), 字符串拼接 (如 `LIKE CONCAT('%', #{propertyName}, '%')`)。
9. **动态更新订单状态：**
[[动态更新订单状态]]
    <mark style="background: #FFF3A3A6;">这个校验orderId非空不能在xml中进行，业务校验并不属于mybatis的范畴，必须要清晰业务分配</mark>
    - 编写一个方法 `updateOrderStatusSelective(Order order)`，只更新 `Order` 对象中非null的属性，特别是 `status` 和 `totalAmount`。`orderId` 必须提供。
        - 知识点: `<set>`, `<if>`.
11. **查询特定状态的订单，并按日期排序：**
    [[查询特定状态的订单，并按日期排序 题解]]
    - 编写一个方法 `findOrdersByStatusAndDateRange(@Param("statusList") List<String> statusList, @Param("startDate") Date startDate, @Param("endDate") Date endDate)`。
        - 如果 `statusList` 不为空，则订单状态必须在列表中 (`IN` 子句)。
        - 如果 `startDate` 不为空，则订单日期必须大于等于 `startDate`。
        - 如果 `endDate` 不为空，则订单日期必须小于等于 `endDate`。
        - 结果按订单日期降序排列。
        - 知识点: `<foreach collection="statusList" ...>`, `<if test="startDate != null">`, `<if test="endDate != null">`, `ORDER BY`。

**高级用法**

11. **批量插入订单项：**
    <mark style="background: #FFF3A3A6;">foreach标签内，param("A"), 有的就把collection=A,不然就写参数的类型，如list等</mark>
    - 编写一个方法 `batchInsertOrderItems(List<OrderItem> items)`，使用 `<foreach>` 构造批量插入语句，一次性插入多个订单项。
        - 知识点: `<foreach>` 用于批量 `INSERT`。
12. **使用SQL片段优化查询：**
    
    - 定义一个SQL片段 `<sql id="customerColumns">customerId, customerName, contactEmail, city</sql>`。
    - 在练习1a和1b的查询中通过 `<include refid="customerColumns"/>` 来使用这个片段。
        - 知识点: `<sql>`, `<include>`.
13. **分页查询某分类下的产品：**
    [[分页查询]]
    - 编写一个方法 `findProductsByCategoryPaged(@Param("category") String category, @Param("offset") int offset, @Param("limit") int limit)`，查询指定分类下的产品，并进行分页。
        - 知识点: `LIMIT`/`OFFSET` (或对应数据库的分页语法), 多参数。
14. **统计每个城市PENDING状态的订单数量：**
    
    - 编写一个方法，返回一个 `List<Map<String, Object>>` 或者自定义DTO，其中每个元素包含 `city` 和 `pendingOrderCount`。只统计状态为 'PENDING' 的订单。
        - 知识点: `GROUP BY`, `COUNT(*)`, JOIN `Customers` 和 `Orders` 表, `resultType="java.util.Map"` 或自定义DTO的 `<resultMap>`。
15. **(思考与挑战) 延迟加载客户的订单：**
    
    - 修改练习5，当查询客户信息时，其订单列表默认不加载。只有当程序显式调用 `customer.getOrders()` 方法时，才执行SQL去查询该客户的订单。
        - 知识点: 嵌套Select查询 (`<collection select="...">`), 延迟加载配置 (`lazyLoadingEnabled`, `aggressiveLazyLoading` in mybatis-config.xml, 或 `<collection fetchType="lazy">`)。注意N+1问题。
