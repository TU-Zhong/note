好的，我们来详细介绍一下 MyBatis 中的结果映射 (Result Mapping)。

**结果映射 (Result Mapping) 的概念**

结果映射是 MyBatis 核心功能之一。<mark style="background: #FFF3A3A6;">它的主要作用是将从数据库查询出来的结果集 (JDBC `ResultSet`) 中的列数据，准确地映射到 Java 对象的属性上</mark>。
<mark style="background: #FFF3A3A6;">怎么映射由自己来定义</mark>

为什么需要结果映射？

1. **列名与属性名不匹配**：数据库表的列名（通常是下划线风格，如 `user_name`）和 Java 对象的属性名（通常是驼峰风格，如 `userName`）可能不一致。
2. **复杂数据结构**：查询结果可能需要映射到包含其他对象（一对一关联）或对象集合（一对多关联）的复杂 Java 对象。
3. **类型转换**：有时需要自定义数据库类型与 Java 类型之间的转换规则。
4. **构造方法注入**：希望通过对象的构造方法来创建并初始化对象。
5. **鉴别器 (Discriminator)**：根据某列的值决定将结果映射到哪个具体的 Java 子类或使用哪个特定的映射规则。

MyBatis 提供了两种主要的方式来进行结果映射：

- **`resultType` (自动映射)**：简单直接，适用于列名和属性名能自动匹配（或通过 `mapUnderscoreToCamelCase` 配置解决）的简单场景。
- **`resultMap` (显式映射)**：功能强大，能处理各种复杂映射场景，是本节重点。

---

**`resultMap` 的用法和核心元素**

`resultMap` 元素是 MyBatis 中最重要也最强大的元素之一。它允许你显式地控制如何将数据库查询结果的列映射到 Java 对象的属性上。

一个基本的 `<resultMap>` 结构如下：

XML

```xml
<resultMap id="唯一的标识" type="目标Java类型的完全限定名或别名">
    <constructor>
        <idArg column="列名" javaType="java类型" jdbcType="JDBC类型"/>
        <arg column="列名" javaType="java类型" jdbcType="JDBC类型"/>
    </constructor>

    <id property="java属性名" column="数据库列名" javaType="java类型" jdbcType="JDBC类型" typeHandler="类型处理器"/>

    <result property="java属性名" column="数据库列名" javaType="java类型" jdbcType="JDBC类型" typeHandler="类型处理器"/>

    <association property="java属性名" javaType="关联对象的类型或别名" resultMap="外部resultMap的id" columnPrefix="列前缀">
        </association>

    <collection property="java属性名" ofType="集合中元素的类型或别名" resultMap="外部resultMap的id" columnPrefix="列前缀">
        </collection>

    <discriminator javaType="列的java类型" column="作为判别依据的列名">
        <case value="特定值" resultMap="此情况使用的resultMap的id" resultType="此情况使用的java类型"/>
        </discriminator>
</resultMap>
```

**主要属性和子元素详解：**

- **`id` (必需)**：此 `<resultMap>` 的唯一标识，供 `<select>` 语句的 `resultMap` 属性引用。
    
- **`type` (必需)**：期望映射成的 Java 对象的完全限定名或在 `<typeAliases>` 中定义的别名。
    
- **`autoMapping` (可选)**：布尔值 (`true`/`false`)。用于控制在此 `resultMap` 中是否启用或禁用自动映射。
    
    - 如果未设置，则继承全局的 `autoMappingBehavior` (默认为 `PARTIAL`，即自动映射未显式配置的属性)。
    - 如果设置为 `true`，则对所有未显式映射的属性尝试自动映射。
    - 如果设置为 `false`，则只映射显式配置的属性，未配置的即使列名和属性名匹配也不会自动映射。
- **`<constructor>`**：用于通过构造方法注入值来创建对象实例。
    
    - **`<idArg>`**：标记构造方法参数是主键。
    - **`<arg>`**：普通构造方法参数。
    - 属性：`column` (数据库列名), `javaType` (参数的Java类型), `jdbcType` (列的JDBC类型), `name` (构造方法参数名，如果参数名与属性名不一致或有多个相同类型的参数时需要), `typeHandler`。
- **`<id>`**：用于映射结果集中的主键列到对象的标识属性。标记为主键有助于 MyBatis 在进行嵌套查询和缓存时提高性能。
    
    - 属性：`property` (Java对象属性名), `column` (数据库列名), `javaType`, `jdbcType`, `typeHandler`。
- **`<result>`**：用于映射普通列到对象的普通属性。
    
    - 属性：`property` (Java对象属性名), `column` (数据库列名), `javaType`, `jdbcType`, `typeHandler`。
- **`javaType`**：指定 Java 属性的类型。通常 MyBatis 可以通过反射推断，但在某些情况下（如处理 `Map<String, Object>`）可能需要显式指定。
    
- **`jdbcType`**：指定数据库列的 JDBC 类型（来自 `java.sql.Types` 枚举）。在某些数据库驱动可能无法正确识别列类型，或者当列允许 `NULL` 值时，指定 `jdbcType` 可以确保正确的行为。
    
- **`typeHandler`**：允许你指定一个自定义的类型处理器，用于在 `jdbcType` 和 `javaType` 之间进行转换。

**使用场景和例子**

#### 1. 基本映射 (列名与属性名不一致)

假设 Java 对象 `User.java`:

Java

```
package com.example.model;
import java.util.Date;

public class User {
    private Integer userId;     // 对应 user_id
    private String fullName;   // 对应 user_full_name
    private String emailAddress; // 对应 email
    private Date registrationDate; // 对应 created_at

    // Getters and Setters
    public Integer getUserId() { return userId; }
    public void setUserId(Integer userId) { this.userId = userId; }
    public String getFullName() { return fullName; }
    public void setFullName(String fullName) { this.fullName = fullName; }
    public String getEmailAddress() { return emailAddress; }
    public void setEmailAddress(String emailAddress) { this.emailAddress = emailAddress; }
    public Date getRegistrationDate() { return registrationDate; }
    public void setRegistrationDate(Date registrationDate) { this.registrationDate = registrationDate; }
}
```

SQL 查询:

SQL

```
SELECT user_id, user_full_name, email, created_at FROM users WHERE user_id = #{id}
```

`UserMapper.xml` 中的 `<resultMap>`:

XML

```
<mapper namespace="com.example.mapper.UserMapper">
    <resultMap id="userResultMap" type="com.example.model.User">
        <id property="userId" column="user_id"/>
        <result property="fullName" column="user_full_name"/>
        <result property="emailAddress" column="email"/>
        <result property="registrationDate" column="created_at"/>
    </resultMap>

    <select id="selectUserById" parameterType="int" resultMap="userResultMap">
        SELECT user_id, user_full_name, email, created_at FROM users WHERE user_id = #{id}
    </select>
</mapper>
```

**说明**：即使开启了 `mapUnderscoreToCamelCase`，对于 `user_full_name` -> `fullName` 和 `email` -> `emailAddress` 这样的映射，还是需要显式指定，因为它不是简单的下划线转驼峰。

#### 2. 关联对象映射 (一对一 `<association>`)

假设一个订单 (`Order`) 关联一个客户 (`Customer`)。

`Customer.java`:

Java

```
package com.example.model;
public class Customer {
    private Integer customerId;
    private String customerName;
    // Getters and Setters...
}
```

`Order.java`:

Java

```
package com.example.model;
import java.util.Date;
public class Order {
    private Integer orderId;
    private Date orderDate;
    private Customer customer; // 关联的客户对象
    // Getters and Setters...
}
```

**方式一：嵌套 `resultMap` (通常用于 JOIN 查询)**

SQL 查询 (JOIN):

SQL

```
SELECT
    o.order_id, o.order_date,
    c.customer_id AS cust_id,  -- 使用别名避免列名冲突
    c.customer_name AS cust_name
FROM orders o
JOIN customers c ON o.customer_id = c.customer_id
WHERE o.order_id = #{id}
```

`OrderMapper.xml`:

XML

```
<mapper namespace="com.example.mapper.OrderMapper">
    <resultMap id="customerNestedMap" type="com.example.model.Customer">
        <id property="customerId" column="cust_id"/>
        <result property="customerName" column="cust_name"/>
    </resultMap>

    <resultMap id="orderWithCustomerMap" type="com.example.model.Order">
        <id property="orderId" column="order_id"/>
        <result property="orderDate" column="order_date"/>
        <association property="customer" javaType="com.example.model.Customer">
            <id property="customerId" column="cust_id"/>
            <result property="customerName" column="cust_name"/>
        </association>
    </resultMap>

    <select id="selectOrderWithCustomerById" parameterType="int" resultMap="orderWithCustomerMap">
        SELECT
            o.order_id, o.order_date,
            c.customer_id AS cust_id,
            c.customer_name AS cust_name
        FROM orders o
        JOIN customers c ON o.customer_id = c.customer_id
        WHERE o.order_id = #{id}
    </select>
</mapper>
```

**使用 `columnPrefix` 优化JOIN查询**

如果不想为JOIN的列取别名，可以使用 `columnPrefix`。 SQL:

SQL

```
SELECT
    o.order_id, o.order_date, o.customer_id, -- Order's own customer_id for join key
    c.customer_id, c.customer_name          -- Customer's details
FROM orders o
JOIN customers c ON o.customer_id = c.customer_id
WHERE o.order_id = #{id}
```

ResultMap:

XML

```
<resultMap id="orderWithCustomerMapPrefixed" type="com.example.model.Order">
    <id property="orderId" column="order_id"/>
    <result property="orderDate" column="order_date"/>
    <association property="customer" javaType="com.example.model.Customer" columnPrefix="c_">
         <id property="customerId" column="customer_id"/> <result property="customerName" column="customer_name"/> </association>
</resultMap>
<select id="selectOrderWithCustomerPrefixed" resultMap="orderWithCustomerMapPrefixed">
    SELECT
        o.order_id, o.order_date,
        c.customer_id AS c_customer_id,  -- 使用前缀匹配的列名
        c.customer_name AS c_customer_name
    FROM orders o
    INNER JOIN customers c ON o.customer_id = c.customer_id
    WHERE o.order_id = #{id}
</select>
```

`columnPrefix` 会自动给 `<association>` 内部所有 `column` 属性值加上这个前缀去匹配结果集中的列。

**方式二：嵌套 `select` 查询 (可能导致 N+1 问题)**

`OrderMapper.xml`:

XML

```
<mapper namespace="com.example.mapper.OrderMapper">
    <resultMap id="orderWithCustomerSelectMap" type="com.example.model.Order">
        <id property="orderId" column="order_id"/>
        <result property="orderDate" column="order_date"/>
        <association property="customer"
                     javaType="com.example.model.Customer"
                     column="customer_id"  REMOTE_STORAGE select="com.example.mapper.CustomerMapper.selectCustomerById"/>
    </resultMap>

    <select id="selectSimpleOrderById" parameterType="int" resultMap="orderWithCustomerSelectMap">
        SELECT order_id, order_date, customer_id FROM orders WHERE order_id = #{id}
    </select>
</mapper>
```

`CustomerMapper.xml` (需要一个对应的查询):

XML

```
<mapper namespace="com.example.mapper.CustomerMapper">
    <select id="selectCustomerById" parameterType="int" resultType="com.example.model.Customer">
        SELECT customer_id AS customerId, customer_name AS customerName FROM customers WHERE customer_id = #{id}
    </select>
</mapper>
```

**注意 N+1 问题**：如果查询100个订单，方式二会导致执行1条订单查询 + 100条客户查询。JOIN查询通常更高效。可以通过开启延迟加载 (`lazyLoadingEnabled`) 和积极的延迟加载 (`aggressiveLazyLoading=false`) 来缓解，但JOIN仍是首选。

#### 3. 关联集合映射 (一对多 `<collection>`)

假设一个作者 (`Author`) 有多篇文章 (`Post`)。

`Post.java`:

Java

```
package com.example.model;
public class Post {
    private Integer postId;
    private String title;
    // Getters and Setters...
}
```

`Author.java`:

Java

```
package com.example.model;
import java.util.List;
public class Author {
    private Integer authorId;
    private String authorName;
    private List<Post> posts; // 关联的文章列表
    // Getters and Setters...
}
```

**方式一：嵌套 `resultMap` (JOIN 查询)**

SQL 查询 (JOIN):

SQL

```
SELECT
    a.author_id, a.author_name,
    p.post_id AS p_post_id,  -- 使用别名
    p.title AS p_title
FROM authors a
LEFT JOIN posts p ON a.author_id = p.author_id
WHERE a.author_id = #{id}
```

`AuthorMapper.xml`:

XML

```
<mapper namespace="com.example.mapper.AuthorMapper">
    <resultMap id="authorWithPostsMap" type="com.example.model.Author">
        <id property="authorId" column="author_id"/>
        <result property="authorName" column="author_name"/>
        <collection property="posts" ofType="com.example.model.Post">
            <id property="postId" column="p_post_id"/>
            <result property="title" column="p_title"/>
            </collection>
    </resultMap>

    <select id="selectAuthorWithPostsById" parameterType="int" resultMap="authorWithPostsMap">
        SELECT
            a.author_id, a.author_name,
            p.post_id AS p_post_id,
            p.title AS p_title
        FROM authors a
        LEFT JOIN posts p ON a.author_id = p.author_id
        WHERE a.author_id = #{id}
    </select>
</mapper>
```

同样，`columnPrefix` 也可以在这里使用。

**方式二：嵌套 `select` 查询 (同样有 N+1 问题)** `AuthorMapper.xml`:

XML

```
<mapper namespace="com.example.mapper.AuthorMapper">
    <resultMap id="authorWithPostsSelectMap" type="com.example.model.Author">
        <id property="authorId" column="author_id"/>
        <result property="authorName" column="author_name"/>
        <collection property="posts"
                    ofType="com.example.model.Post"
                    column="author_id" select="com.example.mapper.PostMapper.selectPostsByAuthorId"/>
    </resultMap>

    <select id="selectSimpleAuthorById" parameterType="int" resultMap="authorWithPostsSelectMap">
        SELECT author_id, author_name FROM authors WHERE author_id = #{id}
    </select>
</mapper>
```

`PostMapper.xml`:

XML

```
<mapper namespace="com.example.mapper.PostMapper">
    <select id="selectPostsByAuthorId" parameterType="int" resultType="com.example.model.Post">
        SELECT post_id AS postId, title FROM posts WHERE author_id = #{authorId}
    </select>
</mapper>
```

#### 4. 鉴别器 (`<discriminator>`)

假设有一个 `Vehicle` 基类，和子类 `Car`, `Truck`。数据库中有一个 `vehicle_type` 列来区分。

`Vehicle.java` (可能是抽象类), `Car.java extends Vehicle`, `Truck.java extends Vehicle`。

`VehicleMapper.xml`:

XML

```
<mapper namespace="com.example.mapper.VehicleMapper">
    <resultMap id="vehicleBaseResultMap" type="com.example.model.Vehicle">
        <id property="vehicleId" column="vehicle_id"/>
        <result property="brand" column="brand"/>
        </resultMap>

    <resultMap id="carResultMap" type="com.example.model.Car" extends="vehicleBaseResultMap">
        <result property="numberOfDoors" column="num_doors"/> </resultMap>

    <resultMap id="truckResultMap" type="com.example.model.Truck" extends="vehicleBaseResultMap">
        <result property="payloadCapacity" column="payload_cap"/> </resultMap>

    <resultMap id="vehicleDiscriminatorMap" type="com.example.model.Vehicle">
        <id property="vehicleId" column="vehicle_id"/>
        <result property="brand" column="brand"/>
        <discriminator javaType="string" column="vehicle_type">
            <case value="CAR" resultMap="carResultMap"/>
            <case value="TRUCK" resultMap="truckResultMap"/>
        </discriminator>
    </resultMap>

    <select id="selectVehicleById" parameterType="int" resultMap="vehicleDiscriminatorMap">
        SELECT vehicle_id, brand, vehicle_type, num_doors, payload_cap
        FROM vehicles
        WHERE vehicle_id = #{id}
    </select>
</mapper>
```

**说明**：

- `extends` 属性允许 `resultMap` 继承另一个 `resultMap` 的配置。
- `<discriminator>` 会检查 `vehicle_type` 列的值：
    - 如果是 "CAR"，则使用 `carResultMap` 进行剩余的映射 (基于 `vehicleBaseResultMap` 并添加 `numberOfDoors`)。
    - 如果是 "TRUCK"，则使用 `truckResultMap`。

---

**总结 `resultMap` 的优势：**

- **精确控制**：完全掌控列到属性的映射过程。
- **处理复杂关系**：轻松实现一对一、一对多等关联关系。
- **代码解耦**：SQL 和 Java 对象的映射逻辑清晰分离。
- **可重用性**：定义好的 `resultMap` 可以在多个 `<select>` 语句中复用。
- **性能考虑**：通过 `<id>` 标签和合理的 JOIN 策略，可以优化复杂查询的性能。

虽然 `resultType` 在简单情况下很方便，但 `resultMap` 提供了处理真实世界复杂数据模型的强大能力和灵活性。掌握 `resultMap` 是精通 MyBatis 的关键。
