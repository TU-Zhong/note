“非默认的集合实现类”指的是，当我们在代码中定义一个接口类型的集合时（比如 `List`、`Set`、`Map`），Java 或框架（如 MyBatis）在实际创建这个集合的实例时，没有使用最常规、最通用的那个实现类，而是选择了另一个具有特定行为或性能特点的实现类。

让我们用一个简单的比喻来解释。

### 接口与实现类的关系：蓝图与建筑

- **接口 (Interface)** - 就像一张**“设计蓝图”**。
    
    - `List` 接口就是一张“列表蓝图”，它规定了“列表”这种东西必须要有 `add()` (添加)、`get()` (获取)、`remove()` (删除) 等功能。但蓝图本身不能住人。
- **实现类 (Implementation Class)** - 就像根据蓝图盖好的**“实体建筑”**。
    
    - `ArrayList` 和 `LinkedList` 都是根据 `List` 这张蓝图盖出来的不同建筑。它们都具备蓝图规定的所有功能，但内部结构和性能特点完全不同。

---

### 什么是“默认”实现类？

当你只声明了接口（蓝图）时，框架需要帮你选择一个最通用、最常见的实现（建筑）来实例化。这个最常见的选择，就是“默认实现类”。

|接口 (蓝图)|默认实现类 (最常见的建筑)|为什么是默认的？|
|:--|:--|:--|
|**`List<T>`**|**`ArrayList<T>`**|内部使用数组，查询快 (`get(index)` 操作 O(1))，是绝大多数场景下的首选，用途最广泛。|
|**`Set<T>`**|**`HashSet<T>`**|提供最快的添加和查找速度 (`add`/`contains` 操作 O(1))，但不保证顺序。|
|**`Map<K, V>`**|**`HashMap<K, V>`**|提供最快的存取速度 (`put`/`get` 操作 O(1))，但也不保证顺序。|

Export to Sheets

在 MyBatis 的 `<collection>` 映射中，如果你这样写：

Java

```
// 在你的 Dish.java 中
private List<DishFlavor> flavors;
```

MyBatis 看到 `List` 接口，它就会默认帮你创建一个 `ArrayList` 来存放 `DishFlavor` 对象。所以 `ArrayList` 就是这里的**默认实现类**。

---

### 什么又是“非默认”的实现类？

“非默认的实现类”就是指除了上面提到的默认选项之外，**其他也符合接口规范（也根据同一张蓝图建造）的类**。你选择它们，通常是为了利用它们特殊的内部结构所带来的性能优势或特定功能。

以下是一些常见的“非默认”选项：

#### 1. `List` 接口的非默认实现类

- **`LinkedList`**:
    
    - **内部结构**: 双向链表。每个元素都像链条一样，记录着它的上一个和下一个元素。
    - **优点**: 在列表的**开头或中间**插入、删除元素非常快。
    - **缺点**: 查询特定位置的元素很慢 (`get(index)`)，因为它需要从头开始一个一个地数过去。
    - **使用场景**: 当你的业务需要频繁地在列表头部或中间添加/删除数据时（虽然在数据库映射中不常见，但在其他算法中很常用）。
- **`CopyOnWriteArrayList`**:
    
    - **内部结构**: 数组。
    - **优点**: **线程安全**。特别适合“读多写少”的并发场景。读取操作完全不加锁，性能很高。
    - **缺点**: 写入操作（`add`, `remove`）非常昂贵，因为它会**复制整个底层数组**。
    - **使用场景**: 在多线程环境下，一个集合被大量线程读取，但很少被修改。

#### 2. `Set` 接口的非默认实现类

- **`LinkedHashSet`**:
    
    - **优点**: 在 `HashSet` 的基础上，额外维护了一个链表来记录元素的**插入顺序**。遍历时，元素会按照你添加它们的顺序出现。
    - **使用场景**: 需要去重，同时又希望保持元素插入时的顺序。
- **`TreeSet`**:
    
    - **优点**: 内部使用红黑树，集合中的元素**始终保持排序状态**（自然排序或自定义比较器排序）。
    - **使用场景**: 需要去重，同时希望元素随时都是有序的。

### 结论与 MyBatis 的关联

当你使用 MyBatis 的 `javaType` 属性时，你正是在告诉 MyBatis：“不要用你的默认选择，请为我使用这个**我指定的非默认实现类**。”

XML

```
<collection property="flavors" 
            ofType="com.sky.entity.DishFlavor"
            javaType="java.util.LinkedList">
    ...
</collection>

<collection property="flavors" 
            ofType="com.sky.entity.DishFlavor"
            javaType="java.util.LinkedHashSet">
    ...
</collection>
```

**总结一下：**

- **默认实现类** (`ArrayList`, `HashSet`) 是最通用、最常见的选择，MyBatis 会自动使用。
- **非默认实现类** (`LinkedList`, `LinkedHashSet` 等) 提供了特殊的性能或功能（如快速增删、保持顺序、排序等）。
- 你只在有**特殊需求**时，才需要通过 `javaType` 属性来指定使用一个“非默认的集合实现类”。在绝大多数情况下，默认的就足够好了。
