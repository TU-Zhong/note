### 数据库定义 (SQL DDL) 和示例数据

SQL

```sql
-- 清理旧表 (可选)
DROP TABLE IF EXISTS student_course;
DROP TABLE IF EXISTS course;

DROP TABLE IF EXISTS customer;

DROP TABLE IF EXISTS "user"; -- user 也是关键字

-- 1. 用户表 (User)
CREATE TABLE "user" (
    user_id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL UNIQUE,
    email VARCHAR(100) NOT NULL UNIQUE,
    registration_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 2. 用户资料表 (UserProfile) - 与User表一对一
CREATE TABLE user_profile (
    profile_id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT NOT NULL UNIQUE, -- 外键, 且唯一保证一对一
    bio TEXT,
    avatar_url VARCHAR(255),
    last_updated TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES "user"(user_id) ON DELETE CASCADE
);

-- 3. 客户表 (Customer)
CREATE TABLE customer (
    customer_id INT PRIMARY KEY AUTO_INCREMENT,
    customer_name VARCHAR(100) NOT NULL,
    contact_email VARCHAR(100),
    city VARCHAR(50)
);

-- 4. 订单表 (Order) - 与Customer表一对多
CREATE TABLE "order" ( -- "order" 因为 order 是关键字
    order_id INT PRIMARY KEY AUTO_INCREMENT,
    customer_id INT NOT NULL, -- 外键
    order_date DATE NOT NULL,
    total_amount DECIMAL(10, 2) NOT NULL,
    status VARCHAR(20) DEFAULT 'PENDING',
    FOREIGN KEY (customer_id) REFERENCES customer(customer_id) ON DELETE CASCADE
);

-- 5. 学生表 (Student)
CREATE TABLE student (
    student_id INT PRIMARY KEY AUTO_INCREMENT,
    student_name VARCHAR(100) NOT NULL,
    major VARCHAR(50)
);

-- 6. 课程表 (Course)
CREATE TABLE course (
    course_id INT PRIMARY KEY AUTO_INCREMENT,
    course_name VARCHAR(100) NOT NULL UNIQUE,
    credits INT NOT NULL
);

-- 7. 学生-课程关联表 (Student_Course) - 学生与课程多对多
CREATE TABLE student_course (
    student_id INT NOT NULL,
    course_id INT NOT NULL,
    enrollment_date DATE,
    grade VARCHAR(2), -- 例如 A, B, C, A+
    PRIMARY KEY (student_id, course_id), -- 复合主键
    FOREIGN KEY (student_id) REFERENCES student(student_id) ON DELETE CASCADE,
    FOREIGN KEY (course_id) REFERENCES course(course_id) ON DELETE CASCADE
);
```

### MyBatis 结果映射练习题

#### 前期准备

1. **配置 MyBatis**: 确保你的 `mybatis-config.xml` 配置正确，包括数据源、事务管理器，并启用了驼峰命名自动映射（推荐）：
    
    XML
    
    ```
    <settings>
        <setting name="mapUnderscoreToCamelCase" value="true"/>
    </settings>
    ```
    
2. **创建 Mapper 接口和 XML 文件**: 为每个主要实体创建 Mapper 接口和对应的 XML 文件（例如 `UserMapper.java`, `UserMapper.xml`, `CustomerMapper.java`, `CustomerMapper.xml` 等）。

#### 练习 1: 一对一映射 (User 和 UserProfile)

**目标**: 查询一个用户及其关联的用户资料。

1. **创建 `UserMapper.java` 接口**:
    
    Java
    
    ```java
    package com.example.mapper;
    import com.example.model.User;
    public interface UserMapper {
        User findUserWithProfileById(int userId);
    }
    ```
    
2. **创建 `UserMapper.xml` 文件**:
    - **任务**: 定义一个 `<resultMap>`，使用 `<association>` 标签来映射 `User` 对象中的 `profile` 属性。
    - 尝试两种方式实现 `<association>`:
        - **方式 A (推荐 - JOIN查询)**: 使用嵌套 `resultMap`。你的 SQL 语句应该使用 `LEFT JOIN` 连接 `user` 表和 `user_profile` 表。
        - **方式 B (了解N+1问题)**: 使用嵌套 `select` 查询。你需要额外创建一个 `UserProfileMapper` 和一个 `findProfileByUserId` 的查询。
    - **思考**: 如果一个用户没有对应的 `user_profile` 记录，`profile` 属性应该是什么？(应该是 `null`，`LEFT JOIN` 可以实现这点)。

**UserMapper.xml (方式 A 提示)**

XML

```xml
<mapper namespace="com.example.mapper.UserMapper">
    <resultMap id="userWithProfileResultMap" type="com.example.model.User">
        <id property="userId" column="user_id"/>
        <result property="username" column="username"/>
        <result property="email" column="email"/>
        <result property="registrationDate" column="registration_date"/>
        <association property="profile" javaType="com.example.model.UserProfile">
            <id property="profileId" column="profile_id"/>
            <result property="bio" column="bio"/>
            <result property="avatarUrl" column="avatar_url"/>
            <result property="lastUpdated" column="last_updated"/>
            </association>
    </resultMap>

    <select id="findUserWithProfileById" parameterType="int" resultMap="userWithProfileResultMap">
        SELECT
            u.user_id,
            u.username,
            u.email,
            u.registration_date,
            p.profile_id,
            p.bio,
            p.avatar_url,
            p.last_updated
        FROM "user" u
        LEFT JOIN user_profile p ON u.user_id = p.user_id
        WHERE u.user_id = #{userId}
    </select>
</mapper>
```

#### 练习 2: 一对多映射 (Customer 和 Order)

**目标**: 查询一个客户及其所有订单。

1. **创建 `CustomerMapper.java` 接口**:
    
    Java
    
    ```java
    package com.example.mapper;
    import com.example.model.Customer;
    public interface CustomerMapper {
        Customer findCustomerWithOrdersById(int customerId);
    }
    ```
    
2. **创建 `CustomerMapper.xml` 文件**:
    - **任务**: 定义一个 `<resultMap>`，使用 `<collection>` 标签来映射 `Customer` 对象中的 `orders` (一个 `List<Order>`) 属性。
    - 尝试两种方式实现 `<collection>`:
        - **方式 A (推荐 - JOIN查询)**: 使用嵌套 `resultMap`。SQL 语句使用 `LEFT JOIN` 连接 `customer` 表和 `order` 表。
        - **方式 B (了解N+1问题)**: 使用嵌套 `select` 查询。你需要额外创建一个 `OrderMapper` 和一个 `findOrdersByCustomerId` 的查询。
    - **思考**: 如果一个客户没有任何订单，`orders` 属性应该是什么？(应该是空列表 `[]`，而不是 `null`)。

**CustomerMapper.xml (方式 A 提示)**

XML

```xml
<mapper namespace="com.example.mapper.CustomerMapper">
    <resultMap id="customerWithOrdersResultMap" type="com.example.model.Customer">
        <id property="customerId" column="customer_id"/>
        <result property="customerName" column="customer_name"/>
        <result property="contactEmail" column="contact_email"/>
        <result property="city" column="city"/>
        <collection property="orders" ofType="com.example.model.Order">
            <id property="orderId" column="order_id"/>
            <result property="orderDate" column="order_date"/>
            <result property="totalAmount" column="total_amount"/>
            <result property="status" column="status"/>
            </collection>
    </resultMap>

    <select id="findCustomerWithOrdersById" parameterType="int" resultMap="customerWithOrdersResultMap">
        SELECT
            c.customer_id,
            c.customer_name,
            c.contact_email,
            c.city,
            o.order_id,
            o.order_date,
            o.total_amount,
            o.status
            -- o.customer_id AS o_customer_id -- 如果需要在Order对象中也保留 customer_id
        FROM customer c
        LEFT JOIN "order" o ON c.customer_id = o.customer_id
        WHERE c.customer_id = #{customerId}
    </select>
</mapper>
```

#### 练习 3: 多对多映射 (Student 和 Course)

**目标**:

- 查询一个学生及其所选的所有课程。
- 查询一门课程及其所有选课学生。

**3.1 查询学生及其课程**

1. **创建 `StudentMapper.java` 接口**:
    
    Java
    
    ```
    package com.example.mapper;
    import com.example.model.Student;
    public interface StudentMapper {
        Student findStudentWithCoursesById(int studentId);
    }
    ```
    
2. **创建 `StudentMapper.xml` 文件**:
    - **任务**: 定义一个 `<resultMap>`，使用 `<collection>` 标签映射 `Student` 对象中的 `courses` (一个 `List<Course>`) 属性。
    - SQL 语句需要 `JOIN` `student` 表, `student_course` 表, 和 `course` 表。
    - 可以只使用嵌套 `resultMap` 的方式。嵌套 `select` 在多对多中会更复杂且更容易导致严重的 N+1 问题。

**StudentMapper.xml (提示)**

XML

```xml
<mapper namespace="com.example.mapper.StudentMapper">
    <resultMap id="studentWithCoursesResultMap" type="com.example.model.Student">
        <id property="studentId" column="s_student_id"/> <result property="studentName" column="s_student_name"/>
        <result property="major" column="s_major"/>
        <collection property="courses" ofType="com.example.model.Course">
            <id property="courseId" column="c_course_id"/>
            <result property="courseName" column="c_course_name"/>
            <result property="credits" column="c_credits"/>
            </collection>
    </resultMap>

    <select id="findStudentWithCoursesById" parameterType="int" resultMap="studentWithCoursesResultMap">
        SELECT
            s.student_id AS s_student_id,
            s.student_name AS s_student_name,
            s.major AS s_major,
            c.course_id AS c_course_id,
            c.course_name AS c_course_name,
            c.credits AS c_credits
            -- sc.enrollment_date,  -- 如果需要映射到 Course 对象的临时属性或封装类
            -- sc.grade             -- 同上
        FROM student s
        LEFT JOIN student_course sc ON s.student_id = sc.student_id
        LEFT JOIN course c ON sc.course_id = c.course_id
        WHERE s.student_id = #{studentId}
    </select>
</mapper>
```

**3.2 查询课程及其学生**

1. **创建 `CourseMapper.java` 接口**:
    
    Java
    
    ```
    package com.example.mapper;
    import com.example.model.Course;
    public interface CourseMapper {
        Course findCourseWithStudentsById(int courseId);
    }
    ```
    
2. **创建 `CourseMapper.xml` 文件**:
    - **任务**: 定义一个 `<resultMap>`，使用 `<collection>` 标签映射 `Course` 对象中的 `students` (一个 `List<Student>`) 属性。
    - SQL 语句需要 `JOIN` `course` 表, `student_course` 表, 和 `student` 表。

**CourseMapper.xml (提示 - 结构与 StudentMapper 类似，只是主客体颠倒)**

XML

```xml
<mapper namespace="com.example.mapper.CourseMapper">
    <resultMap id="courseWithStudentsResultMap" type="com.example.model.Course">
        <id property="courseId" column="c_course_id"/>
        <result property="courseName" column="c_course_name"/>
        <result property="credits" column="c_credits"/>
        <collection property="students" ofType="com.example.model.Student">
            <id property="studentId" column="s_student_id"/>
            <result property="studentName" column="s_student_name"/>
            <result property="major" column="s_major"/>
        </collection>
    </resultMap>

    <select id="findCourseWithStudentsById" parameterType="int" resultMap="courseWithStudentsResultMap">
        SELECT
            c.course_id AS c_course_id,
            c.course_name AS c_course_name,
            c.credits AS c_credits,
            s.student_id AS s_student_id,
            s.student_name AS s_student_name,
            s.major AS s_major
        FROM course c
        LEFT JOIN student_course sc ON c.course_id = sc.course_id
        LEFT JOIN student s ON sc.student_id = s.student_id
        WHERE c.course_id = #{courseId}
    </select>
</mapper>
```

### 练习提示和技巧

- **别名非常重要**: 在 JOIN 查询中，不同表可能有相同的列名（如 `id`, `name`）。在 SQL 中使用列别名，并在 `<resultMap>` 的 `column` 属性中引用这些别名，可以避免歧义。
- **`ofType` vs `javaType`**: `<collection>` 用 `ofType` 指定集合内元素的类型；`<association>` 用 `javaType` 指定关联对象的类型。
- **N+1 问题**: 对于嵌套 `select` 的方式，要特别注意其性能影响。在实际项目中，通常优先考虑 JOIN 方式。
- **测试**: 编写单元测试来验证你的映射是否按预期工作，包括关联对象/集合是否正确加载，以及在没有关联数据时（如用户没有 Profile，客户没有订单）是否能正确处理（返回 `null` 或空集合）。
- **延迟加载 (Lazy Loading)**: 对于嵌套 `select`，可以研究 MyBatis 的延迟加载配置 (`lazyLoadingEnabled`, `aggressiveLazyLoading`)，但这只是缓解，不能完全替代高效的 JOIN。
