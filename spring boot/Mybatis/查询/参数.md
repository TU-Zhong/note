好的，下面我们针对 `<select>` 标签中的 `parameterType` 属性，给出不同使用场景的例子。

`parameterType` 用于指定传递给 SQL 语句的参数的类型。它可以是基本数据类型、包装类、`String`、Java Bean (POJO)、`Map` 等。

**重要提示：**

- 在很多情况下，`parameterType` 是可选的，MyBatis 可以根据 Mapper 接口方法的参数类型自动推断。但显式指定有助于提高可读性。
- 对于多个简单参数，推荐使用 `@Param` 注解，此时 `parameterType` 通常会被 MyBatis 视为一个 `Map`。

---

### 例子1: 单个基本数据类型或包装类或 String

当 Mapper 接口方法只有一个简单类型的参数时。

**Mapper 接口 (`UserMapper.java`):**

Java

```java
package com.example.mapper;

import com.example.model.User;

public interface UserMapper {
    // 1. 基本数据类型 int
    User selectUserById(int userId);

    // 2. 包装类 Integer
    User selectUserByAge(Integer age);

    // 3. String 类型
    User selectUserByUsername(String username);
}
```

**XML 映射文件 (`UserMapper.xml`):**

XML

```xml
<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper
    PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
    "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="com.example.mapper.UserMapper">

    <select id="selectUserById" parameterType="int" resultType="com.example.model.User">
        SELECT id, username, email FROM users WHERE id = #{userId}
        </select>

    <select id="selectUserByAge" parameterType="java.lang.Integer" resultType="com.example.model.User">
        SELECT id, username, email FROM users WHERE age = #{age}
    </select>

    <select id="selectUserByUsername" parameterType="string" resultType="com.example.model.User">
        SELECT id, username, email FROM users WHERE username = #{username}
    </select>

</mapper>
```

**说明:**

- `#{userId}`、`#{age}`、`#{username}` 中的名称对应 Mapper 接口方法的参数名。
- MyBatis 对常用类型有别名，如 `int`、`string`、`integer`。

---

### 例子2: 传递一个 POJO (Java Bean) 对象

当查询条件较多，可以将它们封装到一个对象中传递。

**POJO 类 (`UserSearchCriteria.java`):**

Java

```java
package com.example.model;

public class UserSearchCriteria {
    private String username;
    private String email;
    private Integer status;

    // Getters and Setters
    public String getUsername() { return username; }
    public void setUsername(String username) { this.username = username; }
    public String getEmail() { return email; }
    public void setEmail(String email) { this.email = email; }
    public Integer getStatus() { return status; }
    public void setStatus(Integer status) { this.status = status; }
}
```

**Mapper 接口 (`UserMapper.java`):**

Java

```
package com.example.mapper;

import com.example.model.User;
import com.example.model.UserSearchCriteria;
import java.util.List;

public interface UserMapper {
    List<User> findUsersByCriteria(UserSearchCriteria criteria);
}
```

**XML 映射文件 (`UserMapper.xml`):**

XML

```
<mapper namespace="com.example.mapper.UserMapper">

    <select id="findUsersByCriteria" parameterType="com.example.model.UserSearchCriteria" resultType="com.example.model.User">
        SELECT id, username, email FROM users
        <where>
            <if test="username != null and username != ''">
                AND username LIKE #{username}
            </if>
            <if test="email != null and email != ''">
                AND email = #{email}
            </if>
            <if test="status != null">
                AND status = #{status}
            </if>
        </where>
    </select>

</mapper>
```

**说明:**

- `parameterType="com.example.model.UserSearchCriteria"` 指定了参数是一个 `UserSearchCriteria` 对象。
- 在 SQL 语句中，使用 `#{propertyName}` 来引用 POJO 对象的属性，例如 `#{username}` 对应 `criteria.getUsername()`。

---

### 例子3: 传递一个 Map 对象

当参数不固定或者不想为参数专门创建一个 POJO 时，可以使用 `Map`。

**Mapper 接口 (`UserMapper.java`):**

Java

```
package com.example.mapper;

import com.example.model.User;
import java.util.List;
import java.util.Map;

public interface UserMapper {
    List<User> findUsersByMap(Map<String, Object> params);
}
```

**调用示例:**

Java

```
Map<String, Object> queryParams = new HashMap<>();
queryParams.put("targetUsername", "john.doe");
queryParams.put("minAge", 30);
List<User> users = userMapper.findUsersByMap(queryParams);
```

**XML 映射文件 (`UserMapper.xml`):**

XML

```
<mapper namespace="com.example.mapper.UserMapper">

    <select id="findUsersByMap" parameterType="map" resultType="com.example.model.User">
        SELECT id, username, email FROM users
        <where>
            <if test="targetUsername != null and targetUsername != ''">
                AND username = #{targetUsername}
            </if>
            <if test="minAge != null">
                AND age >= #{minAge}
            </if>
        </where>
    </select>

</mapper>
```

**说明:**

- `parameterType="map"` (或 `java.util.Map`)。
- 在 SQL 语句中，使用 `#{keyName}` 来引用 `Map` 中的键对应的值，例如 `#{targetUsername}` 对应 `params.get("targetUsername")`。

---

### 例子4: 多个简单参数 (使用 @Param 注解)

当 Mapper 接口方法有多个简单类型参数时，推荐使用 `@Param` 注解为每个参数命名。MyBatis 内部会将这些参数封装成一个 `Map`。

**Mapper 接口 (`UserMapper.java`):**

Java

```
package com.example.mapper;

import com.example.model.User;
import org.apache.ibatis.annotations.Param;
import java.util.List;

public interface UserMapper {
    List<User> findUsersByNameAndStatus(@Param("nameParam") String username, @Param("statusParam") Integer status);
}
```

**XML 映射文件 (`UserMapper.xml`):**

XML

```
<mapper namespace="com.example.mapper.UserMapper">

    <select id="findUsersByNameAndStatus" resultType="com.example.model.User">
        SELECT id, username, email FROM users
        WHERE username LIKE #{nameParam} AND status = #{statusParam}
    </select>

    </mapper>
```

**说明:**

- `@Param("nameParam")` 将方法参数 `username` 在 SQL 映射中命名为 `nameParam`。
- `#{nameParam}` 和 `#{statusParam}` 对应 `@Param` 注解中指定的名字。
- 虽然 MyBatis 内部将参数处理为 `Map`，但通常在这种情况下，你不需要在 `<select>` 标签中显式声明 `parameterType="map"`，MyBatis 能够正确处理。

---

### 总结 `parameterType`

|场景|Mapper 接口参数示例|XML `parameterType` (或其别名)|SQL中引用方式示例|
|:--|:--|:--|:--|
|单个基本类型/包装类/String|`int id` 或 `String name`|`int`, `java.lang.Integer`, `string`, `_int` 等|`#{id}`, `#{name}`|
|POJO 对象|`UserSearchCriteria criteria`|对象的完全限定名 (如`com.example.model.User`) 或其别名|`#{propertyName}`|
|Map 对象|`Map<String, Object> params`|`map` 或 `java.util.Map`|`#{keyName}`|
|多个简单类型 (使用 `@Param`)|`@Param("name") String name, @Param("age") int age`|通常省略 (MyBatis 内部视为 `map`)|`#{name}`, `#{age}`|
|列表或数组 (用于 `IN` 子句等)|`List<Integer> ids` 或 `Integer[] idArray`|`list`, `array`, `java.util.List`|(通常在 `<foreach>` 中使用)|

选择哪种 `parameterType` 取决于传递给 SQL 的参数的结构和数量。目标是保持代码的清晰和易于维护。
