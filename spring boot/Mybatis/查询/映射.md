好的，很乐意为您详细介绍 MyBatis 的映射（Mapping）。

MyBatis 的核心思想是将 SQL 语句与 Java 代码分离，通过映射文件（通常是 XML 文件）或注解的方式，将 Java 对象、方法与 SQL 语句关联起来，从而实现对数据库的灵活操作。映射是 MyBatis 的灵魂所在。

我们主要从以下几个方面来理解 MyBatis 的映射：

1. **映射文件 (XML Mapper)**
2. **Mapper 接口 (Java Interface)**
3. **核心映射元素 (SQL 语句标签)**
4. **参数映射 (Parameter Mapping)**    [[参数]]
5. **结果映射 (Result Mapping)**    [[结果映射]]
6. **动态 SQL (Dynamic SQL)**
7. **注解方式的映射 (Annotation-based Mapping)**

---

### 1. 映射文件 (XML Mapper)

这是 MyBatis 最常用也是功能最强大的映射方式。一个 XML 映射文件通常对应一个数据访问接口 (DAO/Mapper 接口)。

- **文件结构和声明:**
    
    - XML 文件通常以 `.xml` 为后缀，例如 `UserMapper.xml`。
    - 文件头部需要有 XML 声明和 DOCTYPE 声明，指向 MyBatis 的 DTD 定义，用于验证 XML 文件的结构。
    
    XML
    
    ```xml
    <?xml version="1.0" encoding="UTF-8" ?>
    <!DOCTYPE mapper
        PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
    ```
    
- **`<mapper>` 根元素:**
    
    - 所有映射配置都包含在 `<mapper>` 元素内。
    - 最重要的属性是 `namespace`。这个属性通常指向一个 Mapper 接口的完全限定名（例如 `com.example.mapper.UserMapper`）。`namespace` 的作用有两个：
        1. **隔离作用域**：使得不同 XML 映射文件中的 `id` 可以重复（尽管不推荐）。
        2. **接口绑定**：将 XML 文件与对应的 Java Mapper 接口关联起来。通过 `namespace + statement id` 可以唯一确定一条 SQL 语句。
    
    XML
    
    ```sql
    <mapper namespace="com.example.mapper.UserMapper">
        </mapper>
    ```

### 2. Mapper 接口 (Java Interface)

Mapper 接口是 Java 代码与 SQL 语句之间的桥梁。

- **定义:**
    
    - 通常是一个普通的 Java 接口，例如 `UserMapper.java`。
    - 接口中的方法名需要与 XML 映射文件中 SQL 语句的 `id` 对应。
    - 方法的参数类型和数量需要与 SQL 语句中的参数占位符对应。
    - 方法的返回值类型需要与 SQL 语句查询结果的类型对应。
    
    Java
    
    ```java
    package com.example.mapper;
    
    import com.example.model.User;
    import java.util.List;
    
    public interface UserMapper {
        User selectUserById(int id);
        List<User> selectAllUsers();
        void insertUser(User user);
        int updateUser(User user);
        int deleteUser(int id);
    }
    ```
    
- **绑定机制:**
    
    - 当 `UserMapper.xml` 的 `namespace` 设置为 `com.example.mapper.UserMapper` 时，MyBatis 会自动将这个 XML 文件与 `UserMapper` 接口关联起来。
    - 调用 `UserMapper` 接口中的方法（例如 `selectUserById`）时，MyBatis 会在对应的 XML 文件中查找 `id` 为 `selectUserById` 的 SQL 语句并执行。

### 3. 核心映射元素 (SQL 语句标签)

这些标签用于定义具体的 SQL 操作。

- **`<select>`:** 用于定义查询语句。
    
    - `id`: 唯一标识，与 Mapper 接口中的方法名对应。
    - `parameterType`: (可选) 传入 SQL 语句的参数的完全限定名或别名。如果方法有多个参数，可以使用 `@Param` 注解或 `Map`，或者直接传递一个包含所有参数的 POJO。
	    parameterType  [[参数]]  <mark style="background: #FFF3A3A6;">多参数时可选择接口内添加@Param注解或者 传入Map对象或者传入 pojo对象</mark>
    - `resultType`: (可选) SQL 查询结果集映射的 Java 对象的完全限定名或别名。适用于简单对象，MyBatis 会自动将列名与属性名进行匹配（忽略大小写，或通过驼峰命名转换）。
	    在mybatis-config中设置：
	    ```xml
	    <settings>  
		<setting name="mapUnderscoreToCamelCase" value="true"/>  
		</settings>
		```
		<mark style="background: #FFF3A3A6;">即可设置匹配驼峰命名</mark>
    - `resultMap`: (可选) 指向一个预定义的 `<resultMap>` 元素的 `id`，用于更复杂的自定义结果映射。`resultType` 和 `resultMap` 不能同时使用。
    
    XML
    
    ```sql
    <select id="selectUserById" parameterType="int" resultType="com.example.model.User">
        SELECT id, username, email, created_at AS createdAt
        FROM users
        WHERE id = #{id}
    </select>
    ```
    
- **`<insert>`:** 用于定义插入语句。
	    <mark style="background: #FFF3A3A6;">执行完成后要调用sqlsession.commit()方法进行提交，不然数据库不予更改</mark>
    - `id`: 唯一标识。
    - `parameterType`: (可选) 传入参数的类型。
    - `useGeneratedKeys`: (可选，默认为 `false`) 设为 `true` 时，表示数据库支持主键自动生成，并且 MyBatis 会将生成的主键值赋给传入参数对象的相应属性。
    - `keyProperty`: (可选) 与 `useGeneratedKeys` 配合使用，指定主键值应该赋给 `parameterType` 对象中的哪个属性。
    - `keyColumn`: (可选) 设置数据库表中表示主键的列名（如果与 `keyProperty` 名称不一致）。
    
    XML
    
    ```sql
    <insert id="insertUser" parameterType="com.example.model.User" useGeneratedKeys="true" keyProperty="id">
        INSERT INTO users (username, email, password)
        VALUES (#{username}, #{email}, #{password})
    </insert>
    ```
    
- **`<update>`:** 用于定义更新语句。
    
    - `id`: 唯一标识。
    - `parameterType`: (可选) 传入参数的类型。
    - 通常返回一个整数，表示受影响的行数。
    
    XML
    
    ```sql
    <update id="updateUser" parameterType="com.example.model.User">
        UPDATE users
        SET username = #{username}, email = #{email}
        WHERE id = #{id}
    </update>
    ```
    
- **`<delete>`:** 用于定义删除语句。
    
    - `id`: 唯一标识。
    - `parameterType`: (可选) 传入参数的类型。
    - 通常返回一个整数，表示受影响的行数。
    
    XML
    
    ```sql
    <delete id="deleteUser" parameterType="int">
        DELETE FROM users WHERE id = #{id}
    </delete>
    ```

### 4. 参数映射 (Parameter Mapping)

MyBatis 如何将 Java 方法的参数传递给 SQL 语句。

- **`#{}` (预编译参数):**
    
    - 这是推荐的参数占位符方式。
    - MyBatis 会将 `#{}` 替换为 `?`，并使用 `PreparedStatement` 来设置参数。这可以有效防止 SQL 注入。
    - `#{propertyName}`: `propertyName` 是传入参数对象的属性名，或者 Map 的键名，或者使用 `@Param` 注解指定的参数名。
    - 可以指定参数的 Java 类型和 JDBC 类型：`#{propertyName, javaType=String, jdbcType=VARCHAR}`。通常 MyBatis 可以自动推断。
- **`${}` (字符串替换):**
    
    - 这会将 `${}` 中的内容直接拼接到 SQL 语句中，**不是预编译的**。
    - **存在 SQL 注入风险**，除非内容是完全可信的或者不是用户输入。
    - 常用于动态表名、列名或 `ORDER BY` 子句。
    - 例如: `ORDER BY ${columnName}`。
- **参数类型 (`parameterType`):**
    
    - **基本类型:** `int`, `String`, `double` 等。<mark style="background: #FFF3A3A6;">如果 Mapper 接口方法有多个基本类型参数，需要使用 </mark>`@Param("paramName")` 注解。
        
        Java
        
        ```
        User findUser(@Param("username") String username, @Param("password") String password);
        ```
        
        XML
        
        ```
        <select id="findUser" resultType="com.example.model.User">
            SELECT * FROM users WHERE username = #{username} AND password = #{password}
        </select>
        ```
        
    - **POJO (Plain Old Java Object):** 直接传递一个对象，SQL 中使用 `#{propertyName}`。
    - **Map:** 传递一个 `Map`，SQL 中使用 `#{keyName}`。

### 5. 结果映射 (Result Mapping)
[[结果映射]]
MyBatis 如何将 SQL 查询结果集映射到 Java 对象。

- **`resultType` (自动映射):**
    
    - 适用于简单场景，当数据库列名与 Java 对象属性名能够自动匹配时（忽略大小写，或通过驼峰命名转换，例如 `created_at` 映射到 `createdAt`）。
    - MyBatis 会创建一个 `resultType` 指定类型的对象，并尝试将查询结果的每一列填充到对象的同名属性中。
- **`<resultMap>` (自定义映射):**
    
    - 用于处理复杂的结果映射，例如：
        - 列名与属性名不匹配。
        - 关联查询 (一对一、一对多)。
        [[这个association,collection,和discriminator标签分别是什么意思，有什么用？]]
        - 鉴别器 (discriminator)。
    - `<resultMap>` 元素定义在 `<mapper>` 根元素下，具有一个 `id` 和一个 `type` (要映射到的 Java 类型)。
    - **子元素:**
        - `<constructor>`: 用于通过构造方法注入值。
        - `<id>`: 标记主键字段，有助于性能优化。
            - `property`: Java 对象属性名。
            - `column`: 数据库列名。
        - `<result>`: 普通字段的映射。
            - `property`: Java 对象属性名。
            - `column`: 数据库列名。
            - `javaType`: Java 属性类型。
            - `jdbcType`: 数据库列的 JDBC 类型。
            - `typeHandler`: 自定义类型处理器。
        - `<association>`: 用于一对一关联映射。
            - `property`: 关联对象在主对象中的属性名。
            - `javaType`: 关联对象的类型。
            - 可以嵌套另一个 `<resultMap>` 或通过 `select` 执行另一个查询。
        - `<collection>`: 用于一对多关联映射。
            - `property`: 集合属性在主对象中的属性名。
            - `ofType`: 集合中元素的类型。
            - 可以嵌套另一个 `<resultMap>` 或通过 `select` 执行另一个查询。
        - `<discriminator>`: 鉴别器，根据某个列的值决定使用哪个映射规则。
    
    **示例 `<resultMap>`:**
    
    XML
    
    ```xml
    <resultMap id="userResultMap" type="com.example.model.User">
        <id property="id" column="user_id"/>
        <result property="username" column="user_name"/>
        <result property="emailAddress" column="email_addr"/>
        <result property="createdAt" column="created_time"/>
        <association property="address" javaType="com.example.model.Address">
            <id property="id" column="addr_id"/>
            <result property="street" column="addr_street"/>
            <result property="city" column="addr_city"/>
        </association>
        <collection property="posts" ofType="com.example.model.Post">
            <id property="id" column="post_id"/>
            <result property="title" column="post_title"/>
            <result property="content" column="post_content"/>
        </collection>
    </resultMap>
    
    <select id="selectUserWithDetails" resultMap="userResultMap">
        SELECT
            u.user_id, u.user_name, u.email_addr, u.created_time,
            a.addr_id, a.addr_street, a.addr_city,
            p.post_id, p.post_title, p.post_content
        FROM
            users u
        LEFT JOIN
            addresses a ON u.address_id = a.addr_id
        LEFT JOIN
            posts p ON u.user_id = p.user_id
        WHERE
            u.user_id = #{id}
    </select>
    ```

### 6. 动态 SQL (Dynamic SQL)

MyBatis 的强大特性之一，允许根据条件动态构建 SQL 语句。

- **常用动态 SQL 标签:**
    - `<if test="expression">`: 条件判断，如果 `expression` 为 `true`，则包含其内容。
        
        XML
        
        ```
        <select id="findActiveUser" resultType="com.example.model.User">
            SELECT * FROM users
            WHERE state = 'ACTIVE'
            <if test="username != null and username != ''">
                AND username LIKE #{username}
            </if>
        </select>
        ```
        
    - `<choose>`, `<when>`, `<otherwise>`: 类似于 Java 的 `switch` 语句。
        
        XML
        
        ```
        <select id="findUserByChoice" resultType="com.example.model.User">
            SELECT * FROM users
            <choose>
                <when test="id != null">
                    WHERE id = #{id}
                </when>
                <when test="username != null and username != ''">
                    WHERE username = #{username}
                </when>
                <otherwise>
                    WHERE state = 'GUEST'
                </otherwise>
            </choose>
        </select>
        ```
        
    - `<where>`: 智能处理 `AND` 或 `OR`。如果其包含的条件都不满足，则不生成 `WHERE` 关键字。如果其内容以 `AND` 或 `OR` 开头，会自动去除。
        
        XML
        
        ```
        <select id="findUserWithWhere" resultType="com.example.model.User">
            SELECT * FROM users
            <where>
                <if test="state != null">
                    state = #{state}
                </if>
                <if test="username != null and username != ''">
                    AND username LIKE #{username}
                </if>
            </where>
        </select>
        ```
        
    - `<set>`: 用于 `UPDATE` 语句，智能处理逗号。如果其包含的条件都不满足，则不生成 `SET` 关键字。如果其内容以逗号结尾，会自动去除。
        
        XML
        
        ```
        <update id="updateUserIfNecessary" parameterType="com.example.model.User">
            UPDATE users
            <set>
                <if test="username != null and username != ''">username = #{username},</if>
                <if test="email != null and email != ''">email = #{email},</if>
            </set>
            WHERE id = #{id}
        </update>
        ```
        
    - `<trim prefix="" suffix="" prefixOverrides="" suffixOverrides="">`: 更通用的前缀/后缀处理标签。
    - `<foreach collection="list|array|map" item="item" index="index" open="(" separator="," close=")">`: 用于遍历集合，常用于构建 `IN` 子句。
        
        XML
        
        ```
        <select id="selectUsersByIds" resultType="com.example.model.User">
            SELECT * FROM users
            WHERE id IN
            <foreach item="id" collection="list" open="(" separator="," close=")">
                #{id}
            </foreach>
        </select>
        ```
        
    - `<sql>` 和 `<include>`: 定义可重用的 SQL 片段。
        
        XML
        
        ```
        <sql id="userColumns">id, username, email</sql>
        <select id="selectSomeUsers" resultType="com.example.model.User">
            SELECT <include refid="userColumns"/> FROM users
        </select>
        ```
        
    - `<bind name="pattern" value="'%' + _parameter.getTitle() + '%'" />`: 创建一个变量，并将其绑定到 OGNL 表达式的结果。

### 7. 注解方式的映射 (Annotation-based Mapping)

对于简单的 SQL 语句，可以直接在 Mapper 接口的方法上使用注解来定义。

- **常用注解:**
    - `@Select`: `select * from users where id = #{id}`
    - `@Insert`: `insert into users (name, email) values (#{name}, #{email})`
    - `@Update`: `update users set name = #{name} where id = #{id}`
    - `@Delete`: `delete from users where id = #{id}`
    - `@Results`, `@Result`, `@Many`, `@One`: 用于结果映射，功能类似于 XML 中的 `<resultMap>`、`<result>`、`<collection>`、`<association>`。
    - `@Param`: 用于给方法参数命名，当方法有多个参数或者参数是集合/数组时使用。
    - `@Options`: 配置一些额外的选项，如 `useGeneratedKeys`, `keyProperty`。
    - `@SelectProvider`, `@InsertProvider`, `@UpdateProvider`, `@DeleteProvider`: 用于构建动态 SQL，指向一个类和方法来提供 SQL 语句字符串。

**示例注解映射:**

Java

```
package com.example.mapper;

import com.example.model.User;
import org.apache.ibatis.annotations.*;
import java.util.List;

public interface UserAnnotationMapper {

    @Select("SELECT id, username, email, created_at AS createdAt FROM users WHERE id = #{userId}")
    @Results({
        @Result(property = "id", column = "id"),
        @Result(property = "username", column = "username"),
        @Result(property = "email", column = "email"),
        @Result(property = "createdAt", column = "created_at")
    })
    User getUserById(@Param("userId") int id);

    @Insert("INSERT INTO users(username, email, password) VALUES(#{username}, #{email}, #{password})")
    @Options(useGeneratedKeys = true, keyProperty = "id")
    int insertUser(User user);

    @Update("UPDATE users SET username=#{username}, email=#{email} WHERE id =#{id}")
    int updateUser(User user);

    @Delete("DELETE FROM users WHERE id =#{id}")
    int deleteUser(@Param("id") int id);

    // 使用 Provider 构建动态 SQL
    @SelectProvider(type = UserSqlBuilder.class, method = "buildGetUsersByName")
    List<User> getUsersByName(@Param("name") String name);

    class UserSqlBuilder {
        public String buildGetUsersByName(@Param("name") String name) {
            // 在 Java 代码中构建 SQL
            return new org.apache.ibatis.jdbc.SQL() {{
                SELECT("*");
                FROM("users");
                if (name != null && !name.isEmpty()) {
                    WHERE("username LIKE #{name} || '%'");
                }
                ORDER_BY("id");
            }}.toString();
        }
    }
}
```

**XML vs 注解:**

- **XML:**
    - **优点:** SQL 与 Java 代码完全分离，更易于维护复杂的 SQL 和动态 SQL，功能更全面。
    - **缺点:** 文件数量可能较多。
- **注解:**
    - **优点:** 对于简单 SQL，更简洁直观，所有定义都在 Java 接口中。
    - **缺点:** 复杂 SQL 或动态 SQL 写起来比较麻烦，SQL 和 Java 代码耦合度高。

通常建议：

- 简单、固定的 SQL 语句可以使用注解。
- 复杂 SQL、动态 SQL、需要高级结果映射的，优先使用 XML。
- 也可以混合使用，一个项目中的某些 Mapper 用 XML，某些用注解。

---

### 总结

MyBatis 的映射机制是其强大功能的核心。通过 XML 映射文件或注解，开发者可以将 SQL 语句从 Java 代码中解耦出来，专注于业务逻辑的实现。

- `Mapper 接口` 定义操作。
- `XML 映射文件` (或注解) 实现操作，将接口方法与 SQL 关联。
- `参数映射` (`#{}` vs `${}`) 负责将 Java 参数传递给 SQL。
- `结果映射` (`resultType` vs `resultMap`) 负责将 SQL 结果转换为 Java 对象。
- `动态 SQL` 提供了在运行时根据条件构建 SQL 的灵活性。

理解并熟练运用这些映射机制，是高效使用 MyBatis 的关键。
