Okay, let's craft the solution for the "查询特定状态的订单，并按日期排序" exercise. This requires combining several dynamic SQL elements (`<where>`, `<if>`, `<foreach>`) along with parameter handling and ordering.

---

### 1. Mapper 接口方法 (`OrderMapper.java`)

Java

```
package com.example.mapper; // 你的包名

import com.example.model.Order;
import org.apache.ibatis.annotations.Param;
import java.util.Date;
import java.util.List;

public interface OrderMapper {

    /**
     * 根据状态列表和日期范围动态查询订单列表，并按订单日期降序排序。
     *
     * @param statusList 允许的状态列表。如果为 null 或空，则不按状态过滤。
     * @param startDate  订单日期的开始范围（包含）。如果为 null，则不限制开始日期。
     * @param endDate    订单日期的结束范围（包含）。如果为 null，则不限制结束日期。
     * @return 符合条件的订单列表，按订单日期降序排列。
     */
    List<Order> findOrdersByStatusAndDateRange(
        @Param("statusList") List<String> statusList,
        @Param("startDate") Date startDate,
        @Param("endDate") Date endDate
    );
}
```

- `@Param` 注解是必需的，因为它有多个参数，MyBatis 需要明确知道如何在 XML 中引用 `statusList`, `startDate`, 和 `endDate`。

---

### 2. Mapper XML 配置 (`OrderMapper.xml`)

XML

```
<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN" "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="com.example.mapper.OrderMapper">

    <select id="findOrdersByStatusAndDateRange" resultType="com.example.model.Order">
        SELECT orderId, customerId, orderDate, totalAmount, status
        FROM Orders
        <where>
            <if test="statusList != null and !statusList.isEmpty()">
                AND status IN
                <foreach item="statusItem" collection="statusList"
                         open="(" separator="," close=")">
                    #{statusItem}
                </foreach>
            </if>

            <if test="startDate != null">
                AND orderDate >= #{startDate}
            </if>

            <if test="endDate != null">
                AND orderDate <= #{endDate}
            </if>
        </where>
        ORDER BY orderDate DESC
    </select>

</mapper>
```

**解释:**

1. **`resultType="com.example.model.Order"`**: 指定查询结果映射到 `Order` 对象列表。
2. **`<where>`**: 智能处理 `WHERE` 子句。如果内部没有任何 `<if>` 条件满足，则不会生成 `WHERE` 子句。如果至少有一个条件满足，则会自动添加 `WHERE` 并移除第一个条件前多余的 `AND`。
3. **`<if test="statusList != null and !statusList.isEmpty()">`**:
    - 检查 `statusList` 参数是否存在且包含元素。如果传入的是 `null` 或者一个空的 `List`，这个条件块将被忽略。
    - `!statusList.isEmpty()` 是 OGNL 表达式，用于检查集合是否为空。
4. **`<foreach item="statusItem" collection="statusList" open="(" separator="," close=")">#{statusItem}</foreach>`**:
    - 这是生成 `IN` 子句内容的关键。
    - `collection="statusList"` 明确指定要迭代的是名为 `statusList` 的参数。
    - 它会为 `statusList` 中的每个字符串 `statusItem` 生成一个 `#{statusItem}` 占位符，并用逗号分隔，最后用括号包围，形成 `IN ('status1', 'status2', ...)` 的效果。
5. **`<if test="startDate != null">` 和 `<if test="endDate != null">`**:
    - 分别检查 `startDate` 和 `endDate` 参数是否为 `null`。
    - 如果不为 `null`，则添加相应的日期比较条件 (`>=` 或 `<=`)。
6. **`ORDER BY orderDate DESC`**:
    - 这部分是静态的，总是在 `WHERE` 子句（如果存在）之后添加，确保结果始终按订单日期降序排列。

### 3. 如何调用和测试

Java

```
import com.example.mapper.OrderMapper;
import com.example.model.Order;
import com.example.util.MyBatisSqlSessionFactoryUtil;
import org.apache.ibatis.session.SqlSession;

import java.text.SimpleDateFormat;
import java.util.Arrays;
import java.util.Date;
import java.util.List;

public class OrderServiceTest {

    public static void main(String[] args) {
        SimpleDateFormat sdf = new SimpleDateFormat("yyyy-MM-dd");

        try (SqlSession session = MyBatisSqlSessionFactoryUtil.openSession()) {
            OrderMapper orderMapper = session.getMapper(OrderMapper.class);

            // 场景1: 查询 PENDING 和 PROCESSING 状态的订单
            System.out.println("--- 查询 PENDING 和 PROCESSING ---");
            List<String> statuses1 = Arrays.asList("PENDING", "PROCESSING");
            List<Order> orders1 = orderMapper.findOrdersByStatusAndDateRange(statuses1, null, null);
            orders1.forEach(o -> System.out.println(o + " Date: " + sdf.format(o.getOrderDate())));

            // 场景2: 查询 2025-03-01 及之后的订单
            System.out.println("\n--- 查询 2025-03-01 之后 ---");
            Date startDate = sdf.parse("2025-03-01");
            List<Order> orders2 = orderMapper.findOrdersByStatusAndDateRange(null, startDate, null);
            orders2.forEach(o -> System.out.println(o + " Date: " + sdf.format(o.getOrderDate())));

            // 场景3: 查询 2025-04-30 及之前的订单
            System.out.println("\n--- 查询 2025-04-30 之前 ---");
            Date endDate = sdf.parse("2025-04-30");
            List<Order> orders3 = orderMapper.findOrdersByStatusAndDateRange(null, null, endDate);
            orders3.forEach(o -> System.out.println(o + " Date: " + sdf.format(o.getOrderDate())));

            // 场景4: 查询 2025-02-01 到 2025-04-30 之间，状态为 DELIVERED 的订单
            System.out.println("\n--- 查询 2025-02-01 到 2025-04-30 状态为 DELIVERED ---");
            List<String> statuses4 = Arrays.asList("DELIVERED");
            Date startDate4 = sdf.parse("2025-02-01");
            Date endDate4 = sdf.parse("2025-04-30");
            List<Order> orders4 = orderMapper.findOrdersByStatusAndDateRange(statuses4, startDate4, endDate4);
            orders4.forEach(o -> System.out.println(o + " Date: " + sdf.format(o.getOrderDate())));

            // 场景5: 所有条件都为 null (查询所有订单，按日期降序)
            System.out.println("\n--- 查询所有订单 ---");
            List<Order> orders5 = orderMapper.findOrdersByStatusAndDateRange(null, null, null);
            orders5.forEach(o -> System.out.println(o + " Date: " + sdf.format(o.getOrderDate())));

        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```

这个题解完整地利用了 MyBatis 的动态 SQL 功能来构建一个灵活的查询方法，能够根据传入参数的不同组合生成不同的 `WHERE` 条件，同时保证了结果的排序。
