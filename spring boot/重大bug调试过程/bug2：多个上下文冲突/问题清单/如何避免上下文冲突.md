### Q: 那我要用AnnotationConfigApplicationContext但我又不想引起冲突，应该怎么办

### A:如果你需要使用 `AnnotationConfigApplicationContext`，但又不想引起与 Spring Boot 的默认上下文初始化冲突，可以通过以下方式实现：

---

### 1. **明确初始化 `AnnotationConfigApplicationContext` 的范围**
`AnnotationConfigApplicationContext` 是一个独立的 Spring 上下文，它可以与 Spring Boot 的默认上下文分开使用。你可以在不干扰 Spring Boot 上下文的情况下，手动初始化并使用它。

#### 示例：
```java
@SpringBootApplication
public class ValueApplication {

    public static void main(String[] args) {
        // 启动 Spring Boot 应用
        ConfigurableApplicationContext springBootContext = SpringApplication.run(ValueApplication.class, args);

        // 手动初始化 AnnotationConfigApplicationContext
        AnnotationConfigApplicationContext customContext = new AnnotationConfigApplicationContext();
        customContext.scan("com.example.custom"); // 扫描自定义包
        customContext.refresh();

        // 从自定义上下文中获取 Bean
        CustomBean customBean = customContext.getBean(CustomBean.class);
        System.out.println(customBean);

        // 从 Spring Boot 上下文中获取 Bean
        ValueBean valueBean = springBootContext.getBean(ValueBean.class);
        System.out.println(valueBean);

        // 关闭自定义上下文
        customContext.close();
    }
}
```

---

### 2. **避免重复扫描相同的包**
确保 `AnnotationConfigApplicationContext` 扫描的包与 Spring Boot 默认扫描的包不重叠，以避免 Bean 冲突。

#### 示例：
```java
@SpringBootApplication(scanBasePackages = "com.example.springboot")
public class ValueApplication {

    public static void main(String[] args) {
        // 启动 Spring Boot 应用
        ConfigurableApplicationContext springBootContext = SpringApplication.run(ValueApplication.class, args);

        // 手动初始化 AnnotationConfigApplicationContext，扫描不同的包
        AnnotationConfigApplicationContext customContext = new AnnotationConfigApplicationContext();
        customContext.scan("com.example.custom"); // 扫描自定义包
        customContext.refresh();

        // 从自定义上下文中获取 Bean
        CustomBean customBean = customContext.getBean(CustomBean.class);
        System.out.println(customBean);

        // 关闭自定义上下文
        customContext.close();
    }
}
```

---

### 3. **使用 `@Import` 导入配置类**
如果你只需要在 `AnnotationConfigApplicationContext` 中加载特定的配置类，可以使用 `@Import` 注解，而不是扫描整个包。

#### 示例：
```java
@SpringBootApplication
public class ValueApplication {

    public static void main(String[] args) {
        // 启动 Spring Boot 应用
        ConfigurableApplicationContext springBootContext = SpringApplication.run(ValueApplication.class, args);

        // 手动初始化 AnnotationConfigApplicationContext，导入配置类
        AnnotationConfigApplicationContext customContext = new AnnotationConfigApplicationContext();
        customContext.register(CustomConfig.class); // 导入配置类
        customContext.refresh();

        // 从自定义上下文中获取 Bean
        CustomBean customBean = customContext.getBean(CustomBean.class);
        System.out.println(customBean);

        // 关闭自定义上下文
        customContext.close();
    }
}

@Configuration
class CustomConfig {
    @Bean
    public CustomBean customBean() {
        return new CustomBean();
    }
}
```

---

### 4. **避免重复初始化数据源等共享资源**
如果你在 `AnnotationConfigApplicationContext` 中使用了与 Spring Boot 相同的资源（如数据源），可能会导致冲突。可以通过以下方式避免：
- 在 `AnnotationConfigApplicationContext` 中禁用自动配置。
- 手动指定不同的资源。

#### 示例：
```java
@SpringBootApplication
public class ValueApplication {

    public static void main(String[] args) {
        // 启动 Spring Boot 应用
        ConfigurableApplicationContext springBootContext = SpringApplication.run(ValueApplication.class, args);

        // 手动初始化 AnnotationConfigApplicationContext，禁用自动配置
        AnnotationConfigApplicationContext customContext = new AnnotationConfigApplicationContext();
        customContext.getEnvironment().setActiveProfiles("custom"); // 使用不同的 Profile
        customContext.scan("com.example.custom"); // 扫描自定义包
        customContext.refresh();

        // 从自定义上下文中获取 Bean
        CustomBean customBean = customContext.getBean(CustomBean.class);
        System.out.println(customBean);

        // 关闭自定义上下文
        customContext.close();
    }
}
```

### 5. **使用 `@Conditional` 控制 Bean 的加载**
如果你需要根据条件加载不同的 Bean，可以使用 `@Conditional` 注解，避免 Bean 冲突。

#### 示例：
```java
@SpringBootApplication
public class ValueApplication {

    public static void main(String[] args) {
        // 启动 Spring Boot 应用
        ConfigurableApplicationContext springBootContext = SpringApplication.run(ValueApplication.class, args);

        // 手动初始化 AnnotationConfigApplicationContext
        AnnotationConfigApplicationContext customContext = new AnnotationConfigApplicationContext();
        customContext.register(CustomConfig.class); // 导入配置类
        customContext.refresh();

        // 从自定义上下文中获取 Bean
        CustomBean customBean = customContext.getBean(CustomBean.class);
        System.out.println(customBean);

        // 关闭自定义上下文
        customContext.close();
    }
}

@Configuration
class CustomConfig {
    @Bean
    @Conditional(OnCustomProfileCondition.class)
    public CustomBean customBean() {
        return new CustomBean();
    }
}

class OnCustomProfileCondition implements Condition {
    @Override
    public boolean matches(ConditionContext context, AnnotatedTypeMetadata metadata) {
        return context.getEnvironment().acceptsProfiles("custom");
    }
}
```

### 总结
通过以上方法，你可以在使用 `AnnotationConfigApplicationContext` 的同时避免与 Spring Boot 的默认上下文冲突。关键点包括：
1. 明确初始化范围，避免重复扫描相同的包。
2. 使用 `@Import` 导入特定配置类。
3. 避免重复初始化共享资源（如数据源）。
4. 使用 `@Conditional` 控制 Bean 的加载。
