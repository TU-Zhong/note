### **从这里开始：阶段一开发步骤**

**目标：** 运行两个客户端，A能看到B上线，B也能看到A上线，并且在A的窗口输入 `list` 能看到B的信息。

#### **第0步：磨刀不误砍柴工 - 熟悉调试器**

在写任何代码之前，**请务必先学会使用 CLion 的调试器**。这是你未来排查问题的最强武器，比到处加 `printf` 效率高无数倍。

1. **设置断点**：打开 `src/main.c` 文件，在 `user_list_init(&user_list);` 这一行的行号旁边用鼠标点一下，会出现一个红点，这就是断点。
2. **启动调试**：点击 CLion 右上角的**甲虫图标(🐞)** 而不是绿色的运行按钮。
3. **观察程序**：程序会运行并在你设置的断点处暂停。在下方的“Debugger”窗口中，你可以看到当前的变量值，并使用“Step Over (F8)”、“Step Into (F7)”等按钮逐行执行代码，观察程序状态的变化。

#### **第1步：实现“上线”广播 (`communicator.c` 和 `protocol_parser.c`)**

1. **实现协议封包**：
    [[build_entry_packet]]
    - 打开 `src/core/protocol_parser.c`。
    - 找到 `build_entry_packet` 函数。
    - 使用 `sprintf_s` 或 `snprintf` 函数，按照 IPMsg 协议格式将版本号、包编号、用户名、主机名等信息格式化成一个字符串，存入 `buffer` 中。
    - **示例**: `snprintf(buffer, buffer_size, "%d:%lu:%s:%s:%d:%s", ...);`
2. **实现广播发送**：
    [[communicator_broadcast_entry]]
    - 打开 `src/core/communicator.c`。
    - 找到 `communicator_broadcast_entry` 函数。
    - 在函数内部，调用 `build_entry_packet` 来生成要发送的消息。
    - 设置目标地址为广播地址（如 `255.255.255.255`）。
    - 使用 `sendto` 这个 Winsock 函数将消息字符串通过 UDP 套接字发送出去。

完成这一步后，你可以在 `main.c` 的 `communicator_broadcast_entry(&communicator);` 后面加一个断点，运行调试，确认你的程序确实发送了上线消息（虽然现在还没人能接收）。

#### **第2步：实现网络监听与消息处理 (`communicator.c`)**

这是核心中的核心。之前我们为 `communicator_init` 留了启动后台线程的坑，现在要填上。

1. **创建监听循环函数**:
    
    - 在 `communicator.c` 中，创建一个新的**内部函数**，例如 `static DWORD WINAPI communicator_listen_loop(LPVOID param)`。这个函数将作为后台线程的主体。
    - 在 `communicator_init` 函数的末尾，使用 `CreateThread` 函数来启动这个 `communicator_listen_loop` 线程。
2. **实现消息接收**:
    
    - 在 `communicator_listen_loop` 函数中，写一个 `while (comm->is_running)` 循环。
    - 循环内部，调用 `recvfrom` 函数。这个函数会**阻塞**，直到网络上收到一个 UDP 包。
    - `recvfrom` 会返回收到的数据、数据长度以及发送方的 IP 地址和端口。
3. **实现协议解包**:
    
    - 打开 `src/core/protocol_parser.c`，实现 `parse_message` 函数。
    - 使用 `sscanf_s` 或者 `strtok_s` 等字符串处理函数，解析从 `recvfrom` 收到的字符串，将各部分填充到 `parsed_message_t` 结构体中。
4. **实现消息分发**:
    
    - 在 `communicator_listen_loop` 中，收到消息并成功调用 `parse_message` 解析后，根据解析出的命令号（`parsed_message.command`）进行处理。
    - 建议再写一个内部函数 `static void process_packet(...)`，用 `switch` 语句来处理不同的命令。

#### **第3步：实现用户列表更新 (`user.c`)**

1. **处理上线消息**:
    
    - 在 `communicator.c` 的 `process_packet` 函数的 `switch` 语句中，添加 `case IPMSG_BR_ENTRY:` 和 `case IPMSG_ANSENTRY:`。
    - 在这个 case 里，从解析出的消息中提取出对方的用户名、主机名、IP和端口，填充到一个临时的 `user_node_t` 结构体中。
    - 调用 `user_list_add` 函数，将这个新用户添加到列表中。
2. **实现 `user_list_add`**:
    
    - 打开 `src/model/user.c`。
    - 在 `user_list_add` 函数中，实现**线程安全**的链表插入操作：
        - 使用 `EnterCriticalSection` 获取锁。
        - 检查用户是否已存在，如果存在则更新信息。
        - 如果不存在，使用 `malloc` 分配一个新的 `user_node_t` 内存空间。
        - 使用 `strcpy_s` 等安全函数复制用户信息到新节点。
        - 将新节点插入到链表的头部。
        - `count` 加一。
        - 使用 `LeaveCriticalSection` 释放锁。

#### **第4步：实现 `list` 命令 (`console.c` 和 `main.c`)**

1. **实现列表显示**:
    
    - 打开 `src/ui/console.c`，实现 `console_display_users` 函数。
    - 它应该先调用 `user_list_get_all` 将用户数据安全地复制到一个临时数组中。
    - 然后遍历这个数组，用 `printf` 格式化地打印出每个用户的索引号、用户名和主机信息。
2. **实现命令处理**:
    
    - 在 `console.c` 的 `console_run` 函数的循环中，使用 `strcmp` 比较用户输入是否为 "list"。
    - 如果是，就调用 `list_users_cb` 这个函数指针。
3. **连接回调函数**:
    
    - 打开 `src/main.c`。
    - 创建一个符合 `command_callback_t` 签名的函数，例如 `static void on_list_command(void* context, int argc, char* argv[])`。
    - 在这个函数里，获取 `console_t` 和 `user_list_t` 的实例，然后调用 `console_display_users`。
    - 在 `main` 函数中，调用 `console_register_callbacks` 时，把这个 `on_list_command` 函数的地址传进去。
