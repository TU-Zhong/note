### 功能一：程序启动与初始化

这是程序运行的第一步，负责准备好所有必要的资源。

1. **入口 (`main.c`)**
    - `main()`
        - `SetConsoleOutputCP(CP_UTF8)` / `SetConsoleCP(CP_UTF8)`: **【关键】** 设置控制台为UTF-8模式，解决中文乱码问题。
        - `WSAStartup()`: 初始化Windows Sockets网络库。
        - `user_list_init()`: 初始化用户列表，主要是创建线程锁。[[主要是创建线程锁]]
        - `console_init()`: 初始化控制台，创建打印锁。
        - `file_transfer_init()`: 初始化文件传输管理器。
        - `console_register_callbacks()`: **【核心】** 将 `main.c` 中实现的各个命令处理函数（如 `on_list_command`, `on_send_command` 等）注册到 `console` 模块中，建立UI和逻辑的连接。
        - `communicator_init()`: **【核心】** 初始化网络通信模块。
            - `socket()`: 创建一个UDP套接字用于后续所有UDP通信。
            - `WSAIoctl(SIO_UDP_CONNRESET)`: 设置UDP套接字属性，防止因ICMP错误导致程序异常。
            - `bind()`: 将套接字绑定到用户指定的端口上，准备接收数据。
            - `_beginthreadex()`: **创建并启动一个独立的后台“监听线程”**，该线程执行 `communicator_listen_loop` 函数，从此开始程序就能在后台接收网络消息了。
        - `communicator_broadcast_entry()`: **【关键动作】** 通过UDP广播自己的“上线”消息，通知网络中的其他用户。
            - -> `protocol_parser.c::build_entry_packet()`: 构建一个符合IPMSG协议的上线数据包。
            - -> `communicator.c::simulate_broadcast()`: 将数据包发送到预设的端口范围。
        - `console_run()`: **【进入主循环】** 启动控制台的输入循环，程序将在这里阻塞，等待用户输入命令。

### 功能二：接收网络数据（后台核心循环）

这个流程在 `communicator_listen_loop` 线程中持续进行，是所有网络事件的入口。

1. **网络监听 (`communicator.c`)**
    
    - `communicator_listen_loop()`
        - `recvfrom()`: **【阻塞】** 在此函数上阻塞，等待网络上传来UDP数据包。
        - (收到数据后) `parse_message()`: 将原始的字节数据包解析成程序可以理解的 `parsed_message_t` 结构体。
        - `process_packet()`: **【中央分发器】** 将解析后的消息交给这个函数进行处理。
2. **消息分发 (`communicator.c`)**
    
    - `process_packet()`
        - `switch (msg->command)`: 根据消息的命令类型（`IPMSG_BR_ENTRY`, `IPMSG_SENDMSG` 等），执行不同的逻辑分支。

### 功能三：用户上线与发现流程

这个流程展示了两个客户端如何互相认识。假设 **用户A** 已在线，**用户B** 刚刚启动。

1. **用户B** 启动，执行“功能一”中的 `communicator_broadcast_entry()`，向网络发送“上线包”。
2. **用户A** 的后台监听线程 (`communicator_listen_loop`) 中的 `recvfrom` 接收到这个“上线包”。
3. **用户A** 的程序调用 `process_packet()`，进入 `case IPMSG_BR_ENTRY:` 分支。
    - -> `user.c::user_list_add()`: 将**用户B**的信息添加到自己的用户列表中。
    - -> `communicator.c::communicator_send_ans_entry()`: 作为回应，向**用户B**单独发送一个“应答上线包”。
4. **用户B** 的后台监听线程接收到**用户A**发来的“应答上线包”。
5. **用户B** 的程序调用 `process_packet()`，进入 `case IPMSG_ANSENTRY:` 分支。
    - -> `user.c::user_list_add()`: 将**用户A**的信息添加到自己的用户列表中。

**最终结果**: A和B都知道对方在线，并保存在各自的用户列表中。

### 功能四：发送聊天消息流程 (`send` 命令)

1. **用户输入 (`console.c`)**
    
    - `console_run()`: `fgets` 获取用户输入的命令 `send 0 你好`。
    - `strncmp` 判断为 `send` 命令，解析出用户编号 `0` 和消息 `你好`。
    - 调用 `send_msg_cb` 回调函数。
2. **逻辑处理 (`main.c`)**
    
    - `on_send_command()` (即 `send_msg_cb`) 被调用。
        - `user.c::user_list_get_all()`: 获取当前所有用户的快照数组。
        - 根据用户编号 `0`，从数组中找到目标用户的IP地址和端口。
        - 调用 `communicator_send_message()` 发送消息。
3. **网络发送 (`communicator.c`)**
    
    - `communicator_send_message()`
        - -> `protocol_parser.c::build_sendmsg_packet()`: 将消息内容封装成符合协议的“发送消息包”。
        - -> `sendto()`: 通过UDP将数据包发送到目标IP和端口。
4. **对方接收**：对方客户端执行“功能二”，在 `process_packet` 中进入 `case IPMSG_SENDMSG:` 分支，最终调用 `console_print_message` 将消息显示出来。

### 功能五：发送文件流程 (`sendfile` 命令)

这是最复杂的流程，涉及UDP信令和TCP数据传输。

1. **用户输入 (`console.c`)**
    
    - `console_run()`: 获取用户输入的 `sendfile 0 C:\test.txt`。
    - `strncmp` 判断为 `sendfile` 命令，解析出用户编号 `0` 和文件路径 `C:\test.txt`。
    - 调用 `send_file_cb` 回调函数。
2. **发送方准备 (`main.c`)**
    
    - `on_send_file_command()` (即 `send_file_cb`) 被调用。
        - `GetFileAttributes()`: 检查文件是否存在。
        - `user_list_get_all()`: 找到目标用户。
        - **【关键】**`file_transfer.c::file_transfer_prepare_send()`: **启动TCP服务**。
            - -> `socket(TCP)`: 创建TCP套接字。
            - -> `bind()`: 绑定到随机端口。
            - -> `getsockname()`: 获取系统分配的TCP端口号（例如 `51234`）。
            - -> `listen()`: 开始监听。
            - -> `_beginthreadex()`: **创建并启动一个“文件发送线程”** (`sender_thread_proc`)，该线程会阻塞在 `accept()` 等待连接。
            - 函数返回获取到的TCP端口号 `51234`。
        - **【关键】**`communicator.c::communicator_send_file_request()`: **发送UDP信令**。
            - -> `protocol_parser.c::build_file_request_packet()`: 构建一个包含文件名、大小和**TCP端口号(51234)** 的“文件请求包”。
            - -> `sendto()`: 将这个UDP请求包发给目标用户。
3. **接收方响应 (对方客户端)**
    
    - `communicator.c::process_packet()`: 接收到UDP请求，进入 `case IPMSG_REQUEST_FILE_SEND:` 分支。
        - 解析出文件名、大小和发送方的TCP端口号 `51234`。
        - 打印“自动接收”提示。
        - **【关键】**`file_transfer.c::file_transfer_start_receive()`: **启动TCP客户端**。
            - -> `_beginthreadex()`: **创建并启动一个“文件接收线程”** (`receiver_thread_proc`)。
4. **TCP连接与数据传输**
    
    - **接收方**的 `receiver_thread_proc` 线程中：
        - `socket(TCP)` -> `connect()`: 连接到发送方的IP和端口 `51234`。
    - **发送方**的 `sender_thread_proc` 线程中：
        - `accept()` 不再阻塞，返回一个新的TCP套接字用于通信。
    - **数据流动**:
        - 发送方线程循环 `fread()` 本地文件 -> `send()` TCP数据。
        - 接收方线程循环 `recv()` TCP数据 -> `fwrite()` 写入新文件。
5. **结束**: 传输完成后，双方各自关闭TCP连接和线程。

### 功能六：程序退出与清理

1. **用户输入 (`console.c`)**
    
    - `console_run()`: `fgets` 获取 `exit` 命令。
    - 调用 `exit_app_cb` 回调。
2. **逻辑处理 (`main.c`)**
    
    - `on_exit_command()` 被调用，将 `console->should_exit` 标志位设为1。
    - `console_run()` 的 `while` 循环条件不再满足，函数返回，`main` 函数继续向下执行。
3. **清理 (`main.c`)**
    
    - `communicator_broadcast_exit()`: 广播“下线通知”。
    - `communicator_destroy()`: 销毁通信模块。
        - 将 `is_running` 设为0。
        - `closesocket()`: 关闭UDP套接字，这将使后台监听线程中的 `recvfrom` 立刻返回一个错误，从而解除阻塞，让线程可以正常退出。
        - `WaitForSingleObject()`: 等待监听线程完全终止。
    - `file_transfer_destroy()`, `user_list_destroy()`, `console_destroy()`: 依次销毁其他模块，释放资源（主要是线程锁）。
    - `WSACleanup()`: 清理网络库。
    - `main` 函数结束，程序退出。
