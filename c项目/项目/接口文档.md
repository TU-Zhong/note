## **API 接口文档**

### **1. `model/user.h` (用户及列表模块)**

这个模块负责定义用户数据结构，并管理一个线程安全的在线用户链表。

#### `int user_list_init(user_list_t* list);`

- **功能描述**: 初始化一个用户列表管理器。这是在使用列表前的第一个必须调用的函数。
- **参数**:
    - `user_list_t* list`: 指向一个 `user_list_t` 结构体的指针。
- **返回值**:
    - `0`: 初始化成功。
    - `-1`: 初始化失败（例如传入的指针为空）。

#### `void user_list_destroy(user_list_t* list);`

- **功能描述**: 销毁一个用户列表，释放所有已分配的内存和系统资源（例如锁）。
- **参数**:
    - `user_list_t* list`: 指向要销毁的 `user_list_t` 结构体的指针。
- **返回值**: 无。

#### `void user_list_add(user_list_t* list, const user_node_t* user_data);`

- **功能描述**:向用户列表中线程安全地添加一个新用户。如果用户已存在，则会更新其信息。
- **参数**:
    - `user_list_t* list`: 目标用户列表。
    - `const user_node_t* user_data`: 指向包含新用户信息的节点的指针。函数内部会复制此数据，而不是直接使用该指针。
- **返回值**: 无。

#### `void user_list_remove(user_list_t* list, const char* ip_address, uint16_t port);`

- **功能描述**: 从用户列表中线程安全地移除一个指定的用户。
- **参数**:
    - `user_list_t* list`: 目标用户列表。
    - `const char* ip_address`: 要移除用户的 IP 地址。
    - `uint16_t port`: 要移除用户的端口号。
- **返回值**: 无。

#### `user_node_t* user_list_find_by_index(user_list_t* list, int index);`

- **功能描述**: 根据索引（从0开始）查找用户。主要用于处理 `send <index> <file>` 这样的命令。**注意**：返回的指针指向内部数据，应尽快使用且不应由调用者释放。
- **参数**:
    - `user_list_t* list`: 目标用户列表。
    - `int index`: 用户的索引号。
- **返回值**:
    - `user_node_t*`: 指向找到的用户节点的指针。
    - `NULL`: 索引越界或未找到。

#### `int user_list_get_all(user_list_t* list, user_node_t dest_array[], int max_count);`

- **功能描述**: 线程安全地复制当前所有在线用户的信息到一个外部数组中。这用于UI显示，避免长时间锁定列表。
- **参数**:
    - `user_list_t* list`: 源用户列表。
    - `user_node_t dest_array[]`: 用于存放用户信息的外部数组。
    - `int max_count`: `dest_array` 的最大容量。
- **返回值**:
    - `int`: 实际复制到数组中的用户数量。

### **2. `ui/console.h` (控制台UI模块)**

此模块负责所有与命令行的输入输出交互。

#### `int console_init(console_t* console);`

- **功能描述**: 初始化控制台UI管理器，特别是其内部的打印锁。
- **参数**:
    - `console_t* console`: 指向一个 `console_t` 结构体的指针。
- **返回值**:
    - `0`: 初始化成功。
    - `-1`: 初始化失败。

#### `void console_destroy(console_t* console);`

- **功能描述**: 销毁控制台UI管理器，释放其占用的资源（主要是锁）。
- **参数**:
    - `console_t* console`: 指向要销毁的 `console_t` 结构体的指针。
- **返回值**: 无。

#### `void console_register_callbacks(console_t* console, void* context, command_callback_t list_cb, command_callback_t send_cb, command_callback_t exit_cb);`

- **功能描述**: 注册命令回调函数。这是将UI层与应用逻辑层“连接”起来的关键函数。
- **参数**:
    - `console_t* console`: 目标控制台。
    - `void* context`: 一个上下文指针，它会被传递给回调函数，通常指向包含所有模块的主程序结构体。
    - `command_callback_t list_cb`: 当用户输入 `list` 命令时要调用的函数指针。
    - `command_callback_t send_cb`: 当用户输入 `send` 命令时要调用的函数指针。
    - `command_callback_t exit_cb`: 当用户输入 `exit` 命令时要调用的函数指针。
- **返回值**: 无。

#### `void console_run(console_t* console);`

- **功能描述**: 启动控制台的阻塞式主循环，等待用户输入并分派命令给已注册的回调函数。
- **参数**:
    - `console_t* console`: 指向要运行的 `console_t` 结构体。
- **返回值**: 无。

#### `void console_print_message(console_t* console, const char* message);`

- **功能描述**: 向控制台线程安全地打印一条普通消息。
- **参数**:
    - `console_t* console`: 目标控制台。
    - `const char* message`: 要打印的字符串。
- **返回值**: 无。

#### `void console_display_users(console_t* console, user_list_t* user_list);`

- **功能描述**: 以格式化的方式，线程安全地打印整个用户列表。
- **参数**:
    - `console_t* console`: 目标控制台。
    - `user_list_t* user_list`: 包含要显示的用户数据的列表。
- **返回值**: 无。

### **3. `core/communicator.h` (UDP通信模块)**

此模块是网络核心，负责收发所有基于UDP的IPMsg协议包。

#### `int communicator_init(communicator_t* comm, user_list_t* user_list, struct Console* console, const char* user_name);`

- **功能描述**: 初始化通信器，包括创建和绑定UDP套接字，存储用户信息，并启动一个后台线程来监听网络消息。
- **参数**:
    - `communicator_t* comm`: 指向要初始化的 `communicator_t` 结构体。
    - `user_list_t* user_list`: 指向用户列表模块的指针，用于在收到消息时更新列表。
    - `struct Console* console`: 指向控制台模块的指针，用于打印网络事件日志。
    - `const char* user_name`: 当前用户的昵称。
- **返回值**:
    - `0`: 初始化成功。
    - `-1`: 初始化失败（如套接字创建失败）。

#### `void communicator_destroy(communicator_t* comm);`

- **功能描述**: 关闭通信器，包括停止监听线程、关闭套接字。
- **参数**:
    - `communicator_t* comm`: 指向要销毁的 `communicator_t` 结构体。
- **返回值**: 无。

#### `void communicator_broadcast_entry(communicator_t* comm);`

- **功能描述**: 向局域网广播一条“用户上线” (`IPMSG_BR_ENTRY`) 消息。
- **参数**:
    - `communicator_t* comm`: 通信器实例。
- **返回值**: 无。

#### `void communicator_broadcast_exit(communicator_t* comm);`

- **功能描述**: 向局域网广播一条“用户下线” (`IPMSG_BR_EXIT`) 消息。
- **参数**:
    - `communicator_t* comm`: 通信器实例。
- **返回值**: 无。

#### `void communicator_send_ans_entry(communicator_t* comm, const char* dest_ip, uint16_t dest_port);`

- **功能描述**: 向指定IP和端口单播发送一条“响应上线” (`IPMSG_ANSENTRY`) 消息。
- **参数**:
    - `communicator_t* comm`: 通信器实例。
    - `const char* dest_ip`: 目标主机的IP地址。
    - `uint16_t dest_port`: 目标主机的端口号。
- **返回值**: 无。

### **4. `core/protocol_parser.h` (IPMsg协议解析模块)**

这是一个工具模块，提供纯函数来构造和解析IPMsg协议字符串。

#### `int build_entry_packet(char* buffer, size_t buffer_size, uint64_t packet_no, const char* user_name, const char* host_name);`

- **功能描述**: 在提供的 `buffer` 中构造一个符合IPMsg格式的“上线”消息字符串。
- **参数**:
    - `char* buffer`: 用于存放结果字符串的缓冲区。
    - `size_t buffer_size`: 缓冲区的最大大小。
    - `uint64_t packet_no`: 数据包的唯一编号。
    - `const char* user_name`: 用户昵称。
    - `const char* host_name`: 主机名。
- **返回值**:
    - `> 0`: 成功，返回构造的字符串的长度。
    - `< 0`: 失败（如缓冲区太小）。

_(其他 `build_..._packet` 函数与此类似，只是构造的消息类型不同)_

#### `int parse_message(const char* packet, size_t packet_len, parsed_message_t* out_msg);`

- **功能描述**: 解析收到的原始网络数据包字符串，并将各部分填充到 `parsed_message_t` 结构体中。
- **参数**:
    - `const char* packet`: 指向收到的原始数据的指针。
    - `size_t packet_len`: 原始数据的长度。
    - `parsed_message_t* out_msg`: 指向一个结构体，用于存放解析结果。
- **返回值**:
    - `0`: 解析成功。
    - `-1`: 解析失败（如格式不正确）。

### **5. `core/file_transfer.h` (TCP文件传输模块)**

此模块负责处理基于TCP的、可靠的文件传输。

#### `int file_transfer_init(file_transfer_manager_t* manager);`

- **功能描述**: 初始化文件传输管理器。
- **参数**:
    - `file_transfer_manager_t* manager`: 指向管理器的指针。
- **返回值**:
    - `0`: 成功。

#### `void file_transfer_destroy(file_transfer_manager_t* manager);`

- **功能描述**: 销毁文件传输管理器。
- **参数**:
    - `file_transfer_manager_t* manager`: 指向管理器的指针。
- **返回值**: 无。

#### `int file_transfer_prepare_send(file_transfer_manager_t* manager, const char* file_path, uint16_t* out_tcp_port);`

- **功能描述**: **发送方**调用此函数。它会创建一个TCP监听套接字在一个可用的端口上，并启动一个线程等待接收方连接。这是一个非阻塞函数。
- **参数**:
    - `file_transfer_manager_t* manager`: 管理器实例。
    - `const char* file_path`: 要发送的文件的完整路径。
    - `uint16_t* out_tcp_port`: 一个出参，函数会把监听的TCP端口号写入此指针指向的地址。
- **返回值**:
    - `0`: 准备成功。
    - `-1`: 准备失败。

#### `int file_transfer_start_receive(file_transfer_manager_t* manager, const char* target_ip, uint16_t target_port, const file_info_t* file_info);`

- **功能描述**: **接收方**调用此函数。它会尝试连接到发送方的IP和端口，如果成功，则启动一个新线程来接收文件数据并写入本地。这是一个非阻塞函数。
- **参数**:
    - `file_transfer_manager_t* manager`: 管理器实例。
    - `const char* target_ip`: 发送方的IP地址。
    - `uint16_t target_port`: 发送方正在监听的TCP端口。
    - `const file_info_t* file_info`: 包含要接收的文件名、大小等信息的结构体。
- **返回值**:
    - `0`: 成功开始接收。
    - `-1`: 连接或启动失败。
