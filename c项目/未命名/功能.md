### 程序功能概述

您的这个 `ipmsg_win.exe` 程序主要有以下三大核心功能：

1. **自动用户发现 (UDP广播)**：启动后会自动向局域网广播自己的存在，并能发现其他同样在运行此程序的在线用户。
2. **即时消息 (UDP单播)**：可以向局域网内的指定用户发送短消息。
3. **文件传输 (TCP直连)**：可以向指定用户发起一个直接的 TCP 连接，来传输任意大小的文件。

### 一、 启动与环境准备

要完整测试所有功能，您至少需要**两个程序实例**来互相通信。

1. **找到程序**： 您的程序 `ipmsg_win.exe` 位于项目的 `cmake-build-debug\bin\` 目录下。
    
2. **准备环境**：
    
    - 打开 **两个** 独立的命令行终端（CMD 或 PowerShell）。
    - 在**每一个**终端中，都使用 `cd` 命令进入 `ipmsg_win.exe` 所在的目录。例如：
        
        Shell
        
        ```
        cd /d D:\coding learning\c\pro\test2\cmake-build-debug\bin
        ```
        
3. **启动程序**：
    
    - 在**第一个终端**里，以“小明”的身份启动程序：
        
        Shell
        
        ```
        .\ipmsg_win.exe 小明
        ```
        
    - 在**第二个终端**里，以“小红”的身份启动程序：
        
        Shell
        
        ```
        .\ipmsg_win.exe 小红
        ```
        
    - 当第二个程序启动后，您会看到两个窗口几乎同时打印出对方上线的系统提示，证明它们已经互相发现了。

### 二、 命令使用详解

程序启动后，您会在一个 `>` 提示符后输入命令来执行操作。

#### 1. `help` - 获取帮助

- **作用**：显示所有可用命令的列表。
- **操作**：在提示符后输入 `help` 并回车。

#### 2. `list` - 查看在线用户

- **作用**：列出当前局域网内所有已发现的用户。
- **操作**：输入 `list` 并回车。
- **预期结果**：您会看到一个列表，每个用户前都有一个 **ID号**，这个ID在发送消息和文件时至关重要。
    
    ```
    --- Online Users ---
    [1] 小红@电脑名称 (192.168.1.101)
    --------------------
    ```
    

#### 3. `refresh` - 刷新在线列表

- **作用**：重新发送一次上线广播，让网络中的其他人能再次发现您。
- **操作**：输入 `refresh` 并回车。当有新用户上线但您的列表里没有时，可以使用此命令。

#### 4. `send` - 发送文本消息

- **作用**：给指定ID的用户发送一条文本消息。
- **格式**：`send <用户ID> <消息内容>`
- **操作示例**：在“小明”的窗口，想给ID为 `1` 的“小红”发消息。
    
    Shell
    
    ```
    send 1 你好，在吗？我准备给你发个文件。
    ```
    
- **预期结果**：
    - 小明的窗口会提示 `[System] Message sent to 小红.`
    - 小红的窗口会显示 `<Message from 小明@...>: 你好，在吗？我准备给你发个文件。`

#### 5. `file` - 发送文件

- **作用**：给指定ID的用户发送一个文件。这是程序最核心的功能。
- **格式**：`file <用户ID> <文件的完整路径>`
- **重要提示**：如果您的文件路径**包含空格**，必须用英文双引号 `"` 将整个路径括起来。
- **操作示例**：小明想把桌面上一个名为 "项目报告.docx" 的文件发给ID为 `1` 的小红。
    
    Shell
    
    ```
    # 路径不含空格
    file 1 C:\Users\XiaoMing\Desktop\report.zip
    
    # 路径含有空格，必须用引号
    file 1 "C:\My Documents\项目报告.docx"
    ```
    
- **预期结果（一个完整的交互流程）**：
    1. **发送方（小明）**：输入命令后，会看到 `[File] Sending file data for '项目报告.docx'...` 的提示。
    2. **接收方（小红）**：屏幕上会立刻出现一个提示，并等待她确认： `[File] Incoming file: 项目报告.docx. Accept? (y/n):`
    3. **接收方（小红）**：在她的窗口中输入 `y`，然后按回车。
    4. **双方**：文件开始传输。传输成功后，发送方会提示 `[File] Sent successfully.`，接收方会提示 `[File] '项目报告.docx' received...`。
    5. **文件位置**：小红可以在 `ipmsg_win.exe` 所在的目录（即 `bin` 目录）下找到接收到的 `项目报告.docx` 文件。

#### 6. `quit` - 退出程序

- **作用**：安全退出程序。
- **操作**：输入 `quit` 并回车。程序会发送一条下线广播，通知网络中的其他人自己已离开，然后关闭。

### 三、 常见问题 (FAQ)

- **问：为什么我看不到任何在线用户？**
    
    - **答：99%的可能性是 Windows 防火墙。** 程序第一次运行时，Windows 会弹出安全警报，请务必勾选所有网络类型（特别是“专用网络”），并点击**“允许访问”**。如果错过，请手动去控制面板的防火墙设置里允许 `ipmsg_win.exe` 通信。
- **问：接收的文件保存在哪里？**
    
    - **答：** 保存在您运行 `ipmsg_win.exe` 的那个目录，也就是 `cmake-build-debug\bin\` 文件夹里。
