### **跨域属性（`crossorigin`）简单解释**

#### **1. 什么是“跨域”？**
想象你开了一家餐厅，你的厨房（网站）里有很多调料（资源）。有一天，你想用隔壁餐厅（另一个网站）的调料（资源），这就叫“跨域”。

#### **2. 为什么要用 `crossorigin`？**
默认情况下，浏览器不允许你随便用隔壁的调料，因为怕不安全。`crossorigin` 就是用来告诉浏览器：“我是正规借调料的，放心！”

#### **3. `crossorigin` 怎么用？**
它有两个值：
- **`anonymous`** → **“借调料，但别告诉他们是我借的。”**
  - 例子：`<link crossorigin="anonymous" href="https://隔壁餐厅.com/调料">`
  - 浏览器会去隔壁借调料，但不会带你的身份信息（比如 cookies）。
- **`use-credentials`** → **“借调料，告诉他们是我借的。”**
  - 例子：`<link crossorigin="use-credentials" href="https://隔壁餐厅.com/调料">`
  - 浏览器会带你的身份信息（比如 cookies）去借调料。

#### **4. 什么时候用 `crossorigin`？**
当你用隔壁的资源（比如字体、图片、脚本）时，如果隔壁餐厅要求你证明身份，就需要加这个属性。

#### **5. 例子**
```html
<link href="https://fonts.googleapis.com/css?family=Roboto" rel="stylesheet" crossorigin="anonymous">
```
- 这里我们从 Google Fonts（隔壁餐厅）借了一个字体（调料），用了 `crossorigin="anonymous"` 告诉浏览器：“去借吧，别带我的身份信息。”

---

### **一句话总结**
`crossorigin` 是用来告诉浏览器：“我要用隔壁的资源，要不要带身份信息？” 就像借调料时，可以选择匿名还是实名！
