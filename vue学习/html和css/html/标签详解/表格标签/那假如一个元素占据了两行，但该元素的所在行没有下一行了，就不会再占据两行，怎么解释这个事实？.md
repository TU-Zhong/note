

用「叠砖块」的比喻来理解：

---

### **原理归纳**
- **`rowspan` 是「向下占位置」**：如果下方没有足够的砖块（行），浏览器会自动缩减合并的行数，只占实际存在的行。

---

### **举个栗子 🌰**
#### **错误写法（只有一行，却想占两行）**：
```html
<table border="1">
  <tr>
    <td rowspan="2">我想占两行</td> <!-- 下方没有第二行！ -->
    <td>B</td>
  </tr>
</table>
```

**实际效果**：
```
| 我想占两行 | B |
```
- **浏览器觉得**：下方没有第二行，所以 `rowspan="2"` 无效，自动变成 `rowspan="1"`。
- **看起来**：这个单元格只占了一行。

---

#### **正确写法（确保行数足够）**：
```html
<table border="1">
  <tr>
    <td rowspan="2">我占两行</td> <!-- 下方有第二行 -->
    <td>B</td>
  </tr>
  <tr>
    <td>C</td> <!-- 第二行 -->
  </tr>
</table>
```

**效果**：
```
| 我占两行 | B | 
|          | C |
```
- **第二行的第一个位置被合并** → 正确占两行。

---

### **总结口诀**
- **`rowspan` 是向下要位置**，下方没行就失效！
- **砖头不够别硬垒**，浏览器会自动修正！

---

### **类比生活中的场景**
假设你有一摞书，想用一本书压住下面两本书：
- **如果只有一本**：你只能压住自己这一本，压不到下面的（因为下面没有书）。
- **如果有两本**：第一本可以压住自己和第二本。

---

### **实际代码测试**
```html
<table border="1">
  <tr>
    <td rowspan="3">我想占三行</td>
    <td>A</td>
  </tr>
  <tr>
    <td>B</td>
  </tr>
</table>
```
**效果**：
```
| 我想占三行 | A |
|            | B |
```
- **浏览器发现只有两行** → `rowspan="3"` 变成 `rowspan="2"`。
