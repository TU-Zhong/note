

用 **“排队占位”** 的比喻来解释这个现象：

---

### **原理分析**
当所有行的单元格都设置 `colspan="3"` 时，浏览器会按以下逻辑处理：
1. **每行的单元格都声明要占 3 列** → 浏览器认为整个表格需要 **每行至少有 3 列**。
2. **但每行只有一个单元格** → 这个单元格强制占满 3 列，导致所有行的实际列数被统一为 **3 列**。
3. **最终效果**：每行只有一个单元格，占满该行 3 列 → 看起来像一个格子铺满整行。

---

### **举个栗子 🌰**
```html
<table border="1">
  <tr>
    <td colspan="3">A</td> <!-- 占满 3 列 -->
  </tr>
  <tr>
    <td colspan="3">B</td> <!-- 占满 3 列 -->
  </tr>
</table>
```
**效果：**
```
|       A       | <!-- 占满一行 -->
|       B       | <!-- 占满一行 -->
```
- 每行只有一个单元格，但每格占满 3 列 → 视觉上显示为单列（因为浏览器自动将总列数统一为 3 列）。

---

### **对比正常情况**
如果表格的列数不一致，浏览器会以 **最大列数** 为准，并自动补全缺失的列：
```html
<table border="1">
  <tr>
    <td colspan="3">A</td> <!-- 占 3 列 -->
  </tr>
  <tr>
    <td>B</td> <!-- 占 1 列 -->
    <td>C</td> <!-- 占 1 列 -->
  </tr>
</table>
```
**效果：**
```
|       A       | <!-- 3 列 -->
| B | C |     | <!-- 第三列自动补空 -->
```

---

### **结论**
- **所有行统一按最大列数计算**：如果每行都声明占 3 列但实际只有一个单元格 → 浏览器强制每行只有一列（占满 3 列的宽度）。
- **视觉表现**：像单列表格，但实际宽度被分为 3 列（只不过其他列为空）。

---

### **生活比喻**
想象排队买票：
- **规则**：每人必须占 3 个位置（`colspan="3"`）。
- **结果**：每行只能站一个人（占满 3 个位置），但队伍总宽度被强制设为 3 个位置 → 看起来像一列队伍，实际宽度是 3 倍！

---

### **代码验证**
```html
<table border="1" style="width: 100%;">
  <tr>
    <td colspan="3">A</td>
  </tr>
  <tr>
    <td colspan="3">B</td>
  </tr>
</table>
```
**效果**：两个大格子各自占满整行（宽度相同，但实际按 3 列分配）。
