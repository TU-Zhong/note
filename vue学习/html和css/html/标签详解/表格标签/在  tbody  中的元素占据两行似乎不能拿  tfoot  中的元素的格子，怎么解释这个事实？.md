

用「分房间的积木」来比喻解释：

---

### **核心原理**
**`<tbody>` 和 `<tfoot>` 是不同的“房间”**，合并行（`rowspan`）只能在 **同一个房间内** 延伸，**不能跨房间占用其他区域的格子**。

---

### **举个栗子 🌰**
#### **错误写法（试图跨房间合并）**
```html
<table>
  <tbody>
    <tr>
      <td rowspan="2">我想跨到 tfoot</td> <!-- 试图合并到 tfoot -->
      <td>A</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <td>B</td>
    </tr>
  </tfoot>
</table>
```

**实际效果：**

| 我想跨到 tfoot | A |
|----------------|---|
| B              |   |
- **浏览器行为**：  
  `rowspan="2"` 只会尝试在 `tbody` 内合并，但由于 `tbody` 只有一行，实际合并无效 → 单元格高度不变。  
  `tfoot` 是完全独立的区域，不会被占用。

---

#### **正确写法（在同一个房间内合并）**
```html
<table>
  <tbody>
    <tr>
      <td rowspan="2">我在 tbody 内合并</td> <!-- 合并 tbody 内的两行 -->
      <td>A</td>
    </tr>
    <tr>
      <td>B</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <td>C</td>
    </tr>
  </tfoot>
</table>
```

**效果：**

| 我在 tbody 内合并 | A |
|--------------------|---|
|                    | B |
| C                  |   |

---

### **生活比喻**
想象一栋楼：
- **`<tbody>` 是二楼**  
- **`<tfoot>` 是一楼**  
- `rowspan` 是 **在二楼往下挂一条绳子**，绳子最多只能挂到二楼的地板，**无法穿透到一楼**！

---

### **总结**
- `rowspan` 只能在 **同一个行组（tbody/thead/tfoot）** 内合并。
- `<tfoot>` 是独立的“房间”，`<tbody>` 的元素无法跨组占用它的位置。
- 浏览器严格遵守表格结构语义，不同区块的格子相互隔离。
