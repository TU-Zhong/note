

以下是 HTML `<input>` 标签的常用属性及其含义的详细列表，按功能分类整理：

---

### 一、核心属性
| 属性         | 值示例          | 说明                                                                 |
|--------------|----------------|--------------------------------------------------------------------|
| `type`       | `text`, `email`, `number` | **输入类型**：定义输入框的行为和样式（如文本、密码、日期等）。必填属性。                |
| `name`       | `username`     | **字段名称**：表单提交时，数据以 `name=value` 的形式发送到后端。必填属性。             |
| `value`      | `默认值`        | **初始值/用户输入值**：定义输入框的默认值，或用户输入后的值。                          |
| `placeholder`| `请输入邮箱`     | **提示文本**：在输入框为空时显示的灰色提示文本（不影响实际值）。                        |

---

### 二、输入限制与验证
| 属性                      | 值示例           | 说明                                                |
| ----------------------- | ------------- | ------------------------------------------------- |
| `required`              | -             | **必填字段**：提交前必须填写此字段（HTML5 验证）。                    |
| `min`/`max`             | `0`, `100`    | **数值/日期范围**：限制 `number`/`date` 等类型的输入范围。          |
| `minlength`/`maxlength` | `5`, `20`     | **文本长度限制**：限制文本输入的最小/最大字符数（如 `text`, `password`）。 |
| `pattern`               | `[A-Za-z]{3}` | **正则表达式验证**：用正则验证输入格式（如邮箱、电话号码）。                  |
| `step`                  | `2`, `0.5`    | **数值步长**：设置 `number`/`range` 类型的增减步长。             |

### 三、行为控制
| 属性          | 值示例      | 说明                                                                 |
|---------------|------------|--------------------------------------------------------------------|
| `readonly`    | -          | **只读**：输入框内容不可修改（但会被提交）。                               |
| `disabled`    | -          | **禁用**：输入框不可用且不会被提交。                                      |
| `autocomplete`| `on`, `off`| **自动填充**：控制浏览器是否记录并自动填充输入历史（如 `off` 用于敏感字段）。   |
| `autofocus`   | -          | **自动聚焦**：页面加载时自动将光标定位到该输入框。                           |

---

### 四、HTML5 新增属性
| 属性             | 适用类型           | 说明                                                                 |
|------------------|------------------|--------------------------------------------------------------------|
| `multiple`       | `file`, `email`  | **多选**：允许上传多个文件（`type="file"`）或输入多个邮箱（`type="email"`）。 |
| `form`           | 任意             | **关联表单**：指定输入框所属的表单（即使不在 `<form>` 标签内）。                |
| `list`           | `text`, `search` | **输入建议**：关联 `<datalist>` 元素，提供预定义输入选项。                    |
| `formaction`     | `submit`         | **覆盖表单提交地址**：仅对 `type="submit"` 有效。                        |

---

### 五、特殊类型属性
| 属性          | 适用类型       | 说明                                                                 |
|---------------|--------------|--------------------------------------------------------------------|
| `accept`      | `file`       | **文件类型限制**：指定上传文件的类型（如 `image/*`, `.pdf`）。               |
| `checked`     | `radio`, `checkbox` | **默认选中**：设置单选或复选框初始选中状态。                              |
| `src`         | `image`      | **图片地址**：定义图片按钮的图片来源（`type="image"`）。                     |

---

### 示例代码
```html
<!-- 文本输入 + 验证 -->
<input type="email" name="user_email" placeholder="邮箱" required pattern="[a-z0-9._%+-]+@[a-z0-9.-]+\.[a-z]{2,}$">

<!-- 数字范围 -->
<input type="number" name="age" min="18" max="100" step="1" value="20">

<!-- 文件上传 -->
<input type="file" name="doc" accept=".pdf,.docx" multiple>

<!-- 关联 datalist -->
<input type="text" list="countries" name="country">
<datalist id="countries">
  <option value="中国">
  <option value="美国">
</datalist>
```

---

### 关键总结
1. **`type` 是核心**：决定输入框行为和验证规则（如 `email` 自动验证邮箱格式）。
2. **`name` 不可少**：表单提交时用于标识字段。
3. **按需使用 HTML5 验证**：如 `required`、`pattern` 可减少 JavaScript 依赖。
4. **注意兼容性**：部分属性（如 `datalist`）在旧浏览器中可能不支持。
