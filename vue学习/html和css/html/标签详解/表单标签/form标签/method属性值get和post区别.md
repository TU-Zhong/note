---

### **GET 和 POST 的区别**  
它们就像寄信的两种方式：

1. **GET（读作“盖特”）**  
   - **像明信片**：你写的内容直接写在信封表面（URL地址栏里），所有人都能看到。  
   - **适合**：搜东西（比如百度搜索）、分享链接（比如商品页面）。  
   - **缺点**：不能写太多字（网址长度有限），不安全（密码不能用这个！）。

   ```html
   <!-- 例子：搜索“披萨” -->
   <form action="/search" method="GET">
     <input type="text" name="keyword">
     <button>搜索</button>
   </form>
   ```
   提交后网址变成：`/search?keyword=披萨`

---

1. **POST（读作“破斯特”）**  
   - **像密封的信封**：内容藏在信封里（不会显示在网址上）。  
   - **适合**：登录账号、提交银行卡号、上传文件。  
   - **优点**：能发很长的内容（比如一篇作文），相对安全（但也要加密！）。

   ```html
   <!-- 例子：登录 -->
   <form action="/login" method="POST">
     <input type="text" name="username">
     <input type="password" name="password">
     <button>登录</button>
   </form>
   ```
   提交后网址不变，看不到密码！

---

### **总结口诀**  
- **想公开 → 用 GET**（比如分享搜索结果链接）。  
- **想隐藏 → 用 POST**（比如密码、大文件）。  
- **重要的事情（钱、密码）一定要用 POST！**  

---

### **举个生活例子** 🌰  
- **GET**：点外卖时告诉朋友：“帮我看看**美团外卖?keyword=牛肉面**”（对方直接打开链接就能看到你要吃啥）。  
- **POST**：偷偷告诉朋友：“微信发你地址和密码，帮我点个牛肉面”（别人看不到你的地址和密码）。  

---

**终极结论：**  
- 默认用 POST，除非你想让数据出现在网址里！
