

父级监听器的触发次数**取决于事件的注册方式**和**事件流的阶段**。以下是详细分析：

---

### 🌟 **核心结论**  
**父级监听器可能触发一次、两次，甚至不触发，具体取决于：**  
1. 监听器的注册阶段（冒泡阶段 vs 捕获阶段）  
2. 是否在子元素中阻止事件传播（如 `e.stopPropagation()`）  
3. 是否通过 `once: true` 设置单次触发  

---

### 🔍 分场景解释（以三层结构为例）
假设有以下 HTML 结构：  
```html
<div class="爷爷">
  <div class="爸爸">
    <div class="儿子"></div>
  </div>
</div>
```

---

#### 场景 1：**父级仅监听冒泡阶段（默认行为）**
```javascript
// 爷爷监听冒泡阶段（第三个参数为 false 或默认不写）
document.querySelector('.爷爷').addEventListener('click', () => {
  console.log('爷爷冒泡阶段触发');
});

// 儿子监听器
document.querySelector('.儿子').addEventListener('click', () => {
  console.log('儿子触发');
});
```

**点击 `.儿子` 时的输出：**  
```
儿子触发 → 爷爷冒泡阶段触发
```  
**父级触发次数：1 次**

---

#### 场景 2：**父级同时监听捕获和冒泡阶段**
```javascript
// 爷爷监听捕获阶段（第三个参数为 true）
document.querySelector('.爷爷').addEventListener('click', () => {
  console.log('爷爷捕获阶段触发');
}, true);

// 爷爷同时监听冒泡阶段
document.querySelector('.爷爷').addEventListener('click', () => {
  console.log('爷爷冒泡阶段触发');
});
```

**点击 `.儿子` 时的输出：**  
```
爷爷捕获阶段触发 → 儿子触发 → 爷爷冒泡阶段触发
```  
**父级触发次数：2 次（捕获 + 冒泡）**

---

#### 场景 3：**子元素阻止事件传播**
```javascript
// 爷爷监听冒泡阶段
document.querySelector('.爷爷').addEventListener('click', () => {
  console.log('爷爷触发');
});

// 儿子阻止事件冒泡
document.querySelector('.儿子').addEventListener('click', (e) => {
  e.stopPropagation(); // 阻止事件继续传播
  console.log('儿子触发（阻止冒泡）');
});
```

**点击 `.儿子` 时的输出：**  
```
儿子触发（阻止冒泡）  
```  
**父级触发次数：0 次（冒泡被阻止）**

---

### ⚙️ 关键控制方法
#### 方法 1：**强制单次触发**  
通过 `{ once: true }` 让监听器只触发一次（无论事件流阶段）：  
```javascript
document.querySelector('.爷爷').addEventListener('click', () => {
  console.log('爷爷触发（仅一次）');
}, { once: true }); // 触发后自动移除监听器
```

#### 方法 2：**精准控制触发阶段**  
- 若只需在捕获阶段触发父级逻辑，使用 `true`  
- 若只需在冒泡阶段触发，使用 `false`（或默认不写）  

---

### 📊 总结表格
| 场景                        | 父级触发次数 | 原因说明                     |
|-----------------------------|--------------|------------------------------|
| 仅监听冒泡阶段（默认）      | 1 次         | 事件冒泡到父级时触发         |
| 同时监听捕获和冒泡          | 2 次         | 两个阶段各触发一次           |
| 子元素阻止传播 (`stopPropagation`) | 0 次         | 事件流被中断                 |
| 使用 `{ once: true }`        | 1 次（永久） | 监听器触发后自动移除         |

---

### 💡 实战建议  
- **事件委托**：如果希望父级监听器统一处理子元素事件，推荐仅在 **冒泡阶段** 监听（默认行为），避免重复触发  
- **性能优化**：若无必要，不要同时在捕获和冒泡阶段监听同一事件  
- **灵活拦截**：在捕获阶段监听适合需要“拦截”事件的场景（如阻止默认行为）
