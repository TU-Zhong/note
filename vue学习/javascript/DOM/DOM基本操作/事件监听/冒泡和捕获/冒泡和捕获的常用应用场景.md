

以下是事件冒泡（Bubbling）和捕获（Capturing）的常用应用场景及示例代码，帮助你更高效地管理事件：

---

### 一、**事件冒泡（Bubbling）的典型场景**
#### 1. **事件委托（Event Delegation）**
[[事件委托的e.target是什么意思]]

**场景**：动态生成子元素时（如列表项），无需为每个子元素单独绑定事件，直接在父元素统一处理。  
```html
<ul id="task-list">
  <!-- 动态添加的 <li> 元素 -->
</ul>
```
```javascript
document.getElementById('task-list').addEventListener('click', (e) => {
  if (e.target.tagName === 'LI') { // 通过冒泡机制捕获子元素事件
    console.log('点击的任務是:', e.target.textContent);
  }
});
```

#### 2. **表单统一验证**
**场景**：在表单提交时统一验证所有输入框。  
```html
<form id="my-form">
  <input type="text" name="username">
  <input type="password" name="password">
</form>
```
```javascript
document.getElementById('my-form').addEventListener('submit', (e) => {
  e.preventDefault();
  const inputs = e.currentTarget.elements;
  // 冒泡阶段统一遍历所有输入域
  validateInputs(inputs); // 自定义验证逻辑
});
```

### 二、**事件捕获（Capturing）的典型场景**
#### 1. **提前拦截事件（如权限检查）**
**场景**：在事件到达目标前拦截，例如检查用户权限后决定是否允许操作。  
```javascript
document.querySelector('.admin-button').addEventListener('click', (e) => {
  if (!user.isAdmin) {
    e.stopPropagation(); // 捕获阶段直接阻止事件传递
    alert('无权限操作！');
  }
}, true); // 设置为捕获阶段监听
```

#### 2. **全局埋点统计**
**场景**：在点击事件到达目标前记录埋点数据（避免被 `stopPropagation` 阻断）。  
```javascript
document.body.addEventListener('click', (e) => {
  logAnalytics(e.target); // 在捕获阶段记录埋点
}, true); // 强制捕获阶段触发
```

---

### 三、**混合使用的特殊场景**
#### 1. **拖拽组件开发**
**场景**：在捕获阶段处理鼠标按下事件（确保优先响应），冒泡阶段处理释放事件。  
```javascript
// 捕获阶段开始拖拽
draggableElement.addEventListener('mousedown', startDrag, true);

// 冒泡阶段结束拖拽
document.addEventListener('mouseup', endDrag); 
```

#### 2. **模态框点击外部关闭**
**场景**：点击模态框外部区域时关闭弹窗，但需避免点击内部元素冒泡误触发。  
```javascript
document.getElementById('modal').addEventListener('click', (e) => {
  e.stopPropagation(); // 阻止模态框内部点击冒泡
}, false);

document.body.addEventListener('click', () => {
  closeModal(); // 捕获点击外部区域（冒泡到此）
}, false);
```

### 📊 对比总结表

| **机制**   | **典型应用场景**                     | **优势**                          | **示例代码关键点**         |
|------------|------------------------------------|-----------------------------------|--------------------------|
| **冒泡**   | 动态元素事件委托、表单统一提交逻辑      | 减少事件监听器数量，性能优化        | `addEventListener('click', handler)` |
| **捕获**   | 权限拦截、全局埋点、组件库高级功能      | 优先处理事件，避免被后续逻辑阻断    | `addEventListener('click', handler, true)` |

---

### ⚠️ 注意事项
1. **避免滥用捕获**：除非必要（如拦截），否则优先使用冒泡（默认行为更符合直觉）。  
2. **阻止传播**：谨慎使用 `e.stopPropagation()`，可能破坏其他监听逻辑。  
3. **性能优化**：通过 `{ once: true }` 参数让监听器只触发一次。  

合理选择冒泡或捕获，可以让你的代码更简洁高效！ 🚀
