### **浏览器里的实际表现**
当点击网页中的按钮时：
1. **捕获阶段**（先从上往下找目标）：  
   `document → <html> → <body> → <div> → <button>`  
   （默认不使用这个阶段）

2. **冒泡阶段**（再从下往上通知）：  
   `<button> → <div> → <body> → <html> → document`  
   （默认事件监听器在这个阶段触发）

---

### **如何控制使用哪个阶段？**
用 `addEventListener` 的第三个参数：
```javascript
// 捕获阶段触发（爷爷先知道）
element.addEventListener('click', handler, true);

// 冒泡阶段触发（孙子先反应，默认行为）
element.addEventListener('click', handler, false); 
// 或直接省略第三个参数
```

---

### **举个代码例子**
<mark style="background: #FFF3A3A6;">观看浏览器开发者工具中的console输出，如果第三个参数是true,那么就是默认捕获阶段，父级元素比子级元素更先被触发；而如果什么都不写，那么就是冒泡阶段，那么子级元素比父级元素更先触发。这两个选择一个触发。</mark>

<mark style="background: #FFF3A3A6;">那是父级元素有和子级元素同样的事件比如"click"才会触发，比如爷爷绑定了"click",孙子也绑定了"click",这样才会在点孙子的时候触发爷爷；如果爷爷绑定的事件不一样，比如“doubleclick”,那么爷爷就不会被触发。</mark>

<mark style="background: #FFF3A3A6;">为什么会触发？可以理解为当你点了儿子这个元素，那么首先浏览器会从顶到尾地查找儿子这个元素，而儿子绑定的事件流也会顺便向下流动，这时沿途碰到了也绑定了相同事件的元素，就同样会触发这个元素所绑定的事件。</mark>

你的描述中说事件"从顶到尾查找儿子时**顺便往下流动**"，这混淆了两个阶段：

- **捕获阶段是主动的“下探”**（默认不触发，需显式监听，第三个参数设置成 true）
- **冒泡阶段是被动的“上浮”**（默认触发）

实际触发父级元素的逻辑在于：

1. **事件委托机制**：父元素监听的是子元素的冒泡行为
2. **传播路径覆盖**：无论捕获还是冒泡阶段，事件都会经过父元素

`document → <html> → <body> → <div> → <div> -> <button>`  
`<button> → <div> → <div> -> <body> → <html> → document`  

[[父级监听能被触发几次？]]
```html
<!DOCTYPE html>  
<html lang="en">  
<head>  
    <meta charset="UTF-8">  
    <title>Title</title>  
    <style>        .爷爷{  
            background-color: red;  
            color: white;  
        }  
        .爷爷plus{  
            background-color: black;  
            color: white;  
        }  
    </style>  
</head>  
<body>  
<div class="爷爷" id="hello">  
    <div class="爸爸">  
        <button class="儿子">戳我</button>  
    </div></div>  
  
<script>  
  // 捕获阶段（爷爷先出手）  
document.querySelector('.爷爷').addEventListener('click', () => {  
    const grandfather=document.getElementById("hello");  
    // grandfather.classList.remove("爷爷");  
    grandfather.classList.toggle("爷爷plus");  
    console.log("爷爷在此！");  
},true);  
  
// 冒泡阶段（儿子先喊疼）  
document.querySelector('.儿子').addEventListener('click', () => {  
    console.log('儿子被戳疼了！');  
});
</script>  
</body>  
</html>
```



**点击按钮后的输出顺序**：  
1. `爷爷知道你要戳了！`（捕获阶段）  
2. `儿子被戳疼了！`（冒泡阶段）

---

### **总结一句话**
- **冒泡**：事件从最里面的元素 **往上冒泡** 通知所有长辈（默认）。  
- **捕获**：事件从最外面的元素 **往下钻** 找目标（需要特殊设置）。  

就像你闯祸时，要么先偷偷告诉家长（捕获），要么先自己承认再让家长知道（冒泡）！ 😉
