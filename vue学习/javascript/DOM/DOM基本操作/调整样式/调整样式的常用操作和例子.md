#### 1. **直接修改 `style` 属性**
<mark style="background: #FFF3A3A6;">element.style和cssRules.style之间的区别</mark>

### **核心区别总结表**
| 特性                | `element.style`                  | `cssRules.style`                  |
|--------------------|----------------------------------|-----------------------------------|
| **作用目标**         | 单个元素                         | 样式表中的规则（影响所有匹配元素）  |
| **优先级**           | 最高（内联样式）                  | 取决于样式表位置和选择器权重        |
| **性能**            | 修改单个元素时轻量                | 批量修改时更高效                   |
| **典型场景**         | 动态调整某个元素的样式            | 全局主题切换、批量样式修改           |
<mark style="background: #FFF3A3A6;">内联样式优先级最高，会覆盖cssRules的修改</mark>
```javascript
// 单个属性修改
element.style.color = "red";
element.style.backgroundColor = "#f0f0f0";

// 批量修改（通过 cssText）
element.style.cssText = "color: red; font-size: 16px;";
```
**特点**：直接作用于<mark style="background: #FFF3A3A6;">内联样式</mark>（优先级高），适用于动态修改少量属性。

#### 3. **操作 CSS 样式表（CSSOM）**
<mark style="background: #FFF3A3A6;">通常用于全局更改样式</mark>
[[cssdom通俗解释]]
stylesheet:**本质**：网页中所有 CSS 样式规则的集合（比如 `<style>` 标签或外部 `.css` 文件）。
<mark style="background: #FFF3A3A6;">内联样式比如</mark>     `<div style="color:red;">`    <mark style="background: #FFF3A3A6;">这种就不会出现在cssRules中</mark>
### <mark style="background: #FFF3A3A6;">**为什么 `cssRules` 不包含内联样式？**</mark>
- `cssRules` 是 CSSOM（CSS 对象模型）的一部分，表示的是样式表中的规则。
- 内联样式属于 HTML 元素的私有属性，不归入任何样式表。<br>
cssRules:**本质**：一个样式表（`stylesheet`）中所有 CSS 规则的集合（比如 `.box { ... }` 就是一个规则）。<br>

<mark style="background: #FFF3A3A6;">cssRules数组中查找特定的类：</mark>
```javascript
// 1. 获取第一个样式表
const stylesheet = document.styleSheets[0];

// 2. 遍历所有规则，找到 .text 对应的规则
let targetRule;
for (let rule of stylesheet.cssRules) {
  if (rule.selectorText === ".text") {
    targetRule = rule;
    break;
  }
}

// 3. 修改颜色
if (targetRule) {
  targetRule.style.color = "blue"; 
  // 页面上所有 .text 元素都会变蓝！
}
```
可以直接改变样式。
```javascript

const stylesheet = document.styleSheets[0];
const rule = stylesheet.cssRules[0];
rule.style.backgroundColor = "blue";
```

#### 4. **使用 `setAttribute`**
```javascript
element.setAttribute("style", "color: green; border: 1px solid black;");
```
**特点**：覆盖所有内联样式，需谨慎使用。

### **注意事项**

#### 1. **样式优先级问题**
- 内联样式（`style` 属性）优先级最高，可能覆盖 CSS 文件或 `<style>` 标签中的样式。
- 使用 `classList` 时，确保 CSS 类中的样式定义正确且优先级足够。

---

#### 2. **性能优化**
- **避免频繁操作 DOM**：在循环中多次修改样式可能导致页面重排（Reflow）和重绘（Repaint），建议批量操作或使用 `documentFragment`。
- **使用 `requestAnimationFrame`**：对动画等高频修改场景，使用 `requestAnimationFrame` 优化性能。

---

#### 3. **兼容性**
- `classList` 方法在 IE10 及以上支持，旧版浏览器需使用 `className` 拼接字符串。
- `cssText` 覆盖原有样式时需注意保留必要属性。

---

#### 4. **单位与值格式**
- 数值属性需添加单位（如 `px`, `%`）：
  ```javascript
  element.style.width = "100px"; // 正确
  element.style.width = 100;     // 错误（无效）
  ```

---

#### 5. **响应式场景**
- 动态修改样式后，可能需要触发布局更新：
  ```javascript
  // 强制重新计算样式（慎用）
  const width = element.offsetWidth;
  ```

---

#### 6. **CSS 变量（Custom Properties）**
- 可通过 JavaScript 修改 CSS 变量：
  ```javascript
  // 修改变量值
  element.style.setProperty("--primary-color", "#ff0000");
  // 移除变量
  element.style.removeProperty("--primary-color");
  ```

---

### **最佳实践**
1. **优先使用 CSS 类**：通过 `classList` 管理样式，保持代码可维护性。
2. **分离关注点**：将样式定义放在 CSS 文件中，JavaScript 仅负责逻辑切换。
3. **慎用内联样式**：避免因优先级过高导致后续维护困难。
4. **测试多浏览器**：确保样式修改在目标浏览器中生效。

根据需求选择合适的方法，可结合框架（如 React/Vue）的状态管理更高效地控制样式。
