


当 HTML 元素同时拥有多个类（如 `<div class="class1 class2">`），浏览器选择样式的规则基于 **CSS选择器的优先级** 和 **样式定义的顺序**。以下是通俗解释：

---

### **规则一：就近原则（后定义的样式覆盖前面的）**
如果多个类对同一属性定义了不同的值，**代码中后出现的类会覆盖前面类的样式**。

#### 例子：
```css
.class1 { color: red; }
.class2 { color: blue; } /* 生效 */
```
```html
<div class="class1 class2">文字颜色是蓝色</div>
```

---

### **规则二：选择器优先级（权重更高者胜出）**
如果不同选择器的优先级不同，**权重更高的选择器定义的样式生效**，与类名的顺序无关。

#### 优先级权重（从高到低）：
1. `!important`（慎用） → `color: red !important;`
2. **内联样式**（直接写在元素上的 `style` 属性） → `<div style="color: green">`
3. **ID选择器** → `#myId { color: yellow; }`
4. **类/伪类/属性选择器** → `.class1 { ... }`
5. **元素选择器/伪元素** → `div { ... }`

#### 例子：
```css
.class1 { color: red; }
div.class1.class2 { color: blue; } /* 生效（权重更高） */
```
```html
<div class="class1 class2">文字颜色是蓝色</div>
```

---

### **规则三：直接冲突时的细节**
1. **同一选择器多次定义**：以最后一次定义的值为准。
2. **不同选择器权重相同**：按代码中出现顺序，后定义的生效。

#### 例子：
```css
.class1.class2 { color: red; }
.class2.class1 { color: blue; } /* 生效（顺序靠后，权重相同） */
```

---

### **总结判断步骤** 🔍
1. 是否有 `!important`？  
   → 有则直接胜出。
2. 是否有内联样式（`style`属性）？  
   → 有则优先于外部或 `<style>` 中的样式。
3. 计算选择器的权重：  
   - ID选择器 > 类选择器 > 元素选择器。
4. 若权重相同，按代码中定义的顺序判断，靠后者生效。

---

### **实际场景练习** 🌟
假设有以下 CSS 和 HTML：
```css
.text { color: red; }         /* 权重：0,0,1,0 */
.special.text { color: blue; }/* 权重：0,0,2,0（更高） */
#title { color: green; }      /* 权重：0,1,0,0（最高） */
```
```html
<div class="text special" id="title" style="color: purple;">
  最终颜色是？
</div>
```

**答案**：  
1. 内联样式 `style="color: purple"` 优先级最高 → **紫色生效**。  
2. 若没有内联样式，则 ID选择器 `#title` 胜出 → **绿色**。  
3. 若也没有 ID，则 `.special.text` 胜出 → **蓝色**。

---

记住：**样式冲突时，浏览器像法官一样层层筛选，直到找到最“权威”的规则** 🧑⚖️！
