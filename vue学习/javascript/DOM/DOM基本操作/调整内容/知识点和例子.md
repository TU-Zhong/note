#### 1. **`innerHTML`**  
**作用**：获取或设置元素的 HTML 内容（支持标签解析）  
**风险**：直接插入未过滤内容可能导致 XSS 攻击  
**示例**：
[[insertAdjacentHTML知识点和例子]]<mark style="background: #FFF3A3A6;">这个函数也是用于插入html代码片段，注意差别</mark>
```javascript
// 设置内容（含 HTML 标签）
document.getElementById("box").innerHTML = "<strong>警告！</strong> 请检查输入";

// 获取内容
const htmlContent = element.innerHTML; 
```

#### 2. **`textContent`**  
[[innerText和textContent的区别]]
[[textContent知识点]]
[[会清空所有的子元素]]
**作用**：安全获取或设置元素的纯文本内容（自动转义 HTML）  
**示例**：
```javascript
// 设置纯文本（自动转义特殊字符）
element.textContent = "用户输入：<script>alert(1)</script>"; // 页面显示为文本，不会执行

// 获取纯文本（忽略标签）
const text = element.textContent; 
```

#### 3. **`innerText`**  
[[innerText和textContent的区别]]
**作用**：获取或设置元素的**可见文本**（考虑 CSS 样式，性能较差）  
**示例**：
```javascript
// 设置可见文本（隐藏元素内容不会被获取）
element.innerText = "新内容";

// 获取可见文本（若元素被隐藏返回空）
const visibleText = element.innerText;
```

### **二、动态创建与插入内容**
#### 1. **`document.createElement`**  
**作用**：安全创建新元素  
**示例**：
```javascript
const div = document.createElement("div");
div.className = "alert";
div.textContent = "操作成功！";
document.body.appendChild(div);
```

#### 2. **`appendChild` / `append`**  
[[appendChild和append的区别]]
<mark style="background: #FFF3A3A6;">最明显的差别应该是后者可以接受字符串做参数，前者不行。</mark>
document不会允许存在同一个节点

**作用**：将元素插入到父级末尾  
```javascript
// appendChild (插入单个元素)
parent.appendChild(child);

// append (支持插入多个元素或文本)
parent.append("提示：", div, "请留意");
```

#### 3. **`insertAdjacentHTML`**  
[[insertAdjacentHTML知识点和例子]]
**作用**：在指定位置插入 HTML 字符串  
**安全建议**：插入前需消毒内容  
```javascript
// 插入到元素前面
element.insertAdjacentHTML("beforebegin", "<p>前置内容</p>");

// 插入到元素内部末尾
element.insertAdjacentHTML("beforeend", "<li>新增项</li>");
```

### **三、内容替换与删除**
[[replaceWith和replaceWithChildre的区别]]
#### <mark style="background: #FFF3A3A6;">**专属注意事项**</mark>
- **`replaceWith()`**：
  - 替换后，原节点的所有属性和事件监听器失效。
  - 如果当前节点是根节点（如 `<html>`），会报错。
  - 原属性样式也会失效

- **`replaceChildren()`**：
  - 无参数调用时，会清空所有子节点：
    ```javascript
    parent.replaceChildren(); // 清空所有子节点
    ```
	  - 支持传入空字符串或 `null` 清空内容。



<mark style="background: #FFF3A3A6;">**小实验：可以用div标签，里面装四个p标签，然后分别给这两个dev标签设置不同背景颜色，然后再分别调用这两个函数，比对差别。**</mark>
#### 1. **`replaceWith`**  
**作用**：替换元素  
```javascript
const newElement = document.createElement("span");
newElement.textContent = "已更新";
oldElement.replaceWith(newElement);
```

#### 2. **`replaceChildren`**  
**作用**：替换所有子元素  
```javascript
parent.replaceChildren(
  document.createElement("div"),
  document.createTextNode("新内容")
);
```

#### 3. **`remove`**  
**作用**：删除元素  
```javascript
element.remove(); // 直接删除
```

### **四、表单内容操作**
#### 1. **`value` 属性**  
```javascript
// 输入框/下拉菜单
const input = document.querySelector("input");
input.value = "默认文本";

// 多行文本框
const textarea = document.querySelector("textarea");
textarea.value = "多行内容";
```

#### 2. **复选框/单选按钮**  
这个要用组合筛选条件[[css选择器有多个匹配条件怎么办]]
<mark style="background: #FFF3A3A6;">来匹配那个特定的选项，如结合id等条件。不过id也是不会重复就是了</mark>
```javascript
const checkbox = document.querySelector("input[type='checkbox']");
checkbox.checked = true; // 勾选

const radio = document.querySelector("input[type='radio']");
radio.checked = true; // 选中
```

### **五、安全操作规范**
#### 1. **避免直接使用 `innerHTML` 插入用户输入**
```javascript
// 错误做法（XSS 风险）
element.innerHTML = userContent;

// 正确做法（使用消毒库）
import DOMPurify from "dompurify";
element.innerHTML = DOMPurify.sanitize(userContent);
```

#### 2. **优先使用 `textContent` 替代 `innerHTML`**
```javascript
// 安全显示用户输入
element.textContent = userContent;
```

### **六、性能优化**
#### 1. **批量操作使用 `DocumentFragment`**
```javascript
const fragment = document.createDocumentFragment();

for (let i = 0; i < 100; i++) {
  const li = document.createElement("li");
  li.textContent = `Item ${i}`;
  fragment.appendChild(li);
}

document.querySelector("ul").appendChild(fragment);
```

#### 2. **减少 DOM 操作次数**
```javascript
// 错误：频繁修改样式
elements.forEach(el => el.style.margin = "10px");

// 正确：通过 CSS 类名批量修改
// CSS 中定义 .m10 { margin: 10px; }
elements.forEach(el => el.classList.add("m10"));
```

### **七、综合示例**
#### 动态生成表格
```javascript
const data = [
  { name: "Alice", age: 25 },
  { name: "Bob", age: 30 }
];

const table = document.createElement("table");
const tbody = document.createElement("tbody");

data.forEach(user => {
  const row = document.createElement("tr");
  
  const nameCell = document.createElement("td");
  nameCell.textContent = user.name;
  
  const ageCell = document.createElement("td");
  ageCell.textContent = user.age;
  
  row.append(nameCell, ageCell);
  tbody.appendChild(row);
});

table.appendChild(tbody);
document.body.appendChild(table);
```

#### 安全渲染用户评论
```javascript
const userComment = '<img src="x" onerror="alert("XSS")">';

// 安全渲染
const safeComment = document.createElement("div");
safeComment.textContent = userComment; // 自动转义为文本
// 或使用 DOMPurify
safeComment.innerHTML = DOMPurify.sanitize(userComment);

document.getElementById("comments").appendChild(safeComment);
```

### **总结**
| **方法**              | **适用场景**                          | **安全性**            |
|-----------------------|-------------------------------------|----------------------|
| `textContent`         | 纯文本内容（优先选择）                 | 高（自动转义 HTML）   |
| `innerHTML`           | 需要解析 HTML 标签时（需消毒）         | 低（需手动处理）      |
| `createElement`       | 动态创建复杂元素结构                   | 高                   |
| `insertAdjacentHTML`  | 灵活插入 HTML 片段（需消毒）           | 中（依赖输入过滤）    |
| 表单 `.value`         | 输入框、下拉菜单等表单控件              | 高                   |

通过合理选择方法，既能实现动态交互，又能确保页面安全性和性能！ 🔒
