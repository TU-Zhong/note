

在 DOM 操作中，当你获取一个元素的 `textContent`（包含其所有子元素的文本），然后重新设置该元素的 `textContent` 时，**所有子元素都会被删除**，并被替换为一个新的文本节点。这意味着原来的子元素结构和排列会被完全覆盖！

---

### **流程解析**
1. **初始状态**：  
   ```html
   <div id="parent">
     父元素文本
     <span>子元素1</span>
     <p>子元素2</p>
   </div>
   ```

2. **获取 `textContent`**：  
   ```javascript
   const parent = document.getElementById("parent");
   const text = parent.textContent; 
   console.log(text); // 输出："父元素文本 子元素1 子元素2"
   ```

3. **重新设置 `textContent`**：  
   ```javascript
   parent.textContent = "新文本"; 
   ```

4. **最终状态**：  
   ```html
   <div id="parent">新文本</div>
   ```  
   - 所有子元素（`<span>` 和 `<p>`）被删除  
   - 仅保留一个文本节点（内容为 `"新文本"`）

---

### **关键知识点**
| **行为**                | **结果**                     | **DOM 变化**               |
|-------------------------|-----------------------------|---------------------------|
| 设置 `textContent`       | 清空所有子节点并设置纯文本内容 | 原有子元素被完全移除         |
| 对比 `innerText`        | 行为相同，但受 CSS 影响       | 同样移除子元素              |
| 对比 `innerHTML`        | 可保留 HTML 结构             | 不会移除子元素（根据设置内容） |

---

### **常见问题**
#### **1. 如何修改父元素文本但保留子元素？**
- **错误做法**：直接设置父元素的 `textContent`  
- **正确方案**：定位到具体的文本节点或子元素修改  
```javascript
// 只修改父元素本身的文本（假设父元素没有其他子元素）
parent.firstChild.textContent = "新父文本"; 

// 或者添加新的文本节点（保留子元素）
const newText = document.createTextNode("新父文本");
parent.insertBefore(newText, parent.firstChild);
```

#### **2. 如何更新父元素文本并保留部分子元素？**
1. **克隆需要保留的子元素**  
2. **修改父元素内容后重新插入**  
```javascript
const parent = document.getElementById("parent");
const childrenToKeep = Array.from(parent.children); // 获取所有子元素

// 清空父元素
parent.textContent = "新文本";

// 重新插入要保留的子元素
childrenToKeep.forEach(child => parent.appendChild(child));
```

---

### **示例场景**
#### **目标**：修改父元素的文本，但保留子元素 `<span>`
```html
<div id="parent">
  原始父文本
  <span>重要子元素</span>
</div>
```

```javascript
const parent = document.getElementById("parent");
const span = parent.querySelector("span"); // 保存子元素引用

// 清空父元素并设置新文本（会删除 span）
parent.textContent = "新父文本";

// 重新插入子元素
parent.appendChild(span);
```

**最终结构**：  
```html
<div id="parent">
  新父文本
  <span>重要子元素</span>
</div>
```

---

### **总结**
- **`textContent` 是核武器**：设置它会清空所有子元素，只保留纯文本。  
- **保留子元素的正确姿势**：  
  1. 提前保存需要保留的子元素引用  
  2. 修改内容后重新插入  
- **替代方案**：使用 `insertAdjacentText` 或操作特定文本节点，避免清空整个 DOM 结构。
