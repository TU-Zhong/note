关于模块导入导出的核心机制，以下是分步解释：

### 一、ES6 模块系统基础
1. **导出模块**：使用 `export` 关键字暴露内容
2. **导入模块**：使用 `import` 关键字获取内容

### 二、`export default` 与普通导出的区别
| 特性                | export default          | export const         |
|---------------------|-------------------------|----------------------|
| 导出次数限制        | 每个模块只能有一个      | 可多个导出           |
| 导入方式            | 直接接收整个默认导出    | 需要解构获取指定导出 |
| 语法示例            | `export default {}`     | `export const foo = {}` |
| 导入示例            | `import X from '...'`   | `import {foo} from '...'` |

### 三、你的代码实例解析
#### 导出端代码：
```javascript
// Person.vue
export default {
  name: 'Person',
  data() {
    return { age: 25 }
  }
}
```

#### 导入端代码：
```javascript
import MyComponent from './Person.vue'
```

#### 工作原理：
1. **导出过程**：
   - `export default` 将右侧的对象作为模块的默认导出
   - 该对象没有固定的变量名标识

2. **导入过程**：
   - `MyComponent` 是导入者自定义的变量名（可使用任意合法名称）
   - 该变量自动绑定到被导入文件的默认导出对象

### 四、命名关系的三个阶段
| 阶段     | 名称示例       | 说明                         |
|----------|---------------|-----------------------------|
| 导出文件 | `Person.vue`  | 物理文件名（重要标识）       |
| 导出对象 | 无固定名称    | 通过 `export default` 匿名导出 |
| 导入使用 | `MyComponent` | 接收者自定义的引用名称       |

### 五、实际应用演示
#### 示例一：基础用法
```javascript
// 导出端
export default { name: 'Person' }

// 导入端（三种等效写法）
import MyComponent from './Person.vue'
import AnyName from './Person.vue'
import Person from './Person.vue' // 推荐使用与组件名一致的名称
```

#### 示例二：注册组件
```vue
<template>
  <!-- 使用组件 -->
  <PersonComponent />
</template>

<script>
// 导入时使用不同名称
import MyCustomName from './Person.vue'

export default {
  components: {
    // 注册时指定模板标签名
    PersonComponent: MyCustomName
  }
}
</script>
```

### 六、Vue 组件的完整名称体系
1. **文件名**：`Person.vue`（物理文件标识）
2. **组件选项名**：`name: 'Person'`（开发工具显示名）
3. **导入变量名**：`MyComponent`（代码中的引用名称）
4. **注册标签名**：最终模板中的使用名称

### 七、命名最佳实践
| 名称类型       | 推荐规范           | 示例              |
|---------------|--------------------|-------------------|
| 文件名         | PascalCase.vue     | `PersonCard.vue`  |
| 组件选项名     | PascalCase         | `name: 'PersonCard'` |
| 导入变量名     | 与组件名一致       | `import PersonCard from './PersonCard.vue'` |
| 注册标签名     | PascalCase 或 kebab-case | `<person-card>` 或 `<PersonCard>` |

### 八、常见问题解答
#### Q1：可以不使用 `export default` 吗？
可以，但需要使用具名导出：
```javascript
// 导出端
export const PersonComponent = { name: 'Person' }

// 导入端
import { PersonComponent } from './Person.vue'
```

#### Q2：导入变量名与组件名不一致是否影响使用？
不会，但建议保持一致性，例如：
```javascript
// 不推荐（易混淆）
import Foo from './Person.vue'

// 推荐（清晰明确）
import Person from './Person.vue'
```

#### Q3：为什么 DevTools 中显示的是组件名而不是导入变量名？
因为 Vue 开发者工具通过 `name` 属性识别组件，与导入变量名无关：
```javascript
export default {
  name: 'Person', // 这里决定 DevTools 的显示
  // ...
}
```

### 九、综合示例
```javascript
// AnimalDog.vue
export default {
  name: 'AnimalDog',
  data: () => ({ bark: 'Woof!' })
}

// AnimalCat.vue
export default {
  name: 'AnimalCat',
  data: () => ({ meow: 'Meow!' })
}

// App.vue
import DogComponent from './AnimalDog.vue'
import CatComponent from './AnimalCat.vue'

export default {
  components: {
    MyDog: DogComponent,    // 注册为 <my-dog>
    SpecialCat: CatComponent // 注册为 <special-cat>
  }
}
```
