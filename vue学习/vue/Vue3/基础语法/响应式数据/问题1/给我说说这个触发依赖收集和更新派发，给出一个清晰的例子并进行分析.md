

以下是关于 Vue3 的依赖收集（track）和更新派发（trigger）的完整示例解析，通过自实现的简易响应式系统演示其核心原理：

```html
<!DOCTYPE html>
<html>
<body>
  <button onclick="changeData()">修改数据</button>
  
  <script>
/* 一、极简响应式系统实现 ---------------------------------- */
const targetMap = new WeakMap() // 依赖存储中心

// 当前正在运行的 effect
let activeEffect = null 

// 副作用注册函数
function effect(fn) {
  activeEffect = fn
  fn() // 首次执行触发依赖收集
  activeEffect = null
}

// 依赖收集函数
function track(target, key) {
  if (!activeEffect) return

  let depsMap = targetMap.get(target)
  if (!depsMap) {
    depsMap = new Map()
    targetMap.set(target, depsMap)
  }

  let dep = depsMap.get(key)
  if (!dep) {
    dep = new Set()
    depsMap.set(key, dep)
  }
  
  dep.add(activeEffect) // 注册依赖关系
  console.log(`[Track] 收集依赖 ${key} →`, activeEffect.name)
}

// 更新派发函数
function trigger(target, key) {
  const depsMap = targetMap.get(target)
  if (!depsMap) return

  const dep = depsMap.get(key)
  if (dep) {
    console.log(`[Trigger] 触发更新 ${key}，执行回调：`, [...dep].map(fn => fn.name))
    dep.forEach(effect => effect())
  }
}

// 响应式对象生成函数
function reactive(obj) {
  return new Proxy(obj, {
    get(target, key) {
      track(target, key)
      return Reflect.get(target, key)
    },
    set(target, key, value) {
      Reflect.set(target, key, value)
      trigger(target, key)
      return true
    }
  })
}

/* 二、实际使用示例 ---------------------------------- */
const state = reactive({ 
  count: 0,
  price: 10
})

// 副作用函数1：总价计算
effect(function totalEffect() {
  console.log('总价更新:', state.count * state.price)
})

// 副作用函数2：数量校验
effect(function checkEffect() {
  if (state.count < 0) {
    console.log('数量不能为负，已自动重置')
    state.count = 0
  }
})

/* 三、交互测试 ---------------------------------- */
function changeData() {
  console.log('\n--- 修改数据 ---')
  state.count += 1
}

// 初始输出：
// 总价更新: 0
// 数量校验: 0
  </script>
</body>
</html>
```

### 执行流程分析（点击按钮时）

#### 步骤 1：修改数据
```javascript
state.count += 1
→ 执行 Proxy 的 set 拦截
→ 触发 trigger(state, 'count')
```

#### 步骤 2：触发更新
```javascript
trigger 查找依赖：
找到依赖 state.count 的两个 effect：
- totalEffect
- checkEffect
依次执行这两个函数
```

#### 步骤 3：执行副作用函数
```javascript
1. 执行 totalEffect → 计算 1 * 10 = 10
2. 执行 checkEffect → 校验通过（1 ≥ 0）
```

#### 步骤 4：二次触发（当校验失败时）
```javascript
// 如果执行：
state.count = -5

// checkEffect 中会触发：
state.count = 0 → 再次触发 set → 新一轮更新
```

### 核心机制图解

```
┌─────────────┐          ┌─────────────┐
│  响应式对象  │          │ 依赖存储中心 │
│  Proxy      │          │ targetMap   │
├─────────────┤          ├─────────────┤
│ count       │◄──track─ ┤ Map         │
│ price       │          │  └─ count → {totalEffect, checkEffect} │
└─────┬───────┘          └──────▲───────┘
      │                         │
      │ set                     │ trigger
      ▼                         │
┌─────────────┐                 │
│ 数据修改事件  │                │
└─────────────┘                 │
      │                         │
      └─────────通知更新─────────┘
```

### 关键概念解析

| 概念          | 作用                                                                 | 对应代码                         |
|---------------|----------------------------------------------------------------------|----------------------------------|
| **effect**    | 副作用函数，表示依赖响应式数据的行为                                   | `effect(() => { ... })`         |
| **track**     | 在数据读取时收集当前 effect 作为依赖                                   | Proxy getter 中的 `track()`     |
| **trigger**   | 在数据修改时通知所有关联的 effect 重新执行                             | Proxy setter 中的 `trigger()`   |
| **targetMap** | 使用 WeakMap 存储对象与键的依赖关系（防止内存泄漏）                    | `const targetMap = new WeakMap()` |

### 性能优化特征

1. **精准更新**  
   - 只触发与修改数据相关联的 effect
   - 例如修改 `price` 不会触发 `checkEffect`

2. **依赖关系动态更新**  
   ```javascript
   effect(() => {
     // 根据条件动态依赖不同数据
     if (condition) {
       console.log(state.a)
     } else {
       console.log(state.b)
     }
   })
   ```
   - 每次执行都会重新收集当前实际使用的依赖

3. **批量更新**  
   ```javascript
   // 连续修改多个属性
   state.a = 1
   state.b = 2
   // 会被合并到同一事件循环中处理
   ```

### 与 Vue2 的对比

| 特性               | Vue2 (defineProperty)               | Vue3 (Proxy)                      |
|--------------------|-------------------------------------|------------------------------------|
| 初始化性能          | 递归转换所有属性（性能消耗大）        | 按需代理（惰性转换）                |
| 数组处理            | 需要重写数组方法                    | 原生支持数组索引修改                |
| 新增属性           | 需要 `Vue.set`                      | 自动检测                           |
| 删除属性           | 需要 `Vue.delete`                   | 自动检测                           |

### 实际应用场景

1. **表单联动验证**  
   ```javascript
   const form = reactive({
     username: '',
     password: '',
     error: ''
   })

   effect(() => {
     if (form.username.length < 6) {
       form.error = '用户名至少6位'
     } else if (form.password.length < 8) {
       form.error = '密码至少8位'
     } else {
       form.error = ''
     }
   })
   ```

2. **数据缓存优化**  
   ```javascript
   const cache = new Map()
   effect(() => {
     const key = JSON.stringify({ a: state.a, b: state.b })
     if (!cache.has(key)) {
       cache.set(key, heavyCalculation(state.a, state.b))
     }
     result.value = cache.get(key)
   })
   ```

3. **动画帧同步**  
   ```javascript
   const pos = reactive({ x: 0, y: 0 })
   
   effect(() => {
     requestAnimationFrame(() => {
       element.style.transform = `translate(${pos.x}px, ${pos.y}px)`
     })
   })
   ```

通过这种响应式机制，Vue3 实现了高效精确的更新系统，使得开发者可以专注于业务逻辑而无需手动管理数据更新。理解这些底层原理有助于编写高性能的 Vue 应用。
