以下是 Vue3 中 `watch` 的常用知识点及示例：

---

### 1. **基本用法**
监听一个响应式数据源的变化，触发回调函数。
```javascript
import { ref, watch } from 'vue';

const count = ref(0);

watch(count, (newValue, oldValue) => {
  console.log(`值变化：${oldValue} → ${newValue}`);
});
```

---

### 2. **监听多个源**
通过数组传入多个监听源，回调参数为对应新值和旧值的数组。
```javascript
const count = ref(0);
const name = ref('Alice');

watch([count, name], ([newCount, newName], [oldCount, oldName]) => {
  console.log(`count变化: ${oldCount}→${newCount}, name变化: ${oldName}→${newName}`);
});
```

---

### 3. **监听响应式对象的属性**
对于 `reactive` 对象，需通过函数返回具体属性。
[[易错点 第一个参数传入对象时不同的传入方法]]
[[watch里面的深度监视是什么意思？]]
```javascript
import { reactive, watch } from 'vue';

const state = reactive({ 
  user: { 
    name: 'Bob' 
  } 
});

// 监听 state.user.name
watch(() => state.user.name, (newName, oldName) => {
  console.log(`用户名变化：${oldName} → ${newName}`);
});
```

---

### 4. **深度监听（Deep Watch）**
[[给我说说为什么可以自己调用自己，stopWatch明明什么都没写为什么newSum到了10之后这个函数就停下来了？]]
对于嵌套对象，使用 `deep: true` 监听深层变化。
```javascript
const state = reactive({ 
  user: { 
    name: 'Bob' 
  } 
});

watch(
  () => state.user,
  (newUser, oldUser) => {
    console.log('user对象深层变化');
  },
  { deep: true }
);
```

---

### 5. **立即执行回调**
通过 `immediate: true` 在初始化时立即执行一次回调。
```javascript
const count = ref(0);

watch(
  count,
  (newValue) => {
    console.log(`初始值：${newValue}`); // 初始化时会立即执行
  },
  { immediate: true }
);
```

---

### 6. **使用 `watchEffect` 自动追踪依赖**
`watchEffect` 自动收集回调中的响应式依赖，并立即执行。
```javascript
const count = ref(0);

watchEffect(() => {
  // 自动追踪 count.value 的依赖
  console.log(`当前值：${count.value}`);
});
```

---

### 7. **`watch` vs `watchEffect` 的区别**
| 特性                | `watch`                          | `watchEffect`                   |
|--------------------|----------------------------------|--------------------------------|
| **依赖源**           | 需显式指定监听源                  | 自动收集回调内的响应式依赖        |
| **立即执行**         | 默认不立即执行（需 `immediate`） | 默认立即执行                    |
| **新旧值获取**       | 提供新旧值                        | 不提供旧值                     |

---

### 示例对比 `watch` 和 `watchEffect`
```javascript
// 使用 watch
const x = ref(0);
watch(x, (newX, oldX) => {
  console.log(`x变化：${oldX} → ${newX}`);
});

// 使用 watchEffect
const y = ref(0);
watchEffect(() => {
  console.log(`y的当前值：${y.value}`); // 自动追踪 y.value
});
```

---

### 总结
- **`watch`**：适合精确监听特定数据，控制执行时机，需要新旧值。
- **`watchEffect`**：适合依赖自动追踪，无需显式声明监听源，立即执行。
- **高级场景**：结合 `{ deep: true }` 或 `immediate: true` 处理复杂逻辑。
