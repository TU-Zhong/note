

以下是 **Vue3 中 `watchEffect` 的常用知识点**以及与 `watch` 的核心区别：

---

### 一、`watchEffect` 的核心特性

#### 1. **自动依赖收集**
- **无需显式声明依赖**：回调函数中所有被读取的响应式变量（`ref`/`reactive`）会被自动追踪。
- **动态更新依赖**：每次执行回调会重新收集最新的依赖。
  ```javascript
  const a = ref(1);
  const b = ref(2);

  watchEffect(() => {
    console.log(a.value); // 自动追踪 a.value
    console.log(b.value); // 自动追踪 b.value
  });
  ```

#### 2. **立即执行**
- **初始化时立即运行一次**：在组件挂载阶段就会执行回调函数。
- **与 `watch` 的区别**：`watch` 默认不会立即执行（需配置 `immediate: true`）。

#### 3. **副作用清理（`onInvalidate`）**
- **参数接收清理函数**：在回调函数第一个参数中注册清理逻辑。
- **适用场景**：取消未完成的异步操作（如请求、定时器）。
  ```javascript
  watchEffect((onInvalidate) => {
    const timer = setTimeout(() => {
      // 异步操作...
    }, 1000);

    onInvalidate(() => {
      clearTimeout(timer); // 清理未完成的定时器
    });
  });
  ```

#### 4. **配置选项**
- **`flush`**：控制回调的执行时机。
  - `pre`（默认）：在组件更新前执行。
  - `post`：在组件更新后执行（适合操作 DOM）。
  - `sync`：同步执行（响应式变更后立即触发）。
  ```javascript
  watchEffect(() => { ... }, { flush: 'post' });
  ```

- **`onTrack` 和 `onTrigger`**（调试用）：
  ```javascript
  watchEffect(
    () => { /* 副作用代码 */ },
    {
      onTrack(e) { console.log('依赖被追踪', e) },
      onTrigger(e) { console.log('依赖触发回调', e) }
    }
  );
  ```

---

### 二、`watchEffect` vs `watch` 的关键区别

| 特性                | `watchEffect`                               | `watch`                                      |
|--------------------|---------------------------------------------|----------------------------------------------|
| **依赖声明**         | 自动收集回调内的响应式依赖                    | 需显式指定监听源（直接值、函数、数组）         |
| **立即执行**         | ✅ 默认立即执行                              | ❌ 默认不执行（需 `immediate: true`）         |
| **新旧值获取**       | ❌ 不提供旧值                               | ✅ 提供新旧值（`newValue`, `oldValue`）       |
| **深度监听**         | ❌ 默认不启用（但可配合 `reactive` 自动深度） | ✅ 支持 `deep: true` 配置                     |
| **适用场景**         | 组合多个依赖的副作用（如计算、DOM 操作）       | 精确监听特定数据的变化，需处理新旧值逻辑       |

---

### 三、何时使用 `watchEffect`？
1. **自动追踪多个依赖**：当副作用逻辑依赖多个变量，且不想手动维护监听列表时。
2. **无需旧值**：只需根据最新状态执行操作（如发送请求、操作 DOM）。
3. **简化代码**：代替简单的 `watch` + `immediate: true` 场景。

---

### 四、示例对比

#### 场景：根据 `id` 和 `type` 加载数据
- **使用 `watchEffect`**（自动追踪依赖）：
  ```javascript
  const id = ref(1);
  const type = ref('user');

  watchEffect(async (onInvalidate) => {
    const currentId = id.value;
    const currentType = type.value;

    // 清理未完成的请求
    let isCancelled = false;
    onInvalidate(() => (isCancelled = true));

    const data = await fetchData(currentId, currentType);
    if (!isCancelled) {
      // 更新数据...
    }
  });
  ```

- **使用 `watch`**（显式声明依赖）：
  ```javascript
  watch([id, type], async ([newId, newType], [oldId, oldType]) => {
    const data = await fetchData(newId, newType);
    // 更新数据...
  }, { immediate: true });
  ```

---

### 五、总结
- **`watchEffect`**：简化依赖管理，适合组合型副作用。
- **`watch`**：精确控制监听源，需要新旧值或深度监听时使用。
