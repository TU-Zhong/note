```typescript
let person=reactive({
  name:'张三',
  age:18,
  car:{
    c1:'奔驰',
    c2:'宝马'
  }
})

function changeC3(){
  person.car={c1:'自行车',c2:'学步车'}
}

watch(person.car,(newValue,oldValue)=>{
  console.log(newValue,oldValue)
})

```
触发了changeC3,但是为什么没有触发回调？


---

### 问题原因分析
**直接监听 `person.car` 时，Vue3 会默认对 `person.car` 启用深度监听（`deep: true`），但它的依赖是 `person.car` 的初始引用。当你替换 `person.car` 为一个新对象时，Vue 的侦听器仍然绑定在旧的引用上，因此不会触发回调。**

---

### 详细解释
#### 1. **`watch(person.car, ...)` 的监听逻辑**
- **初始监听**：Vue 会监听 `person.car` 的初始引用（即 `{ c1: '奔驰', c2: '宝马' }`）。
- **触发条件**：只有当这个初始引用的**内部属性变化**（如 `person.car.c1 = 'xxx'`）时，回调才会触发。
- **替换失效**：如果直接替换 `person.car`（如 `person.car = { ... }`），旧的引用已被丢弃，新的对象未被绑定到侦听器，因此**不会触发回调**。

#### 2. **示例代码的实际情况**
```javascript
function changeC3() {
  person.car = { c1: '自行车', c2: '学步车' }; // ✅ 替换整个对象
}

watch(person.car, (newVal, oldVal) => {
  console.log('触发', newVal, oldVal);
});
```
- **替换 `person.car` 时**：Vue 的侦听器仍然绑定在旧的 `{ c1: '奔驰', ... }` 对象上，因此不会检测到新对象的变化。

---

### 解决方案
**使用函数返回 `person.car`，并显式启用 `deep: true`，确保能追踪引用替换和内部属性变化：**
```javascript
watch(
  () => person.car, // 通过函数动态获取最新的 person.car
  (newVal, oldVal) => {
    console.log('触发', newVal, oldVal);
  },
  { deep: true } // 可选：如果需要监听内部属性变化
);
```

---

### 验证步骤
1. **修改 `watch` 的写法**：
   ```javascript
   watch(
     () => person.car,
     (newVal, oldVal) => {
       console.log('触发', newVal, oldVal);
     }
   );
   ```
2. **调用 `changeC3`**：
   ```javascript
   changeC3(); // 输出：触发 { c1: '自行车', c2: '学步车' } { c1: '奔驰', c2: '宝马' }
   ```

---

### 关键区别
| 场景             | `watch(person.car, ...)` | `watch(() => person.car, ...)` |
|------------------|--------------------------|--------------------------------|
| **监听引用替换**   | ❌ 不触发                | ✅ 触发                       |
| **监听内部属性变化** | ✅ 触发（默认深度监听）  | ❌ 不触发（需显式 `deep: true`） |

---

### 总结
- **直接传递 `person.car`**：仅绑定初始引用，无法感知替换操作。
- **使用函数返回 `() => person.car`**：动态追踪最新引用，支持替换操作。
- 如果需要同时监听内部属性变化，可添加 `{ deep: true }` 配置。
