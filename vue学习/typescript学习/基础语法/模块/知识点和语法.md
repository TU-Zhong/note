TypeScript 的模块化系统允许你将代码拆分为多个文件，并且通过 `export` 和 `import` 关键字来管理和使用这些模块。以下是一些详细的 TypeScript 模块化的知识点和例子：

### 1. **导出 (Export)**
`export` 关键字用于从模块中导出变量、函数、类、接口等，使其可以在其他模块中使用。

#### 1.1 导出单个变量或函数
```typescript
// math.ts
export function square(n: number): number {
  return n * n;
}
```

#### 1.2 导出多个变量或函数
```typescript
// math.ts
export function square(n: number): number {
  return n * n;
}

export function cube(n: number): number {
  return n * n * n;
}
```

#### 1.3 导出默认值
一个模块只能有一个默认导出。
```typescript
// math.ts
export default function square(n: number): number {
  return n * n;
}
```

### 2. **导入 (Import)**
`import` 关键字用于从其他模块中导入导出的变量、函数、类等。

#### 2.1 导入单个导出
```typescript
// main.ts
import { square } from "./math";
console.log(square(4)); // 输出 16
```

#### 2.2 导入多个导出
```typescript
// main.ts
import { square, cube } from "./math";
console.log(square(4)); // 输出 16
console.log(cube(3));   // 输出 27
```

#### 2.3 导入默认导出
```typescript
// main.ts
import square from "./math";
console.log(square(4)); // 输出 16
```

#### 2.4 导入所有导出
你可以使用 `* as` 来导入模块中的所有导出，并将其绑定到一个命名空间对象上。
```typescript
// main.ts
import * as math from "./math";
console.log(math.square(4)); // 输出 16
console.log(math.cube(3));   // 输出 27
```

### 3. **重命名导入和导出**
你可以使用 `as` 关键字在导入或导出时重命名。

#### 3.1 重命名导入
```typescript
// main.ts
import { square as sq } from "./math";
console.log(sq(4)); // 输出 16
```

#### 3.2 重命名导出
```typescript
// math.ts
function square(n: number): number {
  return n * n;
}

export { square as sq };
```

### 4. **重新导出**
你可以使用 `export ... from` 语法将一个模块的内容重新导出到另一个模块中。

```typescript
// mathUtils.ts
export { square } from "./math";
export { cube } from "./math";
```

### 5. **模块解析**
TypeScript 支持多种模块解析策略，包括 `classic` 和 `node`。默认情况下，TypeScript 使用 `node` 模块解析策略，这意味着它会按照 Node.js 的模块解析规则来查找模块。

### 6. **模块编译**
TypeScript 可以将模块编译为不同的模块系统，如 `CommonJS`、`AMD`、`ES6` 等。你可以在 `tsconfig.json` 中通过 `module` 选项来指定目标模块系统。

```json
{
  "compilerOptions": {
    "module": "commonjs"
  }
}
```

### 7. **模块路径别名**
你可以通过配置 `tsconfig.json` 中的 `paths` 选项来简化模块路径。

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@math/*": ["src/math/*"]
    }
  }
}
```

然后在代码中可以这样导入：
```typescript
import { square } from "@math/math";
```

### 8. **动态导入**
TypeScript 支持动态导入，允许你在运行时按需加载模块。

```typescript
async function loadMath() {
  const math = await import("./math");
  console.log(math.square(4)); // 输出 16
}

loadMath();
```

### 9. **模块和命名空间**
虽然 TypeScript 支持命名空间（`namespace`），但在现代 TypeScript 开发中，推荐使用模块化来组织代码，而不是命名空间。

### 10. **模块和类型**
TypeScript 模块不仅可以导出代码，还可以导出类型。

```typescript
// types.ts
export interface Point {
  x: number;
  y: number;
}

// main.ts
import { Point } from "./types";

const p: Point = { x: 1, y: 2 };
```

### 总结
TypeScript 的模块化系统非常强大，允许你将代码组织得更加清晰和可维护。通过 `export` 和 `import`，你可以轻松地在不同模块之间共享代码和类型。掌握这些知识点能够帮助你更好地构建大型 TypeScript 项目。
