

在 TypeScript 中，接口（`interface`）和类型别名（`type`）都是用来定义复杂类型的工具，但它们在使用场景、语法和特性上有显著区别。以下是它们的核心区别、语法和注意事项：

---
[[通俗解释接口和类型别名的区别]]
### **一、核心区别**

| **特性**      | **接口（`interface`）**                         | **类型别名（`type`）**           |
| ----------- | ------------------------------------------- | -------------------------- |
| **定义范围**    | 只能定义对象类型或函数类型                               | 可定义任意类型（对象、联合、交叉、元组、基本类型等） |
| **扩展性**     | 支持通过 `extends` 继承其他接口，或通过 `implements` 被类实现 | 通过交叉类型（`&`）扩展              |
| **声明合并**    | 同名接口会自动合并                                   | 同名类型别名会报错                  |
| **联合/交叉类型** | 不能直接定义联合类型，但可通过继承组合                         | 可直接定义联合类型（\|）、交叉类型（`&`）    |
| **元组**      | 不支持                                         | 支持定义元组类型                   |
| **类的实现**    | 类可以实现接口（`implements`）                       | 类不能实现类型别名                  |
| **性能**      | 更适合大型对象结构的扩展                                | 更适合组合简单或复杂类型               |

---

### **二、语法对比**

#### **1. 接口（`interface`）**
- **定义对象类型**：
  ```typescript
  interface User {
    name: string;
    age: number;
    greet(): void;
  }
  ```
- **继承其他接口**：
  ```typescript
  interface Admin extends User {
    isAdmin: true;
  }
  ```
- **声明合并**（自动合并同名接口）：
  ```typescript
  interface User { name: string; }
  interface User { age: number; }
  // 合并后：{ name: string; age: number; }
  ```

---

#### **2. 类型别名（`type`）**
- **定义任意类型**：
  ```typescript
  type ID = string | number; // 联合类型
  type Point = { x: number; y: number }; // 对象类型
  type Coordinates = [number, number]; // 元组类型
  ```
- **扩展其他类型**（交叉类型 `&`）：
  ```typescript
  type Admin = User & { isAdmin: true };
  ```

---

### **三、使用场景**

#### **优先使用接口的场景**
1. **定义对象或类的形状**：
   ```typescript
   interface Animal {
     name: string;
     move(): void;
   }
   
   class Dog implements Animal {
     name = "Buddy";
     move() { console.log("Running..."); }
   }
   ```
2. **需要声明合并**（如扩展第三方库类型）：
   ```typescript
   // 第一次定义
   interface Window {
     title: string;
   }
   
   // 第二次定义（自动合并）
   interface Window {
     width: number;
   }
   ```

#### **优先使用类型别名的场景**
1. **定义联合或交叉类型**：
   ```typescript
   type Result = { success: true; data: string } | { success: false; error: string };
   ```
2. **定义元组或复杂类型组合**：
   ```typescript
   type Matrix = [number, number, number][];
   ```
3. **简化重复类型**：
   ```typescript
   type Callback = (err: Error | null, data: string) => void;
   ```

---

### **四、注意事项**

#### **接口的注意事项**
1. **不能直接定义联合类型**：
   ```typescript
   interface A { prop: string }
   interface B { prop: number }
   type C = A | B; // ✅ 只能通过类型别名实现
   ```
2. **避免过度合并**：
   - 同名接口的自动合并可能导致意外行为，需谨慎使用。

#### **类型别名的注意事项**
1. **不支持声明合并**：
   ```typescript
   type User = { name: string };
   type User = { age: number }; // ❌ 报错：重复定义
   ```
2. **无法被类直接实现**：
   ```typescript
   type Animal = { name: string; move(): void };
   class Dog implements Animal { ... } // ❌ 报错
   ```

---

### **五、示例对比**

#### **扩展方式对比**
- **接口**（`extends`）：
  ```typescript
  interface Person {
    name: string;
  }
  
  interface Employee extends Person {
    salary: number;
  }
  ```
- **类型别名**（交叉类型 `&`）：
  ```typescript
  type Person = { name: string };
  type Employee = Person & { salary: number };
  ```

#### **联合类型示例**
- **仅类型别名支持**：
  ```typescript
  type Status = "success" | "error";
  type Response = { code: 200; data: string } | { code: 404; error: string };
  ```

---

### **总结**
- **接口**：适合定义对象的结构、类的契约，支持声明合并和继承。
- **类型别名**：适合定义复杂类型组合（联合、交叉、元组等），更灵活但无法合并。

根据需求选择合适的工具，通常推荐：
- **对象和类**：优先用 `interface`。
- **类型组合或复杂类型**：优先用 `type`。
