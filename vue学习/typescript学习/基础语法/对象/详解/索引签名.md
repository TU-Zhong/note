

以下是 TypeScript 中对象索引签名（Index Signatures）的详细知识点和示例：

---

### **1. 索引签名的作用**
索引签名允许你定义对象的动态属性名（键的类型）和对应的值类型，常用于处理未知键名的数据结构（如字典、动态配置等）。

---

### **2. 基本语法**
```typescript
interface Dictionary {
  [key: string]: number; // 字符串作为键，数值作为值
}

interface NumericKeyDictionary {
  [key: number]: string; // 数值作为键，字符串作为值
}
```

---

### **3. 核心规则**
1. **键的类型**：只能是 `string`、`number` 或 `symbol`。
2. **值的类型**：所有动态属性的值类型必须一致。
3. **显式属性**：如果对象有其他显式声明的属性，它们的类型必须与索引签名兼容。

---

### **4. 示例**

#### **示例 1：字符串索引签名**
定义以字符串为键、数值为值的对象：
```typescript
interface Scores {
  [subject: string]: number;
}

let studentScores: Scores = {
  math: 90,
  english: 85,
  // science: "A" // ❌ 错误：值必须为 number
};
```

#### **示例 2：数值索引签名**
定义以数值为键、字符串为值的对象（类似数组）：
```typescript
interface StringArray {
  [index: number]: string;
}

let colors: StringArray = ["Red", "Green", "Blue"];
console.log(colors[0]); // "Red"
```

#### **示例 3：混合显式属性和索引签名**
显式属性必须与索引签名兼容：
```typescript
interface User {
  name: string;        // ✅ 显式属性
  [key: string]: string | number; // 索引签名允许值为 string 或 number
}

let user: User = {
  name: "Alice",
  age: 25,            // ✅
  isAdmin: "false",   // ❌ 错误：值只能是 string 或 number（"false" 是 string，但此处可能逻辑错误）
};
```

#### **示例 4：联合类型的值**
允许值类型为联合类型：
```typescript
interface Data {
  [key: string]: string | number;
}

let data: Data = {
  id: 1,      // ✅
  name: "Alice", // ✅
};
```

---

### **5. 索引签名的注意事项**
#### **(1) 无法约束键的格式**
索引签名无法限制键的具体格式（如必须符合正则表达式），只能通过运行时检查。
```typescript
interface Config {
  [key: string]: string;
}

let config: Config = {
  apiUrl: "https://example.com",
  // 无法约束键必须是小写或符合其他格式！
};
```

#### **(2) 混合 `string` 和 `number` 索引**
如果同时定义 `string` 和 `number` 索引，`number` 索引的值类型必须是 `string` 索引的子类型。
```typescript
interface MixedDictionary {
  [key: string]: any; // 必须包含 number 索引的类型
  [index: number]: string; // ✅ string 是 any 的子类型
}

let dict: MixedDictionary = {
  0: "Hello", // ✅
  name: "Alice", // ✅
};
```

#### **(3) 无法使用联合类型作为索引类型**
键的类型不能是联合类型：
```typescript
interface ErrorExample {
  [key: string | number]: string; // ❌ 错误：索引类型必须为 string、number 或 symbol
}
```

---

### **6. 实际应用场景**
#### **(1) 动态配置对象**
```typescript
interface AppConfig {
  [key: string]: string | number | boolean;
}

let config: AppConfig = {
  port: 3000,
  env: "development",
  debug: true,
};
```

#### **(2) 字典映射**
```typescript
interface WordCount {
  [word: string]: number;
}

let counts: WordCount = {
  apple: 5,
  banana: 3,
};
```

#### **(3) 缓存数据**
```typescript
interface Cache {
  [key: string]: {
    data: any;
    expires: Date;
  };
}

let cache: Cache = {
  userData: {
    data: { name: "Alice" },
    expires: new Date("2025-12-31"),
  },
};
```

---

### **总结**
| **特性**              | **说明**                                      | **示例**                              |
|-----------------------|----------------------------------------------|---------------------------------------|
| 键的类型              | `string`、`number`、`symbol`                 | `[key: string]: number`               |
| 值的类型              | 必须统一或兼容显式属性                       | `[key: string]: string | number`       |
| 显式属性兼容性        | 显式属性需符合索引签名                       | `name: string` 与 `[key: string]: string` |
| 混合索引签名          | `number` 索引值必须是 `string` 索引的子类型   | `[key: string]: any; [index: number]: string` |

掌握索引签名后，可以更灵活地处理动态属性的对象结构！ 🚀
