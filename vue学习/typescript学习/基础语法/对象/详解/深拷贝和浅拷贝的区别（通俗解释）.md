

### 🌟 用“复印文件”来比喻深拷贝和浅拷贝 🌟

想象你有一份文件，里面有一页纸（对象A）和一个贴在上面的便利贴（嵌套对象）。现在你要复印这份文件：

---

### **1. 浅拷贝：只复印最外层**
- **操作**：你只复印了最外面那页纸，但便利贴还是原来的那张（只复制引用）。
- **结果**：
  - 修改复印件的文字：**不影响**原文件。
  - 修改复印件的便利贴：**会影响**原文件的便利贴（因为它们共用同一张便利贴）。
```typescript
// 浅拷贝示例
let original = { a: 1, b: { c: 2 } };
let copy = { ...original }; // 展开运算符（浅拷贝）

copy.a = 999; // 修改顶层属性 ✅ 不影响原对象
copy.b.c = 888; // 修改嵌套属性 ❌ 原对象也被修改！

console.log(original); // { a: 1, b: { c: 888 } }
```

---

### **2. 深拷贝：完整复印所有内容**
- **操作**：你把整份文件（包括便利贴）都重新复印一遍，完全独立。
- **结果**：
  - 修改复印件的文字：**不影响**原文件。
  - 修改复印件的便利贴：**不影响**原文件的便利贴。
```typescript
// 深拷贝示例（JSON.stringify 方式，但有局限）
let original = { a: 1, b: { c: 2 } };
let copy = JSON.parse(JSON.stringify(original)); // 深拷贝

copy.a = 999;
copy.b.c = 888;

console.log(original); // { a: 1, b: { c: 2 } } ✅ 原对象未被修改
```

---

### **一句话总结**
- **浅拷贝**：只复印表面，便利贴共享。
- **深拷贝**：全盘复印，连便利贴都重新写一份。

---

### **什么时候用哪种拷贝？**
- **浅拷贝**：当你不关心嵌套对象是否被共享时（比如简单配置）。
- **深拷贝**：当你需要完全独立的副本时（比如防止数据被意外修改）。
