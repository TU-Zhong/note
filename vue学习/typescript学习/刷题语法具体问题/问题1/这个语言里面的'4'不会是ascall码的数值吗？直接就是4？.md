

在 JavaScript/TypeScript 中，字符 `'4'` 的 **ASCII 码值** 是 `52`，但当你直接将 `'4'` 参与数值运算时，语言会自动将其转换为对应的 **数字 `4`**，而不是使用 ASCII 码值。这是由语言的隐式类型转换规则决定的。

---

### 关键点解析

#### 1. **字符的底层表示**
字符 `'4'` 在计算机中存储的确实是 ASCII 码值 `52`（或 Unicode 码点 `U+0034`）。这是字符的底层二进制表示形式。

#### 2. **隐式类型转换规则**
当字符参与算术运算时，JavaScript/TypeScript 会尝试将字符转换为 **数字类型的字面值**，而非直接使用其 ASCII 码值。例如：
```typescript
const char = '4';
const result = char - 2; // 等价于 4 - 2 = 2（不是 52 - 2 = 50）
```

#### 3. **与 ASCII 码的差异**
若想直接获取字符的 ASCII 码值，需使用 **`charCodeAt()`** 方法：
```typescript
const char = '4';
const asciiValue = char.charCodeAt(0); // 返回 52（ASCII 码值）
console.log(asciiValue - 2);          // 输出 50（52 - 2）
```

---

### 对比实验
| 场景                     | 代码示例               | 结果  | 原因                                      |
|--------------------------|------------------------|-------|------------------------------------------|
| 直接减法运算             | `'4' - 2`              | `2`   | 隐式转换为数字 `4`                       |
| 获取 ASCII 码值后运算    | `'4'.charCodeAt(0) - 2`| `50`  | 显式使用 ASCII 码值 `52`                 |
| 加法操作（字符串拼接）   | `'4' + 2`              | `'42'`| `+` 优先拼接字符串                       |

---

### 类型转换的底层逻辑
1. **字符串转为数字的规则**  
   JavaScript 会尝试解析字符串内容为数字：
   - 若字符串是纯数字组成（如 `'123'`）→ 转换为对应数字（`123`）。
   - 若字符串包含非数字字符（如 `'12a'`）→ 转换为 `NaN`。

2. **显式转换方法**  
   - `Number('4')` → `4`  
   - `parseInt('4')` → `4`  
   - `+'4'` → `4`  

---

### 为什么设计成这样？
- **开发者友好性**：语言希望尽量让 `'4'` 这样的字符串在数值运算中表现为直观的数字 `4`，而非其底层 ASCII 码值。
- **避免混淆**：若直接用 ASCII 码值参与运算，代码可读性会大幅降低（例如 `'A' - 1` 会变成 `64`，而非 `NaN`）。

---

### 总结
| 操作                | 结果       | 本质原因                              |
|---------------------|------------|--------------------------------------|
| `'4' - 4`           | `0`        | 字符 `'4'` 隐式转换为数字 `4`        |
| `'4'.charCodeAt(0)` | `52`       | 显式获取 ASCII 码值                  |
| `'4' + 4`           | `'44'`     | `+` 运算符优先拼接字符串             |

合理利用 `charCodeAt()` 或 `codePointAt()` 可直接操作字符的编码值！ 🚀
